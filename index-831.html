如何用空格填充 Python 字符串？

Arya

2023-02-17 17:19

我想用空格填充一个字符串。我知道以下适用于零：

>>> print  "'%06d'"%4
'000004'

但是当我想要这个时我该怎么办？：

'hi    '

当然我可以测量字符串长度并执行str+" "*leftover，但我想要最短的方法。



解答

你可以这样做 str.ljust(width[, fillchar]) ：

在长度为宽度 的字符串中返回左对齐的字符串。填充是使用指定的 fillchar 完成的（默认为空格）。 如果宽度 小于 ，则返回原始字符串len(s)。

>>> 'hi'.ljust(10)
'hi        '


更多建议

对于即使在格式化复杂字符串时也能使用的灵活方法，您可能应该使用string-formatting mini-language，

使用任一f 弦

>>> f'{"Hi": <16} StackOverflow!'  # Python >= 3.6
'Hi               StackOverflow!'

或str.format()方法

>>> '{0: <16} StackOverflow!'.format('Hi')  # Python >=2.6
'Hi               StackOverflow!'

在 C 和 C++ 中都有效的代码在用每种语言编译时是否会产生不同的行为？

Arya

2023-02-17 17:18

C 和 C++ 有很多差异，并非所有有效的 C 代码都是有效的 C++ 代码。
（“有效”是指具有定义行为的标准代码，即不是特定于实现/未定义/等。）

在使用每种语言的标准编译器编译时，一段在 C 和 C++ 中均有效的代码是否会产生 不同的行为？

为了使它成为一个合理/有用的比较（我正在尝试学习一些实际有用的东西，而不是试图在问题中找到明显的漏洞），让我们假设：

  • 与预处理器无关（这意味着没有 hack with #ifdef __cplusplus，pragmas 等）
  • 任何实现定义的东西在两种语言中都是相同的（例如数字限制等）
  • 我们正在比较每个标准的合理最新版本（例如，C++98 和 C90 或更高版本）
    如果版本很重要，那么请提及每个版本的哪些版本产生不同的行为。



解答

以下在 C 和 C++ 中有效，将（很可能）在iC 和 C++ 中产生不同的值：

int i = sizeof('a');

有关差异的说明，请参阅C/C++ 中的字符大小 ('a') 。

这篇文章中的另一个：

#include <stdio.h>

int  sz = 80;

int main(void)
{
    struct sz { char c; };

    int val = sizeof(sz);      // sizeof(int) in C,
                               // sizeof(struct sz) in C++
    printf("%d\n", val);
    return 0;
}


更多建议

对于 C++ 与 C90，至少有一种方法可以获得未定义实现的不同行为。C90 没有单行注释。只要小心一点，我们就可以使用它来创建一个在 C90 和 C++ 中具有完全不同结果的表达式。

int a = 10 //* comment */ 2 
        + 3;

在 C++ 中，从行尾//到行尾的所有内容都是注释，因此可以这样计算：

int a = 10 + 3;

由于 C90 没有单行注释，因此只有 is/* comment */注释。first/和 the2都是初始化的部分，所以结果是：

int a = 10 / 2 + 3;

因此，正确的 C++ 编译器会给出 13，但严格正确的 C90 编译器会给出 8。当然，我只是在这里选择了任意数字——您可以使用您认为合适的其他数字。

我试图为事件分配热键：

  • 向前
  • 转发历史
  • 下一页

但它不按我想要的方式工作。



解答

  • CTRL+ E（用于编辑列表）
  • CTRL+ F6（用于通过列表切换到下一个编辑器）
    您可以为“下一个编辑器”键分配另一个快捷方式。

是两个官方快捷方式，但它们都涉及要显示的列表。

CTRL+ Page Up/ CTRL+Page Down可以在不显示列表的情况下循环编辑器。

最后一组快捷键的有趣之处在于：
它们没有列在要定义的 Eclipse Keys 快捷键中。
它们可能继承自多页编辑器，使它们成为 特定于操作系统的快捷方式 。
