在 Objective-C 中，如何测试对象类型？

Arya

2023-02-18 02:55

我需要测试对象的类型是NSString还是UIImageView。我怎样才能做到这一点？是否有某种类型的“isoftype”方法？



解答

如果您的对象是myObject，并且您想测试它是否是NSString，代码将是：

[myObject isKindOfClass:[NSString class]]

同样，如果你想测试myObject一个UIImageView：

[myObject isKindOfClass:[UIImageView class]]


更多建议

你可能会使用

- (BOOL)isKindOfClass:(Class)aClass

这是一个方法NSObject。

有关更多信息，请查看NSObject文档。

这就是你如何使用它。

BOOL test = [self isKindOfClass:[SomeClass class]];

你也可以尝试做这样的事情

for(id element in myArray)
{
    NSLog(@"=======================================");
    NSLog(@"Is of type: %@", [element className]);
    NSLog(@"Is of type NSString?: %@", ([[element className] isMemberOfClass:[NSString class]])? @"Yes" : @"No");
    NSLog(@"Is a kind of NSString: %@", ([[element classForCoder] isSubclassOfClass:[NSString class]])? @"Yes" : @"No");    
}

WOFF 字体的 Mime 类型？

Arya

2023-02-18 02:54

WOFF 字体应该用作什么 mime 类型？

我提供 truetype (ttf) 字体和font/truetypeopentype (otf) 字体font/opentype，但我找不到 WOFF 字体的正确格式。

我已经尝试过font/woff、font/webopen和font/webopentype，但 Chrome 仍然抱怨：

“资源解释为字体，但使用 MIME 类型应用程序/八位字节流传输。”

有人知道吗？



解答

更新自 Keith Shaw于 2017 年 6 月 22 日发表的评论：

截至 2017 年 2 月，RFC8081是提议的标准。它定义了字体的顶级媒体类型，因此 WOFF 和 WOFF2 的标准媒体类型如下：

font/woff

font/woff2

2011 年 1 月宣布与此同时 Chromium 将承认

application/x-font-woff

作为 WOFF 的 MIME 类型。我知道这个变化现在在 Chrome 测试版中，如果还不稳定，应该不会太远。


更多建议

对我来说，下一个一直在 .htaccess 文件中工作。

AddType font/ttf .ttf
AddType font/eot .eot
AddType font/otf .otf
AddType font/woff .woff
AddType font/woff2 .woff2

Arya

因此，可以尝试获取以下 JSON 对象：

$ curl -i -X GET http://echo.jsontest.com/key/value/anotherKey/anotherValue
HTTP/1.1 200 OK

Content-Type: application/json; charset=ISO-8859-1
Date: Wed, 30 Oct 2013 22:19:10 GMT
Server: Google Frontend
Cache-Control: private

{
   "anotherKey": "anotherValue",
   "key": "value"
}
$

有没有办法在使用节点或快递的服务器响应中生成完全相同的正文？显然，可以设置标头并指示响应的内容类型将是“application/json”，但是有不同的方式来写入/发送对象。我经常看到的一种是使用以下形式的命令：

response.write(JSON.stringify(anObject));

但是，这有两点可以争论，就好像它们是“问题”一样：

  • 我们正在发送一个字符串。
  • 而且，最后没有换行符。

另一个想法是使用命令：

response.send(anObject);

这似乎是基于 curl 的输出发送一个 JSON 对象，类似于上面的第一个示例。但是，当在终端上再次使用 curl 时，正文末尾没有换行符。那么，如何使用 node 或 node/express 在末尾附加一个新行字符来写下这样的东西呢？



解答

该响应也是一个字符串，如果您想发送经过美化的响应，出于某些尴尬的原因，您可以使用类似JSON.stringify(anObject, null, 3)

Content-Type将标头也设置为也很重要application/json。

var http = require('http');

var app = http.createServer(function(req,res){
    res.setHeader('Content-Type', 'application/json');
    res.end(JSON.stringify({ a: 1 }));
});
app.listen(3000);

// > {"a":1}

美化：

var http = require('http');

var app = http.createServer(function(req,res){
    res.setHeader('Content-Type', 'application/json');
    res.end(JSON.stringify({ a: 1 }, null, 3));
});
app.listen(3000);

// >  {
// >     "a": 1
// >  }

我不确定你为什么要用换行符终止它，但你可以做到JSON.stringify(...) + '\n'这一点。

表达

在 express 中，您可以通过更改选项来做到这一点。

'json replacer'JSON replacer回调，默认为null

'json spaces'用于格式化的 JSON 响应空间，开发中默认为 2，生产中为 0

实际上不建议设置为 40

app.set('json spaces', 40);

然后你可以用一些 json 来回应。

res.json({ a: 1 });

它将使用'json spaces' 配置来美化它。


更多建议

由于 Express.js 3x 响应对象有一个 json() 方法，它可以为您正确设置所有标头并以 JSON 格式返回响应。

例子：

res.json({"foo": "bar"});
