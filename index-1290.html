理解 dict.copy() - 浅还是深？

Arya

2023-02-18 06:04

在阅读 的文档时dict.copy()，它说它制作了字典的浅表副本。我正在关注的书（Beazley 的 Python 参考）也是如此，它说：

m.copy() 方法对映射对象中包含的项目进行浅表复制，并将它们放入新的映射对象中。

考虑一下：

>>> original = dict(a=1, b=2)
>>> new = original.copy()
>>> new.update({'c': 3})
>>> original
{'a': 1, 'b': 2}
>>> new
{'a': 1, 'c': 3, 'b': 2}

所以我假设这也会更新（并添加'c'：3）的值，original因为我正在做一个浅拷贝。就像你为列表做的那样：

>>> original = [1, 2, 3]
>>> new = original
>>> new.append(4)
>>> new, original
([1, 2, 3, 4], [1, 2, 3, 4])

这按预期工作。

由于两者都是浅拷贝，为什么它dict.copy()不能像我预期的那样工作？或者我对浅拷贝和深拷贝的理解有缺陷？



解答

通过“浅复制”，它意味着字典的 内容 不是按值复制，而只是创建一个新的引用。

>>> a = {1: [1,2,3]}
>>> b = a.copy()
>>> a, b
({1: [1, 2, 3]}, {1: [1, 2, 3]})
>>> a[1].append(4)
>>> a, b
({1: [1, 2, 3, 4]}, {1: [1, 2, 3, 4]})

相反，深拷贝将按值复制所有内容。

>>> import copy
>>> c = copy.deepcopy(a)
>>> a, c
({1: [1, 2, 3, 4]}, {1: [1, 2, 3, 4]})
>>> a[1].append(5)
>>> a, c
({1: [1, 2, 3, 4, 5]}, {1: [1, 2, 3, 4]})

所以：

  1. b = a：引用赋值，Makea和b指向同一个对象。

  1. b = a.copy(): 浅拷贝，a会b成为两个孤立的对象，但它们的内容仍然共享同一个引用

  1. b = copy.deepcopy(a): 深度复制，aandb的结构和内容变得完全隔离。


更多建议

举个例子：

original = dict(a=1, b=2, c=dict(d=4, e=5))
new = original.copy()

现在让我们更改“浅”（第一）级别的值：

new['a'] = 10
# new = {'a': 10, 'b': 2, 'c': {'d': 4, 'e': 5}}
# original = {'a': 1, 'b': 2, 'c': {'d': 4, 'e': 5}}
# no change in original, since ['a'] is an immutable integer

现在让我们更深一层地改变一个值：

new['c']['d'] = 40
# new = {'a': 10, 'b': 2, 'c': {'d': 40, 'e': 5}}
# original = {'a': 1, 'b': 2, 'c': {'d': 40, 'e': 5}}
# new['c'] points to the same original['d'] mutable dictionary, so it will be changed

Java 中的 native 关键字有什么用？

Arya

2023-02-18 06:04

在玩这个谜题（这是一个 Java 关键字问答游戏）时，我遇到了native关键字。

Java中的native关键字是干什么用的？



解答

关键字native应用于方法以指示该方法是使用 JNI（Java 本机接口）在本机代码中实现的。


更多建议

它标志着一种方法，它将以其他语言而不是 Java 实现。它与 JNI（Java 本机接口）一起工作。

过去使用本机方法来编写性能关键部分，但随着 Java 变得越来越快，这种方法现在不那么常见了。当前需要使用本地方法

  • 您需要从用其他语言编写的 Java 中调用一个库。

  • 您需要访问只能从其他语言（通常是 C）访问的系统或硬件资源。实际上，许多与真实计算机交互的系统功能（例如磁盘和网络 IO）只能这样做，因为它们调用了本机代码。

另见Java 本机接口规范

如何获取 JavaScript 对象的所有属性值（不知道键）？

Arya

2023-02-18 06:03

如果有一个 JavaScript 对象：

var objects={...};

假设，它有 50 多个属性，在不知道属性名称（也就是不知道“键”）的情况下如何循环获取每个属性值？



解答

通过使用一个简单的for..in循环：

for(var key in objects) {
    var value = objects[key];
}


更多建议

通过使用一个简单的for..in循环：

    var value = objects[key];
