如何为项目中的单个文件禁用 ARC？

Arya

2023-02-17 02:59

我在我的项目中成功地使用了 ARC。然而，我遇到了一些文件（例如，在单元测试和模拟对象中），其中 ARC 的规则现在有点脆弱。我记得听说有一种方法可以在每个文件的基础上禁用 ARC，但我一直找不到这个选项。

这可能吗？如何在每个文件的基础上禁用 ARC？



解答

通过为这些文件添加编译器标志，可以为单个文件禁用 ARC -fno-objc-arc。

您在 Targets -> Build Phases -> Compile Sources 中添加编译器标志。 您必须双击Compiler Flags 下行的右列。您还可以通过按住 cmd 按钮选择文件，然后按 enter 调出标志编辑框，将其添加到多个文件中。（请注意，编辑多个文件将覆盖它可能已经拥有的任何标志。）

我创建了一个示例项目，其中包含一个示例： https: //github.com/jaminguy/NoArc

有关详细信息，请参阅此答案：禁用某些文件的自动引用计数


更多建议

*在 *多个文件上 禁用 ARC ： ****

  1. 在 Xcode 中的 Target/Build Phases/Compile Sources 选择所需的文件
  2. 按回车
  3. 输入 -fno-objc-arc
  4. 按 Enter 或 完成

.gitignore 排除文件夹但包含特定子文件夹

Arya

2023-02-17 02:58

我有application/我添加到.gitignore. 文件夹里面application/是文件夹application/language/gr。我怎样才能包含这个文件夹？

我试过这个

application/
!application/language/gr/



解答

如果您排除application/，那么它下面的所有内容都将始终被排除（即使某些后来的否定排除模式（“unignore”）可能与 下的内容匹配application/）。

为了做你想做的事，你必须“忽略”你想“忽略”的任何东西的每个父目录。通常你最终会成对地为这种情况编写规则：忽略目录中的所有内容，但不是某些特定的子目录。

# you can skip this first one if it is not already excluded by prior patterns
!application/

application/*
!application/language/

application/language/*
!application/language/gr/

注意
尾随/*很重要：

  • 该模式dir/排除了一个名为的目录dir和（隐含地）它下面的所有内容。
    使用dir/，Git 永远不会查看 下的任何内容dir，因此永远不会将任何“取消排除”模式应用于 . 下的任何内容dir。

  • 模式本身dir/*什么也没说dir；它只是排除了dir. 有了dir/*，Git 将处理 的直接内容dir，让其他模式有机会“取消排除”某些内容 ( !dir/sub/)。


更多建议

来自Karsten Blees (kblees)的针对 Git 1.9/2.0（2014 年第一季度）的提交 59856de澄清了这种情况：

gitignore.txt：阐明排除目录的递归性质

一个可选的前缀“ !”，它否定模式；先前模式排除的任何匹配文件将再次包含在内。

如果排除了该文件的父目录，则无法重新包含该文件。 ( *)
( *: 除非在 git 2.8+ 中满足某些条件，见下文）
出于性能原因，Git 不会列出排除的目录，因此包含文件的任何模式都无效，无论它们在哪里定义。

对于以文字“”开头的模式，例如，“”，请\在第一个“”前面放置一个反斜杠（ “ ”）。!``!``\!important!.txt

排除除特定目录以外的所有内容的示例foo/bar（请注意/*- 没有斜线，通配符也会排除 内的所有内容foo/bar）：

 --------------------------------------------------------------
     $ cat .gitignore
     # exclude everything except directory foo/bar
     /*
     !/foo
     /foo/*
     !/foo/bar
 --------------------------------------------------------------

在你的情况下：

application/*
!application/**/
application/language/*
!application/language/**/
!application/language/gr/**

您必须首先将文件夹 列入白名单，然后才能将给定文件夹中的文件列入白名单。

2016 年 2 月/3 月更新：

请注意，对于 git 2.9.x/2.10（2016 年年中？），如果文件的父目录被排除，如果路径中没有通配符，则可能会重新包含该文件re-included 。

Nguyễn Thái Ngọc Duy ( pclouds)正在尝试添加此功能：

  • git v2.7.0 的提交 506d8f1 ，在提交 76b620d git v2.8.0-rc0中还原
  • 提交 5e57f9c git v2.8.0-rc0，...在提交 5cee3493 git 2.8.0中恢复（！） 。

所以对于 git 2.9+，这实际上可以工作，但最终被恢复：

application/
!application/language/gr/

如何删除重复的行？

Arya

2023-02-17 02:58

我需要从一个相当大的 SQL Server 表（即超过 300,000 行）中删除重复的行。

当然，由于标识字段的存在，行不会完全重复RowID。

我的表格

RowID int not null identity(1,1) primary key,
Col1 varchar(20) not null,
Col2 varchar(2048) not null,
Col3 tinyint not null

我怎样才能做到这一点？



解答

假设没有空值，您GROUP BY是唯一列，而SELECTRowIdMIN (or MAX)作为要保留的行。然后，删除所有没有行 ID 的内容：

DELETE FROM MyTable
LEFT OUTER JOIN (
   SELECT MIN(RowId) as RowId, Col1, Col2, Col3 
   FROM MyTable 
   GROUP BY Col1, Col2, Col3
) as KeepRows ON
   MyTable.RowId = KeepRows.RowId
WHERE
   KeepRows.RowId IS NULL

如果您有 GUID 而不是整数，则可以替换

MIN(RowId)

和

CONVERT(uniqueidentifier, MIN(CONVERT(char(36), MyGuidColumn)))


更多建议

另一种可能的方法是

;

--Ensure that any immediately preceding statement is terminated with a semicolon above
WITH cte
     AS (SELECT ROW_NUMBER() OVER (PARTITION BY Col1, Col2, Col3 
                                       ORDER BY ( SELECT 0)) RN
         FROM   #MyTable)
DELETE FROM cte
WHERE  RN > 1;

我ORDER BY (SELECT 0)在上面使用，因为在平局的情况下保留哪一行是任意的。

为了保留最新的，RowID例如你可以使用ORDER BY RowID DESC

执行计划

这个执行计划通常比接受的答案中的执行计划更简单、更有效，因为它不需要自连接。

然而，情况并非总是如此。解决方案可能更受欢迎的一个地方是哈希聚合优先于流聚合的GROUP BY情况。

该ROW_NUMBER解决方案将始终提供几乎相同的计划，而GROUP BY策略则更加灵活。

可能有利于散列聚合方法的因素是

  • 分区列上没有有用的索引
  • 相对较少的组，每个组中的重复项相对较多

在第二种情况的极端版本​​中（如果每个组中有很多重复项的组很少），也可以考虑简单地插入行以保留到新表中，然后 -ing 原始表并将它们复制回来以最小化日志记录，而不是删除一个TRUNCATE表行的比例非常高。
