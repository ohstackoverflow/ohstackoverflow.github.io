jQuery 获取 select onChange 的值

Arya

2023-02-17 08:34

$(this).val();我的印象是我可以通过这样做并将onchange参数应用于选择字段来获得选择输入的值。

它似乎只有在我引用 ID 时才有效。

我如何使用它来做到这一点。



解答

尝试这个-

$('select').on('change', function() {
  alert( this.value );
});


<script src="https://ajax.googleapis.com/ajax/libs/jquery/2.1.1/jquery.min.js"></script>

<select>
    <option value="1">One</option>
    <option value="2">Two</option>
</select>

您还可以参考 onchange 事件-

function getval(sel)
{
    alert(sel.value);
}


<select onchange="getval(this);">
    <option value="1">One</option>
    <option value="2">Two</option>
</select>


更多建议

这对我有帮助。

对于选择：

$('select_tags').on('change', function() {
    alert( $(this).find(":selected").val() );
});

对于单选/复选框：

$('radio_tags').on('change', function() {
    alert( $(this).find(":checked").val() );
});

如何强制 gradle 重新下载依赖项？

Arya

2023-02-17 08:34

我如何告诉 gradle 从存储库重新下载依赖项？



解答

通常，您可以使用命令行选项--refresh- dependencies刷新缓存中的依赖项。您还可以删除缓存文件下~/.gradle/caches。在下一次构建中，Gradle 将尝试再次下载它们。

您的具体用例是什么？你使用动态依赖版本还是SNAPSHOT版本？

在 Unix 系统上，您可以删除 Gradle 使用以下命令下载的所有现有工件（工件和元数据）：

rm -rf $HOME/.gradle/caches/

请注意 --refresh-dependencies 不会总是重新下载每个工件；如果它们与存储库中存在的内容匹配，它将使用现有副本。从 Gradle 用户指南，刷新依赖项：

--refresh-dependencies 选项告诉 Gradle 忽略已解析模块和工件的所有缓存条目。将对所有配置的存储库执行新的解析，重新计算动态版本，刷新模块并下载工件。但是，在可能的情况下，Gradle 会在再次下载之前检查之前下载的工件是否有效。这是通过将存储库中发布的 SHA1 值与现有下载工件的 SHA1 值进行比较来完成的。

[...]

认为使用 --refresh-dependencies 会强制下载依赖项是一种常见的误解。事实并非如此：Gradle 只会执行刷新动态依赖项所严格要求的操作。这可能涉及下载新列表或元数据文件，甚至工件，但如果没有任何变化，影响很小。


更多建议

如果您使用的是最新版本的 Gradle，则可以使用 --refresh-dependencies 选项。

./gradlew build --refresh-dependencies

你可以参考Gradle 手册。

--refresh-dependencies 选项告诉 Gradle 忽略已解析模块和工件的所有缓存条目。 将对所有配置的存储库执行新的解析，重新计算动态版本，刷新模块并下载工件。

Java 如何检查一个文件是否存在？

Arya

2023-02-17 08:33

在用Java （相当于 Perl 的 ）打开文件进行阅读之前，如何检查文件是否存在-e $filename？

SO 上唯一类似的问题涉及写入文件，因此使用FileWriter此处显然不适用的方式进行了回答。

如果可能的话，我更喜欢真正的 API 调用返回 true/false 而不是一些“调用 API 以打开文件并在它抛出异常时捕获你检查文本中的‘无文件’”，但我可以忍受后者。



解答

使用java.io.File：

File f = new File(filePathString);
if(f.exists() && !f.isDirectory()) { 
    // do something
}


更多建议

我建议使用isFile()而不是exists(). 大多数时候，您不仅要检查路径是否指向文件，还要检查它是否存在。请记住，exists()如果您的路径指向一个目录，它将返回 true。

new File("path/to/file.txt").isFile();

new File("C:/").exists()将返回 true 但不允许您将其作为文件打开和读取。
