C++代码：

#include <iostream>
#include <time.h>

using namespace std;

int main() {
    string input_line;
    long line_count = 0;
    time_t start = time(NULL);
    int sec;
    int lps;

    while (cin) {
        getline(cin, input_line);
        if (!cin.eof())
            line_count++;
    };

    sec = (int) time(NULL) - start;
    cerr << "Read " << line_count << " lines in " << sec << " seconds.";
    if (sec > 0) {
        lps = line_count / sec;
        cerr << " LPS: " << lps << endl;
    } else
        cerr << endl;
    return 0;
}

// Compiled with:
// g++ -O3 -o readline_test_cpp foo.cpp

执行 每秒行数

解答

tl;dr：由于 C++ 中不同的默认设置需要更多的系统调用。

默认情况下，cin与 stdio 同步，这会导致它避免任何输入缓冲。如果将它添加到主程序的顶部，您应该会看到更好的性能：

std::ios_base::sync_with_stdio(false);

通常，当缓冲输入流时，不是一次读取一个字符，而是以更大的块读取流。这减少了通常相对昂贵的系统调用的数量。然而，由于FILE*basedstdio和iostreams通常有单独的实现，因此有单独的缓冲区，如果两者一起使用，这可能会导致问题。例如：

int myvalue1;
cin >> myvalue1;
int myvalue2;
scanf("%d",&myvalue2);

如果读取的输入比cin实际需要的多，那么第二个整数值将无法用于函数scanf，它有自己的独立缓冲区。这会导致意想不到的结果。

为避免这种情况，默认情况下，流与stdio. 实现此目的的一种常见方法是根据cin需要使用函数一次读取每个字符stdio。不幸的是，这会带来很多开销。对于少量输入，这不是什么大问题，但是当您读取数百万行时，性能损失就很明显了。

幸运的是，库设计者决定如果您知道自己在做什么，您也应该能够禁用此功能以获得改进的性能，因此他们提供了该方法sync_with_stdio。从这个链接（重点添加）：

如果关闭同步，则允许 C++ 标准流独立缓冲其 I/O， 这在某些情况下可能会快得多 。

如何格式化 Microsoft JSON 日期？

Arya

2023-02-16 22:30

我正在使用jQuery尝试Ajax 。我正在将我的数据放到我的页面上，但我在处理为 Date 数据类型返回的 JSON 数据时遇到了一些问题。基本上，我得到一个看起来像这样的字符串：

/Date(1224043200000)/

来自完全不熟悉 JSON 的人 - 如何将其格式化为短日期格式？这应该在 jQuery 代码中的某个地方处理吗？我试过jQuery.UI.datepicker使用插件$.datepicker.formatDate()但没有成功。

仅供参考：这是我使用此处答案的组合提出的解决方案：

function getMismatch(id) {
  $.getJSON("Main.aspx?Callback=GetMismatch",
    { MismatchId: id },

    function (result) {
      $("#AuthMerchId").text(result.AuthorizationMerchantId);
      $("#SttlMerchId").text(result.SettlementMerchantId);
      $("#CreateDate").text(formatJSONDate(Date(result.AppendDts)));
      $("#ExpireDate").text(formatJSONDate(Date(result.ExpiresDts)));
      $("#LastUpdate").text(formatJSONDate(Date(result.LastUpdateDts)));
      $("#LastUpdatedBy").text(result.LastUpdateNt);
      $("#ProcessIn").text(result.ProcessIn);
    }
  );

  return false;
}

function formatJSONDate(jsonDate) {
  var newDate = dateFormat(jsonDate, "mm/dd/yyyy");
  return newDate;
}

该解决方案从回调方法中获取了我的对象，并使用日期格式库在页面上正确显示了日期。



解答

eval()没有必要。这将正常工作：

var date = new Date(parseInt(jsonDate.substr(6)));

函数substr()取出/Date(部分，parseInt()函数得到整数，)/最后忽略。结果数字被传递到Date构造函数中。

我故意遗漏了基数（ 的第二个参数parseInt）；请参阅下面的评论。

此外，我完全同意Rory 的评论：ISO-8601 日期优于这种旧格式 - 因此这种格式通常不应用于新开发。

对于 ISO-8601 格式的 JSON 日期，只需将字符串传递给Date构造函数：

var date = new Date(jsonDate); //no ugly parsing needed; full timezone support


更多建议

您可以使用它从 JSON 中获取日期：

var date = eval(jsonDate.replace(/\/Date\((\d+)\)\//gi, "new Date($1)"));

然后您可以使用JavaScript 日期格式脚本（压缩和 gzip 后为 1.2 KB）根据需要显示它。
