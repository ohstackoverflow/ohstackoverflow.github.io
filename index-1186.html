merge --squash 和 rebase 有什么区别？

Arya

2023-02-18 03:30

我试图了解壁球和变基之间的区别。据我了解，一个人在做变基时会进行挤压。



解答

和git merge --squash都git rebase --interactive可以产生“压缩的”提交。但它们有不同的用途。

  • git merge --squash abranch

将在目标分支上生成压缩提交，而不标记任何合并关系。（注意：它不会立即产生提交：您需要一个额外的git commit -m "squash branch"）

如果您想完全丢弃源分支，这很有用（从SO 问题中获取的架构）：

git checkout stable

          X               stable
         /
a---b---c---d---e---f---g tmp

到：

git merge --squash tmp
git commit -m "squash tmp"


# In the following graph, G is c--d--e--f--g squashed together

          X-------------G stable
         /
a---b---c---d---e---f---g tmp

然后删除tmp分支。

注意：git merge有一个--commit选项，但不能与 一起使用--squash。 永远 不可能一起使用--commitand --squash。自 Git 2.22.1（2019 年第 3 季度）起，这种不兼容性变得明确：

请参阅Vishal Verma ( )的提交 1d14d0c（2019 年 5 月 24 日）。（由Junio C Hamano合并——提交33f2790，2019年 7 月 25 日）reloadbrain``gitster

merge:--commit拒绝--squash

以前，在--squash提供时，' option_commit' 被静默删除。对于试图显式使用 squash 的无提交行为的用户来说，这可能会让他们感到惊讶--commit。

git/git builtin/merge.c#cmd_merge()现在包括：

if (option_commit > 0)
    die(_("You cannot combine --squash with --commit."));
  • git rebase --interactive

在一个新的基础上重播你的部分或全部提交，允许你压缩（或最近“修复”，参见这个SO 问题），直接进入：

git checkout tmp
git rebase -i stable

   stable
      X----------------G tmp
     /
a---b

如果您选择压缩所有提交tmp（但是，相反merge --squash，您可以选择重播一些，并压缩其他）。

所以区别是：

  • squash不触及您的源分支（tmp此处）并在您想要的位置创建单个提交。
  • rebase允许您 继续使用相同的源分支 （仍然tmp）：
    • 新基地
    • 更干净的历史

`require': 没有要加载的文件 -- mkmf (LoadError)

Arya

2023-02-18 03:30

我试图使用 ruby​​1.9.1 在 Ubuntu Natty Narwhal 11.04 上安装 rails。

我使用apt-get install ruby1.9.1-full包含开发包的 ruby​​ 安装了 ruby​​。我用谷歌搜索了这个错误，所有人都建议我安装我已经拥有的 1.9.1-dev。

Building native extensions.  This could take a while...
ERROR:  Error installing rails:
    ERROR: Failed to build gem native extension.

        /usr/bin/ruby1.8 extconf.rb
extconf.rb:36:in `require': no such file to load -- mkmf (LoadError)
    from extconf.rb:36


Gem files will remain installed in /usr/lib/ruby/gems/1.8/gems/bcrypt-ruby-3.0.1 for inspection.
Results logged to /usr/lib/ruby/gems/1.8/gems/bcrypt-ruby-3.0.1/ext/mri/gem_make.out



解答

在搜索了一些解决方案后，发现-dev需要这个包，而不仅仅是ruby1.8. 所以如果你有ruby1.9.1做

sudo apt-get install ruby1.9.1-dev

或者安装通用的 ruby​​ 版本，使用（根据@lamplightdev 评论）：

sudo apt-get install ruby-dev

应该修复它。

尝试locate mkmf查看该文件是否确实存在。


更多建议

这是对我有用的答案。在上面的评论中，但作为 ubuntu 12.04 ruby​​ 1.8.7 的答案，它应该得到应有的位置

sudo apt-get install ruby-dev
# if above doesnt work make sure you have build essential
sudo apt-get install build-essential

命令行工具不工作 - OS X El Capitan、Sierra、High Sierra、Mojave

Arya

2023-02-18 03:29

git status我刚刚从 Yosemite 升级到 El Capitan（并复制了从 El Capitan 升级到 Sierra 的问题），当我尝试在终端内输入例如，我收到以下错误：

xcrun: error: invalid active developer path
 (/Library/Developer/CommandLineTools), missing xcrun at:
 /Library/Developer/CommandLineTools/usr/bin/xcrun

我没有安装 Xcode，从来没有。有人有解决办法吗？



解答

在这里找到问题的解决方法。

xcode-select --install

这将提示安装必要的命令行工具。


更多建议

如果您对命令有疑问xcode-select --install；例如，我一直遇到网络问题超时，然后尝试在developer.apple.com/downloads（命令行工具 OS X 10.11）为 Xcode 7.1下载 dmg
