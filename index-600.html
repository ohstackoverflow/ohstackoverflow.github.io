SQL 多列排序

Arya

2023-02-17 11:09

如何按 SQL 中的多列和不同方向进行排序。column1将按降序和column2升序排序。



解答

ORDER BY column1 DESC, column2

这column1首先按（降序）对所有内容进行排序，然后在两行或更多行的字段相等column2时按（升序，这是默认值）排序。column1


更多建议

其他答案缺少一个具体的例子，所以这里是：

给定以下 人员 表：

 FirstName |  LastName   |  YearOfBirth
----------------------------------------
  Thomas   | Alva Edison |   1847
  Benjamin | Franklin    |   1706

  Thomas   | Jefferson   |   1826

SELECT * FROM People ORDER BY FirstName DESC, YearOfBirth ASC

“对等方重置连接”是什么意思？

Arya

2023-02-17 11:08

TCP 连接上的“对等方重置连接”错误是什么意思？它是致命错误还是只是通知或与网络故障有关？



解答

这是致命的。远程服务器向您发送了一个 RST 数据包，表示立即断开连接，而不是通常的握手。这绕过了正常的半关闭状态转换。我喜欢这个描述：

“由对等方重置连接”是 TCP/IP 等同于将电话挂回挂机。这比仅仅不回复、留下一个悬而未决更有礼貌。但这不是真正有礼貌的 TCP/IP 对话者所期望的 FIN-ACK。


更多建议

这意味着已收到 TCP RST，现在连接已关闭。当数据包从您的连接端发送但另一端无法识别该连接时，就会发生这种情况；它将发回一个设置了 RST 位的数据包，以强制关闭连接。

如果另一端崩溃然后恢复，或者如果它close()在传输中有来自您的数据时调用套接字，则可能会发生这种情况，这表明您之前发送的某些数据可能尚未收到。

这是否是错误取决于您；如果您发送的信息只是为了远程客户端的利益，那么任何最终数据可能已经丢失可能无关紧要。但是，您应该关闭套接字并释放与连接关联的任何其他资源。

window.onload 与 document.onload

Arya

2023-02-17 11:08

哪个更广泛支持：window.onload或document.onload？



解答

他们什么时候开火？

window.onload

  • 默认情况下，它会在整个页面加载时触发， 包括 其内容（图像、CSS、脚本等）。

在某些浏览器中，它现在接管了 DOM 的角色document.onload并在 DOM 准备就绪时触发。

document.onload

  • 它在 DOM 准备就绪时调用，这可以在图像和其他外部内容加载 之前。

他们的支持情况如何？

window.onload似乎得到了最广泛的支持。事实上，一些最现代的浏览器在某种意义上已经document.onload被window.onload.

浏览器支持问题很可能是许多人开始使用jQuery等库来处理文档准备就绪检查的原因，如下所示：

$(document).ready(function() { /* code here */ });
$(function() { /* code here */ });

为了历史的目的。window.onload对比body.onload：

前阵子在编码论坛上提出了一个关于window.onloadover用法的类似问题body.onload。结果似乎是您应该使用，window.onload因为最好将结构与操作分开。


更多建议

一般的想法是 window.onload 在文档窗口 准备好呈现 时触发，而 document.onload在 DOM 树 （从文档中的标记代码构建） 完成 时触发。

理想情况下，订阅DOM 树事件，允许通过 Javascript 进行屏幕外操作， 几乎不会产生 CPU 负载 。相反，当多个外部资源尚未被请求、解析和加载时， window.onload 可能 需要一段时间才能触发。

►测试场景：

要观察差异以及您选择的 浏览器如何 实现 上述 事件处理程序 ，只需在文档的<body>-- 标记中插入以下代码即可。

<script language="javascript">
window.tdiff = []; fred = function(a,b){return a-b;};
window.document.onload = function(e){ 
    console.log("document.onload", e, Date.now() ,window.tdiff,  
    (window.tdiff[0] = Date.now()) && window.tdiff.reduce(fred) ); 
}
window.onload = function(e){ 
    console.log("window.onload", e, Date.now() ,window.tdiff, 
    (window.tdiff[1] = Date.now()) && window.tdiff.reduce(fred) ); 
}
</script>

►结果：

这是由此产生的行为，可在 Chrome v20（以及可能是大多数最新浏览器）中观察到。

  • 没有document.onload事件。
  • onload在 内声明时触发两次<body>，在 内声明时触发一次<head>（事件随后充当document.onload）。
  • 根据计数器的状态进行计数和操作允许模拟这两种事件行为。
  • 或者window.onload在 HTML- 元素的范围内声明事件处理程序<head>。
