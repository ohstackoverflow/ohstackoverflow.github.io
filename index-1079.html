如何将 JSON 对象转换为 TypeScript 类？

Arya

2023-02-18 00:56

我从远程 REST 服务器读取了一个 JSON 对象。此 JSON 对象具有打字稿类的所有属性（按设计）。如何将接收到的 JSON 对象转换为类型 var？

我不想填充打字稿 var（即有一个采用此 JSON 对象的构造函数）。它很大，逐个子对象和逐个属性地跨子对象复制所有内容将花费大量时间。

更新： 但是您可以将其转换为打字稿界面！



解答

您不能简单地将来自 Ajax 请求的普通 JavaScript 结果转换为原型 JavaScript/TypeScript 类实例。有许多技术可以做到这一点，并且通常涉及复制数据。除非您创建该类的实例，否则它不会有任何方法或属性。它仍然是一个简单的 JavaScript 对象。

如果您只处理数据，则可以只对接口进行强制转换（因为它纯粹是编译时结构），这将需要您使用使用数据实例并对该数据执行操作的 TypeScript 类。

复制数据的一些示例：

  1. 将 AJAX JSON 对象复制到现有对象中
  2. 将 JSON 字符串解析为 JavaScript 中的特定对象原型

本质上，你只是：

var d = new MyRichObject();
d.copyInto(jsonResult);


更多建议

我遇到了同样的问题，我找到了一个可以完成这项工作的库：https://github.com/pleerock/class-transformer。

它是这样工作的：

let jsonObject = response.json() as Object;
let fooInstance = plainToClass(Models.Foo, jsonObject);
return fooInstance;

它支持嵌套的孩子，但你必须装饰你班级的成员。

什么是“运行时”？

Arya

2023-02-18 00:55

我听说过“C 运行时”、“Visual C++ 2008 运行时”、“.NET 公共语言运行时”等。

  • 究竟什么是“ 运行时 ”？
  • 它是什么做的？
  • 它如何与我的代码交互？或者更准确地说，它是如何控制我的代码的？

在 Linux 上编写汇编语言时，我可以使用 INT 指令进行系统调用。那么，运行时是否只是一堆预制函数，将低级函数包装成更抽象的高级函数？但这看起来更像是库的定义，而不是运行时的定义吗？

“运行时”和“运行时库”是两个不同的东西吗？

加 1

这些天，我在想，也许 Runtime和所谓的 Virtual Machine 有一些共同点，比如 JVM。这是导致这种想法的报价：

这个编译过程非常复杂，可以分解成几个抽象层，这些通常涉及三个翻译器：编译器、虚拟机实现和汇编器。---计算系统的要素（介绍，通往硬件领域的道路）

添加 2

《专家 C 编程：深度 C 秘密》一书。第 6 章运行时数据结构是对此问题的有用参考。

添加 3 - 2021 年 2 月 28 日上午 7:31

在获得有关处理器设计的一些知识后，这是我的一些观点。整个计算机只是多层次的 抽象 。它从基本晶体管一直到运行程序。对于任何level N抽象，它的 运行时间level N-1是它下面的抽象的直接部分。是上帝给了我们 0 级抽象。



解答

运行时描述了在您的程序运行时 执行 的软件/指令，尤其是那些您没有明确编写但对于正确执行代码所必需的指令。

像 C 这样的低级语言有非常小的（如果有的话）运行时间。更复杂的语言，如允许动态消息传递的 Objective-C，具有更广泛的运行时间。

运行时代码是库代码是正确的，但库代码是一个更笼统的术语，描述任何 库生成的代码。运行时代码具体是实现语言本身的特性所需要的代码。


更多建议

运行时是一个通用术语，指的是您的代码在其上运行的任何库、框架或平台。

C 和 C++ 运行时是函数的集合。

.NET 运行时包含中间语言解释器、垃圾收集器等。

我如何“git blame”删除的行？

Arya

2023-02-18 00:55

git blame非常适合修改和添加的行，但是我如何才能找到特定先前提交中存在的行最终被删除的时间。我在想bisect，但我希望有一些更方便的东西。

（在你问之前：在这种情况下，我只是做了 agit log -p并搜索了代码行并且（a）一些白痴刚刚 删除 了上一次提交中的重要行并且（b）我就是那个白痴。）



解答

如果您知道该行的内容，那么这是一个理想的用例：

git log -S <string> path/to/file

它向您展示引入或删除该字符串实例的提交。还有-G<regex>which 用正则表达式做同样的事情！查看man git- log并搜索-G和-S选项，或镐（这些功能的友好名称）以获取更多信息。

该-S选项实际上也在联机帮助页的标题中提到git-blame，在描述部分，它给出了一个使用git log -S....


更多建议

我想你真正想要的是

git blame --reverse START..END filename

从联机帮助页：

让历史向前走，而不是向后走。这不是显示出现行的修订，而是显示存在行的最后修订。这需要进行一系列修改，例如 START..END，其中 START 中存在责备路径。

使用git blame reverse，您可以找到该行出现的最后一次提交。您仍然需要获取之后的提交。

您可以使用以下命令来显示反向的 git 日志。显示的第一个提交将是该行最后一次出现，下一个提交将是它被更改或删除时。

git log --reverse --ancestry-path COMMIT^..master
