我怎么知道一个分支是否已经合并到 master 中？

Arya

2023-02-17 02:35

我有一个包含多个分支的 git 存储库。

我怎么知道哪些分支已经合并到主分支？



解答

git branch --merged master列出合并到 master 的分支

git branch --merged 列出合并到HEAD 中的分支（即当前分支的尖端）

git branch --no-merged列出尚未合并的分支

默认情况下，这仅适用于本地分支机构。flag-a会同时显示本地和远程分支，而-rflag 只显示远程分支。


更多建议

您可以使用该git merge-base命令查找两个分支之间的最新共同提交。如果该提交与您的分支负责人相同，则该分支已完全合并。

请注意，它已经执行了此类操作，因为它会拒绝删除 尚未git branch -d完全合并的分支。 __

计算两个日期之间的差异（天数）？

Arya

2023-02-17 02:35

我看到这个问题已经针对Java、JavaScript和PHP得到了回答，但没有针对 C# 得到回答。那么，如何计算 C# 中两个日期之间的天数呢？



解答

假设StartDate并且EndDate是类型DateTime：

(EndDate - StartDate).TotalDays


更多建议

最佳答案是正确的，但是如果您只想将整日作为 int 并且很乐意放弃日期的时间部分，那么请考虑：

(EndDate.Date - StartDate.Date).Days

再次假设StartDate和EndDate属于 类型DateTime。

Visual Studio Code 中的垂直标尺

Arya

2023-02-17 02:34

在 VS Code 中渲染多个标尺

VS Code 的标尺默认配置如下所示。

  "editor.ruler": 80

我在使用默认 VS 代码配置（如上所示） 时遇到的问题是它只呈现一个标尺。在 Sublime Text Editor 中，我可以使用以下 Sublime 配置呈现任意数量的标尺。

  "rulers": [72, 80, 100, 120]

是否可以在 VS Code 中呈现多个标尺。 如果可能的话， VS Code 中的多标尺配置是什么样的？



解答

Visual Studio Code 0.10.10 引入了此功能。要配置它，请转到菜单 文件 → 首选项 → 设置 并将其添加到您的用户或工作区设置中：

"editor.rulers": [80,120]

标尺的颜色可以这样定制：

"workbench.colorCustomizations": {
    "editorRuler.foreground": "#ff4081"
}


更多建议

除了全局"editor.rulers"设置之外，还可以在每种语言级别上进行设置。

例如，Python 项目的风格指南通常指定 79 或 120 个字符，而 Git 提交消息不应超过 50 个字符。

所以在你的settings.json，你会把：

"[git-commit]": {"editor.rulers": [50]},
"[python]": {
    "editor.rulers": [
        79,
        120
    ]
}
