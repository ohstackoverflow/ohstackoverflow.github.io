从文本中删除所有空格

Arya

2023-02-17 14:12 

$("#topNav" + $("#breadCrumb2nd").text().replace(" ", "")).addClass("current");

这是我的代码片段。我想在获取另一个 ID 的文本属性后向 ID 添加一个类。这个问题是，包含我需要的文本的 ID 包含字母之间的间隙。

我想删除空格。我试过了TRIM()，REPLACE()但这只能部分奏效。仅REPLACE()删除第一个空格。



解答

您必须告诉 replace() 重复正则表达式：

.replace(/ /g,'')

g 字符使其成为“全局”匹配，这意味着它会重复搜索整个字符串。在此处阅读有关此内容以及 JavaScript 中可用的其他 RegEx 修饰符的信息。

如果要匹配所有空格，而不仅仅是文字空格字符，请改用\s：

.replace(/\s/g,'')

如果您使用的是足够新的 JavaScript 版本，您也可以使用.replaceAll，但对于您的特定用例，实际上没有任何理由，因为捕获 所有 空格需要一个正则表达式，并且在使用正则表达式时.replaceAll，它必须是全局的，所以你只是结束了额外的输入：

.replaceAll(/\s/g,'')


更多建议

.replace(/\s+/, "")

将仅 替换第一个空格，这包括空格、制表符和换行符。

要替换字符串中的所有空格，您需要使用全局模式

.replace(/\s/g, "")

SSH 密钥 - 仍然要求输入密码和密码

Arya

2023-02-17 14:12

当我克隆存储库时，Github 总是要求我提供用户名和密码，这让我有些“忍受”。我想绕过这一步，因为它是我工作流程中的一个烦恼。

我尝试使用本指南设置 SSH 密钥（我成功地做到了）。https://help.github.com/articles/generating-ssh- keys我成功了。

我的问题是，在克隆存储库（使用 SSH）时，系统仍然要求我提供 github 密码和密码。我的理解是，在设置此 SSH 密钥后，我将不再需要这样做。

我有点不确定要问什么，所以我只陈述我的目标。

我希望能够克隆存储库而不必一直输入我的 Github 信息 。

我的 SSH 密钥缺少什么？如果有人可以提供一些指导或资源，我将不胜感激，因为在 GitHub 中进行 SSH 身份验证时，我总是感到有些迷茫。

据我所知，这是一个测试是否正常工作的命令，这是我的控制台的输出：

~ $ ssh -T git@github.com
Saving password to keychain failed
Enter passphrase for key '/Users/MYNAME/.ssh/id_rsa':
Hi MYNAME! You've successfully authenticated, but GitHub does not provide shell access.

当我输入密码时，应该先失败吗？然后，当我输入我的密码时，它通过了。



解答

如果你使用HTTPsurl，它总是会询问你的用户名/密码。这可以使用@Manavalan Gajapathy 的评论（在此处复制）来解决：

请参阅此github 文档以将远程 URL 从 https 转换为 ssh。要检查远程的 URL 是 ssh 还是 https，请使用git remote -v. 从 https 切换到 ssh：

git remote set-url origin git@github.com:USERNAME/REPOSITORY.git

如果您SSH在克隆/设置遥控器时正确使用：确保您有一个 ssh-agent 来记住您的密码（请参阅@Komu 的回答）。这样，您只需在终端会话中输入一次密码。

如果它仍然很烦人，那么只需设置一个没有密码的 ssh-key。

GitHub 和要点有什么区别？

Arya

2023-02-17 14:11

gist 的目的是什么？它与使用 GitHub 的常规代码共享/维护有何不同？



解答

GitHub 是整个站点。Gists 是该网站上提供的一项特殊服务，即类似于 pastebin 的代码片段。然而，一切都是由 git 修订控制驱动的，因此 gists 也有完整的修订历史。


更多建议

我个人对 Gist 和 GitHub 的看法：

Gist： Gist 是一种与他人共享代码片段和粘贴的简单方法。当您需要与您的同事或朋友分享一段示例代码或技术时使用它。

和

GitHub 另一方面，GitHub 提供了一个平台，可以将整个项目作为一个存储库在团队（私有存储库）或整个世界（公共存储库）中共享。

两者都将记录版本详细信息。
