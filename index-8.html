Python 中有三元条件运算符吗？



解答

是的，它是在 2.5 版本中添加的。表达式语法为：

a if condition else b

首先condition求值，然后根据 的布尔值求值并返回aor中的一个。如果计算为，则计算并返回但被忽略，否则计算并返回但被忽略。b``condition``condition``True``a``b``b``a

这允许短路，因为condition只有当为真时才a被评估并且b根本不被评估，但是condition只有当为假时才b被评估并且a根本不被评估。

例如：

>>> 'true' if True else 'false'
'true'
>>> 'true' if False else 'false'
'false'

请注意，条件是一个 表达式 ，而不是一个 语句 。这意味着您不能在条件表达式中使用 语句 ， 例如pass``=``+=

>>> # it can't be on the left-hand side of `=`.
>>> x = 1 if False else y = 2

>>> # If we parenthesize it instead...
>>> (x = 1) if False else (y = 2)

同样，因为它是一个表达式，所以else部分是 强制性的 ：

# condition isn't met. It doesn't matter if we can verify that
# ahead of time.

但是，您可以使用条件表达式来分配变量，如下所示：

或者例如返回一个值：

将条件表达式视为在两个值之间切换。当我们处于“一个值或另一个”的情况时，我们可以使用它，无论条件是否满足，我们都会对结果 做同样的事情。 我们使用表达式来计算值，然后用它做一些事情。如果您需要根据条件 执行不同的操作，请改用普通if 语句 。

如何修改现有的、未推送的提交消息？

Arya

2023-02-16 18:23

这个问题的答案是 社区的努力。编辑现有答案以改进这篇文章。它目前不接受新的答案或互动。

我在提交消息中写错了东西。

如何更改消息？尚未推送提交。



解答

修改最近的提交消息

git commit --amend

将打开您的编辑器，允许您更改最近提交的提交消息。此外，您可以直接在命令行中设置提交消息：

git commit --amend -m "New commit message"

…然而，这会使多行提交消息或小的更正更难以输入。

确保在执行此操作之前没有任何工作副本更改 ， 否则它们也会被提交。（ 未暂存的 更改将不会提交。）

更改已推送到远程分支的提交消息

如果您已经将您的提交推送到您的远程分支，那么 - 在本地修改您的提交（如上所述）之后 - 您还需要强制推送提交：

git push <remote> <branch> --force
# Or
git push <remote> <branch> -f

警告：强制推送将用本地分支的状态覆盖远程分支 。如果远程分支上有您在本地分支中没有的提交，您 将 丢失这些提交。

警告：在修改您已经与其他人共享的提交时要谨慎。 修改提交实质上是将它们 重写 为具有不同的SHA ID，如果其他人拥有您已重写的旧提交的副本，这就会造成问题。任何拥有旧提交副本的人都需要将他们的工作与您新重写的提交同步，这有时会很困难，因此在尝试重写共享提交历史时确保与其他人协调，或者避免重写共享提交共。

执行交互式变基

另一种选择是使用交互式变基。这使您可以编辑任何要更新的消息，即使它不是最新消息。

要执行 Git squash，请按照以下步骤操作：

// n is the number of commits up to the last commit you want to be able to edit
git rebase -i HEAD~n

压缩提交后 - 选择e/r用于编辑消息的：

关于交互式变基的重要说明

当您使用时git rebase -i HEAD~n，可以有 超过 n 次提交。Git 将“收集”最后 n 个提交中的所有提交，如果在该范围之间的某处存在合并，您也会看到所有提交，因此结果将为 n + 。

好提示：

如果您必须为多个分支执行此操作，并且在修改内容时可能会遇到冲突，请设置git rerere并让 Git 自动为您解决这些冲突。

如何让 Git 忘记已跟踪但现在位于 .gitignore 中的文件？

Arya

2023-02-16 18:22

我把一个之前被 Git 跟踪的文件放到了列表中.gitignore。但是，文件git status在编辑后仍然显示在中。我如何强制 Git 完全忘记该文件？



解答

.gitignore将防止未跟踪的文件被添加（没有add -f）到 Git 跟踪的文件集中。但是，Git 将继续跟踪任何已经被跟踪的文件。

要停止跟踪文件，我们必须将其从索引中删除：

git rm --cached <file>

递归删除文件夹和文件夹中的所有文件：

git rm -r --cached <folder>

从 head 修订版中删除文件将在下一次提交时发生。

警告：虽然这不会从您的本地计算机上删除物理文件，但它会从其他开发人员的计算机上删除他们下一个git pull.


更多建议

下面的一系列命令将从 Git 索引（而不是从工作目录或本地存储库）中删除所有项目，然后将更新 Git 索引，同时尊重 Git 忽略。 附言。 索引=缓存

第一的：

git rm -r --cached .
git add .

然后：

git commit -am "Remove ignored files"

或者作为单线：

git rm -r --cached . && git add . && git commit -am "Remove ignored files"
