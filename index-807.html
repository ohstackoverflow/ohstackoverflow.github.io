postgres：将用户升级为超级用户？

Arya

2023-02-17 16:38

在 postgres 中，如何将现有用户更改为超级用户？由于各种原因，我不想删除现有用户。

# alter user myuser ...?



解答

ALTER USER myuser WITH SUPERUSER;

您可以在ALTER USER 的文档中阅读更多内容


更多建议

要扩展以上内容并进行快速参考：

  • 使用户成为超级用户：ALTER USER username WITH SUPERUSER;
  • 使用户不再是超级用户：ALTER USER username WITH NOSUPERUSER;
  • 只允许用户创建数据库：ALTER USER username CREATEDB;

您还可以使用CREATEROLE和CREATEUSER允许用户权限而不使他们成为超级用户。

在 Python 中进行 HTTP GET 的最快方法是什么？

Arya

2023-02-17 16:38

如果我知道内容将是一个字符串，那么在 Python 中使用 HTTP GET 的最快方法是什么？我正在搜索文档以寻找快速的单行代码，例如：

contents = url.get("http://example.com/foo/bar")

但我使用谷歌所能找到的只是httplib和urllib- 我无法在这些库中找到快捷方式。

标准 Python 2.5 是否具有上述某种形式的快捷方式，或者我应该编写一个函数url_get？

  1. 我宁愿不捕获 shelling out towget或的输出curl。



解答

蟒蛇 3：

import urllib.request
contents = urllib.request.urlopen("http://example.com/foo/bar").read()

蟒蛇2：

import urllib2
contents = urllib2.urlopen("http://example.com/foo/bar").read()

urllib.request和的文档read。


更多建议

使用请求库：

import requests
r = requests.get("http://example.com/foo/bar")

然后你可以做这样的事情：

>>> print(r.status_code)
>>> print(r.headers)
>>> print(r.content)  # bytes
>>> print(r.text)     # r.content as str

通过运行以下命令安装请求：

pip install requests

从 Bash 中的字符串中删除固定前缀/后缀

Arya

2023-02-17 16:37

我想从字符串中删除前缀/后缀。例如，给定：

string="hello-world"
prefix="hell"
suffix="ld"

我如何获得以下结果？

"o-wor"



解答

$ prefix="hell"
$ suffix="ld"
$ string="hello-world"
$ foo=${string#"$prefix"}
$ foo=${foo%"$suffix"}
$ echo "${foo}"
o-wor

这记录在手册的Shell Parameter Expansion部分：

${parameter#word}
${parameter##word}

这个词被扩展以产生一个模式并根据下面描述的规则进行匹配（请参阅模式匹配）。如果模式与参数扩展值的开头相匹配，则扩展的结果是删除了最短匹配模式（case）或#最长匹配模式（##case）的参数扩展值。[…]

${parameter%word}
${parameter%%word}

这个词被扩展以产生一个模式并根据下面描述的规则进行匹配（请参阅模式匹配）。%如果模式与参数扩展值的尾部匹配，则扩展的结果是删除最短匹配模式（案例）或最长匹配模式（案例%%）的参数值。[…]


更多建议

使用 sed：

$ echo "$string" | sed -e "s/^$prefix//" -e "s/$suffix$//"
o-wor

在 sed 命令中，^字符匹配以 开头的文本$prefix，尾部$匹配以 结尾的文本$suffix。

Adrian Frühwirth 在下面的评论中提出了一些很好的观点，但sed对于此目的可能非常有用。$prefix 和 $suffix 的内容由 sed 解释这一事实可以是好也可以是坏——只要你注意，你应该没问题。美妙的是，你可以这样做：

$ prefix='^.*ll'
$ suffix='ld$'
$ echo "$string" | sed -e "s/^$prefix//" -e "s/$suffix$//"
o-wor

这可能正是您想要的，并且比 bash 变量替换更漂亮、更强大。如果你记得能力越大责任越大（正如蜘蛛侠所说），你应该没问题。

可以在http://evc-cit.info/cit052/sed_tutorial.html找到 sed 的快速介绍

关于 shell 及其字符串使用的注释：

对于给出的特定示例，以下内容也适用：

$ echo $string | sed -e s/^$prefix// -e s/$suffix$//

...但仅仅是因为：

  1. echo 不关心它的参数列表中有多少个字符串，并且
  2. $prefix 和 $suffix 中没有空格

在命令行中引用字符串通常是一种很好的做法，因为即使它包含空格，它也会作为单个参数呈现给命令。我们出于同样的原因引用 $prefix 和 $suffix：每个对 sed 的编辑命令都将作为一个字符串传递。我们使用双引号是因为它们允许变量插值；如果我们使用单引号，sed 命令会得到一个文字$prefix，$suffix这肯定不是我们想要的。

还要注意，我在设置变量prefix和时使用了单引号suffix。我们当然不希望字符串中的任何内容被解释，所以我们将它们单引号，这样就不会发生插值。同样，在此示例中可能没有必要，但这是一个很好的习惯。
