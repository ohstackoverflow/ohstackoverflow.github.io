如何将端口映射分配给现有的 Docker 容器？

Arya

2023-02-17 17:52

我不确定我是否在这里误解了什么，但似乎只能通过从图像创建新容器来设置端口映射。有没有办法将端口映射分配给现有的 Docker 容器？



解答

您可以通过直接编辑或 处的hostconfig.json文件来更改端口映射，我相信，如果您快速安装了 Docker。/var/lib/docker/containers/[hash_of_the_container]/hostconfig.json``/var/snap/docker/common/var- lib-docker/containers/[hash_of_the_container]/hostconfig.json

您可以通过命令确定 [hash_of_the_container] docker inspect <container_name>，“Id”字段的值是散列。

  1. 停止容器 ( docker stop <container_name>)。
  2. 停止 docker 服务（根据 Tacsiazuma 的评论）
  3. 更改文件。
  4. 重新启动您的 docker 引擎（以刷新/清除配置缓存）。
  5. 启动容器 ( docker start <container_name>)。

所以你不需要用这种方法创建图像。您还可以在此处更改重启标志。

PS 您可以访问 https://docs.docker.com/engine/admin/了解如何根据您的主机正确重启您的 docker 引擎。我曾经sudo systemctl restart docker重新启动在 Ubuntu 16.04 上运行的 docker 引擎。

以下未跟踪的工作树文件将被合并覆盖，但我不在乎

Arya

2023-02-17 17:51

在我的分支上，我在 .gitignore 中有一些文件

在不同的分支上，这些文件不是。

我想将不同的分支合并到我的分支中，我不在乎这些文件是否不再被忽略。

不幸的是我得到这个：

以下未跟踪的工作树文件将被合并覆盖

我将如何修改我的 pull 命令以覆盖这些文件，而不必自己查找、移动或删除这些文件？



解答

问题是您没有在本地跟踪文件，而是远程跟踪相同的文件，因此为了“拉”您的系统将被迫覆盖不受版本控制的本地文件。

尝试跑步

git add * 
git stash
git pull

这将跟踪所有文件，删除对这些文件的所有本地更改，然后从服务器获取文件。


更多建议

您可以尝试命令从本地清除未跟踪的文件

Git 2.11 及更新版本：

git clean  -d  -f .

旧版本的 Git：

git clean  -d  -f ""

Where-d可以替换为以下内容：

  • -x忽略的文件以及 Git 未知的文件也会被删除。

  • -d除了未跟踪的文件外，还删除未跟踪的目录。

  • -f需要强制它运行。

这是也可能有用的链接。

使用 LINQ 获取一个 List<> 中不在另一个 List<> 中的项目

Arya

2023-02-17 17:51

我会假设有一个简单的 LINQ 查询来执行此操作，我只是不确定如何操作。

鉴于这段代码：

class Program
{
    static void Main(string[] args)
    {
        List<Person> peopleList1 = new List<Person>();
        peopleList1.Add(new Person() { ID = 1 });
        peopleList1.Add(new Person() { ID = 2 });
        peopleList1.Add(new Person() { ID = 3 });

        List<Person> peopleList2 = new List<Person>();
        peopleList2.Add(new Person() { ID = 1 });
        peopleList2.Add(new Person() { ID = 2 });
        peopleList2.Add(new Person() { ID = 3 });
        peopleList2.Add(new Person() { ID = 4 });
        peopleList2.Add(new Person() { ID = 5 });
    }
}

class Person
{
    public int ID { get; set; }
}

我想执行一个 LINQ 查询来给我所有peopleList2不在peopleList1.

这个例子应该给我两个人（ID = 4 & ID = 5）



解答

这可以使用以下 LINQ 表达式解决：

var result = peopleList2.Where(p => !peopleList1.Any(p2 => p2.ID == p.ID));

通过 LINQ 表达这一点的另一种方式，一些开发人员发现它更具可读性：

var result = peopleList2.Where(p => peopleList1.All(p2 => p2.ID != p.ID));

警告： 如评论中所述，这些方法强制执行 O(n*m) 操作。这可能很好，但可能会引入性能问题，尤其是在数据集非常大的情况下。如果这不能满足您的性能要求，您可能需要评估其他选项。由于规定的要求是针对 LINQ 中的解决方案，因此此处不探讨这些选项。一如既往，根据您的项目可能具有的性能要求评估任何方法。


更多建议

如果您覆盖 People 的平等性，那么您还可以使用：

peopleList2.Except(peopleList1)

Except应该比变体快得多，Where(...Any)因为它可以将第二个列表放入哈希表中。Where(...Any)有一个运行时间O(peopleList1.Count * peopleList2.Count)，而基于HashSet<T>（几乎）的变体有一个运行时间O(peopleList1.Count + peopleList2.Count)。

Except隐式删除重复项。这不应该影响你的情况，但对于类似的情况可能是一个问题。

或者，如果您想要快速代码但不想覆盖相等性：

var excludedIDs = new HashSet<int>(peopleList1.Select(p => p.ID));
var result = peopleList2.Where(p => !excludedIDs.Contains(p.ID));

此变体不会删除重复项。
