我已经阅读了关于const和static readonly字段的信息。我们有一些类只包含常量值。它们用于我们系统中的各种事物。所以我想知道我的观察是否正确：

这些恒定值是否应该始终适用static readonly于所有公开的东西？并且只const用于internal//protected值private？

你有什么建议吗？我什至不应该使用static readonly字段，而是使用属性吗？



解答

public static readonly字段有点不寻常；public static属性（只有一个get）会更常见（可能由一个private static readonly字段支持）。

const值直接写入调用站点；这是双刃剑：

  • 如果值是在运行时获取的，可能是从配置中获取的，那么它是无用的
  • 如果更改 const 的值，则需要重建所有客户端
  • 但它可以更快，因为它避免了方法调用......
  • ...有时可能已经被 JIT 内联

如果该值 永远 不会改变，那么 const 就可以了 -Zero等等 合理的 consts ;p 除此之外，static属性更常见。

if/else 在列表理解中

Arya

2023-02-17 01:32

xs我有一个包含字符串和值混合的列表None。我如何使用列表推导式在每个字符串上调用函数，但将值转换None为''（而不是将它们传递给函数）？

我试过：

[f(x) for x in xs if x is not None else '']

但它给出了一个SyntaxError。正确的语法是什么？

如果您尝试创建一个基于条件省略值 的 列表推导，请参阅带条件的列表推导。 __

如果您需要考虑两个以上的条件结果，请注意 Python 的条件表达式不支持elif. 相反，有必要嵌套if/else条件。有关详细信息，请参阅列表理解条件中的 elif。



解答

你完全可以做到。这只是一个订购问题：

[f(x) if x is not None else '' for x in xs]

一般来说，

[f(x) if condition else g(x) for x in sequence]

并且，对于if仅具有条件的列表理解，

[f(x) for x in sequence if condition]

请注意，这实际上使用了不同的语言构造，即条件表达式，它本身不是理解语法的一部分，而if后面的for…in是列表理解的一部分，用于从源可迭代中 过滤元素。

条件表达式可用于您希望根据某些条件在两个表达式值之间进行选择的各种情况。这与存在于其他语言中的三元运算符的?:作用相同。例如：

value = 123
print(value, 'is', 'even' if value % 2 == 0 else 'odd')


更多建议

让我们用这个问题来回顾一些概念。我认为首先了解基本原理是很好的，这样您就可以推断出不同的情况。

其他答案提供了您问题的具体答案。我将首先提供一些一般背景，然后我将回答问题。

基础知识

if/else列表理解中的语句涉及两件事：

  • 列表理解
  • 条件表达式（三元运算符）

1. 列表理解

它们提供了一种创建列表的简洁方法。

它的结构包括：“ 括号包含一个表达式，后跟一个 for 子句，然后是零个或多个 for 或 if 子句 ”。

情况1

这里我们没有条件。iterable 中的每个项目都添加到new_list.

new_list = [expression for item in iterable]
new_list = [x for x in range(1, 10)]
> [1, 2, 3, 4, 5, 6, 7, 8, 9]

案例二

这里我们有一个条件。

示例 1

条件：只有 偶数 会被添加到new_list.

new_list = [expression for item in iterable if condition == True]
new_list = [x for x in range(1, 10) if x % 2 == 0]
> [2, 4, 6, 8]

示例 2

条件：只有3 的倍数的 偶数 才会被加到new_list.

new_list = [expression for item in iterable if condition == True]
new_list = [x for x in range(1, 10) if x % 2 == 0 if x % 3 == 0]
> [6]

但是如果我们使用两个ifin怎么会有一个条件new_list呢？

先验表达式可以写成：

new_list = [x for x in range(1, 10) if x % 2 and x % 3 == 0]
> [6]

我们只使用一个if语句。

这就像做：

new_list = []
for x in range(1, 10):
    if x % 2 == 0 and x % 3 == 0:
        new_list.append(x)
> [6]

示例 3

只是为了争论，你也可以使用or.

条件：将 偶数 或3的倍数加到new_list。

new_list = [x for x in range(1, 10) if x % 2 == 0 or x % 3 == 0]
> [2, 3, 4, 6, 8, 9]

案例三

不止一种情况：

这里我们需要条件表达式（三元运算符）的帮助。

2.条件表达式

什么是条件表达式？顾名思义：具有某些条件的 Python 表达式。

<Exp1> if condition else <Exp2>

首先condition评估。如果condition是True，则<Exp1>评估并返回。如果condition是False，则<Exp2>评估并返回。

具有多个条件的条件表达式：

<Exp1> if condition else <Exp2> if condition else <Exp3>...

来自Real Python的示例：

age = 12
s = 'minor' if age < 21 else 'adult'
> minor

的价值s以age价值为条件。

3.List Comprehensions with Conditionals

我们像这样将列表理解和条件放在一起。

new_list = [<Conditional Expression> for <item> in <iterable>]

new_list = [<Exp1> if condition else <Exp2> if condition else <Exp3> for <item> in <iterable>]

条件： 偶数 加为'even'，数字三加为'number three'，其余加为'odd'。

new_list = ['even' if x % 2 == 0 else 'number three' if x == 3 else 'odd' 
             for x in range(1, 10)]
> ['odd', 'even', 'number three', 'even', 'odd', 'even', 'odd', 'even', 'odd']

问题的答案

[f(x) for x in xs if x is not None else '']

这里我们遇到了列表结构的问题：for x in xs应该在表达式的末尾。

正确方法：

[f(x) if x is not None else '' for x in xs]

确定两个日期范围是否重叠

Arya

2023-02-17 01:32

给定两个日期范围，确定两个日期范围是否重叠的最简单或最有效的方法是什么？

例如，假设我们有由 DateTime 变量StartDate1toEndDate1 和 StartDate2to表示的范围EndDate2。



解答

(StartA <= EndB) 和 (EndA >= StartB)

证明：
令 ConditionA 表示 DateRange A 完全在 DateRange B 之后

_                        |---- DateRange A ------|
|---Date Range B -----|                          _

（如果为真StartA > EndB）

让 ConditionB 表示 DateRange A 完全在 DateRange B 之前

|---- DateRange A -----|                        _ 
_                          |---Date Range B ----|

（如果为真EndA < StartB）

如果 A 和 B 都不为真，则存在重叠 -
（如果一个范围既不完全在另一个范围之后，
也不完全在另一个范围之前，则它们必须重叠。）

现在，德摩根的一项法律规定：

Not (A Or B)<=>Not A And Not B

这转化为：(StartA <= EndB) and (EndA >= StartB)

注意：这包括边缘完全重叠的情况。如果您希望排除它，请将运算符
更改>=为>,<=和<

笔记2。感谢@Baodad，请参阅此博客，实际重叠最少：
{ endA-startA, endA - startB, endB-startA, endB - startB}

(StartA <= EndB) and (EndA >= StartB) (StartA <= EndB) and (StartB <= EndA)

注3。感谢@tomosius，较短的版本如下：
DateRangesOverlap = max(start1, start2) < min(end1, end2)
这实际上是较长实现的语法快捷方式，其中包括额外的检查以验证开始日期是否在结束日期之前或之前。从上面推导出这个：

如果开始日期和结束日期可能乱序，即，如果startA > endAorstartB > endB有可能，那么您还必须检查它们是否有序，这意味着您必须添加两个额外的有效性规则：
(StartA <= EndB) and (StartB <= EndA) and (StartA <= EndA) and (StartB <= EndB)or:
(StartA <= EndB) and (StartA <= EndA) and (StartB <= EndA) and (StartB <= EndB)or,
(StartA <= Min(EndA, EndB) and (StartB <= Min(EndA, EndB))or:
(Max(StartA, StartB) <= Min(EndA, EndB)

但是要实现Min()and Max()，您必须编写代码（为简洁起见，使用 C 三元）：
((StartA > StartB) ? StartA : StartB) <= ((EndA < EndB) ? EndA : EndB)


更多建议

我认为在以下情况下，说这两个范围重叠就足够了：

(StartDate1 <= EndDate2) and (StartDate2 <= EndDate1)
