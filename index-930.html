如何在 TypeScript 中将字符串转换为枚举？

Arya

2023-02-17 20:00

我在 TypeScript 中定义了以下枚举：

enum Color{
    Red, Green
}

现在在我的函数中，我接收颜色作为字符串。我尝试了以下代码：

var green= "Green";
var color : Color = <Color>green; // Error: can't convert string to enum

如何将该值转换为枚举？



解答

TypeScript 0.9 中的枚举是基于字符串+数字的。简单转换不需要类型断言：

enum Color{
    Red, Green
}

// To String
 var green: string = Color[Color.Green];

// To Enum / number
var color : Color = Color[green];

在线试用

我在我的 OSS 书中有关于这个和其他枚举模式的文档：https://basarat.gitbook.io/typescript/type- system/enums


更多建议

从 Typescript 2.1 开始，枚举中的字符串键是强类型的。keyof typeof用于获取有关可用字符串键 ( 1 ) 的信息：

enum Color{
    Red, Green
}

let typedColor: Color = Color.Green;
let typedColorString: keyof typeof Color = "Green";

// Error "Black is not assignable ..." (indexing using Color["Black"] will return undefined runtime)
typedColorString = "Black";

// Error "Type 'string' is not assignable ..." (indexing works runtime)
let letColorString = "Red";
typedColorString = letColorString;

// Works fine
typedColorString = "Red";

// Works fine
const constColorString = "Red";
typedColorString = constColorString

// Works fine (thanks @SergeyT)
let letColorString = "Red";
typedColorString = letColorString as keyof typeof Color;

typedColor = Color[typedColorString];

https://www.typescriptlang.org/docs/handbook/advanced-types.html#index-types

git switch 和 git checkout <branch> 有什么区别

Arya

2023-02-17 19:59

Git 2.23引入了一个新命令git switch——阅读文档后，它看起来几乎相同，git checkout <branchname>有人可以解释一下区别或用例吗？

引入了两个新命令“git switch”和“git restore”来拆分“检出一个分支以推进其历史”和“检出索引和/或树状结构以推进当前的路径”从单个“git checkout”命令中提取“历史记录”。



解答

该switch命令确实与 执行相同的操作checkout，但仅适用于切换分支的那些用法。特别是，不像checkout它不能恢复工作树文件——而是使用restore与switch.

详细解释

正如您在引用的 2.23.0 发行说明部分中所述，引入了switch和命令以将命令拆分为两个单独的部分：restore``checkout

  • “检查一个分支以推进其历史”
  • “检查索引和/或树状结构的路径以推进当前历史”

换句话说，checkout做两件不同的事情，而这个版本将这些不同的事情分成了自己的重点命令。

的双重用途可以在文档checkout中的摘要描述中看到：

git-checkout - 切换分支或恢复工作树文件

添加命令的提交在switch其提交消息中解释了新命令的基本原理：

“git checkout”做太多事情是许多用户混淆的根源（有时甚至会咬老计时器）。为了解决这个问题，该命令将分为两个新命令：switch 和 restore。古老的“git checkout”命令仍然存在，并且会一直存在，直到所有（或大多数用户）都厌倦了它。

由此可见，引入新命令是为了减少混淆，因为它有两个重点命令，而不是一个多用途命令。

请注意，截至 2021 年 12 月，新命令仍列为实验性命令 ( switch, restore)：

此命令是实验性的。行为可能会改变。

  • 用于在切换前创建新分支的-b/选项重命名为/ 。它们也有长选项变体 ( / )，而以前它们只有单字母选项版本。-B``-c``-C``--create``--force-create
  • --detach(or -d) 现在在切换到分离头时总是需要的，以前它对于提交是可选的，但对于分支是必需的。
  • -s用于恢复的源代码树现在由(or ) 选项给出--source，而不是内联参数。
  • --force如果有未合并的条目，则使用 now 切换会失败，而不是忽略它们。--force也已重命名为--discard-changes，并--force作为别名保留。


更多建议

git checkout有点像瑞士军刀，因为它有几个不相关的用途。

如果您修改了一个文件但没有暂存更改，则将git checkout <filename>撤消修改……这是一种取消对文件更改的快速简便的方法。你留在同一个分支。

git checkout <branchname>（如您所述）切换分支。

两个完全不同的目的，如果文件名和分支名称相似，可能会导致混淆。

将它作为两个命令更清楚。

但是，当此列中的一个单元格中的文本太长时，列的宽度会变得超过100px. 我怎样才能禁用此扩展？



解答

我玩了一会儿，因为我很难弄明白。

您需要设置单元格宽度（要么th有效td，我都设置）并将其设置table-layout为fixed. 出于某种原因，单元格宽度似乎只有在设置了表格宽度的情况下才会保持固定（我认为这很愚蠢，但无论如何）。

overflow此外，将该属性设置为hidden以防止任何额外的文本从表中出来也很有用。

你应该确保也为 CSS 保留所有的边框和大小。
