解答

更改您的文件权限...像这样

首先检查谁拥有该目录

ls -la /usr/local/lib/node_modules

它拒绝访问，因为 node_module 文件夹归 root 所有

drwxr-xr-x   3 root    wheel  102 Jun 24 23:24 node_modules

所以这需要通过将 root 更改为您的用户来更改，但首先运行下面的命令以检查您当前的用户如何通过 OS X 中的命令行获取活动用户的名称？

id -un或者whoami

然后换主人

sudo chown -R [owner]:[owner] /usr/local/lib/node_modules

或者

sudo chown -R ownerName: /usr/local/lib/node_modules

或者

sudo chown -R $USER /usr/local/lib/node_modules


更多建议

为了尽量减少权限错误的可能性，您可以将 npm 配置为使用不同的目录。在此示例中，您将在您的主目录中创建和使用一个隐藏目录。

备份您的计算机。在命令行上，在您的主目录中，为全局安装创建一个目录：

mkdir ~/.npm-global

配置 npm 以使用新的目录路径：

npm config set prefix '~/.npm-global'

在您首选的文本编辑器中，打开或创建文件~/.profile并添加以下行：

export PATH=~/.npm-global/bin:$PATH

在命令行上，更新您的系统变量：

source ~/.profile

要测试您的新配置，请在不使用sudo.

什么是 Python 蛋？

Arya

2023-02-17 18:55

我试图了解 Python 包的工作原理。大概eggs是某种打包机制，但如何快速概述它们所扮演的角色，以及关于它们为何有用以及如何创建它们的一些信息？



解答

注意： Egg 包装已被 Wheel 包装取代。

与 Java 中文件的概念相同.jar，它是.zip一些元数据文件重命名的文件.egg，用于将代码作为包分发。

具体来说：Python Eggs 的内部结构

“Python 蛋”是一种逻辑结构，体现了 Python 项目特定版本的发布，包括其代码、资源和元数据。有多种格式可用于对 Python egg 进行物理编码，并且可以开发其他格式。然而，Python eggs 的一个关键原则是它们应该是可发现和可导入的。也就是说，Python 应用程序应该可以轻松高效地找出系统中存在的鸡蛋，并确保所需鸡蛋的内容是可导入的。

该.egg格式非常适合代码的分发和轻松卸载或升级，因为该项目本质上是独立于单个目录或文件中的，没有与任何其他项目的代码或资源混合。它还使得同时安装一个项目的多个版本成为可能，这样各个程序就可以选择他们希望使用的版本。


更多建议

该.egg文件是 Python 包的分发格式。它只是源代码分发或 Windows exe. 但请注意，对于 pure Python，该.egg文件是完全跨平台的。

文件.egg本身本质上是一个.zip文件。如果将扩展名更改为“ zip”，您可以看到它在存档中有文件夹。

另外，如果你有一个.egg文件，你可以使用easy_install

示例：要为一个目录创建一个.egg文件，mymath比如它本身可能有多个 python 脚本，请执行以下步骤：

# setup.py
from setuptools import setup, find_packages
setup(
    name = "mymath",
    version = "0.1",
    packages = find_packages()
    )

然后，从终端执行：

 $ python setup.py bdist_egg

这将生成大量输出，但是当它完成时，您会看到您拥有 三个 新文件夹： build 、 dist 和 mymath.egg-info 。我们唯一关心的文件夹是 dist 文件夹，您可以在其中找到您的.egg文件， mymath-0.1-py3.5.egg 以及您的默认 python（安装）版本号（我在这里：3.5）

解答

首先推荐这篇文章：Java: When to create a final class

如果他们这样做，他们什么时候使用它，这样我就可以更好地理解它并知道什么时候使用它。

类就是 不能扩展的final类。 ****

（这并不意味着对类对象的所有引用都将像它们被声明为一样final。）

这个问题的答案涵盖了何时将类声明为 final 有用：

  • 在 Java 中禁止继承的充分理由？

如果 Java 是面向对象的，你声明一个类final，是不是就停止了类具有对象特性的想法？

在某种意义上是的。

通过将类标记为 final，您可以为该部分代码禁用该语言的强大而灵活的功能。然而，有些类不应该（在某些情况下 不能 ）设计为以一种好的方式考虑子类化。在这些情况下，将类标记为最终类是有意义的，即使它限制了 OOP。（但是请记住，final 类仍然可以扩展另一个非 final 类。）


更多建议

在 Java 中，final不能更改带有修饰符的项目！

这包括最终类、最终变量和最终方法：

  • final 类不能被任何其他类扩展
  • final 变量不能重新赋值
  • final 方法不能被覆盖
