BehaviorSubject 与 Observable？

Arya

2023-02-17 08:04

我正在研究 Angular RxJs 模式，但我不明白 aBehaviorSubject和Observable.

根据我的理解，aBehaviorSubject是一个可以随时间变化的值（可以订阅并且订阅者可以接收更新的结果）。这似乎与Observable.

你什么时候用 an Observablevs a BehaviorSubject？使用BehaviorSubjectover an有什么好处Observable，反之亦然？



解答

BehaviorSubject 是一种主题，主题是一种特殊类型的可观察对象，因此您可以像订阅任何其他可观察对象一样订阅消息。BehaviorSubject 的独特之处在于：

  • 它需要一个初始值，因为它必须始终在订阅时返回一个值，即使它没有收到next()
  • 订阅后，它返回主题的最后一个值。常规可观察对象仅在收到onnext
  • 在任何时候，您都可以使用该方法在不可观察的代码中检索主题的最后一个值getValue()。

与 observable 相比，subject 的独特特征是：

  • 除了作为可观察对象之外，它还是一个观察者，因此除了订阅它之外，您还可以向主题发送值。

此外，您可以使用asObservable()on 方法从行为主题中获取一个可观察对象BehaviorSubject。

Observable 是一个 Generic，并且BehaviorSubject在技术上是 Observable 的子类型，因为 BehaviorSubject 是一个具有特定特性的可观察对象。

BehaviorSubject 示例：

// Behavior Subject

// a is an initial value. if there is a subscription 
// after this, it would get "a" value immediately
let bSubject = new BehaviorSubject("a");

bSubject.next("b");

bSubject.subscribe(value => {
  console.log("Subscription got", value); // Subscription got b, 
                                          // ^ This would not happen 
                                          // for a generic observable 
                                          // or generic subject by default
});

bSubject.next("c"); // Subscription got c
bSubject.next("d"); // Subscription got d

带有常规主题的示例 2：

// Regular Subject

let subject = new Subject();

subject.next("b");

subject.subscribe(value => {
  console.log("Subscription got", value); // Subscription won't get 
                                          // anything at this point
});

subject.next("c"); // Subscription got c
subject.next("d"); // Subscription got d

Subject可以通过和BehaviorSubject使用来创建可观察对象subject.asObservable()。

唯一的区别是您不能将值发送到可观察的 usingnext()方法。

在 Angular 服务中，我会使用BehaviorSubject数据服务，因为 Angular 服务通常在组件和行为主题之前初始化，以确保使用该服务的组件接收到最后更新的数据，即使自组件订阅此数据以来没有新的更新也是如此。


更多建议

Observable：每个观察者的结果不同

一个非常非常重要的区别。由于 Observable 只是一个函数，它没有任何状态，所以对于每一个新的 Observer，它都会一次又一次地执行 observable 创建代码。这导致：

为每个观察者运行代码。如果它是一个 HTTP 调用，它会为每个观察者调用

这会导致重大错误和效率低下

BehaviorSubject （或 Subject ）存储观察者详细信息，仅运行一次代码并将结果提供给所有观察者。

前任：

JSBin：http://jsbin.com/qowulet/edit?js, console

// --- Observable ---
let randomNumGenerator1 = Rx.Observable.create(observer => {
   observer.next(Math.random());
});

let observer1 = randomNumGenerator1
      .subscribe(num => console.log('observer 1: '+ num));

let observer2 = randomNumGenerator1
      .subscribe(num => console.log('observer 2: '+ num));


// ------ BehaviorSubject/ Subject

let randomNumGenerator2 = new Rx.BehaviorSubject(0);
randomNumGenerator2.next(Math.random());

let observer1Subject = randomNumGenerator2
      .subscribe(num=> console.log('observer subject 1: '+ num));

let observer2Subject = randomNumGenerator2
      .subscribe(num=> console.log('observer subject 2: '+ num));

如何在 Java 中将 String 转换为 InputStream？

Arya

2023-02-17 08:04

给定一个字符串：

String exampleString = "example";

如何将其转换为InputStream？



解答

像这样：

InputStream stream = new ByteArrayInputStream(exampleString.getBytes(StandardCharsets.UTF_8));

请注意，这假设您需要一个 InputStream，它是一个字节流，表示编码为 UTF-8 的原始字符串。

对于低于 7 的 Java 版本，替换StandardCharsets.UTF_8为"UTF-8".


更多建议

我发现使用Apache Commons IO让我的生活更轻松。

String source = "This is the source of my input stream";
InputStream in = org.apache.commons.io.IOUtils.toInputStream(source, "UTF-8");

您可能会发现该库还提供了许多其他快捷方式，可以完成您可以在项目中使用的常见任务。

如何在 Git 中克隆单个分支？

Arya

2023-02-17 08:03

我在 .net 中有一个本地 Git 存储库~/local_repo。它有几个分支：

$ git branch
* master
  rails
  c
  c++

要克隆本地存储库，我会这样做：

$ git clone ~/local_repo new_repo
Initialized empty Git repository in /home/username/new_repo/.git/

masternew_repo分支指向local_repomaster分支，我可以push/pull。

但是我无法克隆另一个分支。我只想拉出我想要的分支（例如rails），以便新存储库有一个默认情况master下推送到 的分支和从local_repo的分支拉出的分支。我如何完成此操作，或者可能与跟踪 masterrails类似？local_repo``local_repo



解答

注意： git1.7.10 （2012 年 4 月）实际上只允许 克隆一个分支 ：

# clone only the remote primary HEAD (default: origin/master)
git clone <url> --single-branch

# as in:
git clone <url> --branch <branch> --single-branch <folder>

笔记：

  • <url>是远程仓库的 URL，不引用克隆的分支
  • <folder>是您要克隆存储库的本地文件夹

您可以在以下位置看到它t5500-fetch-pack.sh：

test_expect_success 'single branch clone' '
  git clone --single-branch "file://$(pwd)/." singlebranch
'

东武 评论说：

这在进行浅克隆时是隐式的。
这是git clone --depth 1节省带宽的最简单方法。

自 Git 1.9.0（2014 年 2 月）起，浅克隆支持数据传输（推/拉），因此该选项现在更加有用。
更多信息请参阅“ （浅克隆）是否git clone --depth 1比它看起来更有用？ ”。

“撤消”浅克隆在“将浅克隆转换为完整克隆”(git 1.8.3+)中有详细说明

# unshallow the current branch
git fetch --unshallow

# for getting back all the branches (see Peter Cordes' comment)
git config remote.origin.fetch refs/heads/*:refs/remotes/origin/*
git fetch --unshallow

正如克里斯评论的那样：

让丢失的分支逆转的神奇线--single-branch是（git v2.1.4）：

git config remote.origin.fetch +refs/heads/*:refs/remotes/origin/*
git fetch --unshallow

在 Git 2.26（2020 年第一季度）中，“ git clone --recurse-submodules --single-branch” 现在在克隆子模块时使用相同的单分支选项 。

请参阅Emily Shaffer ( )的提交 132f600、提交 4731957（2020 年 2 月 21 日）。（由Junio C Hamano合并——提交b22db26，2020年 3 月 5 日）nasamuffin
gitster

clone: 在 --recurse-submodules 期间传递 --single-branch

签字人：Emily Shaffer
确认人：Jeff King

以前，git clone --recurse-submodules --single- branch即使超级项目只克隆了一个分支，执行“”也会导致子模块克隆所有分支。

--single-branch通过子模块帮助器框架进行管道传输，以便clone稍后将其添加到 ' '。


更多建议

一种方法是执行以下操作。

git clone user@git-server:project_name.git -b branch_name /your/folder

branch_name您选择的分支在哪里，“/your/folder”是该分支的目标文件夹。确实，这会带来其他分支，让您有机会来回合并。

更新

现在，从 Git 1.7.10 开始，您现在可以执行此操作

git clone user@git-server:project_name.git -b branch_name --single-branch /your/folder
