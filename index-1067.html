解答

不同 C# 版本的解决方案

至少具有 .NET Core 3.0 或 .NET Standard 2.1 的 C# 8

public static class StringExtensions
{
    public static string FirstCharToUpper(this string input) =>
        input switch
        {
            null => throw new ArgumentNullException(nameof(input)),
            "" => throw new ArgumentException($"{nameof(input)} cannot be empty", nameof(input)),
            _ => string.Concat(input[0].ToString().ToUpper(), input.AsSpan(1))
        };
}

由于 .NET Core 3.0 / .NET Standard 2.1String.Concat()支持ReadonlySpan<char>如果我们使用.AsSpan(1)而不是 .NET可以节省一次分配.Substring(1)。

＃8

public static class StringExtensions
{
    public static string FirstCharToUpper(this string input) =>
        input switch
        {
            null => throw new ArgumentNullException(nameof(input)),
            "" => throw new ArgumentException($"{nameof(input)} cannot be empty", nameof(input)),

C# 7

public static class StringExtensions
{
    public static string FirstCharToUpper(this string input)
    {
        switch (input)
        {
            case null: throw new ArgumentNullException(nameof(input));
            case "": throw new ArgumentException($"{nameof(input)} cannot be empty", nameof(input));
            default: return input[0].ToString().ToUpper() + input.Substring(1);
        }
    }
}

真的很老的答案

public static string FirstCharToUpper(string input)

    if (String.IsNullOrEmpty(input))
        throw new ArgumentException("ARGH!");

这个版本比较短。要获得更快的解决方案，请查看Diego 的回答。

public static string FirstCharToUpper(string input)
{

        throw new ArgumentException("ARGH!");
    return input.First().ToString().ToUpper() + input.Substring(1);
}

可能最快的解决方案是Darren 的（甚至有一个基准），尽管我会更改它的string.IsNullOrEmpty(s)验证以抛出异常，因为最初的要求期望第一个字母存在以便它可以大写。请注意，此代码适用于通用字符串，而不是特别适用于Textbox.


更多建议

public string FirstLetterToUpper(string str)
{
    if (str == null)
        return null;

    if (str.Length > 1)
        return char.ToUpper(str[0]) + str.Substring(1);

    return str.ToUpper();
}

旧答案：这使得每个首字母大写

public string ToTitleCase(string str)
{
    return CultureInfo.CurrentCulture.TextInfo.ToTitleCase(str.ToLower());
}

更多建议

Git 目标文件已损坏（正如其他答案中所指出的那样）。这可能发生在机器崩溃等情况下。

我有同样的事情。在阅读了此处的其他热门答案后，我找到了使用以下命令修复损坏的 Git 存储库的最快方法（在包含该文件夹的 Git 工作目录中执行.git）：

（请务必先备份您的 Git 存储库文件夹！）

find .git/objects/ -type f -empty | xargs rm
git fetch -p
git fsck --full

这将首先删除所有导致整个存储库损坏的 空对象文件，然后从远程存储库中 获取丢失的对象 （以及最新更改），然后进行完整的 对象存储检查 。哪一个，在这一点上，应该成功而没有任何错误（虽然可能仍然有一些警告！）

附言。这个答案表明您在某处（例如在 GitHub 上）有 Git 存储库的远程副本，而损坏的存储库是与仍然完好无损的远程存储库相关联的本地存储库。如果不是这种情况，请不要尝试按照我推荐的方式修复它。

检查另一个字符串中是否存在多个字符串

Arya

2023-02-18 00:35

如何检查数组中的任何字符串是否存在于另一个字符串中？

例如：

a = ['a', 'b', 'c']
s = "a123"
if a in s:
    print("some of the strings found in s")
else:
    print("no strings found in s")

如何替换该if a in s:行以获得适当的结果？



解答

您可以使用 any ：

a_string = "A string is more than its parts!"
matches = ["more", "wholesome", "milk"]

if any([x in a_string for x in matches]):

类似地，要检查是否找到了列表中的 所有 字符串，请使用 all 而不是any。


更多建议

any()是迄今为止最好的方法，如果你想要的只是Trueor False，但如果你想具体知道哪些字符串/字符串匹配，你可以使用一些东西。

如果您想要第一场比赛（False默认情况下）：

match = next((x for x in a if x in str), False)

如果您想获得所有匹配项（包括重复项）：

matches = [x for x in a if x in str]

如果您想获得所有非重复匹配项（无视顺序）：

matches = {x for x in a if x in str}

如果您想以正确的顺序获取所有非重复匹配项：

matches = []
for x in a:
    if x in str and x not in matches:
        matches.append(x)
