如何在 Android XML 可绘制文件中定义圆形？

Arya

2023-02-17 12:35

我在查找 Android XML 中形状定义的文档时遇到了一些问题。我想在 XML 文件中定义一个用纯色填充的简单圆圈，以将其包含到我的布局文件中。

遗憾的是，android.com 上的文档没有涵盖 Shape 类的 XML 属性。我想我应该使用ArcShape来绘制一个圆，但是没有关于如何设置大小、颜色或从圆弧中制作圆所需的角度的说明。



解答

这是一个简单的圆形，作为 Android 中的可绘制对象。

<?xml version="1.0" encoding="utf-8"?>
<shape
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="oval">

   <solid 
       android:color="#666666"/>

   <size 
       android:width="120dp"
        android:height="120dp"/>
</shape>


更多建议

将此设置为您的视图背景

<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="oval">
    <stroke
        android:width="1dp"
        android:color="#78d9ff"/>
</shape>

对于实心圆使用：

<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="oval">
    <solid
        android:color="#48b3ff"/>
</shape>

实心描边：

<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="oval">
    <solid android:color="#199fff"/>
    <stroke
        android:width="2dp"
        android:color="#444444"/>
</shape>

注意 ：要使oval形状显示为圆形，在这些示例中，您使用此形状作为其背景的视图应该是正方形，或者您必须将形状标签的height和属性设置为相等的值。width

是否可以在 Git 中移动/重命名文件并维护它们的历史记录？

Arya

2023-02-17 12:34

我想重命名/移动 Git 中的项目子树，将其从

/project/xyz

到

/components/xyz

如果我使用普通的git mv project components，那么所有的提交历史都会xyz project丢失。有没有办法移动它以保持历史？



解答

Git 检测重命名而不是将操作与提交持久化，因此无论您使用git mv与否都mv没有关系。

该log命令采用--follow在重命名操作之前继续历史记录的参数，即，它使用试探法搜索相似的内容。

要查找完整的历史记录，请使用以下命令：

git log --follow ./path/to/file

返回对象的 ECMAScript 6 箭头函数

Arya

2023-02-17 12:33

当从箭头函数返回一个对象时，由于语法上的歧义，似乎有必要使用一个额外的集合{}和一个关键字。return

这意味着我不能写p => {foo: "bar"}，但必须写p => { return {foo: "bar"}; }。

如果箭头函数返回对象以外的任何东西，则 和{}是return不必要的，例如：p => "foo".

p => {foo: "bar"}回报undefined。

修改后的p => {"foo": "bar"}抛出 “SyntaxError：意外标记：' :'”。

有什么明显的我想念的吗？



解答

您必须将返回的对象文字包装到括号中。否则花括号将被视为表示函数的主体。以下作品：

p => ({ foo: 'bar' });

您不需要将任何其他表达式括在括号中：

p => 10;
p => 'foo';
p => true;
p => [1,2,3];
p => null;
p => /^foo$/;

等等。

参考：MDN - 返回对象字面量


更多建议

您可能想知道为什么语法有效（但未按预期工作）：

var func = p => { foo: "bar" }

这是因为JavaScript 的标签语法：

所以如果你将上面的代码转换为 ES5，它应该是这样的：

var func = function (p) {
  foo:
  "bar"; //obviously no return here!
}
