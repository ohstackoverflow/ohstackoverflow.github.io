.gitignore 被 Git 忽略

Arya

2023-02-17 00:38

.gitignoreGit 似乎忽略了我的文件 -.gitignore文件可能已损坏？Git 期望哪种文件格式、语言环境或文化？

我的.gitignore：

# This is a comment
debug.log
nbproject/

输出自git status：

# On branch master
# Your branch is ahead of 'origin/master' by 1 commit.
#
# Untracked files:
#   (use "git add <file>..." to include in what will be committed)
#
#       debug.log
#       nbproject/
nothing added to commit but untracked files present (use "git add" to track)

我不想debug.log出现nbproject/在未跟踪的文件列表中。

我应该从哪里着手解决这个问题？



解答

固定的。好的，我在 Windows 上的记事本中创建了 .gitignore 文件，但它无法正常工作。当我在 Linux 上查看 .gitignore 文件时，它看起来像是有组织的乱码 - 也许记事本写出了 Unicode 而不是 ASCII 或任何 8 位。

所以我在我的 Linux 机器上重写了这个文件，当我把它拉回 Windows 时它工作正常！欢呼！


更多建议

如果 Git 似乎没有注意到您对.gitignore文件所做的更改，您可能需要检查以下几点：

  • 可能有一个全局.gitignore文件可能会干扰您的本地文件

  • 当你在 .gitignore 文件中添加一些东西时，试试这个：

    git add [uncommitted changes you want to keep] && git commit

    git rm -r --cached . git add . git commit -m "fixed untracked files"

  • 如果你从 .gitignore 文件中删除了一些东西，并且上面的步骤可能不起作用，如果你发现上面的步骤不起作用，试试这个 ：

    git add -f [files you want to track again]

    git commit -m "Refresh removing files from .gitignore file."

    // For example, if you want the .java type file to be tracked again, // The command should be: // git add -f *.java

在 JavaScript 中停止 setInterval 调用

Arya

2023-02-17 00:38

我setInterval(fname, 10000);在 JavaScript 中每 10 秒调用一个函数。是否可以在某些事件中停止调用它？

我希望用户能够停止重复刷新数据。



解答

setInterval()返回一个间隔 ID，您可以将其传递给clearInterval()：

var refreshIntervalId = setInterval(fname, 10000);

/* later */
clearInterval(refreshIntervalId);

setInterval()请参阅和 的文档clearInterval()。


更多建议

如果将 的返回值设置setInterval为变量，则可以使用clearInterval来停止它。

var myTimer = setInterval(...);
clearInterval(myTimer);

抽象方法和虚方法有什么区别？

Arya

2023-02-17 00:37

抽象方法和虚方法有什么区别？在哪些情况下建议使用抽象方法或虚拟方法？哪一个是最好的方法？



解答

抽象函数不能具有功能。 您基本上是在说，任何子类都必须提供他们自己的此方法版本，但是它太笼统了，甚至无法尝试在父类中实现。

一个虚函数 ，基本上是说看，这里的功能对于子类来说可能不够好，也可能不够好。所以如果足够好，就使用这个方法，如果不够好，那就覆盖我，并提供你自己的功能。


更多建议

抽象函数没有实现，只能在抽象类上声明。这会强制派生类提供实现。

虚函数提供默认实现，它可以存在于抽象类或非抽象类中。

例如：

public abstract class myBase
{
    //If you derive from this class you must implement this method. notice we have no method body here either
    public abstract void YouMustImplement();

    //If you derive from this class you can change the behavior but are not required to
    public virtual void YouCanOverride()
    { 
    }
}

public class MyBase
{
   //This will not compile because you cannot have an abstract method in a non-abstract class
    public abstract void YouMustImplement();
}
