如何在 JavaScript 中触发事件？

Arya

2023-02-17 19:55

我已使用 将事件附加到文本框addEventListener。它工作正常。当我想从另一个函数以编程方式触发事件时，我的问题就出现了。

我该怎么做？



解答

注意：initEvent方法现已弃用。其他答案具有最新和推荐的做法。

您可以在 IE 8或更低版本上使用 fireEvent，在 大多数 其他浏览器上使用 W3C 的dispatchEvent 。要创建要触发的事件，您可以使用或取决于浏览器。 **** __createEvent``createEventObject

这是一段不言自明的代码（来自原型），它dataavailable在 上触发一个事件element：

var event; // The custom event that will be created
if(document.createEvent){
    event = document.createEvent("HTMLEvents");
    event.initEvent("dataavailable", true, true);
    event.eventName = "dataavailable";
    element.dispatchEvent(event);
} else {
    event = document.createEventObject();
    event.eventName = "dataavailable";
    event.eventType = "dataavailable";
    element.fireEvent("on" + event.eventType, event);
}

// Dispatch/Trigger/Fire the event
document.dispatchEvent(event);

在子域和域之间共享 cookie

Arya

我有两个问题。我知道如果我在 cookie 中将域指定为.example.com（带前导点），则所有子域都可以共享一个 cookie。

可以访问在（没有子域）subdomain.example.com中创建的 cookie ？example.com``www

如果在中创建，可以example.com（没有www子域）访问 cookie吗subdomain.example.com？



解答

如果你像这样设置一个 cookie：

Set-Cookie: name=value

那么 cookie 将 只适用于请求域 ，并且只会发送到完全相同域的请求，而不是任何其他子域。（请参阅什么是仅主机 cookie？）

如果标头中存在属性，则两个不同的域（例如example.comand subdomain.example.com，或sub1.example.comand sub2.example.com）只能共享 cookie ：domain

Set-Cookie: name=value; domain=example.com

域属性必须“域匹配”请求 URL 才能有效，这基本上意味着它必须是请求域或超级域。因此，这适用于问题中的两个示例，以及两个单独的子域之间的共享。

然后，此 cookie 将针对的任何子域 example.com 发送，包括嵌套的子域，如subsub.subdomain.example.com. （请记住，还有其他属性可以限制 cookie 的范围以及浏览器发送它的时间，例如path或Secure）。

由于域匹配的工作方式，如果您想要sub1.example.com并sub2.example.com共享 cookie，那么您也可以将它们共享给sub3.example.com.

也可以看看：

  • www 与 no-www 和 cookie
  • setcookie.net：一个你可以尝试的网站（免责声明：由我开发，针对这个问题）

关于属性中前导点的注释domain：在早期的RFC 2109中，只有带有前导点 ( domain=.example.com) 的域可以跨子域使用。但这无法与顶级域共享，因此您所要求的在旧规范中是不可能的。

但是，较新的规范RFC 6265会忽略任何前导点，这意味着您可以在子域和顶级域上使用 cookie。


更多建议

请大家注意，您可以从域的子域设置 cookie。

（在请求的响应中发送subdomain.example.com）

Set-Cookie: name=value; Domain=example.com // GOOD

但是您 _ 不能_ 在子域上设置来自域的 cookie。

（在请求的响应中发送example.com）

Set-Cookie: name=value; Domain=subdomain.example.com // Browser rejects cookie

为什么？

根据规范，RFC 6265 第 5.3.6 节存储模型，

如果规范化请求主机与域属性不 匹配 ：完全忽略 cookie 并中止这些步骤。

和RFC 6265 第 5.1.3 节域匹配，

域匹配

如果至少满足以下条件之一，则字符串域匹配给定的域字符串：

  1. 域字符串和字符串是相同的。（请注意，此时域字符串和字符串都已规范化为小写。）

  2. 以下所有条件都成立：

    • The domain string is a suffix of the string.

    • The last character of the string that is not included in the domain string is a %x2E (".") character.

    • The string is a host name (i.e., not an IP address).

从 Flask 视图返回 JSON 响应

Arya

2023-02-17 19:54

我有一个函数可以使用 Pandas 分析 CSV 文件并生成包含摘要信息的字典。我想将结果作为 Flask 视图的响应返回。如何返回 JSON 响应？

@app.route("/summary")
def summary():
    d = make_summary()
    # send it back as json



解答

视图可以直接返回 Python dict 或 list，Flask 会jsonify自动调用。

@app.route("/summary")
def summary():
    d = make_summary()
    return d

对于旧的 Flask 版本，或者要返回不同的 JSON 可序列化对象，导入并使用jsonify.

from flask import jsonify

@app.route("/summary")
def summary():
    d = make_summary()
    return jsonify(d)


更多建议

jsonify序列化您传递给 JSON 的数据。如果您想自己序列化数据，请通过使用和jsonify构建响应来执行操作。status=200``mimetype='application/json'

from flask import json

@app.route('/summary')
def summary():
    data = make_summary()
    response = app.response_class(
        response=json.dumps(data),
        status=200,
        mimetype='application/json'
    )
    return response
