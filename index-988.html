PUT、DELETE、HEAD 等方法在大多数 Web 浏览器中都可用吗？

Arya

2023-02-17 21:58

我在这里看到了几个问题，比如How to debug RESTful services，其中提到：

不幸的是，同一个浏览器不允许我测试 HTTP PUT、DELETE，甚至在某种程度上什至是 HTTP POST。

我还听说浏览器仅支持 GET 和 POST，来自其他一些来源，例如：

  • http://www.packetizer.com/ws/rest.html
  • http://www.mail-archive.com/jmeter-user@jakarta.apache.org/msg13518.html
  • http://www.xml.com/cs/user/view/cs_msg/1098

然而，Firefox 中的一些快速测试表明发送PUT和DELETE请求按预期工作——XMLHttpRequest成功完成，并且请求以正确的方法显示在服务器日志中。我是否缺少某些方面，例如跨浏览器兼容性或不明显的限制？



解答

不。HTML 5 规范提到：

method 和formmethod 内容属性是具有以下关键字和状态的枚举属性：

关键字 get ，映射到状态 GET，表示 HTTP GET 方法。GET 方法应该只请求和检索数据，不应有其他作用。

关键字 post ，映射到状态 POST，表示 HTTP POST 方法。POST 方法请求服务器接受提交的表单数据进行处理，这可能导致将项目添加到数据库、创建新网页资源、更新现有页面或所有上述结果.

关键字 dialog ，映射到状态对话框，表示提交表单的目的是关闭表单所在的对话框（如果有），否则不提交。

这些属性的无效值默认是 GET 状态

即 HTML 表单仅支持 GET 和 POST 作为 HTTP 请求方法。一个解决方法是通过 POST 使用隐藏的表单字段通过 POST 隧道其他方法，该字段由服务器读取并相应地分派请求。

然而， GET 、 POST 、 PUT 和 DELETE 被 所有主要网络浏览器（IE、Firefox、Safari、Chrome、Opera）中的 XMLHttpRequest 实现（即 AJAX 调用）所支持。


更多建议

HTML 表单支持 GET 和 POST。（HTML5 曾一度添加了 PUT/DELETE，但这些都被删除了。）

XMLHttpRequest支持所有方法，包括 CHICKEN，尽管一些方法名称不区分大小写（每个 HTTP 方法都是区分大小写的）并且出于安全原因根本不支持一些方法名称（例如 CONNECT）。

Fetch API还支持除 CONNECT、TRACE 和 TRACK 之外的任何方法，这些方法出于安全原因被禁止。

浏览器正在慢慢收敛于 XMLHttpRequest 指定的规则，但正如其他评论所指出的那样，仍然存在一些差异。

“echo -n” 打印“-n”

Arya

2023-02-17 21:58

echo我的脚本有问题：

echo -n "Some string..."

印刷

-n Some string...

并移动到下一行。在控制台中，它在没有换行符的情况下正常工作：

Some string...



解答

该命令有多个版本echo，具有不同的行为。显然，用于脚本的 shell 使用了无法识别-n.

该printf命令具有更加一致的行为。echo适用于像 之类的简单事物echo hello，但我建议使用它printf来处理更复杂的事物。

你在什么系统上，你的脚本使用什么 shell？


更多建议

bash有一个名为“echo”的“内置”命令：

$ type echo
echo is a shell builtin

此外，还有一个“echo”命令是一个正确的可执行文件（即 shell forks 和 execs /bin/echo，而不是解释echo和执行它）：

$ ls -l /bin/echo
-rwxr-xr-x 1 root root 22856 Jul 21  2011 /bin/echo

echoeither的关于\cand的行为各-n不相同。你最好的选择是使用printf，它在我看过的四种不同的 *NIX 风格上可用：

$ printf "a line without trailing linefeed"
$ printf "a line with trailing linefeed\n"

related_name 有什么用？

Arya

2023-02-17 21:57

related_name对 onManyToManyField和fields 有用的参数是什么ForeignKey？例如，给定以下代码，效果是什么related_name='maps'？

class Map(db.Model):
    members = models.ManyToManyField(User, related_name='maps',
                                     verbose_name=_('members'))



解答

该related_name属性指定从模型返回到您的模型的反向关系的名称User。

如果您不指定 a related_name，Django 会使用您的模型名称自动创建一个后缀_set，例如User.map_set.all()。

如果你 确实 指定了，例如related_name=maps在User模型上，User.map_set仍然可以工作，但User.maps.语法显然更清晰，更不笨重；因此，例如，如果您有一个用户对象current_user，您可以使用它来获取与 有关系的模型current_user.maps.all()的所有实例。Map``current_user

Django 文档有更多详细信息。


更多建议

要添加到现有答案 - 相关名称是必须的，以防模型中有 2 个 FK 指向同一个表。例如在物料清单的情况下

@with_author 
class BOM(models.Model): 
    name = models.CharField(max_length=200,null=True, blank=True)
    description = models.TextField(null=True, blank=True)
    tomaterial =  models.ForeignKey(Material, related_name = 'tomaterial')
    frommaterial =  models.ForeignKey(Material, related_name = 'frommaterial')
    creation_time = models.DateTimeField(auto_now_add=True, blank=True)
    quantity = models.DecimalField(max_digits=19, decimal_places=10)

因此，当您必须访问此数据时，您只能使用相关名称

 bom = material.tomaterial.all().order_by('-creation_time')

否则它不起作用（至少在同一张表有 2 个 FK 的情况下，我无法跳过相关名称的使用。）
