如何修复 Git 分离头？

Arya

2023-02-16 23:18

我在我的存储库中做一些工作并注意到一个文件有本地更改。我不再需要它们，所以我删除了文件，以为我可以签出一份新副本。我想做 Git 相当于

svn up .

使用git pull似乎没有用。一些随机搜索将我带到了一个有人推荐的网站

git checkout HEAD^ src/

（src是包含已删除文件的目录）。

现在我发现我有一个独立的头脑。 我不知道那是什么。我该如何撤消？



解答

分离的头意味着你不再在一个分支上，你已经检查了历史中的单个提交（在这种情况下是在 HEAD 之前的提交，即 HEAD^）。

如果你想 删除 与分离的 HEAD 关联的更改

您只需要签出您所在的分支机构，例如

git checkout master

下次你更改了文件并想将其恢复到索引中的状态时，不要先删除文件，只需执行

git checkout -- path/to/foo

这会将文件 foo 恢复到它在索引中的状态。

如果你想让 你 的更改与分离的 HEAD 相关联

  1. 运行git branch tmp- 这会将您的更改保存在一个名为tmp.
  2. 跑步git checkout master
  3. 如果您想将所做的更改合并到 中master，请git merge tmp从master分支运行。master运行后你应该在分支上git checkout master。


更多建议

如果您不想丢失已更改的文件，则可以推送它们。我已经在分离模式下提交了它们，之后你可以移动到一个临时分支以便稍后集成到 master 中。

git commit -m "....."
git branch my-temporary-work
git checkout master
git merge my-temporary-work

如何附加到文件？

Arya

2023-02-16 23:17

如何附加到文件而不是覆盖它？



解答

将模式设置open()为"a"（附加）而不是"w"（写入）：

with open("test.txt", "a") as myfile:
    myfile.write("appended text")

该文档列出了所有可用的模式。


更多建议

您需要通过将“a”或“ab”设置为模式来以追加模式打开文件。参见 open() 。

当您使用“a”模式打开时，写入位置将 始终 位于文件末尾（追加）。您可以使用“a+”打开以允许读取、向后查找和读取（但所有写入仍将位于文件末尾！）。

例子：

>>> with open('test1','wb') as f:
        f.write('test')
>>> with open('test1','ab') as f:
        f.write('koko')
>>> with open('test1','rb') as f:
        f.read()
'testkoko'

注意 ：使用 'a' 与使用 'w' 打开并查找到文件末尾不同 - 考虑如果另一个程序打开文件并在查找和写入之间开始写入可能会发生什么。在某些操作系统上，使用“a”打开文件可以保证所有后续写入都将自动附加到文件末尾（即使文件因其他写入而增长）。

关于“a”模式如何运行的更多细节（ 仅在 Linux 上测试 ）。即使您向后查找，每次写入都会附加到文件末尾：

>>> f = open('test','a+') # Not using 'with' just to simplify the example REPL session
>>> f.write('hi')
>>> f.seek(0)
>>> f.read()
'hi'
>>> f.seek(0)
>>> f.write('bye') # Will still append despite the seek(0)!
>>> f.seek(0)
>>> f.read()
'hibye'

事实上，fopen 联机帮助页指出：

以追加模式（a 作为模式的第一个字符）打开文件会导致对该流的所有后续写入操作发生在文件末尾，就像在调用之前一样：

fseek(stream, 0, SEEK_END);

旧的简化答案（不使用with）：

示例：（ 在真实程序中 用于with关闭文件- 请参阅文档）

>>> open("test","wb").write("test")
>>> open("test","a+b").write("koko")
>>> open("test","rb").read()
'testkoko'

我怎样才能一次删除我所有的 Git 存储？

Arya

2023-02-16 23:16

我怎样才能一次删除我所有的Git存储？

具体来说，我的意思是，输入一个命令。



解答

以下命令删除您所有的藏匿处：

git stash clear

来自git 文档：

clear

删除所有隐藏状态。

重要警告： 这些状态将被修剪，并且可能无法恢复（...）。


更多建议

此命令使您能够查看所有隐藏的更改：

git stash list

以下是使用它清除所有隐藏更改的命令：

git stash clear

现在，如果你想从隐藏区域中删除一个隐藏的更改：

git stash drop stash@{index} # Index will be shown after getting stash list

注意：git stash list使您能够从 git 的存储区获取索引。
