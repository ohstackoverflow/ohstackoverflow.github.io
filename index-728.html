Python2 中的 dict.items() 和 dict.iteritems() 有什么区别？

Arya

2023-02-17 14:33

dict.items()和之间有任何适用的区别吗dict.iteritems()？

来自Python 文档：

dict.items()：返回字典的（键，值）对列表的副本 。

dict.iteritems()：在字典的（键，值）对上返回一个 迭代器。

如果我运行下面的代码，每个代码似乎都返回对同一对象的引用。有什么我遗漏的细微差别吗？

#!/usr/bin/python
d={1:'one',2:'two',3:'three'}
print 'd.items():'
for k,v in d.items():
   if d[k] is v: print '\tthey are the same object'
   else: print '\tthey are different'
print 'd.iteritems():'   
for k,v in d.iteritems():
   if d[k] is v: print '\tthey are the same object'
   else: print '\tthey are different'

输出：

d.items():
    they are the same object
    they are the same object
    they are the same object
d.iteritems():
    they are the same object
    they are the same object
    they are the same object



解答

这是进化的一部分。

最初，Pythonitems()构建了一个真正的元组列表并返回了它。这可能会占用大量额外内存。

然后，一般将生成器引入该语言，并将该方法重新实现为名为 的迭代器生成器方法iteritems()。原来的保留是为了向后兼容。

Python 3 的变化之一是items()现在返回视图，并且list永远不会完全构建。该iteritems()方法也消失了，因为items()在 Python 3 中的工作方式与viewitems()在 Python 2.7 中一样。


更多建议

dict.items()返回二元组列表 ( [(key, value), (key, value), ...])，而dict.iteritems()是生成二元组的生成器。前者最初需要更多的空间和时间，但访问每个元素的速度很快，而第二个最初需要的空间和时间更少，但生成每个元素的时间要多一些。

如何获取 NumPy 数组中 N 个最大值的索引？

Arya

2023-02-17 14:32

NumPy 提出了一种通过 获取数组最大值索引的方法np.argmax。

N我想要类似的东西，但返回最大值的索引。

例如，如果我有一个数组，[1, 3, 2, 4, 5]，那么将返回对应于元素的nargmax(array, n=3)索引。[4, 3, 1]``[5, 4, 3]



解答

argpartition较新的 NumPy 版本（1.8 及更高版本）有一个为此调用的函数。要获得四个最大元素的索引，请执行

>>> a = np.array([9, 4, 4, 3, 3, 9, 0, 4, 6, 0])
>>> a
array([9, 4, 4, 3, 3, 9, 0, 4, 6, 0])

>>> ind = np.argpartition(a, -4)[-4:]
>>> ind
array([1, 5, 8, 0])

>>> top4 = a[ind]
>>> top4
array([4, 9, 6, 9])

与 不同argsort，此函数在最坏情况下以线性时间运行，但返回的索引未排序，从评估结果可以看出a[ind]。如果您也需要，请在之后对它们进行排序：

>>> ind[np.argsort(a[ind])]
array([1, 8, 5, 0])

以这种方式获得排好序的前 k 个 元素需要 O( n + k log k ) 时间。


更多建议

我能想到的最简单的是：

>>> import numpy as np
>>> arr = np.array([1, 3, 2, 4, 5])
>>> arr.argsort()[-3:][::-1]
array([4, 3, 1])

这就涉及到对数组的完整排序。我想知道是否numpy提供了一种内置的方法来进行部分排序；到目前为止我还没有找到一个。

如果这个解决方案被证明太慢（尤其是对于 small n），可能值得考虑在Cython中编写一些东西。
