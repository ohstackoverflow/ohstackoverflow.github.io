在正则表达式的上下文中，“懒惰”和“贪婪”是什么意思？

Arya

2023-02-17 15:02

这两个术语是什么意思？



解答

贪婪会尽可能多地消费。从http://www.regular-expressions.info/repeat.html我们看到了尝试将 HTML 标签与<.+>. 假设您有以下内容：

<em>Hello World</em>

你可能认为<.+>(.表示 任何非换行符 ，+表示 一个或多个 ) 只会匹配 the<em>和 the </em>，而实际上它会非常贪婪，从第一个<到最后一个>。这意味着它将匹配<em>Hello World</em>而不是你想要的。

让它变得懒惰 ( <.+?>) 将防止这种情况。通过?在 之后添加 ，我们告诉它 尽可能少地+重复，所以它遇到的第一个就是我们要停止匹配的地方。 __>

我鼓励您下载RegExr，这是一个很棒的工具，可以帮助您探索正则表达式——我一直都在使用它。


更多建议

'Greedy' 表示匹配最长的字符串。

'Lazy' 表示匹配最短的字符串。

例如，贪心h.+l匹配'hell'但'hello'懒惰h.+?l匹配'hel'。

我在 3D 空间中有两点：

a = (ax, ay, az)
b = (bx, by, bz)

我想计算它们之间的距离：

dist = sqrt((ax-bx)^2 + (ay-by)^2 + (az-bz)^2)

我如何使用 NumPy 执行此操作？我有：

import numpy
a = numpy.array((ax, ay, az))
b = numpy.array((bx, by, bz))



解答

使用numpy.linalg.norm：

dist = numpy.linalg.norm(a-b)

这是有效的，因为 欧氏距离 是 l2 范数 ord ，并且参数的默认值为numpy.linalg.norm2。有关更多理论，请参阅 数据挖掘简介 ：

Cygwin 和 MinGW 有什么区别？

Arya

2023-02-17 15:01

我想让我的 C++ 项目跨平台，我正在考虑使用 Cygwin/MinGW。但是它们之间有什么区别呢？

另一个问题是我是否能够在没有 Cygwin/MinGW 的系统上运行二进制文件？



解答

作为简化，它是这样的：

  • 在 Cygwin 中编译一些东西，你正在 为 Cygwin 编译它。

  • 在 MinGW 中编译一些东西，你正在 为 Windows 编译它。

什么是Cygwin？

Cygwin 是一个兼容层，通过模拟基于 Unix 的操作系统提供的许多基本接口，例如管道、Unix 风格的文件和目录访问等，可以很容易地将简单的基于 Unix 的应用程序移植到 Windows由POSIX标准记录。Cygwin 还捆绑了 GNU Compiler Collection 的一个端口和 Cygwin 环境的一些其他工具。

如果您有使用 POSIX 接口的现有源代码，您可以在进行很少甚至没有更改后将其编译为与 Cygwin 一起使用，从而大大简化移植基于简单 IO 的 Unix 代码以在 Windows 上使用的过程。

Cygwin 的缺点

使用 Cygwin 进行编译涉及将您的程序与 Cygwin 运行时环境链接，该环境通常作为动态链接库与您的程序一起分发cygwin1.dll。这个库是开源的，需要使用它的软件共享一个兼容的开源许可证，即使你单独分发 dll，因为头文件和接口都包含在内。因此，这对您如何许可您的代码施加了一些限制。

什么是 MinGW？

MinGW 是用于本机 Windows 的 GNU 编译器工具的发行版，包括 GNU Compiler Collection、GNU Binutils 和 GNU Debugger。还包括允许开发本机 Windows 应用程序的头文件和库。因此，这将作为Microsoft Visual C++套件的开源替代品。

有可能使用 MinGW 编译一些最初打算用 Microsoft Visual C++ 进行编译的东西，并进行相对较小的修改。

默认情况下，在 MinGW 的 GCC 中编译的代码将编译为本机 Windows 目标，包括 .exe 和 .dll 文件，但您也可以使用正确的设置进行交叉编译，因为您基本上使用的是 GNU 编译器工具套件。

尽管 MingW 包含一些头文件和接口代码，允许您的代码与 Windows API 交互，但与常规标准库一样，这不会对您创建的软件施加许可限制。

MinGW 的缺点

使用 MinGW 为 Windows 编译的软件必须使用 Windows 自己的 API 进行文件和 IO 访问。如果您要将 Unix/Linux 应用程序移植到 Windows，这可能意味着对代码进行重大更改，因为不能再使用 POSIX 类型的 API。

其他注意事项

对于任何重要的软件应用程序，例如使用图形界面、多媒体或访问系统设备的软件应用程序，您都超出了 Cygwin 可以为您做的事情的界限，并且需要进一步的工作来使您的代码跨平台。但是，可以通过使用跨平台工具包或框架来简化此任务，这些工具包或框架允许编码一次并使您的代码在任何平台上都能成功编译。如果你从一开始就使用这样的框架，你不仅可以减少移植到另一个平台时的头痛，而且你可以在所有平台上使用相同的图形小部件——窗口、菜单和控件——如果你正在编写一个GUI 应用程序，并让它们对用户来说是原生的。

例如，开源Qt 框架是一种流行且全面的跨平台开发框架，允许构建跨操作系统（包括 Windows）工作的图形应用程序。还有其他这样的框架。除了大型框架之外，还有数以千计的更专业的软件库，它们支持多种平台，让您不必担心为不同平台编写不同代码。

当您从头开始开发跨平台软件时，通常没有任何理由使用 Cygwin。在 Windows 上编译时，您的目标通常是使您的代码能够使用 MingW 或 Microsoft Visual C/C++ 或两者进行编译。在 Linux/*nix 上编译时，您通常会直接使用 GNU 编译器和工具进行编译。


更多建议

Cygwin 是在 Windows 上创建一个完整的 UNIX/POSIX 环境的尝试。为此，它使用各种 DLL。虽然这些 DLL 包含在 GPLv3+ 中，但它们的许可证包含一个例外，不会强制派生作品包含在 GPLv3+ 中。MinGW 是一个 C/C++ 编译器套件，它允许您在不依赖此类 DLL 的情况下创建 Windows 可执行文件——您只需要正常的 MSVC 运行时，它们是任何正常 Microsoft Windows 安装的一部分。

您还可以获得一个小型的 UNIX/POSIX 类环境，使用名为MSYS的 MinGW 编译。它几乎没有 Cygwin 的所有功能，但对于想要使用 MinGW 的程序员来说是理想的选择。
