无法加载身份验证插件“caching_sha2_password”

Arya

2023-02-17 20:38

我将 MySQL - 8.0 与 MySQL Workbench 连接起来并收到以下错误：

无法加载身份验证插件“caching_sha2_password”：dlopen（/usr/local/mysql/lib/plugin/caching_sha2_password.so，2）：找不到图像

我也尝试过使用其他客户端工具。

有什么解决办法吗？



解答

您可以通过使用以下 Alter 命令更改用户来更改用户密码的加密：

ALTER USER 'username'@'ip_address' IDENTIFIED WITH mysql_native_password BY 'password';

或者

我们可以通过使用旧密码插件来避免这个错误：

首先更改 my.cnf 文件中的身份验证插件 for Linux / my.ini 文件在 Windows 中 ：

[mysqld]

default_authentication_plugin=mysql_native_password

重新启动 mysql 服务器以使更改生效并尝试通过 MySQL 与任何 mysql 客户端连接。

如果仍然无法连接并出现以下错误：

Unable to load plugin 'caching_sha2_password'

这意味着您的用户需要上述插件。因此，尝试在更改默认插件后使用 create user 或 grant 命令创建新用户。然后新用户需要本机插件，你就可以连接 MySQL。

谢谢

解答

它如何成为关键字和类型的实例？

这并不奇怪。和true都是false关键字，作为文字，它们具有类型 ( bool)。nullptr是一个类型为 的 指针文字std::nullptr_t，它是一个纯右值（您不能使用 获取它的地址&）。

  • 4.10关于指针转换说类型的纯右值std::nullptr_t是空指针常量，并且整数空指针常量可以转换为std::nullptr_t. 相反的方向是不允许的。这允许为指针和整数重载函数，并传递nullptr以选择指针版本。通过NULL或0会混淆选择int版本。

  • nullptr_t对整数类型的转换需要一个 , 并且与对整数类型reinterpret_cast的转换具有相同的语义（映射实现已定义）。(void*)0Areinterpret_cast不能转换nullptr_t为任何指针类型。如果可能，依赖隐式转换或使用static_cast.

  • 标准要求sizeof(nullptr_t)是sizeof(void*).

为什么在 C++11 中使用 nullptr？它是什么？为什么 NULL 不够？

C++ 专家Alex Allain 在这里说得很完美（我的强调以粗体显示）：

...假设您有以下两个函数声明：

void func(int n);
void func(char *s);

func( NULL ); // guess which function gets called?

虽然看起来会调用第二个函数——毕竟，你传递的似乎是一个指针——但实际上是第一个函数被调用！ 麻烦的是，因为 NULL 是 0，而 0 是一个整数，所以将调用 func 的第一个版本。是的，这种事情不会一直发生，但一旦发生，就会非常令人沮丧和困惑。如果您不知道正在发生的事情的详细信息，它很可能看起来像一个编译器错误。 看起来像编译器错误的语言功能并不是您想要的。

输入 nullptr。 在 C++11 中，nullptr 是一个新关键字，可以（并且应该！）用于表示 NULL 指针；换句话说，无论您之前在何处编写 NULL，都应该改用 nullptr。 程序员，你并不清楚 ，（每个人都知道 NULL 是什么意思）， 但它对编译器来说更明确 ，编译器将不再看到到处都是 0 被用作指针时具有特殊含义。

Allain 以这样的方式结束了他的文章：

不管这一切——C++11 的经验法则是只要nullptr你NULL过去会使用，就开始使用。
