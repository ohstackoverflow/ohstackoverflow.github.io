如何判断一个数是不是2的幂

Arya

2023-02-17 17:32

今天我需要一个简单的算法来检查一个数是否是 2 的幂。

该算法需要是：

  1. 简单的
  2. 正确的任何ulong值。

我想出了这个简单的算法：

private bool IsPowerOfTwo(ulong number)
{
    if (number == 0)
        return false;

    for (ulong power = 1; power > 0; power = power << 1)
    {
        // This for loop used shifting for powers of 2, meaning
        // that the value will become 0 after the last shift
        // (from binary 1000...0000 to 0000...0000) then, the 'for'
        // loop will break out.

        if (power == number)
            return true;
        if (power > number)
            return false;
    }
    return false;
}

但后来我想：如何检查 log 2 x 是否恰好是一个整数？当我检查 2^63+1 时，Math.Log()由于四舍五入而恰好返回 63。所以我检查了 2 的 63 次方是否等于原始数字，它是，因为计算是在doubles 中完成的，而不是精确的数字。

private bool IsPowerOfTwo_2(ulong number)
{
    double log = Math.Log(number, 2);
    double pow = Math.Pow(2, Math.Round(log));
    return pow == number;
}

这返回了true给定的错误值：9223372036854775809。

有更好的算法吗？



解答

这个问题有一个简单的技巧：

bool IsPowerOfTwo(ulong x)
{
    return (x & (x - 1)) == 0;
}

请注意，此函数将报告true，0这不是 的幂2。如果你想排除它，方法如下：

bool IsPowerOfTwo(ulong x)
{
    return (x != 0) && ((x & (x - 1)) == 0);
}

解释

首先是 MSDN 定义中的按位二进制 & 运算符：

二元 & 运算符是为整数类型和 bool 预定义的。对于整数类型，& 计算其操作数的逻辑按位与。对于 bool 操作数， & 计算其操作数的逻辑与；也就是说，当且仅当它的两个操作数都为真时，结果才为真。

现在让我们来看看这一切是如何发生的：

该函数返回布尔值 (true / false) 并接受一个 unsigned long 类型的传入参数（在本例中为 x）。为了简单起见，让我们假设有人传递了值 4 并像这样调用函数：

bool b = IsPowerOfTwo(4)

现在我们将每次出现的 x 替换为 4：

return (4 != 0) && ((4 & (4-1)) == 0);

好吧，我们已经知道 4 != 0 评估为真，到目前为止一切顺利。但是关于：

((4 & (4-1)) == 0)

这当然转化为：

((4 & 3) == 0)

但究竟是什么4&3？

4 的二进制表示是 100，3 的二进制表示是 011（记住 & 采用这些数字的二进制表示）。所以我们有：

100 = 4
011 = 3

想象一下，这些值就像初等加法一样叠加起来。运算&符表示如果两个值都等于 1，则结果为 1，否则为 0。因此1 & 1 = 1、1 & 0 = 0、0 & 0 = 0和0 & 1 = 0。所以我们算一下：

100
011
----
000

结果只是 0。所以我们回头看看我们的 return 语句现在翻译成什么：

return (4 != 0) && ((4 & 3) == 0);

现在翻译成：

return true && (0 == 0);



return true && true;

我们都知道这true && true很简单true，这表明对于我们的示例，4 是 2 的幂。

纠正那个问题。

身份验证与授权

Arya

2023-02-17 17:31

Web 应用程序的上下文有何不同？我经常看到缩写“auth”。它代表 授权 还是 授权 ？或者两者兼而有之？



解答

身份验证 是确定某人确实是他们声称的人的过程。

授权 是指确定允许谁做什么的规则。例如，Adam 可能被授权创建和删除数据库，而 Usama 仅被授权读取。

这两个概念是完全正交和独立的，但都是安全设计的核心，如果 其中 任何一个都不正确，就会打开妥协的途径。

就web应用而言，粗略地说，身份验证是当你检查登录凭据以查看你是否识别用户已登录，而授权是当你在你的访问控制中查找你是否允许用户查看、编辑、删除或创建内容。


更多建议

简而言之，请。:-)

身份验证 = 登录名 + 密码 （你是谁）

授权 = 权限 （允许你做什么）

简短的“auth”最有可能指代第一个或两者。

如何将所有 git 提交合并为一个？

Arya

2023-02-17 17:31

您如何将整个存储库压缩到第一次提交？

我可以 rebase 到第一个提交，但这会让我有 2 个提交。有没有办法在第一个提交之前引用提交？



解答

Perhaps the easiest way is to just create a new repository with current state of the working copy. If you want to keep all the commit messages you could first do git log > original.log and then edit that for your initial commit message in the new repository:

rm -rf .git
git init
git add .
git commit

or

git log > original.log
# edit original.log as desired
rm -rf .git
git init
git add .
git commit -F original.log


更多建议

一个班轮

您可以直接从创建一个 squash-all 提交HEAD， 根本不需要变基 ，只需运行：

git reset $(git commit-tree HEAD^{tree} -m "A new start")

在中创建别名~/.gitconfig 

[alias]
  squash-all = "!f(){ git reset $(git commit-tree HEAD^{tree} \"$@\");};f"

然后运行： git squash-all -m "a brand new start"

注意：从 标准输入 或通过 -m /选项提供提交消息 -F ， 就像 git commit 命令一样 。请参阅git-commit-tree 手册。

或者，您可以使用以下命令创建别名：

git config --global alias.squash-all '!f(){ git reset $(git commit-tree HEAD^{tree} "$@");};f'

解释

  1. 通过创建单个提交git commit-tree

什么git commit-tree HEAD^{tree} -m "A new start"是：

根据提供的树对象创建一个新的提交对象，并在标准输出上发出新的提交对象 ID。除非给出 -m 或 -F 选项，否则从标准输入读取日志消息。

表达式HEAD^{tree}代表 对应的树对象HEAD，即你当前分支的顶端。请参阅树对象和提交对象。

  1. 将当前分支重置为新提交

然后git reset简单地将当前分支重置为新创建的提交对象。

这样，工作区中的任何内容都不会被触及，也不需要变基/压缩，这使得它非常快。所需时间与存储库大小或历史深度无关。

变体：来自项目模板的新回购

这对于使用另一个存储库作为模板/原型/种子/骨架在新项目中创建“初始提交”很有用。例如：

cd my-new-project
git init
git fetch --depth=1 -n https://github.com/toolbear/panda.git
git reset --hard $(git commit-tree FETCH_HEAD^{tree} -m "initial commit")

这避免了将模板仓库添加为远程（origin或其他方式）并将模板仓库的历史折叠到您的初始提交中。
