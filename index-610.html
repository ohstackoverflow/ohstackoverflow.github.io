从远程服务器拉取某个分支

Arya

2023-02-17 11:24

假设有人创建了一个分支xyz。如何xyz从远程服务器（例如GitHubxyz ）拉取分支并将其合并到本地存储库中的现有分支？

将分支推送到 Git 的 答案给我错误“！[拒绝]”并提到“非快进”。



解答

但是我得到一个错误“！[rejected]”和一些关于“非快进”的东西

那是因为 Git 无法将分支中的更改合并到您当前的母版中。假设您已经签出 branch master，并且想要合并到远程 branch other- branch。当你这样做时：

$ git pull origin other-branch

Git 基本上是这样做的：

$ git fetch origin other-branch && git merge other-branch

也就是说， apull只是 afetch后跟 a merge。但是，当pull-ing 时，Git 只有 在可以执行 快进合并other-branch 时 才会合并。 快进 合并是一种合并，其中您尝试合并到的分支的头部是您要合并的分支头部的 直接后代。 例如，如果您有这个历史树，那么合并将导致快进合并： __other-branch

O-O-O-O-O-O
^         ^
master    other-branch

但是，这不是 快 进合并：

    v master
O-O-O
\
 \-O-O-O-O
         ^ other-branch

要解决您的问题，请先 获取 远程分支：

$ git fetch origin other-branch

然后将其合并到您当前的分支（我假设是master），并修复任何合并冲突：

$ git merge origin/other-branch
# Fix merge conflicts, if they occur
# Add merge conflict fixes
$ git commit    # And commit the merge!


更多建议

只需明确地跟踪您的远程分支，一个简单的git pull将做您想要的：

git branch -f remote_branch_name origin/remote_branch_name
git checkout remote_branch_name

后者是本地操作。

或者更适合GitHub 上关于分叉的文档：

git branch -f new_local_branch_name upstream/remote_branch_name

撤消导致合并冲突的 git stash pop

Arya

2023-02-17 11:24

我开始更改我的代码库，却没有意识到我在一个旧主题分支上。为了转移它们，我想把它们藏起来，然后将它们应用到 master 的一个新分支。我曾经git stash pop将正在进行的更改转移到这个新分支，忘记了在创建新分支之前我没有将新更改拉入 master。这导致了一堆合并冲突，并丢失了我的更改的干净存储（因为我使用了 pop）。

一旦我正确地重新创建了新分支，我怎样才能恢复我隐藏的更改以正确应用它们？



解答

事实证明，Git 足够聪明，不会在应用不干净时丢弃存储。我能够通过以下步骤达到所需的状态：

  1. 要解除合并冲突：（git reset HEAD .注意尾随点）
  2. 保存冲突的合并（以防万一）：git stash
  3. 回到主人：git checkout master
  4. 要拉取最新的更改：git fetch upstream; git merge upstream/master
  5. 要更正我的新分支：git checkout new-branch; git rebase master
  6. 要应用正确的隐藏更改（现在位于堆栈的第二位）：git stash apply stash@{1}


更多建议

幸运的是， 在git stash pop发生冲突的情况下不会更改存储！ ****

所以不用担心，只需清理代码并重试即可。

假设你的代码库之前是干净的，你可以通过以下方式回到那个状态：git checkout -f
​​ 然后做你忘记的事情，例如git merge missing-branch
在那之后再次触发git stash pop你得到 相同的 存储，之前发生冲突。

请记住：存储是安全的，但是，工作目录中 未提交的 更改当然不是。他们可能会搞砸。

如何删除 Redis 中的所有内容？

Arya

2023-02-17 11:23

我想删除所有密钥。我想要清除所有内容并给我一个空白数据库。

有没有办法在Redis客户端中执行此操作？



解答

使用 redis-cli：

  • FLUSHDB – 从连接的当前数据库中删除所有键。
  • FLUSHALL – 从所有数据库中删除所有键。

例如，在您的外壳中：

redis-cli flushall


更多建议

抬头，这FLUSHALL可能是矫枉过正。FLUSHDB是一个只刷新数据库的。FLUSHALL将消灭整个服务器。与服务器上的每个数据库一样。由于问题是关于刷新数据库的，我认为这是一个足够重要的区别，值得单独回答。
