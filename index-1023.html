GitHub“致命：远程源已经存在”

Arya

2023-02-17 23:06

我正在尝试按照Michael Hartl 的 Rails 教程进行操作，但遇到了一个错误。

我在 GitHub 上注册，发布了一个新的 SSH 密钥，并创建了一个新的存储库。但是当我在终端中输入下一行时，出现以下错误：

Parkers-MacBook-Pro:.ssh ppreyer$ git remote add origin git@github.com:ppreyer/first_app.git
fatal: remote origin already exists.

只是想知道是否还有其他人遇到过这个问题？



解答

简洁版本：

您应该只更新现有的遥控器：

$ git remote set-url origin git@github.com:ppreyer/first_app.git

长版：

如错误消息所示，已经有一个配置了相同名称的远程。因此，您可以添加具有不同名称的新遥控器，或者在不需要时更新现有遥控器。

要添加一个新的遥控器，例如调用github而不是origin（显然已经存在于您的系统中），请执行以下操作：

$ git remote add github git@github.com:ppreyer/first_app.git

不过请记住，在本教程的任何地方，您都可以看到 “origin” ，您应该将其替换为 “github” 。例如$ git push origin master现在应该是$ git push github master。

但是，如果您想查看已经存在的origin遥控器是什么，您可以执行$ git remote -v. 如果您认为这是错误的，您可以像这样更新它：

$ git remote set-url origin git@github.com:ppreyer/first_app.git


更多建议

这有效：

git remote rm origin
git remote add origin git@github.com:username/myapp.git

我可以向现有静态类添加扩展方法吗？

Arya

2023-02-17 23:06

我是 C# 中扩展方法的粉丝，但在向静态类（例如Console.

例如，如果我想向 中添加一个Console名为“ WriteBlueLine”的扩展名，这样我就可以：

Console.WriteBlueLine("This text is blue");

我通过添加一个本地公共静态方法来尝试此操作，并将其Console作为“ this”参数……但没有骰子！

public static class Helpers {
    public static void WriteBlueLine(this Console c, string text)
    {
        Console.ForegroundColor = ConsoleColor.Blue;
        Console.WriteLine(text);
        Console.ResetColor();
    }
}

这没有添加一个 ' WriteBlueLine' 方法到Console......我做错了吗？还是要求不可能的事情？



解答

否。扩展方法需要对象的实例变量（值）。但是，您可以围绕ConfigurationManager接口编写一个静态包装器。如果您实现包装器，则不需要扩展方法，因为您可以直接添加该方法。

 public static class ConfigurationManagerWrapper
 {
      public static ConfigurationSection GetSection( string name )
      {
         return ConfigurationManager.GetSection( name );
      }

      .....

      public static ConfigurationSection GetWidgetSection()
      {
          return GetSection( "widgets" );
      }
 }


更多建议

可以向 C# 中的类添加静态扩展吗？不，但你可以这样做：

public static class Extensions
{
    public static T Create<T>(this T @this)
        where T : class, new()
    {
        return Utility<T>.Create();
    }
}

public static class Utility<T>
    where T : class, new()
{
    static Utility()
    {
        Create = Expression.Lambda<Func<T>>(Expression.New(typeof(T).GetConstructor(Type.EmptyTypes))).Compile();
    }
    public static Func<T> Create { get; private set; }
}

这是它的工作原理。虽然您不能在技术上编写静态扩展方法，但此代码却利用了扩展方法中的漏洞。该漏洞是您可以在空对象上调用扩展方法而不会出现空异常（除非您通过@this 访问任何内容）。

所以这里是你将如何使用它：

    var ds1 = (null as DataSet).Create(); // as oppose to DataSet.Create()
    // or
    DataSet ds2 = null;
    ds2 = ds2.Create();

    // using some of the techniques above you could have this:
    (null as Console).WriteBlueLine(...); // as oppose to Console.WriteBlueLine(...)

现在为什么我选择调用默认构造函数作为示例，为什么我不在第一个代码片段中返回 new T() 而不执行所有这些 Expression 垃圾？那么今天是你的幸运日，因为你得到了 2fer。任何高级 .NET 开发人员都知道，new T() 很慢，因为它生成对 System.Activator 的调用，后者在调用它之前使用反射获取默认构造函数。该死的微软！但是我的代码直接调用对象的默认构造函数。

静态扩展会比这更好，但绝望的时代需要采取绝望的措施。

^M 字符在 Vim 中是什么意思？

Arya

2023-02-17 23:05

我一直在获取^M角色.vimrc，但它破坏了我的配置。



解答

Unix 使用 0xA 作为换行符。Windows 使用两个字符的组合：0xD 0xA。0xD 是回车符。^M恰好是vim显示0xD的方式（0x0D = 13，M是英文字母表中的第13个字母）。

您可以^M通过运行以下命令删除所有字符：

:%s/^M//g

Where^M是通过按住Ctrl并键入v，m然后再松开来输入的Ctrl。有时缩写为^V^M，但请注意，您必须按照上一句中的说明输入它，而不是按字面意思键入。

此表达式将^M用空字符串（即无）替换所有出现的位置。我用它来删除^M从 Windows 复制到 Unix（Solaris、Linux、OSX）的文件。


更多建议

:%s/\r//g
