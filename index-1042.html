如何访问 Flask 路由中的查询字符串？

Arya

2023-02-17 23:50

如何访问 Flask 路由中的查询参数或查询字符串？从 Flask 文档中看并不明显。

下面的示例路由/data说明了我想要访问该数据的上下文。如果有人请求类似的东西example.com/data?abc=123，我想访问字符串?abc=123或能够检索参数的值，例如abc。

@app.route("/data")
def data():
    # query_string = ???
    return render_template("data.html")



解答

from flask import request

@app.route('/data')
def data():
    # here we want to get the value of user (i.e. ?user=some-value)
    user = request.args.get('user')


更多建议

完整的 URL 可用request.url，查询字符串可用request.query_string.decode()。

这是一个例子：

from flask import request

@app.route('/adhoc_test/')
def adhoc_test():

    return request.query_string

要访问在查询字符串中传递的单个已知参数，您可以使用request.args.get('param'). 据我所知，这是“正确”的做法。

ETA：在继续之前，您应该问问自己为什么需要查询字符串。我从来不需要拉入原始字符串——Flask 具有以抽象方式访问它的机制。除非有令人信服的理由不这样做，否则您应该使用它们。

您应该将 .gitignore 提交到 Git 存储库中吗？

Arya

2023-02-17 23:48

.gitignore你认为提交到 Git 仓库是一个好习惯吗？

有些人不喜欢它，但我认为它很好，因为您可以跟踪文件的历史记录。不是吗？



解答

通常是的，.gitignore对每个想要使用存储库的人都有用。有时你会想忽略更多私人的东西（也许你经常创造LOG或其他东西。在那些情况下你可能不想把它强加给任何人。


更多建议

你通常 会 commit .gitignore。事实上，我个人会尽量确保我的索引在我不处理某事时始终保持干净。（git status应该什么都不显示。）

在某些情况下，您想要忽略确实不是项目特定的东西。例如，您的文本编辑器可能会创建自动*~备份文件，或者另一个示例是.DS_Store由 OS X 创建的文件。

我会说，如果其他人抱怨这些规则使您的 . 变得混乱.gitignore，请将它们排除在外，而不是将它们放在全局排除文件中。

默认情况下，此文件位于$XDG_CONFIG_HOME/git/ignore（默认为~/.config/git/ignore），但可以通过设置选项更改此位置core.excludesfile。例如：

git config --global core.excludesfile ~/.gitignore

只需根据您的喜好创建和编辑全局排除文件；它将应用于您在该机器上工作的每个 git 存储库。

JPA JoinColumn 与 mappedBy

Arya

2023-02-17 23:48

有什么区别：

@Entity
public class Company {

    @OneToMany(cascade = CascadeType.ALL , fetch = FetchType.LAZY)
    @JoinColumn(name = "companyIdRef", referencedColumnName = "companyId")
    private List<Branch> branches;
    ...
}

和

@Entity
public class Company {

    @OneToMany(cascade = CascadeType.ALL , fetch = FetchType.LAZY, 
    mappedBy = "companyIdRef")
    private List<Branch> branches;
    ...
}



解答

@JoinColumn 可以用在关系的双方。问题是关于@JoinColumn在@OneToMany侧面使用（罕见情况）。这里的要点是 物理信息重复 （列名）以及 未优化的 SQL 查询将产生一些额外的UPDATE语句。

根据文档：

由于在 JPA 规范中 多对一 （几乎）始终是双向关系的 所有者方 ，因此一对多关联被注释为@OneToMany(mappedBy=...)

@Entity
public class Troop {
    @OneToMany(mappedBy="troop")
    public Set<Soldier> getSoldiers() {
    ...
}

@Entity
public class Soldier {
    @ManyToOne
    @JoinColumn(name="troop_fk")
    public Troop getTroop() {
    ...
}

Troop``Soldier通过 troop 属性与双向一对多关系。您不必（不得）在侧面定义任何物理映射mappedBy。

要将双向一对多映射，将 一对多方作为拥有方 ，您必须删除该元素并将mappedBy多对一设置@JoinColumn为假。此解决方案未优化，会产生一些额外的语句。insertable``updatable``UPDATE

@Entity
public class Troop {
    @OneToMany
    @JoinColumn(name="troop_fk") //we need to duplicate the physical information
    public Set<Soldier> getSoldiers() {
    ...
}

@Entity
public class Soldier {
    @ManyToOne
    @JoinColumn(name="troop_fk", insertable=false, updatable=false)
    public Troop getTroop() {
    ...
}
