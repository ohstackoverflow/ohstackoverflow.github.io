使用 JavaScript 获取下拉列表中的选定值

Arya

2023-02-16 22:00

如何使用 JavaScript 从下拉列表中获取选定的值？

<form>
  <select id="ddlViewBy">
    <option value="1">test1</option>
    <option value="2" selected="selected">test2</option>
    <option value="3">test3</option>
  </select>
</form>

运行代码片段隐藏结果



解答

给定一个看起来像这样的选择元素：

<select id="ddlViewBy">
  <option value="1">test1</option>
  <option value="2" selected="selected">test2</option>
  <option value="3">test3</option>
</select>

运行这段代码：

var e = document.getElementById("ddlViewBy");
var value = e.value;
var text = e.options[e.selectedIndex].text;

结果是：

value == 2
text == "test2"

交互示例：

显示代码片段

var e = document.getElementById("ddlViewBy");
function onChange() {
  var value = e.value;
  var text = e.options[e.selectedIndex].text;
  console.log(value, text);
}
e.onchange = onChange;
onChange();


<form>
  <select id="ddlViewBy">
    <option value="1">test1</option>
    <option value="2" selected="selected">test2</option>
    <option value="3">test3</option>
  </select>
</form>


更多建议

纯 JavaScript：

var e = document.getElementById("elementId");
var value = e.options[e.selectedIndex].value;
var text = e.options[e.selectedIndex].text;

查询：

$("#elementId").val(); // The value of the selected option

  value: 'item_1_id',

JavaBean 究竟是什么？

Arya

2023-02-16 21:59

我认为，我理解“Bean”是一个具有属性和 getter/setter 的 Java 类。
据我了解，它相当于一个 Cstruct。真的吗？

另外， a和 regular之间是否存在真正的 句法 差异？是否有任何特殊定义或？JavaBean``class
Interface

基本上，为什么有这个术语？

另外Serializable接口是什么意思？



解答

JavaBean 只是一个标准的. 它是一个普通的 Java class，除了它遵循某些约定：

  1. 所有属性都是私有的（使用getters/setters）
  2. 公共无参数构造函数
  3. 工具Serializable。

就是这样。这只是一个约定。但是很多图书馆都依赖于它。

关于Serializable，来自API 文档：

类的可序列化性由实现 java.io.Serializable 接口的类启用。不实现此接口的类将不会对其任何状态进行序列化或反序列化。可序列化类的所有子类型本身都是可序列化的。序列化接口没有方法或字段，仅用于标识可序列化的语义。

换句话说，可序列化对象可以写入流，因此可以写入文件、对象数据库等任何东西。

此外，JavaBean 和另一个类之间没有句法差异——如果一个类遵循标准，它就是一个 JavaBean。

它有一个术语，因为该标准允许库以编程方式对您以预定义方式定义的类实例执行操作。例如，如果一个库想要流式传输您传递给它的任何对象，它知道它可以，因为您的对象是可序列化的（假设该库要求您的对象是适当的 JavaBeans）。


更多建议

有一个术语让它听起来很特别。现实远没有那么神秘。

基本上，一个“Bean”：

  • 是一个可序列化的对象（也就是说，它实现了java.io.Serializable，并且正确地实现了），即
  • 具有“属性”，其 getter 和 setter 只是具有特定名称的方法（例如，getFoo()是“Foo”属性的 getter），并且
  • 有一个公共的零参数构造函数（因此它可以随意创建并通过设置其属性进行配置）。

至于Serializable：这只不过是一个“标记接口”（一个不声明任何函数的接口），它告诉 Java 实现类同意（并暗示它能够）“序列化”——一个转换的过程一个实例到一个字节流中。这些字节可以存储在文件中，通过网络连接发送等，并且有足够的信息允许 JVM（至少，知道对象类型的）稍后重建对象——可能在不同的实例中应用程序，甚至在另一台机器上！

当然，为了做到这一点，类必须遵守某些限制。其中最主要的是所有实例字段必须是基本类型（int、bool 等）、某些也可序列化的类的实例，或者标记为这样transientJava 就不会尝试包含它们。（这当然意味着transient字段将无法通过流传输。具有transient字段的类应该准备好在必要时重新初始化它们。）

一个不能遵守这些限制的类不应该实现Serializable（而且，IIRC，Java 编译器甚至不会 让它 这样做。）

为什么 GCC 不将 a*a*a*a*a*a 优化为 (a*a*a)*(a*a*a)？

Arya

2023-02-16 21:58

我正在对科学应用程序进行一些数值优化。pow(a,2)我注意到的一件事是，GCC 会通过将调用编译成 来优化调用a*a，但调用pow(a,6)没有优化，实际上会调用库函数pow，这大大降低了性能。（相比之下，英特尔 C++ 编译器可执行文件icc将消除对库的调用pow(a,6)。）

我很好奇的是，当我替换pow(a,6)为a*a*a*a*a*a使用 GCC 4.5.1 和选项“ -O3 -lm -funroll-loops -msse4”时，它使用了 5mulsd条指令：

movapd  %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm14, %xmm13

而如果我写(a*a*a)*(a*a*a)，它会产生

movapd  %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm14, %xmm13
mulsd   %xmm13, %xmm13

它将乘法指令的数量减少到 3。icc具有类似的行为。

为什么编译器无法识别这种优化技巧？



解答

因为Floating Point Math 不是 Associative。在浮点乘法中对操作数进行分组的方式会影响答案的数值准确性。

因此，大多数编译器对重新排序浮点计算非常保守，除非他们可以确定答案将保持不变，或者除非你告诉他们你不关心数值准确性。例如：允许 gcc 重新关联浮点操作的 gcc选项-fassociative-math，甚至-ffast-math允许更积极地权衡准确性与速度的选项。


更多建议

Lambdageek 正确地指出，由于结合律不适用于浮点数，因此a*a*a*a*a*ato(a*a*a)*(a*a*a)可能会更改值。这就是 C99 不允许它的原因（除非用户通过编译器标志或 pragma 特别允许）。通常，假设程序员写下她所做的是有原因的，编译器应该尊重这一点。如果你愿意(a*a*a)*(a*a*a)，写那个。

不过，写起来可能很痛苦；为什么编译器不能在你使用时做[你认为的]正确的事情pow(a,6)？因为这样做是 错误的。 在具有良好数学库的平台上，pow(a,6)比a*a*a*a*a*aor更准确(a*a*a)*(a*a*a)。只是为了提供一些数据，我在我的 Mac Pro 上进行了一个小实验，测量在 [1,2] 之间的所有单精度浮点数计算 a^6 时的最严重错误：

worst relative error using    powf(a, 6.f): 5.96e-08
worst relative error using (a*a*a)*(a*a*a): 2.94e-07
worst relative error using     a*a*a*a*a*a: 2.58e-07

使用pow而不是乘法树将误差范围减少了 4 倍 。编译器不应该（并且通常不会）进行增加错误的“优化”，除非用户许可这样做（例如通过-ffast- math）。

请注意，GCC 提供了__builtin_powi(x,n)的替代方法pow( )，它应该生成内联乘法树。如果您想以准确性换取性能，但又不想启用快速数学，请使用它。
