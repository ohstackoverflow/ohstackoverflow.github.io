测试是否存在嵌套的 JavaScript 对象键

Arya

2023-02-17 11:23

如果我有一个对象的引用：

var test = {};

可能（但不是立即）具有嵌套对象，例如：

{level1: {level2: {level3: "level3"}}};

检查深层嵌套对象中属性是否存在的最佳方法是什么？

alert(test.level1);产量undefined，但alert(test.level1.level2.level3);失败了。

我目前正在做这样的事情：

if(test.level1 && test.level1.level2 && test.level1.level2.level3) {
    alert(test.level1.level2.level3);
}

但我想知道是否有更好的方法。



解答

如果您不想要 a ，则必须逐步执行此操作，TypeError因为如果其中一个成员是nullor undefined，并且您尝试访问成员，则会抛出异常。

您可以简单地catch排除异常，也可以创建一个函数来测试多个级别的存在，如下所示：

function checkNested(obj /*, level1, level2, ... levelN*/) {
  var args = Array.prototype.slice.call(arguments, 1);

  for (var i = 0; i < args.length; i++) {
    if (!obj || !obj.hasOwnProperty(args[i])) {
      return false;
    }
    obj = obj[args[i]];
  }
  return true;
}

var test = {level1:{level2:{level3:'level3'}} };

checkNested(test, 'level1', 'level2', 'level3'); // true
checkNested(test, 'level1', 'level2', 'foo'); // false

ES6 更新：

这是原始函数的较短版本，使用 ES6 功能和递归（它也是正确的尾调用形式）：

function checkNested(obj, level,  ...rest) {
  if (obj === undefined) return false
  if (rest.length == 0 && obj.hasOwnProperty(level)) return true
  return checkNested(obj[level], ...rest)
}

但是，如果您想获取嵌套属性的值而不仅仅是检查它是否存在，这里有一个简单的单行函数：

function getNested(obj, ...args) {
  return args.reduce((obj, level) => obj && obj[level], obj)
}

const test = { level1:{ level2:{ level3:'level3'} } };
console.log(getNested(test, 'level1', 'level2', 'level3')); // 'level3'
console.log(getNested(test, 'level1', 'level2', 'level3', 'length')); // 6
console.log(getNested(test, 'level1', 'level2', 'foo')); // undefined
console.log(getNested(test, 'a', 'b')); // undefined

上面的函数允许你获取嵌套属性的值，否则将返回undefined。

更新 2019-10-17：

可选链接提案在ECMAScript 委员会流程中达到了第 3 阶段，这将允许您通过使用令牌（?.新的 可选链接运算符） 安全地访问深层嵌套的属性：

const value = obj?.level1?.level2?.level3

如果访问的任何级别是null或undefined表达式将undefined自行解析为。

该提案还允许您安全地处理方法调用：

obj?.level1?.method();

上面的表达式会产生undefinedif obj, obj.level1, or obj.level1.methodare nullor undefined，否则会调用函数。

您可以使用可选的链接插件 开始使用 Babel 的此功能。

从Babel 7.8.0开始，默认支持 ES2020

在 Babel REPL 上检查这个例子。

🎉🎉更新：2019 年 12 月 🎉🎉

可选链提案最终在 TC39 委员会 2019 年 12 月的会议上进入了第 4 阶段。这意味着此功能将成为 ECMAScript 2020 标准的一部分。


更多建议

这是我从 Oliver Steele 那里得到的一个模式：

var level3 = (((test || {}).level1 || {}).level2 || {}).level3;
alert( level3 );

事实上，整篇文章都是在讨论如何在 javascript 中执行此操作。他决定使用上述语法（一旦你习惯了它就不难阅读）作为习惯用法。

为什么 python setup.py 在 Travis CI 上说无效命令 'bdist_wheel'？

Arya

2023-02-17 11:22

我的 Python 包有一个setup.py在 Ubuntu Trusty 和新的 Vagrant Ubuntu Trusty VM 上本地构建良好的，当我像这样配置它时：

sudo apt-get install python python-dev --force-yes --assume-yes --fix-broken
curl --silent --show-error --retry 5 https://bootstrap.pypa.io/get-pip.py | sudo python2.7
sudo -H pip install setuptools wheel virtualenv --upgrade

但是当我在 Travis CI Trusty Beta VM 上做同样的事情时：

- sudo apt-get install python python-dev --force-yes --assume-yes --fix-broken
- curl --silent --show-error --retry 5 https://bootstrap.pypa.io/get-pip.py | sudo python2.7
- sudo -H pip install setuptools wheel virtualenv --upgrade

我得到：

python2.7 setup.py bdist_wheel
usage: setup.py [global_opts] cmd1 [cmd1_opts] [cmd2 [cmd2_opts] ...]
   or: setup.py --help [cmd1 cmd2 ...]
   or: setup.py --help-commands
   or: setup.py cmd --help
error: invalid command 'bdist_wheel'

这为什么我不能在 python 中创建一个轮子？是相关的，但请注意我正在安装 wheel 并升级 setuptools。



解答

必须安装wheel软件包。一切都是最新的，但仍然出现错误。

pip install wheel

然后

python setup.py bdist_wheel

工作没有问题。


更多建议

在 AWS 上Ubuntu 18.04 new machine，需要以下安装：

sudo apt-get install gcc libpq-dev -y
sudo apt-get install python-dev  python-pip -y
sudo apt-get install python3-dev python3-pip python3-venv python3-wheel -y
pip3 install wheel

特别是最后一行是必须的。
但是，可能需要 3 行作为先决条件。

在我的 package.json 中，我写了"main": "dist/index.js".

在 Node.js 中一切正常，但 TypeScript：

import {Injector} from '@ts-stack/di';

找不到模块“@ts-stack/di”的声明文件。'/path/to/node_modules/@ts-stack/di/dist/index.js' 隐式具有 'any' 类型。

然而，如果我按如下方式导入，那么一切正常：

import {Injector} from '/path/to/node_modules/@ts-stack/di/dist/index.js';

我究竟做错了什么？



解答

对于安装自己的 npm 包的情况

如果您使用的是第三方软件包，请参阅下面的回答。

.js从中删除。"main": "dist/index.js"_package.json

"main": "dist/index",

还要根据 TypeScript 文档typings添加package.json：

"main": "dist/index",
"typings": "dist/index",

该文件夹dist是 TS 编译器存储模块文件的地方。


更多建议

'foo'如果您要导入不提供任何类型的第三方模块，无论是在库本身还是在@types/foo包中（从DefinitelyTyped存储库生成），那么您可以通过在带扩展名的文件.d.ts。.d.tsTypeScript在与查找普通文件相同的位置查找文件：.ts如tsconfig.json.

// foo.d.ts
declare module 'foo';

然后，当您导入时，foo它只会被输入为any.

或者，如果您想滚动自己的类型，您也可以这样做：

// foo.d.ts
declare module 'foo' {
    export function getRandomNumber(): number
}

然后这将正确编译：

import { getRandomNumber } from 'foo';
const x = getRandomNumber(); // x is inferred as number

您不必为模块提供完整的类型，只需为您实际使用的位提供足够的类型（并且需要正确的类型），因此如果您使用的 API 数量相当少，这就特别容易做到。

另一方面，如果您不关心外部库的类型并希望将所有没有类型的库导入为any，您可以将其添加到具有扩展名的文件中.d.ts：

declare module '*';

这样做的好处（和缺点）是你可以导入任何东西，TS 会编译。
