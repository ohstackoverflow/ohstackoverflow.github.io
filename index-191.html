如何配置 git 以忽略本地某些文件？

Arya

2023-02-17 00:28

我可以忽略本地文件而不污染其他人的全局 git 配置吗？我的 git 状态中有未跟踪的垃圾邮件文件，但我不想为本地分支中的每个随机未跟踪文件提交 git 配置更改。



解答

来自相关的 Git 文档：

特定于特定存储库但不需要与其他相关存储库共享的模式（例如，存在于存储库中但特定于一个用户的工作流的辅助文件）应该放入文件中$GIT_DIR/info/exclude。

该.git/info/exclude文件具有与任何文件相同的格式.gitignore。另一种选择是设置core.excludesFile为包含全局模式的文件的名称。

请注意，如果您已经有未暂存的更改，则必须在编辑忽略模式后运行以下命令：

git update-index --assume-unchanged <file-list>

_注意：这是一个$GIT_DIR_在整个git 手册中使用的符号，只是为了指示 git 存储库的路径。如果设置了环境变量，那么它将覆盖您所在的任何存储库的位置，这可能不是您想要的。

编辑 ：另一种方法是使用：

git update-index --skip-worktree <file-list>

通过以下方式反转：

git update-index --no-skip-worktree <file-list>


更多建议

更新 ：考虑改用git update-index --skip-worktree [<file>...]，谢谢@danShumway！参见Borealid 对这两个选项的区别的解释。

旧答案：

如果您需要忽略对跟踪文件的本地更改（我们有对配置文件的本地修改），请使用git update-index --assume-unchanged [<file>...].

如何将多个提交合并到另一个分支作为单个压缩提交？

Arya

2023-02-17 00:27

我有一个远程 Git 服务器，这是我要执行的场景：

  • 我为每个错误/功能创建一个不同的 Git 分支

  • 我继续使用非官方 Git 消息在那个 Git 分支中提交我的代码

  • 在顶级存储库中，我们必须使用官方 Git 消息对一个错误进行一次提交

那么我怎样才能将我的分支合并到远程分支，以便他们只为我的所有签入提交一次提交（我什至想为此提供提交消息）？



解答

假设你的错误修复分支被调用bugfix并且你想将它合并到master：

git checkout master
git merge --squash bugfix
git commit

这将从分支中获取所有提交bugfix，将它们压缩为 1 个提交，并将其与您的master分支合并。

解释 ：

git checkout master

切换到您的master分支。

git merge --squash bugfix

从分支中获取所有提交bugfix并将其分组为 1 提交与您当前的分支。
（没有出现合并提交；您可以在提交之前手动解决冲突）

git commit

从合并的更改创建单个提交。

省略该-m参数可让您修改草稿提交消息，其中包含在完成提交之前来自已压缩提交的每条消息。


更多建议

最终让我明白这一点的是一条评论：

git checkout main
git merge --squash feature

相当于做：

git checkout feature
git diff main > feature.patch
git checkout main
patch -p1 < feature.patch
git add .

当我想将一个功能分支与 105(!!) 提交合并并将它们全部压缩成一个时，我不想这样做，git rebase -i origin/master因为我需要分别解决 每个 中间提交（或至少是那些git 无法弄清楚自己）。使用git merge --squash让我得到我想要的结果，即合并整个功能分支的单个提交。而且，我最多只需要进行一次手动冲突解决。

针对存储的 Git diff

Arya

2023-02-17 00:27

我如何才能看到取消隐藏对当前工作树所做的更改？我想知道在应用它们之前会进行哪些更改！



解答

查看最近的存储：

git stash show -p

查看任意存储：

git stash show -p stash@{1}

从git stash联机帮助页：

默认情况下，该命令显示 diffstat，但它会接受 git diff 已知的任何格式（例如，git stash show -p stash@{1} 以补丁形式查看第二个最近的存储）。


更多建议

要查看最近的存储：

git stash show -p

要查看任意存储：

git stash show -p stash@{1}

此外，我使用 git diff 将存储与任何分支进行比较。

您可以使用：

git diff stash@{0} master

查看与分支主机相比的所有更改。

或者您可以使用：

git diff --name-only stash@{0} master

为了轻松找到仅更改的文件名。
