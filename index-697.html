我已经开发了 1 或 2 周的应用程序，就在昨天，我将我的 iPhone 5S 更新到了 iOS 8 GM。一切正常，我也可以在我的设备上进行测试，直到我从手机中删除该应用程序并想重新构建。出现以下错误：

Could not launch "My App"
process launch failed: Security

当我用模拟器测试时，它工作正常。这是因为 iOS 8 GM 更新吗？我该如何解决这个启动问题？我希望能够在我的 iPhone 和模拟器中进行测试。



解答

如果您收到此消息，则该应用程序已安装在您的设备上。你必须点击图标。它会询问您是否 真的 要运行它。说“ yes”，然后说Build & Run again。

从 开始iOS 9，需要去Settings→ General→ Device Management→ Developer App→ `Trust``。

在 某些 版本的上iOS，您必须转至Settings→ General→ Profile。


更多建议

Xcode 7 的更新答案 ：点击应用程序不再有效（从 beta 1 开始，它只显示一条“不受信任的企业开发人员”消息，只有一个关闭按钮）。

要修复，请打开“设置”应用，转到“常规”/“个人资料”，然后您会看到您的个人资料。将其标记为可信，一切应该会再次开始正常工作。

针对 iOS 9.2.1 和 Xcode 7.2.1 更新：

转到：设置 > 通用 > 设备管理 > 从开发者应用程序中选择应用程序 > 信任应用程序。

如果 PyPy 快 6.3 倍，为什么我不应该使用 PyPy 而不是 CPython？

Arya

2023-02-17 13:42

我听说过很多关于PyPy项目的信息。他们声称它比他们网站上的CPython解释器快 6.3 倍。

每当我们谈论像 Python 这样的动态语言时，速度都是最重要的问题之一。为了解决这个问题，他们说 PyPy 快了 6.3 倍。

第二个问题是并行性，臭名昭著的全局解释器锁（GIL）。为此，PyPy 表示它可以提供 GIL-less Python。

如果 PyPy 能够解决这些巨大的挑战，那么它阻碍更广泛采用的弱点是什么？也就是说，是什么阻止了像我这样的典型 Python 开发人员 现在 转向 PyPy ？



解答

注意： 与 2013 年提出此问题时相比，PyPy 现在更加成熟且得到了更好的支持。避免从过时的信息中得出结论。

  1. 正如其他人很快提到的那样，PyPy 对 C 扩展的支持很薄弱。它有支持，但通常速度比 Python 慢，而且它充其量是不确定的。因此，许多模块只需要 CPython。 PyPy 不支持 numpy 。一些扩展仍然不受支持（ Pandas 、 SciPy 等），在进行更改之前查看支持的包列表。 请注意，列表中标记为不受支持的许多包现在都受支持。
  2. Python 3 支持 目前处于试验阶段。 刚刚达到稳定！ 截至 2014 年 6 月 20 日，PyPy3 2.3.1 - Fulcrum 已发布！
  3. 对于很多人使用 Python 的“脚本”，PyPy 有时实际上并不快。这些是短期运行的程序，可以做一些简单而小的事情。因为 PyPy 是一个 JIT 编译器，所以它的主要优势来自运行时间长和类型简单（例如数字）。与 CPython 相比，PyPy 的预 JIT 速度可能很差。
  4. 惯性 。迁移到 PyPy 通常需要重组，这对某些人和组织来说简直就是太多的工作。

这些是影响我的主要原因，我会说。


更多建议

该站点并未 声称 PyPy 比 CPython 快 6.3 倍。去引用：

所有基准测试的几何平均值比 CPython 快 0.16 或 6.3 倍

这与您所做的一揽子声明 截然不同 ，当您了解其中的差异时，您至少会理解为什么不能只说“使用 PyPy”的一组原因。这听起来像是我在吹毛求疵，但理解为什么这两种说法完全不同是至关重要的。

打破它：

  • 他们所做的声明仅适用于他们使用的基准。它绝对没有说明您的程序（除非您的程序与他们的基准测试之一完全相同）。

  • 该声明是关于一组基准的 平均值。 没有人声称即使对于他们测试过的程序，运行 PyPy 也会带来 6.3 倍的改进。

  • 没有人声称 PyPy 甚至可以运行 CPython 运行的所有程序 ， 更不用说更快了。
