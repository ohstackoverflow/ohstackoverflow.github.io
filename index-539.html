外部 JAR 上的“转换为 Dalvik 格式失败，错误 1”

Arya

2023-02-17 09:34

在 Eclipse 中的 Android 应用程序中，出现以下错误。

意外的顶级异常：
java.lang.IllegalArgumentException：已添加：Lorg/xmlpull/v1/XmlPullParser；
....
转换为 Dalvik 格式失败，出现错误 1

只有当我将特定的外部 JAR 文件添加到我的项目时才会出现此错误。我搜索了很长时间以寻找可能的解决方案，但没有一个有效。

我什至尝试更改为 Android 1.6 而不是 1.5（我使用的当前版本）。



解答

我解决了这个问题。

这是 JAR 文件冲突。

似乎我的构建路径上有两个 JAR 文件，它们包含相同的包和类。

smack.jar和android_maps_lib-1.0.2

从其中一个 JAR 文件中删除这个包解决了这个问题。

如何更改分支正在跟踪的远程？

Arya

2023-02-17 09:33

存储central库必须在新服务器上设置，所以我在本地存储库上创建了一个新的远程服务器，并推送到那里。

但是现在当我这样做时git pull，它声称我是最新的。这是错误的——它告诉我 旧的 远程分支，而不是新的，我知道事实上有新的提交要获取。

如何更改我的本地分支机构以跟踪不同的远程分支机构？

我可以在 git 配置文件中看到这一点，但我不想把事情搞砸。

[branch "master"]
    remote = oldserver
    merge = refs/heads/master



解答

使用 git v1.8.0 或更高版本：

git branch branch_name --set-upstream-to your_new_remote/branch_name

或者你可以使用-u开关

git branch branch_name -u your_new_remote/branch_name

使用 git v1.7.12 或更早版本

git branch --set-upstream branch_name your_new_remote/branch_name


更多建议

对我来说，解决方法是：

git remote set-url origin https://some_url/some_repo

然后：

git push

这非常烦人，不得不评论、构建和取消注释，构建变得非常令人厌烦。



解答

我只是有同样的问题。Visual Studio 未生成正在引用的项目。

书面说明：

  1. 右键单击该解决方案，然后单击“属性”。
  2. 单击左侧的配置。
  3. 确保选中“构建”下找不到的项目的复选框。如果它已经被选中，取消选中，点击应用并再次选中复选框。
  4. （可选）您必须对解决方案属性的发布和调试模式执行此操作。

更多建议

这在较新版本的 Visual Studio 中仍然会发生（我刚刚在 Visual Studio 2013 上发生过）：

要尝试的另一件事是关闭 Visual Studio 并删除.suo该文件旁边的文件.sln。（它将在您下次（或退出 Visual Studio）时重新生成Save all）。

我在另一台机器上向解决方案添加新项目然后拉入修订时遇到了这个问题，但文件.suo在其他情况下也可能损坏并导致非常奇怪的 Visual Studio 行为，因此删除它是其中之一我总是尝试的事情。

请注意，删除.suo文件将重置解决方案的启动项目。
