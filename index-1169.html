'$(this)' 和 'this' 有什么区别？

Arya

2023-02-18 03:07

我目前正在学习本教程：jQuery 入门

对于下面的两个例子：

$("#orderedlist").find("li").each(function (i) {
    $(this).append(" BAM! " + i);
});
$("#reset").click(function () {
    $("form").each(function () {
        this.reset();
    });
});

请注意，在第一个示例中，我们使用$(this)在每个元素内附加一些文本li。在第二个例子中我们this在重置表单时直接使用。

$(this)似乎比 . 更经常被使用this。

我的猜测是在第一个示例中，$()将每个li元素转换为一个理解函数的 jQuery 对象append()，而在第二个示例中reset()可以直接在表单上调用。

基本上我们需要$()特殊的 jQuery-only 函数。

它是否正确？



解答

$()是的，只有在使用 jQuery 时才需要。如果你想要 jQuery 的帮助来做 DOM 的事情，请记住这一点。

$(this)[0] === this

基本上每次您返回一组元素时，jQuery 都会将其转换为jQuery 对象。如果您知道只有一个结果，那么它将位于第一个元素中。

$("#myDiv")[0] === document.getElementById("myDiv");

等等...


更多建议

$()是 jQuery 构造函数。

this是对调用的 DOM 元素的引用。

所以基本上，在 中$(this)，您只是将thisin$()作为参数传递，这样您就可以调用 jQuery 方法和函数。

JQuery - $ 未定义

Arya

2023-02-18 03:06

我有一个简单的 jquery 点击事件

<script type="text/javascript">
    $(function() {
        $('#post').click(function() {
            alert("test"); 
        });
    });
</script>

和 site.master 中定义的 jquery 引用

<script src="<%=ResolveUrl("~/Scripts/jquery-1.3.2.js")%>" type="text/javascript"></script>

我已经检查过脚本是否被正确解析，我能够看到标记并直接在萤火虫中查看脚本，所以我一定被发现了。但是，我仍然得到：

$ 未定义

并且 jquery 都不起作用。$(document).ready我也尝试过 this like和 jQuery 等的各种变体。

这是 .net 3.5 上的 MVC 2 应用程序，我确定我真的很密集，谷歌上到处都说检查文件是否正确引用，我已经检查并再次检查，请指教！：/



解答

该错误只能由以下三种情况之一引起：

  1. 您的 JavaScript 文件未正确加载到您的页面中
  2. 你有一个拙劣的 jQuery 版本。这可能是因为有人编辑了核心文件，或者插件可能覆盖了 $ 变量。
  3. 在页面完全加载之前运行 JavaScript，因此，在 jQuery 完全加载之前。

首先，确保正确调用了什么脚本，它应该看起来像

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js" type="text/javascript"></script>

并且不应具有属性 async 或 defer 。

然后您应该检查Firebug 网络 面板以查看文件是否确实被正确加载。如果不是，它将以红色突出显示并在其旁边显示“404”。如果文件加载正确，则表示问题是 2。

确保所有 jQuery javascript 代码都在代码块内运行，例如：

$(document).ready(function () {
  //your code here
});

这将确保在jQuery 初始化 后 加载您的代码。

最后要检查的一件事是确保在加载 jQuery 之前没有加载任何插件。 插件扩展了“$”对象，所以如果你在加载 jQuery 核心之前加载一个插件，那么你会得到你描述的错误。

注意： 如果您正在加载不需要 jQuery 运行的代码，则不需要将其放置在 jQuery 就绪处理程序中。该代码可以使用document.readyState.


更多建议

可能是在调用 jquery 脚本之前调用了脚本标记。

<script type="text/javascript" src="js/script.js"></script>

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js" type="text/javascript"></script>

这导致 $ 未定义

将 jquery.js 放在你的脚本标签之前，它会起作用；）像这样：

<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js" type="text/javascript"></script>

<script type="text/javascript" src="js/script.js"></script>

将 UTF-8 编码的 NSData 转换为 NSString

Arya

2023-02-18 03:06

我有NSData从 Windows 服务器编码的 UTF-8，我想将它转换为NSStringiPhone。由于数据包含在两个平台上具有不同值的字符（如度数符号）， 如何将数据转换为字符串？



解答

如果数据不是空终止的，你应该使用-initWithData:encoding:

NSString* newStr = [[NSString alloc] initWithData:theData encoding:NSUTF8StringEncoding];

如果数据以 null 结尾，则应改用to 避免末尾的-stringWithUTF8String:额外内容。\0

NSString* newStr = [NSString stringWithUTF8String:[theData bytes]];

（请注意，如果输入未正确进行 UTF-8 编码，您将得到nil.）

斯威夫特变体：

let newStr = String(data: data, encoding: .utf8)
// note that `newStr` is a `String?`, not a `String`.

如果数据以 null 结尾，您可以通过安全的方式删除该 null 字符，或者使用类似于上面的 Objective-C 版本的不安全方式。

// safe way, provided data is \0-terminated
let newStr1 = String(data: data.subdata(in: 0 ..< data.count - 1), encoding: .utf8)
// unsafe way, provided data is \0-terminated
let newStr2 = data.withUnsafeBytes(String.init(utf8String:))


更多建议

你可以调用这个方法

+(id)stringWithUTF8String:(const char *)bytes.
