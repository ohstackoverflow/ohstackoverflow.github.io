为什么C++头文件中使用#ifndef和#define？

Arya

2023-02-17 18:09

我通常在头文件的开头看到这样的代码：

#ifndef HEADERFILE_H
#define HEADERFILE_H

在文件的末尾是

#endif

这样做的目的是什么？



解答

这些被称为#include 守卫。

包含标头后，它会检查是否HEADERFILE_H定义了唯一值（在本例中为 ）。然后，如果它没有被定义，它会定义它并继续到页面的其余部分。

当代码再次被包含时，第一次ifndef失败，导致一个空白文件。

这可以防止任何标识符的双重声明，例如类型、枚举和静态变量。


更多建议

#ifndef <token>
/* code */
#else
/* code to include if the token is defined */
#endif

#ifndef检查给定的令牌是否#defined在文件的前面或包含的文件中；如果没有，它包括它和结束语句之间的代码#else，如果没有#else，则包括#endif语句。#ifndef通常用于通过在包含文件后定义一个标记并检查该标记是否未设置在该文件的顶部来使头文件幂等。

#ifndef _INCL_GUARD
#define _INCL_GUARD
#endif

如何按索引从 std::vector<> 中删除元素？

Arya

2023-02-17 18:09

我有一个 std::vector，我想删除第 n 个元素。我怎么做？

std::vector<int> vec;

vec.push_back(6);
vec.push_back(-17);
vec.push_back(12);

vec.erase(???);



解答

要删除单个元素，您可以这样做：

std::vector<int> vec;

vec.push_back(6);
vec.push_back(-17);
vec.push_back(12);

// Deletes the second element (vec[1])
vec.erase(std::next(vec.begin()));

或者，一次删除多个元素：

// Deletes the second through third elements (vec[1], vec[2])
vec.erase(std::next(vec.begin(), 1), std::next(vec.begin(), 3));


更多建议

std::vector 上的 erase 方法重载了，所以调用它可能更清楚

vec.erase(vec.begin() + index);

当您只想删除一个元素时。

如何在 Objective-C 中测试字符串是否为空？

Arya

2023-02-17 18:08

在 Objective-C 中如何测试 an 是否NSString为空？



解答

你可以检查是否[string length] == 0。这将检查它是否是一个有效但空的字符串 (@"") 以及它是否为 nil，因为调用lengthnil 也会返回 0。


更多建议

马克的回答是正确的。但我会借此机会包括一个指向 Wil Shipley 的 generalized 的指针isEmpty，他在他的博客上分享了它：

static inline BOOL IsEmpty(id thing) {
return thing == nil
|| ([thing respondsToSelector:@selector(length)]
&& [(NSData *)thing length] == 0)
|| ([thing respondsToSelector:@selector(count)]
&& [(NSArray *)thing count] == 0);
}
