什么是聚合根？

Arya

2023-02-18 04:11

我试图了解如何正确使用存储库模式。聚合根的中心概念不断出现。当在网络和 Stack Overflow 上搜索关于什么是聚合根的帮助时，我一直在寻找关于它们的讨论以及指向本应包含基本定义的页面的死链接。

在存储库模式的上下文中， 什么是聚合根？



解答

在存储库模式的上下文中，聚合根是您的客户端代码从存储库加载的唯一对象。

存储库封装了对子对象的访问 - 从调用者的角度来看，它会自动加载它们，或者在加载根的同时或者在实际需要它们时（如延迟加载）。

例如，您可能有一个Order对象封装了对多个LineItem对象的操作。您的客户端代码永远不会LineItem直接加载对象，只会Order加载包含它们的对象，这将是您域的那部分的聚合根。


更多建议

来自埃文斯 DDD：

AGGREGATE 是关联对象的集群，为了数据更改，我们将其视为一个单元。每个 AGGREGATE 都有一个根和一个边界。边界定义了 AGGREGATE 内部的内容。根是包含在 AGGREGATE 中的单个特定实体。

和：

根是 AGGREGATE 中唯一允许外部对象持有对 [.] 的引用的成员

这意味着聚合根是唯一可以从存储库加载的对象。

Customer一个例子是包含实体和实体的模型Address。我们永远不会Address直接从模型访问实体，因为如果没有关联的上下文，它就没有意义Customer。所以我们可以说 和Customer一起Address形成一个聚合，这Customer是一个聚合根。

'/' 和 '//' 用于除法时有什么区别？

Arya

2023-02-18 04:11

使用一个比另一个有好处吗？在 Python 2 中，它们似乎都返回相同的结果：

>>> 6/3
2
>>> 6//3
2



解答

在 Python 3.x 中，5 / 2将返回2.5并将5 // 2返回2. 前者是浮点除法，后者是 _ 底除法 ，有时也叫 _ 整数除法 。

在 Python 2.2 或更高版本的 2.x 行中，整数没有区别，除非您执行 a from __future__ import division，这会导致 Python 2.x 采用 3.x 行为。

不管以后的导入，5.0 // 2都会返回2.0，因为那是地板分割运算的结果。

您可以在PEP 238：更改除法运算符 中找到详细说明。


更多建议

Python 2.x 说明：

为 Python 2.x 行澄清，/既不是楼层划分也不是真正的划分。

/ 当两个 args 都是floor division 时，但是当 _ 其中一个 argsint是 true division 时。 _ ****float

导入错误：没有模块名称 urllib2

Arya

2023-02-18 04:10

这是我的代码：

import urllib2.request

response = urllib2.urlopen("http://www.google.com")
html = response.read()
print(html)

有什么帮助吗？



解答

如urllib2文档中所述：

该urllib2模块已拆分为 Python 3 中的多个模块，名为urllib.request和urllib.error。在将您的源代码转换为 Python 3 时，该2to3工具将自动调整导入。

所以你应该说

from urllib.request import urlopen
html = urlopen("http://www.google.com/").read()
print(html)

您当前编辑的代码示例不正确，因为您说的urllib.urlopen("http://www.google.com/")不是urlopen("http://www.google.com/").


更多建议

对于使用 Python 2（测试版本 2.7.3 和 2.6.8）和 Python 3（3.2.3 和 3.3.2+）的脚本，请尝试：

#! /usr/bin/env python
try:
    # For Python 3.0 and later
    from urllib.request import urlopen
except ImportError:
    # Fall back to Python 2's urllib2
    from urllib2 import urlopen
html = urlopen("http://www.google.com/")
print(html.read())
