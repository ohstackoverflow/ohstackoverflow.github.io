如何在特定索引处将项目插入到数组中 (JavaScript)

Arya

2023-02-16 19:11

我正在寻找一种 JavaScript 数组插入方法，其样式如下：

arr.insert(index, item)

最好使用 jQuery，但此时任何 JavaScript 实现都可以。



解答

您需要 splice 本机数组对象上的函数。

arr.splice(index, 0, item);将在指定位置item插入（先删除项目，即它只是一个插入）。arr``index``0

在这个例子中，我们将创建一个数组并将一个元素添加到它的索引 2 中：

var arr = [];
arr[0] = "Jani";
arr[1] = "Hege";
arr[2] = "Stale";
arr[3] = "Kai Jim";
arr[4] = "Borge";

console.log(arr.join()); // Jani,Hege,Stale,Kai Jim,Borge
arr.splice(2, 0, "Lene");
console.log(arr.join()); // Jani,Hege,Lene,Stale,Kai Jim,Borge

运行代码片段隐藏结果


更多建议

Array.insert您可以通过这样做来实现该方法：

Array.prototype.insert = function ( index, ...items ) {
    this.splice( index, 0, ...items );
};

然后你可以像这样使用它：

var arr = [ 'A', 'B', 'E' ];
arr.insert(2, 'C', 'D');

// => arr == [ 'A', 'B', 'C', 'D', 'E' ]

如何从分支中删除提交？

Arya

2023-02-16 19:10

如何从我的分支历史记录中删除提交？我应该使用吗git reset --hard HEAD？



解答

注意： git reset --hard 将删除您的工作目录更改 。在运行此命令之前，请务必 存储要保留的任何本地更改 。

假设你正坐在那个提交上，那么这个命令就会搞砸它......

git reset --hard HEAD~1

意思HEAD~1是 head 之前的提交。

或者，您可以查看 的输出git log，找到要备份到的提交的提交 ID，然后执行以下操作：

git reset --hard <sha1-commit-id>

如果你已经推了它，你需要用力推才能摆脱它......

git push origin HEAD --force

但是 ，如果其他人可能已经取消了它，那么您最好开始一个新分支。因为当他们拉动时，它只会合并到他们的工作中，然后您会再次将其推回。

如果您已经推送，最好使用git revert, 创建一个“镜像”提交来撤消更改。但是，两次提交都将在日志中。

仅供参考 -git reset --hard HEAD如果您想摆脱正在进行的工作，那就太好了。它会将您重置为最近的提交，并清除工作树和索引中的所有更改。

最后，如果您需要找到您“删除”的提交，它通常会存在，git reflog除非您对存储库进行了垃圾回收。


更多建议

如果您尚未将提交推送到任何地方，则可以使用git rebase -i删除该提交。首先，找出该提交的时间（大约）。然后做：

git rebase -i HEAD~N

这~N意味着 rebase 最后一次N提交（N例如，必须是一个数字HEAD~10）。然后，您可以编辑 Git 提供给您的文件以删除有问题的提交。保存该文件后，Git 将重写以下所有提交，就好像您删除的那个不存在一样。

Git Book 中有一个很好的部分介绍了带有图片和示例的变基。

不过要小心，因为如果你改变了你在别处推送的东西 ， 除非你打算强制推送，否则将需要另一种方法。

使用“for”循环遍历字典

Arya

2023-02-16 19:10 

d = {'x': 1, 'y': 2, 'z': 3}

for key in d:
    print(key, 'corresponds to', d[key])

Python 是如何识别它只需要key从字典中读取的呢？是key一个特殊的关键字，还是仅仅是一个变量？



解答

key只是一个变量名。

for key in d:

将简单地遍历字典中的键，而不是键和值。要遍历键和值，您可以使用以下命令：

对于 Python 3.x：

for key, value in d.items():

对于 Python 2.x：

for key, value in d.iteritems():

要自己测试，请将单词更改key为poop。

在 Python 3.x 中，iteritems()被简单地替换items()，它返回一个由 dict 支持的类似集合的视图，类似iteritems()但更好。这在 2.7 中也可以作为viewitems().

该操作items()将适用于 2 和 3，但在 2 中它将返回字典对的列表(key, value)，这不会反映调用后发生的对字典的更改items()。如果你想要 3.x 中的 2.x 行为，你可以调用list(d.items()).


更多建议

并不是说 key 是一个特殊的词，而是字典实现了迭代器协议。您可以在您的课堂上执行此操作，例如，请参阅此问题了解如何构建类迭代器。

对于字典，它是在 C 级别实现的。详细信息可在PEP 234中找到。特别是标题为“字典迭代器”的部分：

  • 字典实现了一个 tp_iter 槽，它返回一个高效的迭代器，该迭代器迭代字典的键。[...] 这意味着我们可以写

    for k in dict: ...

这相当于，但比

>     for k in dict.keys(): ...

只要不违反对字典修改的限制（通过循环或其他线程）。

  • 将方法添加到显式返回不同类型迭代器的字典中：

    for key in dict.iterkeys(): ...

    for value in dict.itervalues(): ...

    for key, value in dict.iteritems(): ...

这意味着这for x in dict是简写for x in dict.iterkeys()。

在 Python 3 中，dict.iterkeys()不再支持 ,dict.itervalues()和。dict.iteritems()使用dict.keys(),dict.values()而dict.items()不是。
