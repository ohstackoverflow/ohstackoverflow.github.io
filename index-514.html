“cat << EOF”在 bash 中如何工作？

Arya

2023-02-17 08:55

我需要编写一个脚本来向程序输入多行输入 ( psql)。

经过一番谷歌搜索后，我发现以下语法有效：

cat << EOF | psql ---params
BEGIN;

`pg_dump ----something`

update table .... statement ...;

END;
EOF

这正确地构造了多行字符串（从BEGIN;到END;，包括在内）并将其作为输入通过管道传输到psql.

但是我不知道它是如何/为什么起作用的，有人可以解释一下吗？

我主要指的是cat << EOF，我知道>输出到文件，>>附加到文件，<从文件读取输入。

究竟是<<做什么的？

有它的手册页吗？



解答

这称为 heredoc 格式 ，用于将字符串提供到标准输入中。有关详细信息，请参阅https://en.wikipedia.org/wiki/Here_document#Unix_shells 。

来自man bash:

这里文件

这种类型的重定向指示 shell 从当前源读取输入，直到看到仅包含单词（没有尾随空格）的行。

然后，所有读取到该点的行都将用作命令的标准输入。

这里文档的格式是：

          <<[-]word
                  here-document
          delimiter

不对word 执行参数扩展、命令替换、算术扩展或路径名扩展。如果 word 中的任何字符被引用，则 分隔符是对 word 去除引号的结果，并且不会扩展 here-document中的行。 如果 word 未被引用，则 here-document 的所有行都将进行参数扩展、命令替换和算术扩展。在后一种情况下，字符序列\<newline>将被忽略，并且\必须用于引用字符\、$和```。

如果重定向运算符是，则所有前导制表符都将从输入行和包含 定界符的<<-行中删除。这允许 shell 脚本中的 here-documents 以自然的方式缩进。 ****

如何列出 npm 用户安装的包

Arya

2023-02-17 08:55

如何仅 在 npm 中列出用户安装/环境包？

当我这样做时npm -g list，它会输出每个包及其依赖项。相反，我希望看到安装在 当前 工作项目或环境中的包。



解答

npm list -g --depth=0
  • npm ：Node.js 包管理器命令行工具
  • list -g ：显示在用户文件夹中找到的每个包的树（没有选项-g它只显示当前目录的包）
  • --depth 0 / --depth=0 : 避免在树视图中包含每个包的依赖项


更多建议

您可以使用以下命令获取所有全局安装模块的列表：

ls `npm root -g`

如何设置TextView textStyle如粗体、斜体

Arya

2023-02-17 08:54

如何设置TextView在不使用 XML 布局的情况下在 Java 中设置样式（粗体或斜体）？

换句话说，我需要android:textStyle用 Java 编写。



解答

textView.setTypeface(null, Typeface.BOLD_ITALIC);
textView.setTypeface(null, Typeface.BOLD);
textView.setTypeface(null, Typeface.ITALIC);
textView.setTypeface(null, Typeface.NORMAL);

保留以前的字体

textView.setTypeface(textView.getTypeface(), Typeface.BOLD_ITALIC)


更多建议

试试这个设置TextView为粗体或斜体

textView.setTypeface(textView.getTypeface(), Typeface.BOLD);
textView.setTypeface(textView.getTypeface(), Typeface.ITALIC);
textView.setTypeface(textView.getTypeface(), Typeface.BOLD_ITALIC);
