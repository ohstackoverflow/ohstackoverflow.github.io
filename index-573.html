链接到 Javadoc 中的外部 URL？

Arya

2023-02-17 10:27

就像是：

/**
 * See {@linktourl http://google.com}
 */



解答

这将创建一个包含链接的“另见”标题，即：

/**
 * @see <a href="http://google.com">http://google.com</a>
 */

将呈现为：

另请参阅： http:
//google.com

而这个：

/**
 * See <a href="http://google.com">http://google.com</a>
 */

将创建一个内嵌链接：

请参阅http://google.com


更多建议

取自javadoc 规范

@see <a href="URL#value">label</a>: 添加由 定义的链接URL#value。是URL#value相对或绝对 URL。Javadoc 工具通过查找小于号 ( <) 作为第一个字符来将这种情况与其他情况区分开来。

例如 ：@see <a href="http://www.google.com">Google</a>

如何检查 PHP 数组是关联的还是顺序的？

Arya

2023-02-17 10:26

PHP 将所有数组视为关联数组，因此没有任何内置函数。谁能推荐一种相当有效的方法来检查数组是否 “是列表” （仅包含从 0 开始的数字键）？

基本上，我希望能够区分这一点：

$sequentialArray = [
    'apple', 'orange', 'tomato', 'carrot'
];

和这个：

$assocArray = [
    'fruit1' => 'apple',
    'fruit2' => 'orange',
    'veg1' => 'tomato',
    'veg2' => 'carrot'
];



解答

您问了两个不完全相同的问题：

  • 一、如何判断数组是否只有数字键
  • 其次，如何判断一个数组是否有 连续的 数字键，从0开始

考虑您实际需要哪些行为。（可能两者都适合您的目的。）

Captain kurO 很好地回答了第一个问题（简单地检查所有键是否为数字）。

对于第二个问题（检查数组是否为零索引和顺序），可以使用以下函数：

function isAssoc(array $arr)
{
    if (array() === $arr) return false;
    return array_keys($arr) !== range(0, count($arr) - 1);
}

var_dump(isAssoc(['a', 'b', 'c'])); // false
var_dump(isAssoc(["0" => 'a', "1" => 'b', "2" => 'c'])); // false
var_dump(isAssoc(["1" => 'a', "0" => 'b', "2" => 'c'])); // true
var_dump(isAssoc(["a" => 'a', "b" => 'b', "c" => 'c'])); // true


更多建议

仅检查数组是否具有非整数键（而不是数组是顺序索引还是零索引）：

function has_string_keys(array $array) {
  return count(array_filter(array_keys($array), 'is_string')) > 0;
}

如果至少有一个字符串键，$array将被视为一个关联数组。

Moment.js 转换为日期对象

Arya

2023-02-17 10:26

使用 Moment.js 我无法将正确的时刻对象转换为具有时区的日期对象。我无法获得正确的日期。

例子：

var oldDate = new Date(),
    momentObj = moment(oldDate).tz("MST7MDT"),
    newDate = momentObj.toDate();
console.log("start date " + oldDate)
console.log("Format from moment with offset " + momentObj.format())
console.log("Format from moment without offset " + momentObj.utc().format())
console.log("(Date object) Time with offset " + newDate)
console.log("(Date object) Time without offset "+ moment.utc(newDate).toDate())



解答

使用它来将 moment 对象转换为 date 对象：

来自http://momentjs.com/docs/#/displaying/as-javascript-date/

moment().toDate();

产量：

Tue Nov 04 2014 14:04:01 GMT-0600 (CST)


更多建议

只要您使用所需区域的数据初始化了moment-timezone，您的代码就会按预期工作。

您正在正确地将时间转换为时区，这反映在 的第二行输出中momentObj.format()。

切换到 UTC 不仅会降低偏移量，还会变回 UTC 时区。如果您要这样做，则.tz()根本不需要原始电话。你可以这样做moment.utc()。

也许您只是想更改输出格式字符串？如果是这样，只需指定您想要的方法参数format：

momentObj.format("YYYY-MM-DD HH:mm:ss")

关于代码的最后几行 - 当您使用 返回对象时Date，toDate()您将放弃 moment.js 的行为并返回到 JavaScript 的行为。JavaScriptDate对象将始终在其运行的计算机的本地时区打印。moment.js 对此无能为力。

其他一些小事：

  • 虽然 moment constructor 可以 使用Date，但通常最好不要使用。对于“现在”，不要使用moment(new Date()). 相反，只需使用moment(). 两者都可以工作，但这是不必要的冗余。如果您从字符串中解析，请将该字符串直接传递给 moment. 不要试图首先解析它Date。您会发现 moment 的解析器更加可靠。

  • 像这样的时区MST7MDT是出于向后兼容的原因。它们源于 POSIX 风格的时区，只有少数在 TZDB 数据中。除非绝对必要，否则您应该使用诸如America/Denver.
