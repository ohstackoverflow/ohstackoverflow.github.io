在文件中查找和替换并覆盖文件不起作用，它会清空文件

Arya

2023-02-17 21:49

我想通过命令行在 HTML 文件上运行查找和替换。

我的命令看起来像这样：

sed -e s/STRING_TO_REPLACE/STRING_TO_REPLACE_IT/g index.html > index.html

当我运行它并随后查看文件时，它是空的。它删除了我文件的内容。

当我再次恢复文件后运行它时：

sed -e s/STRING_TO_REPLACE/STRING_TO_REPLACE_IT/g index.html

是stdout文件的内容，已经执行了查找替换。

为什么会这样？



解答

当 shell 在命令行中看到时> index.html，它会打开文件index.html进行 写入 ，擦除所有以前的内容。

要解决此问题，您需要将-i选项传递给以sed进行内联更改并在就地进行更改之前创建原始文件的备份：

sed -i.bak s/STRING_TO_REPLACE/STRING_TO_REPLACE_IT/g index.html

如果没有 .bak，该命令将在某些平台上失败，例如 Mac OSX。


更多建议

另一种有用的模式是：

sed -e 'script script' index.html > index.html.tmp && mv index.html.tmp index.html

这具有几乎相同的效果，但不使用该-i选项，另外意味着，如果 sed 脚本由于某种原因失败，输入文件不会被破坏。此外，如果编辑成功，则不会留下任何备份文件。这种习语在 Makefile 中很有用。

很多 seds 都有这个-i选项，但不是全部；posix sed 是没有的。因此，如果您的目标是便携性，最好避免使用。

Git - 到处忽略node_modules文件夹

Arya

2023-02-17 21:48

我有一个包含多个其他项目的项目：

  • 主要项目
    • 迷你项目 1
    • 迷你项目 2

全部包含node_modules文件夹。我希望 git 忽略该文件夹，无论它是从根文件夹开始的。像这样添加到 .gitignore 中：

*node_modules/*



解答

添加node_modules/或node_modules到.gitignore文件以忽略node_modules当前文件夹中调用的所有目录和任何子文件夹，如下图所示。


更多建议

在项目目录的终端中使用 通用的一行代码：

touch .gitignore && echo "node_modules/" >> .gitignore && git rm -r --cached node_modules ; 混帐状态

无论您是否创建了一个.gitignore，无论您是否已添加node_modules到 git 跟踪，它都有效。

然后提交并推送.gitignore更改。

解释

touch``.gitignore如果文件不存在，将生成该文件。

echo并将>>附加node_modules/在 的末尾.gitignore，导致node_modules文件夹和所有子文件夹被忽略。

git rm -r --cached``node_modules如果之前添加了文件夹，则从 git 控件中删除该文件夹。否则，这将显示一个警告pathspec 'node_modules' did not match any files，它没有副作用，您可以放心地忽略它。这些标志导致删除是递归的并包括缓存。

git status显示新的更改。.gitignore将出现一个更改，而node_modules不会出现，因为它不再被 git 跟踪。
