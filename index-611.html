“TypeError：在 Python 3 中处理文件内容时，需要一个类似字节的对象，而不是‘str’”

Arya

2023-02-17 11:26

我最近迁移到了 Python 3.5。此代码在 Python 2.7 中正常运行：

with open(fname, 'rb') as f:
    lines = [x.strip() for x in f.readlines()]

for line in lines:
    tmp = line.strip().lower()
    if 'some-pattern' in tmp: continue
    # ... code

但是在 3.5 中，在线上if 'some-pattern' in tmp: continue，我收到一条错误消息：

TypeError: a bytes-like object is required, not 'str'

我无法.decode()在 的任一侧使用 解决问题in，也无法使用 解决问题

    if tmp.find('some-pattern') != -1: continue

出了什么问题，我该如何解决？



解答

您以二进制模式打开文件：

with open(fname, 'rb') as f:

这意味着从文件中读取的所有数据都作为bytes对象返回，而不是str. 然后您不能在包含测试中使用字符串：

if 'some-pattern' in tmp: continue

你必须使用一个bytes对象来测试tmp：

if b'some-pattern' in tmp: continue

或者将文件作为文本文件打开，而不是通过将'rb'模式替换为'r'.


更多建议

您可以使用.encode()

例子：

'Hello World'.encode()

如错误所述，为了将字符串写入文件，您需要先将其编码为类似字节的对象，并将encode()其编码为字节字符串。

Xcode 7 错误：“缺少...的 iOS 分发签名身份”

Arya

2023-02-17 11:25

我试图将我的应用程序上传到 iTunes Connect resp。AppStore 并收到以下错误：

无法找到或生成匹配的签名资产

Xcode 尝试定位或生成匹配的签名资产，但由于以下问题未能成功。

Missing iOS Distribution signing identity for ... Xcode 可以为您申请一个。

在我设置新的开发机器之前，通过 Xcode 7 将开发者帐户从旧机器导出到新机器。

我该怎么做才能解决这个问题？



解答

来自苹果-

感谢您提请社区注意此事，并对您遇到的问题深表歉意。此问题源于在您的系统和登录钥匙串中都有一份过期的 WWDR 中间证书。要解决此问题，您应该首先下载并安装新的WWDR 中间证书（通过双击该文件）。接下来，在钥匙串访问应用程序中，选择系统钥匙串。请务必在“查看”菜单中选择“显示过期证书”，然后删除过期版本的 Apple Worldwide Developer Relations Certificate Authority Intermediate 证书（2016 年 2 月 14 日过期）。您的证书现在应该在 Keychain Access 中显示为有效，并且可用于 Xcode 以提交到 App Store。

正如下面的评论中所指出的，过期的证书也需要从该部分中删除login，以及：

对于所有尽管有说明仍无法正常工作的人...有两个过期的 WWDR 证书。一个在登录钥匙串中，另一个在系统中。您必须同时删除它们才能使事情正常进行


更多建议

我今天也面临同样的问题。以下步骤解决了我的问题。

  1. 下载https://developer.apple.com/certificationsauthority/AppleWWDRCA.cer
  2. 双击安装到钥匙串。
  3. 然后在 Keychain 中，在 Keychain 应用程序中选择 View -> "Show Expired Certificates"。
  4. 它将列出所有过期的证书。
  5. 从“登录”选项卡中删除“Apple Worldwide Developer Relations Certificate Authority certificates”
  6. 并将其从“系统”选项卡中删除。

现在你准备好了。

如何防止“git diff”使用寻呼机？

Arya

2023-02-17 11:25

是否有命令行开关传递给默认git diff使用寻呼机的其他命令？less我知道这些方法存在：

  • git diff | cat... 删除所有语法突出显示
  • git config --global core.pager cat将全局 .gitconfig 中的寻呼机设置为cat
  • export GIT_PAGER=cat

但我更喜欢命令行开关。



解答

--no-pager到 Git 会告诉它不要使用寻呼机。如果输出适合单个屏幕，将选项传递-F给将告诉它不分页。less

用法：

git --no-pager diff

评论中的其他选项包括：

# Set an evaporating environment variable to use 'cat' for your pager
GIT_PAGER=cat git diff

# Tells 'less' not to paginate if less than a page
export LESS="-F -X $LESS"
# ...then Git as usual
git diff


更多建议

正如前面提到的答案，如果内容少于一个屏幕，将-F选项传递给会导致它退出。less但是，这样做之后，屏幕会重置，您最终看不到内容。

该-X选项消除了该行为。因此，我使用以下命令根据内容量启用条件分页：

git config --global --replace-all core.pager "less -F -X"
