是否可以将一个 CSS 文件包含在另一个文件中？

Arya

2023-02-17 11:18

是否可以将一个 CSS 文件包含在另一个文件中？



解答

是的：

@import url("base.css");

笔记：

  • 该@import规则必须先于所有其他规则（除了@charset）。
  • 额外的@import语句需要额外的服务器请求。作为一种替代方法，将所有 CSS 连接到一个文件中以避免多个 HTTP 请求。例如，将base.cssand的内容复制special.css到base-special.cssand reference only中base-special.css。


更多建议

是的。可以将 CSS 文件导入另一个 CSS 文件。

它必须是样式表中使用@import 规则的第一条规则。

@import "mystyle.css";
@import url("mystyle.css");

唯一需要注意的是，较旧的网络浏览器不支持它。事实上，这是对旧浏览器隐藏 CSS 样式的 CSS“hack”之一。

请参阅此列表以了解浏览器支持。

我正在尝试使用以下命令行将我的代码部署到 heroku：

git push heroku master

但出现以下错误：

Permission denied (publickey).
fatal: The remote end hung up unexpectedly

我已经上传了我的公共 SSH 密钥，但它仍然出现此错误。



解答

您必须将您的公钥上传到 Heroku：

heroku keys:add ~/.ssh/id_rsa.pub

如果您没有公钥，Heroku 会提示您自动添加一个可以无缝运行的公钥。只需使用：

heroku keys:add

要清除所有以前的密钥，请执行以下操作：

heroku keys:clear

要显示所有现有密钥，请执行以下操作：

heroku keys

每次推送时 Git 都会询问用户名

Arya

2023-02-17 11:17

每当我尝试推入我的 repo 时，git 都会要求两者username & password。

我每次重新输入密码都没有问题，但问题是输入用户名。我用来https克隆我的存储库。

那么，我该如何配置 git，使其不会username在每个git push.

我是 linux 的新手，但 Windows 中的 IIRCgit push只要求输入密码。



解答

编辑（由@dk14 根据版主和评论的建议）

警告：如果您使用credential.helper storefrom the answer，您的密码将完全未加密（“原样”）存储在~/.git- credentials. 请查阅下面的评论部分或“链接”部分的答案，特别是如果您的雇主对安全问题零容忍。

即使被接受，它也没有回答实际 OP 关于仅省略用户名（而不是密码）的问题。对于有这个确切问题的读者，@grawity 的回答可能会派上用场。

原始答案（来自@Alexander Zhu）：

您可以使用以下命令存储您的凭据

$ git config credential.helper store
$ git push http://example.com/repo.git
Username: <type your username>
Password: <type your password>

我也建议你阅读
$ git help credentials


更多建议

您可以在.git/config本地存储库的文件中完成此操作。该文件包含一个名为“remote”的部分和一个名为“url”的条目。'url' 条目应包含您正在谈论的存储库的 https 链接。

当您在主机“url”前加上您的用户名时，git不应再询问您的用户名。这是一个例子：

url = https://username@repository-url.com
