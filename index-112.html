如何在 SQL Server 中将多行文本连接成单个文本字符串

Arya

2023-02-16 21:45

考虑一个包含名称的数据库表，其中包含三行：

Peter
Paul
Mary

有没有一种简单的方法可以将它变成一个字符串Peter, Paul, Mary？



解答

如果您使用的是 SQL Server 2017 或 Azure，请参阅Mathieu Renda 的回答。

当我尝试连接具有一对多关系的两个表时，我遇到了类似的问题。在 SQL 2005 中，我发现该XML PATH方法可以非常轻松地处理行的串联。

如果有一张表叫STUDENTS

SubjectID       StudentName
----------      -------------
1               Mary
1               John
1               Sam
2               Alaina
2               Edward

我预期的结果是：

SubjectID       StudentName
----------      -------------
1               Mary, John, Sam
2               Alaina, Edward

我使用了以下内容T-SQL：

SELECT Main.SubjectID,
       LEFT(Main.Students,Len(Main.Students)-1) As "Students"
FROM
    (
        SELECT DISTINCT ST2.SubjectID, 
            (
                SELECT ST1.StudentName + ',' AS [text()]
                FROM dbo.Students ST1
                WHERE ST1.SubjectID = ST2.SubjectID
                ORDER BY ST1.SubjectID
                FOR XML PATH (''), TYPE
            ).value('text()[1]','nvarchar(max)') [Students]
        FROM dbo.Students ST2
    ) [Main]

如果您可以在开头连接逗号并使用substring跳过第一个逗号，那么您可以以更紧凑的方式做同样的事情，这样您就不需要进行子查询：

SELECT DISTINCT ST2.SubjectID, 
    SUBSTRING(
        (
            SELECT ','+ST1.StudentName  AS [text()]
            FROM dbo.Students ST1
            WHERE ST1.SubjectID = ST2.SubjectID
            ORDER BY ST1.SubjectID
            FOR XML PATH (''), TYPE
        ).value('text()[1]','nvarchar(max)'), 2, 1000) [Students]
FROM dbo.Students ST2


更多建议

此答案可能会返回 意外结果要获得一致的结果，请使用其他答案中详述的 FOR XML PATH 方法之一。

使用COALESCE：

DECLARE @Names VARCHAR(8000) 
SELECT @Names = COALESCE(@Names + ', ', '') + Name 
FROM People

只是一些解释（因为这个答案似乎得到了相对正常的看法）：

  • Coalesce 实际上只是一个有用的作弊工具，它可以完成两件事：

1）不需要@Names用空字符串值初始化。

2) 不需要在最后剥离额外的分隔符。

  • 如果一行有 NULL Name 值，上面的解决方案将给出不正确的结果（如果有 NULL ， NULL 将在该行之后变为@Names NULL ，并且下一行将再次作为空字符串重新开始。可以使用以下两个之一轻松修复解决方案：

    DECLARE @Names VARCHAR(8000) SELECT @Names = COALESCE(@Names + ', ', '') + Name FROM People WHERE Name IS NOT NULL

或者：

DECLARE @Names VARCHAR(8000) 
SELECT @Names = COALESCE(@Names + ', ', '') + 
    ISNULL(Name, 'N/A')
FROM People

取决于您想要的行为（第一个选项只是过滤掉 NULL ，第二个选项将它们保留在带有标记消息的列表中[用适合您的任何内容替换'N/A']）。

Java 中的“实现 Runnable”与“扩展线程”

Arya

2023-02-16 21:44

从我花在线程上的时间开始Java，我发现了这两种编写线程的方法：

带 工具Runnable：

public class MyRunnable implements Runnable {
    public void run() {
        //Code
    }
}
//Started with a "new Thread(new MyRunnable()).start()" call

或者，使用 extendsThread：

public class MyThread extends Thread {
    public MyThread() {
        super("MyThread");
    }
    public void run() {
        //Code
    }
}
//Started with a "new MyThread().start()" call

这两个代码块有什么显着差异吗？



解答

是的：IMO，implementsRunnable是首选的方式。您并没有真正专门化线程的行为。你只是给它一些东西来运行。这意味着组合是 哲学上 “更纯粹”的方式。

实际上，这意味着您也可以从另一个类实现和扩展......您还可以通过 Java 8 的 lambda 表达式 实现。Runnable``Runnable


更多建议

tl;dr：实现 Runnable 更好。 但是，警告很重要。

一般来说，我会推荐使用类似Runnablerather than 的东西Thread，因为它允许您将您的工作仅与您选择的并发性松散耦合。例如，如果您使用 aRunnable并稍后决定这实际上不需要它自己的Thread，您可以只调用 threadA.run()。

警告： 在这里，我强烈反对使用原始线程。我更喜欢使用Callables和FutureTasks（来自 javadoc：“A cancellable asynchronous computation”）。现代并发支持的超时、适当取消和线程池的集成对我来说都比一堆原始线程有用得多。

跟进： 有一个FutureTask构造函数允许您使用 Runnables（如果这是您最熟悉的）并且仍然可以从现代并发工具中获益。引用 javadoc：

如果您不需要特定结果，请考虑使用以下形式的结构：

Future<?> f = new FutureTask<Object>(runnable, null)

因此，如果我们将 their 替换runnable为 your threadA，我们将得到以下内容：

new FutureTask<Object>(threadA, null)

另一个让您更接近 Runnables 的选项是ThreadPoolExecutor。您可以使用execute方法传入一个 Runnable 来执行“将来某个时间的给定任务”。

如果您想尝试使用线程池，上面的代码片段将变成如下所示（使用 Executors.newCachedThreadPool ()工厂方法）：

ExecutorService es = Executors.newCachedThreadPool();
es.execute(new ThreadA());

如何恢复 Git 中丢失的存储？

Arya

2023-02-16 21:43

我经常使用git stash和git stash pop来保存和恢复工作树中的更改。昨天，我对我的工作树进行了一些更改，这些更改是我隐藏和弹出的，然后我对我的工作树进行了更多更改。我想回去查看昨天隐藏的更改，但git stash pop似乎删除了对相关提交的所有引用。

我知道如果我使用git stash.git /refs/stash 包含 用于创建存储的提交的引用。而 .git/logs/refs/stash 包含 整个存储。但是那些引用在git stash pop. 我知道提交仍在我的某个存储库中，但我不知道它是什么。

有没有一种简单的方法可以恢复昨天的隐藏提交引用？



解答

一旦你知道你删除的存储提交的哈希值，你就可以将它作为存储应用：

git stash apply $stash_hash

或者，您可以为它创建一个单独的分支

git branch recovered $stash_hash

之后，您可以使用所有常规工具做任何您想做的事情。当你完成后，把树枝吹走。

寻找哈希

如果你刚刚弹出它并且终端仍然打开，你仍然会在屏幕上打印哈希值git stash pop（感谢 Dolda）。

否则，您可以在 Linux、Unix 或 Windows 的 Git Bash 中使用它来找到它：

git fsck --no-reflog | awk '/dangling commit/ {print $3}'

...或使用适用于 Windows 的 PowerShell：

git fsck --no-reflog | select-string 'dangling commit' | foreach { $_.ToString().Split(" ")[2] }

这将在提交图的提示处向您显示所有提交，这些提交不再被任何分支或标记引用——每个丢失的提交，包括您曾经创建的每个隐藏提交，都将位于该图中的某个位置。

找到你想要的隐藏提交的最简单方法可能是将该列表传递给gitk：

gitk --all $( git fsck --no-reflog | awk '/dangling commit/ {print $3}' )

...或者如果使用适用于 Windows 的 PowerShell，请查看emragins 的答案。

这将启动一个存储库浏览器，向您显示 存储库中的每一次提交 ，无论它是否可访问。

如果您更喜欢控制台上的漂亮图形而不是单独的 GUI 应用程序，则可以将gitk其替换为类似的东西。git log --graph --oneline --decorate

要发现隐藏提交，请查找以下形式的提交消息：

某些分支 上的 WIP ： commithash 一些旧的提交消息

注意 ：如果您在提供消息时没有提供消息，提交消息将仅采用这种形式（以“WIP on”开头）git stash。


更多建议

如果您没有关闭终端，只需查看输出git stash pop，您就会得到已删除存储的对象 ID。它通常看起来像这样：

$ git stash pop
[...]
Dropped refs/stash@{0} (2ca03e22256be97f9e40f08e6d6773c7d41dbfd1)

（请注意，这git stash drop也会产生相同的行。）

要取回该存储，只需运行git branch tmp 2cae03e，您就会将其作为一个分支获取。要将其转换为存储，请运行：

git stash apply tmp
git stash

将它作为一个分支还可以让你自由地操作它；例如，挑选它或合并它。
