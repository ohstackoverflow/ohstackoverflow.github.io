JavaScript 中是否有“空合并”运算符？

Arya

2023-02-17 01:40

Javascript 中是否有空合并运算符？

例如，在 C# 中，我可以这样做：

String someString = null;
var whatIWant = someString ?? "Cookies!";

我能为 Javascript 找到的最佳近似是使用条件运算符：

var someString = null;
var whatIWant = someString ? someString : 'Cookies!';

恕我直言，这有点恶心。我可以做得更好吗？



解答

更新

JavaScript 现在支持nullish 合并运算符 (??)。当其左侧操作数为nullor时，它返回其右侧undefined操作数，否则返回其左侧操作数。

旧答案

使用前请检查兼容性。

C# 空合并运算符 ( ??) 的 JavaScript 等价物使用逻辑 OR ( ||)：

var whatIWant = someString || "Cookies!";

在某些情况下（下文会阐明）行为与 C# 的行为不匹配，但这是在 JavaScript 中分配默认值/替代值的一般、简洁的方式。

澄清

无论第一个操作数的类型如何，如果将其转换为布尔值导致false，赋值将使用第二个操作数。当心以下所有情况：

alert(Boolean(null)); // false
alert(Boolean(undefined)); // false
alert(Boolean(0)); // false
alert(Boolean("")); // false
alert(Boolean("false")); // true -- gotcha! :)

这意味着：

var whatIWant = null || new ShinyObject(); // is a new shiny object
var whatIWant = undefined || "well defined"; // is "well defined"
var whatIWant = 0 || 42; // is 42
var whatIWant = "" || "a million bucks"; // is "a million bucks"
var whatIWant = "false" || "no way"; // is "false"


更多建议

function coalesce() {
    var len = arguments.length;
    for (var i=0; i<len; i++) {
        if (arguments[i] !== null && arguments[i] !== undefined) {
            return arguments[i];
        }
    }
    return null;
}

var xyz = {};
xyz.val = coalesce(null, undefined, xyz.val, 5);

// xyz.val now contains 5

此解决方案的工作方式类似于 SQL 合并函数，它接受任意数量的参数，如果没有参数则返回 null。它的行为类似于 C# ?? “”、false 和 0 被视为 NOT NULL 并因此算作实际值的意义上的运算符。如果你来自.net 背景，这将是最自然的感觉解决方案。

我如何强制 Python 的print函数将缓冲的输出刷新到屏幕上？

对于重复的关闭器： 如果初学者问一个关于尝试在末尾不使用换行符的情况下立即显示输出的问题， 请改用 当末尾没有换行符时为什么打印输出不立即显示在终端中？关闭问题。当前的问题不够好，因为提出问题的人可能 没有 缓冲或冲洗的概念；另一个问题旨在首先解释这些概念，而这个问题是关于技术细节的。



解答

在 Python 3 中，print可以采用可选flush参数：

print("Hello, World!", flush=True)

在 Python 2 中，调用 后print，执行：

import sys
sys.stdout.flush()

默认情况下，print打印到（有关文件对象sys.stdout的更多信息，请参阅文档）。

Node.js module.exports 的用途是什么？如何使用它？

Arya

2023-02-17 01:39

Node.js 的用途是什么module.exports？如何使用它？

我似乎找不到关于此的任何信息，但它似乎是 Node.js 的一个相当重要的部分，因为我经常在源代码中看到它。

根据Node.js 文档：

模块

对当前module. 特别module.exports是与 exports 对象相同。src/node.js有关详细信息，请参阅。

但这并没有真正的帮助。

究竟是module.exports做什么的，一个简单的例子是什么？



解答

module.exports是作为调用结果实际返回的对象require。

该exports变量最初设置为同一个对象（即它是一个简写的“别名”），因此在模块代码中您通常会这样写：

let myFunc1 = function() { ... };
let myFunc2 = function() { ... };
exports.myFunc1 = myFunc1;
exports.myFunc2 = myFunc2;

导出（或“公开”）内部作用域的函数myFunc1和myFunc2.

在调用代码中，您将使用：

const m = require('./mymodule');
m.myFunc1();

最后一行显示的结果require（通常）只是一个可以访问其属性的普通对象。

注意：如果你覆盖，exports那么它将不再引用module.exports. 因此，如果您希望将一个新对象（或函数引用）分配给，那么exports您还应该将该新对象分配给module.exports

值得注意的是，添加到exports对象的名称不必与要添加的值的模块的内部范围名称相同，因此您可以：

let myVeryLongInternalName = function() { ... };
exports.shortName = myVeryLongInternalName;
// add other objects, functions, as required

其次是：

const m = require('./mymodule');
m.shortName(); // invokes module.myVeryLongInternalName


更多建议

这已经得到回答，但我想补充一些说明......

您可以同时使用exports和module.exports将代码导入到您的应用程序中，如下所示：

var mycode = require('./path/to/mycode');

您将看到的基本用例（例如在 ExpressJS 示例代码中）是您在 .js 文件中设置对象的属性，exports然后使用require()

因此，在一个简单的计数示例中，您可以：

（计数器.js）：

var count = 1;

exports.increment = function() {
    count++;
};

exports.getCount = function() {
    return count;
};

...然后在您的应用程序中（web.js 或任何其他 .js 文件）：

var counting = require('./counter.js');

console.log(counting.getCount()); // 1
counting.increment();
console.log(counting.getCount()); // 2

简单来说，您可以将所需文件视为返回单个对象的函数，您可以将属性（字符串、数字、数组、函数等）添加到通过将它们设置为 on 返回的对象exports。

有时您希望从调用返回的对象require()是一个您可以调用的函数，而不仅仅是一个具有属性的对象。在这种情况下，您还需要设置module.exports，如下所示：

（说你好.js）：

module.exports = exports = function() {
    console.log("Hello World!");
};

(应用程序.js):

var sayHello = require('./sayhello.js');
sayHello(); // "Hello World!"

exports 和 module.exports 之间的区别在此处的答案中得到了更好的解释。
