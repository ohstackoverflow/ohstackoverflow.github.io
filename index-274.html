SQL Server 检查表是否存在

Arya

2023-02-17 02:48

我希望这是关于如何使用 SQL 语句检查 SQL Server 2000/2005 中是否存在表的最终讨论。

这里有两种可能的方法。哪一个是标准/最好的方法？

第一种方式：

IF EXISTS (SELECT 1 
           FROM INFORMATION_SCHEMA.TABLES 
           WHERE TABLE_TYPE='BASE TABLE' 
           AND TABLE_NAME='mytablename') 
   SELECT 1 AS res ELSE SELECT 0 AS res;

第二种方式：

IF OBJECT_ID (N'mytablename', N'U') IS NOT NULL 
   SELECT 1 AS res ELSE SELECT 0 AS res;

MySQL 提供了简单的

SHOW TABLES LIKE '%tablename%';

陈述。我正在寻找类似的东西。



解答

对于这样的查询，最好始终使用视图INFORMATION_SCHEMA。这些视图（大部分）在许多不同的数据库中是标准的，并且很少在不同版本之间发生变化。

要检查表是否存在，请使用：

IF (EXISTS (SELECT * 
                 FROM INFORMATION_SCHEMA.TABLES 
                 WHERE TABLE_SCHEMA = 'TheSchema' 
                 AND  TABLE_NAME = 'TheTable'))
BEGIN
    --Do Stuff
END


更多建议

另请注意，如果出于任何原因您需要检查临时表，您可以这样做：

if OBJECT_ID('tempdb..#test') is not null
 --- temp table exists

使用 CSS 保持 div 的纵横比

Arya

2023-02-17 02:47

我想创建一个响应div，可以随着窗口宽度的变化而改变它的宽度/高度。

是否有任何 CSS 规则允许高度根据宽度变化， 同时保持其纵横比 ？

我知道我可以通过 JavaScript 做到这一点，但我更愿意只使用 CSS。



解答

只需创建一个<div>带有百分比值的包装器padding-bottom，如下所示：

.demoWrapper {
  padding: 10px;
  background: white;
  box-sizing: border-box;
  resize: horizontal;
  border: 1px dashed;
  overflow: auto;
  max-width: 100%;
  height: calc(100vh - 16px);
}

div {
  width: 100%;
  padding-bottom: 75%;
  background: gold; /** <-- For the demo **/
}


<div class="demoWrapper">
  <div></div>
</div>

它将导致<div>高度等于其容器宽度的 75%（4:3 纵横比）。

这依赖于填充的事实：

百分比是根据生成的框的包含块的 宽度计算的[...]（来源： w3.org，强调我的）

其他宽高比和 100% 宽度的底边距值：

aspect ratio  | padding-bottom value
--------------|----------------------
    16:9      |       56.25%
    4:3       |       75%
    3:2       |       66.66%
    8:5       |       62.5%

在 div 中放置内容：

为了保持 div 的纵横比并防止其内容拉伸它，您需要添加一个绝对定位的孩子并将其拉伸到包装器的边缘：

div.stretchy-wrapper {
  position: relative;
}

div.stretchy-wrapper > div {
  position: absolute;
  top: 0; bottom: 0; left: 0; right: 0;
}

更多建议

有几种方法可以在像 div 这样的元素上 指定固定的宽高比，这里是其中的两种：

1.CSSaspect-ratio属性

div {
  aspect-ratio: 1 / 1;
  width: 50%;
  background: teal;

<div>aspect-ratio: 1 / 1;</div>

这是最简单灵活的解决方案 。它直接为元素指定固定的宽度与高度（或高度与宽度）纵横比。这意味着您还可以根据元素 height 指定纵横比。
它不依赖于父级宽度（如填充技术）或视口大小（如以下vw单元技术），它依赖于元素自身的宽度或高度有关 MDN 的更多信息。与其他解决方法相比，这就是它如此强大的原因。

如何挑选多个提交

Arya

2023-02-17 02:47

我有两个分支。Commita是一个的头部，而另一个有b, c, d,e和fon top of a。我想在没有提交的情况下移动、和c到d第一个分支。使用 cherry pick 很简单：checkout 第一个分支 cherry-pick 一个一个地检查第二个分支到第一个分支。但是有什么办法可以在一个命令中挑选所有的东西吗？e``f``b``c``f``c``f

这是场景的直观描述（感谢JJD）：



解答

Git 1.7.2 引入了挑选一系列提交的能力。从发行说明：

git cherry-pick学会了选择一系列提交（例如cherry-pick A..B和cherry-pick --stdin），也是如此git revert；rebase [-i]但是，这些不支持更好的排序控制。

A要 cherry-pick 从提交到提交的所有提交B（A早于B），请运行：

git cherry-pick A^..B

如果你想 忽略 A 本身，运行：

git cherry-pick A..B

来自评论的注释：

  • A应该早于B，或者A应该来自另一个分支。
  • 在 Windows 上，它应该是A^^..B因为插入符需要转义，或者应该是"A^..B"（双引号）。
  • 在zshshell 中，它应该是'A^..B'（单引号），因为插入符号是一个特殊字符。
  • 有关说明，请参阅Gabriel Staples 的回答。

（在评论中感谢 damian、JB Rainsberger、sschaef、Neptilo、Pete 和 TMin。）


更多建议

如果您有选择性的修订要合并，比如来自 A、B、C、D、E、F、G、H、I、J 提交的 A、C、F、J，只需使用以下命令：

git cherry-pick A C F J
