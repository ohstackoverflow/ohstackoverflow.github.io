在背景中拉伸和缩放 CSS 图像 - 仅使用 CSS

Arya

2023-02-17 09:19

我希望我的背景图像根据浏览器视口大小进行拉伸和缩放。

我在 Stack Overflow 上看到了一些可以完成这项工作的问题，例如 Stretch and scale CSS background 。background它运行良好，但我想使用而不是标签放置图像img。

在那个标签中img放置了一个标签，然后我们使用 CSS 向img标签致敬。

width:100%; height:100%;

它有效，但这个问题有点陈旧，并指出在 CSS 3 中调整背景图像的大小会很好地工作。我已经在第一个示例中尝试过这个示例，但它对我来说没有用。

有没有一个很好的方法来做声明background-image？



解答

CSS3 有一个不错的小属性，叫做background-size:cover.

这会缩放图像，以便背景区域完全被背景图像覆盖，同时保持纵横比。整个区域都将被覆盖。但是，如果调整后的图像的宽度/高度太大，则部分图像可能不可见。


更多建议

您可以使用 CSS3 属性来很好地完成它。它会调整比例，因此不会出现图像失真（尽管它会放大小图像）。请注意，它尚未在所有浏览器中实现。

background-size: 100%;

如何在 Django 查询集过滤中执行不等于？

在 Django 模型 QuerySets 中，我看到有一个__gt和__lt用于比较值，但是有一个__ne或!=（ 不等于 ）吗？我想使用不等于过滤掉。例如，对于

Model:
    bool a;
    int x;

我想要做

results = Model.objects.exclude(a=True, x!=5)

语法!=不正确。我也试过了__ne。

我最终使用了：

results = Model.objects.exclude(a=True, x__lt=5).exclude(a=True, x__gt=5)



解答

您可以为此使用Q 对象。它们可以用~运算符取反并像普通的 Python 表达式一样组合：

from myapp.models import Entry
from django.db.models import Q

Entry.objects.filter(~Q(id=3))

将返回所有条目，但 ID 除外3：

[<Entry: Entry object>, <Entry: Entry object>, <Entry: Entry object>, ...]


更多建议

您的查询似乎有双重否定，您想排除所有不是 5 的行 ，x所以换句话说，您想要包括所有5 的行。我相信这会成功：x __

results = Model.objects.filter(x=5).exclude(a=True)

要回答您的具体问题，没有“不等于”字段查找，但这可能是因为 Django 具有可用的filter和exclude方法，因此您始终可以切换逻辑以获得所需的结果。

GNU Makefile 变量赋值 =、?=、:= 和 += 之间有什么区别？

Arya

2023-02-17 09:18

任何人都可以清楚地解释变量赋值在 Makefile 中是如何工作的。

有什么区别：

 VARIABLE = value
 VARIABLE ?= value
 VARIABLE := value
 VARIABLE += value

我已经阅读了GNU Make 手册中的部分，但对我来说仍然没有意义。



解答

懒集

VARIABLE = value

变量的正常设置，但该value字段中提到的任何其他变量都在使用该变量时用它们的值递归扩展，而不是声明它时的值

立即设置

VARIABLE := value

通过简单扩展内部值来设置变量 - 其中的值在声明时扩展。

如果不存在则延迟设置

VARIABLE ?= value

仅当变量没有值时才设置变量。value总是在VARIABLE访问时评估。它相当于

ifeq ($(origin VARIABLE), undefined)
  VARIABLE = value
endif

有关详细信息，请参阅文档。

附加

VARIABLE += value

将提供的值附加到现有值（如果变量不存在，则设置为该值）


更多建议

使用 = 会导致变量被赋值。如果变量已经有一个值，它将被替换。这个值在使用的时候会被扩展。例如：

HELLO = world
HELLO_WORLD = $(HELLO) world!

# This echoes "world world!"
echo $(HELLO_WORLD)

HELLO = hello

# This echoes "hello world!"
echo $(HELLO_WORLD)

使用 := 类似于使用=。但是，不是在使用时扩展值，而是在赋值期间扩展它。例如：

HELLO = world
HELLO_WORLD := $(HELLO) world!

# This echoes "world world!"
echo $(HELLO_WORLD)

HELLO = hello

# Still echoes "world world!"
echo $(HELLO_WORLD)

HELLO_WORLD := $(HELLO) world!

# This echoes "hello world!"
echo $(HELLO_WORLD)

如果 以前未分配变量，则使用为变量 ?= 分配一个值。如果变量之前被分配了一个空值（）， 我认为 它仍然被认为是集合。否则，功能完全一样。 VAR= =

using += 与 using 类似=，但不是替换值，而是将值附加到当前值，中间有一个空格。如果变量以前设置为:=， 我认为 它会被扩展。 结果值在我认为 使用时会扩展。例如：

HELLO_WORLD = hello
HELLO_WORLD += world!

# This echoes "hello world!"
echo $(HELLO_WORLD)

如果HELLO_WORLD = $(HELLO_WORLD) world!使用了类似的东西，就会导致递归，这很可能会结束 Makefile 的执行。如果A := $(A) $(B)使用 were，结果将与 using 不完全相同，+=因为Bis expanded with :=while+=不会导致B被扩展。
