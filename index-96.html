如何显示已上演的更改？

Arya

2023-02-16 21:13

我上演了一些要提交的更改。如何查看为下一次提交暂存的所有文件的差异？有一个方便的单线吗？

git status仅显示暂存文件的名称，但我想查看实际差异。

手册git-diff(1)页说：

git diff [--选项] [--] […]

这个表单是为了查看你相对于索引所做的更改（下一次提交的暂存区）。换句话说，差异是你 可以 告诉 git 进一步添加到索引但你仍然没有。您可以使用 git-add(1) 暂存这些更改。



解答

它应该只是：

git diff --cached

--cached表示针对当前HEAD. --staged是的同义词--cached。

--staged并--cached没有指向HEAD，只是相对于的区别HEAD。git add --patch如果您使用（或git add -p）挑选要提交的内容，--staged将返回上演的内容。


更多建议

一个简单的图形使这一点更清楚：

比较差异

显示工作目录和索引之间的变化。这显示了已更改但未准备提交的内容。

git diff --cached

显示索引和 HEAD 之间的变化（这是该分支上的最后一次提交）。这显示了已添加到索引并为提交暂存的内容。

git 差异头

显示工作目录和 HEAD 之间的所有变化（包括索引中的变化）。这显示了自上次提交以来的所有更改，无论它们是否已准备好提交。

执行“git export”（如“svn export”）？

Arya

2023-02-16 21:13

我一直想知道是否有一个好的“git export”解决方案可以在没有存储库目录的情况下创建树的副本.git。我至少知道三种方法：

  1. git clone然后删除.git存储库目录。
  2. git checkout-index暗指此功能，但以“只需将所需的树读入索引...”开头，我不完全确定该怎么做。
  3. git-export是一个第三方脚本，本质上是先git clone进入一个临时位置，然后rsync --exclude='.git'进入最终目的地。

这些解决方案都没有真正让我感到满意。最接近的svn export可能是选项 1，因为两者都要求目标目录首先为空。但是选项 2 似乎更好，假设我可以弄清楚将树读入索引的含义。



解答

实现这一目标的最简单方法可能是使用git archive. 如果你真的只需要展开的树，你可以这样做。

git archive master | tar -x -C /somewhere/else

大多数时候我需要从 git 中“导出”一些东西，无论如何我都想要一个压缩的存档，所以我会做这样的事情。

git archive master | bzip2 >source-tree.tar.bz2

压缩包：

git archive --format zip --output /full/path/to/zipfile.zip master

git help archive有关更多详细信息，它非常灵活。

请注意，即使存档不包含 .git 目录，它也会包含其他隐藏的特定于 git 的文件，如 .gitignore、.gitattributes 等。如果你不希望它们出现在存档中，请确保你在 .gitattributes 文件中使用 export-ignore 属性并在归档之前提交它。阅读更多...

注意：如果你对导出索引感兴趣，命令是

git checkout-index -a -f --prefix=/destination/path/

（有关详细信息，请参阅格雷格的回答）


更多建议

我发现了选项 2 的含义。从存储库中，您可以执行以下操作：

git checkout-index -a -f --prefix=/destination/path/

路径末尾的斜杠很重要，否则会导致文件位于 /destination 中，前缀为“路径”。

由于在正常情况下索引包含存储库的内容，因此“将所需的树读入索引”没有什么特别要做的。它已经在那里了。

该-a标志是检查索引中所有文件所必需的（我不确定在这种情况下省略该标志意味着什么，因为它不符合我的要求）。该-f标志强制覆盖输出中的任何现有文件，此命令通常不会这样做。

这似乎是我正在寻找的那种“git export”。

如何获取 Git 中当前提交的哈希值？

Arya

2023-02-16 21:12

如何在 Git 中获取当前提交的哈希值？



解答

要将任何扩展对象引用转换为散列，请使用 git-rev-parse ：

git rev-parse HEAD

或者

git rev-parse --verify HEAD

要检索短哈希：

git rev-parse --short HEAD

要将 引用 （例如 branches 和 tags ）转换为散列，请使用git show-refand git for-each-ref。


更多建议

要获取缩短的提交哈希，请使用%h格式说明符：

git log --pretty=format:'%h' -n 1

%H代表长提交哈希。此外，-1可以直接代替-n 1.
