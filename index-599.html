在 PostgreSQL 中创建数据库副本

Arya

2023-02-17 11:08

将整个数据库（其结构和数据）复制到 pgAdmin 中的新数据库的正确方法是什么？



解答

Postgres 允许在创建新数据库时使用服务器上的任何现有数据库作为模板。我不确定 pgAdmin 是否在创建数据库对话框中为您提供了选项，但如果没有，您应该能够在查询窗口中执行以下操作：

CREATE DATABASE newdb WITH TEMPLATE originaldb OWNER dbuser;

不过，您可能会得到：

ERROR:  source database "originaldb" is being accessed by other users

要断开所有其他用户与数据库的连接，您可以使用以下查询：

SELECT pg_terminate_backend(pg_stat_activity.pid) FROM pg_stat_activity 
WHERE pg_stat_activity.datname = 'originaldb' AND pid <> pg_backend_pid();


更多建议

贝尔答案的命令行版本：

createdb -O ownername -T originaldb newdb

这应该在数据库管理员的权限下运行，通常是 postgres。

AngularJS：调用 $scope.$apply() 时防止错误 $digest already in progress

Arya

2023-02-17 11:07

我发现自从以角度构建应用程序以来，我越来越需要手动将页面更新到我的范围。

$apply()我知道的唯一方法是从我的控制器和指令的范围内调用。这个问题是它不断向控制台抛出一个错误：

错误：$digest 已经在进行中

有谁知道如何避免此错误或以不同的方式实现相同的目标？



解答

不要使用这种模式 ——这最终会导致比它解决的错误更多的错误。即使您认为它解决了某些问题，但事实并非如此。

您可以$digest通过检查来检查 a 是否已经在进行中$scope.$$phase。

if(!$scope.$$phase) {
  //$digest or $apply
}

$scope.$$phase将返回"$digest"or"$apply"如果$digestor$apply正在进行中。我相信这些状态之间的区别在于$digest将处理当前作用域及其子作用域的手表，并将$apply处理所有作用域的观察者。

对于@dnc253 的观点，如果您发现自己经常打电话$digest或$apply打电话，那么您可能做错了。当由于在 Angular 范围之外触发​​ DOM 事件而需要更新范围的状态时，我通常发现我需要消化。例如，当 Twitter 引导程序模式被隐藏时。有时 DOM 事件会在 a$digest进行时触发，有时不会。这就是我使用这张支票的原因。

如果有人知道的话，我很想知道一种更好的方法。

来自评论：@anddoutoi

angular.js 反模式

  1. 不要这样做if (!$scope.$$phase) $scope.$apply()，这意味着您$scope.$apply()在调用堆栈中的位置不够高。

如何访问 Javascript 对象的第一个属性？

Arya

2023-02-17 11:06

有没有一种优雅的方法来访问对象的第一个属性......

  1. 你不知道你的房产名称的地方
  2. 不使用类似循环for .. in或 jQuery 的$.each

例如，我需要foo1在不知道 foo1 名称的情况下访问对象：

var example = {
    foo1: { /* stuff1 */},
    foo2: { /* stuff2 */},
    foo3: { /* stuff3 */}
};



解答

var obj = { first: 'someVal' };
obj[Object.keys(obj)[0]]; //returns 'someVal'

Object.values(obj)[0]; // returns 'someVal'

使用它，您还可以通过索引访问其他属性。请注意！Object.keys或Object.values根据 ECMAScript 不保证退货顺序，但所有主要浏览器实现都非正式地保证，请阅读https://stackoverflow.com/a/23202095了解详细信息。


更多建议

尝试for … in循环并在第一次迭代后中断：

for (var prop in object) {
    // object[prop]
    break;
}
