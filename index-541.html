自动或使用快捷方式删除尾随空格

Arya

2023-02-17 09:37

有没有办法删除 Visual Studio Code 中的尾随空格（自动或使用快捷方式）？

我一直在命令面板和编辑器设置中寻找它，但找不到我要找的东西。



解答

您可以在文件保存时 从设置中启用空格修剪：

  1. 打开 Visual Studio Code 用户设置（菜单 文件 → 首选项 → 设置 → 用户设置选项 卡）。
  2. 单击窗口右上角的图标。这将打开一个文档。
  3. 将新"files.trimTrailingWhitespace": true设置添加到用户设置文档（如果尚不存在）。这样您就不会直接编辑默认设置，而是添加它。
  4. 保存用户设置文件。

我们还添加了一个新命令来手动触发此操作（来自命令面板的 Trim Trailing Whitespace ）。


更多建议

菜单 代码 → 首选项 → 设置 →

检查 “修剪尾随空格” 选项 - “启用后，将在保存文件时修剪尾随空格” 。

我需要隐藏所有 权限被拒绝的 消息：

find . > files_and_folders

当出现这样的消息时，我正在试验。我需要收集所有没有出现的文件夹和文件。

是否可以将权限级别定向到files_and_folders文件？

如何同时隐藏错误？



解答

笔记：

  • 这个答案可能比用例保证更深入，并且 find 2>/dev/null在许多情况下可能已经足够好了。它可能仍然对跨平台的观点和它对一些高级 shell 技术的讨论感兴趣，以便找到尽可能健壮的解决方案，即使所防范的情况可能在很大程度上是假设的。

如果您的 shell 是bash或zsh，则有 一个解决方案既可靠又相当简单 ，仅使用 符合 POSIX 的find功能；虽然bash它本身不是 POSIX 的一部分，但大多数现代 Unix 平台都带有它，使这个解决方案具有广泛的可移植性：

find . > files_and_folders 2> >(grep -v 'Permission denied' >&2)

笔记：

  • 如果您的系统配置为显示 本地化 错误消息，请find在下面的调用前加上LC_ALL=C( LC_ALL=C find ...) 以确保报告 英文grep -v 'Permission denied'消息，以便按预期工作。但是，总是会显示任何错误消息， 然后 它也将以英文显示。

  • >(...)是一个（很少使用的） 输出 进程替换，它允许将输出（在本例中为 stderr output( 2>) 重定向到命令内部的标准输入>(...)。
    除了bashand之外zsh， 原则上ksh也支持它们，但试图将它们与重定向结合起来 stderr ，就像这里 ( ) 所做的那样，似乎被默默地忽略了 (在)。 2> >(...)``ksh 93u+

    • grep -v 'Permission denied'过滤 掉 ( ) 所有包含该短语的-v行（来自命令的 stderr 流）并将剩余行输出到 stderr ( )。find``Permission denied``>&2

    • grep注意：有些 的输出可能在完成 后 到达的可能性很小find，因为整个命令不会等待内部命令>(...)完成。在 中bash，您可以通过附加| cat到命令来防止这种情况。

这种方法是：

  • robust ：grep仅适用于 错误消息 （而不适用于文件路径和错误消息的组合，可能导致误报），并且除权限被拒绝的错误消息之外的错误消息被传递到 stderr。

  • side-effect free :find的退出代码被保留：无法访问遇到的至少一个文件系统项导致退出代码1（尽管这不会告诉您是否发生了除权限被拒绝之外的错误（也）） 。

符合 POSIX 标准的解决方案：

完全符合 POSIX 的解决方案要么有局限性，要么需要额外的工作。

如果 无论如何find都要在 文件 中捕获（或完全抑制）的输出，那么Jonathan Leffler 的答案中基于管道的解决方案简单、健壮且符合 POSIX 标准：

find . 2>&1 >files_and_folders | grep -v 'Permission denied' >&2

请注意，重定向的顺序很重要：2>&1必须 先到 。

预先在文件中捕获 stdout 输出允许通过管道 仅2>&1发送错误消息，然后可以明确地对其进行操作。 __grep

唯一的 缺点是 整个退出代码 将是grep命令的，而不是find，在这种情况下意味着：如果根本 没有错误或 只有 权限被拒绝的错误，退出代码将是1（信号 失败 ），否则（除了拒绝许可之外的错误）0- 这与意图相反。
也就是说，无论如何find很少使用退出代码，因为除了 基本 故障（例如传递不存在的路径）之外，它通常传达的信息很少。
然而，具体的案例甚至只有 一些 由于缺少权限而无法访问的输入路径的一部分 反映 在find的退出代码中（在 GNU 和 BSD 中）：如果处理的 任何 文件find发生权限拒绝错误，则退出代码设置为. __1

以下变体解决了这个问题：

find . 2>&1 >files_and_folders | { grep -v 'Permission denied' >&2; [ $? -eq 1 ]; }

现在，退出代码指示是否发生了 除此之外的 Permission denied任何错误：1如果是，0否则。
换句话说：退出代码现在反映了命令的真实意图：0如果根本没有错误或 仅 发生权限被拒绝的错误，则报告成功 ()。
可以说，这比仅传递 的find退出代码更好，如顶部的解决方案所示。

gniourf_gniourf在评论中提出了一个（仍然符合 POSIX 标准的） 使用复杂重定向的解决方案的概括 ，它 甚至可以使用将文件路径打印到 stdout 的默认行为：

{ find . 3>&2 2>&1 1>&3 | grep -v 'Permission denied' >&3; } 3>&2 2>&1

简而言之：自定义文件描述符3用于临时交换 stdout ( 1) 和 stderr ( 2)，以便可以通过 stdout 单独 传输错误消息grep。

如果没有这些重定向，数据（文件路径） 和 错误消息都将通过 stdout 传输到管道grep，然后grep将无法区分 错误消息 Permission denied和 名称恰好包含 短语. 的（假设的）文件Permission denied。

然而，与第一个解决方案一样，报告的退出代码将是grep's，而不是find's，但可以应用与上述相同的修复。

什么是实现多个构造函数的干净的“pythonic”方式？

Arya

2023-02-17 09:36

我找不到明确的答案。__init__据我所知，一个 Python 类中不能有多个函数。那么我该如何解决这个问题呢？

Cheese假设我有一个用属性调用的类number_of_holes。我怎样才能有两种创建奶酪对象的方法......

  1. 一个像这样需要很多洞的人：parmesan = Cheese(num_holes = 15)。
  2. 一个不接受任何参数而只是随机化number_of_holes属性的方法：gouda = Cheese()。

我只能想到一种方法来做到这一点，但这看起来很笨重：

class Cheese():
    def __init__(self, num_holes = 0):
        if (num_holes == 0):
            # Randomize number_of_holes
        else:
            number_of_holes = num_holes

你怎么说？还有别的办法吗？



解答

实际上None对于“魔术”值要好得多：

class Cheese():
    def __init__(self, num_holes = None):
        if num_holes is None:
            ...

现在，如果您想要完全自由地添加更多参数：

class Cheese():
    def __init__(self, *args, **kwargs):
        #args -- tuple of anonymous arguments
        #kwargs -- dictionary of named arguments
        self.num_holes = kwargs.get('num_holes',random_holes())

*args为了更好地解释and的概念**kwargs（您实际上可以更改这些名称）：

def f(*args, **kwargs):
   print 'args: ', args, ' kwargs: ', kwargs

>>> f('a')
args:  ('a',)  kwargs:  {}
>>> f(ar='a')
args:  ()  kwargs:  {'ar': 'a'}
>>> f(1,2,param=3)
args:  (1, 2)  kwargs:  {'param': 3}

http://docs.python.org/reference/expressions.html#calls


更多建议

如果num_holes=None您只需要__init__.

如果你想要多个独立的“构造函数”，你可以将它们作为类方法提供。这些通常称为工厂方法。在这种情况下，您可以使用默认值num_holesbe 0。

class Cheese(object):
    def __init__(self, num_holes=0):
        "defaults to a solid cheese"
        self.number_of_holes = num_holes

    @classmethod
    def random(cls):
        return cls(randint(0, 100))

    @classmethod
    def slightly_holey(cls):
        return cls(randint(0, 33))

    @classmethod
    def very_holey(cls):
        return cls(randint(66, 100))

现在像这样创建对象：

gouda = Cheese()
emmentaler = Cheese.random()
leerdammer = Cheese.slightly_holey()
