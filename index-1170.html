你如何在 Javascript 中四舍五入到小数点后一位？

Arya

2023-02-18 03:08

您可以将 javascript 中的数字四舍五入到小数点后的 1 个字符（正确四舍五入）吗？

我尝试了 *10、round、/10，但它在 int 的末尾留下了两位小数。



解答

Math.round(num * 10) / 10有效，这是一个例子......

var number = 12.3456789
var rounded = Math.round(number * 10) / 10
// rounded is 12.3

如果你希望它有一位小数，即使那是 0，那么添加...

var fixed = rounded.toFixed(1)
// fixed is always to 1 d.p.
// NOTE: .toFixed() returns a string!

// To convert back to number format
parseFloat(number.toFixed(2))
// 12.34
// but that will not retain any trailing zeros

// So, just make sure it is the last step before output,
// and use a number format during calculations!

编辑：添加带有精度函数的圆...

使用这个原则，作为参考，这里有一个方便的小圆函数，它需要精确...

function round(value, precision) {
    var multiplier = Math.pow(10, precision || 0);
    return Math.round(value * multiplier) / multiplier;
}

... 用法 ...

round(12345.6789, 2) // 12345.68
round(12345.6789, 1) // 12345.7

...默认舍入到最接近的整数（精度为 0）...

round(12345.6789) // 12346

...并可用于四舍五入到最接近的 10 或 100 等...

round(12345.6789, -1) // 12350
round(12345.6789, -2) // 12300

...以及正确处理负数...

round(-123.45, 1) // -123.4
round(123.45, 1) // 123.5

... 并且可以与 toFixed 结合使用以一致地格式化为字符串 ...

round(456.7, 2).toFixed(2) // "456.70"


更多建议

var number = 123.456;

console.log(number.toFixed(1)); // should round to 123.5

解答

最初，我在 2012 年提到 git archive （参见Jared Forsyth的回答和Robert Knight的回答），自git1.7.9.5（2012 年 3 月）以来，Paul Brannan的回答：

git archive --format=tar --remote=origin HEAD:path/to/directory -- filename | tar -O -xf -

但是：在 2013 年，远程 https://github.com URL不再可能。
请参阅旧页面“我可以存档存储库吗？ ”

当前（2018 年）页面“关于在 GitHub 上存档内容和数据”建议使用第三方服务，例如GHTorrent或GH Archive。

所以你也可以处理本地副本/克隆：

如果您有本答案中提到的裸存储库的本地副本，您也可以执行以下操作，

git --no-pager --git-dir /path/to/bar/repo.git show branch:path/to/file >file

或者你必须首先克隆 repo，这意味着你可以获得完整的历史记录：

    • 启用稀疏结帐选项 ( git config core.sparsecheckout true)

    • 重新阅读工作树以仅显示您需要的内容

重新阅读工作树：

$ git read-tree -m -u HEAD

这样，您最终会得到一个工作树，其中包含您想要的内容（即使它只是一个文件）

在稀疏结帐文件中，前者以以下形式出现：

而后者成为一对形式的模式：

或者在匹配顶层文件的特殊情况下：

/A/B/C

当这种模式位于稀疏结帐文件中并且启用了圆锥模式时，这会导致稀疏结帐表现出意外行为。

具体来说，对于上述模式，非锥形模式下的稀疏结帐将仅包含位于“ /A/B/C”的目录或文件。

什么时候用Task.Delay，什么时候用Thread.Sleep？

Arya

2023-02-18 03:07

是否有关于何时使用Task.Delay与Thread.Sleep的良好规则？

  • 具体来说，是否有最低价值可以提供一个比另一个有效/高效的值？
  • 最后，由于 Task.Delay 导致异步/等待状态机上的上下文切换，使用它是否有开销？



解答

Thread.Sleep当你想阻塞当前线程时使用。

await Task.Delay当您想要逻辑延迟而不阻塞当前线程时使用。

效率不应该是这些方法的首要问题。它们在现实世界中的主要用途是作为 I/O 操作的重试计时器，以秒而不是毫秒为单位。


更多建议

Task.Delay和之间最大的区别Thread.Sleep是Task.Delay旨在异步运行。在同步代码中使用没有意义Task.Delay。在异步代码中使用是一个非常糟糕的主意Thread.Sleep。

通常你会Task.Delay()用await关键字调用：

await Task.Delay(5000);

或者，如果您想在延迟之前运行一些代码：

var sw = new Stopwatch();
sw.Start();
Task delay = Task.Delay(5000);
Console.WriteLine("async: Running for {0} seconds", sw.Elapsed.TotalSeconds);
await delay;

猜猜这会打印什么？运行 0.0070048 秒。如果我们移动await delay上面Console.WriteLine的，它将打印 Running for 5.0020168 seconds。

让我们看看与 的区别Thread.Sleep：

class Program
{
    static void Main(string[] args)
    {
        Task delay = asyncTask();
        syncCode();
        delay.Wait();
        Console.ReadLine();
    }

    static async Task asyncTask()
    {
        var sw = new Stopwatch();
        sw.Start();
        Console.WriteLine("async: Starting");
        Task delay = Task.Delay(5000);
        Console.WriteLine("async: Running for {0} seconds", sw.Elapsed.TotalSeconds);
        await delay;
        Console.WriteLine("async: Running for {0} seconds", sw.Elapsed.TotalSeconds);
        Console.WriteLine("async: Done");
    }

    static void syncCode()
    {
        var sw = new Stopwatch();
        sw.Start();
        Console.WriteLine("sync: Starting");
        Thread.Sleep(5000);
        Console.WriteLine("sync: Running for {0} seconds", sw.Elapsed.TotalSeconds);
        Console.WriteLine("sync: Done");
    }
}

尝试预测这将打印什么......

异步：启动
异步：运行 0.0070048 秒
同步：启动
异步：运行 5.0119008 秒
异步：完成
同步：运行 5.0020168 秒
同步：完成

此外，有趣的是，它要准确Thread.Sleep得多，ms 精度并不是真正的问题，而Task.Delay最少需要 15-30ms。与它们具有的 ms 精度相比，这两个函数的开销是最小的（Stopwatch如果您需要更精确的东西，请使用 Class）。Thread.Sleep仍然占用你的线程，Task.Delay在你等待的时候释放它去做其他的工作。
