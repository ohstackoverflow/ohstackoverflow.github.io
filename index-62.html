如何在 Git 存储库中查找和恢复已删除的文件？

Arya

2023-02-16 20:06

假设我在 Git 存储库中。我删除一个文件并提交该更改。我继续工作并做出更多承诺。然后，我发现我需要在删除该文件后恢复该文件。

我知道我可以使用 签出文件git checkout <commit> -- filename.txt，但我不知道该文件何时被删除。

  1. 如何找到删除给定文件名的提交？
  2. 我如何将该文件恢复到我的工作副本中？



解答

查找影响给定路径的最后一次提交。由于该文件不在 HEAD 提交中，因此之前的提交必须已将其删除。

git rev-list -n 1 HEAD -- <file_path>

然后使用脱字符号 ( ) 签出之前提交时的版本^：

git checkout <deleting_commit>^ -- <file_path>

或者在一个命令中，if$file是有问题的文件。

git checkout $(git rev-list -n 1 HEAD -- "$file")^ -- "$file"

如果您使用的是 zsh 并启用了 EXTENDED_GLOB 选项，插入符号将不起作用。您可以改用~1。

git checkout $(git rev-list -n 1 HEAD -- "$file")~1 -- "$file"


更多建议

  1. 获取所有已删除文件的提交，以及已删除的文件：
    git log --diff-filter=D --summary

记下所需的提交哈希，例如e4e6d4d5e5c59c69f3bd7be2.

  1. 将删除的文件从之前的一个提交 ( ~1) 恢复到上面确定的提交 ( e4e6d4d5e5c59c69f3bd7be2)：
    git checkout e4e6d4d5e5c59c69f3bd7be2~1 path/to/file.ext

注意~1.

使用 Git 挑选提交意味着什么？

Arya

2023-02-16 20:05

做什么git cherry-pick <commit>？



解答

Git 中的 Cherry picking 意味着从一个分支中选择一个提交并将其应用到另一个分支。

这与其他方式形成对比，例如merge和rebase通常将许多提交应用到另一个分支。

也可以cherry-pick 多个提交，但合并是 cherry-picking 的首选方式。

  1. 确保您在要应用提交的分支上。

     git switch master

  2. 执行以下操作：

     git cherry-pick <commit-hash>

注意：

  1. 如果您从公共分支机构挑选，您应该考虑使用
     git cherry-pick -x <commit-hash>

这将生成标准化的提交消息。这样，您（和您的同事）仍然可以跟踪提交的来源，并可能避免将来发生合并冲突。

  1. 如果您在提交上附有注释，则它们不会遵循 cherry-pick。要将它们也带过来，您必须使用：
     git notes copy <from> <to>

附加链接：

  • git官方指南页面
  • git cherry pick 指南


更多建议

这段引自：Version Control with Git

使用 git cherry-pick 命令 git cherry-pick commit 应用当前分支上指定提交引入的更改。它将引入一个新的、独特的提交。 严格来说，使用 git cherry-pick 不会改变存储库中的现有历史记录； 相反，它增加了历史。与通过应用差异的过程引入更改的其他 Git 操作一样，您可能需要解决冲突以完全应用给定提交的更改。命令 git cherry- pick 通常用于将特定提交从存储库中的一个分支引入到另一个分支。一个常见的用途是将提交从维护分支转发或反向移植到开发分支。

$ git checkout rel_2.3
$ git cherry-pick dev~2 # commit F, below

如何测试具有私有方法、字段或内部类的类？

Arya

2023-02-16 20:04

如何使用 JUnit 测试具有内部私有方法、字段或嵌套类的类？

仅仅为了能够运行测试而更改方法的访问修饰符似乎很糟糕。



解答

如果您有一些遗留 Java 应用程序，并且不允许更改方法的可见性，那么测试私有方法的最佳方法是使用反射。

在内部，我们使用助手来获取/设置private和private static变量以及调用private和private static方法。以下模式将使您可以执行与私有方法和字段相关的几乎所有操作。当然，你不能通过private static final反射改变变量。

Method method = TargetClass.getDeclaredMethod(methodName, argClasses);
method.setAccessible(true);
return method.invoke(targetObject, argObjects);

对于字段：

Field field = TargetClass.getDeclaredField(fieldName);
field.setAccessible(true);
field.set(object, value);

笔记：

  1. TargetClass.getDeclaredMethod(methodName, argClasses)让您研究private方法。同样的事情适用于getDeclaredField.
  2. 需要setAccessible(true)与私人玩耍。


更多建议

测试私有方法的最佳方式是通过另一个公共方法。如果无法做到这一点，则以下条件之一为真：

  1. 私有方法是死代码
  2. 您正在测试的类附近有设计气味
  3. 您尝试测试的方法不应该是私有的
