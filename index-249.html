  • #!/bin/sh -
  • ETC

我依稀记得很久以前听说在末尾添加破折号可以防止有人向您的脚本传递命令，但找不到任何详细信息。



解答

您应该使用 #!/usr/bin/env bash 可移植性：不同的 nixes 放在不同的地方，使用是运行第一个在. 而 不是 。bash``/usr/bin/env``bash``PATH sh``bash*


更多建议

我建议使用：

#!/bin/bash

它不是 100% 可移植的（一些系统放置bash在 以外的位置/bin），但许多现有脚本使用#!/bin/bash压力的事实迫使各种操作系统/bin/bash至少建立到主要位置的符号链接。

替代方案：

#!/usr/bin/env bash

已被建议——但不能保证该env命令在/usr/bin（而且我使用过它不在的系统）。bash此外，此表单将使用当前用户中的第一个实例$PATH，这可能不是合适的 bash shell 版本。

（但/usr/bin/env应该可以在任何相当现代的系统上工作，因为env它在/usr/bin或因为系统做了一些事情让它工作。我上面提到的系统是 SunOS 4，我可能已经有 25 年没有使用它了。）

git stash pop 和 git stash apply 的区别

Arya

2023-02-17 02:07

我已经使用了git stash pop一段时间了。我最近发现了这个git stash apply命令。当我尝试它时，它似乎与git stash pop.

git stash pop和 和有什么不一样git stash apply？



解答

git stash pop 在应用它之后丢弃 （默认情况下最上面的）存储，而git stash apply 将它留在存储列表中 以供以后可能重用（或者你可以然后git stash drop它）。

除非在 之后发生冲突，否则会发生这种情况git stash pop，在这种情况下，它不会删除存储，让它的行为与 完全一样git stash apply。

另一种看待它的方式：git stash pop是git stash apply && git stash drop。


更多建议

得到了这个有用的链接，它说明了差异，正如 John Zwinck 所说的那样，以及git stash pop.

例如，假设您的隐藏更改与您首次创建隐藏后所做的其他更改冲突。pop 和 apply 都将有助于触发合并冲突解决模式，使您能够很好地解决此类冲突……并且都不会摆脱隐藏，即使您可能也期待 pop。由于很多人都希望 stashes 只是一个简单的堆栈，这通常会导致他们后来不小心弹出相同的 stashes，因为他们认为它已经消失了。

如何在 TypeScript 中将字符串转换为数字？

Arya

2023-02-17 02:07

给定数字的字符串表示形式，如何将其转换为numberTypeScript 类型？

var numberString: string = "1234";
var numberValue: number = /* what should I do with `numberString`? */;



解答

与在 JavaScript 中完全一样，您可以使用parseIntorparseFloat函数，或者简单地使用一元+运算符：

var x = "32";
var y: number = +x;

所有提到的技术都将具有正确的类型，并将正确解析简单的十进制整数字符串，如"123"，但对于其他各种可能预期的情况（如"123.45"）和极端情况（如null），其行为会有所不同。
