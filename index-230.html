如何在 MySQL 中重置 AUTO_INCREMENT

Arya

2023-02-17 01:37

我怎样才能 _ 重置_AUTO_INCREMENT的？

我希望它重新开始计数1。



解答

您可以使用以下方法重置计数器：

ALTER TABLE tablename AUTO_INCREMENT = 1

对于InnoDB，您不能将auto_increment值设置为低于或等于当前最高索引。（引自ViralPatel）：

请注意，您不能将计数器重置为小于或等于任何已使用的值。对于 MyISAM，如果该值小于或等于 AUTO_INCREMENT 列中当前的最大值，则将该值重置为当前最大值加一。对于InnoDB，如果该值小于当前列中的最大值，则不会发生错误，并且不会更改当前序列值。

请参阅 如何使用另一个表中的 MAX 值重置 MySQL AutoIncrement？ 关于如何动态获得可接受的值。


更多建议

SET  @num := 0;

UPDATE your_table SET id = @num := (@num+1);

ALTER TABLE your_table AUTO_INCREMENT =1;

如何重命名 Ruby on Rails 迁移中的数据库列？

Arya

2023-02-17 01:36

我错误地命名了一个列hased_password而不是hashed_password.

如何更新数据库架构，使用迁移重命名此列？



解答

rename_column :table, :old_column, :new_column

您可能想要创建一个单独的迁移来执行此操作。（FixColumnName随意重命名。）：

bin/rails generate migration FixColumnName
# creates  db/migrate/xxxxxxxxxx_fix_column_name.rb

然后编辑迁移以执行您的意愿：

# db/migrate/xxxxxxxxxx_fix_column_name.rb
class FixColumnName < ActiveRecord::Migration
  def self.up
    rename_column :table_name, :old_column, :new_column
  end

  def self.down
    # rename back if you need or do something else or do nothing
  end
end

对于 Rails 3.1 使用：

虽然up和down方法仍然适用，但 Rails 3.1 收到了一种change方法，它“知道如何迁移您的数据库并在迁移回滚时将其撤消，而无需编写单独的向下方法”。

有关详细信息，请参阅“活动记录迁移”。

rails g migration FixColumnName

class FixColumnName < ActiveRecord::Migration
  def change
    rename_column :table_name, :old_column, :new_column
  end
end

如果您碰巧有一大堆要重命名的列，或者需要一遍又一遍地重复表名的东西：

rename_column :table_name, :old_column1, :new_column1
rename_column :table_name, :old_column2, :new_column2
...

你可以用来change_table让事情变得更整洁：

class FixColumnNames < ActiveRecord::Migration
  def change
    change_table :table_name do |t|
      t.rename :old_column1, :new_column1
      t.rename :old_column2, :new_column2
      ...
    end
  end
end

然后就像db:migrate往常一样，或者你如何处理你的业务。

对于 Rails 4：

在创建Migration用于重命名列的 时，Rails 4 会生成一个change方法，而不是上一节中提到的up和。down生成的change方法是：

$ > rails g migration ChangeColumnName

这将创建一个类似于以下内容的迁移文件：

class ChangeColumnName < ActiveRecord::Migration
  def change
    rename_column :table_name, :old_column, :new_column
  end
end


更多建议

在我看来，在这种情况下，最好使用rake db:rollback，然后编辑您的迁移并再次运行rake db:migrate。

但是，如果您不想丢失列中的数据，则使用rename_column.

Android Studio 中的“assets”文件夹放在哪里？

Arya

2023-02-17 01:36

我对文件夹感到困惑assets。它不是在 Android Studio 中自动创建的，几乎所有讨论它的论坛都在谈论 Eclipse。

Assets目录在Android Studio中如何配置？



解答

由于 Android Studio 使用新的基于 Gradle 的构建系统，您应该将assets/其放入源集（例如，src/main/assets/）。

在一个典型的 Android Studio 项目中，您将拥有一个带有源集app/的模块main/（app/src/main/在项目根目录之外），因此您的主要资产将进入app/src/main/assets/. 然而：

  • 如果您需要特定于构建类型的资产，例如debugvs release，您可以为这些角色创建源集（例如，。app/src/release/assets/）

  • 您的产品口味也可以具有带有资产的源集（例如，app/src/googleplay/assets/）

  • 您的仪器测试可以有一个androidTest带有自定义资产（例如，app/src/androidTest/assets/）的源集，但一定要询问InstrumentationRegistryfor getContext()，而不是getTargetContext()，以访问这些资产

另外，快速提醒一下：资产在运行时是只读的。使用内部存储、外部存储或存储访问框架来读取/写入内容。


更多建议

让 Android Studio 为您代劳。

  1. 在 Android Studio（ 1.0 及更高版本 ）中，右键单击文件夹并导航到 Assets Folder.

  1. 在下一个屏幕上，只需单击 Finish 。

瞧！它将 assets 在 main 目标源集中创建文件夹。
