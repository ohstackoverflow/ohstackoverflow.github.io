“下游”和“上游”的定义

Arya

2023-02-17 07:01

我开始使用 Git 并遇到了“上游”和“下游”这两个术语。我以前见过这些，但从未完全理解它们。这些术语在 SCM（软件配置管理工具）和源代码的上下文中意味着什么？



解答

在源代码控制方面，当您从存储库复制（克隆、签出等）时，您处于下游。信息“顺流”流向你。

当您进行更改时，您通常希望将它们发送回“上游”，以便它们进入该存储库，以便从同一来源提取的每个人都在处理所有相同的更改。这主要是一个社会问题，即每个人如何协调他们的工作，而不是源代码控制的技术要求。您希望将更改放入主项目中，这样您就不会跟踪不同的开发路线。

有时您会读到包或发布经理（人，而不是工具）谈论将更改提交到“上游”。这通常意味着他们必须调整原始来源，以便他们可以为他们的系统创建一个包。他们不想继续进行这些更改，因此如果他们将它们“上游”发送到原始来源，他们就不必在下一个版本中处理相同的问题。


更多建议

当您阅读git tag手册页时：

git 的一个重要方面是它是分布式的，而分布式在很大程度上意味着系统中没有固有的“上游”或“下游”。

，这仅仅 意味着没有 绝对的 上游回购或下游回购。
这些概念在两个回购协议之间总是相关的，并且取决于数据流动的方式：

如果“yourRepo”已将“otherRepo”声明为远程，则 ：

  • 您是 从上游 “otherRepo”拉取的（“otherRepo”是“您的上游 ” ，而您是“ otherRepo 的 下游”）。
  • 您正在 向上游推送 （“otherRepo”仍然是“上游”，信息现在返回到那里）。

注意“来自”和“为了”：你不仅仅是“下游”，你是“来自 /为了 ”的下游，因此是相对的方面。

DVCS（分布式版本控制系统）的转折是：你不知道下游实际上是什么，除了你自己的 repo 相对于你声明的远程 repos。

  • 你知道上游是什么（你从中提取或推送的回购协议）
  • 您不知道下游是由什么组成的（其他回购从您的回购中拉出或推送到 您的回购 ）。

基本上：

就“数据流 ”而言，您的回购位于来自上游回购（“拉自”）并返回（相同或其他）上游回购（“推入”）的流量的底部（“下游”） ).

git-rebase您可以在手册页中看到带有“从上游 REBASE 恢复”段落的插图：

这意味着您正在 从发生变基的“上游”回购中拉出 ，并且您（“下游”回购）陷入后果（大量重复提交，因为上游变基的分支重新创建了与您相同的分支的提交有本地）。

这很糟糕，因为对于一个“上游”回购协议，可能有 许多 下游回购协议（即从上游回购协议中拉取的回购协议，带有重新设置的分支），所有这些回购协议都有可能处理重复提交。

同样，与“数据流”类比，在 DVCS 中，“上游”的一个错误命令可能对下游产生“ 涟漪效应 ”。

注意：这不仅限于数据。
它也适用于 parameters ，因为 git 命令（如“瓷器”命令）经常在内部调用其他 git 命令（“管道”命令）。请参阅rev- parse手册页：

许多 git 瓷器命令混合使用标志（即以破折号 ' ' 开头的参数-）和参数，这些标志和参数用于git rev- list它们在内部使用的底层命令， 以及用于它们在 . 下游使用的其他命令的标志和参数git rev-list。该命令用于区分它们。

shell 命令到 tar 目录，不包括某些文件/文件夹

Arya

2023-02-17 07:01

是否有一个简单的 shell 命令/脚本支持从归档中排除某些文件/文件夹？

我有一个目录需要与一个子目录一起存档，该子目录包含许多我不需要备份的非常大的文件。

不完全解决方案：

该tar --exclude=PATTERN命令匹配给定的模式并排除那些文件，但我需要忽略特定文件和文件夹（完整文件路径），否则可能会排除有效文件。

我还可以使用 find 命令创建文件列表并排除我不想存档的文件并将列表传递给 tar，但这只适用于少量文件。我有几万。

我开始认为唯一的解决方案是创建一个包含要排除的文件/文件夹列表的文件，然后使用 rsync--exclude-from=file将所有文件复制到 tmp 目录，然后使用 tar 存档该目录。

有人能想到更好/更有效的解决方案吗？

编辑： Charles Ma 的解决方案效果很好。最大的问题是--exclude='./folder' 必须 在 tar 命令的开头。完整命令（首先是 cd，因此备份是相对于该目录的）：

cd /folder_to_backup
tar --exclude='./folder' --exclude='./upload/folder2' -zcvf /backup/filename.tgz .



解答

您可以为 tar 设置多个排除选项，因此

$ tar --exclude='./folder' --exclude='./upload/folder2' -zcvf /backup/filename.tgz .

等会工作。确保放在源和目标项目 之前--exclude 。


更多建议

您可以使用--excludefor tar 排除目录。

如果你想存档除了/usr你可以使用的所有东西：

tar -zcvf /all.tgz / --exclude=/usr

在你的情况下可能是这样的

tar -zcvf archive.tgz arc_dir --exclude=dir/ignore_this_dir
