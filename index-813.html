同一台计算机上的多个 GitHub 帐户？

Arya

2023-02-17 16:48

尝试在我的计算机上处​​理我实际的“工作”回购和 GitHub 上的回购。

首先设置了工作帐户，一切正常。

但是，我的帐户似乎无法推送到我的存储库，该存储库是在不同的帐户/电子邮件下设置的。

我试过将我的工作密钥复制到我的帐户，但这会引发错误，因为密钥当然只能附加到一个帐户。

如何使用各自的 GitHub 凭据向两个帐户推送/拉取它们？



解答

您需要做的就是使用多个 SSH 密钥对配置 SSH 设置。

  • 此链接很容易访问（感谢 Eric）：

http://code.tutsplus.com/tutorials/quick-tip-how-to-work-with-github-and- multiple-accounts--net-22574

  • 生成 SSH 密钥 (Win/msysgit)：

https://help.github.com/articles/generating-an-ssh-key/

第一个链接的相关步骤：

  1. 生成 SSH 密钥：
    ssh-keygen -t ed25519 -C "john@doe.example.com"

按照提示确定一个名称，例如id_ed25519_example_company.

  1. 将 SSH 公钥从 GitHub 复制~/.ssh/id_ed25519_doe_company.pub并告诉 ssh 密钥：

    ssh-add ~/.ssh/id_ed25519_doe_company

  2. 在其中创建一个包含以下内容的config文件：~/.ssh

    Host github-doe-company

    HostName github.com User git IdentityFile ~/.ssh/id_ed25519_doe_company

  3. 添加您的遥控器：

    git remote add origin git@github-doe-company:username/repo.git

或更改使用：

    git remote set-url origin git@github-doe-company:username/repo.git

此外，如果您使用不同的角色处理多个存储库，则需要确保您的各个存储库相应地覆盖了用户设置：

设置用户名、电子邮件和 GitHub 令牌——覆盖个人存储库的设置https://help.github.com/articles/setting- your-commit-email-address-in-git/

注意： 你们中的一些人可能需要不同的电子邮件用于不同的存储库，从 git 2.13 开始，您可以通过编辑位于以下位置的全局配置文件来在目录基础上设置电子邮件：~/.gitconfig使用条件，如下所示：

[user]
    name = Default Name
    email = defaultemail@example.com

[includeIf "gitdir:~/work/"]
    path = ~/work/.gitconfig

然后您的工作特定配置~/work/.gitconfig将如下所示：

[user]
    name = Pavan Kataria
    email = pavan.kataria@example.com

感谢@alexg在评论中告知我这一点。


更多建议

使用 HTTPS：

将远程 url 更改为 https:

git remote set-url origin https://USERNAME@github.com/USERNAME/PROJECTNAME.git

你很高兴去：

git push

为确保提交显示为由 USERNAME 执行，还可以为此项目设置user.name和user.email ：

git config user.name USERNAME
git config user.email USERNAME@example.com

@class 与 #import

Arya

2023-02-17 16:47

据我了解，如果 ClassA 需要包含 ClassB 标头，而 ClassB 需要包含 ClassA 标头以避免任何循环包含，则应该使用前向类声明。我也明白 an#import很简单ifndef，因此包含只发生一次。

我的询问是这样的：什么时候使用#import，什么时候使用@class？有时如果我使用@class声明，我会看到一个常见的编译器警告，如下所示：

warning: receiver 'FooController' is a forward class and corresponding @interface may not exist.

真的很想理解这一点，而不是仅仅删除@class前向声明并抛出一个#importin 来消除编译器给我的警告。



解答

如果您看到此警告：

警告：接收器“MyCoolClass”是前向类，对应的@interface 可能不存在

你需要#import这个文件，但你可以在你的实现文件（.m）中做到这一点，并@class在你的头文件中使用声明。

@class不会（通常）消除对文件的需求#import，它只是将需求降低到更接近信息有用的地方。

例如

如果你说@class MyCoolClass，编译器知道它可能会看到类似这样的东西：

MyCoolClass *myObject;

除了是一个有效的类之外，它不必担心任何其他事情MyCoolClass，并且它应该为指向它的指针预留空间（真的，只是一个指针）。因此，在您的标头中，@class90% 的时间就足够了。

但是，如果您需要创建或访问myObject的成员，则需要让编译器知道这些方法是什么。此时（可能在您的实现文件中），您需要#import "MyCoolClass.h"告诉编译器除了“这是一个类”之外的其他信息。


更多建议

三个简单的规则：

  • 头文件（文件）中只有#import超类和采用的协议.h。
  • #import所有类和协议，您在实现中将消息发送到（.m文件）。
  • 转发其他所有内容的声明。

如果你在实现文件中做前向声明，那么你可能做错了什么。

Go 中的可选参数？

Arya

2023-02-17 16:47

Go 可以有可选参数吗？或者我可以只定义两个具有相同名称和不同数量参数的不同函数吗？



解答

Go 没有可选参数，也不支持方法重载：

如果方法分派也不需要进行类型匹配，则它会得到简化。使用其他语言的经验告诉我们，具有相同名称但不同签名的各种方法偶尔有用，但在实践中也可能令人困惑和脆弱。仅按名称匹配并要求类型一致是 Go 类型系统中的一个主要简化决策。


更多建议

实现诸如可选参数之类的东西的一个好方法是使用可变参数。该函数实际上接收您指定的任何类型的切片。

func foo(params ...int) {
    fmt.Println(len(params))
}

func main() {
    foo()
    foo(1)
    foo(1,2,3)
}
