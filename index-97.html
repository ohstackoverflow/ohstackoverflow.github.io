使用 jQuery 禁用/启用输入？

Arya

2023-02-16 21:15 

$input.disabled = true;

或者

$input.disabled = "disabled";

哪个是标准方法？反过来，如何启用禁用的输入？



解答

jQuery 1.6+

要更改disabled属性，您应该使用该.prop()功能。

$("input").prop('disabled', true);
$("input").prop('disabled', false);

jQuery 1.5 及以下版本

该.prop()函数不存在，但.attr()类似：

设置禁用属性。

$("input").attr('disabled','disabled');

要再次启用，正确的方法是使用.removeAttr()

$("input").removeAttr('disabled');

在任何版本的 jQuery 中

你总是可以依赖实际的 DOM 对象，如果你只处理一个元素，它可能比其他两个选项快一点：

// assuming an event handler thus 'this'
this.disabled = true;

.prop()使用or方法的优点.attr()是您可以为一堆选定的项目设置属性。

注意： 在 1.6 中，有一种.removeProp()方法听起来很像removeAttr()，但 不应将其用于 本机属性，例如'disabled'文档摘录：

注意：不要使用此方法删除本机属性，例如已选中、已禁用或已选中。这将完全删除该属性，并且一旦删除，就无法再次添加到元素中。使用 .prop() 将这些属性设置为 false。

事实上，我怀疑这个方法有很多合法用途，布尔道具的完成方式是你应该将它们设置为 false 而不是像 1.5 中的“属性”对应物那样“删除”它们


更多建议

只是为了新的约定 && 使其适应性向前发展（除非 ECMA6(?????) 发生巨大变化）：

$(document).on('event_name', '#your_id', function() {
    $(this).removeAttr('disabled');
});

和

$(document).off('event_name', '#your_id', function() {
    $(this).attr('disabled','disabled');   
});

一次捕获多个异常？

Arya

2023-02-16 21:14

不鼓励简单地捕捉System.Exception。相反，应该只捕获“已知”异常。

现在，这有时会导致不必要的重复代码，例如：

try
{
    WebId = new Guid(queryString["web"]);
}
catch (FormatException)
{
    WebId = Guid.Empty;
}
catch (OverflowException)
{
    WebId = Guid.Empty;
}

我想知道：有没有办法捕获这两个异常并只调用WebId = Guid.Empty一次？

给定的示例相当简单，因为它只是一个GUID. 但是想象一下您多次修改一个对象的代码，如果其中一个操作预期失败，您想要“重置” object. 但是，如果出现意外异常，我还是想抛得高一点。



解答

捕捉System.Exception并打开类型

catch (Exception ex)            
{                
    if (ex is FormatException || ex is OverflowException)
    {
        WebId = Guid.Empty;
        return;
    }

    throw;
}

.gitignore 和 .gitkeep 之间有什么区别？

Arya

2023-02-16 21:14

.gitignore和之间有什么区别.gitkeep？它们是名称不同的同一事物，还是它们都具有不同的功能？

我似乎无法找到关于.gitkeep.



解答

.gitkeep没有记录，因为它不是 Git 的一个特性。

Git不能添加一个完全空的目录。想要在 Git 中跟踪空目录的人们已经创建了将调用的文件放在.gitkeep这些目录中的约定。该文件可以被称为任何东西；Git 没有给这个名字赋予特殊意义。

.gitignore将文件添加到空目录以跟踪它们有一个竞争惯例，但有些人认为这是令人困惑的，因为目标是保留空目录，而不是忽略它们；.gitignore还用于列出 Git 在查找未跟踪文件时应忽略的文件。


更多建议

.gitkeep只是一个占位符。一个虚拟文件，因此 Git 不会忘记目录，因为 Git 只跟踪文件。

如果您想要一个空目录并确保它对 Git 保持“干净”，请创建一个.gitignore包含以下行的目录：

# .gitignore sample
# Ignore all files in this dir...
*

# ... except for this one.
!.gitignore

如果你希望只有一种类型的文件对 Git 可见，下面是一个如何过滤除所有文件之外的.gitignore所有.txt文件的示例：

# .gitignore to keep just .txt files
# Filter everything...
*

# ... except the .gitignore...
!.gitignore

# ... and all text files.
!*.txt
