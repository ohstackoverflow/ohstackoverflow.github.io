如何找到foreach索引？

Arya

2023-02-17 22:15

是否可以找到foreach索引？

在一个for循环中，如下所示：

for ($i = 0; $i < 10; ++$i) {
   echo $i . ' ';
}

$i会给你索引。

我必须使用for循环还是有某种方法可以在foreach循环中获取索引？



解答

foreach($array as $key=>$value) {
    // do stuff
}

$key``$array是每个元素的索引


更多建议

您可以在您的 中添加一个 hack foreach，例如在每次运行时递增的字段，这正是循环for在数字索引数组中为您提供的内容。这样的字段将是一个需要手动管理（增量等）的伪索引。

Aforeach会以你的价值的形式给你你的索引$key，所以这样的 hack 应该是不必要的。

例如，在 foreach

$index = 0;
foreach($data as $key=>$val) {
    // Use $key as an index, or...

    // ... manage the index this way..
    echo "Index is $index\n";
    $index++;
}

是否有将 PHP 数组复制到另一个数组的函数？

Arya

2023-02-17 22:15

是否有将 PHP 数组复制到另一个数组的函数？

尝试复制 PHP 数组时，我已经被烧了几次。我想将对象内部定义的数组复制到它外部的全局数组。



解答

在 PHP 中，除对象外的所有变量都通过称为写时复制的机制分配，而对象则通过引用分配。这意味着对于具有标量值的数组，$b = $a已经简单地为您提供了一个副本：

$a = array();
$b = $a;
$b['foo'] = 42;
var_dump($a);

将产生：

array(0) {
}

而对于对象，

$a = new StdClass();
$b = $a;
$b->foo = 42;
var_dump($a);

产量：

object(stdClass)#1 (1) {
  ["foo"]=>
  int(42)
}

在另一个答案中解释了数组元素也可能是需要克隆的对象的边缘情况

您可能会对诸如 之类的错综复杂的内容感到困惑ArrayObject，它是一个行为与数组完全相同的对象。然而，作为一个对象，它具有引用语义。

编辑：@AndrewLarsson 在下面的评论中提出了一个观点。PHP 有一个称为“引用”的特殊功能。它们有点类似于 C/C++ 等语言中的指针，但又不完全相同。如果您的数组包含引用，那么当数组本身通过副本传递时，引用仍将解析为原始目标。这当然通常是理想的行为，但我认为值得一提。


更多建议

PHP 默认会复制数组。PHP 中的引用必须是显式的。

$a = array(1,2);
$b = $a; // $b will be a different array
$c = &$a; // $c will be a reference to $a

更新

这也适用于我。

rm /usr/local/var/postgres/postmaster.pid



解答

有类似的问题；pid 文件阻止 postgres 启动。要解决这个问题：

$ rm /usr/local/var/postgres/postmaster.pid
$ brew services restart postgresql

然后一切都很好。

更新：

对于 Apple M1 (Big Sur) 用户，请执行以下操作：

$ rm /opt/homebrew/var/postgres/postmaster.pid
$ brew services restart postgresql

更新 （2022 年 11 月 15 日） ：

对于 Apple M1 / M2 (Ventura) 用户，只需执行以下操作：

$ bundle install (just to make sure pg is installed)
$ brew services restart postgresql


更多建议

当 brew 升级 postgres 时，有时会发生这种情况，导致数据文件与新服务器不兼容。

就我而言，它发生在从 9.3 升级到 9.4时。

OS X/自制软件：

尝试运行postgres -D /usr/local/var/postgres——如果 postgres 启动失败，它会给你一个更详细的输出。或者打开日志文件/usr/local/var/log/postgres.log（/opt/homebrew/var/log/postgres.log在 Mac M1 上）并查找 “FATAL: database files are incompatible with server”这一行

在我的例子中，运行rm -rf /usr/local/var/postgres && initdb /usr/local/var/postgres -E utf8删除了我的旧数据库，然后重新初始化了 postgres 数据库模式。 （这会破坏您的数据）

感谢https://github.com/Homebrew/homebrew/issues/35240的解决方案。可以在此处找到彻底重新安装的完整说明：如何完全卸载和重新安装 Homebrew Postgres - Test Double Blog（同样，如果您使用的是 M1 Mac，则将 /opt/homebrew/var 路径替换为 /用户/本地/变量 ）

重新生成我的数据库（使用rake db:create）后，一切都恢复正常。
