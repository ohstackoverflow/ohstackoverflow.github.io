为什么使用“==”或“is”比较字符串有时会产生不同的结果？

Arya

2023-02-17 03:31

两个字符串变量设置为相同的值。s1 == s2总是返回True，但s1 is s2有时返回False。

如果我打开我的 Python 解释器并进行相同的is比较，它会成功：

>>> s1 = 'text'
>>> s2 = 'text'
>>> s1 is s2
True

为什么是这样？



解答

is是身份测试，并且==是平等测试。您的代码中发生的事情将在解释器中模拟如下：

>>> a = 'pub'
>>> b = ''.join(['p', 'u', 'b'])
>>> a == b
True
>>> a is b
False

所以，难怪他们不一样，对吧？

换句话说：a is b相当于id(a) == id(b)


更多建议

这里的其他答案是正确的：is用于 身份 比较，而==用于 相等 比较。由于您关心的是相等性（两个字符串应包含相同的字符），在这种情况下，运算符is完全错误，您应该==改为使用。

交互式工作的原因is是（大多数）字符串文字在默认情况下是interned 的。来自维基百科：

Interned strings 加速字符串比较，这有时是严重依赖具有字符串键的哈希表的应用程序（例如编译器和动态编程语言运行时）的性能瓶颈。在没有实习的情况下，检查两个不同的字符串是否相等涉及检查两个字符串的每个字符。这很慢有几个原因：它在字符串长度上本质上是 O(n)；它通常需要从多个内存区域读取，这需要时间；并且读取会填满处理器缓存，这意味着可用于其他需求的缓存较少。对于 interned 字符串，在原始的 intern 操作之后进行简单的对象标识测试就足够了；这通常作为指针相等性测试来实现，通常只是一条完全没有内存引用的机器指令。

因此，当您的程序中有两个具有相同值的字符串文字（按字面意义输入到程序源代码中的单词，用引号括起来时），Python 编译器将自动保留字符串，使它们同时存储内存位置。（请注意，这并不 总是 会发生，并且发生这种情况的规则非常复杂，所以请不要在生产代码中依赖这种行为！）

由于在您的交互式会话中，两个字符串实际上都存储在相同的内存位置，因此它们具有相同的 identity ，因此is运算符按预期工作。但是如果你通过一些其他方法构造一个字符串（即使那个字符串包含 完全相同 的字符），那么这个字符串可能是 相等的 ，但它不是 同一个字符串 ——也就是说，它有一个不同的 标识 ，因为它是存储在内存中的不同位置。

`self` 参数的目的是什么？为什么需要它？

Arya

2023-02-17 03:30

考虑这个例子：

class MyClass:
    def func(self, name):
        self.name = name

我知道那是指self的具体实例MyClass。但是为什么必须func显式包含self为参数呢？为什么我们需要self在方法的代码中使用？其他一些语言将此隐式化，或使用特殊语法代替。

对于与 语言无关 的设计决策考虑，请参阅What is the advantage of having this/self pointer mandatory explicit? .

要关闭OP 省略方法参数并获得的 调试问题 ，请使用TypeError: method() takes 1 positional argument but 2 were given instead. 如果 OP在方法体中省略并得到一个，请考虑How can I call a function within a class? .self``TypeError``self.``NameError



解答

您需要使用的原因self.是因为 Python 不使用特殊语法来引用实例属性。Python 决定以一种使方法所属的实例自动 传递 但不自动 接收 的方式来处理方法：方法的第一个参数是调用该方法的实例。这使得方法与函数完全相同，并让您使用实际名称（尽管这self是惯例，当您使用其他名称时人们通常会对您皱眉。）self代码并不特殊，它只是另一个对象.

Python 可以做一些其他的事情来区分普通名称和属性——像 Ruby 那样的特殊语法，或者像 C++ 和 Java 那样需要声明，或者可能更不同的东西——但它没有。Python 就是为了让事情变得明确，让什么是什么变得显而易见，虽然它并不是在所有地方都这样做，但它确实为实例属性做到了。这就是为什么分配给一个实例属性需要知道要分配给什么实例，这就是为什么它需要self..


更多建议

假设您有一个类ClassA，其中包含定义为的方法methodA：

def methodA(self, arg1, arg2):
    # do something

并且objectA是这个类的一个实例。

现在 whenobjectA.methodA(arg1, arg2)被调用时，python 在内部为您将其转换为：

ClassA.methodA(objectA, arg1, arg2)

变量self指的是对象本身。
