更多建议

受Vladimir Reshetnikov上述回答的启发，我找到了一种非常舒适的方式，您可以通过 拖放 或通过 “发送到” 上下文菜单立即使用 Visual Studio 比较两个文件。它只需要做一些准备工作，您需要做一次，然后它就像一把瑞士军刀一样有用。

Visual Studio 已经拥有你需要的一切，只需要一些配置步骤就可以让它工作：

使用拖放比较文件

准备：

  1. 使用您喜欢的文本编辑器创建一个新的 批处理文件 。输入以下内容：
    @echo off

    setlocal set vspath=C:\Program Files (x86)\Microsoft Visual Studio\2017\Enterprise\Common7\IDE start "Compare files" /B /MIN "%vspath%\devenv.exe" /diff %2 %1 First:'%2' Second:'%1'

您可能会注意到我在批处理中颠倒了%1和参数。%2这是因为我注意到文件浏览器将 第二个文件 作为第一个参数传递，然后将 第一个文件 作为第二个参数传递。

  1. 保存此代码以 VS_FileCompare.cmd 供使用，并vspath在需要时进行修改以匹配devenv.exe（取决于您当前使用的 Visual Studio 版本，请参见 脚注 *））

  2. 创建一个名为 “文件比较”的 快捷方式 并将其放置在 桌面 上（如下面的动画中所用），以便始终可以将文件拖放到它上面或直接将批处理文件放置在桌面上。就这样！ *VS_FileCompare.cmd *

用法：

  1. 通过+打开 Windows 资源管理器Win``E

  2. 在资源管理器中选择要比较的两个文件

  3. 拖放它们，如下面的动画所示：

  1. 几秒钟后（取决于 Visual Studio 的启动时间），结果将显示在 Visual Studio 中：

注意： 如果 Visual Studio 已经打开，则不会有任何危害。在这种情况下，它只会在正在运行的 Visual Studio 实例中打开一个新窗口。因此您可以比较多个文件对，但请确保您一次只选择了两个文件。

替代方式：发送到上下文菜单

这是使用 VS_FileCompare.cmd 上一节中提到的批处理文件的替代方法。它允许使用上下文菜单的 发送到文件夹 来比较文件。

准备：

  1. 为批处理文件创建快捷方式 “Compare2Files VS” VS_FileCompare.cmd 并将其复制到 SendTo 文件夹中。通过+打开 Windows 资源管理器Win``E
  2. 通过进入文件资源管理器的地址栏（如此处所述）打开SendTo 文件夹。 然后，把准备好的快捷方式放到这个文件夹中。 shell:sendto

用法：

  1. 通过+打开 Windows 资源管理器Win``E

  2. 在资源管理器中选择要比较的两个文件

  3. 假设批处理文件的快捷方式 VS_FileCompare.cmd 名为 “Compare2Files VS” ，您可以选择这两个文件，右键单击并选择 “发送到”→“Compare2Files VS” 以调用比较，如下所示：

  1. 几秒钟后（取决于 Visual Studio 的启动时间），结果将显示在 Visual Studio 中：

提示： 如果您喜欢 SendTo 文件夹方法，您可以做更多的事情 - 例如，您可以直接通过 SendTo 打开一个命令行界面，它从正确的路径（所选文件所在的路径）开始。看这里了解如何做到这一点。您甚至可以将它与脚本结合起来以获得更高的权限，只需付出一点额外的努力。

如何使用自定义分隔符将多行文件名合并为一个

Arya

2023-02-18 06:35

如何将结果ls -1加入一行并用我想要的任何内容分隔它？



解答

与第一个选项类似，但省略了尾随定界符

ls -1 | paste -sd "," -


更多建议

编辑 ：如果您希望定界符为逗号，只需“ ls -m ”

啊，力量和简单！

ls -1 | tr '\n' ','

将逗号“ , ”更改为您想要的任何内容。请注意，这包括一个“尾随逗号”（对于以换行符结尾的列表）
