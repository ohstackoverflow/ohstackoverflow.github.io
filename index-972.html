通过“git reset --hard”从丢失未提交的更改中恢复

Arya

2023-02-17 21:25

有什么方法可以从 a 恢复对工作目录的未提交更改git reset --hard HEAD？



解答

一般来说，您 无法取回未提交的更改。

先前暂存的更改 ( git add) 应该可以从索引对象中恢复，因此如果您这样做了，请使用git fsck --lost- found来定位与其相关的对象。（这会将对象写入目录.git/lost-found/；您可以从那里git show <filename>查看每个文件的内容。）

如果没有，这里的答案是：查看您的备份。 也许您的编辑器/IDE 将临时副本存储 在 /tmp 或 C:\TEMP 以及类似的东西下。[1]

git reset HEAD@{1}

这将恢复到以前的 HEAD

[1] vim eg 可选地存储持久撤销， eclipse IDE 存储 本地历史 ；这些功能可能会节省您的时间**

在 Android Studio 中搜索整个项目中所有出现的字符串

Arya

2023-02-17 21:23

我刚刚开始使用 Android Studio (IntelliJ)，现在我正在寻找在我的项目中的任何文件中查找字符串出现的功能。例如：我想找到所有包含字符串“ .getUuid()”的文件

右上角的搜索没有给我正确的结果，而且我认为我无法在“编辑”>“查找”下找到此功能。

有人能指出我正确的方向吗？



解答

TLDR： MacOS 上的⌃⇧F 将打开“在路径中查找”对话框。

首先，这个 IDEA 有一个很好的“ Find Usages”命令。当光标位于某些字段、方法等上时，可以在上下文菜单中找到它。

它是上下文感知的，据我所知，这是查找类、方法或字段用法的最佳方式。

或者，您可以使用

编辑 > 查找 > 在路径中查找...

dialog ，它允许您搜索整个工作区。

在 IDEA 13 中还有一个很棒的“ Search Everywhere”选项，默认情况下由双 Shift 调用。它允许您在项目、文件、类、设置等中进行搜索。

您也可以使用“”从项目结构对话框中搜索Find in Path…。只需在具体目录上用鼠标右键调用它，搜索范围就会限定在该目录及其子目录内。

享受！


更多建议

在基于 Windows、macOS 或 Linux 的机器上的 Android Studio 中，使用快捷键Ctrl+ Shift+F搜索整个项目中的任何字符串。Ctrl考虑到+F用于在当前文件中搜索，这很容易记住。所以也只需按一下Shift。

如何从 Unix 上的文本文件中提取预定范围的行？

Arya

2023-02-17 21:22

我有一个 ~23000 行的 SQL 转储，其中包含多个数据库的数据。我需要提取此文件的特定部分（即单个数据库的数据）并将其放入新文件中。我知道我想要的数据的开始行号和结束行号。

有谁知道 Unix 命令（或一系列命令）从文件中提取第 16224 行和 16482 行之间的所有行，然后将它们重定向到新文件中？



解答

sed -n '16224,16482p;16483q' filename > newfile

从sed 手册：

p - 打印出模式空间（到标准输出）。此命令通常只与 -n 命令行选项一起使用。

n - 如果未禁用自动打印，则打印模式空间，然后无论如何，将模式空间替换为下一行输入。如果没有更多输入，则 sed 退出而不处理更多命令。

q - 退出sed而不处理任何更多的命令或输入。请注意，如果未使用 -n 选项禁用自动打印，则会打印当前模式空间。

和

sed 脚本中的地址可以是以下任何一种形式：

number 指定行号将仅匹配输入中的该行。

可以通过指定以逗号 (,) 分隔的两个地址来指定地址范围。地址范围匹配从第一个地址匹配的位置开始的行，并继续直到第二个地址匹配（包括）。


更多建议

sed -n '16224,16482 p' orig-data-file > new-file

其中 16224,16482 是起始行号和结束行号，包括在内。这是 1 索引的。-n抑制将输入作为输出回显，这显然是您不想要的；数字表示使以下命令运行的行范围；该命令p打印出相关行。
