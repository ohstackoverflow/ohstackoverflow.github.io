我有两个分支： master 和 dev

我想从dev 分支创建一个“功能分支” 。

目前在分支 dev ，我做：

git checkout -b myfeature dev

... （一些工作）

git commit -am "blablabla"
git push origin myfeature

但是，在可视化我的分支之后，我得到：

--**master**
------0-----0-----0-----0-----0
------------------------**dev**----**myfeature**

我的意思是分支似乎快进合并了，我不明白为什么......

我究竟做错了什么？

您如何从另一个分支分支出来并推回功能分支的远程存储库？

所有这一切都在一个分支模型中，就像这里描述的那样。



解答

如果您喜欢您发布的链接中的方法，请查看Git Flow。

这是他为该工作流创建的一组脚本。

但要回答你的问题：

git checkout -b myFeature dev

从 dev 创建 MyFeature 分支。做你的工作然后 __

git commit -am "Your message"

现在将您的更改合并到 dev 而无需快进

git checkout dev
git merge --no-ff myFeature

现在将更改推送到服务器

git push origin dev
git push origin myFeature

你会看到你想要的样子。


更多建议

如果你想从 Git 中的任何现有分支创建一个新分支，只需按照选项操作即可。

首先更改/签出要创建新分支的分支。例如，如果您有以下分支，例如：

  • 掌握
  • 开发
  • 分支机构1

因此，如果您想在名为“branch1” 的分支下创建一个名为 _ “subbranch_of_b1” 的新分支，请按照以下步骤操作： _ ****

  1. 结帐或更改为 “branch1”

     git checkout branch1

  2. 现在使用以下命令在 “branch1”下创建名为 “subbranch_of_b1” 的新分支。 ****

     git checkout -b subbranch_of_b1 branch1

以上将创建一个名为 *在分支 *branch1下创建一个名为 subbranch_of_b1 的新分支（请注意，branch1在上面的命令中不是强制性的，因为 HEAD 当前指向它，如果您在不同的分支上，您可以精确它）。

  1. 现在，在使用 subbranch_of_b1 之后，您可以在本地或远程提交并推送或合并它。

将 subbranch_of_b1 推送到远程：

 git push origin subbranch_of_b1

我如何使用 Mathematica 查找 Waldo？

Arya

2023-02-17 01:29

这周末一直困扰着我：解决 Where's Waldo 问题的好方法是什么？ [北美以外的 “Wally” ] 拼图，使用 Mathematica（图像处理和其他功能）？

这是我到目前为止所拥有的，一个通过调暗一些非红色来稍微降低视觉复杂性的功能：

whereIsWaldo[url_] := Module[{waldo, waldo2, waldoMask},
    waldo = Import[url];
    waldo2 = Image[ImageData[
        waldo] /. {{r_, g_, b_} /;
          Not[r > .7 && g < .3 && b < .3] :> {0, 0,
          0}, {r_, g_, b_} /; (r > .7 && g < .3 && b < .3) :> {1, 1,
          1}}];
    waldoMask = Closing[waldo2, 4];
    ImageCompose[waldo, {waldoMask, .5}]
]

以及此“有效”的 URL 示例：

whereIsWaldo["http://www.findwaldo.com/fankit/graphics/IntlManOfLiterature/Scenes/DepartmentStore.jpg"]

（沃尔多在收银台旁）：



解答

我找到沃尔多了！

我是怎么做到的

首先，我过滤掉所有非红色的颜色

waldo = Import["http://www.findwaldo.com/fankit/graphics/IntlManOfLiterature/Scenes/DepartmentStore.jpg"];
red = Fold[ImageSubtract, #[[1]], Rest[#]] &@ColorSeparate[waldo];

接下来，我正在计算此图像与简单黑白图案的相关性，以找到衬衫中的红色和白色过渡。

corr = ImageCorrelate[red, 
   Image@Join[ConstantArray[1, {2, 4}], ConstantArray[0, {2, 4}]], 
   NormalizedSquaredEuclideanDistance];

我过去常常Binarize挑选出图像中具有足够高相关性的像素，并在它们周围绘制白色圆圈以强调它们Dilation

pos = Dilation[ColorNegate[Binarize[corr, .12]], DiskMatrix[30]];

我不得不在关卡上玩一点。如果水平太高，就会发现太多误报。

最后我将这个结果与原始图像结合起来得到上面的结果

found = ImageMultiply[waldo, ImageAdd[ColorConvert[pos, "GrayLevel"], .5]]


更多建议

我对“做到这一点的防弹方法”的猜测（想想中央情报局随时在任何卫星图像中发现沃尔多，而不仅仅是没有竞争元素的单一图像，如条纹衬衫）......我会在沃尔多的许多图像上训练玻尔兹曼机- 他坐着、站着、闭塞等的所有变化；衬衫、帽子、相机和所有作品。您不需要大量的 Waldos 语料库（也许 3-5 个就足够了），但越多越好。

这将为以任何正确排列发生的各种元素分配概率云，然后（通过分割）确定平均对象大小，将源图像分割成最像个人的对象单元（考虑可能的遮挡和姿势变化)，但由于 Waldo 图片通常包括很多人，比例大致相同，这应该是一项非常容易的任务，然后将这些片段输入预训练的玻尔兹曼机。它会给你每个人都是 Waldo 的概率。拿一个概率最高的。

这就是当今 OCR、邮政编码阅读器和无笔画手写识别的工作方式。基本上你知道答案就在那里，你或多或少知道它应该是什么样子，其他一切可能有共同的元素，但绝对是“不是它”，所以你不必理会“不是它”，你只看“它”在所有你以前见过的可能“它”中的可能性（例如，在邮政编码中，你训练 BM 仅 1 秒，仅 2 秒，仅 3 秒，等等，然后喂给每个数字到每台机器，然后选择一个最有信心的机器。这比单个神经网络学习所有数字的特征要好得多。
