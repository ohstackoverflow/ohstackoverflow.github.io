是否可以在 Count() 中指定条件？

Arya

2023-02-18 05:26

是否可以在中指定条件Count()？我想只计算位置列中具有例如“经理”的行。

我想在计数语句中执行此操作，而不是使用WHERE; 我问这个是因为我需要同时计算经理和其他人SELECT（在这个例子中Count(Position = Manager), Count(Position = Other))这样的事情WHERE对我没有用。



解答

如果你不能只用一个子句限制查询本身，你可以使用聚合只计算非空值的where事实：count

select count(case Position when 'Manager' then 1 else null end)
from ...

您还可以sum以类似的方式使用聚合：

select sum(case Position when 'Manager' then 1 else 0 end)
from ...


更多建议

假设您不想限制返回的行，因为您也在聚合其他值，您可以这样做：

select count(case when Position = 'Manager' then 1 else null end) as ManagerCount
from ...

假设在同一列中您有经理、主管和团队负责人的值，您可以像这样获得每个值：

select count(case when Position = 'Manager' then 1 else null end) as ManagerCount,
    count(case when Position = 'Supervisor' then 1 else null end) as SupervisorCount,
    count(case when Position = 'Team Lead' then 1 else null end) as TeamLeadCount,
from ...

heroku - 如何查看所有日志

Arya

2023-02-18 05:26

我在 heroku 上有一个小应用程序。每当我想查看日志时，我都会转到命令行并执行

heroku logs

那只显示了大约 100 行。有没有办法在 heroku 上查看我们的应用程序的完整日志？



解答

更新（感谢 dawmail333）：

heroku logs -n 1500

或者，实时跟踪日志

heroku logs -t

Heroku 日志文档

如果你需要超过几千行，你可以使用 heroku 的Syslog Drains

或者（旧方法）：

$ heroku run rails c
File.open('log/production.log', 'r').each_line { |line| puts line }


更多建议

heroku 中的日志记录有了很大的改进！

$ heroku logs -n 500

更好的！

$ heroku logs --tail

参考资料： http: //devcenter.heroku.com/articles/logging

更新

这些不再是附加组件，而是默认功能的一部分 :)

如何将多个参数传递给 PowerShell 中的一个函数？

Arya

2023-02-18 05:25

如果我有一个接受多个字符串参数的函数，第一个参数似乎获得分配给它的所有数据，其余参数作为空参数传入。

一个快速测试脚本：

Function Test([string]$arg1, [string]$arg2)
{
    Write-Host "`$arg1 value: $arg1"
    Write-Host "`$arg2 value: $arg2"
}

Test("ABC", "DEF")

生成的输出是

$arg1 value: ABC DEF
$arg2 value:

正确的输出应该是：

$arg1 value: ABC
$arg2 value: DEF

这似乎在多台机器上的 v1 和 v2 之间是一致的，所以很明显，我做错了什么。谁能指出具体是什么？



解答

在 PowerShell（所有版本）中调用函数的参数 是用空格分隔的，而不是逗号分隔的 。此外，括号是完全不必要的，如果或更高版本处于活动状态，则会在 PowerShell 2.0（或更高版本）中导致解析错误。带括号的参数仅在 .NET 方法中使用。Set-StrictMode -Version 2

function foo($a, $b, $c) {
   "a: $a; b: $b; c: $c"
}

ps> foo 1 2 3
a: 1; b: 2; c: 3


更多建议

已经提供了正确的答案，但这个问题似乎很普遍，需要为那些想要了解其中微妙之处的人提供一些额外的细节。
