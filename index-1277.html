Flex-box：将最后一行与网格对齐

我有一个简单的 flex-box 布局，容器如下：

.grid {
  display: flex;

  justify-content: space-between;

现在我希望最后一行中的项目与其他项目对齐。justify-content: space-between;应该使用，因为可以调整网格的宽度和高度。

解答

添加一个::after自动填充空间的。无需污染您的 HTML。这是一个显示它的codepen：http://codepen.io/DanAndreasson/pen/ZQXLXj

.grid {
  display: flex;
  flex-flow: row wrap;
  justify-content: space-between;
}

.grid::after {
  content: "";
  flex: auto;
}


更多建议

正如其他海报所提到的那样——没有一种干净的方法可以将最后一行与 flexbox 左对齐（至少按照当前规范）

然而，就其价值而言：使用CSS 网格布局模块，这非常容易生成：

基本上相关的代码归结为：

ul {
  display: grid; /* 1 */
  grid-template-columns: repeat(auto-fill, 100px); /* 2 */
  grid-gap: 1rem; /* 3 */
  justify-content: space-between; /* 4 */
}

1）使容器元素成为网格容器

2) 将网格设置为宽度为 100px 的自动列。（请注意自动填充的使用（与auto-fit- 它（对于 1 行布局）将空轨道折叠为 0 - 导致项目扩展以占用剩余空间。这将导致合理的'space-between ' 当网格只有一行时的布局，在我们的例子中这不是我们想要的。（查看此演示以了解它们之间的区别））。

3) 为网格行和列设置间隙/间距 - 在这里，因为想要一个“间隔”布局 - 间隙实际上是最小间隙，因为它会根据需要增长。

4) 类似于flexbox。

显示代码片段

ul {
  display: grid;
  grid-template-columns: repeat(auto-fill, 100px);
  grid-gap: 1rem;
  justify-content: space-between;

  /* boring properties */
  list-style: none;
  background: wheat;
  padding: 2rem;
  width: 80vw;
  margin: 0 auto;
}

li {
  height: 50px;
  border: 1px solid green;
}


<ul>
  <li></li>
  <li></li>
  <li></li>
  <li></li>
  <li></li>

Java 相当于 C# 中的#region

Arya

2023-02-18 05:45

我想在Eclipse 中使用区域进行代码折叠；这怎么能在 Java 中完成呢？

C#中的示例用法：

#region name
//code
#endregion



解答

没有这样的 标准 等价物。某些 IDE（例如 Intellij 或 Eclipse）可以根据所涉及的代码类型（构造函数、导入等）进行折叠，但没有什么比#region.

ExpressionChangedAfterItHasBeenCheckedError 解释

Arya

2023-02-18 05:44

请向我解释为什么我不断收到此错误：ExpressionChangedAfterItHasBeenCheckedError: Expression has changed after it was checked.

显然，我只在开发模式下得到它，它不会在我的生产构建中发生，但它非常烦人，我根本不明白在我的开发环境中出现不会出现在产品上的错误的好处 - -可能是因为我缺乏理解。

通常，修复很简单，我只是将导致错误的代码包装在 setTimeout 中，如下所示：

setTimeout(()=> {
    this.isLoading = true;
}, 0);

或者使用这样的构造函数强制检测更改constructor(private cd: ChangeDetectorRef) {}：：

this.isLoading = true;
this.cd.detectChanges();

但是为什么我经常遇到这个错误？我想了解它，以便将来避免这些 hacky 修复。



解答

一旦我理解了Angular Lifecycle Hooks及其与变更检测的关系，就会有很多理解。

我试图让 Angular 更新绑定到*ngIf元素的全局标志，并且我试图在ngOnInit()另一个组件的生命周期挂钩内更改该标志。

根据文档，这个方法在 Angular 已经检测到变化之后被调用：

在第一个 ngOnChanges() 之后调用一次。

因此更新其中的标志ngOnChanges()不会启动更改检测。然后，一旦更改检测自然再次触发，标志的值已更改并抛出错误。

就我而言，我改变了这个：

constructor(private globalEventsService: GlobalEventsService) {

}

ngOnInit() {
    this.globalEventsService.showCheckoutHeader = true;
}

对此：

constructor(private globalEventsService: GlobalEventsService) {
    this.globalEventsService.showCheckoutHeader = true;
}

ngOnInit() {

}

它解决了问题:)


更多建议

此错误表明您的应用程序中存在真正的问题，因此抛出异常是有意义的。

在devMode变化检测中，在每次常规变化检测运行后添加一个额外的轮次，以检查模型是否已更改。

如果模型在常规变化检测和附加变化检测之间发生了变化，这表明要么

  • 变化检测本身引起了变化
  • 方法或 getter 每次被调用时都会返回不同的值

这两者都很糟糕，因为不清楚如何进行，因为模型可能永远不会稳定。

如果 Angular 在模型稳定之前运行变更检测，它可能会永远运行下去。如果 Angular 不运行变更检测，那么视图可能不会反映模型的当前状态。

另请参阅Angular2 中的生产模式和开发模式有什么区别？
