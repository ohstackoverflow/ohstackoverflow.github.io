从命令行向“git commit -m”添加换行符

Arya

2023-02-17 07:43

我从命令行使用 Git，并试图git commit -m ""在不进入 Vim 的情况下向提交消息添加换行符（使用 ）。

这可能吗？



解答

当然，如何完成取决于您的 shell。在 Bash 中，您可以在消息周围使用单引号，并且可以将引号保持打开状态，这将使 Bash 提示换行，直到您关闭引号。像这样：

git commit -m 'Message

goes
here'

或者，您可以使用“此处文档”（也称为 heredoc）：

git commit -F- <<EOF
Message

goes
here
EOF


更多建议

如果您只想要标题行和内容行，则可以使用：

git commit -m "My head line" -m "My content line."

请注意，这会创建单独的段落 - 而不是行。所以每两行之间会有一个空行-m，eg：

My head line

My content line.

Kotlin 中 Java 静态方法的等价物是什么？

Arya

2023-02-17 07:43

staticKotlin 中没有关键字。

在 Kotlin 中表示staticJava 方法的最佳方式是什么？



解答

您将函数放在“伴随对象”中。

所以java代码是这样的：

class Foo {
  public static int a() { return 1; }
}

会变成

class Foo {
  companion object {
     fun a() : Int = 1
  }
}

然后您可以从 Kotlin 代码内部使用它作为

Foo.a();

但是在 Java 代码中，您需要将其称为

Foo.Companion.a();

（这也适用于 Kotlin。）

如果您不喜欢必须指定位，Companion您可以添加@JvmStatic注释或命名您的同伴类。

从文档：

伴生对象

类中的对象声明可以用关键字标记companion：

class MyClass {
   companion object Factory {
       fun create(): MyClass = MyClass()
   }
}

可以通过简单地使用类名作为限定符来调用伴随对象的成员：

val instance = MyClass.create()

...

但是，在 JVM 上，如果使用注释，则可以将伴随对象的成员生成为真正的静态方法和字段@JvmStatic。有关详细信息，请参阅Java 互操作性部分。

添加@JvmStatic注释看起来像这样

class Foo {
  companion object {
    @JvmStatic
    fun a() : Int = 1;
  }
}

然后它将作为一个真正的 Java 静态函数存在，可以从 Java 和 Kotlin 访问，如Foo.a().

如果只是不喜欢名称Companion，那么您还可以为伴随对象提供一个明确的名称，如下所示：

class Foo {
  companion object Blah {
    fun a() : Int = 1;
  }
}

这会让你以同样的方式从 Kotlin 调用它，但是从类似 Java 的方式调用它Foo.Blah.a()（它也可以在 Kotlin 中工作）。


更多建议

文档建议用 包级函数 解决大部分静态函数的需求。它们只是在源代码文件的类之外声明。文件的包可以在文件的开头用 package 关键字指定。

宣言

package foo

fun bar() = {}

用法

import foo.bar

或者

import foo.*

您现在可以调用该函数：

bar()

或者如果您不使用 import 关键字：

foo.bar()

如果您不指定包，则可以从根目录访问该函数。

如果您只有 java 经验，这可能看起来有点奇怪。原因是 kotlin 不是严格面向对象的语言。你可以说它支持类外的方法。

编辑：他们已经编辑了文档，不再包含关于推荐包级功能的句子。这是上面提到的原件。

将远程重置为某个提交

Arya

2023-02-17 07:42

我想放弃提交后所做的所有更改<commit-hash>。所以我做了：

git reset --hard <commit-hash>

现在我想用我的遥控器做同样的事情。我怎样才能做到这一点？之后我做了一些提交（和推送），<commit-hash>我只想放弃它们。只是在途中出现了 严重 错误，我不想让它变得比现在更糟。;(

我基本上想倒回我origin/master的<commit-hash>



解答

假设您的分支master在此处和远程都被调用，并且您的远程被调用，origin您可以这样做：

 git reset --hard <commit-hash>
 git push -f origin master

但是，如果其他人正在使用您的远程存储库并拉取了您的更改，您应该避免这样做。在这种情况下，最好还原不需要的提交，然后照常推送。

更新： 你已经在下面解释过其他人已经撤回了你推送的更改，所以最好 创建一个新的提交来恢复所有这些更改 。 在Jakub Narębski 的这个回答 中，您可以很好地解释执行此操作的选项。哪一个最方便取决于您要还原多少提交，以及哪种方法对您最有意义。

由于从您的问题中可以清楚地看出您已经使用过git reset --hard重置master分支，因此您可能需要先使用git reset --hard ORIG_HEAD将分支移回原来的位置。（与往常一样git reset --hard，确保它git status是干净的，您在正确的分支上，并且您知道git reflog这是一种恢复明显丢失的提交的工具。）您还应该检查ORIG_HEAD指向正确提交的点，使用git show ORIG_HEAD.

故障排除：

如果您收到类似“ ! [remote rejected] a60f7d85 - > master (pre-receive hook declined) ”的消息

那么您必须允许为特定分支重写分支历史记录。例如在 BitBucket 中它说“不允许重写分支历史”。有一个名为Allow rewriting branch history您必须选中的复选框。


更多建议

使用其他一些答案可能会导致不必要的本地状态丢失。 更改远程并非固有地需要本地更改。如果您选择了错误的提交返回，此方法仍然会破坏您的远程，但即使这样您通常也可以找到正确的提交并重试。

您 必须 在您希望远程匹配的本地存储库中的某处进行所需的提交。

  1. 不要 做 任何重置。
  2. 用于git log查找您希望远程所在的提交。用于git log -p查看更改或git log --graph --all --oneline --decorate查看紧凑的树。
  3. 复制提交的散列、标记或（如果是提示）其分支名称。
  4. 运行如下命令：
    git push --force <remote> <commit-ish>:<the remote branch>

例如

    git push --force origin 606fdfaa33af1844c86f4267a136d4666e576cdc:master

或者

    git push --force staging v2.4.0b2:releases

如果强制推送失败，它可能被遥控器禁用了。receive.denyNonFastForwards这可以通过临时更改和receive.denyDeletes中的一个或两个来解决。如果您的遥控器托管在没有 shell 访问权限的服务上，则它可能具有您可以更改的设置以允许强制推送。

我在步骤 2 中使用方便的别名 ( git go) 来查看历史记录，可以像这样添加：

git config --global alias.go 'log --graph --all --decorate --oneline'
