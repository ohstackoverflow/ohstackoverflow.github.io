我如何在 JavaScript 中获取时间戳？

Arya

2023-02-16 18:55

我想要一个代表当前日期和时间的数字，例如Unix timestamp。



解答

以毫秒为单位的时间戳

要获取自Unix 纪元以来的毫秒数，请调用Date.now：

Date.now()

或者，使用一元运算符+调用Date.prototype.valueOf：

+ new Date()

或者，valueOf直接调用：

new Date().valueOf()

要支持 IE8 和更早版本（请参阅兼容性表），请为以下内容创建 填充程序Date.now：

if (!Date.now) {
    Date.now = function() { return new Date().getTime(); }
}

或者，getTime直接调用：

new Date().getTime()

以秒为单位的时间戳

获取自Unix epoch以来的秒数，即 Unix timestamp ：

Math.floor(Date.now() / 1000)

或者，使用 bitwise-or to floor 稍微快一些，但可读性也较差并且将来可能会中断（请参阅解释1、2）：

Date.now() / 1000 | 0

以毫秒为单位的时间戳（更高分辨率）

使用performance.now：

var isPerformanceSupported = (
    window.performance &&
    window.performance.now &&
    window.performance.timing &&
    window.performance.timing.navigationStart
);

var timeStampInMs = (
    isPerformanceSupported ?
    window.performance.now() +
    window.performance.timing.navigationStart :
    Date.now()
);

console.log(timeStampInMs, Date.now());


更多建议

我喜欢这个，因为它很小：

+new Date

我也喜欢这个，因为它同样简短并且与现代浏览器兼容，超过 500 人投票认为它更好：

Date.now()

撤消尚未推送的 Git 合并

Arya

2023-02-16 18:55

我不小心git merge some_other_branch在我当地的 master 分支上跑了。我还没有将更改推送到 origin master。如何撤消合并？

合并后，git status说：

# On branch master
# Your branch is ahead of 'origin/master' by 5 commits.

我如何撤消所有这些提交？



解答

检查git reflog哪个提交是合并之前的提交（git reflog将是比 更好的选择git log）。然后您可以使用以下方法重置它：

git reset --hard commit_sha

还有另一种方法：

git reset --hard HEAD~1

它会让你回到 1 提交。

请注意，任何已修改和未提交/未隐藏的文件都将重置为其未修改状态 。为了让他们隐藏更改或查看--merge下面的选项。

正如@Velmont 在他的回答中建议的那样，在这种直接的情况下使用：

git reset --hard ORIG_HEAD

可能会产生更好的结果，因为它应该保留您的更改。ORIG_HEAD将在合并发生之前直接指向提交，因此您不必自己寻找它。

另一个提示是使用--merge开关而不是--hard因为它不会不必要地重置文件：

git reset --merge ORIG_HEAD

- 合并

重置索引并更新工作树中 和 HEAD 之间不同的文件，但保留索引和工作树之间不同的文件（即具有尚未添加的更改）。


更多建议

假设你的本地主人没有领先于原点/主人，你应该能够做到

git reset --hard origin/<branch-name>

因此，假设您在 上执行此操作master，那么您的本地master分支机构应该看起来与origin/master.

如何将“Null”（真实姓氏！）传递给 ActionScript 3 中的 SOAP Web 服务

Arya

2023-02-16 18:54

我们有一个姓 Null 的员工。当姓氏被用作搜索词时（现在这种情况经常发生），我们的员工查找应用程序就会被终止。收到的错误（感谢 Fiddler！）是：

<soapenv:Fault>
   <faultcode>soapenv:Server.userException</faultcode>
   <faultstring>coldfusion.xml.rpc.CFCInvocationException: [coldfusion.runtime.MissingArgumentException : The SEARCHSTRING parameter to the getFacultyNames function is required but was not passed in.]</faultstring>

可爱吧？

参数类型为string.

我在用：

  • WSDL（肥皂）
  • 弹性 3.5
  • 动作脚本 3
  • 冷聚变 8

请注意，当从 ColdFusion 页面将 Web 服务作为对象调用时， 不会 发生该错误。



解答

追踪它

起初我以为这是一个强制错误，它null被强制执行"null"并通过了测试"null" == null。它不是。 我很接近，但非常非常错误。 对于那个很抱歉！

从那以后，我在 wonderfl.net 上做了很多摆弄，并跟踪了mx.rpc.xml.*. 在XMLEncoder（在 3.5 源代码中）的第 1795 行，在 中setValue，所有 XMLEncoding 归结为

currentChild.appendChild(xmlSpecialCharsFilter(Object(value)));

这与以下内容基本相同：

currentChild.appendChild("null");

根据我原来的小提琴，这段代码返回一个空的 XML 元素。但为什么？

原因

根据评论员 Justin Mclean 在错误报告FLEX-33664中的说法，以下是罪魁祸首（请参阅我的小提琴中的最后两个测试来验证这一点）：

var thisIsNotNull:XML = <root>null</root>;
if(thisIsNotNull == null){
    // always branches here, as (thisIsNotNull == null) strangely returns true
    // despite the fact that thisIsNotNull is a valid instance of type XML
}

当currentChild.appendChild传递 string时"null"，它首先将其转换为带有 text 的根 XML 元素null，然后根据 null 文字测试该元素。这是一个弱相等性测试，因此包含 null 的 XML 被强制转换为 null 类型，或者 null 类型被强制转换为包含字符串“null”的根 xml 元素，并且测试通过了它可能会失败的地方。一种解决方法可能是在检查 XML（或其他任何东西）是否为“空值”时始终使用严格的相等性测试。

解决方案

除了在每个该死的 ActionScript 版本中修复此错误之外，我能想到的唯一合理的解决方法是测试字段是否为“null”并将 它们转义为 CDATA 值。

CDATA 值是改变整个文本值的最合适方法，否则会导致编码/解码问题。 例如，十六进制编码适用于单个字符。当您转义元素的整个文本时，首选 CDATA 值。这样做的最大原因是它保持了人类的可读性。


更多建议

在xkcd note上，Bobby Tables 网站有很好的建议，可以避免在各种语言（包括ColdFusion）的 SQL 查询中对用户数据（在本例中为字符串“Null”）的不当解释。

从问题中不清楚这是问题的根源，并且考虑到第一个答案的评论中指出的解决方案（将参数嵌入结构中）似乎可能是其他原因。
