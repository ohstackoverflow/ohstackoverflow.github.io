JavaScript 检查变量是否存在（已定义/已初始化）

Arya

2023-02-16 21:52

哪种检查变量是否已初始化的方法更好/正确？（假设变量可以容纳任何东西（字符串、整数、对象、函数等））

if (elem) { // or !elem

或者

if (typeof elem !== 'undefined') {

或者

if (elem != null) {



解答

运算typeof符将检查变量是否真的未定义。

if (typeof variable === 'undefined') {
    // variable is undefined
}

与其他运算符不同，该运算符在与未声明的变量一起使用时typeof不会抛出 ReferenceError异常。

但是，请注意typeof null将返回"object"。我们必须小心避免将变量初始化为null. 为了安全起见，这是我们可以使用的：

if (typeof variable === 'undefined' || variable === null) {
    // variable is undefined or null
}

Which equals operator (== vs ===) should be used in JavaScript comparisons?

我是否提交由 npm 5 创建的 package-lock.json 文件？

Arya

2023-02-16 21:51

npm 5 今天发布，其中一项新功能包括通过创建文件进行确定性安装package-lock.json。

这个文件应该保存在源代码管理中吗？

我假设它类似于yarn.lockand composer.lock，它们都应该保存在源代码管理中。



解答

是的，package-lock.json旨在签入源代码管理。如果您使用的是 npm 5+，您可能会在命令行中看到此通知：created a lockfile as package-lock.json. You should commit this file.根据npm help package-lock.json：

package-lock.json对于 npm 修改树node_modules或package.json. 它描述了生成的确切树，以便后续安装能够生成相同的树，而不管中间依赖项更新如何。

此文件旨在提交到源存储库中 ，并用于各种目的：

  • 描述依赖关系树的单一表示，这样可以保证队友、部署和持续集成安装完全相同的依赖关系。

  • 为用户提供一种工具，让他们可以“穿越”到之前的状态，node_modules而无需提交目录本身。

  • 通过可读的源代码控制差异来促进树更改的更大可见性。

  • 并通过允许 npm 跳过以前安装的包的重复元数据解析来优化安装过程。

关于它的一个关键细节package-lock.json是它不能发布，如果在顶层包以外的任何地方找到它都会被忽略。它与 npm- shrinkwrap.json 共享一种格式，本质上是同一个文件，但允许发布。除非部署 CLI 工具或以其他方式使用发布过程来生成生产包，否则不建议这样做。

如果 和package-lock.json都npm-shrinkwrap.json出现在包的根目录中，package- lock.json将被完全忽略。

如何在 Bash 中将变量设置为命令的输出？

Arya

2023-02-16 21:51

我有一个非常简单的脚本，类似于以下内容：

#!/bin/bash
VAR1="$1"
MOREF='sudo run command against $VAR1 | grep name | cut -c7-'
echo $MOREF

当我从命令行运行此脚本并将参数传递给它时，我没有得到任何输出。但是，当我运行变量中包含的命令时$MOREF，我能够获得输出。

如何获取需要在脚本中运行的命令的结果，将其保存到变量中，然后将该变量输出到屏幕上？



解答

除了反引号command，命令替换$(command)还可以用or完成"$(command)"，我觉得这样更容易阅读，并且允许嵌套。

OUTPUT=$(ls -1)
echo "${OUTPUT}"

MULTILINE=$(ls \
   -1)
echo "${MULTILINE}"

引用 ( ) 对保留 多行变量值"很重要；它在赋值的右侧是可选的，因为不执行分词，所以可以正常工作。 ****OUTPUT=$(ls -1)


更多建议

$(sudo run command)

如果您要使用撇号，则需要``, 而不是'`. 这个字符称为“反引号”（或“重音符”）：

#!/bin/bash
VAR1="$1"
VAR2="$2"
MOREF=`sudo run command against "$VAR1" | grep name | cut -c7-`
echo "$MOREF"
