如何将本地未提交的更改合并到另一个 Git 分支？

Arya

2023-02-17 17:57

我怎样才能在 Git 中执行以下操作？

我当前的分支是 branch1，我做了一些本地更改。但是，我现在意识到我实际上是想将这些更改应用到 branch2。有没有办法应用/合并这些更改，以便它们成为 branch2 上的本地更改，而无需在 branch1 上提交它们？



解答

由于您的文件尚未提交branch1：

git stash
git checkout branch2
git stash pop

或者

git stash
git checkout branch2
git stash list       # to check the various stash made in different branch
git stash apply x    # to select the right one

以上是rbento回答的更长更明确的版本：

git stash
git stash branch branch2

它用：

git stash branch <branchname> [<stash>]

  • 创建并签出一个新分支，名称<branchname>从最初创建的提交开始<stash>，
  • 将记录的更改应用于<stash>新的工作树和索引。

如果成功，并且<stash>是 form 的引用stash@{<revision>}，则它会删除<stash>.

如果您运行的分支git stash push已经发生足够多的更改而git stash apply由于冲突而失败，这将很有用。
由于存储条目应用在运行时 HEAD 的提交之上git stash，因此它恢复了最初存储的状态而没有冲突。

正如benjohn所评论的（参见git stash手册页）：

要同时存储当前未跟踪的（新添加的）文件，请添加参数-u，因此：

git stash -u


更多建议

隐藏、临时提交和变基可能都有些矫枉过正。如果您还没有将更改的文件添加到索引中，那么您可以只检查另一个分支。

git checkout branch2

只要您正在编辑的文件在 branch1 和 branch2 之间没有不同，这就会起作用。它将使您留在 branch2 上，并保留工作更改。如果它们不同，那么您可以指定要将本地更改与通过使用-m结帐选项切换分支引入的更改合并。

git checkout -m branch2

如果您已将更改添加到索引，那么您将希望先通过重置来撤消这些更改。（这将保留您的工作副本，它只会删除暂存的更改。）

git reset

div 中的图像在图像下方有额外的空间

Arya

2023-02-17 17:57

为什么在下面的代码中的高度div大于的高度img？图像下方有一个间隙，但它似乎不是填充/边距。

图像下方的间隙或额外空间是什么？

#wrapper {
  border: 1px solid red;
  width:200px;
}
img {
  width:200px;
}


<div id="wrapper">
  <img src="http://i.imgur.com/RECDV24.jpg" />
</div>



解答

默认情况下，图像是内联渲染的，就像字母一样，所以它与 a、b、c 和 d 位于同一行。

该行下方有空间用于您在 g、j、p 和 q 等字母上找到的下行字母。

你可以：

  • 调整vertical-align图像的 以将其定位在其他位置（例如middle） 或

  • 改变display所以它不是内联的。

    div { border: solid black 1px; margin-bottom: 10px; }

    align-middle img {

    vertical-align: middle; }

    align-base img {

    vertical-align: bottom; }

    display img {

    display: block; }

为什么 git 的“拉取请求”不称为“推送请求”？

Arya

2023-02-17 17:56

用于将分支与官方存储库合并的术语是“拉取请求”。这令人困惑，因为我似乎在请求将我的更改推送到官方存储库。

为什么它被称为拉取请求而不是推送请求？



解答

如果您的存储库中有代码更改，并希望将其移动到目标存储库，则：

  • “推送”是您强制将更改存在于目标存储库 ( git push) 中。
  • “拉”是目标存储库抓取您的更改以显示在那里（git pull来自其他存储库）。

“拉取请求”是您请求目标存储库获取您的更改。

“推送请求”将是请求您推送更改的目标存储库。


更多建议

当您发送拉取请求时，您是在要求（请求）官方回购所有者从您自己的回购中提取一些更改。因此“拉取请求”。
