如果字符串的第一个字符为 0，则删除它

Arya

2023-02-17 14:14

如果第一个字符是 0，我想删除字符串的第一个字符。0 可以出现不止一次。

是否有一个简单的函数来检查第一个字符并在它为 0 时将其删除？

现在，我正在尝试使用 JSslice()函数，但它非常笨拙。



解答

您可以使用以下方法删除字符串的第一个字符substring：

var s1 = "foobar";
var s2 = s1.substring(1);
alert(s2); // shows "oobar"

要删除字符串开头的所有 0：

var s = "0000test";
while(s.charAt(0) === '0')
{
 s = s.substring(1);
}


更多建议

非常易读的代码是将.substring()开始设置为第二个字符 (1) 的索引（第一个字符的索引为 0）。该方法的第二个参数.substring()实际上是可选的，所以你甚至不需要调用.length()...

TL;DR ：从字符串中删除第一个字符：

str = str.substring(1);

……是的，就这么简单……

删除一些特定字符：

正如@Shaded 建议的那样，只需循环此字符串，而字符串的第一个字符是“不需要的”字符......

var yourString = "0000test";
var unwantedCharacter = "0";
//there is really no need for === check, since we use String's charAt()
while( yourString.charAt(0) == unwantedCharacter ) yourString = yourString.substring(1);
//yourString now contains "test"

他还指出，如果 slice 的参数是负数，它们会从末尾引用字符串。Substring 和 substr 没有。

何时使用 IMG 与 CSS 背景图片？

Arya

2023-02-17 14:13

这个问题的答案是 社区的努力。编辑现有答案以改进这篇文章。它目前不接受新的答案或互动。

IMG在什么情况下使用 HTML标签显示图像比使用 CSS更合适background-image，反之亦然？

因素可能包括可访问性、浏览器支持、动态内容或任何类型的技术限制或可用性原则。



解答

正确使用 IMG

  1. IMG如果您打算让人们打印您的页面并且您希望默认包含图像，请使用。—杰蒂
  2. 当图像具有重要的语义意义时使用IMG(with text)，例如警告图标。这确保图像的含义可以在所有用户代理中传达，包括屏幕阅读器。alt

IMG 的实用用途

  1. IMG如果图像是内容的一部分，例如徽标或图表或人物（真人，而不是照片人物），请使用plus alt 属性。— sanchothefat
  2. IMG如果您依赖浏览器缩放比例来呈现与文本大小成比例的图像，请使用。
  3. 用于IE6 中IMG的多个叠加图像。

  4. IMG 与 a 一起使用z-index以拉伸背景图像以填充其整个窗口。
    注意，CSS3 background-size 不再适用；请参阅下面的#6。

  5. 使用imginstead ofbackground-image可以显着提高背景动画的性能。

何时使用 CSS 背景图片

  1. 如果图像不是内容的一部分，请使用 CSS 背景图像。— sanchothefat
  2. 在进行文本图像替换时使用 CSS 背景图像，例如。段落/标题。— sanchothefat
  3. background-image如果您打算让人们打印您的页面并且您不希望默认包含图像，请使用。—杰蒂
  4. background-image如果您需要缩短下载时间，请使用CSS sprites。
  5. background-image如果您只需要图像的一部分可见，请使用 CSS 精灵。
  6. 使用background-imagewithbackground-size:cover以拉伸背景图像以填充其整个窗口。


更多建议

对我来说这是一个非黑即白的决定。如果图像是内容的一部分，例如徽标或图表或人物（真人，而不是照片人物），则使用标签<img />加 alt 属性。对于其他所有内容，都有 CSS 背景图像。

另一个使用 CSS 背景图像的时间是在对文本进行图像替换时，例如。段落/标题。

如何修复来自 Perl 的语言环境设置警告

Arya

2023-02-17 14:13

当我运行时perl，我收到警告：

perl：警告：设置语言环境失败。
perl：警告：请检查您的区域设置：
    LANGUAGE =（未设置），
    LC_ALL =（未设置），
    LANG = "en_US.UTF-8"
支持并安装在您的系统上。
perl：警告：回落到标准语言环境（“C”）。

我如何解决它？



解答

您的操作系统不知道en_US.UTF-8.

你没有提到具体的平台，但我可以重现你的问题：

% uname -a
OSF1 hunter2 V5.1 2650 alpha
% perl -e 退出
perl：警告：设置语言环境失败。
perl：警告：请检查您的区域设置：
    LC_ALL =（未设置），
    LANG = "en_US.UTF-8"
    支持并安装在您的系统上。
perl：警告：回落到标准语言环境（“C”）。

我的猜测是您使用 ssh 从较新的台式机连接到这台较旧的主机。/etc/ssh/sshd_config包含很常见

AcceptEnv LANG LC_*

这允许客户端将这些环境变量的值传播到新会话中。

如果您不需要完整的语言环境，该警告会提示您如何压制它：

% env LANG=C perl -e exit
%

或者使用 Bash：

$ LANG=C perl -e 退出
$

要永久修复，请选择以下之一

  1. 在较旧的主机上，LANG在 shell 的初始化文件中设置环境变量。
  2. 在客户端修改您的环境， 例如 ，而不是ssh hunter2使用命令LANG=C ssh hunter2。
  3. SendEnv LANG LC_*如果您拥有管理员权限，请通过在 本地 文件中注释掉该行来阻止 ssh 发送环境变量/etc/ssh/ssh_config。（感谢这个答案。有关更多信息，请参见OpenSSH 的Bug 1285。）
