在 Java 中将“ArrayList<String>”转换为“String[]”

Arya

2023-02-17 04:11

我如何将ArrayList<String>对象转换为String[]Java 中的数组？



解答

List<String> list = ..;
String[] array = list.toArray(new String[0]);

例如：

List<String> list = new ArrayList<String>();
//add some stuff
list.add("android");
list.add("apple");
String[] stringArray = list.toArray(new String[0]);

toArray()不传递任何参数的方法返回Object[]。所以你必须传递一个数组作为参数，它将用列表中的数据填充并返回。您也可以传递一个空数组，但您也可以传递一个具有所需大小的数组。

重要更新 ：最初使用上面的代码new String[list.size()]。然而，这篇博文表明，由于 JVM 优化，new String[0]现在使用更好。


更多建议

Java 8 中的替代方案：

String[] strings = list.stream().toArray(String[]::new);

Java 11+：

String[] strings = list.toArray(String[]::new);

在 Bash 脚本中通过管道传输到/从剪贴板

Arya

2023-02-17 04:10

是否可以在 Bash 中通过管道传输到/从剪贴板？

无论是管道到/从设备句柄还是使用辅助应用程序，我都找不到任何东西。

例如，如果/dev/clip是链接到剪贴板的设备，我们可以这样做：

cat /dev/clip        # Dump the contents of the clipboard
cat foo > /dev/clip  # Dump the contents of "foo" into the clipboard



解答

2018年答案

使用剪贴板 cli。它适用于 macOS、Windows、Linux、OpenBSD、FreeBSD 和 Android，没有任何实际问题。

安装它：

npm install -g clipboard-cli

然后你可以这样做：

echo foo | clipboard

如果需要，您可以通过在、或cb中添加以下内容来别名：.bashrc``.bash_profile``.zshrc

alias cb=clipboard


更多建议

确保您正在使用alias xclip="xclip -selection c"，否则您将无法使用Ctrl+进行粘贴v。

示例：运行后echo -n test | xclip，Ctrl+v将粘贴test

如何列出包含给定提交的分支？

Arya

2023-02-17 04:10

我如何查询 git 以找出哪些分支包含给定的提交？gitk通常会列出分支，除非有太多，在这种情况下它只会说“很多（38）”或类似的东西。我需要知道完整列表，或者至少某些分支是否包含提交。



解答

从git-branch 手册页：

 git branch --contains <commit>

仅列出包含指定提交的分支（如果未指定，则为 HEAD）。暗示--list。

 git branch -r --contains <commit>

还列出 远程跟踪分支 （如user3941992下面的回答中所述），即“与远程分支有直接关系的本地分支”。

正如Carl Walsh所指出的，这仅适用于默认的 refspec

fetch = +refs/heads/*:refs/remotes/origin/*

如果您需要包含其他 ref 命名空间（pull request，Gerrit，...），您需要添加新的 refspec，然后再次获取：

git config --add remote.origin.fetch "+refs/pull/*/head:refs/remotes/origin/pr/*"
git fetch
git branch -r --contains <commit>

另请参阅这篇git ready文章。

该--contains标签将确定某个提交是否已被引入您的分支。也许您从您认为已应用的补丁中获得了提交 SHA，或者您只是想检查您最喜欢的将内存使用量减少 75% 的开源项目的提交是否已经提交。

$ git log -1 tests
commit d590f2ac0635ec0053c4a7377bd929943d475297
Author: Nick Quaranto <nick@quaran.to>
Date:   Wed Apr 1 20:38:59 2009 -0400

    Green all around, finally.

$ git branch --contains d590f2
  tests
* master

注意： 如果提交在 远程跟踪分支上，请添加-a选项。
（如下MichielB评论）

git branch -a --contains <commit>

MatrixFrog评论说它只显示哪些分支包含那个 确切的 提交。
如果您想知道哪些分支包含“等效”提交（即哪些分支已经精心挑选了该提交），那就是 git cherry ：

因为git cherry 比较的是变更集而不是提交 ID (sha1) ，所以您可以使用它git cherry来确定您在本地进行的提交是否已<upstream>在不同的提交 ID 下应用。
例如，如果您<upstream>通过电子邮件提供补丁而不是直接推送或拉取提交，就会发生这种情况。

           __*__*__*__*__> <upstream>
          /
fork-point
          \__+__+__-__+__+__-__+__> <head>

（在这里，标记为 ' ' 的提交-不会显示在 中git cherry，这意味着它们已经存在于 中<upstream>。）


更多建议

你可以运行：

git log <SHA1>..HEAD --ancestry-path --merges

从输出中最后一次提交的评论中，您可以找到 原始分支 名称

例子：

       c---e---g--- feature
      /         \
-a---b---d---f---h---j--- master

git log e..master --ancestry-path --merges

commit h
Merge: g f
Author: Eugen Konkov <>
Date:   Sat Oct 1 00:54:18 2016 +0300

    Merge branch 'feature' into master
