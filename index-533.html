然而，还有一个更有前途的主题是“扩展一切”，特别关注属性和静态类甚至字段。

更新：2016 年 8 月

由于 dotnet 团队发布了C# 7.0 中的新功能以及来自Mads Torgensen的评论：

扩展属性：我们有一个（出色的！）实习生在整个夏天将它们作为实验与其他类型的扩展成员一起实现。我们仍然对此感兴趣，但这是一个很大的变化，我们需要确信它是值得的。

扩展属性和其他成员似乎仍然是 Roslyn 未来版本中包含的很好的候选者，但可能不是 7.0 版本。

更新时间：2017 年 5 月

扩展成员 已关闭，因为 扩展所有问题 的副本也已关闭。主要讨论实际上是关于广义上的类型可扩展性。该功能现在 作为提案在此处 进行跟踪，并已从 7.0 里程碑 中删除。

更新：2017 年 8 月 - C# 8.0 提议的功能

虽然它仍然只是一个 提议的 功能，但我们现在对它的语法有了更清晰的认识。请记住，这也将是扩展方法的新语法：

public interface IEmployee 
{
    public decimal Salary { get; set; }
}

public class Employee
{
    public decimal Salary { get; set; }
}

public extension MyPersonExtension extends Person : IEmployee
{
    private static readonly ConditionalWeakTable<Person, Employee> _employees = 
        new ConditionalWeakTable<Person, Employee>();


    public decimal Salary
    {
        get 
        {
            // `this` is the instance of Person
            return _employees.GetOrCreate(this).Salary; 
        }
        set 
        {
            Employee employee = null;
            if (!_employees.TryGetValue(this, out employee)
            {
                employee = _employees.GetOrCreate(this);
            }
            employee.Salary = value;
        }
    }
}

IEmployee person = new Person();
var salary = person.Salary;

类似于部分类，但在不同的程序集中编译为单独的类/类型。请注意，您还可以通过这种方式添加静态成员和运算符。正如Mads Torgensen 播客中提到的， _ 扩展没有任何状态（因此它不能将私有实例成员添加到类中），这意味着您将无法添加链接到实例的私有实例数据_ 。调用的原因是它意味着管理内部字典并且它可能很困难（内存管理等......）。为此，您仍然可以使用前面描述的TypeDescriptor/ConditionalWeakTable技术和属性扩展，将其隐藏在一个不错的属性下。

正如这个问题所暗示的那样，语法仍然可能会发生变化。例如，extends可以替换为for某些人可能感觉更自然且与 Java 相关性较低的内容。

2018 年 12 月更新 - 角色、扩展和静态接口成员

扩展一切都 没有到 C# 8.0，因为一些缺点解释为这个GitHub 票的结尾。于是，就有了改进设计的探索。在这里，Mads Torgensen 解释了什么是 角色和扩展 以及它们的不同之处：

角色允许在给定类型的特定值上实现接口。扩展允许在特定代码区域内对给定类型的所有值实现接口。

    }

从字符串列表中删除空字符串

Arya

2023-02-17 09:23

我想从 python 中的字符串列表中删除所有空字符串。

我的想法是这样的：

while '' in str_list:
    str_list.remove('')

还有更多的pythonic方法可以做到这一点吗？



解答

我会用filter：

str_list = filter(None, str_list)
str_list = filter(bool, str_list)
str_list = filter(len, str_list)
str_list = filter(lambda item: item, str_list)

Python 3 从 返回一个迭代器filter，因此应该包含在对list()

str_list = list(filter(None, str_list))


更多建议

使用列表理解是最 Pythonic 的方式：

>>> strings = ["first", "", "second"]
>>> [x for x in strings if x]
['first', 'second']

如果列表必须就地修改，因为有其他引用必须看到更新的数据，那么使用切片赋值：

strings[:] = [x for x in strings if x]
