Mockito 可以捕获多次调用的方法的参数吗？

Arya

2023-02-18 03:48

我有一个被调用两次的方法，我想捕获第二次方法调用的参数。

这是我尝试过的：

ArgumentCaptor<Foo> firstFooCaptor = ArgumentCaptor.forClass(Foo.class);
ArgumentCaptor<Foo> secondFooCaptor = ArgumentCaptor.forClass(Foo.class);
verify(mockBar).doSomething(firstFooCaptor.capture());
verify(mockBar).doSomething(secondFooCaptor.capture());
// then do some assertions on secondFooCaptor.getValue()

但是我得到一个TooManyActualInvocations异常，因为 Mockito 认为doSomething应该只调用一次。

如何验证第二次调用的参数doSomething？



解答

我觉得应该是

verify(mockBar, times(2)).doSomething(...)

来自mockito javadoc的示例：

ArgumentCaptor<Person> peopleCaptor = ArgumentCaptor.forClass(Person.class);
verify(mock, times(2)).doSomething(peopleCaptor.capture());

List<Person> capturedPeople = peopleCaptor.getAllValues();
assertEquals("John", capturedPeople.get(0).getName());
assertEquals("Jane", capturedPeople.get(1).getName());


更多建议

自 Mockito 2.0 以来，还可以使用静态方法Matchers.argThat(ArgumentMatcher)。在 Java 8 的帮助下，现在编写起来更清晰、更易读：

verify(mockBar).doSth(argThat((arg) -> arg.getSurname().equals("OneSurname")));
verify(mockBar).doSth(argThat((arg) -> arg.getSurname().equals("AnotherSurname")));

如果您依赖于较低的 Java 版本，那么也不错：

verify(mockBar).doSth(argThat(new ArgumentMatcher<Employee>() {
        @Override
        public boolean matches(Object emp) {
            return ((Employee) emp).getSurname().equals("SomeSurname");
        }
    }));

当然，这些都不能验证调用顺序——为此你应该使用InOrder：

InOrder inOrder = inOrder(mockBar);

inOrder.verify(mockBar).doSth(argThat((arg) -> arg.getSurname().equals("FirstSurname")));
inOrder.verify(mockBar).doSth(argThat((arg) -> arg.getSurname().equals("SecondSurname")));

请查看mockito-java8项目，它可以进行如下调用：

verify(mockBar).doSth(assertArg(arg -> assertThat(arg.getSurname()).isEqualTo("Surname")));

Linux：复制并创建目标目录（如果不存在）

Arya

2023-02-18 03:48

我想要一个命令（或者可能是 cp 的一个选项）来创建目标目录（如果它不存在）。

例子：

cp -? file /path/to/copy/file/to/is/very/deep/there



解答

mkdir -p "$d" && cp file "$d"

（没有这样的选项cp）。


更多建议

如果以下两项都为真：

  1. 您使用的是 GNU 版本cp（而不是 Mac 版本），并且
  2. 您正在从一些现有的目录结构中复制，您只需要重新创建它

那么你可以用--parents的标志来做到这一点cp。从信息页面（可在http://www.gnu.org/software/coreutils/manual/html_node/cp- invocation.html#cp-invocation或使用info cpor查看man cp）：

--parents
     Form the name of each destination file by appending to the target
     directory a slash and the specified name of the source file.  The
     last argument given to `cp' must be the name of an existing
     directory.  For example, the command:

          cp --parents a/b/c existing_dir

     copies the file `a/b/c' to `existing_dir/a/b/c', creating any
     missing intermediate directories.

例子：

/tmp $ mkdir foo
/tmp $ mkdir foo/foo
/tmp $ touch foo/foo/foo.txt
/tmp $ mkdir bar
/tmp $ cp --parents foo/foo/foo.txt bar
/tmp $ ls bar/foo/foo
foo.txt

将宽度/高度设置为百分比减去像素

Arya

2023-02-18 03:47

我正在尝试创建一些可重复使用的 CSS 类以提高网站的一致性并减少混乱，并且我一直在尝试标准化我经常使用的东西。

我有一个容器<div>，我不想为其设置高度（因为它会根据它在网站上的位置而有所不同），里面是一个 header <div>，然后是一个 无序列表 的项目，所有这些都应用了 CSS他们。

它看起来很像这样：

我希望 无序列表 占据容器中剩余的空间<div>，因为标题<div>很高18px。我只是不知道如何将列表的高度指定为“100%减号的结果18px”。

我已经在 SO 的其他几个上下文中看到过这个问题，但我认为对于我的特殊情况再次询问是值得的。在这种情况下有人有什么建议吗？



解答

您可以使用calc：

height: calc(100% - 18px);

请注意，某些旧浏览器不支持 CSS3calc()功能，因此可能需要实现该功能的供应商特定版本：

/* Firefox */
height: -moz-calc(100% - 18px);
/* WebKit */
height: -webkit-calc(100% - 18px);
/* Opera */
height: -o-calc(100% - 18px);
/* Standard */
height: calc(100% - 18px);


更多建议

对于一些不同的方法，您可以在列表中使用类似这样的方法：

position: absolute;
top: 18px;
bottom: 0px;
width: 100%;

只要父容器有position: relative;
