UUID有多独特？

Arya

2023-02-17 21:13

使用 UUID 来唯一标识某物有多安全（我将它用于上传到服务器的文件）？据我了解，它基于随机数。然而，在我看来，如果有足够的时间，它最终会自我重复，只是出于偶然。是否有更好的系统或某种类型的模式来缓解这个问题？



解答

非常安全：

一个人每年被陨石击中的风险估计为 170 亿分之一，这意味着概率约为 0.00000000006 (6 × 10 -11 )，相当于创建数十万亿个 UUID 的几率在一年内有一个副本。换句话说，仅在接下来的 100 年内每秒生成 10 亿个 UUID 后，仅创建一个副本的概率约为 50%。

警告：

但是，这些概率仅在使用足够的熵生成 UUID 时成立。否则，重复的概率可能会高得多，因为统计离散度可能较低。在分布式应用程序需要唯一标识符的情况下，即使合并来自许多设备的数据，UUID 也不会发生冲突，每个设备上使用的种子和生成器的随机性必须在应用程序的生命周期内可靠。如果这不可行，RFC4122 建议改用名称空间变体。

资料来源：维基百科关于通用唯一标识符的文章的 随机 UUID 重复概率 部分（链接指向 2016 年 12 月的修订，然后编辑修改了该部分）。

另请参阅同一通用唯一标识符文章 Collisions 中关于同一主题的当前部分。


更多建议

如果“给定足够的时间”是指 100 年，而你以每秒十亿次的速度创造它们，那么是的，100 年后你有 50% 的机会发生碰撞。

如何用 grep 排除一个词？

Arya

2023-02-17 21:12

我需要这样的东西：

grep ^"unwanted_word"XXXXXXXX



解答

您可以使用grep 的-v(for --invert-match) 选项来做到这一点：

grep -v "unwanted_word" file | grep XXXXXXXX

grep -v "unwanted_word" file将过滤具有的行unwanted_word，并将grep XXXXXXXX仅列出具有 pattern 的行XXXXXXXX。

编辑：

从您的评论看来，您想列出所有没有unwanted_word. 在这种情况下，您只需要：

grep -v 'unwanted_word' file


更多建议

我将问题理解为“如何匹配一个词但排除另一个词”，其中一个解决方案是串联两个 grep：第一个 grep 找到想要的“word1”，第二个 grep 排除“word2”：

grep "word1" | grep -v "word2"

在我的例子中：我需要区分“plot”和“#plot”，grep 的“word”选项不会这样做（“#”不是字母数字）。

如何从 C# 中的泛型方法返回 NULL？

Arya

2023-02-17 21:12

我有一个带有这个（虚拟）代码的通用方法（是的，我知道 IList 有谓词，但我的代码没有使用 IList，而是使用其他一些集合，无论如何这与问题无关......）

    static T FindThing<T>(IList collection, int id) where T : IThing, new()
    {
        foreach (T thing in collection)
        {
            if (thing.Id == id)
                return thing;
        }
        return null;  // ERROR: Cannot convert null to type parameter 'T' because it could be a value type. Consider using 'default(T)' instead.
    }

这给了我一个构建错误

“无法将 null 转换为类型参数‘T’，因为它可能是值类型。请考虑改用‘default(T)’。”

我可以避免这个错误吗？



解答

三个选项：

  • 返回default（或对于旧版本的 C#），这意味着如果是引用类型（或可为 null 的值类型）、for 、for等default(T)，您将返回。（默认值表（C# 参考））null``T``0``int``'\0'``char
  • 如果您乐于将T约束限制为引用类型where T : class，然后null正常返回
  • 如果您乐于使用约束将其限制T为不可为空的值类型where T : struct，那么您可以再次null从返回值为的方法正常返回T?- 请注意，这不是 返回 空引用，而是返回空值可为 null 的值类型。


更多建议

return default(T);
