SQL Server 中 1/1/1753 的意义是什么？

Arya

2023-02-18 06:38

为什么是 1753？他们有什么反对1752的？我的曾曾曾曾曾曾曾曾曾曾祖父会非常生气。



解答

在 SQL Server 中使用 1st January 1753( 1753-01-01) 作为日期时间的最小日期值的决定可以追溯到它的Sybase 起源。

日期本身的重要性可以归因于这个人。

菲利普·斯坦霍普，第四代切斯特菲尔德伯爵。谁通过英国议会指导了1750 年日历（新式）法案。这为英国及其当时的殖民地采用公历制定了法律。

1752 年的英国日历中有一些缺失的日子（互联网档案链接），最终根据儒略历进行了调整。1752年9月3日至1752年9月13日丢失。

Kalen Delaney这样解释了选择

那么，如果有 12 天的损失，您如何计算日期呢？例如，如何计算从 1492 年 10 月 12 日到 1776 年 7 月 4 日之间的天数？你包括那些失踪 12 天的人吗？为避免必须解决此问题，最初的 Sybase SQL Server 开发人员决定不允许 1753 年之前的日期。您可以使用字符字段存储更早的日期，但不能将任何日期时间函数与存储在字符中的更早日期一起使用领域。

1753 年的选择似乎有点以英国为中心，但是因为欧洲许多天主教国家在英国实施之前已经使用了 170 年的日历（最初由于教会的反对而推迟）。相反，许多国家直到很久以后才改革他们的日历，1918 年在俄罗斯。事实上，1917 年的十月革命是在公历 11 月 7 日开始的。

Joe 的回答中提到的两者datetime和新数据类型都没有尝试考虑这些地方差异，而只是使用公历。datetime2

所以随着更大范围的datetime2

SELECT CONVERT(VARCHAR, DATEADD(DAY,-5,CAST('1752-09-13' AS DATETIME2)),100)

退货

Sep  8 1752 12:00AM

数据类型的最后一点datetime2是，它使用远期公历倒推到它实际发明之前，因此在处理历史日期方面用途有限。

这与其他软件实现形成对比，例如 Java公历类，它默认遵循儒略历直到 1582 年 10 月 4 日，然后在新的公历中跳转到 1582 年 10 月 15 日。它可以正确处理该日期之前的闰年 Julian 模型和该日期之后的 Gregorian 模型。调用者可以通过调用 来更改切换日期setGregorianChange()。

可以在此处找到一篇相当有趣的文章，其中讨论了采用日历的一些更多特点。

如何从 Axios 中的 HTTP 错误获取状态码？

Arya

2023-02-18 06:37

这可能看起来很愚蠢，但我试图在 Axios 中请求失败时获取错误数据。

axios
  .get('foo.example')
  .then((response) => {})
  .catch((error) => {
    console.log(error); //Logs a string: Error: Request failed with status code 404
  });

除了字符串，是否有可能获得一个可能包含状态代码和内容的对象？例如：

Object = {status: 404, reason: 'Not found', body: '404 Not found'}



解答

toString你看到的是对象的方法返回的字符串error。（error不是字符串。）

如果已从服务器收到响应，则该error对象将包含以下response属性：

axios.get('/foo')
  .catch(function (error) {
    if (error.response) {
      console.log(error.response.data);
      console.log(error.response.status);
      console.log(error.response.headers);
    }
  });


更多建议

使用 TypeScript，很容易找到你想要的正确类型。

这使一切变得更容易，因为您可以获得具有自动完成功能的类型的所有属性，因此您可以知道响应和错误的正确结构。

import { AxiosResponse, AxiosError } from 'axios'

axios.get('foo.example')
  .then((response: AxiosResponse) => {
    // Handle response
  })
  .catch((reason: AxiosError) => {
    if (reason.response!.status === 400) {
      // Handle 400
    } else {
      // Handle else
    }
    console.log(reason.message)
  })

此外，您可以将参数传递给这两种类型，以告诉您在内部期望什么，response.data如下所示：

import { AxiosResponse, AxiosError } from 'axios'
axios.get('foo.example')
  .then((response: AxiosResponse<{user:{name:string}}>) => {
    // Handle response
  })
  .catch((reason: AxiosError<{additionalInfo:string}>) => {
    if (reason.response!.status === 400) {
      // Handle 400
    } else {
      // Handle else
    }
    console.log(reason.message)
  })

AngularJS ngClass 条件

Arya

2023-02-18 06:37

有没有什么办法可以表达像ng-class条件一样的东西？

例如，我尝试了以下操作：

<span ng-class="{test: 'obj.value1 == \'someothervalue\''}">test</span>

这段代码的问题在于，无论是什么obj.value1，类测试总是应用于元素。这样做：

<span ng-class="{test: obj.value2}">test</span>

只要obj.value2不等于真值，就不会应用该类。现在我可以通过这样做来解决第一个示例中的问题：

<span ng-class="{test: checkValue1()}">test</span>

函数checkValue1看起来像这样：

$scope.checkValue1 = function() {
  return $scope.obj.value === 'somevalue';
}

我只是想知道这是否应该ng- class工作。我还在构建一个自定义指令，我想在其中执行与此类似的操作。但是，我找不到观看表达式的方法（也许这是不可能的，也是它这样工作的原因）。

这是一个plnkr来说明我的意思。



解答

您的第一次尝试几乎是正确的，它应该在没有引号的情况下工作。

{test: obj.value1 == 'someothervalue'}

这是一个plnkr。

ngClass 指令适用于任何求值为 truthy 或 falsey 的表达式，有点类似于 Javascript 表达式，但有一些区别，您可以在此处阅读相关信息。如果您的条件太复杂，那么您可以使用返回 truthy 或 falsey 的函数，就像您在第三次尝试中所做的那样。

补充一下：您还可以使用逻辑运算符来形成逻辑表达式，例如

ng-class="{'test': obj.value1 == 'someothervalue' || obj.value2 == 'somethingelse'}"


更多建议

在 ng-repeat 中使用 ng-class

<table>
    <tbody>
            <tr ng-repeat="task in todos"
                ng-class="{'warning': task.status == 'Hold' , 'success': task.status == 'Completed',
              'active': task.status == 'Started', 'danger': task.status == 'Pending' } ">
                <td>{{$index + 1}}</td>
                <td>{{task.name}}</td>
                <td>{{task.date|date:'yyyy-MM-dd'}}</td>
                <td>{{task.status}}</td>
            </tr>
    </tbody>
</table>

对于 task.status 中的每个状态，该行使用不同的类。
