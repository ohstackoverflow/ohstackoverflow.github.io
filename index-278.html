Vim 中的 remap、noremap、nnoremap 和 vnoremap 映射命令有什么区别？

Arya

2023-02-17 02:54

remapVim 中的,noremap​​ ,nnoremap和mapping 命令有什么区别vnoremap？



解答

remap是一个使映射递归工作的 选项。 默认情况下它是打开的，我建议你保持这种状态。其余为 映射命令 ，说明如下：

:map并且:noremap是各种映射命令的 递归 和 非递归版本。 例如，如果我们运行：

:map j gg           (moves cursor to first line)
:map Q j            (moves cursor to first line)
:noremap W j        (moves cursor down one line)

然后：

  • j将映射到gg.
  • Q 也 将映射到gg，因为j将针对递归映射进行扩展。
  • W将映射到j（而不是gg），因为j不会针对非递归映射进行扩展。

现在记住 Vim 是一个 模态编辑器 。它有 普通 模式、 可视 模式等多种模式。

对于这些映射集中的每一个，都有一个在正常模式、视觉模式、选择模式和运算符模式（和）下工作的映射，一个在正常模式（和）下工作的映射，一个在视觉模式（和）下工作的映射，等等。:map``:noremap``:nmap``:nnoremap``:vmap``:vnoremap

有关这方面的更多指导，请参阅：

:help :map
:help :noremap
:help recursive_mapping
:help :map-modes


更多建议

我认为 Vim 文档应该已经解释了这些命令命名背后的含义。仅仅告诉你他们做什么并不能帮助你记住这些名字。

map是所有递归映射命令的“根”。根形式适用于“正常”、“视觉+选择”和“操作员等待”模式。（我在语言学中使用术语“根” 。）

noremap是所有非递归映射命令的“根”。根形式适用于与 相同的模式map。（认为nore​​前缀意味着“非递归”。）

（请注意，还有适用于插入和命令行的!模式。）map!

请参阅下文了解“递归”在这种情况下的含义。

在模式字母前加上n修改映射工作的模式。它可以选择适用模式列表的一个子集（例如，仅“视觉”），或选择其他不适用的模式（例如map“插入”）。

使用help map-modes将向您展示一些表格，这些表格解释了如何控制映射适用于哪些模式。

模式字母：

  • n: 正常
  • v：视觉和选择
  • o: 运营商待定
  • x: 仅视觉
  • s: 仅选择
  • i： 插入
  • c： 命令行
  • l：插入、命令行、正则表达式搜索（和其他。统称为“Lang-Arg”伪模式）

“ 递归 ”表示映射展开为一个结果，然后该结果展开为另一个结果，以此类推。

当其中之一为真时，扩展停止：

  1. 结果不再映射到其他任何东西。
  2. 已应用非递归映射（即“noremap”[或其类似物之一]是最终扩展）。

那时，Vim 对最终结果的默认“含义”被应用/执行。

“ 非递归 ”表示映射仅扩展一次，然后应用/执行该结果。

例子：

 nmap K H
 nnoremap H G
 nnoremap G gg

上述原因K扩展到H，然后H扩展到G并停止。它因 而停止nnoremap，它会立即展开并停止。G将被执行的意思（即“跳转到最后一行”）。至多一个非递归映射将应用于扩展链（这将是最后一次发生的扩展）。

Gto的映射gg仅在您按下时适用G，但在您按下时不适用K。K无论是否递归映射，此映射都不会影响按下G，因为是第 2 行导致 的扩展K停止，因此不会使用第 3 行。

NP、NP-Complete 和 NP-Hard 之间有什么区别？

Arya

2023-02-17 02:54

NP 、 NP-Complete 和 NP-Hard 之间有什么区别？

我知道网络上有很多资源。我想阅读您的解释，原因是它们可能与那里的内容不同，或者有一些我不知道的东西。



解答

我假设您正在寻找直观的定义，因为技术定义需要相当长的时间才能理解。首先，让我们记住一个初步需要的概念来理解这些定义。

  • 决策问题：回答 是 或 否 的问题。

现在，让我们定义那些 复杂性类 。

P

P 是一个复杂度类，表示可以在多项式时间内解决的所有决策问题的集合 。

也就是说，给定一个问题实例，可以在多项式时间内确定答案是或否。

例子

给定一个连通图G，它的顶点可以用两种颜色着色，这样就没有边是单色的了吗？

算法：从任意顶点开始，将其着色为红色，将其所有邻居着色为蓝色，然后继续。当您用完所有顶点或您被迫使一条边的两个端点颜色相同时停止。

NP

NP 是一个复杂性类，表示所有决策问题的集合，对于这些决策问题，答案为“是”的实例具有可以在多项式时间内验证的证明。

这意味着如果有人给我们一个问题的实例和证明（有时称为见证）证明答案是肯定的，我们可以在多项式时间内检查它是否正确。

例子

整数因式分解 在 NP 中。n这是给定整数和 的问题m，是否有一个整数与 相除f（1 < f < m是的一个小因数）？f``n``f``n

这是一个决策问题，因为答案是“是”或“否”。如果有人给我们一个问题的实例（所以他们给我们整数n和m）和一个整数f与1 < f < m，并声称这f是（证书）的一个因素，我们可以通过执行除法在 多项式时间n内检查答案。 __n / f

NP完全

NP-Complete 是一个复杂性类，它表示 NP 中所有问题的集合，X对于这些问题，可以将任何其他 NP 问题减少Y到X多项式时间内。

直觉上，这意味着Y如果我们知道如何快速解决问题，我们就可以X快速解决问题。准确地说，Y可归约为X，如果存在多项式时间算法将的f实例转换为多项式时间的实例，并且具有答案为是的属性，当且仅当答案为是时。y``Y``x = f(y)``X``y``f(y)

例子

3-SAT. 这是一个问题，其中我们给出了 3 子句析取 (OR) 的连词 (AND)，形式为

(x_v11 OR x_v21 OR x_v31) AND 
(x_v12 OR x_v22 OR x_v32) AND 
...                       AND 
(x_v1n OR x_v2n OR x_v3n)

其中每个x_vij是布尔变量或来自有限预定义列表的变量的否定(x_1, x_2, ... x_n)。

可以证明， 每个 NP 问题都可以简化为 3-SAT 。对此的证明是技术性的，需要使用 NP 的技术定义（ 基于非确定性图灵机 ）。这被称为 库克定理 。

使 NP 完全问题变得重要的是，如果可以找到确定性多项式时间算法来解决其中一个问题，则每个 NP 问题都可以在多项式时间内解决（一个问题可以解决所有问题）。

NP难

直觉上，这些问题 至少与 NP 完全问题一样难 。请注意，NP-hard 问题 不一定是 NP 问题 ，也不 一定是决策问题 。

这里的精确定义是， _一个问题X是 NP-hard，如果存在一个 NP-complete 问题Y，那么它Y可以X在多项式时间内_减少到。

但是由于任何 NP 完全问题都可以在多项式时间内还原为任何其他 NP 完全问题，因此所有 NP 完全问题都可以在多项式时间内还原为任何 NP 难问题。那么，如果在多项式时间内存在一个 NP-hard 问题的解，则在多项式时间内存在所有 NP 问题的解。

例子

停机问题是一个 NP-hard 问题。P这是给定程序和输入的问题I，它会停止吗？这是一个决策问题，但它不在 NP 中。很明显，任何 NP 完全问题都可以简化为这个问题。再举一个例子，任何 NP 完全问题都是 NP 难的。

我最喜欢的 NP 完全问题是扫雷问题。

P = NP

这是计算机科学中最著名的问题，也是数学科学中最重要的突出问题之一。事实上，Clay Institute正在提供 100 万美元来解决这个问题（Stephen Cook 在 Clay 网站上的文章非常好）。

很明显 P 是 NP 的子集。悬而未决的问题是 NP 问题是否具有确定性多项式时间解。人们普遍认为他们没有。这是最近一篇关于 P = NP 问题的最新（和重要性）的杰出文章：The Status of the P versus NP problem。

关于这个主题的最好的书是Garey 和 Johnson 的Computers and Intractability。

撤消 git pull，如何将 repos 恢复到旧状态

Arya

2023-02-17 02:53

有什么方法可以恢复或撤消 git pull 以便我的源/回购将回到执行 git pull 之前的旧状态？我想这样做是因为它合并了一些我不想这样做的文件，但只合并了其他剩余的文件。那么，我想取回那些文件，这可能吗？

编辑：我想撤消 git merge 以进行澄清。看到一些答案后，我这样做了

git reflog
bb3139b... HEAD@{0}: pull : Fast forward
01b34fa... HEAD@{1}: clone: from ...name...

现在，我该怎么办？做的git reset --hard可以吗？我不想再搞砸了，所以要求详细步骤？



解答

运行git pull按顺序执行以下任务：

  1. git fetch
  2. git merge

合并步骤将已设置的分支合并到您的配置中。您想要撤消 合并 步骤，但可能不想撤消 提取 （没有多大意义，也没有必要）。

要撤消 合并 ，请使用git reset --hard将本地存储库重置为以前的状态；使用git-reflog找到先前状态的 SHA-1，然后重置为它。

警告

本节中列出的命令删除所有未提交的更改，可能会导致工作丢失：

git reset --hard

或者，重置为特定时间点，例如：

git reset --hard master@{"10 minutes ago"}


更多建议

与 jkp 的回答相同，但这是完整的命令：

git reset --hard a0d3fe6

通过执行找到 a0d3fe6 的位置

git reflog

并查看您要撤消的点。
