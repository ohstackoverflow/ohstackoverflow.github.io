jQuery/JavaScript 来替换损坏的图像

Arya

2023-02-18 02:17

我有一个包含一堆图像的网页。有时图片不可用，因此客户端浏览器中会显示损坏的图片。

我如何使用 jQuery 获取图像集，将其过滤为损坏的图像然后替换 src？

--我认为使用 jQuery 会更容易，但事实证明使用纯 JavaScript 解决方案要容易得多，即 Prestaul 提供的解决方案。



解答

onError使用 JavaScript 处理图像事件以重新分配其来源：

function imgError(image) {
    image.onerror = "";
    image.src = "/images/noimage.gif";
    return true;
}



<img src="image.png" onerror="imgError(this);"/>

或者没有 JavaScript 函数：

<img src="image.png" onError="this.onerror=null;this.src='/images/noimage.gif';" />

以下兼容性表列出了支持错误工具的浏览器：

http://www.quirksmode.org/dom/events/error.html


更多建议

我使用内置error处理程序：

$("img").error(function () {
    $(this).unbind("error").attr("src", "broken.gif");
});

编辑： 该方法在jquery 1.8error()及更高版本中已弃用。相反，您应该改用：.on("error")

$("img").on("error", function () {
    $(this).attr("src", "broken.gif");
});

Hibernate 抛出 MultipleBagFetchException - 不能同时获取多个包

Arya

2023-02-18 02:17

Hibernate 在 SessionFactory 创建期间抛出这个异常：

org.hibernate.loader.MultipleBagFetchException：不能同时获取多个包

这是我的测试用例：

父类.java

@Entity
public Parent {

 @Id
 @GeneratedValue(strategy=GenerationType.IDENTITY)
 private Long id;

 @OneToMany(mappedBy="parent", fetch=FetchType.EAGER)
 // @IndexColumn(name="INDEX_COL") if I had this the problem solve but I retrieve more children than I have, one child is null.
 private List<Child> children;

}

孩子.java

@Entity
public Child {

 @Id
 @GeneratedValue(strategy=GenerationType.IDENTITY)
 private Long id;

 @ManyToOne
 private Parent parent;

}

这个问题怎么样？我能做些什么？

编辑

好的，我遇到的问题是另一个“父”实体在我的父母内部，我的真实行为是这样的：

父类.java

@Entity
public Parent {

 @Id
 @GeneratedValue(strategy=GenerationType.IDENTITY)
 private Long id;

 @ManyToOne
 private AnotherParent anotherParent;

 @OneToMany(mappedBy="parent", fetch=FetchType.EAGER)
 private List<Child> children;

}

另一个父类.java

@Entity
public AnotherParent {

 @Id
 @GeneratedValue(strategy=GenerationType.IDENTITY)
 private Long id;

 @OneToMany(mappedBy="parent", fetch=FetchType.EAGER)
 private List<AnotherChild> anotherChildren;

}

Hibernate 不喜欢带有 的两个集合FetchType.EAGER，但这似乎是一个错误，我没有做不寻常的事情......

FetchType.EAGER从问题中移除Parent或AnotherParent解决问题，但我需要它，所以真正的解决方案是使用@LazyCollection(LazyCollectionOption.FALSE)而不是FetchType（感谢Bozho提供的解决方案）。



解答

我认为较新版本的 hibernate（支持 JPA 2.0）应该可以解决这个问题。但除此之外，您可以通过使用以下方式注释集合字段来解决它：

@LazyCollection(LazyCollectionOption.FALSE)

fetchType请记住从注释中删除该属性@*ToMany。

但请注意，在大多数情况下 aSet<Child>比 更合适List<Child>，所以除非您真的需要 a List- 去Set

但请注意，使用集合 不会 消除Vlad Mihalcea 在他的回答中描述的底层 笛卡尔积 ！


更多建议

只需从List一种类型更改为另Set一种类型。

但请注意，您 不会 消除Vlad Mihalcea 在他的回答中所描述的 底层笛卡尔积 ！

Windows 和 Linux 目录名称中禁止使用哪些字符？

Arya

2023-02-18 02:16

我知道这/在 Linux 中是非法的，而以下在 Windows 中是非法的（我认为）* . " / \ [ ] : ; | ,

我还缺少什么？

然而，我需要一份全面的指南，并且需要一份考虑到双字节字符的指南。链接到外部资源对我来说很好。

我需要首先使用可能包含禁止字符的名称在文件系统上创建一个目录，因此我打算用下划线替换这些字符。然后我需要将此目录及其内容写入一个 zip 文件（使用 Java），因此任何有关 zip 目录名称的额外建议将不胜感激。



解答

禁用文件名字符的“综合指南”在 Windows 上不起作用，因为它保留了文件名和字符。是的，像和其他这样的字符* " ?是被禁止的，但是有无数的名字只由被禁止的有效字符组成。例如，空格和点是有效的文件名字符，但禁止仅由这些字符组成的名称。

Windows 不区分大写和小写字符，因此A如果命名的文件夹a已经存在，则无法创建命名的文件夹。PRN更糟糕的是，像andCON和许多其他看似允许的名称被保留且不允许使用。Windows 也有几个长度限制；如果移动到另一个文件夹，则在一个文件夹中有效的文件名可能会变得无效。命名文件和文件夹的规则在 Microsoft 文档中。

通常，您不能使用用户生成的文本来创建 Windows 目录名称。如果您希望允许用户命名任何他们想要的名称，您必须创建安全的名称，例如A、AB等A2，将用户生成的名称及其等效路径存储在应用程序数据文件中，并在您的应用程序中执行路径映射。

如果您绝对必须允许用户生成的文件夹名称，则判断它们是否无效的唯一方法是捕获异常并假定该名称无效。即使这样也充满了危险，因为拒绝访问、脱机驱动器和驱动器空间不足而抛出的异常与因无效名称而抛出的异常重叠。你正在打开一个巨大的伤害罐头。
