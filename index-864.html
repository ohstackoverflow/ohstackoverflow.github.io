JavaScript 对象的查询字符串编码

Arya

2023-02-17 18:12

有没有一种快速简单的方法可以将 JavaScript 对象编码为string可以通过GET请求传递的对象？

没有jQuery，没有其他框架——只有普通的 JavaScript :)



解答

像这样：

serialize = function(obj) {
  var str = [];
  for (var p in obj)
    if (obj.hasOwnProperty(p)) {
      str.push(encodeURIComponent(p) + "=" + encodeURIComponent(obj[p]));
    }
  return str.join("&");
}

console.log(serialize({
  foo: "hi there",
  bar: "100%"
}));
// foo=hi%20there&bar=100%25

这一个还转换递归对象（使用 PHP“数组”表示法查询字符串）：

serialize = function(obj, prefix) {
  var str = [],
    p;
  for (p in obj) {
    if (obj.hasOwnProperty(p)) {
      var k = prefix ? prefix + "[" + p + "]" : p,
        v = obj[p];
      str.push((v !== null && typeof v === "object") ?
        serialize(v, k) :
        encodeURIComponent(k) + "=" + encodeURIComponent(v));
    }
  }
  return str.join("&");
}

console.log(serialize({
  foo: "hi there",
  bar: {
    blah: 123,
    quux: [1, 2, 3]
  }
}));
// foo=hi%20there&bar%5Bblah%5D=123&bar%5Bquux%5D%5B0%5D=1&bar%5Bquux%5D%5B1%5D=2&bar%5Bquux%5D%5B2%5D=3


更多建议

只需使用URLSearchParams这适用于所有当前浏览器

new URLSearchParams(object).toString()

如何克隆到非空目录？

Arya

2023-02-17 18:12

我有目录 A 和与目录 B 匹配的文件。目录 A 可能有其他需要的文件。目录 B 是一个 git 仓库。

我想将目录 B 克隆到目录 A 但 git-clone 不允许我这样做，因为该目录是非空的。

我希望它只是克隆 .git 并且因为所有文件都匹配我可以从那里去吗？

我无法克隆到一个空目录中，因为我在目录 A 中有不在目录 B 中的文件，我想保留它们。

复制 .git 不是一个选项，因为我想要 refs 来推/拉，我不想手动设置它们。

有什么办法吗？

更新：我认为这可行，任何人都可以看到任何问题吗？-->

cd a
git clone --no-hardlinks --no-checkout ../b a.tmp 
mv a.tmp/.git .
rm -rf a.tmp
git unstage # apparently git thinks all the files are deleted if you don't do this



解答

在以下 shell 命令中existing-dir是一个目录，其内容与 git 存储库中的跟踪文件相匹配repo-to-clone。

# Clone just the repository's .git folder (excluding files as they are already in
# `existing-dir`) into an empty temporary directory
git clone --no-checkout repo-to-clone existing-dir/existing-dir.tmp # might want --no-hardlinks for cloning local repo

# Move the .git folder to the directory with the files.
# This makes `existing-dir` a git repo.
mv existing-dir/existing-dir.tmp/.git existing-dir/

# Delete the temporary directory
rmdir existing-dir/existing-dir.tmp
cd existing-dir

# git thinks all files are deleted, this reverts the state of the repo to HEAD.
# WARNING: any local changes to the files will be lost.
git reset --hard HEAD

android:layout_weight 是什么意思？

Arya

2023-02-17 18:11

我不明白如何使用这个属性。谁能告诉我更多相关信息？



解答

使用layout_weight您可以指定多个视图之间的大小比例。例如，您有 aMapView和 atable应该向地图显示一些附加信息。地图应使用屏幕的 3/4，表格应使用屏幕的 1/4。然后将 的 设置layout_weight为map3，将layout_weight的 设置table为 1。

要让它工作，您还必须将高度或宽度（取决于您的方向）设置为 0px。


更多建议

简而言之， layout_weight 指定要分配给 View 的布局中有多少额外空间。

LinearLayout 支持为单个孩子分配权重。此属性为视图分配一个“重要性”值，并允许它扩展以填充父视图中的任何剩余空间。视图的默认权重为零。

计算分配孩子之间的任何剩余空间

一般来说，公式是：

分配给孩子的空间=（孩子的个人体重）/（线性布局中每个孩子的体重总和）

示例 1

如果有三个文本框，其中两个声明权重为 1，而第三个文本框没有权重 (0)，则剩余空间分配如下：

第一个文本框 = 1/(1+1+0)

第二个文本框 = 1/(1+1+0)

第三个文本框 = 0/(1+1+0)

示例 2

假设我们在水平行中有一个文本标签和两个文本编辑元素。标签没有layout_weight指定，因此它占用渲染所需的最小空间。如果layout_weight两个文本编辑元素中的每一个都设置为 1，则父布局中的剩余宽度将在它们之间平分（因为我们声称它们同样重要）。

计算：

第一个标签 = 0/(0+1+1)

第二个文本框 = 1/(0+1+1)

第三个文本框 = 1/(0+1+1)

相反，如果第一个文本框的 alayout_weight为 1，第二个文本框的 a 为layout_weight2，则剩余空间的三分之一将分配给第一个，三分之二分配给第二个（因为我们要求第二个一个更重要）。

计算：

第一个标签 = 0/(0+1+2)

第二个文本框 = 1/(0+1+2)

第三个文本框 = 2/(0+1+2)
