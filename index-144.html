C++中的extern "C"有什么作用？

Arya

2023-02-16 23:00

extern "C"放入C++ 代码到底有什么作用？

例如：

extern "C" {
   void foo();
}



解答

extern "C"使 C++ 中的函数名称具有 C 链接（编译器不会破坏名称），以便客户端 C 代码可以使用仅包含函数声明的 C 兼容头文件链接到（使用）您的函数。您的函数定义包含在二进制格式中（由您的 C++ 编译器编译），然后客户端 C 链接器将使用 C 名称链接到该格式。

由于 C++ 有函数名的重载而 C 没有，C++ 编译器不能只使用函数名作为链接的唯一 id，因此它通过添加有关参数的信息来破坏名称。AC 编译器不需要破坏名称，因为您不能在 C 中重载函数名称。当您在extern "C"C++ 中声明函数具有链接时，C++ 编译器不会将参数/参数类型信息添加到用于链接的名称中。

如您所知，您可以明确指定extern "C"与每个单独的声明/定义的链接，或者使用块将一系列声明/定义分组以具有特定链接：

extern "C" void foo(int);
extern "C"
{
   void g(char);
   int i;
}

如果您关心技术细节，它们在 C++03 标准的第 7.5 节中列出，这里是一个简短的摘要（强调extern "C"）：

  • extern "C"是链接规范
  • 每个编译器都 需要 提供“C”链接
  • 链接规范应仅出现在名称空间范围内
  • 所有的函数类型、函数名和变量名都有语言链接 见理查德的评论： 只有具有外部链接的函数名和变量名才有语言链接
  • 具有不同语言链接的两个函数类型是不同的类型，即使在其他方面相同
  • Linkage specs嵌套，inner one决定最终的linkage
  • extern "C"被类成员忽略
  • 至多一个具有特定名称的函数可以具有“C”链接（无论命名空间如何）
  • extern "C" 强制函数具有外部链接（不能使其成为静态的） 参见 Richard 的评论： inside is valid; 如此声明的实体具有内部链接，因此没有语言链接 static``extern "C"
  • 从 C++ 到用其他语言定义的对象的链接以及从其他语言到用 C++ 定义的对象的链接是实现定义的和语言相关的。只有两种语言实现的对象布局策略足够相似，才能实现这种联动

你会经常在 C 头文件中看到这样的代码：

#ifdef __cplusplus
extern "C" {

// all of your legacy C code here

如何停止跟踪并忽略对 Git 中文件的更改？

Arya

2023-02-16 22:58

我克隆了一个包含一些.csproj文件的项目。我不需要/喜欢我的本地csproj文件被 Git 跟踪（或在创建补丁时被调出），但很明显它们在项目中是需要的。

我已经添加*.csproj到我的 LOCAL .gitignore，但文件已经在回购中。

当我键入 git status 时，它会显示我csproj对跟踪或提交补丁不感兴趣的更改。

如何从我的个人存储库中删除这些文件的“跟踪”（但将它们保留在源代码中以便我可以使用它们），以便在执行状态（或创建补丁）时看不到更改？

有没有正确/规范的方法来处理这种情况？



解答

只需调用git rm --cached要从修订控制中删除的每个文件就可以了。只要您的本地忽略模式正确，您就不会在 git status 的输出中看到这些文件。

请注意，此解决方案会从存储库中删除文件，因此所有开发人员都需要维护自己的本地（不受修订控制的）文件副本

为了防止 git 检测这些文件中的更改，您还应该使用此命令：

git update-index --assume-unchanged [path]

你可能想做什么：（ 从下面@Ryan Taylor 回答）

  1. 这是告诉 git 你想要你自己的文件或文件夹的独立版本。例如，您不想覆盖（或删除）生产/暂存配置文件。

git update-index --skip-worktree <path-name>

完整答案在此 URL 中： http: //source.kohlerville.com/2009/02/untrack-files-in-git/

在 C++ 中将 int 转换为字符串的最简单方法

Arya

2023-02-16 22:58

在 C++ 中从 转换int为等效项的最简单方法是什么？string我知道两种方法。有更容易的方法吗？

(1)

int a = 10;
char *intStr = itoa(a);
string str = string(intStr);

(2)

int a = 10;
stringstream ss;
ss << a;
string str = ss.str();



解答

C++11 引入std::stoi（以及每种数字类型的变体）和std::to_string，Catoi和的对应物itoa，但用std::string.

#include <string>

std::string s = std::to_string(42);

因此是我能想到的最短路线。您甚至可以使用关键字省略命名类型auto：

auto s = std::to_string(42);

注意：参见 [string.conversions] （n3242中的 21.5 ）


更多建议

C++20： std::format现在是惯用的方式。

C++17：

几年后与@v.oddou 进行了讨论，C++17 提供了一种方法来执行最初基于宏的类型不可知解决方案（保留在下面），而无需通过 宏 丑陋。

// variadic template
template < typename... Args >
std::string sstr( Args &&... args )
{
    std::ostringstream sstr;
    // fold expression
    ( sstr << std::dec << ... << args );
    return sstr.str();
}

用法：

int i = 42;
std::string s = sstr( "i is: ", i );
puts( sstr( i ).c_str() );

Foo x( 42 );
throw std::runtime_error( sstr( "Foo is '", x, "', i is ", i ) );

C++98：

由于“converting ... to string”是一个反复出现的问题，我总是在我的 C++ 源代码的中央标头中定义SSTR()宏：

#include <sstream>

#define SSTR( x ) static_cast< std::ostringstream & >( \
        ( std::ostringstream() << std::dec << x ) ).str()

用法尽可能简单：

int i = 42;
std::string s = SSTR( "i is: " << i );
puts( SSTR( i ).c_str() );

Foo x( 42 );
throw std::runtime_error( SSTR( "Foo is '" << x << "', i is " << i ) );

以上是 C++98 兼容的（如果你不能使用 C++11 std::to_string），不需要任何第三方包含（如果你不能使用 Boost lexical_cast<>）；不过，这两种其他解决方案都具有更好的性能。
