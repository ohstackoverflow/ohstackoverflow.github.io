如何仅为调试构建运行 Visual Studio 构建后事件

Arya

2023-02-17 20:22

如何将构建后事件限制为仅针对一种类型的构建运行？

我正在使用事件将 DLL 文件复制到本地 IIS 虚拟目录，但我不希望在发布模式下的构建服务器上发生这种情况。



解答

构建前和构建后事件作为批处理脚本运行。您可以对 执行条件语句$(ConfigurationName)。

例如

if $(ConfigurationName) == Debug xcopy something somewhere


更多建议

仅供参考，您不需要使用 goto。shell IF命令可以与圆括号一起使用：

if $(ConfigurationName) == Debug (
  copy "$(TargetDir)myapp.dll" "c:\delivery\bin" /y
  copy "$(TargetDir)myapp.dll.config" "c:\delivery\bin" /y
) ELSE (
  echo "why, Microsoft, why".
)

pip 使用不正确的缓存包版本，而不是用户指定的版本

Arya

2023-02-17 20:21

我需要专门安装psycopg2 v2.4.1。我不小心做了：

 pip install psycopg2

代替：

 pip install psycopg2==2.4.1

这将安装 2.4.4 而不是早期版本。

现在，即使在我 pip uninstall psycopg2 并尝试使用正确的版本重新安装之后，pip 似乎正在重新使用它第一次下载的缓存。

如何强制 pip 清除其下载缓存并使用我在命令中包含的特定版本？



解答

如果使用 pip 6.0 或更新版本，请尝试添加--no-cache-dir选项 ( source )。

如果使用早于 pip 6.0 的 pip，请使用pip install -U pip.


更多建议

清除适合您系统的缓存目录

Linux 和 Unix

~/.cache/pip  # and it respects the XDG_CACHE_HOME directory.

操作系统

~/Library/Caches/pip

视窗

%LocalAppData%\pip\Cache

更新

使用 pip 20.1 或更高版本，您可以通过在命令行中键入以下内容轻松找到操作系统的完整路径：

pip cache dir

我的 Ubuntu 安装的示例输出：

➜ pip3 cache dir
/home/tawanda/.cache/pip

如何在 Ruby 中写入文件？

Arya

2023-02-17 20:21

我需要从数据库中读取数据，然后将其保存在文本文件中。

我怎样才能在 Ruby 中做到这一点？Ruby 有文件管理系统吗？



解答

The Ruby File class will give you the ins and outs of ::new and ::open but its parent, the IO class, gets into the depth of #read and #write.


更多建议

您可以使用简短版本：

File.write('/path/to/file', 'Some glorious content')

它返回写入的长度；有关详细信息和选项，请参阅::write 。

要附加到文件，如果它已经存在，请使用：

File.write('/path/to/file', 'Some glorious content', mode: 'a')
