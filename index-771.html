Git 中的“起源”是什么？

Arya

2023-02-17 15:43

当我跑步时：

git push origin branchname

究竟是什么origin，为什么我必须在分支名称之前输入它？



解答

origin是 您系统上 特定远程存储库的 别名。 它实际上不是该存储库的属性。 __

通过做

git push origin branchname

你是说推送到origin存储库。不需要命名远程存储库origin：事实上，同一个存储库可以为另一个开发人员使用不同的别名。

Remotes 只是存储库 URL 的 别名。 您可以通过使用查看属于每个远程的 URL

git remote -v

在push命令中，您可以使用 遥控器 ，也可以直接使用 URL 。使用 URL 的示例：

git push git@github.com:git/git.git master


更多建议

origin不是远程 存储 库名称。它是一个本地 别名 ，设置为代替远程存储库 URL 的键。

它避免了用户在提示推送时必须键入整个远程 URL。

此名称是默认设置的，并且是 Git 第一次从远程克隆时的约定。

此别名不是硬编码的，可以使用以下命令提示符进行更改：

git remote rename origin mynewalias

查看http://git-scm.com/docs/git-remote以获得进一步的说明。

git reset --hard HEAD 留下未跟踪的文件

Arya

2023-02-17 15:43

据我了解，当我运行时git reset --hard HEAD，它应该重置为您所拉内容的原始版本。不幸的是，它会留下文件，因为它git status显示了大量未跟踪文件。

您如何告诉 git “只需将其恢复到上次拉动时的状态，仅此而已”？



解答

您必须使用git clean -f -d来删除工作副本中未跟踪的文件和目录。您可以添加以删除被忽略的文件，在这个优秀的 SO answer-x中有更多信息。

如果您需要将带有子模块的整个存储库重置为 master 上的状态，请运行此脚本：

git fetch origin master
git checkout --force -B master origin/master
git reset --hard
git clean -fdx
git submodule update --init --recursive --force
git submodule foreach git fetch
git submodule foreach git checkout --force -B master origin/master
git submodule foreach git reset --hard
git submodule foreach git clean -fdx


更多建议

git reset --hard && git clean -df

或者，zsh 提供一个 'gpristine' 别名：

alias gpristine='git reset --hard && git clean -df'

这真的很方便。

选修的：

-x该命令还有一个选项git clean。这也将删除“git ignored”文件，因此如果需要，也可以添加此选项。

如果在分叉的回购上工作，请确保从正确的回购/分支中获取和重置，例如：

git fetch upstream && git reset --hard upstream/master && git clean -df

如何从标签创建新分支？

Arya

2023-02-17 15:42

我想从现有标签创建一个新的主分支。说我有一个标签v1.0。如何从此标签创建新分支？



解答

哇，这比我想象的要容易：

git checkout -b newbranch v1.0


更多建议

如果您只是想创建一个新分支而不立即更改它，您可以执行以下操作：

git branch newbranch v1.0
