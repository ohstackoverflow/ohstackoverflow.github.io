如何正确清理 Python 对象？

Arya

2023-02-18 02:36 

class Package:
    def __init__(self):
        self.files = []

    # ...

    def __del__(self):
        for file in self.files:
            os.unlink(file)

__del__(self)以上失败并出现 AttributeError 异常。我知道Python 不保证__del__()在调用时存在“全局变量”（在此上下文中的成员数据？） 。如果是这种情况并且这是异常的原因，我如何确保对象正确销毁？



解答

我建议使用 Python 的with语句来管理需要清理的资源。使用显式close()语句的问题是你不得不担心人们根本忘记调用它或者忘记将它放在一个finally块中以防止发生异常时资源泄漏。

要使用该with语句，请使用以下方法创建一个类：

def __enter__(self)
def __exit__(self, exc_type, exc_value, traceback)

在上面的示例中，您将使用

class Package:
    def __init__(self):
        self.files = []

    def __enter__(self):
        return self

    # ...

    def __exit__(self, exc_type, exc_value, traceback):
        for file in self.files:
            os.unlink(file)

然后，当有人想使用您的课程时，他们会执行以下操作：

with Package() as package_obj:
    # use package_obj

变量 package_obj 将是 Package 类型的一个实例（它是该方法返回的值__enter__）。__exit__无论是否发生异常，它的方法都会被自动调用。

您甚至可以更进一步采用这种方法。在上面的示例中，有人仍然可以在不使用子句的情况下使用其构造函数实例化 Package with。你不希望这种情况发生。您可以通过创建定义__enter__和__exit__方法的 PackageResource 类来解决此问题。然后，将在方法内部严格定义 Package 类__enter__并返回。这样，调用者永远无法在不使用with语句的情况下实例化 Package 类：

class PackageResource:
    def __enter__(self):
        class Package:
            ...
        self.package_obj = Package()
        return self.package_obj

    def __exit__(self, exc_type, exc_value, traceback):
        self.package_obj.cleanup()

您将按如下方式使用它：

with PackageResource() as package_obj:
    # use package_obj


更多建议

标准方法是使用atexit.register：

# package.py
import atexit
import os

class Package:
    def __init__(self):
        self.files = []
        atexit.register(self.cleanup)

    def cleanup(self):
        print("Running cleanup...")
        for file in self.files:
            print("Unlinking file: {}".format(file))
            # os.unlink(file)

但是你应该记住，这将保留所有创建的实例，Package直到 Python 终止。

Demo 使用上面的代码保存为 package.py ：

$ python
>>> from package import *
>>> p = Package()
>>> q = Package()
>>> q.files = ['a', 'b', 'c']
>>> quit()
Running cleanup...
Unlinking file: a
Unlinking file: b
Unlinking file: c
Running cleanup...

如何使用类型提示指定多个返回类型

Arya

2023-02-18 02:35

我在 python 中有一个函数可以返回 abool或 a list。有没有办法使用类型提示指定返回类型？

例如，这是正确的做法吗？

def foo(id) -> list or bool:
    ...



解答

从文档 -联合类型：

联合对象保存对多个类型对象|进行（按位或）运算的值。这些类型主要用于类型注释。与.typing.Union

这种用法|是在 Python 3.10 中添加的。因此，表示多个返回数据类型的正确方法是：

def foo(client_id: str) -> list | bool:

对于早期版本，请使用typing.Union：

from typing import Union


def foo(client_id: str) -> Union[list, bool]:

但请注意，不强制执行键入。Python 仍然是一种动态类型的语言。注释语法的开发是为了在代码发布到生产环境之前的开发过程中提供帮助。正如PEP 484所述，“在运行时不会进行类型检查。”

>>> def foo(a: str) -> list:
...     return "Works"
... 
>>> foo(1)
'Works'

如您所见，我正在传递一个 int 值并返回一个 str。但是，__annotations__将被设置为各自的值。

>>> foo.__annotations__ 
{'return': <class 'list'>, 'a': <class 'str'>}

请通过PEP 483了解更多关于类型提示的信息。另请参阅什么是 Python 3.5 中的类型提示？？

请注意，这仅适用于 _ Python 3.5_ 及更高版本。PEP 484中明确提到了这一点。


更多建议

如果有人在这里搜索“如何指定多个返回值的类型？”，请使用[type_value1, ..., type_valueN].

在 Python 3.9+ 中：

def f() -> tuple[dict, str]:
    a = {1: 2}
    b = "hello"
    return a, b

在早期版本中，使用typing.Tuple：

from typing import Tuple

def f() -> Tuple[dict, str]:
    ...

在 jQuery 中设置文本区域的值

Arya

2023-02-18 02:35

我正在尝试使用 jquery 和以下代码在 textarea 字段中设置一个值：

$("textarea#ExampleMessage").attr("value", result.exampleMessage);

问题是，一旦执行此代码，它不会更改文本区域中的文本吗？

但是，当执行alert($("textarea#ExampleMessage").attr("value"))新设置的值时返回？



解答

你试过瓦尔吗？

$("textarea#ExampleMessage").val(result.exampleMessage);


更多建议

Textarea 没有 value 属性，它的值来自标签之间，即：<textarea>my text</textarea>，它不像输入字段 ( <input value="my text" />)。这就是 attr 不起作用的原因:)
