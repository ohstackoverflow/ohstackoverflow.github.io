在 React / React Native 中使用构造函数与 getInitialState 有什么区别？

Arya

2023-02-17 17:25

我见过两者可以互换使用。

两者的主要用例是什么？有优点/缺点吗？一个是更好的做法吗？



解答

这两种方法不可互换。使用 ES6 类时应在构造函数中初始化状态，getInitialState使用React.createClass.

请参阅有关 ES6 类主题的官方 React 文档。

class MyComponent extends React.Component {
  constructor(props) {
    super(props);
    this.state = { /* initial state */ };
  }
}

相当于

var MyComponent = React.createClass({
  getInitialState() {
    return { /* initial state */ };
  },
});

可以使用哪些技术在 JavaScript 中定义类，它们的权衡是什么？

Arya

2023-02-17 17:24

我更喜欢在大型项目中使用 OOP，比如我现在正在从事的项目。我需要在 JavaScript 中创建几个类，但如果我没记错的话，至少有几种方法可以做到这一点。语法是什么，为什么要那样做？

我想避免使用第三方库——至少一开始是这样。
在寻找其他答案时，我找到了文章 使用 JavaScript 进行面向对象编程，第一部分：继承 - Doc JavaScript ，其中讨论了 JavaScript 中的面向对象编程。有没有更好的继承方式？



解答

这是不使用任何外部库的方法：

// Define a class like this
function Person(name, gender){

   // Add object properties like this
   this.name = name;
   this.gender = gender;
}

// Add methods like this.  All Person objects will be able to invoke this
Person.prototype.speak = function(){
    alert("Howdy, my name is" + this.name);
};

// Instantiate new objects with 'new'
var person = new Person("Bob", "M");

// Invoke methods like this
person.speak(); // alerts "Howdy, my name is Bob"

现在真正的答案比这复杂得多。例如，JavaScript 中没有类这样的东西。JavaScript 使用prototype基于 - 的继承方案。

此外，还有许多流行的 JavaScript 库，它们在 JavaScript 中有自己的近似类功能的风格。您至少要查看Prototype和jQuery。

决定其中哪一个是“最好的”是在 Stack Overflow 上开始一场圣战的好方法。如果您正在着手进行一个更大的 JavaScript 密集型项目，那么学习一个流行的库并按照他们的方式去做绝对值得。我是一个原型专家，但 Stack Overflow 似乎更倾向于 jQuery。

就只有“一种方法”而言，不依赖于外部库，我写的方式就差不多了。


更多建议

在 JavaScript 中定义类的最佳方式是不定义类。

严重地。

面向对象有几种不同的风格，其中一些是：

  • 基于类的 OO（首先由 Smalltalk 引入）
  • 基于原型的 OO（首先由 Self 引入）
  • 基于多方法的 OO（我认为首先由 CommonLoops 引入）
  • 基于谓词的 OO（不知道）

可能还有其他我不知道的。

JavaScript 实现了基于原型的 OO。在基于原型的 OO 中，新对象是通过复制其他对象（而不是从类模板实例化）创建的，方法直接存在于对象中而不是类中。继承是通过委托完成的：如果一个对象没有方法或属性，它会在它的原型（即它被克隆的对象）上查找，然后是原型的原型等等。

换句话说：没有类。

JavaScript 实际上对该模型进行了很好的调整：构造函数。您不仅可以通过复制现有对象来创建对象，可以这么说，您还可以“凭空”构造它们。如果你用这个关键字调用一个函数new，那个函数就变成了一个构造函数，这个this关键字将不会指向当前对象，而是指向一个新创建的“空”对象。因此，您可以按照自己喜欢的方式配置对象。通过这种方式，JavaScript 构造函数可以承担传统基于类的 OO 中类的角色之一：充当新对象的模板或蓝图。

现在，JavaScript 是一种非常强大的语言，因此如果您愿意，可以很容易地 在 JavaScript 中 实现基于类的 OO 系统。但是，只有在确实需要时才应该这样做，而不仅仅是因为 Java 就是这样做的。

如何从链接中删除蓝色下划线？



解答

您不是应用于text-decoration: none;锚点 ( .boxhead a)，而是应用于跨度元素 ( .boxhead)。

尝试这个：

.boxhead a {
    color: #FFFFFF;
    text-decoration: none;
}


更多建议

锚标签（链接）还有visited、hover、link、active等伪类。确保您的样式适用于有问题的州，并且没有其他样式冲突。

例如：

a:hover, a:visited, a:link, a:active
{
    text-decoration: none;
}

有关用户操作伪类 :hover、:active 和 :focus 的 更多信息，请参见W3.org 。 ****
