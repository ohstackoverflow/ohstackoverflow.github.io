Angular 2+ 中 ngShow 和 ngHide 的等价物是什么？

Arya

2023-02-17 16:25

我有许多元素希望在特定条件下可见。

在 AngularJS 中我会写

<div ng-show="myVar">stuff</div>

我如何在 Angular 2+ 中执行此操作？



解答

该hidden属性可用于此

[hidden]="!myVar"

也可以看看

  • https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/hidden

问题

hidden但是有一些问题，因为它可能与display属性的 CSS 冲突。

看看 Plunker 示例some是如何不被隐藏的，因为它有一个样式 ****

:host {display: block;}

放。（这在其他浏览器中可能表现不同——我用 Chrome 50 测试过）

解决方法

您可以通过添加来修复它

[hidden] { display: none !important;}

在index.html.

另一个陷阱

hidden="false"
hidden="{{false}}"
hidden="{{isHidden}}" // isHidden = false;

是一样的

hidden="true"

并且不会显示该元素。

hidden="false"``"false"将分配被认为是真实的字符串。
只有值false或删除属性才会真正使元素可见。

使用{{}}还会将表达式转换为字符串，并且不会按预期工作。

只有绑定 with[]才能按预期工作，因为 thisfalse被指定为 asfalse而不是"false".

*ngIf 对比[hidden]

*ngIf有效地从 DOM 中删除它的内容，同时[hidden]修改display属性，只指示浏览器不显示内容，但 DOM 仍然包含它。


更多建议

使用[hidden]属性：

[hidden]="!myVar"

或者你可以使用*ngIf

*ngIf="myVar"

这是显示/隐藏元素的两种方式。唯一的区别是：*ngIf将从 DOM 中删除元素，同时[hidden]通过将元素保留在 DOM 中来告诉浏览器使用 CSSdisplay属性显示/隐藏元素。

我什么时候应该为函数/方法写关键字“inline”？

Arya

2023-02-17 16:25

inline我什么时候应该在 C++ 中为函数/方法编写关键字？

看到一些答案后，一些相关的问题：

  • 我什么时候 不 应该在 C++ 中为函数/方法编写关键字“inline”？

  • 编译器何时会不知道何时使函数/方法“内联”？

  • 如果一个应用程序是多线程的 ，当一个人为一个函数/方法写“内联”时，这有关系吗？



解答

哦，伙计，我最讨厌的人之一。

inline更像是static或extern不仅仅是告诉编译器内联您的函数的指令。extern, static,inline是链接指令，几乎专门由链接器使用，而不是编译器。

据说inline向编译器提示您认为应该内联该函数。这在 1998 年可能是正确的，但十年后编译器不需要这样的提示。更不用说人类在优化代码时通常是错误的，因此大多数编译器会完全忽略“提示”。

  • static - 变量/函数名称不能在其他翻译单元中使用。链接器需要确保它不会意外使用来自另一个翻译单元的静态定义的变量/函数。

  • extern - 在此翻译单元中使用此变量/函数名称，但如果未定义则不要抱怨。链接器会将其整理出来并确保所有尝试使用某些外部符号的代码都有其地址。

  • inline - 这个函数会在多个翻译单元中定义，不用担心。链接器需要确保所有翻译单元都使用变量/函数的单个实例。

注意： 通常，声明模板inline是没有意义的，因为它们已经具有链接语义inline。但是，需要inline使用模板的显式特化和实例化。

具体回答你的问题：

  • 我什么时候应该在 C++ 中为函数/方法编写关键字“inline”？

仅当您希望在标头中定义函数时。更确切地说，只有当函数的定义可以出现在多个翻译单元中时。在头文件中定义小的（如在一个衬里中）函数是个好主意，因为它为编译器提供了更多信息，以便在优化代码时使用。它还会增加编译时间。

  • 我什么时候不应该在 C++ 中为函数/方法编写关键字“inline”？

不要仅仅因为您认为如果编译器内联您的代码会运行得更快就添加内联。

  • 编译器何时会不知道何时使函数/方法“内联”？

通常，编译器能够比您做得更好。但是，如果编译器没有函数定义，则它没有内联代码的选项。在最大限度优化的代码中，private无论您是否要求，通常所有方法都是内联的。

为了防止在 GCC 中内联，请使用__attribute__(( noinline ))，而在 Visual Studio 中，请使用__declspec(noinline).

  • 如果一个应用程序是多线程的，当一个人为一个函数/方法写“内联”时，这有关系吗？

多线程不会以任何方式影响内联。

如何在 C# 中克隆通用列表？

Arya

2023-02-17 16:24

我在 C# 中有一个通用对象列表，并希望克隆该列表。列表中的项目是可克隆的，但似乎没有可执行的选项list.Clone()。

有没有简单的方法解决这个问题？



解答

您可以使用扩展方法。

static class Extensions
{
    public static IList<T> Clone<T>(this IList<T> listToClone) where T: ICloneable
    {
        return listToClone.Select(item => (T)item.Clone()).ToList();
    }
}
