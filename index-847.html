从 Python 中的字符串中删除特定字符

Arya

2023-02-17 17:44

我正在尝试使用 Python 从字符串中删除特定字符。这是我现在正在使用的代码。不幸的是，它似乎对字符串没有任何作用。

for char in line:
    if char in " ?.!/;:":
        line.replace(char,'')

我该如何正确地做到这一点？



解答

Python 中的字符串是 不可变的 （无法更改）。正因为如此， 的作用line.replace(...)只是创建一个新的字符串，而不是改变旧的。您需要 重新绑定 （分配）它，line以便让该变量采用新值，并删除这些字符。

此外，你这样做的方式相对来说会有点慢。对于有经验的 pythonators 来说，这也可能有点令人困惑，他们会看到一个双重嵌套的结构，并想一想更复杂的事情正在发生。

从 Python 2.6 和更新的 Python 2.x 版本 * 开始，您可以改为使用str.translate,（ 请参阅 下面的 Python 3 答案 ）：

line = line.translate(None, '!@#$')

或正则表达式替换为re.sub

import re
line = re.sub('[!@#$]', '', line)

括号中的字符构成一个 字符类 。line该类中的任何字符都被第二个参数替换为sub：一个空字符串。

蟒蛇 3 答案

在 Python 3 中，字符串是 Unicode。你必须翻译得有点不同。kevpie 在对其中一个答案的评论中提到了这一点，并且在str.translate.

在调用translateUnicode字符串的方法时，不能传递我们上面使用的第二个参数。您也不能None作为第一个参数传递。相反，您将翻译表（通常是字典）作为唯一参数传递。该表将字符的 序数值 （即调用它们的结果ord）映射到应该替换它们的字符的序数值，或者——对我们有用——None指示它们应该被删除。

因此，要使用 Unicode 字符串进行上述舞蹈操作，您可以调用类似

translation_table = dict.fromkeys(map(ord, '!@#$'), None)
unicode_line = unicode_line.translate(translation_table)

这里dict.fromkeys和map用于简洁地生成包含的字典

{ord('!'): None, ord('@'): None, ...}

更简单的是，正如另一个答案所说，就地创建翻译表：

unicode_line = unicode_line.translate({ord(c): None for c in '!@#$'})

或者，正如Joseph Lee提出的那样，创建相同的翻译表str.maketrans：

unicode_line = unicode_line.translate(str.maketrans('', '', '!@#$'))

  • 为了与早期的 Python 兼容，您可以创建一个“空”转换表来代替传递None：

    import string line = line.translate(string.maketrans('', ''), '!@#$')

这里string.maketrans用于创建一个 转换表 ，它只是一个包含序数值为 0 到 255 的字符的字符串。


更多建议

我在这里错过了重点，还是只是以下内容：

string = "ab1cd1ef"
string = string.replace("1", "")

print(string)
# result: "abcdef"

把它放在一个循环中：

a = "a!b@c#d$"
b = "!@#$"
for char in b:
    a = a.replace(char, "")

print(a)
# result: "abcd"

命令行上的 Node.js 版本？（不是 REPL）

Arya

2023-02-17 17:44

我想在命令行上获取 Node.js 的版本。我期待运行如下命令：

node -version

但这是行不通的。有人知道命令行是什么吗？（即不是 REPL）



解答

命令行是：

node -v

或者

node --version

笔记：

如果node -v不起作用，但nodejs -v确实起作用，则表明您的系统上的某些设置不正确。有关修复它的方法，请参阅其他问题。


更多建议

如果您指的是 shell 命令行，则可以使用以下任一方法：

node -v

node --version

只需键入node version就会导致 node.js 尝试加载名为 version 的模块，除非您喜欢使用混淆的模块名称，否则该模块不存在。

如何确定类型是否实现了 C# 反射的接口

Arya

2023-02-17 17:44

反射 是否提供C#了一种方法来确定某些给定System.Type类型是否为某些接口建模？

public interface IMyInterface {}

public class MyType : IMyInterface {}

// should yield 'true'
typeof(MyType)./* ????? */MODELS_INTERFACE(IMyInterface);



解答

你有几个选择：

  1. typeof(IMyInterface).IsAssignableFrom(typeof(MyType))
  2. typeof(MyType).GetInterfaces().Contains(typeof(IMyInterface))
  3. 使用 C# 6，您可以使用typeof(MyType).GetInterface(nameof(IMyInterface)) != null

对于通用接口，它有点不同。

typeof(MyType).GetInterfaces().Any(i => i.IsGenericType && i.GetGenericTypeDefinition() == typeof(IMyInterface<>))


更多建议

使用Type.IsAssignableFrom：

typeof(IMyInterface).IsAssignableFrom(typeof(MyType));
