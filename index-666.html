如何防止为 Node.js (package.json) 安装“devDependencies”NPM 模块？

Arya

2023-02-17 12:55

我的 package.json 文件中有这个（简化版）：

{
  "name": "a-module",
  "version": "0.0.1",
  "dependencies": {
    "coffee-script":      ">= 1.1.3"
  },
  "devDependencies": {
    "stylus":             ">= 0.17.0"
  }
}

我在 Mac 10.6.8 上使用 NPM 版本 1.1.1。

当我从项目根目录运行以下命令时，它会同时安装dependencies 和 devDependencies：

npm install

我的印象是这个命令安装了devDependencies：

npm install --dev

我如何使它npm install只安装dependencies（因此生产环境只获得那些模块），而类似的东西npm install --dev同时安装dependencies和devDependencies？



解答

在开发环境（默认）中的包目录中运行时，该命令将与其他npm install命令一起安装。devDependencies``dependencies

在 8.x 及以上版本中，使用--omit=dev标志仅安装常规依赖项：

npm install --omit=dev

无论环境变量的值如何，这将 只 dependencies安装，而不安装。devDependencies``NODE_ENV

如果您使用 6.x 或更早版本，则需要改用该--only=prod标志。

注意：
在npm v3.3.0之前（2015-08-13），该选项被调用--production，即

npm install --production

您可能还需要--no-optional标志。


更多建议

我也遇到了这个问题！npm install 有点令人困惑，网络帖子不断引入 -d/--dev 标志，就好像存在明确的“开发”安装模式一样。

  • npm install 将同时安装“ dependencies ”和“ devDependencies ”

  • npm install --production 只会安装“ 依赖项 ”

  • npm install --dev 只会安装“ devDependencies ”

Java中如何将数组转换为集合

Arya

2023-02-17 12:54

我想用 Java 将数组转换为 Set。有一些明显的方法可以做到这一点（即使用循环），但我想要一些更简洁的方法，例如：

java.util.Arrays.asList(Object[] a);

有任何想法吗？



解答

像这样：

Set<T> mySet = new HashSet<>(Arrays.asList(someArray));

在 Java 9+ 中，如果 unmodifiable set 没问题：

Set<T> mySet = Set.of(someArray);

在 Java 10+ 中，泛型类型参数可以从数组组件类型中推断出来：

var mySet = Set.of(someArray);

当心

Set.of 抛出 IllegalArgumentException - 如果 someArray 中有任何重复元素。查看更多详细信息：https://docs.oracle.com/en/java/javase/11/docs/api/java.base/java/util/Set.html#of(E...)


更多建议

Set<T> mySet = new HashSet<T>();
Collections.addAll(mySet, myArray);

这是JDK 6 中的Collections.addAll(java.util.Collection, T...) 。

另外： 如果我们的数组充满了原语怎么办？

对于 JDK < 8，我只会编写一个明显的for循环来一次完成包装和添加到集合。

对于 JDK >= 8，一个有吸引力的选择是这样的：

Arrays.stream(intArray).boxed().collect(Collectors.toSet());

Git什么时候刷新远程分支列表？

Arya

2023-02-17 12:54

使用git branch --all显示所有 远程 和 本地 分支。Git 什么时候刷新这个列表？

在拉/推？以及如何使用Git Bash刷新它？



解答

更新远程分支的本地列表：

git remote update origin --prune

显示（本地）Git 知道的所有本地和远程分支：

git branch -a


更多建议

OP 没有要求清理所有遥控器，而是要求清理默认遥控器的所有分支。

git fetch --prune应该使用什么。

设置git config remote.origin.prune true为--prune自动。在这种情况下，justgit fetch也会从本地副本中删除陈旧的远程分支。另请参阅 使用 Git fetch 或 pull 自动修剪 。

请注意，这不会清除不再跟踪远程分支的本地分支。请参阅 如何修剪远程不再存在的本地跟踪分支 。
