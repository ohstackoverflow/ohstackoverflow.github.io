如何有条件地向 React 组件添加属性？

Arya

2023-02-17 08:51

有没有办法只在满足特定条件时才向 React 组件添加属性？

我应该在渲染后基于 Ajax 调用添加 required 和 readOnly 属性以形成元素，但我看不出如何解决这个问题，因为这与readOnly="false"完全省略该属性不同。

下面的示例应该可以解释我想要什么，但它不起作用。

（解析错误：意外的标识符）

function MyInput({isRequired}) {
  return <input classname="foo" {isRequired ? "required" : ""} />
}



解答

显然，对于某些属性，如果传递给它的值不真实，React 足够智能，可以忽略该属性。例如：

const InputComponent = function() {
    const required = true;
    const disabled = false;

    return (
        <input type="text" disabled={disabled} required={required} />
    );
}

将导致：

<input type="text" required>

更新： 如果有人对这种情况如何/为何发生感到好奇，您可以在 ReactDOM 的源代码中找到详细信息，特别是在DOMProperty.js文件的第30和167行。


更多建议

juandemarco 的回答通常是正确的，但这里有另一种选择。

按照你喜欢的方式构建一个对象：

var inputProps = {
  value: 'foo',
  onChange: this.handleChange
};

if (condition) {
  inputProps.disabled = true;
}

使用传播渲染，也可以选择传递其他道具。

<input
    value="this is overridden by inputProps"
    {...inputProps}
    onChange={overridesInputProps}
 />

有 Pytz 时区列表吗？

Arya

2023-02-17 08:51

我想知道 Python 库 pytz 中时区参数的所有可能值是什么。怎么做？



解答

您可以列出所有可用的时区pytz.all_timezones：

In [40]: import pytz
In [41]: pytz.all_timezones
Out[42]: 
['Africa/Abidjan',
 'Africa/Accra',
 'Africa/Addis_Ababa',
 ...]

还有pytz.common_timezones：

In [45]: len(pytz.common_timezones)
Out[45]: 403

In [46]: len(pytz.all_timezones)
Out[46]: 563


更多建议

不要创建自己的列表 -pytz有一个内置的集合：

import pytz
set(pytz.all_timezones_set)  
>>> {'Europe/Vienna', 'America/New_York', 'America/Argentina/Salta',..}

然后您可以 应用时区 ：

import datetime
tz = pytz.timezone('Pacific/Johnston')
ct = datetime.datetime.now(tz=tz)
>>> ct.isoformat()
2017-01-13T11:29:22.601991-05:00

或者， 如果您已经有一个datetime可 识别 TZ 的 对象 （不是天真）：

# This timestamp is in UTC
my_ct = datetime.datetime.now(tz=pytz.UTC)

# Now convert it to another timezone
new_ct = my_ct.astimezone(tz)
>>> new_ct.isoformat()
2017-01-13T11:29:22.601991-05:00

如何从 Windows 命令行获取应用程序退出代码？

Arya

2023-02-17 08:50

我正在运行一个程序，想看看它的返回码是什么（因为它根据不同的错误返回不同的代码）。

我知道在 Bash 中我可以通过运行来做到这一点

回声$？

在 Windows 上使用 cmd.exe 时我该怎么办？



解答

“退出代码”存储在名为 的 shell 变量中errorlevel。

设置errorlevel在控制台应用程序的末尾。Windows 应用程序的行为略有不同；请参阅下面的@gary的回答。

使用if命令关键字errorlevel进行比较：

if errorlevel <n> (<statements>)

当错误级别大于或等于 n 时，它将执行 语句 。执行细节。 __if /?

名为 的 shell 变量errorlevel包含作为字符串的值，并且可以通过用 % 换行来取消引用。

示例脚本：

my_nifty_exe.exe

rem Give resolution instructions for known exit codes.
rem Ignore exit code 1.
rem Otherwise give a generic error message.

if %errorlevel%==7 (
   echo "Replace magnetic tape."
) else if %errorlevel%==3 (
   echo "Extinguish the printer."
) else if errorlevel 2 (
   echo Unknown Error: %errorlevel% refer to Run Book documentation.
) else (
   echo "Success!"
)

警告：名为 errorlevel 的环境变量（如果存在）将覆盖名为 errorlevel 的 shell 变量。if errorlevel测试不受影响。


更多建议

测试ErrorLevel适用于 控制台应用程序，但正如 dmihailescu 所暗示的，如果您尝试从命令提示符运行 窗口 应用程序（例如基于 Win32 的应用程序） ，这将不起作用。窗口应用程序将在后台运行，控制权将立即返回到命令提示符（很可能带有 0ErrorLevel表示进程已成功 创建 ）。当窗口应用程序最终退出时，其退出状态将丢失。

不过，与其使用其他地方提到的基于控制台的 C++ 启动器，更简单的替代方法是使用命令提示符的命令启动窗口应用程序START /WAIT。这将启动带窗口的应用程序，等待它退出，然后将控制权返回到命令提示符，并在 中设置进程的退出状态ErrorLevel。

start /wait something.exe
echo %errorlevel%
