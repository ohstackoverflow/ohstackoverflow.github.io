基于另一个单元格值的条件格式

Arya

2023-02-18 06:08

我正在使用 Google 表格作为日常仪表板。我需要的是根据另一个单元格 - C5 的值更改单元格 B5 的背景颜色。如果 C5 大于 80%，则背景颜色为绿色，但如果低于 80%，则为琥珀色/红色。

这是否适用于 Google 表格功能，或者我是否需要插入脚本？



解答

注意： 下面解释中说“B5”的时候，其实就是“B{current_row}”的意思，所以C5是B5，C6是B6，以此类推。除非您指定 $B$5 - 那么您指的是一个特定的单元格。

截至 2015 年，Google 表格支持此功能：https ://support.google.com/drive/answer/78413#formulas

在您的情况下，您需要在 B5 上设置条件格式。

  • 使用“ 自定义公式是 ”选项并将其设置为=B5>0.8*C5。
  • 将“ 范围 ”选项设置为B5。
  • 设置所需的颜色

您可以重复此过程为背景或文本或色阶添加更多颜色。

更好的是，通过使用“ Range ”中的范围，使单个规则适用于所有行。假设第一行是标题的示例：

  • 在 B2 条件格式中，将“ 自定义公式为 ”设置为=B2>0.8*C2。
  • 将“ 范围 ”选项设置为B2:B。
  • 设置所需的颜色

将与前面的示例类似，但适用于所有行，而不仅仅是第 5 行。

范围也可用于“自定义公式是”，因此您可以根据列值为整行着色。


更多建议

再举一个例子：

如果你有从 A 到 D 的列，并且需要突出显示整行（例如从 A 到 D），如果 B 是“完整的”，那么你可以按照以下方式进行：

"Custom formula is":  =$B:$B="Completed" 
Background Color:     red 
Range:                A:D

当然，如果你有更多的列，你可以将 Range 改为 A:T。

如果 B 包含“Complete”，则使用以下搜索：

"Custom formula is":  =search("Completed",$B:$B) 
Background Color:     red 
Range:                A:D

Swift 语言中的感叹号是什么意思？

Arya

2023-02-18 06:07

Swift 编程语言指南有以下示例：

class Person {
    let name: String
    init(name: String) { self.name = name }
    var apartment: Apartment?
    deinit { println("\(name) is being deinitialized") }
}

class Apartment {
    let number: Int
    init(number: Int) { self.number = number }
    var tenant: Person?
    deinit { println("Apartment #\(number) is being deinitialized") }
}

var john: Person?
var number73: Apartment?

john = Person(name: "John Appleseed")
number73 = Apartment(number: 73)

//From Apple's “The Swift Programming Language” guide (https://developer.apple.com/library/content/documentation/Swift/Conceptual/Swift_Programming_Language/AutomaticReferenceCounting.html)

然后在将公寓分配给此人时，他们使用感叹号来“打开实例”：

john!.apartment = number73

“解包实例”是什么意思？为什么有必要？它与仅执行以下操作有何不同：

john.apartment = number73

我对 Swift 语言很陌生。只是想把基础知识弄下来。

更新：
我遗漏的一大难题（没有直接在答案中说明 - 至少在撰写本文时没有说明）是当你执行以下操作时：

var john: Person?

正如我最初认为的那样，这并不意味着“john是类型Person并且它可能是零”。我只是误解了Person并且Person?是完全不同的类型。一旦我明白了这一点，所有其他的?，!疯狂的，以及下面的伟大答案，就变得更有意义了。



解答

“解包实例”是什么意思？为什么有必要？

据我所知（这对我来说也是很新的）......

术语“包装”意味着我们应该 将 Optional 变量视为礼物，包裹在闪亮的纸中，它可能（遗憾的是！）是空的 。

当“包装”时，可选变量的值是一个具有两个可能值的枚举（有点像布尔值）。此枚举描述变量是否包含值 ( Some(T)) 或不包含 ( None)。

如果有值，可以通过“展开”变量（获取 from T）来获取Some(T)。

与有何john!.apartment = number73不同john.apartment = number73？（释义）

如果你写一个可选变量的名称（例如 text john，没有!），这是指“包装”的枚举（Some/None），而不是值本身（T）。Sojohn不是 的实例Person，并且它没有成员apartment：

john.apartment
// 'Person?' does not have a member named 'apartment'

实际Person值可以通过多种方式展开：

  • “强制展开”：（如果存在则john!给出Person值，如果为零则运行时错误）
  • “可选绑定”：（如果值存在则if let p = john { println(p) }执行）println
  • “可选链接”：（john?.learnAboutSwift()如果该值存在则执行此虚构的方法）

我猜你会选择其中一种方式来解包，这取决于 nil 情况下应该发生什么，以及发生这种情况的可能性有多大。这种语言设计强制明确处理 nil 情况，我认为这提高了 Obj-C 的安全性（在 Obj-C 中很容易忘记处理 nil 情况）。

更新 ：

感叹号也用在声明“隐式展开可选”的语法中。

到目前为止的示例中，john变量已声明为var john:Person?，并且它是可选的。如果您想要该变量的实际值，则必须使用上述三种方法之一对其进行解包。

如果它被声明为var john:Person!相反，该变量将是一个隐式解包可选（请参阅 Apple 书中带有此标题的部分）。访问值时不需要解包这种变量，john无需额外语法即可使用。但是苹果的书说：

当变量有可能在稍后变为 nil 时，不应使用隐式展开的可选值。如果您需要在变量的生命周期内检查 nil 值，请始终使用普通的可选类型。

更新 2 ：

Mike Ash 的文章“ Interesting Swift Features ”给出了可选类型的一些动机。我认为这很棒，文字清晰。

更新 3 ：

另一篇关于感叹号的 隐式解包可选用法的 有用文章： Chris Adamson 的“ Swift and the Last Mile ”。文章解释说，这是 Apple 用于声明其 Objective-C 框架使用的可能包含 nil 的类型的实用措施。将类型声明为可选（使用?）或隐式展开（使用!）是“安全性和便利性之间的权衡”。在本文给出的示例中，Apple 选择将类型声明为隐式展开，使调用代码更方便，但安全性较低。

也许 Apple 可能会在未来梳理他们的框架，消除隐式解包（“可能永远不会为零”）参数的不确定性，并将它们替换为可选的（“在特定情况下肯定可以为零 [希望，记录在案！] 情况”）或标准非- 可选（“永远不会为零”）声明，基于其 Objective-C 代码的确切行为。


更多建议

这是我认为的区别：

var john: Person?

意味着约翰可以为零

john?.apartment = number73

编译器会将这一行解释为：

if john != nil {
    john.apartment = number73
}

尽管

john!.apartment = number73

编译器会将这一行简单地解释为：

john.apartment = number73

因此，using!将展开 if 语句，并使其运行得更快，但如果 john 为 nil，则会发生运行时错误。

所以这里的wrap不是内存包裹，而是代码包裹，这里是用if语句包裹，因为苹果很注重运行时的性能，所以他们想给你一个办法使您的应用程序以最佳性能运行。

更新：

4 年后回到这个答案，因为我在 Stackoverflow 中获得了最高的声誉 :) 我当时有点误解了展开的含义。现在 4 年后，我相信这里解包的意思是将代码从其原始的紧凑形式展开。它还意味着消除该对象周围的模糊性，因为根据定义我们不确定它是否为 nil。就像上面 Ashley 的回答一样，把它想象成一个不能包含任何东西的礼物。但我仍然认为解包是代码解包，而不是像使用枚举那样基于内存的解包。
