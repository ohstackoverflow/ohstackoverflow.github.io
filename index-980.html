查明单选按钮是否用 JQuery 检查？

Arya

2023-02-17 21:44

我可以设置一个单选按钮来检查好，但我想做的是设置一种“监听器”，当某个单选按钮被选中时激活。

以下面的代码为例：

$("#element").click(function()
{ 
    $('#radio_button').attr("checked", "checked");
});

它添加了一个 checked 属性，一切都很好，但我将如何添加警报。例如，在没有点击功能的帮助下选中单选按钮时弹出？



解答

$('#element').click(function() {
   if($('#radio_button').is(':checked')) { alert("it's checked"); }
});


更多建议

如果您有一组单选按钮共享相同的名称属性，并且在提交或某些事件时您想检查这些单选按钮中的一个是否被选中，您可以通过以下代码简单地完成此操作：

$(document).ready(function(){
  $('#submit_button').click(function() {
    if (!$("input[name='name']:checked").val()) {
       alert('Nothing is checked!');
        return false;
    }
    else {
      alert('One of the radio buttons is checked!');
    }
  });
});

Javascript / Chrome - 如何从 webkit 检查器复制一个对象作为代码

Arya

2023-02-17 21:43

我正在我的 javascript 中执行 console.log 语句以记录 javascript 对象。我想知道是否有一种方法，一旦完成 - 将该对象复制为 javascript 代码。我想要做的是将使用 ajax 创建的对象转换为静态 javascript 对象，以便将 xml 提要解析为静态 javascript 对象，以便文件可以在本地运行，而无需服务器。我在 chrome 检查器窗口中包含了该对象的屏幕截图，因此您可以看到我正在尝试做什么。



解答

  1. 右键单击 Chrome 控制台中的对象，然后Store as Global Variable从上下文菜单中进行选择。它将返回类似于temp1变量名的内容。

  2. Chrome 也有一个copy()方法，所以copy(temp1)在控制台中应该将该对象复制到剪贴板。

关于递归对象的注意事项： 如果您尝试复制递归对象，您将得到[object Object]. 出路是尝试copy(JSON.stringify(temp1))，该对象将作为有效的 JSON 完全复制到您的剪贴板，因此您可以使用众多资源之一按您的意愿格式化它。

如果您收到Uncaught TypeError: Converting circular structure to JSON消息，您可以使用JSON.stringify的第二个参数（这是一个过滤函数）来过滤掉有问题的循环属性。有关更多详细信息，请参阅此Stack Overflow 答案。


更多建议

在 Chrome 89 或更高版本中，您只需右键单击控制台中的一个对象并选择Copy Object( ref )。这也适用于 Chrome 开发人员工具内的其他一些地方，例如在调试时或在网络请求的响应选项卡内。

copy另一种选择是按原样使用命令：

var x = { a: 1, b: 2 };
copy(x);

Xcode 6 错误：Interface Builder 文件中的未知类

Arya

2023-02-17 21:41

我升级到 Xcode 6 beta 4，现在我的应用程序不断崩溃并显示消息

Interface Builder 文件中的未知类 X。

它崩溃是因为据说 Xcode 找不到我在 Storyboard 中链接的自定义类，但它显示它们在 Xcode 界面中正确链接。

我肯定一切都正确链接。我唯一的其他选择可能是删除整个故事板文件并从头开始，因为它可能会以某种方式损坏。

编辑：我还想补充一点，我尝试过清理、重置模拟器、扰乱构建阶段等。这些都不起作用。



解答

我在输入问题时解决了这个问题。我想我会回答我的问题并将它留在这里，以供在使用 Xcode 6 beta 4 时可能遇到此问题的任何其他人使用。

要解决此问题，您需要在 Storyboard 中选择每个自定义类对象（这包括任何自定义视图，甚至是自定义视图控制器本身）。

然后选择这些对象，打开身份检查器，在“自定义类”下你应该看到模块选项。最后：

  • 在 Module 文本框内单击，然后按 Enter 键。
  • 或者 （更新 2022），勾选“Inherit Module From Target”选项。

就是这样！我所有自定义对象的当前模块一定在 Xcode 6 beta 4 中以某种方式在内部错误设置。但是在检查器中没有视觉指示。

请注意，如果在 Module 文本框内按 enter 不起作用，请尝试选择右侧的箭头并手动选择当前模块，然后清除文本框并按 enter。您也可以尝试在类文本框内按回车键（尽管这通常是为了解决不同的问题）。

更多建议

有时 Xcode 会遗漏 customModule="AppName" customModuleProvider="target"

要修复它，请打开故事板作为源代码并替换此行：

<viewController storyboardIdentifier="StoryboardId" id="SomeID" customClass="CustomClass"
sceneMemberID="viewController">

对此：

<viewController storyboardIdentifier="StoryboardId" id="SomeID" customClass="CustomClass"
 customModule="AppName" customModuleProvider="target" sceneMemberID="viewController">
