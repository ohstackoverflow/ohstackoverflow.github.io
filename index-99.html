如何删除所有已合并的 Git 分支？

Arya

2023-02-16 21:20

如何删除已经合并的分支？我可以一次删除它们，而不是一个一个地删除每个分支吗？



解答

注意：如果您的工作流将这些分支作为可能的祖先，您可以添加其他分支以排除，例如 master 和 dev。通常我从“sprint- start”标签分支出来，master、dev 和 qa 不是祖先。

首先，列出在远程合并的本地跟踪分支（考虑使用-r标志列出所有远程跟踪分支）。

git branch --merged

您可能会看到几个您不想删除的分支。我们可以添加一些参数来跳过我们不想删除的重要分支，例如 master 或 develop。以下命令将跳过 master 分支和其中包含 dev 的任何内容。

git branch --merged| egrep -v "(^\*|master|main|dev)"

如果你想跳过，你可以像下面这样将它添加到egrep命令中。分支skip_branch_name不会被删除。

git branch --merged| egrep -v "(^\*|master|main|dev|skip_branch_name)"

删除所有已经合并到当前检出分支的本地分支：

git branch --merged | egrep -v "(^\*|master|main|dev)" | xargs git branch -d

您可以看到 master 和 dev 被排除在外，以防它们是祖先。

您可以删除合并的本地分支：

git branch -d branchname

如果未合并，请使用：

git branch -D branchname

要从远程使用中删除它：

git push --delete origin branchname

git push origin :branchname    # for really old git

从远程删除分支后，您可以通过以下方式修剪以摆脱远程跟踪分支：

git remote prune origin

或修剪单个远程跟踪分支，正如其他答案所建议的那样：

git branch -dr branchname


更多建议

要删除远程上已合并的所有分支：

git branch -r --merged | grep -v master | sed 's/origin\//:/' | xargs -n 1 git push origin

在较新版本的 Git 中

git branch -r --merged | grep -v master | sed 's/origin\///' | xargs -n 1 git push --delete origin

更新（@oliver；因为不适合评论，但答案已经足够了） ：如果你在分支 ABC 上，那么 ABC 将出现在结果中，git branch -r --merged因为没有指定分支，所以分支默认为当前分支，和一个分支总是符合合并到自身的条件（因为分支和自身之间没有区别！）。

所以要么指定分支：

git branch -r --merged master | grep -v master ...

或第一个结账大师：

git checkout master | git branch -r --merged | grep -v ...

如何将 Git 管理项目中的所有本地更改恢复到以前的状态？

Arya

2023-02-16 21:19

我跑了git status，告诉我一切都是最新的，没有本地变化。

然后我做了几个连续的改变，意识到我想扔掉一切，回到我原来的状态。这个命令会为我做吗？

git reset --hard HEAD



解答

要恢复对工作副本所做的更改，请执行以下操作：

git checkout .

或者等效地，对于 git 版本 >= 2.23：

git restore .

要恢复对索引所做的更改（即，您已添加的更改），请执行此操作。 警告这会将你所有未推送的提交重置为 master！ :

git reset

要还原您已提交的更改：

git revert <commit 1> <commit 2>

要删除未跟踪的文件（例如，新文件、生成的文件）：

git clean -f

或未跟踪的目录（例如，新的或自动生成的目录）：

git clean -fd


更多建议

注意：您可能还想运行

git clean -fd

作为

git reset --hard

不会删除 未 跟踪的文件，因为 git-clean 将从跟踪的根目录中删除任何不在 git 跟踪下的文件。警告 - 小心这个！首先使用 git- clean 运行试运行，看看它将删除什么。

当您收到错误消息时，这也特别有用

~"performing this command will cause an un-tracked file to be overwritten"

当你做几件事时可能会发生这种情况，当你和你的朋友都添加了一个同名的新文件时更新工作副本，但他首先将其提交到源代码管理中，而你不关心删除未跟踪的副本.

在这种情况下，试运行也有助于显示将被覆盖的文件列表。

使用 jQuery 从下拉列表（选择框）中获取选定的文本

Arya

2023-02-16 21:19

如何从 jQuery 的下拉列表 中获取选定的文本（而不是选定的值）？



解答

$("#yourdropdownid option:selected").text();


更多建议

尝试这个：

$("#myselect :selected").text();

对于 ASP.NET 下拉列表，您可以使用以下选择器：

$("[id*='MyDropDownId'] :selected")
