如何检测 JavaScript 是否被禁用？

Arya

2023-02-17 17:36

今天早上有一个帖子询问有多少人禁用了 JavaScript。然后我开始想知道可以使用什么技术来确定用户是否禁用了它。

有谁知道一些简短/简单的方法来检测 JavaScript 是否被禁用？我的目的是发出警告，如果浏览器没有启用 JS，该站点将无法正常运行。

最终我想将它们重定向到能够在没有 JS 的情况下工作的内容，但我需要这个检测作为占位符来开始。



解答

我假设您正在尝试决定是否提供 JavaScript 增强的内容。最好的实现会干净利落地降级，这样网站仍然可以在没有 JavaScript 的情况下运行。我还假设您的意思是服务器端检测，而不是<noscript>出于无法解释的原因使用该元素。

没有很好的方法来执行服务器端 JavaScript 检测。作为替代方案，可以使用 JavaScript 设置 cookie，然后在后续页面视图中使用服务器端脚本测试该 cookie。然而，这不适合决定传送什么内容，因为它无法区分没有 cookie 的访客与新访客或不接受 JavaScript set cookie 的访客。

带参数的 RedirectToAction

Arya

2023-02-17 17:36

因此，我有一个从锚点调用的动作，Site/Controller/Action/ID哪里ID是int.

稍后我需要从 Controller 重定向到同一个 Action。

有没有聪明的方法来做到这一点？目前我正在存储ID临时数据，但是当你返回后再次按 f5 刷新页面时，临时数据消失了，页面崩溃了。



解答

您可以将 id 作为 RedirectToAction() 方法的 routeValues 参数的一部分传递。

return RedirectToAction("Action", new { id = 99 });

这将导致重定向到 Site/Controller/Action/99。不需要临时或任何类型的视图数据。


更多建议

根据我的研究， Kurt 的回答应该是正确的，但是当我尝试它时，我必须这样做才能让它真正为我工作：

return RedirectToAction( "Main", new RouteValueDictionary( 
    new { controller = controllerName, action = "Main", Id = Id } ) );

如果我没有指定控制器和其中的操作，RouteValueDictionary它就不起作用。

同样，当这样编码时，第一个参数 (Action) 似乎被忽略了。所以如果只是在Dict中指定controller，期望第一个参数指定Action，也是不行的。

如果您稍后再来，请先尝试 Kurt 的回答，如果您仍然有问题，请尝试这个。

如何更改 Visual Studio Code 中的缩进？

Arya

2023-02-17 17:35

对于每个 typescript 文件，visual studio code 使用 8 个空格的自动缩进。这对我的口味来说有点太多了，但我找不到在哪里改变它。

也许它可以作为设置使用，但名称不同，因为我找不到与缩进相关的任何内容。

更新

我目前正在使用Prettier 代码格式化程序，它通过在保存时自动格式化来解决所有格式化问题（如果没有语法错误）



解答

您可以在全局User级别或Workspace级别更改此设置。

打开设置：点击左下角的齿轮，然后点击Settings如下图。

然后，进行以下 2 项更改：（tabSize在搜索栏中键入）

  1. 取消选中复选框Detect Indentation
  2. 将选项卡大小更改为 2/4（尽管我强烈认为 2 对于 JS 是正确的。哈哈 :)）
