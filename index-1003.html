System.Timers.Timer 与 System.Threading.Timer

Arya

2023-02-17 22:26

我最近一直在检查一些可能的计时器，并且System.Threading.Timer是System.Timers.Timer我认为需要的计时器（因为它们支持线程池）。

我正在制作游戏，我计划使用所有类型的事件、不同的时间间隔等。

哪个最好？



解答

这篇文章提供了一个相当全面的解释：

“比较 .NET Framework 类库中的计时器类”- 也可作为 .chm 文件使用

具体区别似乎是它System.Timers.Timer面向多线程应用程序，因此通过其SynchronizationObject属性是线程安全的，而System.Threading.Timer具有讽刺意味的是它不是开箱即用的线程安全。

我认为两者之间没有区别，因为它与您的间隔有多小有关。


更多建议

System.Threading.Timer是一个普通的计时器。它会在线程池线程（来自工作池）上调用您。

System.Timers.Timer是System.ComponentModel.Component包装 a 的 a System.Threading.Timer，并提供一些用于在特定线程上分派的附加功能。

System.Windows.Forms.Timer而是包装一个本机仅消息 HWND并使用窗口计时器在该 HWND 消息循环中引发事件。

如果您的应用程序没有 UI，并且您希望尽可能使用最轻量级和通用的 .Net 计时器（因为您很乐意弄清楚自己的线程/调度），那么它在框架中就可以了System.Threading.Timer。

我不完全清楚所谓的“非线程安全”问题是什么System.Threading.Timer。也许它与这个问题中的问题相同：Thread-safety of System.Timers.Timer vs System.Threading.Timer，或者也许每个人都意味着：

  1. 使用定时器时很容易编写竞争条件。例如看到这个问题：Timer (System.Threading) thread safety

  2. 计时器通知的重新进入，您的计时器事件可以在您完成 第一个事件处理之前触发并 再次 呼叫您。例如看到这个问题：Thread-safe execution using System.Threading.Timer and Monitor __

如何将字符串拆分为单词列表？

Arya

2023-02-17 22:25

如何拆分句子并将每个单词存储在列表中？例如，给定一个字符串"these are words"，如何得到一个列表["these", "are", "words"]？

要拆分其他定界符，请参阅在 python 中用定界符拆分字符串。

要拆分为单个字符，请参阅如何将字符串拆分为字符列表？.



解答

给定一个字符串sentence，这会将每个单词存储在一个名为的列表中words：

words = sentence.split()


更多建议

text要在任何连续运行的空格上拆分字符串：

words = text.split()

在自定义分隔符上拆分字符串，text例如","：

words = text.split(",")

变量words将是 a并包含来自分隔符上的 split 的list单词。text

如何避免在 Excel VBA 中使用 Select

Arya

2023-02-17 22:25

我听说过很多关于在 Excel VBA 中使用的可以理解的厌恶.Select，但我不确定如何避免使用它。我发现如果我能够使用变量而不是Select函数，我的代码将更易于重用。ActiveCell但是，如果不使用，我不确定如何引用事物（如等） Select。

我找到了关于范围的这篇文章和关于不使用 select 的好处的这个例子，但我找不到关于 如何 使用 select 的任何内容。



解答

如何避免选择的一些例子

使用Dim的变量

Dim rng as Range

Set变量到需要的范围。引用单个单元格范围的方法有很多种：

Set rng = Range("A1")
Set rng = Cells(1, 1)
Set rng = Range("NamedRange")

或多单元格范围：

Set rng = Range("A1:B10")
Set rng = Range("A1", "B10")
Set rng = Range(Cells(1, 1), Cells(10, 2))
Set rng = Range("AnotherNamedRange")
Set rng = Range("A1").Resize(10, 2)

您 可以 使用该Evaluate方法的快捷方式，但这效率较低，通常应在生产代码中避免使用。

Set rng = [A1]
Set rng = [A1:B10]

以上所有示例均指 活动工作表 上的单元格。除非你特别想只使用活动工作表，否则最好Worksheet也将变量调暗：

Dim ws As Worksheet
Set ws = Worksheets("Sheet1")
Set rng = ws.Cells(1, 1)
With ws
    Set rng = .Range(.Cells(1, 1), .Cells(2, 10))
End With

如果您 确实 想使用ActiveSheet，为了清楚起见，最好明确说明。但要小心，因为某些Worksheet方法会更改活动表。

Set rng = ActiveSheet.Range("A1")

同样，这是指 活动工作簿 。除非你特别想只使用ActiveWorkbookor ThisWorkbook，否则最好Workbook也 Dim 一个变量。

Dim wb As Workbook
Set wb = Application.Workbooks("Book1")
Set rng = wb.Worksheets("Sheet1").Range("A1")

如果您 确实 想使用ActiveWorkbook，为了清楚起见，最好明确说明。但是要小心，因为很多WorkBook方法都会改变活动书。

Set rng = ActiveWorkbook.Worksheets("Sheet1").Range("A1")

您还可以使用该ThisWorkbook对象来引用包含运行代码的书。

Set rng = ThisWorkbook.Worksheets("Sheet1").Range("A1")

一段常见的（坏的）代码是打开一本书，获取一些数据然后再次关闭

这不好：

Sub foo()
    Dim v as Variant
    Workbooks("Book1.xlsx").Sheets(1).Range("A1").Clear
    Workbooks.Open("C:\Path\To\SomeClosedBook.xlsx")
    v = ActiveWorkbook.Sheets(1).Range("A1").Value
    Workbooks("SomeAlreadyOpenBook.xlsx").Activate
    ActiveWorkbook.Sheets("SomeSheet").Range("A1").Value = v
    Workbooks(2).Activate
    ActiveWorkbook.Close()
End Sub

它会更好：

Sub foo()
    Dim v as Variant
    Dim wb1 as Workbook
    Dim  wb2 as Workbook
    Set wb1 = Workbooks("SomeAlreadyOpenBook.xlsx")
    Set wb2 = Workbooks.Open("C:\Path\To\SomeClosedBook.xlsx")
    v = wb2.Sheets("SomeSheet").Range("A1").Value
    wb1.Sheets("SomeOtherSheet").Range("A1").Value = v
    wb2.Close()
End Sub

将范围传递给您的Subs 和Functions 作为范围变量：

Sub ClearRange(r as Range)
    r.ClearContents
    '....
End Sub

Sub MyMacro()
    Dim rng as Range
    Set rng = ThisWorkbook.Worksheets("SomeSheet").Range("A1:B10")
    ClearRange rng
End Sub

您还应该将方法（例如Find和Copy）应用于变量：

Dim rng1 As Range
Dim rng2 As Range
Set rng1 = ThisWorkbook.Worksheets("SomeSheet").Range("A1:A10")
Set rng2 = ThisWorkbook.Worksheets("SomeSheet").Range("B1:B10")
rng1.Copy rng2

如果要遍历一系列单元格，通常最好（更快）先将范围值复制到变体数组并遍历它：

Dim dat As Variant
Dim rng As Range
Dim i As Long

Set rng = ThisWorkbook.Worksheets("SomeSheet").Range("A1:A10000")
dat = rng.Value  ' dat is now array (1 to 10000, 1 to 1)
for i = LBound(dat, 1) to UBound(dat, 1)
    dat(i,1) = dat(i, 1) * 10 ' Or whatever operation you need to perform
next
rng.Value = dat ' put new values back on sheet

这是一个小的尝试器，可以尝试什么。


更多建议

应避免.Select, .Activate, Selection, , , 等Activecell的两个主要原因Activesheet``Activeworkbook

  1. 它会减慢你的代码。
  2. 它通常是运行时错误的主要原因。

我们如何避免它？

1） 直接与相关对象打交道

考虑这段代码

Sheets("Sheet1").Activate
Range("A1").Select
Selection.Value = "Blah"
Selection.NumberFormat = "@"

这段代码也可以写成

With Sheets("Sheet1").Range("A1")
    .Value = "Blah"
    .NumberFormat = "@"
End With

2） 如果需要声明你的变量。上面相同的代码可以写成

Dim ws as worksheet

Set ws = Sheets("Sheet1")

With ws.Range("A1")
    .Value = "Blah"
    .NumberFormat = "@"
End With

这是一个很好的答案，但我在这个主题上缺少的是我们真正需要 Activate 的时间。每个人都说它不好，但没有人解释任何使用它有意义的情况。

无法 避免 使用.Activate/.Select. （当我遇到它们时会添加更多链接）

  1. 当您想要向用户展示工作表以便用户可以看到它时。
  2. 从您被迫使用的表单控件运行时，诸如工作宏之类的场景会返回错误.Activate
  3. 当通常的方法Text To Columns/.Formula = .Formula不起作用时，您可能不得不求助于.Select
