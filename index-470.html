删除空数组元素

Arya

2023-02-17 07:46

根据用户提交的内容，我的数组中的某些元素是空字符串。我需要删除那些元素。我有这个：

foreach($linksArray as $link)
{
    if($link == '')
    {
        unset($link);
    }
}
print_r($linksArray);

但它不起作用。$linksArray仍然有空元素。我也试过用这个empty()函数来做，但结果是一样的。



解答

当您处理字符串数组时，您可以简单地使用array_filter()，它可以方便地为您处理所有这些：

print_r(array_filter($linksArray));

请记住， 如果未提供回调 ，则数组中等于FALSE（请参阅转换为布尔值）的所有条目都将被删除。因此，如果您需要保留精确的字符串元素'0'，您将需要一个自定义回调：

// PHP 7.4 and later
print_r(array_filter($linksArray, fn($value) => !is_null($value) && $value !== ''));

// PHP 5.3 and later
print_r(array_filter($linksArray, function($value) { return !is_null($value) && $value !== ''; }));

// PHP < 5.3
print_r(array_filter($linksArray, create_function('$value', 'return $value !== "";')));

注意：如果在删除空元素后需要重新索引数组，请使用：$linksArray = array_values(array_filter($linksArray));


更多建议

您可以使用array_filter删除空元素：

$emptyRemoved = array_filter($linksArray);

如果您(int) 0的阵列中有，您可以使用以下内容：

$emptyRemoved = remove_empty($linksArray);

function remove_empty($array) {
  return array_filter($array, '_remove_empty_internal');
}

function _remove_empty_internal($value) {
  return !empty($value) || $value === 0;
}

编辑： 也许你的元素本身不是空的，而是包含一个或多个空格......你可以在使用之前使用以下内容array_filter

$trimmedArray = array_map('trim', $linksArray);

mysql_fetch_array()/mysql_fetch_assoc()/mysql_fetch_row()/mysql_num_rows 等...期望参数 1 是资源

Arya

2023-02-17 07:46

我正在尝试从 MySQL 表中选择数据，但收到以下错误消息之一：

mysql_fetch_array() 期望参数 1 是资源，给定的布尔值

这是我的代码：

$username = $_POST['username'];
$password = $_POST['password'];

$result = mysql_query('SELECT * FROM Users WHERE UserName LIKE $username');

while($row = mysql_fetch_array($result)) {
    echo $row['FirstName'];
}



解答

查询可能由于各种原因而失败，在这种情况下，mysql_* 和 mysqli 扩展将从false它们各自的查询函数/方法返回。您需要测试该错误情况并相应地进行处理。

mysql_ 扩展名 ：

注意 mysql_函数已弃用并已在 php 版本 7 中删除。

$result在将其传递给之前检查mysql_fetch_array。你会发现这是false因为查询失败。mysql_query有关可能的返回值和如何处理它们的建议，请参阅 [ ][1] 文档。

$username = mysql_real_escape_string($_POST['username']);
$password = $_POST['password'];
$result = mysql_query("SELECT * FROM Users WHERE UserName LIKE '$username'");

if($result === FALSE) { 
    trigger_error(mysql_error(), E_USER_ERROR);
}

while($row = mysql_fetch_array($result))
{
    echo $row['FirstName'];
}


更多建议

当您的查询中出现导致查询失败的错误时，将显示此错误消息。它会在使用时表现出来：

  • mysql_fetch_array/mysqli_fetch_array()
  • mysql_fetch_assoc()/mysqli_fetch_assoc()
  • mysql_num_rows()/mysqli_num_rows()

注意 ：如果您的查询没有影响任何行，则不会出现 此 错误。只有语法无效的查询才会产生此错误。

故障排除步骤

  • 确保您已将开发服务器配置为显示所有错误。您可以通过将其放在文件顶部或配置文件中来执行此操作error_reporting(-1);：如果您有任何语法错误，它会向您指出。

  • 使用mysql_error()。mysql_error()将报告 MySQL 在执行查询时遇到的任何错误。

示例用法：

    mysql_connect($host, $username, $password) or die("cannot connect"); 
mysql_select_db($db_name) or die("cannot select DB");

$sql = "SELECT * FROM table_name";
$result = mysql_query($sql);

if (false === $result) {
    echo mysql_error();
}

  • 从 MySQL 命令行或类似phpMyAdmin 的工具运行查询。如果您的查询中有语法错误，它会告诉您它是什么。

  • 确保你的报价是正确的。查询周围缺少引号或值可能会导致查询失败。

  • 确保你正在逃避你的价值观。查询中的引号可能会导致查询失败（并且还会使您面临 SQL 注入）。用于mysql_real_escape_string()转义您的输入。

  • 确保您没有混合mysqli_*使用mysql_*。它们不是一回事，不能一起使用。（如果您要选择一个或另一个mysqli_*，请坚持使用 。请参阅下文了解原因。）

其他提示

mysql_*函数不应该用于新代码。它们不再维护，社区已开始弃用过程。相反，您应该了解准备好的语句并使用PDO或MySQLi。如果您无法决定，本文将帮助您做出选择。如果您想学习，这里有很好的 PDO 教程。

如何在 Python 中获取绝对文件路径

Arya

2023-02-17 07:45

给定诸如 之类的路径"mydir/myfile.txt"，如何在 Python 中找到文件的绝对路径？例如，在 Windows 上，我可能会得到：

"C:/example/cwd/mydir/myfile.txt"



解答

>>> import os
>>> os.path.abspath("mydir/myfile.txt")
'C:/example/cwd/mydir/myfile.txt'

如果它已经是绝对路径，也可以使用：

>>> import os
>>> os.path.abspath("C:/example/cwd/mydir/myfile.txt")
'C:/example/cwd/mydir/myfile.txt'


更多建议

您可以使用新的 Python 3.4 库pathlib。（您也可以使用 获得它用于 Python 2.6 或 2.7 pip install pathlib。）作者写道：“这个库的目的是提供一个简单的类层次结构来处理文件系统路径和用户对它们执行的常见操作。”

要在 Windows 中获取绝对路径：

>>> from pathlib import Path
>>> p = Path("pythonw.exe").resolve()
>>> p
WindowsPath('C:/Python27/pythonw.exe')
>>> str(p)
'C:\\Python27\\pythonw.exe'

或者在 UNIX 上：

>>> from pathlib import Path
>>> p = Path("python3.4").resolve()
>>> p
PosixPath('/opt/python3/bin/python3.4')
>>> str(p)
'/opt/python3/bin/python3.4'

文档在这里： https: //docs.python.org/3/library/pathlib.html
