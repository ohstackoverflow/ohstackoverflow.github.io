从原始 GitHub 存储库中拉取新的更新到分叉的 GitHub 存储库中

Arya

2023-02-17 12:42

我在 GitHub 上分叉了某人的存储库，并想使用在原始存储库中进行的提交和更新来更新我的版本。这些是在我分叉我的副本后制作的。

我怎样才能提取在源中所做的更改并将它们合并到我的存储库中？



解答

您必须将原始存储库（您分叉的存储库）添加为远程。

来自关于分叉存储库的 GitHub 文档：

一旦克隆完成，你的 repo 将有一个名为“ origin”的远程，它指向你在 GitHub 上的分支。
不要让这个名字混淆了你，这并不指向你从中分叉的原始回购。为了帮助您跟踪该 repo，我们将添加另一个名为“upstream”的远程：

$ cd PROJECT_NAME
$ git remote add upstream

https://github.com/ORIGINAL_OWNER/ORIGINAL_REPOSITORY.git $ git fetch upstream

# then: (like "git pull" which is fetch + merge)
$ git merge upstream/master master

# or, better, replay your local work on top of the fetched branch
# like a "git pull --rebase"
$ git rebase upstream/master

还有一个命令行工具（gh）可以方便上面的操作。

这是它如何工作的视觉效果：

另请参阅“ Git 叉子实际上是 Git 克隆吗？ ”。


更多建议

除了 VonC 的回答之外，您还可以根据自己的喜好进一步调整它。

从远程分支获取后，您仍然需要合并提交。我会更换

$ git fetch upstream

和

$ git pull upstream master

因为 git pull 本质上是 git fetch + git merge。

如何使用 Node.js 解决“找不到模块”错误？

Arya

2023-02-17 12:41

从 GitHub 拉下模块并按照说明构建它后，我尝试使用以下方法将其拉入现有项目：

> npm install ../faye

这似乎可以解决问题：

> npm list
/home/dave/src/server
└─┬ faye@0.7.1
  ├── cookiejar@1.3.0
  ├── hiredis@0.1.13
  └── redis@0.7.1

但是 Node.js 找不到模块：

> node app.js
node.js:201
        throw e; // process.nextTick error, or 'error' event on first tick
              ^
Error: Cannot find module 'faye'
    at Function._resolveFilename (module.js:334:11)

    at Object.<anonymous> (/home/dave/src/server/app.js:2:12)
    at Module._compile (module.js:432:26)

    at Module.load (module.js:351:31)

我真的很想了解这里发生了什么，但对于下一步该去哪里我有点不知所措。有什么建议么？



解答

使用npm install仅将模块安装到当前目录（在名为 的子目录中node_modules）。app.js 位于下面吗home/dave/src/server/？如果没有，并且您想从任何目录使用该模块，则需要使用全局安装它npm install -g。

我通常在本地安装大多数包，以便将它们与我的项目代码一起签入。

更新（8/2019）：

现在你可以使用package-lock.json文件，它是在 npm 修改你的 node_modules 目录时自动生成的。因此，您可以省去签入包，因为它会package-lock.json跟踪您当前正在使用的 node_modules 的确切版本。package-lock.json要从而不是package.json使用命令安装包npm ci。

更新（3/2016）：

我的回复收到了很多批评，特别是我签入了我的代码所依赖的包。几天前，有人取消发布了他们所有的包（https://kodfabrik.com/journal/i-ve- just-liberated-my-modules），这破坏了 React、Babel 和几乎所有其他东西。希望现在很清楚，如果你有生产代码，你不能依赖 NPM 实际为你维护你的依赖关系。


更多建议

我有一个非常相似的问题。删除整个node_modules文件夹并重新安装对我有用：

rm -rf node_modules
npm install

如何 git-cherry-pick 只更改某些文件？

Arya

2023-02-17 12:41

如果我想合并到一个 Git 分支中，仅对在特定提交中更改的某些文件所做的更改包括对多个文件的更改，如何实现？

假设调用的 Git 提交对文件, ,和stuff进行了更改，但我只想合并对文件和 的更改。这听起来像是一份工作，但只知道如何合并整个提交，而不是文件的子集。A``B``C``D``stuff``A``B``git cherry- pick``cherry-pick



解答

cherry-pick -n我会用( )来做--no-commit，它可以让你在提交之前检查（和修改）结果：

git cherry-pick -n <commit>

# unstage modifications you don't want to keep, and remove the
# modifications from the work tree as well.
# this does work recursively!
git checkout HEAD <path>

# commit; the message will have been stored for you by cherry-pick
git commit

如果绝大多数修改都是您不想要的，而不是检查单个路径（中间步骤），您可以重置所有内容，然后添加您想要的内容：

# unstage everything
git reset HEAD

# stage the modifications you do want
git add <path>

# make the work tree match the index
# (do this from the top level of the repo)
git checkout .


更多建议

其他方法对我不起作用，因为提交对许多其他文件有很多更改和冲突。我想到的只是

git show SHA -- file1.txt file2.txt | git apply -

它实际上并没有add文件或为您提交，因此您可能需要跟进

git add file1.txt file2.txt
git commit -c SHA

或者如果你想跳过添加你可以使用--cached参数git apply

git show SHA -- file1.txt file2.txt | git apply --cached -

您也可以对整个目录做同样的事情

git show SHA -- dir1 dir2 | git apply -
