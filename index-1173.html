如何从项目中删除 CocoaPods？

Arya

2023-02-18 03:12

从项目中删除 CocoaPods 的正确方法是什么？我想删除整个 CocoaPod。由于我的客户强加的一些限制，我无法使用它。我只需要一个 xcodeproj 而不是 xcworkspace。



解答

从项目中删除 CocoaPods 是可能的，但目前不能通过 CLI 自动执行。首先，如果您遇到的唯一问题是无法使用 an，xcworkspace则可以xcodeproj通过使用--no- integrate将生成Pods.xcodeproj但不是工作区的标志来将 CocoaPods 与 s 一起使用。然后你可以将它xcodeproj作为子项目添加到你的 main xcodeproj。

如果你真的想删除所有 CocoaPods 集成，你需要做一些事情：

注意 如果操作不当，编辑其中一些内容可能会破坏您的主要项目。我强烈建议您将您的项目签入源代码管理以防万一。此外，这些说明适用于 CocoaPods 版本0.39.0，它们可能会随着新版本的变化而变化。

  1. 删除独立文件（Podfile Podfile.lock和您的Pods目录）
  2. 删除生成的xcworkspace
  3. 打开您的文件，删除对和（在组中）xcodeproj的引用Pods.xcconfig``libPods.a``Frameworks
  4. 在您的Build Phases删除下Copy Pods Resources，Embed Pods Frameworks和Check Pods Manifest.lock阶段。
  5. 这可能看起来很明显，但您需要以其他方式集成第 3 方库或从您的代码中删除对它们的引用。

在这些步骤之后，您应该设置一个xcodeproj在集成 CocoaPods 之前就存在的文件。如果我遗漏了任何内容，请告诉我，我会对此进行编辑。

此外，我们一直在寻找有关如何改进 CocoaPods 的建议，因此如果您遇到问题，请将它们提交到我们的问题跟踪器中，以便我们想出解决它们的方法！

编辑

正如Jack Wu 在评论中所示，有一个第三方 CocoaPods 插件可以为您自动执行这些步骤。它可以在这里找到。请注意，它是第三方插件，可能并不总是在 CocoaPods 更新时更新。另请注意，它是由 CocoaPods 核心团队成员制作的，因此问题不会成为问题。


更多建议

pod deintegrate和pod clean两个指定的命令，用于从您的项目/存储库中删除 CocoaPod。

这是完整的命令集：

$ sudo gem install cocoapods-deintegrate cocoapods-clean
$ pod deintegrate
$ pod cache clean --all
$ rm Podfile

原始解决方案在这里找到：https://medium.com/@icanhazedit/remove-uninstall-deintegrate- cocoapods-from-your-xcode-ios-project-c4621cee5e42#.wd00fj2e5

CocoaPod 文档pod deintegrate：https ://guides.cocoapods.org/terminal/commands.html#pod_deintegrate

debug1: identity file /Users/tudouya/.ssh/vm/vm_id_rsa.pub type 1

.ssh/config您的文件（或）中可能有一行/etc/ssh_config如下所示：

IdentityFile .../.ssh/vm/vm_id_rsa.pub

您需要从文件名中删除“.pub”扩展名：

IdentityFile .../.ssh/vm/vm_id_rsa

嵌入 Base64 图像

Arya

2023-02-18 03:11

纯粹出于好奇，Base64 图像嵌入在哪些浏览器中工作？我指的是这个。

我意识到对于大多数事情来说，这通常不是一个好的解决方案，因为它会增加页面大小——我只是好奇。

一些例子：

HTML:

<img alt="Embedded Image" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAADIA..." />

CSS：

div.image {
  width:100px;
  height:100px;
  background-image:url(data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAADIA...);
}



解答

更新：2017-01-10

现在所有主流浏览器都支持数据 URI。IE 从版本 8 开始也支持嵌入图像。

http://caniuse.com/#feat=datauri

以下 Web 浏览器现在支持数据 URI：

  • 基于 Gecko，例如 Firefox、SeaMonkey、XeroBank、Camino、Fennec 和 K-Meleon
  • Konqueror，通过 KDE 的 KIO 从输入/输出系统
  • Opera（包括 Nintendo DSi 或 Wii 等设备） * 基于WebKit的，如Safari（包括iOS上）、Android的浏览器、Epiphany和Midori（WebKit是Konqueror的KHTML引擎的衍生产品，但Mac OS X不共享KIO架构所以实现不同），以及Webkit /基于Chromium，例如Chrome
  • 三叉戟
    • Internet Explorer 8：Microsoft 出于安全原因限制了对某些“不可导航”内容的支持，包括担心数据 URI 中嵌入的 JavaScript 可能无法被脚本过滤器（例如基于 Web 的电子邮件客户端使用的过滤器）解释。在版本 8[3] 中，数据 URI 必须小于 32 KiB。
    • 仅以下元素和/或属性支持数据 URI [4]：
    • 对象（仅图像）
    • 图片
    • 输入类型=图像
    • 关联
    • 接受 URL 的 CSS 声明，例如 background-image、background、list-style-type、list- style 等。
    • Internet Explorer 9：Internet Explorer 9 没有 32KiB 限制，允许在更广泛的元素中使用。
    • TheWorld Browser：一个 IE shell 浏览器，内置了对 Data URI scheme 的支持

http://en.wikipedia.org/wiki/Data_URI_scheme#Web_browser_support


更多建议

大多数现代桌面浏览器（如 Chrome、Mozilla 和 Internet Explorer）都支持编码为数据 URL 的图像。但在某些移动浏览器中显示数据 URL 存在问题：Android Stock Browser 和 Dolphin Browser 不会显示 嵌入的 JPEG 。

我推荐您使用以下工具进行在线 base64 编码/解码：

  • 编码成Base64格式

  • 从 Base64 格式解码

选中“格式化为数据 URL”选项以格式化为数据 URL。
