如何修复自制软件权限？

Arya

2023-02-17 15:08

我已经卸载并安装了 Homebrew 3 次，因为它似乎永远不允许我安装任何东西，因为它在大多数安装结束时拒绝我的权限。

作为示例，我将发布我目前面临的这个 libjpeg 下载场景。

我尝试安装 libjpeg 并得到：

$ brew install libjpeg
==> Downloading https://downloads.sf.net/project/machomebrew/Bottles/jpeg-8d.mountain_lion.bottle.1.tar.gz

You can try again using `brew link jpeg'

'brew link jpeg' 结果

Error: Permission denied - /usr/local/opt/jpeg

这是我的酿造医生读到的

$ brew doctor

jpeg

这个权限问题导致无法在任何东西上使用 brew，我真的很感激任何建议。



解答

我能够通过chown在文件夹上使用来解决问题：

sudo chown -R "$USER":admin /usr/local

此外，您（很可能）必须在以下方面做同样的事情/Library/Caches/Homebrew：

sudo chown -R "$USER":admin /Library/Caches/Homebrew

显然，我sudo以前曾以一种改变我的文件夹权限的方式使用过/usr/local，从现在开始，所有使用 brew 的安装都被证明是成功的。

这个答案来自gitHub 的自制问题跟踪器


更多建议

macOS High Sierra 用户的新命令，因为无法chown打开/usr/local：

bash/zsh:

sudo chown -R $(whoami) $(brew --prefix)/*

fish:

sudo chown -R (whoami) (brew --prefix)/*

参考：无法在 High Sierra 中 chown /usr/local

Java 中的 == 和 equals() 有什么区别？

Arya

2023-02-17 15:07

我想澄清一下我是否理解正确：

  • ==是引用比较，即两个对象指向同一个内存位置
  • .equals()计算对象中值的比较



解答

一般来说，你的问题的答案是“是”，但是......

  • .equals(...)只会比较写的东西来比较，不多也不少。
  • 如果一个类没有重写 equals 方法，那么它默认使用equals(Object o)最近的重写了这个方法的父类的方法。
  • 如果没有父类提供覆盖，则它默认为最终父类 Object 中的方法，因此您只剩下该方法了Object#equals(Object o)。根据对象 API，这与==; 也就是说， 当且仅当 两个变量都引用同一个对象时，如果它们的引用是一个且相同，它才返回 true。因此，您将测试 对象相等性 而不是 功能相等性 。
  • hashCode如果您覆盖，请始终记住覆盖equals，以免“破坏合同”。根据 API，如果hashCode()两个对象的方法表明它们是等效的，则从两个对象的方法返回的结果 必须 相同。equals反之则 不一定 成立。


更多建议

关于 String 类：

equals() 方法 比较 String 实例（在堆上）内的“值”，而不管两个对象引用是否引用相同的 String 实例。如果任何两个 String 类型的对象引用都引用同一个 String 实例，那就太好了！如果两个对象引用引用两个不同的 String 实例.. 它没有什么区别。它是每个正在比较的 String 实例中的“值”（即：字符数组的内容）。

另一方面，“==” 运算符比较 两个对象引用 的值，看它们是否引用同一个 String 实例 。如果两个对象引用的值“引用”同一个 String 实例，那么布尔表达式的结果将是“true”..duh。另一方面，如果两个对象引用的值“引用” 不同的 String 实例 （即使两个 String 实例具有相同的“值”，即每个 String 实例的字符数组的内容相同）布尔表达式的结果将为“false”。

与任何解释一样，让它沉入其中。

我希望这可以解决一些问题。

对于整个项目搜索：

3. 范围（在表单部分）> 封闭项目（单选按钮选择）。
