如何遍历目录中的文件并更改路径并为文件名添加后缀

Arya

2023-02-17 14:55

我需要编写一个脚本，用不同的参数启动我的程序。我开始我的程序：

./MyProgram.exe Data/data1.txt [Logs/data1_Log.txt].

这是我想做的伪代码：

for each filename in /Data do
  for int i = 0, i = 3, i++
    ./MyProgram.exe Data/filename.txt Logs/filename_Log{i}.txt
  end for
end for

我如何从第一个参数创建第二个参数，使其看起来像 dataABCD_Log1.txt 并启动我的程序？



解答

首先注意几点：当您用作Data/data1.txt参数时，它真的应该是/Data/data1.txt（带前导斜杠）吗？另外，外循环应该只扫描 .txt 文件，还是扫描 /Data 中的所有文件？这是一个答案，假设/Data/data1.txt只有 .txt 文件：

#!/bin/bash
for filename in /Data/*.txt; do
    for ((i=0; i<=3; i++)); do
        ./MyProgram.exe "$filename" "Logs/$(basename "$filename" .txt)_Log$i.txt"
    done
done

笔记：

  • /Data/*.txt扩展为 /Data 中文本文件的路径（ 包括 /Data/ 部分）
  • $( ... )运行 shell 命令并在命令行中的那个点插入它的输出
  • basename somepath .txt输出 somepath 的基础部分，从末尾删除 .txt （例如/Data/file.txt-> file）

Data/file.txt如果您需要使用而不是运行 MyProgram /Data/file.txt，请使用"${filename#/}"删除前导斜杠。另一方面，如果您真的Data不想/Data扫描，只需使用for filename in Data/*.txt.


更多建议

每当您通过 globbing 遍历文件时，最好避免 glob 不匹配的极端情况（这会使循环变量扩展到（不匹配的）glob 模式字符串本身）。

例如：

for filename in Data/*.txt; do
    [ -e "$filename" ] || continue
    # ... rest of the loop body
done

参考：Bash 陷阱

nvm 在新的终端会话中保持“忘记”节点

Arya

2023-02-17 14:55

在 OS X 中使用新的终端会话时，nvm忘记节点版本并默认为无：

$ nvm ls:

         .nvm
     v0.11.12
     v0.11.13

我必须nvm use v.0.11.13在每个会话中继续击球：

         .nvm
     v0.11.12
->   v0.11.13

我已经尝试了brew安装和官方安装脚本。

我的 .profile 酿造版本：

#nvm
export NVM_DIR=~/.nvm
source $(brew --prefix nvm)/nvm.sh

对于 install.sh 脚本：

$ curl https://raw.githubusercontent.com/creationix/nvm/v0.10.0/install.sh | bash

#nvm
export NVM_DIR="/Users/farhad/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"  # This loads nvm

关于我做错了什么的任何线索？



解答

尝试nvm alias default。例如：

$ nvm alias default 0.12.7

这会在您的 shell 中设置默认节点版本。然后通过关闭 shell 窗口，打开一个新窗口来验证更改是否持续存在，然后：node --version


更多建议

为node自身设置别名以避免稍后更新默认别名以及节点版本更新。

nvm alias default node
