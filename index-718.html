如何在 TypeScript 中为对象动态分配属性？

Arya

2023-02-17 14:19

如果我想以编程方式为 Javascript 中的对象分配属性，我会这样做：

var obj = {};
obj.prop = "value";

但是在 TypeScript 中，这会产生一个错误：

“{}”类型的值上不存在属性“prop”

我应该如何将任何新属性分配给 TypeScript 中的对象？



解答

索引类型

可以表示obj为any，但这违背了使用打字稿的全部目的。obj = {}暗示obj是一个Object. 将其标记为any没有意义。为了实现所需的一致性，可以按如下方式定义接口。

interface LooseObject {
    [key: string]: any
}

var obj: LooseObject = {};

或者使其紧凑：

var obj: {[k: string]: any} = {};

LooseObject可以接受任何字符串作为键和any类型作为值的字段。

obj.prop = "value";
obj.prop2 = 88;

该解决方案的真正优点在于您可以在界面中包含类型安全字段。

interface MyType {
    typesafeProp1?: number,
    requiredProp1: string,
    [key: string]: any
}

var obj: MyType ;
obj = { requiredProp1: "foo"}; // valid
obj = {} // error. 'requiredProp1' is missing
obj.typesafeProp1 = "bar" // error. typesafeProp1 should be a number

obj.prop = "value";
obj.prop2 = 88;

Record<Keys,Type>公用事业类型

更新（2020 年 8 月）：@transang 在评论中提出了这个问题

Record<Keys,Type>是打字稿中的实用程序类型。对于属性名称未知的键值对，它是一种更简洁的替代方法。值得注意的是，这是泛型where和areRecord<Keys,Type>的命名别名。IMO，这值得一提{[k: Keys]: Type}``Keys``Type

为了比较，

var obj: {[k: string]: any} = {};

成为

var obj: Record<string,any> = {}

MyType现在可以通过扩展 Record 类型来定义

interface MyType extends Record<string,any> {
    typesafeProp1?: number,
    requiredProp1: string,
}

虽然这回答了原始问题，但@GreeneCreations在此处的回答可能会提供有关如何解决该问题的另一种观点。


更多建议

或者一口气：

  var obj:any = {}
  obj.prop = 5;

如何在 Flask 中提供静态文件

Arya

2023-02-17 14:18

所以这很尴尬。我有一个我放在一起的应用程序Flask，现在它只是提供一个带有一些 CSS 和 JS 链接的静态 HTML 页面。而且我找不到文档中Flask描述返回静态文件的位置。是的，我可以使用render_template，但我知道数据没有模板化。我本来认为send_file或者url_for是正确的事情，但我无法让这些工作。与此同时，我正在打开文件、阅读内容，并Response使用适当的 mimetype 进行装配：

import os.path

from flask import Flask, Response


app = Flask(__name__)
app.config.from_object(__name__)


def root_dir():  # pragma: no cover
    return os.path.abspath(os.path.dirname(__file__))


def get_file(filename):  # pragma: no cover
    try:
        src = os.path.join(root_dir(), filename)
        # Figure out how flask returns static files
        # Tried:
        # - render_template
        # - send_file
        # This should not be so non-obvious
        return open(src).read()
    except IOError as exc:
        return str(exc)


@app.route('/', methods=['GET'])
def metrics():  # pragma: no cover
    content = get_file('jenkins_analytics.html')
    return Response(content, mimetype="text/html")


@app.route('/', defaults={'path': ''})
@app.route('/<path:path>')
def get_resource(path):  # pragma: no cover
    mimetypes = {
        ".css": "text/css",
        ".html": "text/html",
        ".js": "application/javascript",
    }
    complete_path = os.path.join(root_dir(), path)
    ext = os.path.splitext(path)[1]
    mimetype = mimetypes.get(ext, "text/html")
    content = get_file(complete_path)
    return Response(content, mimetype=mimetype)


if __name__ == '__main__':  # pragma: no cover
    app.run(port=80)

有人想为此提供代码示例或网址吗？我知道这将非常简单。



解答

在生产环境中，在您的应用程序前面配置 HTTP 服务器（Nginx、Apache 等）来为来自/static静态文件夹的请求提供服务。专用 Web 服务器非常擅长高效地提供静态文件，尽管与低容量的 Flask 相比，您可能不会注意到差异。

Flask 自动创建一个/static/<path:filename>路由，该路由将在定义您的 Flask 应用程序的 Python 模块旁边的文件夹filename下提供任何服务。static用于url_for链接到静态文件：url_for('static', filename='js/analytics.js')

您还可以使用send_from_directory从您自己的路径中的目录提供文件。这需要一个基本目录和一个路径，并确保路径包含在目录中，这样可以安全地接受用户提供的路径。这在您想要在提供文件之前检查某些内容的情况下很有用，例如登录用户是否具有权限。

from flask import send_from_directory

@app.route('/reports/<path:path>')
def send_report(path):
    return send_from_directory('reports', path)

不要 将send_fileorsend_static_file与用户提供的路径一起使用。这将使您面临目录遍历攻击。send_from_directory旨在安全地处理已知目录下用户提供的路径，如果路径试图转义目录，则会引发错误。

如果您在内存中生成文件而不将其写入文件系统，则可以将对象传递BytesIO给以send_file像文件一样为它提供服务。在这种情况下，您需要将其他参数传递给send_file，因为它无法推断文件名或内容类型等信息。


更多建议

如果您只想移动静态文件的位置，那么最简单的方法是在构造函数中声明路径。在下面的示例中，我已将我的模板和静态文件移动到一个名为web.

app = Flask(__name__,
            static_url_path='', 
            static_folder='web/static',
            template_folder='web/templates')
  • static_url_path=''从 URL 中删除任何前面的路径（即 default /static）。
  • static_folder='web/static'``web/static将文件夹中找到的任何文件作为静态文件提供。
  • template_folder='web/templates'同样，这会更改模板文件夹。

使用此方法，以下 URL 将返回一个 CSS 文件：

<link rel="stylesheet" type="text/css" href="/css/bootstrap.min.css">

IntelliJ：永远不要使用通配符导入

Arya

2023-02-17 14:18

有没有办法告诉 IntelliJ 永远不要使用通配符导入？在“设置 > 代码样式 > 导入”下，我可以看到您可以在使用通配符导入的 IntelliJ 之前指定“类计数”。但是，如果我不想使用通配符导入，是否可以关闭此功能？

我试过输入 -1 或将该字段留空，但这只是告诉 IntelliJ 始终使用通配符导入。显然，一个不太好的解决方案是输入一个高得离谱的数字，这样你就永远不会遇到通配符导入，但我希望有一个更好的方法来关闭它。



解答

很明显为什么要禁用它：强制 IntelliJ 单独包含每个导入。它使人们更容易弄清楚您正在使用的类的确切来源。

点击工具栏上的Settings“扳手”图标，打开“Code Style”下的“Imports”，勾选“Use single class import”选项。您还可以完全删除“Packages to use import with *”下的条目，或者指定一个阈值，仅*当包中的各个类超过该阈值时才使用“”。

更新： 在 IDEA 13 中“使用单类导入”不会阻止通配符导入。解决方案是转到Preferences（在 macOS 上为⌘+ /在 Windows 和 Linux 上为+ + ）设置并设置为更高的值。任何超过 99 的值似乎都可以正常工作。,``Ctrl``Alt``S``> Editor > Code Style > Java > Imports tab``Class count to use import with '*'``Names count to use static import with '*'


更多建议

  1. 文件\设置... ( Ctrl+ Alt+ S)
  2. 项目设置 > 编辑器 > 代码风格 > Java > 导入选项卡
  3. 将类数 设置为使用带有“*”的导入到 999
  4. 将名称计数 设置为使用带“*”的静态导入到 999
