Git mergetool 生成不需要的 .orig 文件

Arya

2023-02-18 04:15

当我使用 Kdiff3（以及我尝试过的其他合并工具）执行合并冲突解决时，我注意到在解决时*.orig创建了一个文件。有没有办法让它不创建那个额外的文件？



解答

一个可能的解决方案git config：

git config --global mergetool.keepBackup false

执行合并后，带有冲突标记的原始文件可以另存为带.orig扩展名的文件。
如果此变量设置为false则不保留此文件。
默认为true（即保留备份文件）。

另一种方法是不添加或忽略这些文件，正如这篇gitguru 文章中所建议的那样，

git mergetool使用“”后缀保存文件的合并冲突版本.orig。
在添加和提交合并或添加*.orig到您的.gitignore.

Berik在评论中建议使用：

find . -name \*.orig 
find . -name \*.orig -delete

Charles Bailey在他的回答中建议注意 内部 diff 工具设置 ，无论 git 设置是什么，它也可以生成这些备份文件。

  • kdiff3 有自己的设置（请参阅其手册中的“目录合并”）。
  • 其他工具如 WinMerge 可以有自己的备份文件扩展名（WinMerge：，如其手册.bak中所述）。

因此，您还需要重置这些设置。


更多建议

你必须小心使用，kdiff3因为 whilegit mergetool可以配置为.orig在合并期间保存文件，默认行为kdiff3也是.orig独立于git mergetool.

您必须确保mergetool备份已关闭：

git config --global mergetool.keepBackup false

并且 kdiff3 的设置设置为不创建备份：

Configure/Options => Directory Merge => Backup Files (*.orig)

如何在不删除内容的情况下取消暂存大量文件

Arya

2023-02-18 04:14

我不小心添加了很多临时文件git add -A

我设法使用以下命令取消暂存文件并设法删除脏索引。

git ls-files -z | xargs -0 rm -f
git diff --name-only --diff-filter=D -z | xargs -0 git rm --cached

上述命令列在git help rm. 但遗憾的是，我的文件在执行时也被删除了，即使我已经提供了缓存选项。如何在不丢失内容的情况下清除索引？

如果有人可以解释此管道操作的工作方式，这也会有所帮助。



解答

git reset

如果你想要的只是撤消一个过分热心的“git add”运行：

git reset

您的更改将取消暂存并准备好供您根据需要重新添加。

不要跑git reset --hard。

它不仅会取消暂存您添加的文件，还会还原您在工作目录中所做的任何更改。如果您在工作目录中创建了任何新文件，它 不会 删除它们。


更多建议

如果你有一个原始的回购协议（或者 HEAD 没有设置） [1] 你可以简单地

rm .git/index

当然，这将需要您重新添加您 确实 想要添加的文件。

[1] 注意（如评论中所述）这通常只会在回购是全新的（“原始”）或没有提交时发生。从技术上讲，只要没有结帐或工作树。

只是让它更清楚:)

如何在 C# 中用单个空格替换多个空格？

Arya

2023-02-18 04:14

如何在 C# 中用一个空格替换字符串中的多个空格？

例子：

1 2 3  4    5

将会：

1 2 3 4 5



解答

string sentence = "This is a sentence with multiple    spaces";
RegexOptions options = RegexOptions.None;
Regex regex = new Regex("[ ]{2,}", options);     
sentence = regex.Replace(sentence, " ");


更多建议
