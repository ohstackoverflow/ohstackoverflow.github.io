如何从从 git 克隆的项目中删除版本跟踪？

Arya

2023-02-17 08:40

我想从项目目录中删除所有版本跟踪。

执行此操作的正确方法是什么？

我可以执行 shell 命令吗，例如：

rm -rf .git

从我的项目目录或有没有办法将其作为 Git 命令执行？

我使用这个项目来学习 Git，并意识到我在项目早期犯了一些错误，包括移动、重命名和删除文件。我想删除所有 Git 跟踪并重新开始使用git init.



解答

Git 用于信息的所有数据都存储在 中.git/，因此删除它应该没问题。当然，请确保您的工作副本处于您想要的状态，因为其他所有内容都会丢失。.git文件夹是隐藏的，因此请确保打开“显示隐藏的文件、文件夹和磁盘”选项。

从那里，您可以运行git init以创建一个新的存储库。


更多建议

rm -rf .git应该足够了。这将消除所有与 Git 相关的信息。

解答

  1. 转到 文件 > 设置 > 构建、执行、部署 > 编译器 > Java 编译器如果在 Mac 上，它位于 Intellij IDEA > 首选项... > 构建、执行、部署 > Java 编译器
  2. 将目标字节码版本 更改为您正在使用的模块的 1.8 。

如果你正在使用 Maven

将编译器插件添加到pom.xml顶层project节点下：

<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
            </configuration>
        </plugin>
    </plugins>
</build>

什么文件使用 .md 扩展名，我应该如何编辑它们？

Arya

2023-02-17 08:39

在 GitHub 上，有几个项目有README.md文件。好像是一个简单的格式文件，用来表达文字和图片。

我想某处有编辑器或语法解释。

在哪里可以找到文件的介绍.md？



解答

Markdown 是一种纯文本文件格式。扩展名.md和.markdown只是用Markdown语法编写的文本文件。如果你的 repo 中有一个Readme.md，GitHub 将在你的 repo 的主页上显示内容。阅读文档：

  • 标准降价
  • GitHub 风格的 Markdown

您可以Readme.md在 GitHub 本身中编辑该文件。点击 Readme.md，你会发现一个编辑按钮。您可以预览您的更改，甚至可以从那里提交它们。

由于是文本文件，可以使用记事本或Notepad++ (Windows)、TextEdit (Mac) 或任何其他文本编辑器对其进行编辑和修改。存在专门的编辑器，可以在您键入时自动解析降价并生成预览，而其他编辑器则将各种语法着色和装饰应用于显示的文本。不过在这两种情况下，保存的文件仍然是可读的文本文件。

如果你想创建一个md带预览的文件，并且不想安装任何特殊的编辑器，你可以使用像dillinger.io和stackedit.io这样的在线编辑器。他们提供实时预览。您还可以将文件导出到Google Drive或Dropbox。


更多建议

如果您正在寻找编辑器，我建议您使用http://dillinger.io/。它是一个简单的基于浏览器的文本编辑器，可以即时呈现 Markdown。

但是，如果您更喜欢应用程序，并且您使用的是 OS X，则可以尝试Mou。它非常好并且充满了例子。
