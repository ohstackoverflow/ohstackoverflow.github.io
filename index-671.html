设置超时还是设置间隔？

Arya

2023-02-17 13:04

据我所知，这两个 javascript 的行为方式相同：

选项A：

function myTimeoutFunction()
{
    doStuff();
    setTimeout(myTimeoutFunction, 1000);
}

myTimeoutFunction();

选项B：

function myTimeoutFunction()
{
    doStuff();
}

myTimeoutFunction();
setInterval(myTimeoutFunction, 1000);

使用setTimeout和setInterval有什么区别吗？



解答

他们本质上尝试做同样的事情，但该setInterval方法将比该setTimeout方法更准确，因为setTimeout等待 1000 毫秒，运行函数然后设置另一个超时。所以等待时间实际上超过 1000 毫秒（如果您的函数需要很长时间才能执行，则等待时间会更长）。

尽管有人可能认为它会 恰好setInterval每 1000 毫秒执行一次，但重要的是要注意它也会延迟，因为 JavaScript 不是一种多线程语言，这意味着 - 如果脚本的其他部分正在运行 - 间隔将有等待它完成。 ****setInterval

在这个 Fiddle中，您可以清楚地看到超时会落后，而间隔几乎一直在几乎 1 次调用/秒（脚本正在尝试这样做）。如果将顶部的速度变量更改为 20 之类的小值（意味着它将尝试每秒运行 50 次），则间隔永远不会达到平均每秒 50 次迭代。

延迟几乎总是可以忽略不计，但如果你正在编程一些非常精确的东西，你应该选择一个 自调整定时器 （它本质上是一个基于超时的定时器，它会根据它创建的延迟不断调整自己）

有什么不同吗？

是的。Timeout 在调用 setTimeout() 后执行一定时间；Interval 在前一个间隔触发后执行一定时间。

如果您的 doStuff() 函数需要一段时间才能执行，您会注意到差异。例如，如果我们用 表示对 setTimeout/setInterval 的调用.，用 触发超时/间隔，用*执行 JavaScript 代码[-----]，时间线如下所示：

• 表示无法立即执行其代码的间隔触发，而是被挂起。

因此，间隔尝试“赶上”以回到计划。但是，它们不会一个接一个地排队：每个时间间隔只能有一个等待执行。（如果他们都排队，浏览器将留下一个不断扩大的未完成执行列表！）

.    *    •    •    x    •    •    x
     [------][------][------][------]

x 表示无法执行或被挂起的间隔触发，因此被丢弃。

如果您的 doStuff() 函数习惯性地花费比为其设置的时间间隔更长的时间来执行，浏览器将占用 100% 的 CPU 来尝试为其提供服务，并且响应速度可能会变慢。

你用哪个，为什么？

Chained-Timeout 为浏览器提供了有保证的空闲时间；Interval 尝试确保它正在运行的功能尽可能接近其预定时间执行，但以牺牲浏览器 UI 可用性为代价。

我会考虑一次性动画的间隔，我希望它尽可能流畅，而链式超时对于页面加载时一直发生的持续动画更为礼貌。对于要求不高的用途（例如每 30 秒触发一次的普通更新程序或其他），您可以安全地使用其中任何一个。

在浏览器兼容性方面，setTimeout 早于 setInterval，但您今天遇到的所有浏览器都支持这两者。多年来最后一个掉队的是 WinMo <6.5 中的 IE Mobile，但希望它现在也已经过去了。

如何使用保存属性名称的变量检查对象属性是否存在？

Arya

2023-02-17 13:04

我正在检查是否存在一个对象属性，该属性的变量保存有问题的属性名称。

var myObj;
myObj.prop = "exists";
var myProp = "p"+"r"+"o"+"p";

if(myObj.myProp){
    alert("yes, i have that property");
};

这是undefined因为它正在寻找myObj.myProp，但我希望它检查myObj.prop



解答

var myProp = 'prop';
if(myObj.hasOwnProperty(myProp)){
    alert("yes, i have that property");
}

或者

var myProp = 'prop';
if(myProp in myObj){
    alert("yes, i have that property");
}

或者

if('prop' in myObj){
    alert("yes, i have that property");
}

请注意，它hasOwnProperty不会检查继承的属性，而in会检查。例如'constructor' in myObj是真的，但myObj.hasOwnProperty('constructor')不是。


更多建议

您可以使用 hasOwnProperty ，但根据参考资料，您在使用此方法时需要 引号：

if (myObj.hasOwnProperty('myProp')) {
    // do something
}

https://developer.mozilla.org/en- US/docs/Web/JavaScript/Reference/Global_Objects/Object/hasOwnProperty

另一种方法是使用 in 运算符，但您也需要在此处使用 引号：

if ('myProp' in myObj) {
    // do something
}

https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/in

内置open函数中模式a、a+、w、w+、r+的区别？

Arya

2023-02-17 13:03

在python内置的open函数中，modes w, a, w+, a+, 和之间的具体区别是什么r+？

特别是，该文档暗示所有这些都将允许写入文件，并说它打开文件的具体目的是“附加”、“写入”和“更新”，但没有定义这些术语的含义。



解答

打开方式与C标准库函数完全相同 fopen() 。

BSDfopen联机帮助页对它们的定义如下：

 The argument mode points to a string beginning with one of the following
 sequences (Additional characters may follow these sequences.):

 ``r''   Open text file for reading.  The stream is positioned at the
         beginning of the file.

 ``r+''  Open for reading and writing.  The stream is positioned at the
         beginning of the file.

 ``w''   Truncate file to zero length or create text file for writing.
         The stream is positioned at the beginning of the file.

 ``w+''  Open for reading and writing.  The file is created if it does not
         exist, otherwise it is truncated.  The stream is positioned at
         the beginning of the file.

 ``a''   Open for writing.  The file is created if it does not exist.  The
         stream is positioned at the end of the file.  Subsequent writes
         to the file will always end up at the then current end of file,
         irrespective of any intervening fseek(3) or similar.

 ``a+''  Open for reading and writing.  The file is created if it does not
         exist.  The stream is positioned at the end of the file.  Subse-
         quent writes to the file will always end up at the then current
         end of file, irrespective of any intervening fseek(3) or similar.
