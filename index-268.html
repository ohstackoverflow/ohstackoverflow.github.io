如何使用 MySQL 中的命令行获取用户帐户列表？

Arya

2023-02-17 02:37

我正在使用 MySQL 命令行实用程序，可以浏览数据库。现在我需要查看用户帐户列表。我怎样才能做到这一点？

我正在使用 MySQL 5.4.1 版。



解答

使用此查询：

SELECT User FROM mysql.user;

这将输出如下表格：

+-------+
| User  |
+-------+
| root  |
+-------+
| user2 |
+-------+

正如 Matthew Scharley在此答案的评论中指出的User那样，如果您只想查看唯一的用户名，则可以按列分组。


更多建议

我发现这种格式最有用，因为它包括主机字段，这在 MySQL 中很重要，可以区分用户记录。

select User,Host from mysql.user;

为什么允许在带有某些 Unicode 字符的注释中执行 Java 代码？

Arya

2023-02-17 02:36

以下代码生成输出“Hello World!” （不，真的，试试看）。

public static void main(String... args) {

   // The comment below is not a typo.
   // \u000d System.out.println("Hello World!");
}

这样做的原因是 Java 编译器将 Unicode 字符解析\u000d为新行并转换为：

public static void main(String... args) {

   // The comment below is not a typo.
   //
   System.out.println("Hello World!");
}

从而导致评论被“执行”。

既然这可以用来“隐藏”恶意代码或任何邪恶的程序员可以想到的东西， 为什么在注释中允许它 ？

为什么 Java 规范允许这样做？



解答

Unicode 解码发生在任何其他词汇翻译之前。这样做的主要好处是它使得在 ASCII 和任何其他编码之间来回切换变得微不足道。您甚至不需要弄清楚评论的开始和结束位置！

正如JLS 第 3.3 节所述，这允许任何基于 ASCII 的工具来处理源文件：

[...] Java 编程语言指定了一种将用 Unicode 编写的程序转换为 ASCII 的标准方法，该方法将程序更改为可以由基于 ASCII 的工具处理的形式。[...]

这为平台独立性（支持的字符集的独立性）提供了基本保证，这一直是 Java 平台的关键目标。

能够在文件中的任何位置编写任何 Unicode 字符是一个巧妙的功能，在用非拉丁语言记录代码时，在注释中尤其重要。它可以以如此微妙的方式干扰语义的事实只是一个（不幸的）副作用。

更多建议

由于这还没有解决，这里有一个解释，为什么 Unicode 转义的翻译发生在任何其他源代码处理之前：

它背后的想法是它允许在不同字符编码之间无损地转换 Java 源代码。今天，Unicode 得到了广泛的支持，这看起来不是问题，但在当时，来自西方国家的开发人员从他的亚洲同事那里收到一些包含亚洲字符的源代码，进行一些修改并不容易（包括编译和测试它）并将结果发回，所有这些都不会损坏任何东西。

因此，Java 源代码可以用任何编码编写，并允许在标识符、字符和文字以及注释中使用各种字符String。然后，为了无损地传输它，目标编码不支持的所有字符都被替换为它们的 Unicode 转义字符。

这是一个可逆的过程，有趣的是，翻译可以通过一个不需要了解任何 Java 源代码语法的工具来完成，因为翻译规则不依赖于它。这是因为在编译器内部对它们的实际 Unicode 字符的转换也独立于 Java 源代码语法。这意味着您可以在两个方向上执行任意数量的翻译步骤，而无需更改源代码的含义。

这是另一个甚至没有提到的奇怪功能的原因：\uuuuuuxxxx语法：

u当翻译工具正在转义字符并遇到一个已经是转义序列的序列时，它应该在序列中插入一个附加值，转换\ucafe为\uucafe. 意思没有改变，但是当转换到另一个方向时，该工具应该只删除一个u并仅替换包含单个的序列u的 Unicode 字符。这样，即使 Unicode 转义符在来回转换时也能保留其原始形式。我想，没有人使用过该功能......

将 JavaScript 字符串全部转为小写

Arya

2023-02-17 02:36

如何将 JavaScript 字符串值转换为全部小写字母？

例子："Your Name"到"your name"



解答

var lowerCaseName = "Your Name".toLowerCase();


更多建议

使用String对象的toLowerCase或toLocaleLowerCase方法。不同之处在于将考虑用户/主机的当前区域设置。根据ECMAScript 语言规范 (ECMA-262)的第 15.5.4.17 节，......toLocaleLowerCase``toLocaleLowerCase

…与 toLowerCase 完全相同，除了它的结果旨在为主机环境的当前语言环境生成正确的结果，而不是与语言环境无关的结果。只有在少数情况下（例如土耳其语）才会有所不同，在这种情况下，该语言的规则与常规 Unicode 大小写映射冲突。

例子：

var lower = 'Your Name'.toLowerCase();

另请注意，实现了toLowerCase和函数以在 任何 值类型上通用地工作。因此，您甚至可以在非对象上调用这些函数。这样做将意味着在更改结果字符串值中每个字符的大小写之前自动转换为字符串值。例如，您可以在这样的日期直接 申请：toLocaleLowerCase String toLowerCase

var lower = String.prototype.toLowerCase.apply(new Date());

这实际上等同于：

var lower = new Date().toString().toLowerCase();

第二种形式通常因其简单性和可读性而受到青睐。在早期版本的 IE 上，第一个的好处是它可以使用一个null值。toLowerCaseapply or toLocaleLowerCaseon的结果null会产生null（而不是错误条件）。
