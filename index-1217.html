快速读取非常大的表作为数据框

Arya

2023-02-18 04:18

我有非常大的表（3000 万行），我想将其作为数据帧加载到 R 中。它read.table()有很多方便的功能，但似乎在实现中有很多逻辑会减慢速度。在我的例子中，我假设我提前知道列的类型，该表不包含任何列标题或行名，并且没有我必须担心的任何病态字符。

我知道在表中读取列表使用scan()可以非常快，例如：

datalist <- scan('myfile',sep='\t',list(url='',popularity=0,mintime=0,maxtime=0)))

但是我将其转换为数据框的一些尝试似乎将上述性能降低了 6 倍：

df <- as.data.frame(scan('myfile',sep='\t',list(url='',popularity=0,mintime=0,maxtime=0))))

有更好的方法吗？或者很可能完全不同的问题解决方法？



解答

几年后的更新

这个答案是旧的，R 已经移动了。调整read.table运行速度快一点几乎没有什么好处。您的选择是：

  1. 使用vroomtidyverse 包vroom将数据从 csv/tab 分隔文件直接导入 R tibble。请参阅赫克托的回答。

  2. 使用freadindata.table将 csv/tab 分隔文件中的数据直接导入 R。请参阅mnel 的回答。

  3. read_table在（2015 年 4 月起在 CRAN 上）使用readr。这很像fread上面的工作。 链接中的自述文件 解释了这两个函数之间的区别（readr目前声称比“慢 1.5-2 倍” data.table::fread）。

  4. read.csv.rawfromiotools为快速读取 CSV 文件提供了第三种选择。

  5. 尝试在数据库而不是平面文件中存储尽可能多的数据。（作为更好的永久存储介质，数据以二进制格式传入和传出 R，速度更快。）在包read.csv.sql中sqldf，如JD Long 的回答中所述，将数据导入临时 SQLite 数据库，然后读取它进入 R。另请参阅：包，以及包RODBC页面的反向依赖部分。为您提供一种数据类型，它伪装成数据框，但实际上是底层的 MonetDB，从而提高了性能。使用其功能导入数据。允许您直接使用存储在多种类型数据库中的数据。DBI``MonetDB.R``monetdb.read.csv``dplyr

  6. 以二进制格式存储数据也有助于提高性能。使用saveRDS/ readRDS（见下文）、HDF5 格式的h5或包，或包中的/ 。rhdf5``write_fst``read_fst``fst

原来的答案

无论您使用 read.table 还是 scan，都可以尝试一些简单的操作。

  1. Set nrows= 数据中的记录数 ( nmaxin scan)。

  2. 确保comment.char=""关闭评论的解释。

  3. colClasses使用in显式定义每列的类read.table。

  4. 设置multi.line=FALSE还可以提高扫描性能。

如果这些都不起作用，则使用其中一个分析包来确定哪些线路正在减慢速度。read.table也许你可以根据结果写一个简化版本。

另一种方法是在将数据读入 R 之前对其进行过滤。

或者，如果问题是您必须定期读取它，则使用这些方法一次读取数据，然后将数据帧保存为二进制 blob save saveRDS，然后下次您可以更快地检索它 load readRDS.

DT[5,d:=""]

解决与二进制文件的 Git 冲突

Arya

2023-02-18 04:18

我一直在 Windows (msysgit) 上使用 Git 来跟踪我一直在做的一些设计工作的变化。

今天我一直在另一台 PC 上工作（带有远程回购brian），我现在正试图将今天完成的编辑合并回我笔记本电脑上的常规本地版本。

在我的笔记本电脑上，我习惯于git pull brian master将更改拉入我的本地版本。除了主要的 InDesign 文档外，一切都很好——这表明存在冲突。

PC 上的版本 ( brian) 是我想保留的最新版本，但我不知道什么命令告诉 repo 使用这个版本。

我尝试直接将文件复制到我的笔记本电脑上，但这似乎破坏了整个合并过程。

谁能指出我正确的方向？



解答

git checkout对于这种情况，接受--oursor选项。--theirs所以如果你有合并冲突，并且你知道你只是想要你正在合并的分支中的文件，你可以这样做：

$ git checkout --theirs -- path/to/conflicted-file.txt

使用该文件的版本。同样，如果你知道你想要你的版本（不是被合并的那个）你可以使用

$ git checkout --ours -- path/to/conflicted-file.txt


更多建议

您必须手动解决冲突（将文件复制过来），然后像这样提交文件（无论您是复制过来还是使用本地版本）

git commit -a -m "Fix merge conflict in test.foo"

Git 通常在合并后自动提交，但当它检测到无法自行解决的冲突时，它会应用它找到的所有补丁，剩下的留给您手动解决和提交。Git Merge 手册页、Git- SVN 速成课程或此博客条目可能会阐明它应该如何工作。

编辑： 请参阅下面的帖子，您实际上不必自己复制文件，但可以使用

git checkout --ours -- path/to/file.txt
git checkout --theirs -- path/to/file.txt

选择所需文件的版本。只有当您想要混合使用两个版本时，才需要复制/编辑文件。

请将 mipadis 答案标记为正确答案。

如何在 android 中以编程方式设置图像视图的色调？

Arya

2023-02-18 04:17

需要为图像视图设置色调...我按以下方式使用它：

imageView.setColorFilter(R.color.blue,android.graphics.PorterDuff.Mode.MULTIPLY);

但它并没有改变...



解答

更新 ： @ADev 在他的回答中
有更新的解决方案，但他的解决方案需要更新的支持库 - 25.4.0 或更高版本。

您可以通过以下方式在代码中很容易地更改色调：

imageView.setColorFilter(Color.argb(255, 255, 255, 255));// 白色色调

如果你想要颜色那么

imageView.setColorFilter(ContextCompat.getColor(context, R.color.COLOR_YOUR_COLOR), android.graphics.PorterDuff.Mode.MULTIPLY);

对于矢量绘图

imageView.setColorFilter(ContextCompat.getColor(context, R.color.COLOR_YOUR_COLOR), android.graphics.PorterDuff.Mode.SRC_IN);


更多建议

大多数答案指的是 using setColorFilterwhich 不是最初要求的。

用户@Tad的答案是正确的，但它只适用于 API 21+。

要在所有 Android 版本上设置色调，请使用ImageViewCompat：

ImageViewCompat.setImageTintList(imageView, ColorStateList.valueOf(yourTint));

请注意，yourTint在这种情况下必须是“color int”。如果你有像 这样的颜色资源R.color.blue，你需要先加载 color int：

ContextCompat.getColor(context, R.color.blue);
