我怎样才能在 Git 中只存储分阶段的更改？

Arya

2023-02-17 18:34

有没有办法可以只存储我的分阶段更改？我遇到问题的场景是我在给定时间处理了几个错误，并且有几个未暂存的更改。我希望能够单独暂存这些文件，创建我的 .patch 文件，并将它们隐藏起来，直到代码获得批准。这样，当它被批准时，我可以隐藏我的整个（当前）会话，弹出那个错误并推送代码。

我会以错误的方式解决这个问题吗？我是否误解了 git 如何以其他方式工作来简化我的流程？



解答

这个答案被接受但很旧。 现在你可以git stash -S看到https://stackoverflow.com/a/70231955/683157

原始答案： 是的，可以使用 DOUBLE STASH

  1. 暂存所有需要存储的文件。
  2. 运行git stash --keep-index。 此命令将创建一个包含所有 更改（ 已暂存和未暂存 ）的存储，但会将已暂存的更改保留在您的工作目录中（仍处于已暂存状态）。
  3. 运行git stash push -m "good stash"（添加-u标志以在存储中包含新文件）
  4. 现在你"good stash"只有 暂存文件 。

现在，如果您在存储之前需要未暂存的文件，只需应用第一个存储（ 创建的那个--keep-index），现在您可以删除存储到的文件"good stash"。

享受


更多建议

git stash push -S

自 2.35 以来的旧
git stash push --staged#

仅存储当前暂存的更改。这类似于基本的git commit，除了状态被提交到存储而不是当前分支。

老的

使用最新的 git 你可以使用--patch选项

git stash push --patch   # since 2.14.6

git stash save --patch   # for older git versions

git 会询问您文件中的每个更改是否添加到存储中。
你只要回答y或n

**DOUBLE STASH的 旧 UPD
别名： ****

git config --global alias.stash-staged '!bash -c "git stash --keep-index; git stash push -m "staged" --keep-index; git stash pop stash@{1}"'

现在您可以暂存文件，然后运行git stash-staged​​.
结果， 您暂存的文件将保存到 stash 中 。

如果您不想保留暂存文件并希望将它们移动到存储中。然后你可以添加另一个别名并运行git move-staged：

git config --global alias.move-staged '!bash -c "git stash-staged;git commit -m "temp"; git stash; git reset --hard HEAD^; git stash pop"'

在 .CPP 文件中存储 C++ 模板函数定义

Arya

2023-02-17 18:33

我有一些模板代码，我更愿意将其存储在 CPP 文件中，而不是在标头中内联。我知道只要您知道将使用哪些模板类型就可以做到这一点。例如：

.h文件

class foo
{
public:
    template <typename T>
    void do(const T& t);
};

.cpp 文件

template <typename T>
void foo::do(const T& t)
{
    // Do something with t
}

template void foo::do<int>(const int&);
template void foo::do<std::string>(const std::string&);

请注意最后两行 - foo::do 模板函数仅用于 int 和 std::strings，因此这些定义意味着应用程序将链接。

我的问题是 - 这是一个令人讨厌的 hack 还是可以与其他编译器/链接器一起使用？目前我只将此代码与 VS2008 一起使用，但希望移植到其他环境。



解答

您描述的问题可以通过在标头中定义模板或通过您上面描述的方法来解决。

我建议阅读C++ FAQ Lite中的以下几点：

  • 为什么我不能将模板类的定义与其声明分开并将其放入 .cpp 文件中？
  • 如何避免模板函数出现链接器错误？
  • C++ 关键字导出如何帮助解决模板链接器错误？

他们详细介绍了这些（和其他）模板问题。


更多建议

对于此页面上的其他人想知道显式模板专业化（或至少在 VS2008 中）的正确语法是什么（我也是），其以下...

在你的 .h 文件中......

template<typename T>
class foo
{
public:
    void bar(const T &t);
};

在你的 .cpp 文件中

template <class T>
void foo<T>::bar(const T &t)
{ }

// Explicit template instantiation
template class foo<int>;

localStorage 值的最大大小是多少？

Arya

2023-02-17 18:33

由于localStorage（当前）仅支持字符串作为值，并且为了做到这一点，对象需要在存储之前进行字符串化（存储为 JSON 字符串），因此是否存在关于值长度的定义限制。

有谁知道是否有适用于所有浏览器的定义？



解答

引用维基百科关于网络存储的文章：

Web 存储可以简单地视为对 cookie 的改进，提供更大的存储容量（ Google Chrome 中每个来源 10 MB（ https://plus.google.com/u/0/+FrancoisBeaufort/posts/S5Q9HqDB8bh），Mozilla Firefox和 Opera；Internet Explorer 中每个存储区域 10 MB）和更好的编程接口。

还引用了John Resig 的一篇文章[发表于 2007 年 1 月]：

储存空间

这意味着，使用 DOM 存储，您拥有比强加于 Cookie 的典型用户代理限制多得多的存储空间。但是，提供的数量没有在规范中定义，用户代理也没有有意义地广播。

如果您查看 Mozilla 源代码，我们会发现 5120KB 是整个域的默认存储大小。与典型的 2KB cookie 相比，这为您提供了更多的工作空间。

然而，这个存储区域的大小可以由用户自定义 （因此 5MB 的存储区域不能保证，也不暗示）和用户代理（例如 Opera，可能只提供 3MB - 但只有时间会证明一切。 )


更多建议

实际上 Opera 没有 5MB 的限制。当应用程序需要更多时，它会增加限制。用户甚至可以为域选择“无限存储”。

您可以自己轻松测试 localStorage 限制/配额。
