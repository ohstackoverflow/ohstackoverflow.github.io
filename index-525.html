npx 是一个 npm 包运行器（x 可能代表 eXecute）。使用 npx 的 一种常见方法是临时或试用下载并运行包。

如何使用“*ngIf else”？

Arya

2023-02-17 09:12

*ngIf else我正在使用 Angular，我想在此示例中使用（自版本 4 起可用）：

<div *ngIf="isValid">
  content here ...
</div>

<div *ngIf="!isValid">
 other content here...
</div>

我怎样才能实现相同的行为ngIf else？



解答

角度 4 和 5 ：

使用else：

<div *ngIf="isValid;else other_content">
    content here ...
</div>

<ng-template #other_content>other content here...</ng-template>

您还可以使用then else：

<div *ngIf="isValid;then content else other_content">here is ignored</div>
<ng-template #content>content here...</ng-template>
<ng-template #other_content>other content here...</ng-template>

或者then单独：

<div *ngIf="isValid;then content"></div>
<ng-template #content>content here...</ng-template>

<ng-template>: 是 Angular 自己根据 MDN<template>实现的标签：

HTML<template>元素是一种用于保存客户端内容的机制，这些内容不会在页面加载时呈现，但随后可能会在运行时使用 JavaScript 实例化。


更多建议

在 Angular 4.xx 中

您可以通过四种方式使用ngIf来实现简单的 if - else 过程：

  1. 只需使用 If

    <div *ngIf="isValid">
If isValid is true

  2. 将If 与 Else 一起 使用（请注意 templateName ）

    <div *ngIf="isValid; else templateName">
If isValid is true

    If isValid is false

  3. 使用 If with Then （请注意 templateName ）

    <div *ngIf="isValid; then templateName">

    If isValid is true

  4. 将If 与 Then 和 Else 一起 使用 ****

    <div *ngIf="isValid; then thenTemplateName else elseTemplateName">
Here is never showing

    If isValid is true

    If isValid is false

在 SQL Server 中删除记录后重置标识种子

Arya

2023-02-17 09:12

我已将记录插入到 SQL Server 数据库表中。该表定义了一个主键，自动增量身份种子设置为“是”。这样做主要是因为在 SQL Azure 中，每个表都必须定义一个主键和标识。

但由于我必须从表中删除一些记录，这些表的身份种子将受到干扰，索引列（自动生成，增量为 1）将受到干扰。

删除记录后如何重置身份列，以便该列按数字升序排列？

身份列不用作数据库中任何地方的外键。



解答

管理命令DBCC CHECKIDENT用于重置身份计数器。命令语法是：

DBCC CHECKIDENT (table_name [, { NORESEED | { RESEED [, new_reseed_value ]}}])
[ WITH NO_INFOMSGS ]

例子：

DBCC CHECKIDENT ('[TestTable]', RESEED, 0);
GO

它在以前版本的 Azure SQL 数据库中不受支持，但现在受支持。

感谢Solomon Rutzky，该命令的文档现已修复。
