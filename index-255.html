我可以将多个 MySQL 行连接到一个字段中吗？

Arya

2023-02-17 02:17

使用MySQL，我可以做类似的事情：

SELECT hobbies FROM peoples_hobbies WHERE person_id = 5;

我的输出：

shopping
fishing
coding

但我只想要 1 行，1 列：

预期输出：

shopping, fishing, coding

原因是我从多个表中选择多个值，在所有连接之后我得到的行比我想要的多得多。

我在MySQL Doc上寻找过一个函数，它看起来不像CONCATorCONCAT_WS函数接受结果集。

那么这里有人知道该怎么做吗？



解答

您可以使用GROUP_CONCAT：

SELECT person_id,
   GROUP_CONCAT(hobbies SEPARATOR ', ')
FROM peoples_hobbies
GROUP BY person_id;

正如 Ludwig 在他的评论中所说，您可以添加DISTINCT运算符以避免重复：

SELECT person_id,
   GROUP_CONCAT(DISTINCT hobbies SEPARATOR ', ')
FROM peoples_hobbies
GROUP BY person_id;

正如 Jan 在他们的评论中所说，您还可以在使用 imploding 之前对值进行排序ORDER BY：

SELECT person_id, 
       GROUP_CONCAT(hobbies ORDER BY hobbies ASC SEPARATOR ', ')
FROM peoples_hobbies
GROUP BY person_id;

正如 Dag 在他的评论中所说，结果有 1024 字节的限制。要解决此问题，请在查询之前运行此查询：

SET group_concat_max_len = 2048;

当然，您可以2048根据自己的需要进行更改。要计算和分配值：

SET group_concat_max_len = CAST(
                     (SELECT SUM(LENGTH(hobbies)) + COUNT(*) * LENGTH(', ')
                           FROM peoples_hobbies
                           GROUP BY person_id) AS UNSIGNED);


更多建议

查看GROUP_CONCAT您的 MySQL 版本 (4.1) 是否支持它。有关详细信息，请参阅文档。

它看起来像：

  SELECT GROUP_CONCAT(hobbies SEPARATOR ', ') 
  FROM peoples_hobbies 
  WHERE person_id = 5 
  GROUP BY 'all';

如何找到数字数组的总和

Arya

2023-02-17 02:16

给定一个数组[1, 2, 3, 4]，我如何找到它的元素的总和？（在这种情况下，总和为10。）

我认为$.each可能会有用，但我不确定如何实现它。



解答

推荐（使用默认值减少）

Array.prototype.reduce可用于遍历数组，将当前元素值添加到先前元素值的总和中。

console.log(
  [1, 2, 3, 4].reduce((a, b) => a + b, 0)
)
console.log(
  [].reduce((a, b) => a + b, 0)
)

没有默认值

你得到一个 TypeError

console.log(
  [].reduce((a, b) => a + b)
)

在 ES6 的箭头函数之前

console.log(
  [1,2,3].reduce(function(acc, val) { return acc + val; }, 0)
)

console.log(
  [].reduce(function(acc, val) { return acc + val; }, 0)
)

非数字输入

如果非数字是可能的输入，您可能想要处理它？

console.log(
  ["hi", 1, 2, "frog"].reduce((a, b) => a + b)
)

let numOr0 = n => isNaN(n) ? 0 : n

console.log(
  ["hi", 1, 2, "frog"].reduce((a, b) => 
    numOr0(a) + numOr0(b))
)

不推荐的 危险 评估使用

我们可以使用eval来执行 JavaScript 代码的字符串表示。使用 Array.prototype.join 函数将数组转换为字符串，我们将 [1,2,3] 更改为“1+2+3”，计算结果为 6。

console.log(
  eval([1,2,3].join('+'))
)

//This way is dangerous if the array is built
// from user input as it may be exploited eg:

eval([1,"2;alert('Malicious code!')"].join('+'))

当然，显示警报并不是可能发生的最糟糕的事情。我将其包括在内的唯一原因是作为 Ortund 问题的答案，因为我认为它没有得到澄清。

自定义 HTTP 标头：命名约定

Arya

2023-02-17 02:16

我们的一些用户要求我们在我们发送给他们的请求的 HTTP 标头 中包含与他们的帐户相关的数据，甚至是他们从我们的 API 获得的响应。 在命名 、 格式 等方面添加自定义 HTTP 标头的一般约定是什么？

另外，请随意发布您在网络上偶然发现的这些的任何巧妙用法；我们正在尝试使用最好的作为目标来实现这一点 :)



解答

建议 是他们的名字以“X-” 开头 。例如X-Forwarded-For，X-Requested-With。RFC 2047的第 5 节中也提到了这一点。

更新 1 ：2011 年 6 月，发布了第一份IETF 草案， 反对 对非标准标头使用“X-”前缀的建议。原因是当以“X-”为前缀的非标准标头成为标准时，删除“X-”前缀会破坏向后兼容性，迫使应用程序协议支持这两个名称（例如，x-gzip&gzip现在是等效的）。所以，官方的建议是在没有“X-”前缀的情况下 合理地 命名它们。

更新 2 ：2012 年 6 月，弃用使用“X-”前缀的建议已正式成为RFC 6648。以下是相关的引用：

3. 给新参数创造者的建议

...

  1. 不应在其参数名称前加上“X-”或类似结构。

4. 给协议设计者的建议

...

  1. 不应禁止注册带有“X-”前缀或类似构造的参数。

  2. 不得规定具有“X-”前缀或类似构造的参数需要被理解为非标准化。

  3. 不得规定没有“X-”前缀或类似构造的参数需要被理解为标准化。

请注意，“不应该”（“不鼓励”）与“不得”（“禁止”）不同，另请参阅RFC 2119以了解有关这些关键字的其他规范。换句话说，您可以继续使用“X-”前缀的标头，但官方不再推荐它，您绝对不能将它们记录为公共标准。

摘要 ：

  • 官方的建议是在没有“X-”前缀的情况下 合理地命名它们
  • 您可以继续使用“X-”前缀的标头，但官方不再推荐它，您绝对不能将它们记录为公共标准


更多建议

这个问题值得重读。实际提出的问题与 CSS 属性中的供应商前缀不同，在这种情况下，面向未来并考虑供应商支持和官方标准是合适的。实际提出的问题更类似于选择 URL 查询参数名称。没有人应该关心它们是什么。但是，自定义名称的名称间距是完全有效的——也是常见的，也是正确的——要做的事情。

理由：
它是关于 开发人员之间的约定，用于自定义的、特定于应用程序的标头 ——“ 与他们的帐户相关的数据 ”——这与供应商、标准机构或第三方实施的协议无关，除了开发人员有问题的只是需要避免服务器、代理或客户端可能有其他预期用途的标头名称。因此，给出的“X-Gzip/Gzip”和“X-Forwarded- For/Forwarded-For”示例没有实际意义。提出的问题是关于私有 API 上下文中的约定，类似于 URL 查询参数命名约定。这是偏好和名称间距的问题；担心没有“X”的任何代理或供应商都支持“X-ClientDataFoo”

“X-”前缀没有什么特别或神奇的地方，但它有助于清楚地表明它是一个自定义标头。事实上，RFC-6648 等有助于支持使用“X-”前缀的案例，因为——随着 HTTP 客户端和服务器的供应商放弃前缀——你的特定应用程序、私有 API、个人数据—— passing-mechanism 正在变得更好地隔离与少量官方保留标题名称的名称空间冲突。也就是说，我个人的偏好和建议是更进一步，例如“X-ACME- ClientDataFoo”（如果您的小部件公司是“ACME”）。

恕我直言，IETF 规范不足以具体回答 OP 的问题，因为它无法区分完全不同的用例：(A) 供应商一方面引入新的全球适用功能，如“Forwarded- For”，与 (B)应用程序开发人员向/从客户端和服务器传递特定于应用程序的字符串。该规范仅涉及前者 (A)。这里的问题是 (B) 是否有约定。有。它们涉及按字母顺序将参数分组在一起，并将它们与许多与标准相关的类型 (A) 标头分开。使用“X-”或“X-ACME-”前缀对 (B) 来说既方便又合法，并且与 (A) 不冲突。越多供应商停止对 (A) 使用“X-”，(B) 就会变得越清晰。

示例：
谷歌（在各种标准机构中占有一席之地）——截至今天，20141102 对我的回答进行了轻微编辑——目前使用“X-Mod-Pagespeed”来指示其 Apache 模块的版本参与转换给定的响应。是否真的有人建议 Google 应该使用“Mod-Pagespeed”而不使用“X-”，和/或请求 IETF 支持它的使用？

摘要：
如果您在您的应用程序中使用自定义 HTTP 标头（有时作为 cookie 的适当替代方案）来将数据传入/传出您的服务器，并且这些标头明确地不打算在您的上下文之外使用应用程序，使用“X-”或“X-FOO-”前缀将它们命名为间距是一种合理且常见的约定。
