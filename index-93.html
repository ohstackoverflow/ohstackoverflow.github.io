如何创建一个像链接一样的 HTML 按钮？

Arya

2023-02-16 21:07

如何创建一个像链接一样的 HTML 按钮？因此，单击该按钮会将用户重定向到一个页面。

我希望它易于访问，并且 URL 中的额外字符或参数最少。



解答

HTML

纯 HTML 方法是将其放在一个属性中<form>，您可以在其中指定所需的目标 URL action。

<form action="https://google.com">
    <input type="submit" value="Go to Google" />
</form>

如有必要，在表单上设置 CSSdisplay: inline;以使其与周围的文本保持一致。除了<input type="submit">上面的示例，您还可以使用<button type="submit">. 唯一的区别是该<button>元素允许子元素。

您本能地希望能够使用<button href="https://google.com">与元素类似的东西，但不幸的是不，根据HTML 规范<a>，这个属性不存在。

CSS

如果允许使用 CSS，只需使用<a>您设计的样式，使其看起来像一个使用appearance属性的按钮（仅在 Internet Explorer 中不受支持）。

<a href="https://google.com" class="button">Go to Google</a>



a.button {
    -webkit-appearance: button;
    -moz-appearance: button;
    appearance: button;

    text-decoration: none;
    color: initial;
}

或者从众多 CSS 库中选择一个，例如Bootstrap。

<a href="https://google.com" class="btn btn-primary">Go to Google</a>

JavaScript

如果允许 JavaScript，请设置window.location.href.

<input type="button" onclick="location.href='https://google.com';" value="Go to Google" />

除了<input type="button">上面的示例，您还可以使用<button>. 唯一的区别是该<button>元素允许子元素。


更多建议

<button onclick="location.href='http://www.example.com'" type="button">
         www.example.com</button>

请注意，该type="button"属性很重要，因为它的缺失值默认是 Submit Button state。

深度克隆对象

Arya

2023-02-16 21:07

我想做类似的事情：

MyObject myObj = GetMyObj(); // Create and fill a new object
MyObject newObj = myObj.Clone();

然后对未反映在原始对象中的新对象进行更改。

我并不经常需要这个功能，所以当有必要时，我会创建一个新对象，然后单独复制每个属性，但它总是让我觉得有更好或更优雅的处理方式情况。

我如何克隆或深复制一个对象，以便可以修改克隆的对象而不会在原始对象中反映任何更改？



解答

一种方法是实现ICloneable接口（在此处描述，所以我不会反省），这是我不久前在代码项目上找到的一个很好的深度克隆对象复制器，并将其合并到我们的代码中。正如其他地方提到的，它要求您的对象是可序列化的。

using System;
using System.IO;
using System.Runtime.Serialization;
using System.Runtime.Serialization.Formatters.Binary;

/// <summary>
/// Reference Article http://www.codeproject.com/KB/tips/SerializedObjectCloner.aspx
/// Provides a method for performing a deep copy of an object.
/// Binary Serialization is used to perform the copy.
/// </summary>
public static class ObjectCopier
{
    /// <summary>
    /// Perform a deep copy of the object via serialization.
    /// </summary>
    /// <typeparam name="T">The type of object being copied.</typeparam>
    /// <param name="source">The object instance to copy.</param>
    /// <returns>A deep copy of the object.</returns>
    public static T Clone<T>(T source)
    {
        if (!typeof(T).IsSerializable)
        {
            throw new ArgumentException("The type must be serializable.", nameof(source));
        }

        // Don't serialize a null object, simply return the default for that object
        if (ReferenceEquals(source, null)) return default;

        using var Stream stream = new MemoryStream();
        IFormatter formatter = new BinaryFormatter();
        formatter.Serialize(stream, source);
        stream.Seek(0, SeekOrigin.Begin);
        return (T)formatter.Deserialize(stream);
    }
}

这个想法是它序列化您的对象，然后将其反序列化为一个新对象。好处是当对象变得太复杂时，您不必担心克隆所有内容。

如果您更喜欢使用 C# 3.0 的新扩展方法，请将方法更改为具有以下签名：

public static T Clone<T>(this T source)
{
   // ...
}

现在方法调用简单地变成了objectBeingCloned.Clone();.

编辑 （2015 年 1 月 10 日）我想重新审视这个，提到我最近开始使用 (Newtonsoft) Json 来做这个，它应该更轻，并且避免了 [Serializable] 标签的开销。（ 注意 @atconway 在评论中指出私有成员不会使用 JSON 方法克隆）

/// <summary>
/// Perform a deep Copy of the object, using Json as a serialization method. NOTE: Private members are not cloned using this method.
/// </summary>
/// <typeparam name="T">The type of object being copied.</typeparam>
/// <param name="source">The object instance to copy.</param>
/// <returns>The copied object.</returns>
public static T CloneJson<T>(this T source)
{            
    // Don't serialize a null object, simply return the default for that object
    if (ReferenceEquals(source, null)) return default;

    // initialize inner objects individually
    // for example in default constructor some list property initialized with some values,
    // but in 'source' these items are cleaned -
    // without ObjectCreationHandling.Replace default constructor values will be added to result
    var deserializeSettings = new JsonSerializerSettings {ObjectCreationHandling = ObjectCreationHandling.Replace};

    return JsonConvert.DeserializeObject<T>(JsonConvert.SerializeObject(source), deserializeSettings);
}


更多建议

我想要一个克隆器，用于主要由基元和列表组成的非常简单的对象。如果您的对象是开箱即用的 JSON 可序列化对象，那么此方法就可以解决问题。这不需要修改或实现克隆类的接口，只需要像 JSON.NET 这样的 JSON 序列化程序。

public static T Clone<T>(T source)
{
    var serialized = JsonConvert.SerializeObject(source);
    return JsonConvert.DeserializeObject<T>(serialized);
}

另外，您可以使用此扩展方法

public static class SystemExtension
{
    public static T Clone<T>(this T source)
    {
        var serialized = JsonConvert.SerializeObject(source);
        return JsonConvert.DeserializeObject<T>(serialized);
    }
}

CSS 中的 `margin` 和 `padding` 有什么区别？

Arya

2023-02-16 21:06

margin和CSS之间有什么区别padding？在什么样的情况下：

  • 两者都有效。
  • 只有margin合适的。
  • 只有padding合适的。



解答

TL;DR： 默认情况下，我在任何地方都使用边距，除非我有边框或背景并且想增加可见框内的空间。

对我来说，padding 和 margin 之间最大的区别是垂直边距会自动折叠，而 padding 不会。

考虑两个元素一个在另一个之上，每个元素的填充为1em。此填充被认为是元素的一部分并且始终被保留。

所以你最终会得到第一个元素的内容，然后是第一个元素的填充，然后是第二个元素的填充，然后是第二个元素的内容。

因此，这两个元素的内容将最终分开2em。

现在用 1em 边距替换该填充。边距被认为在元素之外，相邻项目的边距将重叠。

所以在这个例子中，你将得到第一个元素的内容，然后是1em组合边距的内容，然后是第二个元素的内容。所以这两个元素的内容只是1em相差而已。

当你知道你想说1em一个元素周围的间距时，这真的很有用，不管它旁边是什么元素。

另外两个很大的区别是填充包含在点击区域和背景颜色/图像中，但不包含边距。

div.box > div { height: 50px; width: 50px; border: 1px solid black; text-align: center; }
div.padding > div { padding-top: 20px; }
div.margin > div { margin-top: 20px; }


<h3>Default</h3>
<div class="box">
  <div>A</div>
  <div>B</div>
  <div>C</div>
</div>

<h3>padding-top: 20px</h3>
<div class="box padding">
  <div>A</div>
  <div>B</div>
  <div>C</div>
</div>

<h3>margin-top: 20px; </h3>
<div class="box margin">
  <div>A</div>
  <div>B</div>
  <div>C</div>
</div>


更多建议

边距在块元素的外部，而填充在内部。

  • 使用边距将块与其外部的东西分开
  • 使用填充将内容从块的边缘移开。
