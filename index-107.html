以下是否会使对象满足枚举在 JavaScript 中具有的所有特征？就像是：

my.namespace.ColorEnum = {
  RED : 0,
  GREEN : 1,
  BLUE : 2
}

// later on

if(currentColor == my.namespace.ColorEnum.RED) {
  // whatever
}

还是有其他方法可以做到这一点？



解答

从 1.8.5 开始可以密封和冻结对象，所以将上面的定义为：

const DaysEnum = Object.freeze({"monday":1, "tuesday":2, "wednesday":3, ...})

或者

const DaysEnum = {"monday":1, "tuesday":2, "wednesday":3, ...}
Object.freeze(DaysEnum)

瞧！JS 枚举。

但是，这不会阻止您将不需要的值分配给变量，这通常是枚举的主要目标：

let day = DaysEnum.tuesday
day = 298832342 // goes through without any errors

Makefile 中 .PHONY 的用途是什么？

Arya

2023-02-16 21:35

.PHONYMakefile 中的含义是什么？我经历过这个，但它太复杂了。

有人可以用简单的术语向我解释吗？



解答

默认情况下，Makefile 目标是“文件目标”——它们用于从其他文件构建文件。Make 假设它的目标是一个文件，这使得编写 Makefile 相对容易：

foo: bar
  create_one_from_the_other foo bar

但是，有时您希望 Makefile 运行不代表文件系统中的物理文件的命令。很好的例子是共同目标“clean”和“all”。情况可能并非如此，但您 可能clean在主目录中有一个名为的文件。在这种情况下，Make 会感到困惑，因为默认情况下clean目标将与该文件相关联，并且 Make 只会在文件似乎不是最新的依赖项时运行它。

这些特殊目标称为 假 目标，您可以明确告诉 Make 它们与文件无关，例如：

.PHONY: clean
clean:
  rm -rf *.o

现在make clean将按预期运行，即使您确实有一个名为clean.

就 Make 而言，虚假目标只是一个总是过时的目标，因此无论何时您询问make <phony_target>，它都会运行，独立于文件系统的状态。一些make通常是虚假的常见目标是：all、、、、、、、、。install``clean``distclean``TAGS``info``check


更多建议

假设您有install目标，这在 makefile 中很常见。如果您不 使用 ，.PHONY并且一个名为 的文件install存在于与 Makefile 相同的目录中，则 什么都不make install做。这是因为 Make 将规则解释为“执行某某配方以创建名为”的文件。由于文件已经存在，并且它的依赖关系没有改变，所以什么也不会做。 __install

但是，如果您创建install目标 PHONY，它会告诉 make 工具该目标是虚构的，并且 make 不应期望它创建实际文件。因此它不会检查install文件是否存在，这意味着：a）如果文件确实存在，它的行为将不会改变，并且 b）stat()不会调用 extra。

通常，Makefile 中所有不生成与目标名称同名的输出文件的目标都应该是 PHONY。这通常包括all、install、clean、distclean等。

获取屏幕大小、当前网页和浏览器窗口

Arya

2023-02-16 21:35

我怎样才能得到windowWidth, windowHeight, pageWidth, pageHeight, screenWidth, screenHeight, pageX, pageY, screenX,screenY这将适用于所有主流浏览器？



解答

您可以使用 jQuery 获取窗口或文档的大小：

// Size of browser viewport.
$(window).height();
$(window).width();

// Size of HTML document (same as pageHeight/pageWidth in screenshot).
$(document).height();
$(document).width();

对于屏幕尺寸，您可以使用screen对象：

window.screen.height;
window.screen.width;


更多建议

这有你需要知道的一切：获取视口/窗口大小

但简而言之：

var win = window,
    doc = document,
    docElem = doc.documentElement,
    body = doc.getElementsByTagName('body')[0],
    x = win.innerWidth || docElem.clientWidth || body.clientWidth,
    y = win.innerHeight|| docElem.clientHeight|| body.clientHeight;
alert(x + ' × ' + y);

const width  = window.innerWidth || document.documentElement.clientWidth ||
