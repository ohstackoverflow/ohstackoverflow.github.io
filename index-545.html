有没有办法 git checkout 以前的分支？

Arya

2023-02-17 09:42

我有点想要cd -git 的等价物。如果我在分店master结帐foo，我希望能够输入类似“git checkout -返回”的内容master，并能够再次输入“返回”之类的内容foo。

有这样的东西存在吗？会不会很难实施？



解答

来自1.6.2 的发行说明

@{-1}是一种引用您所在的最后一个分支的方法。这
不仅在需要对象名称的地方被接受，而且在任何需要分支名称的地方都被接受，并且就像您键入分支名称一样。
例如git branch --track mybranch @{-1}, git merge @{-1}, 和
git rev-parse --symbolic-full-name @{-1}将按预期工作。

和

git checkout -是 的简写git checkout @{-1}。

要查看以前结帐的列表：

i=0; while [ $? -eq 0 ]; do i=$((i+1)); echo -n "$i. "; git rev-parse --symbolic-full-name @{-$i} 2> /dev/null; done

这个 Bash 单行脚本并不完美，但它应该适用于大多数情况。 请注意，有时数字可能会跳过。

提示： 您可以将其添加.bashrc为function.


更多建议

现在最简单的方法是：

git checkout -

...这是以下的别名：

git checkout @{-1}

如果您想了解更多相关信息，我在这里写了整篇文章：检查 Git 中的上一个分支。

我们什么时候需要用大括号括住 shell 变量？

Arya

2023-02-17 09:42

在shell脚本中，我们什么时候使用{}when expanding variables？

例如，我看到了以下内容：

var=10        # Declare variable

echo "${var}" # One use of the variable
echo "$var"   # Another use of the variable

有显着差异，还是只是风格？一个比另一个更受欢迎吗？



解答

在这个特定的例子中，它没有区别。但是，如果您想扩展字符串中的变量，则{}in很有用${}``foo

"${foo}bar"

since"$foobar"会扩展由 标识的变量foobar。

在以下情况下也无条件需要大括号：

  • 扩展数组元素，如${array[42]}
  • 使用参数扩展操作，如${filename%.*}（删除扩展）
  • 将位置参数扩展到 9 以上："$8 $9 ${10} ${11}"

在任何地方都这样做，而不仅仅是在可能有歧义的情况下， 可以 被认为是良好的编程习惯。这既是为了保持一致性，也是为了避免像 那样的意外$foo_$bar.jpg，在这种情况下，下划线成为变量名称的一部分在视觉上并不明显。


更多建议

变量的声明和赋值有$和没有{}。你必须使用

var=10

分派。为了从变量中读取（换句话说，“扩展”变量），您必须使用$.

$var      # use the variable
${var}    # same as above
${var}bar # expand var, and append "bar" too
$varbar   # same as ${varbar}, i.e expand a variable called varbar, if it exists.

这有时让我感到困惑 - 在其他语言中，我们以相同的方式引用变量，无论它是在赋值的左侧还是右侧。但是 shell 脚本是不同的，它$var=10并不像您想象的那样！

更多建议

如何在 Flexbox 中垂直和水平居中元素

以下是两种通用的居中解决方案。

一个用于垂直对齐的弹性项目 ( flex-direction: column)，另一个用于水平对齐的弹性项目 ( flex-direction: row)。

在这两种情况下，居中的 div 的高度都可以是可变的、未定义的、未知的，等等。居中的 div 的高度无关紧要。

这是两者的 HTML：

<div id="container"><!-- flex container -->

    <div class="box" id="bluebox"><!-- flex item -->
        <p>DIV #1</p>
    </div>

    <div class="box" id="redbox"><!-- flex item -->
        <p>DIV #2</p>
    </div>

</div>

CSS （不包括装饰样式）

当 flex 项目垂直堆叠时：

#container {
    display: flex;           /* establish flex container */
    flex-direction: column;  /* make main axis vertical */
    justify-content: center; /* center items vertically, in this case */
    align-items: center;     /* center items horizontally, in this case */
    height: 300px;
}

.box {
    width: 300px;
    margin: 5px;
    text-align: center;     /* will center text in <p>, which is not a flex item */
}

flex-direction根据上面的代码调整规则。

#container {
    display: flex;
    flex-direction: row;     /* make main axis horizontal (default setting) */
    justify-content: center; /* center items horizontally, in this case */
    align-items: center;     /* center items vertically, in this case */
    height: 300px;
}

将弹性项目的内容居中

flex 格式化上下文 的范围仅限于父子关系。flex 容器的后代不参与 flex 布局，并且会忽略 flex 属性。本质上，flex 属性不能继承到子项之外。

因此，您总是需要将display: flexor应用于display: inline-flex父元素，以便将 flex 属性应用于子元素。

为了垂直和/或水平居中文本或包含在弹性项目中的其他内容，使项目成为（嵌套的）弹性容器，并重复居中规则。

.box {
    display: flex;
    justify-content: center;
    align-items: center;        /* for single line flex container */
    align-content: center;      /* for multi-line flex container */
}

此处有更多详细信息：How to vertically align text inside a flexbox?

或者，您可以应用于margin: auto弹性项目的内容元素。

p { margin: auto; }

在此处了解弹性auto边距：对齐弹性项目的方法（请参阅方框#56）。

将多行弹性项目居中

当 flex 容器有多行时（由于换行），该align-content属性对于横轴对齐是必需的。

从规格：

8.4. 包装柔性线：align-content财产

align-content当横轴上有额外空间时，该属性会在弹性容器内对齐弹性容器的行，类似于如何justify- content在主轴内对齐各个项目。 请注意，此属性对单行弹性容器没有影响。
