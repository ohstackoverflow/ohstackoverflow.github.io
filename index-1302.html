将元组扩展为参数

Arya

2023-02-18 06:22

假设我有一个像这样的功能：

def myfun(a, b, c):
    return (a * 2, b + c, c + b)

给定一个元组some_tuple = (1, "foo", "bar")，我如何使用some_tuple调用myfun来获取结果(2, "foobar", "barfoo")

我知道可以定义myfun它直接接受元组，但我想调用现有的myfun.

另请参阅：（星号）和 *（双星号）运算符在函数调用中的含义是什么？.



解答

myfun(*some_tuple)完全按照您的要求进行。运算*符简单地解压缩元组（或任何可迭代对象）并将它们作为位置参数传递给函数。阅读有关解包参数的更多信息。


更多建议

请注意，您还可以扩展参数列表的一部分：

myfun(1, *("foo", "bar"))

如何显示在 MySQL 上执行的最后一个查询？

Arya

2023-02-18 06:22

是否有任何查询/方式来显示在所有 服务器上执行的最后查询？



解答

对于 MySQL >= 5.1.12 的用户，您可以在运行时全局控制此选项：

  1. 执行SET GLOBAL log_output = 'TABLE';
  2. 执行SET GLOBAL general_log = 'ON';
  3. 看看表mysql.general_log

如果您更喜欢输出到文件而不是表格：

  1. SET GLOBAL log_output = "FILE"; 默认值 。
  2. SET GLOBAL general_log_file = "/path/to/your/logfile.log";
  3. SET GLOBAL general_log = 'ON';

我更喜欢这种方法来编辑 .cnf 文件，因为：

  1. 您没有编辑my.cnf文件并可能永久打开日志记录
  2. 您不会在文件系统周围四处寻找查询日志 - 甚至更糟，因为需要完美的目的地而分心。/var/log /var/data/log /opt /home/mysql_savior/var
  3. 您不必重新启动服务器并中断与它的任何当前连接。
  4. 重新启动服务器会让您回到开始的位置（默认情况下日志仍然关闭）

有关详细信息，请参阅MySQL 5.1 参考手册 - 服务器系统变量 - general_log


更多建议

您可以为此类诊断启用一般查询日志。不过，通常您不会在生产服务器上记录所有 SELECT 查询，这是性能杀手。

编辑您的 MySQL 配置，例如 /etc/mysql/my.cnf - 查找或添加这样的行

[mysqld]
log = /var/log/mysql/mysql.log

重新启动 mysql 以获取该更改，现在您可以

tail -f /var/log/mysql/mysql.log

嘿 presto，您可以在查询进入时观看它们。

“尝试加载格式不正确的程序”，即使平台相同

Arya

2023-02-18 06:21

我在 64 位系统上从 32 位非托管 DLL 调用函数。我得到的是：

BadImageFormatException：尝试加载格式不正确的程序。（HRESULT 异常：0x8007000B）

起初，我将我的项目设置为 Any CPU 平台，所以我将它们都更改为 x86，但仍然出现此错误。这真的是我所知道的唯一解决方法。

DLL 没有损坏或其他任何东西，因为我可以将它们与其他程序一起使用（我没有源代码）。我想也许它没有找到依赖关系，但我检查了一下，它们都在那里。DllNotFoundException另外，在那种情况下它不会抛出 a 吗？

我还可以做些什么？在您说“改为使用 64 位非托管 DLL”之前，让我指出没有。;)



解答

不知何故，配置管理器中的 构建 复选框未选中我的可执行文件，因此它仍在使用旧的 Any CPU 构建运行。在我修复该问题后，Visual Studio 抱怨它无法调试程序集，但通过重新启动修复了该问题。
