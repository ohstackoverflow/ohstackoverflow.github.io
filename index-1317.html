尝试远程添加 Git 存储库时收到“fatal: Not a git repository”

Arya

2023-02-18 06:44

我正在关注本教程。一切正常，直到我在我的本地机器上运行它（在$VARIABLE用它们的实际值替换 s 之后）：

git remote add nfsn ssh://$USERNAME@$SERVER/home/private/git/$REPONAME.git

我收到以下错误消息：

致命：不是 git 存储库（或任何父目录）：.git

我如何通过这一步？



解答

您是否初始化了一个本地 Git 存储库，应该将此远程添加到其中？

你的本地目录有.git文件夹吗？

尝试git init。


更多建议

如果您在当前工作目录不在 Git 存储库中时尝试使用 Git 命令，则会出现此错误。这是因为，默认情况下，Git 会查找存储库目录（在项目根目录内？），正如我对“除非我在项目目录中，否则 Git 不会显示日志”的回答所.git指出的：

根据Linux Kernel Git 官方文档，默认情况下GIT_DIR[an environment variable] 是否设置为查找目录（在当前工作目录中？）：.git

如果GIT_DIR设置了环境变量，则它会指定要使用的路径，而不是.git存储库基础的默认路径。

您要么需要cd进入存储库/工作副本，要么首先没有初始化或克隆存储库，在这种情况下，您需要在要放置存储库的目录中初始化存储库：

git init

或者克隆一个存储库

git clone <remote-url>
cd <repository>

Grunt watch error - 等待...致命错误：watch ENOSPC

Arya

2023-02-18 06:43

Waiting...Fatal error: watch ENOSPC为什么我在运行 watch 任务时得到了？我该如何解决这个问题？



解答

在做了一些研究后找到了解决方案。运行以下命令。

echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf && sudo sysctl -p

对于 Arch Linux，将此行添加到 /etc/sysctl.d/99-sysctl.conf：

fs.inotify.max_user_watches=524288


更多建议

任何时候你需要跑去sudo something ...解决问题，你都应该停下来想想发生了什么。虽然这里接受的答案是完全有效的，但它是在治疗症状而不是问题。有点相当于购买更大的边箱来解决以下问题：错误，无法将更多垃圾装载到小马身上。小马已经装了这么多垃圾，小马累得晕倒了。

另一种方法（可能类似于从小马身上取出多余的垃圾并放入垃圾场）是运行：

npm dedupe

然后去祝贺你自己让小马开心。
