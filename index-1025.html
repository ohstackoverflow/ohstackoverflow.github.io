带有泛型的 Typescript 箭头函数的语法是什么？

Arya

2023-02-17 23:10

打字稿手册目前没有关于箭头功能的内容。可以使用以下语法对普通函数进行通用类型化：示例：

function identity<T>(arg: T): T {
    return arg;
}

箭头函数的语法是什么？



解答

语言规范在 p.64f 上说

< T > ( ... ) => { ... } 形式的构造可以解析为带有类型参数的箭头函数表达式或应用于没有类型参数的箭头函数的类型断言。它被解析为前者[..]

例子：

// helper function needed because Backbone-couchdb's sync does not return a jqxhr
let fetched = <
           R extends Backbone.Collection<any> >(c:R) => {
               return new Promise(function (fulfill, reject) {
                   c.fetch({reset: true, success: fulfill, error: reject})
               });
           };


更多建议

如果你在一个.tsx文件中，你不能只写<T>，但这有效：

const foo = <T, >(x: T) => x;

与黑客攻击相反extends {}，这种黑客攻击至少保留了意图。

OS X：相当于 Linux 的 wget

Arya

2023-02-17 23:10

我怎样才能在普通 OS X 系统上从 Un*x shell 脚本执行 HTTP GET？（安装第三方软件不是一种选择，因为它必须在我无法控制的许多不同系统上运行）。

例如，如果我在本地启动 Mercurial 服务器执行 hg serve ：

... $ hg serve

然后，从具有 wget 命令的 Linux 我执行 wget：

... $  wget http://127.0.0.1:8000
--2010-12-31 22:18:25--  http://127.0.0.1:8000/
Connecting to 127.0.0.1:8000... connected.
HTTP request sent, awaiting response... 200 Script output follows
Length: unspecified [text/html]
Saving to: `index.html

在我启动“hg serve” 命令的终端上，我确实可以看到 HTTP GET 成功了：

127.0.0.1 - - [30/Dec/2010 22:18:17] "GET / HTTP/1.0" 200 -

因此，在 Linux 上，从 shell 脚本执行 HTTP GET 的一种方法是使用 wget （当然，如果安装了该命令）。

还有什么其他方法可以做相当于 wget 的事情？特别是，我正在寻找可以在原装 OS X 安装上运行的东西。



解答

以下本机命令将起作用：

curl http://127.0.0.1:8000 -o outfile

请注意，curl默认情况下不遵循重定向。要告诉它这样做，请添加-L到参数列表。


更多建议

brew install wget

Homebrew是 OSX 的包管理器，类似于 yum、apt-get、choco、emerge 等。请注意，您还需要安装 Xcode 和命令行工具。事实上，任何在 OSX 中使用命令行的人都会想要安装这些东西。

如果您不能或不想使用自制软件，您还可以：

手动安装 wget：

curl -# "http://ftp.gnu.org/gnu/wget/wget-1.17.1.tar.xz" -o "wget.tar.xz"
tar xf wget.tar.xz
cd wget-1.17.1
./configure --with-ssl=openssl -with-libssl-prefix=/usr/local/ssl && make -j8 && make install

或者，使用 bash 别名：

function _wget() { curl "${1}" -o $(basename "${1}") ; };
alias wget='_wget'

Integrated Security = True 和 Integrated Security = SSPI 之间有什么区别？

Arya

2023-02-17 23:09

我有两个使用 Integrated Security 的应用程序。一个在连接字符串中分配Integrated Security = true，另一个设置Integrated Security = SSPI。

SSPI集成安全性之间和true在集成安全性上下文中有什么区别？



解答

根据微软的说法，它们是同一回事。

当 时false，在连接中指定了用户 ID 和密码。如果为真，则使用当前 Windows 帐户凭据进行身份验证。
可识别的值为true, false, yes, no, 和sspi（强烈推荐），相当于true.


更多建议

Integrated Security=true;不适用于所有 SQL 提供程序，它在与提供程序一起使用时抛出异常OleDb。

所以基本上Integrated Security=SSPI;是首选，因为可以与SQLClient&OleDB提供商一起使用。

这是根据MSDN 的完整语法集 - 连接字符串语法 (ADO.NET)
