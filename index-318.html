简单的面试问题变得更难了：给定数字 1..100，找出正好给定 k 的缺失数字

Arya

2023-02-17 03:54

前段时间我有一次有趣的求职面试经历。这个问题开始真的很简单：

Q1 ：我们有一个包，里面装着数字1，，，…… ，。每个数字只出现一次，所以有 100 个数字。现在从包里随机抽取一个号码。找到丢失的号码。2``3``100

当然，我以前听过这个面试问题，所以我很快按照以下思路回答：

A1 ：嗯，数字之和1 + 2 + 3 + … + N是(N+1)(N/2)（参见维基百科：算术级数之和）。对于N = 100，总和是5050。

因此，如果袋子中存在所有数字，则总和将恰好为5050. 由于少了一个数，和会比这个少，差就是那个数。O(N)所以我们可以在时间和空间中找到那个缺失的数字O(1)。

至此我以为自己做的很好了，没想到问题突然来了个转折：

Q2 ：是的，但是现在如果缺少 两个数字，你会怎么做？

我以前从未见过/听说过/考虑过这种变化，所以我惊慌失措，无法回答这个问题。面试官坚持要知道我的思维过程，所以我提到也许我们可以通过与预期产品进行比较来获得更多信息，或者在从第一次通过中收集到一些信息后进行第二次通过等，但我真的只是在拍摄在黑暗中，而不是实际上有一条通向解决方案的清晰路径。

面试官确实试图鼓励我，说有第二个方程确实是解决问题的一种方法。在这一点上，我有点不高兴（因为事先不知道答案），并询问这是否是一种通用的（读作：“有用的”）编程技术，或者它是否只是一个技巧/陷阱答案。

面试官的回答让我很吃惊：你可以概括这个技术来找到 3 个缺失的数字。事实上，您可以将其概括为找到 k 个 缺失的数字。

Qk ：如果袋子里恰好缺少 k 个数字，你将如何有效地找到它？

这是几个月前的事了，我还是不明白这是什么技术。显然有一个Ω(N)时间下限，因为我们必须至少扫描所有数字一次，但面试官坚持认为求解技术的 时间 和 空间O(N)复杂度（减去时间输入扫描）是在 k 而不是 N 中定义的。

所以这里的问题很简单：

  • 你会如何解决 Q2 ？
  • 你会如何解决 Q3 ？
  • 你会如何解决 Qk ？

说明

  • 通常有 N 个 数字来自 1.. N ，而不仅仅是 1..100。
  • 我不是在寻找明显的基于集合的解决方案，例如使用位 set，通过指定位的值对每个数字的存在/不存在进行编码，因此O(N)在额外空间中使用位。我们负担不起与 N 成比例的任何额外空间。
  • 我也不是在寻找明显的排序优先方法。这个和基于集合的方法在面试中值得一提（它们很容易实现，并且取决于 N ，可以非常实用）。我正在寻找 Holy Grail 解决方案（实现起来可能实用也可能不实用，但仍然具有所需的渐近特性）。

因此，同样，您当然必须扫描 中的输入O(N)，但您只能捕获少量信息（根据 k 而不是 N 定义），然后必须以某种方式找到 k 缺失的数字。



解答

这是Dimitris Andreou 链接的摘要。

记住 i 次幂的总和，其中 i=1,2,..,k。这将问题简化为求解方程组

a 1 + a 2 + ... + a k = b 1

a 1 2 + a 2 2 + ... + a k 2 = b 2

...

a 1 k + a 2 k + ... + a k k = b k

使用牛顿恒等式，知道 b i允许计算

c 1 = a 1 + a 2 + ... a k

c 2 = a 1 a 2 + a 1 a 3 + ... + a k-1 a k

...

c k = a 1 a 2 ... a k

如果展开多项式 (xa 1 )...(xa k )，则系数将恰好为 c 1 , ..., c k - 请参阅Viète 的公式。由于每个多项式因子都是唯一的（多项式环是欧几里得域），这意味着 a i是唯一确定的，直到排列。

这结束了记忆力足以恢复数字的证明。对于常量 k，这是一个很好的方法。

然而，当 k 变化时，计算 c 1 ,...,c k的直接方法非常昂贵，因为例如 c k是所有缺失数字的乘积，大小为 n!/(nk)!。为了克服这个问题，在 Z q域中执行计算，其中 q 是一个素数，使得 n <= q < 2n - 它存在于Bertrand 的假设中。不需要更改证明，因为公式仍然成立，并且多项式的因式分解仍然是唯一的。您还需要一种用于有限域分解的算法，例如Berlekamp或Cantor- Zassenhaus的算法。

常量 k 的高级伪代码：

  • 计算给定数的 i 次方
  • 减去未知数的 i 次方的总和。称总和为 b i。
  • 使用牛顿恒等式计算 b i的系数；称他们为 c i。基本上，c 1 = b 1；c 2 = (c 1 b 1 - b 2 )/2; 请参阅维基百科了解确切的公式
  • 因式分解多项式 x k -c 1 x k-1 + ... + c k。
  • 多项式的根是所需的数字 a 1 , ..., a k。

对于不同的 k，使用例如 Miller-Rabin 找到一个素数 n <= q < 2n，并执行所有数字减少模 q 的步骤。

编辑：此答案的先前版本指出，可以使用特征为 2 (q=2^(log n)) 的有限域代替Z q ，其中 q 是质数。事实并非如此，因为牛顿公式要求除以 k 以内的数字。


更多建议

您会通过阅读 Muthukrishnan 的几页 - 数据流算法：难题 1：查找缺失数字来找到 它。 它准确地显示了您正在寻找的概括 。可能这就是您的面试官阅读的内容以及他提出这些问题的原因。

另请参阅 sdcvvc 的直接相关答案，其中还包括伪代码（万岁！无需阅读那些棘手的数学公式 :)）（谢谢，干得好！）。

如何检查地图是否包含 Go 中的键？

Arya

2023-02-17 03:53

我知道我可以遍历m地图

for k, v := range m { ... }

并寻找一个键，但是是否有更有效的方法来测试键在地图中的存在？



解答

以下是检查地图是否包含键的方法。

val, ok := myMap["foo"]
// If the key exists
if ok {
    // Do something
}

这将初始化两个变量。val是映射中“foo”的值（如果存在），如果不存在则为“零值”（在本例中为空字符串）。ok是一个如果密钥存在bool则将被设置为。true

如果需要，您可以将其缩短为单行。

if val, ok := myMap["foo"]; ok {
    //do something here
}

Go 允许您在 if 语句中的条件（注意分号）之前放置一个初始化语句。val这样做的结果是和的范围ok将限于 if 语句的主体，如果您只需要在那里访问它们，这将很有帮助。


更多建议

除了The Go Programming Language Specification，您还应该阅读Effective Go。在地图部分，他们说，除其他外：

尝试使用映射中不存在的键获取映射值将返回映射中条目类型的零值。例如，如果映射包含整数，查找不存在的键将返回 0。集合可以实现为值类型为 bool 的映射。将map条目设置为true将值放入集合中，然后通过简单的索引进行测试。

attended := map[string]bool{
    "Ann": true,
    "Joe": true,
    ...
}

if attended[person] { // will be false if person is not in the map
    fmt.Println(person, "was at the meeting")
}

有时您需要区分缺失条目和零值。是否有“UTC”条目或 0 因为它根本不在地图中？您可以使用多重赋值的形式进行区分。

var seconds int
var ok bool
seconds, ok = timeZone[tz]

由于显而易见的原因，这被称为“逗号 ok”成语。在此示例中，如果存在 tz，则秒数将被适当设置并且 ok 将为真；如果不是，seconds 将被设置为零并且 ok 将为 false。这是一个将它与一个漂亮的错误报告放在一起的函数：

func offset(tz string) int {
    if seconds, ok := timeZone[tz]; ok {
        return seconds
    }
    log.Println("unknown time zone:", tz)
    return 0
}

要测试地图中的存在而不用担心实际值，您可以使用空白标识符 (_) 代替值的常用变量。

_, present := timeZone[tz]

功能分支变基后 Git 推送被拒绝

Arya

2023-02-17 03:53

好的，我以为这是一个简单的 git 场景，我错过了什么？

我有一个master分支和一个feature分支。我做了一些工作master，一些feature，然后更多master。我最终得到这样的结果（字典顺序意味着提交的顺序）：

A--B--C------F--G  (master)
       \    
        D--E  (feature)

我可以毫无问题地git push origin master保持远程master更新，也可以git push origin feature（打开时feature）为我的工作维护远程备份feature。到现在为止，我们都很好。

但是现在我想在 master 上的提交feature之上进行 rebase F--G，所以我git checkout feature和git rebase master. 还好。现在我们有：

A--B--C------F--G  (master)
                 \
                  D'--E'  (feature)

问题： 当我想备份新的变基feature分支时git push origin feature， 推送被拒绝， 因为树由于变基而改变了。这只能通过git push --force origin feature.

--force我讨厌在不确定自己是否需要它的情况下使用它。那么，我需要它吗？变基 是否必然 意味着下一个push应该是--force完整的？

这个功能分支不与任何其他开发人员共享，所以我对强制 推送没有任何问题，我不会丢失任何数据，这个问题更概念化。



解答

问题是git push假设远程分支可以快速转发到您的本地分支，也就是说本地和远程分支之间的所有区别都在于本地在末尾有一些新的提交，如下所示：

Z--X--R         <- origin/some-branch (can be fast-forwarded to Y commit)
       \        
        T--Y    <- some-branch

当您执行git rebase提交时，D 和 E 将应用于新的基础并创建新的提交。这意味着在变基之后你有类似的东西：

A--B--C------F--G--D'--E'   <- feature-branch
       \  
        D--E                <- origin/feature-branch

在那种情况下，远程分支无法快速转发到本地。虽然，理论上本地分支可以合并到远程（显然在这种情况下你不需要它），但是因为git push只执行快进合并它会抛出错误。

而--force选项所做的只是忽略远程分支的状态并将其设置为您正在推送的提交。所以简单地用 localgit push --force origin feature-branch覆盖。origin/feature-branch``feature-branch

master在我看来，只要你是唯一一个在该分支上工作的人，将功能分支变基并强制将它们推回到远程存储库就可以了。


更多建议

而不是使用-f或--force开发人员应该使用

--force-with-lease

为什么？因为它会检查远程分支的变化，这绝对是个好主意。假设 James 和 Lisa 在同一个功能分支上工作，而 Lisa 已经推送了一个提交。James 现在重新定位他的本地分支，并在尝试推送时被拒绝。当然 James 认为这是由于 rebase 和 uses 导致的--force，并且会重写 Lisa 的所有更改。如果 James 使用过，--force-with-lease他会收到警告，提示有人提交了其他人。我不明白为什么有人会在 rebase 之后使用--force而不是push 。--force-with-lease
