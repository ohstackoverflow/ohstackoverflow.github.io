我们如何制作 xkcd 风格的图表？

Arya

2023-02-17 15:10

显然，人们已经想出了如何在 Mathematica和LaTeX 中制作xkcd风格的图表。我们可以在 R 中做到吗？Ggplot2-ers？geom_xkcd 和/或 theme_xkcd？

我猜在基础图形中，par(xkcd=TRUE)？我该怎么做？

作为 ggplot2 中的第一个尝试（如下所示更优雅），将 jitter 参数添加到一条线上可以产生很棒的手绘效果。所以 -

ggplot(mapping=aes(x=seq(1,10,.1), y=seq(1,10,.1))) + 
  geom_line(position="jitter", color="red", size=2) + theme_bw()

这是一个很好的例子——但轴和字体看起来更棘手。不过，字体似乎已解决（如下）。处理轴的唯一方法是将它们消隐并手工绘制吗？有更优雅的解决方案吗？特别是在ggplot2中，是否可以修改新主题系统中的element_line以采用类似jitter的参数？



解答

您可能需要考虑以下软件包：

xkcd 包：以 XKCD 风格绘制 ggplot2 图形。

library(xkcd)
vignette("xkcd-intro")

  • 条形图：


更多建议

沿着与其他一些答案相同的思路思考，我已经“取消绘制”图表并添加了 x 轴标签位置的灵活性（这在 xkcd 中似乎很常见）和任意标签图表。

请注意，我在加载 Humor Sans 字体并手动将其下载到工作目录时遇到了一些问题。

和代码...

library(ggplot2)
library(extrafont)

### Already have read in fonts (see previous answer on how to do this)
loadfonts()

### Set up the trial dataset 
data <- NULL
data$x <- seq(1, 10, 0.1)
data$y1 <- sin(data$x)
data$y2 <- cos(data$x)
data$xaxis <- -1.5

data <- as.data.frame(data)

### XKCD theme
theme_xkcd <- theme(
    panel.background = element_rect(fill="white"), 
    axis.ticks = element_line(colour=NA),
    panel.grid = element_line(colour="white"),
    axis.text.y = element_text(colour=NA), 
    axis.text.x = element_text(colour="black"),
    text = element_text(size=16, family="Humor Sans")
    )

 ### Plot the chart
 p <- ggplot(data=data, aes(x=x, y=y1))+
      geom_line(aes(y=y2), position="jitter")+
      geom_line(colour="white", size=3, position="jitter")+
      geom_line(colour="red", size=1, position="jitter")+
      geom_text(family="Humor Sans", x=6, y=-1.2, label="A SIN AND COS CURVE")+
      geom_line(aes(y=xaxis), position = position_jitter(h = 0.005), colour="black")+
      scale_x_continuous(breaks=c(2, 5, 6, 9), 
      labels = c("YARD", "STEPS", "DOOR", "INSIDE"))+labs(x="", y="")+
      theme_xkcd

ggsave("xkcd_ggplot.jpg", plot=p, width=8, height=5)

将 NumPy 数组转储到 csv 文件中

Arya

2023-02-17 15:09

如何以人类可读的格式将 NumPy 数组转储到 csv 文件中？



解答

numpy.savetxt将数组保存到文本文件。

import numpy
a = numpy.asarray([ [1,2,3], [4,5,6], [7,8,9] ])
numpy.savetxt("foo.csv", a, delimiter=",")


更多建议

你可以使用pandas。它确实需要一些额外的内存，因此并非总是可行，但它非常快速且易于使用。

import pandas as pd 
pd.DataFrame(np_array).to_csv("path/to/file.csv")

如果您不想要标题或索引，请使用to_csv("/path/to/file.csv", header=None, index=None)

如何保护 Python 代码不被用户阅读？

Arya

2023-02-17 15:08

我正在用 Python 开发一个软件，将分发给我雇主的客户。我的雇主想要使用限时许可文件来限制软件的使用。

如果我们分发.py文件甚至.pyc文件，将很容易（反编译和）删除检查许可证文件的代码。

另一个方面是我的雇主不希望我们的客户阅读代码，担心代码或至少是“新颖的想法”可能被盗。

有没有好的方法来处理这个问题？



解答

Python 是一种字节码编译解释型语言，很难锁定。即使您使用像py2exe这样的 exe 打包程序，可执行文件的布局也是众所周知的，并且 Python 字节码也很容易理解。

通常在这种情况下，您必须做出权衡。保护代码到底有多重要？那里有真正的秘密吗（例如银行转账的对称加密密钥），或者您只是偏执狂？选择可以让您以最快的速度开发出最好的产品的语言，并现实地看待您的新颖想法的价值。

如果你决定你真的需要安全地执行许可证检查，将它写成一个小的 C 扩展，这样许可证检查代码就可以非常困难（但并非不可能！）进行逆向工程，并将大部分代码留在 Python 中.
