如何在 C 中生成一个随机整数？

Arya

2023-02-17 18:37

C中有生成随机整数的函数吗？或者我必须使用第三方库吗？



解答

注意 ：不要用于rand()安全。如果您需要加密安全号码，请参阅此答案。

#include <time.h>
#include <stdlib.h>

srand(time(NULL));   // Initialization, should only be called once.
int r = rand();      // Returns a pseudo-random integer between 0 and RAND_MAX.

在 Linux 上，您可能更喜欢使用random 和 srandom。


更多建议

rand()中的函数返回<stdlib.h>一个介于 0 和 之间的伪随机整数RAND_MAX。您可以用来srand(unsigned int seed)设置种子。

%通常的做法是将运算符与 with 结合使用rand()以获得不同的范围（但请记住，这会在一定程度上影响一致性）。例如：

/* random int between 0 and 19 */
int r = rand() % 20;

如果你 真的 关心统一性，你可以这样做：

/* Returns an integer in the range [0, n).
 *
 * Uses rand(), and so is affected-by/affects the same seed.
 */
int randint(int n) {
  if ((n - 1) == RAND_MAX) {
    return rand();
  } else {
    // Supporting larger values for n would requires an even more
    // elaborate implementation that combines multiple calls to rand()
    assert (n <= RAND_MAX)

    // Chop off all of the values that would cause skew...
    int end = RAND_MAX / n; // truncate skew
    assert (end > 0);
    end *= n;

    // ... and ignore results from rand() that fall above that limit.
    // (Worst case the loop condition should succeed 50% of the time,
    // so we can expect to bail out of this loop pretty quickly.)
    int r;
    while ((r = rand()) >= end);

    return r % n;
  }
}

如果我自己抛出 JavaScript 异常（例如，throw "AArrggg"），我如何获取堆栈跟踪（在 Firebug 中或其他方式中）？现在我刚收到消息。

编辑 ：正如下面许多人发布的那样，可以获取 JavaScript 异常的堆栈跟踪，但 我 想获取异常的堆栈跟踪。例如：

function foo() {
    bar(2);
}
function bar(n) {
    if (n < 2)
        throw "Oh no! 'n' is too small!"
    bar(n-1);
}

调用时，我想获得一个堆栈跟踪，其中包括对, ,foo的调用。foo``bar``bar



解答

编辑 2（2017 年）：

在所有现代浏览器中，您可以简单地调用：console.trace(); （MDN 参考）

编辑 1（2013 年）：

正如对原始问题的评论中指出的那样，更好（更简单）的解决方案是使用对象stack的属性，Error如下所示：

function stackTrace() {
    var err = new Error();
    return err.stack;
}

这将生成如下输出：

DBX.Utils.stackTrace@http://localhost:49573/assets/js/scripts.js:44
DBX.Console.Debug@http://localhost:49573/assets/js/scripts.js:9
.success@http://localhost:49573/:462
x.Callbacks/c@http://localhost:49573/assets/js/jquery-1.10.2.min.js:4
x.Callbacks/p.fireWith@http://localhost:49573/assets/js/jquery-1.10.2.min.js:4
k@http://localhost:49573/assets/js/jquery-1.10.2.min.js:6
.send/r@http://localhost:49573/assets/js/jquery-1.10.2.min.js:6

提供调用函数的名称以及 URL、它的调用函数等。

原版（2009 年）：

此代码段的修改版本可能会有所帮助：

function stacktrace() { 
  function st2(f) {
    return !f ? [] : 
        st2(f.caller).concat([f.toString().split('(')[0].substring(9) + '(' + f.arguments.join(',') + ')']);
  }
  return st2(arguments.callee.caller);
}


更多建议

Chrome/Chromium 和其他使用 V8 的浏览器，以及 Firefox，有一个方便的接口来通过对象stack的属性获取堆栈跟踪Error：

    try {
        // Code throwing an exception
        throw new Error();
    } catch(e) {
        console.log(e.stack);
    }

请参阅V8 文档中的详细信息

如何使方法返回类型通用？

Arya

2023-02-17 18:36

考虑这个例子（OOP 书籍中的典型例子）：

我有一个Animal班级，每个班级Animal都有很多朋友。
和子类Dog，如Duck、Mouse等，它们添加了特定的行为bark()，如quack()等。

这是Animal课程：

public class Animal {
    private Map<String,Animal> friends = new HashMap<>();

    public void addFriend(String name, Animal animal){
        friends.put(name,animal);
    }

    public Animal callFriend(String name){
        return friends.get(name);
    }
}

这是一些带有大量类型转换的代码片段：

Mouse jerry = new Mouse();
jerry.addFriend("spike", new Dog());
jerry.addFriend("quacker", new Duck());

((Dog) jerry.callFriend("spike")).bark();
((Duck) jerry.callFriend("quacker")).quack();

有什么办法可以对返回类型使用泛型来摆脱类型转换，这样我就可以说

jerry.callFriend("spike").bark();
jerry.callFriend("quacker").quack();

下面是一些初始代码，返回类型作为从未使用过的参数传递给方法。

public<T extends Animal> T callFriend(String name, T unusedTypeObj){
    return (T)friends.get(name);        
}

有没有办法在运行时找出返回类型而不使用额外的参数instanceof？或者至少通过传递类型的类而不是虚拟实例。
我知道泛型用于编译时类型检查，但是有解决方法吗？



解答

你可以callFriend这样定义：

public <T extends Animal> T callFriend(String name, Class<T> type) {
    return type.cast(friends.get(name));
}

然后这样称呼它：

jerry.callFriend("spike", Dog.class).bark();
jerry.callFriend("quacker", Duck.class).quack();

此代码的好处是不会生成任何编译器警告。当然，这实际上只是前通用时代投射的更新版本，并没有增加任何额外的安全性。


更多建议

你可以这样实现它：

@SuppressWarnings("unchecked")
public <T extends Animal> T callFriend(String name) {
    return (T)friends.get(name);
}

（是的，这是合法的代码；请参阅Java Generics: Generic type defined as return type only。）

返回类型将从调用者推断出来。但是，请注意@SuppressWarnings注释：它告诉您 此代码不是类型安全的 。你必须自己验证它，或者你可以ClassCastExceptions在运行时得到。

不幸的是，您使用它的方式（没有将返回值分配给临时变量），让编译器满意的唯一方法是这样调用它：

jerry.<Dog>callFriend("spike").bark();

虽然这可能比转换好一点，但您最好给类Animal一个抽象talk()方法，正如 David Schmitt 所说。
