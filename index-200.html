在 Android Studio 中重命名包

Arya

2023-02-17 00:43

您如何在基于 IntelliJ IDEA 的新 IDE Android Studio 中重命名包？

是否包含自动重构？

我想进行批量重构，但我不知道怎么做。我使用 Eclipse 工作了两年，在 Eclipse 中它是一个单击操作。



解答

在 Android Studio 中，你可以这样做：

例如，如果要更改com.example.app为my.awesome.game，则：

  1. 在您的 _ 项目面板_ 中，单击小齿轮图标 ( )

  2. 取消选中该Compact Empty Middle Packages选项

  1. 您的包目录现在将被分解为单独的目录

  2. 单独选择要重命名的每个目录，然后：

  3. 右键单击它

  4. 选择Refactor

  5. 点击Rename

  6. 在弹出对话框中，单击而Rename Package不是重命名目录

  7. 输入新名称并点击 Refactor

  8. 单击底部的 “Do Refactor”

  9. 花一分钟时间让 Android Studio 更新所有更改

  10. 注意：com在 Android Studio 中重命名时，可能会给出警告。在这种情况下，选择 全部重命名

  1. 现在打开您的 _ Gradle 构建文件_ （build.gradle- 通常app或mobile）。如果尚未自动更新，请将applicationIdin更新为您的新包名称和 Sync Gradle：defaultConfig

  1. 您可能需要更改package=清单中的属性。

  2. 清洁和重建。

  1. 完毕！ 不管怎样，Android Studio 需要让这个过程更简单一些。


更多建议

另一个好方法是：首先通过右键单击 Java 文件夹 → New → Package 创建一个具有所需名称的新包。

然后，选择所有类并将其拖到新包中。Android Studio 会到处重构包名。

最后，删除旧包。

完毕。

很重要：

如果使用此方法，则必须手动将 AndroidManifest.xml 和 build.gradle 文件更改为新包。

“javascript:void(0)”是什么意思？

Arya

2023-02-17 00:42 

<a href="javascript:void(0)" id="loginlink">login</a>

我见过href很多次这样的s，但我不知道那到底是什么意思。



解答

运算void符计算给定的表达式，然后返回undefined。

运算void符通常仅用于获取undefined原始值，通常使用“ void(0)”（相当于“ void 0”）。undefined在这些情况下，可以改用全局变量（假设它没有被分配给非默认值）。

此处提供了解释：voidoperator。

href您希望对链接执行此操作的原因是，通常javascript:URL 会将浏览器重定向到评估该 JavaScript 的结果的纯文本版本。但如果结果为undefined，则浏览器停留在同一页面上。void(0)只是一个简短的脚本，其计算结果为undefined.


更多建议

除了技术答案外，javascript:void还意味着作者做错了。

没有充分的理由使用javascript:伪 URL(*)。在实践中，如果任何人尝试“书签链接”、“在新选项卡中打开链接”等操作，都会导致混淆或错误。这种情况经常发生，现在人们已经习惯了中键点击换新标签：它看起来像一个链接，你想在新标签中阅读它，但事实证明它根本不是一个真正的链接，并在单击中键时给出不需要的结果，如空白页或 JS 错误。

<a href="#">是一个常见的替代方案，可以说它可能不那么糟糕。但是，您必须记住return false从您的onclick事件处理程序中防止链接被跟踪并向上滚动到页面顶部。

在某些情况下，可能会有指向链接的实际有用位置。例如，如果您有一个控件，您可以单击它打开一个以前隐藏的，使用链接到它<div id="foo">是有意义的。<a href="#foo">或者，如果有一种非 JavaScript 方式来做同样的事情（例如，'thispage.php?show=foo' 将 foo 设置为可见），您可以链接到它。

否则，如果链接仅指向某些脚本，则它不是真正的链接，不应如此标记。通常的方法是将、或不带an 添加onclick到 an 中，并以某种方式设置样式以明确您可以单击它。这就是 StackOverflow [在撰写本文时所做的；现在它使用]。<span>``<div>``<a>``href``href="#"

这样做的缺点是你失去了键盘控制，因为你不能跳到 span/div/bare-a 上或用空格激活它。这是否真的是一个缺点取决于元素打算采取什么样的行动。您可以通过tabIndex向元素添加 a 并监听 Space 按键来尝试模仿键盘交互性。但它永远不会 100% 重现真实的浏览器行为，尤其是因为不同的浏览器可以不同地响应键盘（更不用说非可视化浏览器了）。

如果你真的想要一个不是链接但可以通过鼠标或键盘正常激活的元素，你想要的是一个<button type="button">（或者<input type="button">同样好，对于简单的文本内容）。如果需要，您始终可以使用 CSS 重新设计它的样式，使其看起来更像一个链接而不是一个按钮。但是因为它的行为像一个按钮，所以你应该如何标记它。

(*: 无论如何，在站点创作中。显然它们对小书签很有用。javascript:伪 URL 是一种概念上的怪异：定位器不指向某个位置，而是调用当前位置内的活动代码。它们造成了巨大的安全性浏览器和 webapps 的问题，并且不应该由 Netscape 发明。）
