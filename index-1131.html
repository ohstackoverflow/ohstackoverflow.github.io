转置/解压缩功能（zip 的倒数）？

Arya

2023-02-18 02:13

我有一个 2 项元组列表，我想将它们转换为 2 个列表，其中第一个列表包含每个元组中的第一项，第二个列表包含第二项。

例如：

original = [('a', 1), ('b', 2), ('c', 3), ('d', 4)]
# and I want to become...
result = (['a', 'b', 'c', 'd'], [1, 2, 3, 4])

有没有内置函数可以做到这一点？



解答

在 2.x 中，zip是它自己的逆！前提是您使用特殊的 * 运算符。

>>> zip(*[('a', 1), ('b', 2), ('c', 3), ('d', 4)])
[('a', 'b', 'c', 'd'), (1, 2, 3, 4)]

这等效于将zip列表的每个元素作为单独的参数调用：

zip(('a', 1), ('b', 2), ('c', 3), ('d', 4))

除了参数直接传递给zip（在转换为元组之后），所以不需要担心参数的数量变得太大。

在 3.x 中，zip 返回一个惰性迭代器，但这被简单地转换了：

>>> list(zip(*[('a', 1), ('b', 2), ('c', 3), ('d', 4)]))
[('a', 'b', 'c', 'd'), (1, 2, 3, 4)]


更多建议

你也可以这样做

result = ([ a for a,b in original ], [ b for a,b in original ])

它 应该 更好地扩展。特别是如果 Python 在除非需要的情况下不扩展列表推导式方面做得很好的话。

（顺便说一下，它生成一个 2 元组（对）列表，而不是像那样的元组列表zip。）

如果生成器而不是实际列表是可以的，那么这样做：

result = (( a for a,b in original ), ( b for a,b in original ))

在您请求每个元素之前，生成器不会遍历列表，但另一方面，它们会保留对原始列表的引用。

寻找“cmake clean”命令来清除 CMake 输出

Arya

2023-02-18 02:13

正如make clean删除 makefile 生成的所有文件一样，我想对 CMake 做同样的事情。我经常发现自己手动浏览目录删除文件，如cmake_install.cmake和CMakeCache.txt，以及CMakeFiles文件夹。

cmake clean是否有类似自动删除所有这些文件的命令？理想情况下，这应该遵循当前目录文件中定义的递归结构CMakeLists.txt。



解答

CMake 3.X

CMake 3.X 提供了一个“干净”的目标。

cmake --build C:/foo/build/ --target clean

来自 3.0.2 的 CMake 文档：

--clean-first  = Build target 'clean' first, then build.
                 (To clean only, use --target 'clean'.)

CMake 2.X

cmake cleanCMake 版本 2.X 中没有

我通常将项目构建在一个文件夹中，例如“build”。所以如果我想make clean，我可以rm -rf build。

与根“CMakeLists.txt”位于同一目录中的“build”文件夹通常是一个不错的选择。要构建您的项目，您只需将 CMakeLists.txt 的位置作为参数提供给 cmake。例如：cd <location-of-cmakelists>/build && cmake ..。（来自@ComicSansMS）


更多建议

在 Git 无处不在的这些日子里，您可能会忘记 CMake 并使用git clean -d -f -x，它将删除所有不受源代码控制的文件。

有没有一种方法可以使用命令行 cURL 进行重定向？

Arya

2023-02-18 02:12

我知道在 php 脚本中：

curl_setopt($ch, CURLOPT_FOLLOWLOCATION, true);

将遵循重定向。有没有一种方法可以使用命令行 cURL 进行重定向？



解答

使用位置标头标志：

curl -L <URL>


更多建议

如前所述， 要遵循重定向， 您可以使用标志-L或--location：

curl -L http://www.example.com

但是， 如果你想限制重定向的数量 ，添加参数--max-redirs

--max-redirs <num>

设置允许的最大重定向关注数。如果使用-L, --location，则此选项可用于防止 curl 遵循“荒谬”的重定向。默认情况下，该限制设置为 50 次重定向。将此选项设置为 -1 以使其无限制。如果多次使用此选项，将使用最后一个。
