适用于操作系统的 Android Studio 中的代码格式化快捷方式

Arya

2023-02-17 10:52

我已经开始使用Android Studio进行开发。在 Eclipse 中我使用的是Ctrl++ ，但在 Android StudioShift中F它不起作用。这将是不同的。

如何跳转到.java文件中的任何方法？我在 Eclipse 中使用Ctrl+ 。O

是可用的么？代码格式化的快捷方式是什么？



解答

窗口：Ctrl+ Alt+L

Linux：Ctrl+ Shift+ Alt+L

macOS：Option+ Command+L

参考：Key Commands和此处是适用于Windows/Linux 用户和Mac 用户的所有命令。

由于 Rohit 在 Ubuntu 中遇到格式代码快捷方式的问题，这是由于在 Ubuntu 中使用Ctrl+ Alt+L键锁定屏幕。

我发现 Ubuntu 首先处理这个键盘快捷键。所以你应该将Ctrl+ Alt+L键盘快捷键绑定到其他东西，这样它就不会与 Ubuntu 冲突。

脚步

  1. 转到 系统工具 → 系统设置 → 键盘 → 快捷方式 选项卡 → 系统 → 锁定屏幕 。

  2. 选择New Accelerator... 行，然后按带有 Alpha 键的任意特殊键（例如Shift+ L）。您应该已经成功更改了键盘快捷键。

  3. 检查键盘快捷键现在是否可以在 Android Studio 中使用。

替代方法

Ctrl+ Alt+ Shift+ L（显示对话框）

  • Windows 和 Linux：转到菜单 文件 → 设置 → 键盘映射
  • macOS：转到菜单 Preferences → Keymap 并从下拉菜单中选择 Eclipse。

使用菜单栏

  • 选择要重新格式化的代码

  • 单击菜单栏中的 代码，然后选择 重新格式化代码 ，如下图所示：

识别关系和非识别关系有什么区别？

Arya

2023-02-17 10:51

我一直无法完全理解差异。你能描述这两个概念并使用真实世界的例子吗？



解答

  • 标识 关系 是指子表中一行的存在取决于父表中的一行。这可能会造成混淆，因为现在通常的做法是为子表创建一个伪键，而 不是 将外键设为子表主键的父部分。形式上，“正确”的做法是将外键作为子键的一部分。但逻辑关系是，孩子不能没有父母存在。

示例：APerson有一个或多个电话号码。如果他们只有一个电话号码，我们可以简单地将其存储在 的一列中Person。由于我们想要支持多个电话号码，我们创建了第二个表PhoneNumbers，其主键包括person_id引用该Person表。

我们可能认为电话号码属于一个人，即使它们被建模为单独表格的属性。这是一个强有力的线索，表明这是一种识别关系（即使我们没有字面上包含person_id在 的主键中PhoneNumbers）。

  • 非标识关系 是指父项的主键属性 不得 成为子项的主键属性。一个很好的例子是查找表，例如Person.state引用主键的外键States.state。Person是关于 的子表States。但是 in 中的一行Person不是由其state属性标识的。即state不是主键的一部分Person。

非标识关系可以是 optional 或 mandatory ，这意味着外键列分别允许 NULL 或不允许 NULL。

另见我对仍然对识别与非识别关系感到困惑的回答


更多建议

现实世界还有另一种解释：

一本书属于一个所有者，一个所有者可以拥有多本书。但是，这本书也可以在没有所有者的情况下存在，并且它的所有权可以从一个所有者更改为另一个所有者。一本书和所有者之间的关系是一种非识别关系。

然而，一本书是由一个作者写的，而这个作者可以写多本书。但是，这本书需要由作者撰写——没有作者就无法存在。因此，书与作者之间的关系是一种识别关系。

如何删除 Docker 镜像？

Arya

2023-02-17 10:51

我在 OS X 10.8.4（Mountain Lion）下的 Vagrant 下运行 Docker，每当我尝试删除保存的图像时，我都会收到错误消息：

$ docker rmi some-image-id
2013/07/15 hh:mm:ss unexpected JSON input

根据rmi帮助，正确的语法是docker rmi IMAGE [IMAGE...]，我不确定该怎么做。

如何删除图像？

$ docker version
Client version: 0.4.8
Server version: 0.4.8
Go version: go1.1



$docker info
Containers: 1
Images: 3

有趣的是，当我运行时docker ps，根本没有容器出现。运行docker images显示四 (4)base个图像和一 (1) 个node图像。



解答

尝试docker rmi node。那应该有效。

查看所有创建的容器就像docker ps -a.

要删除所有现有容器（不是图像！）运行docker rm $(docker ps -aq)


更多建议

以下是一些删除 docker 镜像/容器的方法：

删除单个图像

docker rmi image_name:version/image-id

删除所有图像

docker rmi $(docker images -qf "dangling=true")

杀死容器并删除它们：

docker rm $(docker kill $(docker ps -aq))

注意：替换kill为stop优雅关机

删除除“my-image”之外的所有图像

使用 grep 删除除my-image和之外的所有内容ubuntu

docker rmi $(docker images | grep -v 'ubuntu\|my-image' | awk {'print $3'})

或者 （没有 awk）

docker rmi $(docker images --quiet | grep -v $(docker images --quiet ubuntu:my-image))
