在 .NET 4.0 项目中引用 .NET 2.0 混合模式程序集需要哪些“附加配置”？

Arya

2023-02-18 05:30

我有一个项目，我想在其中使用一些 .NET 4.0 功能，但核心要求是我可以使用针对 2.X 编译的 System.Data.SQLite 框架。我看到提到这是可能的，例如这里接受的答案，但我不知道如何真正实现这一点。

当我在引用 2.X 程序集的同时尝试运行我的 4.0 项目时，我得到：

混合模式程序集是针对运行时版本“v2.0.50727”构建的，如果没有额外的配置信息，则无法在 4.0 运行时中加载。

需要什么“附加配置”？



解答

为了使用 CLR 2.0 混合模式程序集 ，您需要修改 App.Config 文件以包括：

<?xml version="1.0"?><configuration>  <startup useLegacyV2RuntimeActivationPolicy="true">    <supportedRuntime version="v4.0" sku=".NETFramework,Version=v4.0"/>  </startup></configuration>

关键是useLegacyV2RuntimeActivationPolicy国旗。这会导致 CLR 使用最新版本 (4.0) 加载您的混合模式程序集。没有这个，它不会工作。

请注意，这仅适用于混合模式 (C++/CLI) 程序集。您可以加载所有托管的 CLR 2 程序集，而无需在app.config.


更多建议

此论坛帖子位于 .NET Framework 开发人员中心。它可能会提供一些见解。

（添加到应用程序的配置文件。）

<configuration>
  <startup useLegacyV2RuntimeActivationPolicy="true">
    <supportedRuntime version="v4.0"/>
  </startup>
</configuration>

LLVM到底是什么？

Arya

2023-02-18 05:30

我一直听说 LLVM。它在 Perl 中，然后在 Haskell 中，然后有人在其他语言中使用它？它是什么？

  • 它与 GCC 到底有什么区别（观点 = 安全等）？



解答

LLVM 是一个用于构建、优化和生成中间和/或二进制机器代码的库。

LLVM 可以用作编译器框架，您可以在其中提供“前端”（解析器和词法分析器）和“后端”（将 LLVM 的表示形式转换为实际机器代码的代码）。

LLVM 还可以充当 JIT 编译器——它支持 x86/x86_64 和 PPC/PPC64 程序集生成，并具有旨在提高编译速度的快速代码优化。

不幸的是，自 2013 年起禁用，在演示页面上可以使用从 C 或 C++ 代码生成的 LLVM 机器代码。


更多建议

LLVM 的一个很好的总结是这样的：

在前端，您有 Perl 和许多其他高级语言。在后端，您拥有直接在机器上运行的本机代码。

中间是您的中间代码表示。如果每种高级语言都可以用这种 LLVM IR 格式表示，那么基于这种 IR 的分析工具就可以很容易地重用——这就是基本原理。

django OneToOneField 和 ForeignKey 有什么区别？

Arya

2023-02-18 05:29

OneToOneFieldDjango和有什么区别ForeignKey？



解答

The Definitive Guide to DjangoOneToOneField(SomeModel)之间和ForeignKey(SomeModel, unique=True)中所述的差异： __

一对一字段

一对一的关系。从概念上讲，这类似于ForeignKeywith unique=True，但关系的“反面”将直接返回单个对象。

与“反向”关系相反OneToOneField，ForeignKey“反向”关系返回 a QuerySet。

例子

例如，如果我们有以下两个模型（下面是完整的模型代码）：

  1. Car模型用途OneToOneField(Engine)
  2. Car2模型用途ForeignKey(Engine2, unique=True)

从内部python manage.py shell执行以下命令：

OneToOneField例子

>>> from testapp.models import Car, Engine
>>> c = Car.objects.get(name='Audi')
>>> e = Engine.objects.get(name='Diesel')
>>> e.car
<Car: Audi>

ForeignKey举例unique=True_

>>> from testapp.models import Car2, Engine2
>>> c2 = Car2.objects.get(name='Mazda')
>>> e2 = Engine2.objects.get(name='Wankel')
>>> e2.car2_set.all()
[<Car2: Mazda>]

型号代码

from django.db import models

class Engine(models.Model):
    name = models.CharField(max_length=25)

    def __unicode__(self):
        return self.name

class Car(models.Model):
    name = models.CharField(max_length=25)
    engine = models.OneToOneField(Engine)

    def __unicode__(self):
        return self.name

class Engine2(models.Model):
    name = models.CharField(max_length=25)

    def __unicode__(self):
        return self.name

class Car2(models.Model):
    name = models.CharField(max_length=25)
    engine = models.ForeignKey(Engine2, unique=True, on_delete=models.CASCADE)

    def __unicode__(self):
        return self.name


更多建议

AForeignKey是多对一的关系。因此，一个Car对象可能有很多Wheel. 因此，每个人Wheel都会有一个属于ForeignKey它Car的。AOneToOneField就像 的一个实例Engine，其中一个Car对象 最多 只有一个。
