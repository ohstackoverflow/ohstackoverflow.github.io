如何按类查找元素

Arya

2023-02-17 23:40

我在使用 Beautifulsoup 解析具有“class”属性的 HTML 元素时遇到问题。代码看起来像这样

soup = BeautifulSoup(sdata)
mydivs = soup.findAll('div')
for div in mydivs: 
    if (div["class"] == "stylelistrow"):
        print div

我在脚本完成“之后”在同一行收到错误。

File "./beautifulcoding.py", line 130, in getlanguage
  if (div["class"] == "stylelistrow"):
File "/usr/local/lib/python2.6/dist-packages/BeautifulSoup.py", line 599, in __getitem__
   return self._getAttrMap()[key]
KeyError: 'class'

我如何摆脱这个错误？



解答

您可以使用 BS3 优化搜索以仅查找具有给定类的那些 div：

mydivs = soup.find_all("div", {"class": "stylelistrow"})


更多建议

从文档中：

从 Beautiful Soup 4.1.2 开始，您可以使用关键字参数按 CSS 类进行搜索 class_：

soup.find_all("a", class_="sister")

在这种情况下是：

soup.find_all("div", class_="stylelistrow")

它也适用于：

soup.find_all("div", class_="stylelistrowone stylelistrowtwo")

如何从字符串中删除所有换行符

Arya

2023-02-17 23:39

我在 textarea 中有一个文本，我使用 .value 属性读出它。

现在我想Enter使用 .replace 和正则表达式从我的文本中删除所有换行符（当您按下时产生的字符），但是我如何在正则表达式中指示换行符？

如果那不可能，还有其他方法吗？



解答

换行符（更好：换行符）可以是回车（CR，\r在旧 Mac 上），换行符（LF，\n在 Unices 上，包括 Linux）或 CR 后跟 LF（\r\n在 WinDOS 上）。（与另一个答案相反，这与字符编码 无关 。）

RegExp因此，匹配所有变体的最有效文字是

/\r?\n|\r/

如果要匹配字符串中的所有换行符，请使用全局匹配，

/\r?\n|\r/g

分别。然后继续按照replace其他几个答案中建议的方法进行操作。（可能您 不想 删除换行符，而是用其他空格替换它们，例如空格字符，以便单词保持不变。）

为什么使用片段，何时使用片段而不是活动？

Arya

2023-02-17 23:39

在 Android API 11+ 中，谷歌发布了一个名为Fragment.

在视频中，谷歌建议尽可能（link1，link2），我们应该使用片段而不是活动，但他们没有解释确切原因。

片段的目的是什么以及它们的一些可能用途（除了一些可以通过简单视图/布局轻松实现的 UI 示例）？

我的问题是关于片段：

  1. 使用片段的目的是什么？
  2. 与使用活动/视图/布局相比，使用片段有哪些优点和缺点？

奖金问题：

  1. 你能给出片段的一些真正有趣的用途吗？谷歌在他们的视频中没有提到的东西？
  2. 在片段和包含它们的活动之间进行通信的最佳方式是什么？
  3. 使用片段时要记住的最重要的事情是什么？您的经验有什么提示和警告吗？



解答

1 & #2 使用片段的目的是什么？与使用活动/视图/布局相比，使用片段的优缺点是什么？

Fragments 是 Android 用于创建可重用用户界面的解决方案。您可以使用活动和布局（例如使用包含）实现一些相同的事情。然而; 片段被连接到 Android API，从 HoneyComb 开始。让我详细说明；

  • 的ActionBar。如果你想在上面使用选项卡来导航你的应用程序，你很快就会看到该ActionBar.TabListener界面为你提供了一个FragmentTransaction作为该onTabSelected方法的输入参数。您可能会忽略这一点，并做一些其他的聪明的事情，但您将反对 API，而不是使用它。

  • FragmentManager以非常巧妙的方式为您处理“后退” 。返回并不意味着回到上一个活动，就像常规活动一样。这意味着回到以前的片段状态。

  • 您可以使用 cool ViewPagerwith aFragmentPagerAdapter来创建滑动界面。该FragmentPagerAdapter代码比常规适配器更简洁，并且它控制各个片段的实例化。

  • 如果您在尝试为手机和平板电脑创建应用程序时使用 Fragments，您的生活将会轻松很多。由于片段与 Honeycomb+ API 紧密相关，您将希望在手机上使用它们以及重用代码。这就是兼容性库派上用场的地方。

  • 您甚至可以而且应该将片段用于仅适用于手机的应用程序。如果您考虑到便携性。我使用ActionBarSherlock和兼容性库来创建“ICS 外观”应用程序，这些应用程序看起来一直到 1.6 版都一样。您可以获得最新的功能ActionBar，例如带有选项卡、溢出、拆分操作栏、viewpager 等。

奖金 2

片段之间通信的最佳方式是意图。当您按下 Fragment 中的某些内容时，您通常会调用StartActivity()其中的数据。意图会传递到您启动的活动的所有片段。


更多建议

不确定你指的是什么视频，但我怀疑他们是说你应该使用片段而不是活动，因为它们不能直接互换。Dev Guide中其实有一个相当详细的条目，可以考虑阅读它以了解详细信息。

简而言之，片段存在于活动中，每个活动可以承载许多片段。与活动一样，它们具有特定的生命周期，与活动不同，它们不是顶级应用程序组件。片段的优势包括代码重用和模块化（例如，在许多活动中使用相同的列表视图），包括构建多窗格界面的能力（主要用于平板电脑）。主要缺点是（一些）增加了复杂性。您通常可以通过（自定义）视图以非标准且不太健壮的方式实现相同的目的。
