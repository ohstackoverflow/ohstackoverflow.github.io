我可以在 .gitconfig 中为自己指定多个用户吗？

Arya

2023-02-17 07:30

在我的 中~/.gitconfig，我在 下列出了我的个人电子邮件地址[user]，因为这是我想用于 Github 存储库的地址。

但是，我最近也开始使用 git 工作。我公司的 git repo 允许我提交，但是当它发出新变更集的公告时，它说它们来自 Anonymous，因为它无法识别我的电子邮件地址——至少，这是我的理论.gitconfig。

是否可以[user]在中指定多个定义.gitconfig？或者是否有其他方法可以覆盖.gitconfig某个目录的默认设置？在我的例子中，我检查了所有的工作代码~/worksrc/——有没有办法.gitconfig只为那个目录（及其子目录）指定一个？



解答

您可以将单个存储库配置为使用覆盖全局配置的特定用户/电子邮件地址。从 repo 的根目录运行

git config user.name "Your Name Here"
git config user.email your@email.com

而默认用户/电子邮件是在您的 ~/.gitconfig 中配置的

git config --global user.name "Your Name Here"
git config --global user.email your@email.com


更多建议

从 git 2.13 开始，可以使用新引入的Conditional includes来解决这个问题。

一个例子：

全局配置 ~/.gitconfig

[user]
    name = John Doe
    email = john@doe.tld

[includeIf "gitdir:~/work/"]
    path = ~/work/.gitconfig

工作特定配置 ~/work/.gitconfig

[user]
    email = john.doe@company.tld

请记住，[includeIf...]应该遵循[user]顶部的默认值。

如何为 Spring Boot 应用程序配置端口

Arya

2023-02-17 07:29

如何配置 Spring Boot 应用程序侦听的 TCP/IP 端口，使其不使用默认端口 8080。



解答

正如文档中所说，要么使用 jvm 的命令行选项设置server.port为系统属性，-Dserver.port=8090要么application.properties添加/src/main/resources/

server.port=8090

对于随机端口使用：

server.port=0

同样application.yml添加/src/main/resources/：

server:
  port: 8090


更多建议

在 Spring Boot 应用程序中更改嵌入式 Tomcat 中的端口有两种主要方法。

修改application.properties

首先你可以试试 /resources 文件夹下的 application.properties 文件：

server.port = 8090

修改虚拟机选项

第二种方式，如果你想避免修改任何文件并签入你只在本地需要的东西，你可以使用 vm arg：

转到运行 - > 编辑配置 -> VM 选项

-Dserver.port=8090

此外，如果您需要更多信息，可以在此处查看以下博客文章：Changing the port on a Spring Boot Application
