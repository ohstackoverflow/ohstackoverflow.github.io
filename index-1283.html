Arya

2023-02-18 05:54

我如何\n用真正的编辑器中显示的新行替换 Sublime Text，以便：

foo\nbar

变成：

foo
bar

当我在其中查看文件时在编辑器中。



解答

打开正则表达式搜索和替换（搜索和替换栏最左侧的图标或快捷方式Alt+ R）

查找内容：\\n
替换为：\n


更多建议

错误：不止一个模块匹配。使用 skip-import 选项跳过将组件导入最近的模块

Arya

2023-02-18 05:53

当我尝试在角度 cli 中创建组件时，它向我显示此错误。我该如何摆脱它？

错误：不止一个模块匹配。使用 skip-import 选项跳过将组件导入最近的模块。

我正在使用角度 cli 版本：1.4.1



解答

使用 --module 参数指定模块。例如，如果主模块是 app.module.ts，运行：

ng g c new-component --module app

或者，如果您在其他目录中，则

ng g c component-name --module ../


更多建议

试试这个： 它对我有用

ng generate component componentName --module=app.module

可以使用 PHP 短标签吗？

Arya

2023-02-18 05:53

以下是根据官方文档提供的信息：

有四对不同的开始和结束标记可以在 PHP 中使用。其中两个，<?php ?>和<script language="php"> </script>，总是可用的。另外两个是短标签和 ASP 样式标签，可以从 php.ini 配置文件中打开和关闭。因此，虽然有些人觉得短标签和 ASP 样式标签很方便，但它们的可移植性较差， 通常不推荐使用 。

根据我的经验，大多数服务器 都 启用了短标签。打字

<?=

远比打字方便

<?php echo

程序员的方便是一个重要因素，那么 为什么 不推荐它们呢？



解答

PHP 短标记 ( <?) 和速记回显标记 ( <?=)之间必须有明确的区别

前者被PHP Coding standard禁止，主要是出于常识，因为如果您必须将代码移动到不支持它的服务器（并且您无法启用它），它就是一个 PITA。正如您所说，许多共享主机 确实 支持短标签，但“很多”并不是全部。如果你想分享你的脚本，最好使用完整的语法。

而速记回显标签<?=不能被禁用，因此完全可以接受使用。

我同意这<?对程序员来说更容易，<?php但只要您每次都使用相同的表格，就可以进行批量查找和替换。

我根本不买可读性作为理由。大多数认真的开发人员都可以选择语法高亮显示。

正如 ThiefMaster 在评论中提到的那样， 从 PHP 5.4 开始，<?= ... ?>无论短标签设置如何，到处都支持标签。这应该意味着它们可以安全地用于可移植代码，但这确实意味着对 PHP 5.4+ 的依赖。如果你想支持 pre-5.4 并且不能保证短标签，你仍然需要使用<?php echo ... ?>.

此外，您需要知道 ASP 标记 <% 、 %> 、 <%= 和脚本标记已从 PHP 7 中删除。因此，如果您想支持长期可移植代码并希望切换到最现代的工具，请考虑更改这部分代码。


更多建议

我太喜欢<?=$whatever?>放手了。从来没有遇到过问题。我会等到它咬我的屁股。严肃地说，85% 的（我的）客户在 极少数 情况下可以访问 php.ini。其他 15% 使用主流托管服务提供商，而且几乎所有服务提供商都启用了它们。我爱他们。
