如何将本地未提交的更改合并到另一个 Git 分支？

Arya

2023-02-17 17:57

我怎样才能在 Git 中执行以下操作？

我当前的分支是 branch1，我做了一些本地更改。但是，我现在意识到我实际上是想将这些更改应用到 branch2。有没有办法应用/合并这些更改，以便它们成为 branch2 上的本地更改，而无需在 branch1 上提交它们？



解答

由于您的文件尚未提交branch1：

git stash
git checkout branch2
git stash pop

或者

git stash
git checkout branch2
git stash list       # to check the various stash made in different branch
git stash apply x    # to select the right one

以上是rbento回答的更长更明确的版本：

git stash
git stash branch branch2

它用：

git stash branch <branchname> [<stash>]

  • 创建并签出一个新分支，名称<branchname>从最初创建的提交开始<stash>，
  • 将记录的更改应用于<stash>新的工作树和索引。

如果成功，并且<stash>是 form 的引用stash@{<revision>}，则它会删除<stash>.

如果您运行的分支git stash push已经发生足够多的更改而git stash apply由于冲突而失败，这将很有用。
由于存储条目应用在运行时 HEAD 的提交之上git stash，因此它恢复了最初存储的状态而没有冲突。

正如benjohn所评论的（参见git stash手册页）：

要同时存储当前未跟踪的（新添加的）文件，请添加参数-u，因此：

git stash -u


更多建议

隐藏、临时提交和变基可能都有些矫枉过正。如果您还没有将更改的文件添加到索引中，那么您可以只检查另一个分支。

git checkout branch2

只要您正在编辑的文件在 branch1 和 branch2 之间没有不同，这就会起作用。它将使您留在 branch2 上，并保留工作更改。如果它们不同，那么您可以指定要将本地更改与通过使用-m结帐选项切换分支引入的更改合并。

git checkout -m branch2

如果您已将更改添加到索引，那么您将希望先通过重置来撤消这些更改。（这将保留您的工作副本，它只会删除暂存的更改。）

git reset
