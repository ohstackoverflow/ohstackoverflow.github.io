require、include、require_once 和 include_once 之间的区别？

Arya

2023-02-17 02:31

在 PHP 中：

  • 我应该什么时候使用requirevs. include？
  • 我应该什么时候使用require_oncevs. include_once？



解答

有require也include_once有。

所以你的问题应该是...

  1. 我应该什么时候使用requirevs. include？
  2. 我应该什么时候使用require_oncevs.require

此处描述了 1 的答案。

require() 函数与 include() 相同，只是它处理错误的方式不同。如果发生错误，include() 函数会生成警告，但脚本会继续执行。require() 生成致命错误，脚本将停止。

可以在此处找到 2 的答案。

require_once() 语句与 require() 相同，除了 PHP 将检查文件是否已被包含，如果是，则不再包含（要求）它。


更多建议

使用

  • require当您的应用程序 需要
    该文件时，例如重要的消息模板或包含配置变量的文件，没有这些配置变量应用程序将中断。 __

  • require_once
    当文件包含会在后续包含时产生错误的内容，例如function important() { /* important code */}在您的应用程序中肯定需要，但由于函数无法重新声明，因此不应再次包含。

  • 在不需要文件时包含
    ，并且在找不到文件时应用程序流程应该继续，例如非常适合从当前范围引用变量的模板或其他内容

  • include_once
    可选的依赖项会在后续加载时产生错误，或者由于 HTTP 开销，您不希望出现两次远程文件包含

但基本上，何时使用哪个取决于您。
