值得使用 Python 的 re.compile 吗？

Arya

2023-02-18 00:40

在 Python 中对正则表达式使用编译有什么好处吗？

h = re.compile('hello')
h.match('hello world')

对比

re.match('hello', 'hello world')



解答

我有很多运行已编译的正则表达式 1000 次与即时编译的经验，并且没有注意到任何可察觉的差异。显然，这是轶事，当然不是 反对 编译的有力论据，但我发现差异可以忽略不计。

编辑：快速浏览实际的 Python 2.5 库代码后，我看到 Python 会在您无论如何使用它们时（包括re.match()对根本不会节省很多时间——只节省了检查缓存（内部类型的键查找dict）所花费的时间。

来自模块 re.py（评论是我的）：

def match(pattern, string, flags=0):
    return _compile(pattern, flags).match(string)

def _compile(*key):

    # Does cache check at top of function
    cachekey = (type(key[0]),) + key
    p = _cache.get(cachekey)
    if p is not None: return p

我仍然经常预编译正则表达式，但只是为了将它们绑定到一个漂亮的、可重用的名称，而不是为了获得任何预期的性能提升。


更多建议

对我来说，最大的好处re.compile是能够将正则表达式的定义与其使用分开。

即使是一个简单的表达式，例如0|[1-9][0-9]*（以 10 为基数，不带前导零的整数）也可能足够复杂，以至于您宁愿不必重新输入它，也不必检查是否有拼写错误，然后在开始调试时必须重新检查是否有拼写错误. 另外，使用诸如 num 或 num_b10 之类的变量名比使用0|[1-9][0-9]*.

当然可以存储字符串并将它们传递给 re.match；但是，这 不太 可读：

num = "..."
# then, much later:
m = re.match(num, input)

与编译：

num = re.compile("...")
# then, much later:
m = num.match(input)

虽然很接近，但第二行的最后一行在重复使用时感觉更自然、更简单。
