查找已安装的 npm 包的版本

Arya

2023-02-16 20:31

如何找到已安装的 Node.js 或 npm 包 的版本？

这会打印 npm 本身的版本：

npm -v <package-name>

这会打印出一个神秘的错误：

npm version <package-name>

这将在注册表中 打印包版本（即可用的最新版本）：

npm view <package-name> version

我如何获得 已安装的版本 ？



解答

用于npm list本地包或npm list -g全局安装的包。

您可以通过将其名称作为参数传递来查找特定包的版本。例如，npm list grunt将导致：

projectName@projectVersion /path/to/project/folder
└── grunt@0.4.1

或者，您可以在不传递包名称作为参数的情况下运行npm list以查看所有包的版本：

├─┬ cli-color@0.1.6
│ └── es5-ext@0.7.1
├── coffee-script@1.3.3
├── less@1.3.0
├─┬ sentry@0.1.2
│ ├── file@0.2.1
│ └── underscore@1.3.3
└── uglify-js@1.2.6

您还可以添加--depth=0参数以列出没有依赖项的已安装软件包。


更多建议

另一种快速找出 本地 安装了哪些包且 没有它们的依赖项的 方法是使用：

npm list --depth=0

这给你类似的东西

├── bower@0.8.6
├── grunt@0.4.1
├── grunt-bower-requirejs@0.4.3
├── grunt-contrib-clean@0.4.1
├── grunt-contrib-coffee@0.7.0
├── grunt-contrib-copy@0.4.1
├── grunt-contrib-imagemin@0.1.4
├── grunt-contrib-jshint@0.1.1
├── grunt-contrib-livereload@0.1.2
├── grunt-contrib-requirejs@0.4.1
├── grunt-regarde@0.1.1
└── grunt-svgmin@0.1.0

显然，可以在全球范围内使用npm list -g --depth=0.

如果你安装了很多包，这个方法会更清晰。

要找出哪些包需要更新，您可以使用npm outdated -g --depth=0.
