谷歌浏览器将本地主机重定向到 https

Arya

2023-02-18 07:02

当我使用 Chrome 调试 Visual Studio 项目时，浏览器会尝试重定向到与我的网址等效的 https。我没有在 Web 项目中启用 SSL，起始 URL 是 http URL。当我使用 FireFox 或 IE 进行调试时，我没有遇到这个问题。

我确实重新安装了 Chrome，解决了一天的问题。没有下载任何插件，第二天又出现了这个问题。

是什么让 Chrome 将本地主机重定向到 https？

网络检查显示：请求 URL：data：text/html，chromewebdata 请求标头显示临时标头 User- Agent：Mozilla/5.0（Windows NT 6.3；WOW64）AppleWebKit/537.36（KHTML，如 Gecko）Chrome/36.0.1985.143 Safari/ 537.36

这些选项卡中没有预览和响应数据。



解答

我相信这是由 HSTS 引起的 - 请参阅http://en.wikipedia.org/wiki/HTTP_Strict_Transport_Security

如果您有（开发）任何其他发送 HSTS 标头的本地主机站点......

例如Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

... 然后根据 的值max-age，未来对 localhost 的请求将需要通过 HTTPS 提供服务。

为了解决这个问题，我做了以下事情。

  • 在 Chrome 地址栏中 键入以下内容：

chrome://net-internals/#hsts

  • 在页面的最底部有查询域文本框 - 验证浏览器是否知道 localhost。 如果显示“未找到”，则这不是您要找的答案。
  • 如果是，请使用上面的文本框删除本地主机域
  • 您的站点现在应该可以使用普通的旧 HTTP

这不是一个永久的解决方案，但至少可以让它在项目之间工作。如果有人知道如何从 HSTS 列表中永久排除本地主机，请告诉我:)

更新 - 2017 年 11 月

Chrome 最近将此设置移至该部分下方

删除域安全策略

更新 - 2017 年 12 月

如果您使用的是 .dev 域，请参阅下面的其他答案，因为 Chrome（和其他）通过预加载的 HSTS 强制使用 HTTPS。


更多建议

我在 Chrome 中遇到了同样的问题，我尝试使用BigJump 的解决方案没有成功。

我通过强制刷新解决了我的问题，如本博客所示（最初来自这个SuperUser 答案）。

确保您的地址栏使用的是 http 方案，然后执行这些步骤，可能要执行几次：

  1. 打开开发人员工具面板 (CTRL+SHIFT+I)
  2. 单击并按住重新加载图标/右键单击重新加载图标。
  3. 将打开一个菜单。
  4. 从此菜单中选择第三个选项（“清空缓存和硬重新加载”）
