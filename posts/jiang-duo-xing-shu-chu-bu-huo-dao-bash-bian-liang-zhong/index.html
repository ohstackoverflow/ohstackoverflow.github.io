将多行输出捕获到 Bash 变量中

Arya

2023-02-17 18:57

我有一个输出以下内容的脚本“myscript”：

abc
def
ghi

在另一个脚本中，我调用：

declare RESULT=$(./myscript)

并$RESULT获得价值

abc def ghi

有没有办法用换行符或 '\n' 字符存储结果，以便我可以用 ' echo -e' 输出它？



解答

实际上，RESULT 包含你想要的——演示：

echo "$RESULT"

你展示的就是你从中得到的：

echo $RESULT

如评论中所述，不同之处在于（1）变量的双引号版本 ( echo "$RESULT") 保留值的内部间距，与变量中表示的完全相同——换行符、制表符、多个空格和所有——而 (2 ) 未加引号的版本 ( echo $RESULT) 将一个或多个空格、制表符和换行符的每个序列替换为一个空格。因此 (1) 保留输入变量的形状，而 (2) 创建可能很长的单行输出，其中“单词”由单个空格分隔（其中“单词”是一系列非空白字符；需要' 不能是任何单词中的任何字母数字）。


更多建议

另一个陷阱是命令替换——$()去掉尾随的换行符。可能并不总是很重要，但如果你真的想保留输出的 内容 ，你将不得不使用另一行和一些引用：

RESULTX="$(./myscript; echo x)"
RESULT="${RESULTX%x}"

如果您想处理 所有 可能的文件名（以避免未定义的行为，如操作错误的文件），这一点尤其重要。
