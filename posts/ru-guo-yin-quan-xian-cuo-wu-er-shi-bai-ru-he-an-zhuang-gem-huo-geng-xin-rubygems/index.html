如果因权限错误而失败，如何安装 gem 或更新 RubyGems

Arya

2023-02-17 22:33

我正在尝试使用 来安装 gemgem install mygem或使用 更新 RubyGems gem update --system，但它因以下错误而失败：

ERROR:  While executing gem ... (Gem::FilePermissionError)
You don't have write permissions for the /Library/Ruby/Gems/2.0.0 directory.

有谁知道如何解决这个问题？



解答

您没有写入 /Library/Ruby/Gems/1.8 目录的权限。

就是这个意思，你没有在那里写的权限。

那是 Apple 安装的 Ruby 版本，供他们自己使用。 虽然如果您知道自己在做什么 ，可以对其进行微小的修改，但因为您不确定权限问题，所以我认为继续沿着这条轨道前进并不是一个好主意。

相反，我强烈建议您考虑使用rbenv或RVM来管理一个单独的 Ruby，安装到您的主目录中的沙箱中，您可以修改/折叠/主轴/更改而不用担心弄乱系统 Ruby。

在两者之间，我使用 rbenv，尽管我过去经常使用 RVM。rbenv 采用更“不干涉”的方法来管理您的 Ruby 安装。RVM 具有很多功能并且非常强大，但结果是更具侵入性。在任何一种情况下，在开始安装您选择的任何一种之前， 请阅读 它们的安装文档几次。
