为什么元字符周围有时需要空格？

Arya

2023-02-18 05:07

几个月前，我在手臂上纹了一个叉子炸弹，我跳过了空白，因为我觉得没有它们看起来更好。但令我沮丧的是， 有时 （并非总是）当我在 shell 中运行它时，它不会启动 fork 炸弹，而只会给出语法错误。

bash: syntax error near unexpected token `{:'

昨天它发生在我试图在朋友的Bash shell 中运行它时，然后我添加了空格，它突然起作用了，:(){ :|:& };:而不是:(){:|:&};:

空格重要吗？我在手臂上纹了一个语法错误吗？！

它似乎总是在zsh中工作，但在 Bash 中却不行。

一个相关的问题没有解释任何关于空格的内容，这确实是我的问题； 为什么 Bash 需要空格才能正确解析它？



解答

在 BASH 中有一个分隔标记的字符列表。这些字符称为 元字符 ，它们是|, &, ;, (, ), <, >, space 和 tab 。另一方面，大括号 ({和}) 只是组成单词的普通字符。

省略前面的第二个空格}就可以了，因为它&是一个元字符。因此，你的纹身应该至少有一个空格字符。

:(){ :|:&};:


更多建议

只纹一个

#!/bin/zsh

在它之上shebang，你会没事的。
