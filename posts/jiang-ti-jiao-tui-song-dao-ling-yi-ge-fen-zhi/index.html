将提交推送到另一个分支

Arya

2023-02-18 01:13

是否可以将更改从一个分支提交并推送到另一个分支。

假设我在 BRANCH1 中提交了更改并想将它们推送到 BRANCH2 。

从 BRANCH1 开始，执行以下操作是否有效：

git push origin **BRANCH2**

然后重置 BRANCH1？



解答

这几乎会奏效。

推送到非默认分支时，需要指定源 ref 和目标 ref：

git push origin branch1:branch2

或者

git push <remote> <branch with new changes>:<branch you are pushing to>


更多建议

当然，尽管只有当它是 BRANCH2 的快进或者你强制它时它才会起作用。做这样的事情的正确语法是

git push <remote> <source branch>:<dest branch>

有关其工作原理的更多详细信息，请参阅git push 手册页上对“refspec”的描述。另请注意，强制推送和重置都是“重写历史”的操作，除非您绝对确定您知道自己在对任何远程存储库和其他方面做什么，否则不应由胆小的人尝试拥有同一项目的分支/克隆的人。
