类就是 不能扩展的final类。 ****

（这并不意味着对类对象的所有引用都将像它们被声明为一样final。）

这个问题的答案涵盖了何时将类声明为 final 有用：

  • 在 Java 中禁止继承的充分理由？

如果 Java 是面向对象的，你声明一个类final，是不是就停止了类具有对象特性的想法？

在某种意义上是的。

通过将类标记为 final，您可以为该部分代码禁用该语言的强大而灵活的功能。然而，有些类不应该（在某些情况下 不能 ）设计为以一种好的方式考虑子类化。在这些情况下，将类标记为最终类是有意义的，即使它限制了 OOP。（但是请记住，final 类仍然可以扩展另一个非 final 类。）


更多建议

在 Java 中，final不能更改带有修饰符的项目！

这包括最终类、最终变量和最终方法：

  • final 类不能被任何其他类扩展
  • final 变量不能重新赋值
  • final 方法不能被覆盖
