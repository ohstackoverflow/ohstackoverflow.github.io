使用当前更改创建 Git 分支

Arya

2023-02-17 05:03

我开始在我的 master 分支上工作，认为我的任务很容易。过了一会儿，我意识到这需要更多的工作，我想在一个新的分支机构中完成所有这些工作。

我怎样才能创建一个新分支并在不弄脏master 的 情况下进行所有这些更改？



解答

如果您还没有进行任何提交，那么只有 (1: branch) 和 (3: checkout) 就足够了。
或者，在一个命令中： git checkout -b newBranch

使用Git 2.23+（2019 年第 3 季度），新命令git switch将在一行中创建分支（使用相同类型的reset --hard，因此 请注意其效果 ）：

# First, save your work in progress!
git stash

# Then, one command to create *and* switch to a new branch
git switch -f -c topic/wip HEAD~3

或者，按照Alia的回答中的建议，使用git switch -m, without git stash：

git switch -c topic/wip -m

--merge

如果您对当前分支和要切换到的分支之间的一个或多个文件进行了不同的本地修改，该命令将拒绝切换分支以在上下文中保留您的修改。

然而，使用这个选项， 当前分支、你的工作树内容和新分支之间的三向合并就完成了，你将在新分支上 。

当发生合并冲突时，冲突路径的索引条目不会合并，您需要解决冲突并将已解决的路径标记为git add（或者git rm如果合并应导致删除路径）。

如git reset手册页中所述：

$ git stash                # (0) Save your work in progress
$ git branch topic/wip     # (1)
$ git reset --hard HEAD~3  # (2)  NOTE: use $git reset --soft HEAD~3 (explanation below)
$ git checkout topic/wip   # (3)
  1. 您已经做出了一些承诺，但意识到它们在“”分支中还为时过早master。您希望在主题分支中继续完善它们，因此创建topic/wip当前分支的“”分支HEAD。
  2. 倒回master分支以摆脱这三个提交。
  3. 切换到“ topic/wip”分支并继续工作。

再次重申：一种新方法（自 2019 年和 Git2.23 以来）在一条命令中完成所有操作：

git switch -f -c topic/wip HEAD~3

注意：由于git reset --hard命令的“破坏性”效果（它确实会重置索引和工作树。工作树中跟踪文件的任何更改都会<commit>被丢弃），我宁愿选择：

$ git reset --soft HEAD~3  # (2)

这将确保我不会丢失任何私人文件（未添加到索引中）。
该--soft选项根本不会触及索引文件或工作树（但会将 head 重置为<commit>，就像所有模式一样）。


更多建议

就像这个问题中所述：Git：从 master 上未标记/未提交的更改创建一个分支：不需要存储。

只需使用：

git checkout -b feature/newbranch

任何未提交的工作将被带到新分支。

如果您尝试推送，您将收到以下消息

fatal: 当前分支feature/newbranch没有上游分支。要推送当前分支并将远程设置为上游，请使用

git push --set-upstream origin feature/newbranch

按照建议远程创建分支：

git push --set-upstream origin feature/newbranch
