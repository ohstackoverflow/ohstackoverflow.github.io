模板中关键字“typename”和“class”的区别？

Arya

2023-02-17 16:20

对于模板，我已经看到了两个声明：

template < typename T >
template < class T >

有什么不同？

在下面的示例中（摘自关于模板的德语维基百科文章），这些关键字究竟意味着什么？

template < template < typename, typename > class Container, typename Type >
class Example
{
     Container< Type, std::allocator < Type > > baz;
};



解答

typename并且class在指定模板的基本情况下可以互换：

template<class T>
class Foo
{
};

和

template<typename T>
class Foo
{
};

是等价的。

typename话虽如此，在某些特定情况下和之间存在差异class。

第一个是在依赖类型的情况下。typename用于在引用依赖于另一个模板参数的嵌套类型时声明，例如typedef本例中的：

template<typename param_t>
class Foo
{
    typedef typename param_t::baz sub_t;
};

您实际上在问题中显示的第二个，尽管您可能没有意识到：

template < template < typename, typename > class Container, typename Type >

当指定 模板 template 时，class必须像上面那样使用关键字——在这种情况下它 不能 与 互换 （注意：因为 C++17 在这种情况下允许使用这两个关键字） 。typename __

class您还必须在显式实例化模板时使用：

template class Foo<int>;

我确定还有其他情况我错过了，但底线是：这两个关键字不等价，这些是您需要使用一个或另一个的常见情况。


更多建议

对于命名模板参数，typename和class是等价的。§14.1.2：

模板参数中的类和类型名之间没有语义差异。

typename但是在使用模板时在另一个上下文中是可能的 - 向编译器提示您指的是依赖类型。§14.6.2：

在模板声明或定义中使用并依赖于模板参数的名称被假定为不命名类型，除非适用的名称查找找到类型名称或名称由关键字 typename 限定。

例子：

typename some_template<T>::some_type

如果没有，typename编译器通常无法判断您是否指的是类型。
