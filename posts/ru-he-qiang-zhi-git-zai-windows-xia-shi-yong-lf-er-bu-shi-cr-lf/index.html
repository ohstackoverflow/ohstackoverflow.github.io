如何强制 Git 在 Windows 下使用 LF 而不是 CR+LF？

Arya

2023-02-18 04:46

LF我想强制 Git 在 Windows 下使用not检出文件CR+LF。
我检查了两个配置选项，但无法找到正确的设置组合。

我想将所有文件转换为具有LF换行符并将其保留LF在文件中。

备注：我使用过autocrlf = input，但这只会在您提交文件时修复文件。
我想强迫它让他们使用LF.

可能我不是很清楚：存储库已经在使用，但是使用Git for WindowsLF签出的文件正在使用，我想强制 Git 使用: forcing Unix line endings 来获取它们。CR+LF``LF ****

$ git config --list | grep crlf
core.autocrlf=input



解答

在 Windows 中获得 LF 结尾的正确方法是首先设置core.autocrlf为false：

git config --global core.autocrlf false

如果您使用的是 msysgit，则需要这样做，因为它true在其系统设置中将其设置为。

现在 git 不会做任何行结束规范化。如果您希望您签入的文件被规范化，请执行以下操作：text=auto在您的.gitattributes所有文件中设置：

* text=auto

并设置core.eol为lf：

git config --global core.eol lf

现在您还可以通过运行将单个存储库切换到 crlf（在工作目录中！）

git config core.eol crlf

完成配置后，您可能希望 git 规范化 repo 中的所有 文件 。为此，请转到存储库的根目录并运行以下命令：

git rm --cached -rf .
git diff --cached --name-only -z | xargs -n 50 -0 git add -f

如果您现在希望 git 也规范化您工作目录 中的文件，请运行以下命令：

git ls-files -z | xargs -0 rm
git checkout .


更多建议

我经常回到这个问题，尽管它的其他答案都不适合我。
也就是说，对我来说正确的答案是其他答案的混合。

我发现有效的是：

git config --global core.eol lf
git config --global core.autocrlf input

对于在设置这些全局设置后检出的回购（Git 存储库），所有内容都将按回购中的任何内容检出——希望是LF( \n)。
任何CRLF将转换为仅LF在签入（提交）时。

使用您已经签出的现有存储库 - 在存储库中具有正确的行结尾但不是您的工作副本 - 您可以运行以下命令来修复它：

git rm -rf --cached .
git reset --hard HEAD

rm这将在没有提示的情况下递归地（）删除-r（） ，从当前目录（）-f删除除您已编辑的文件（）之外的所有文件。然后将所有这些文件返回到它们具有真实行尾的状态（与回购中的内容匹配）。--cached``.``reset

如果您需要修复存储库中文件的行尾，我建议您使用一个编辑器，它可以让您像 IntelliJ 或 Sublime Text 一样批量执行此操作，但我相信任何好的编辑器都可能支持这一点。
