如何在迭代时从通用列表中删除元素？

Arya

2023-02-18 03:05

我正在寻找一种更好的 模式 来处理每个需要处理的元素列表，然后根据结果从列表中删除。

您不能.Remove(element)在 a 内部使用foreach (var element in X)（因为它会导致Collection was modified; enumeration operation may not execute.异常）...您也不能使用for (int i = 0; i < elements.Count(); i++)并且.RemoveAt(i)因为它会破坏您在集合中相对于i.

有没有一种优雅的方法可以做到这一点？



解答

使用 for 循环反向迭代列表：

for (int i = safePendingList.Count - 1; i >= 0; i--)
{
    // some code
    // safePendingList.RemoveAt(i);
}

例子：

var list = new List<int>(Enumerable.Range(1, 10));
for (int i = list.Count - 1; i >= 0; i--)
{
    if (list[i] > 5)
        list.RemoveAt(i);
}
list.ForEach(i => Console.WriteLine(i));

或者，您可以使用带有谓词的RemoveAll 方法来测试：

safePendingList.RemoveAll(item => item.Value == someValue);

这是一个用于演示的简化示例：

var list = new List<int>(Enumerable.Range(1, 10));
Console.WriteLine("Before:");
list.ForEach(i => Console.WriteLine(i));
list.RemoveAll(i => i > 5);
Console.WriteLine("After:");
list.ForEach(i => Console.WriteLine(i));


更多建议

 foreach (var item in list.ToList()) {
     list.Remove(item);
 }

如果您 将“.ToList()”添加 到您的列表（或 LINQ 查询的结果），您可以直接从“列表”中删除“项目”，而不会出现可怕的“ Collection was modified; enumeration operation may not execute ”。错误。编译器制作“列表”的副本，以便您可以安全地对数组进行删除。

虽然 这种模式 不是非常有效，但它具有自然的感觉并且 足够灵活以应对几乎任何情况 。例如，当您想将每个“项目”保存到数据库并仅在数据库保存成功时将其从列表中删除时。
