需要将 git 分支重置为原始版本

Arya

2023-02-17 19:30

我不小心在一个我不应该有一段时间的分支上工作，所以我从它分支出来并给它起了一个合适的名字。现在我想覆盖我不应该在原始版本（github）上的分支。是否有捷径可寻？我尝试删除分支，然后重新设置跟踪分支，但它只是给了我再次使用的版本。



解答

如果你还没有推送到原点，你可以将你的分支重置为上游分支：

git checkout mybranch
git reset --hard origin/mybranch

（确保您在单独的分支中引用您的最新提交，就像您在问题中提到的那样）

请注意，在重置之后，mybranch@{1}指的是重置之前的旧提交。

但如果您已经推送，请参阅“创建 git 分支，并将原始状态恢复为上游状态”以了解其他选项。

使用 Git 2.23（2019 年 8 月） ，这将是一个命令： git switch 。
即：git switch -C mybranch origin/mybranch

例子

C:\Users\vonc\git\git>git switch -C master origin/master
Reset branch 'master'
Branch 'master' set up to track remote branch 'master' from 'origin'.
Your branch is up to date with 'origin/master'.

这会像往常一样恢复索引和工作树git reset --hard。

正如Brad Herman评论的那样，areset --hard会 删除任何新文件或将修改后的文件重置为 HEAD 。

git clean -f -d实际上，为了确保您从“干净的石板”开始，重置后的a将确保工作树与您刚刚重置到的分支完全相同 。

这篇博文建议使用这些别名（master仅适用于分支，但您可以调整/扩展这些别名）：

[alias]
   resetorigin = !git fetch origin && git reset --hard origin/master &&

git clean -f -d resetupstream = !git fetch upstream && git reset --hard upstream/master && git clean -f -d

然后你可以输入：

git resetupstream

或者

git resetorigin


更多建议

有一种更简单的方法可以做到这一点：

git reset --hard @{u}

@{u}是无论您的跟踪分支是什么的快捷方式，因此如果您在master并且跟踪origin/master，@{u}则指向origin/master。

使用它的好处是您不必记住（或输入）您的跟踪分支的全名。你也可以做一个别名：

git-reset-origin="git reset --hard @{u}"

无论您当前所在的分支如何，它都会起作用。
