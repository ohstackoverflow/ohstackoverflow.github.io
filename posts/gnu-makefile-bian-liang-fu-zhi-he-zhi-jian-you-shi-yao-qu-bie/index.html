GNU Makefile 变量赋值 =、?=、:= 和 += 之间有什么区别？

Arya

2023-02-17 09:18

任何人都可以清楚地解释变量赋值在 Makefile 中是如何工作的。

有什么区别：

 VARIABLE = value
 VARIABLE ?= value
 VARIABLE := value
 VARIABLE += value

我已经阅读了GNU Make 手册中的部分，但对我来说仍然没有意义。



解答

懒集

VARIABLE = value

变量的正常设置，但该value字段中提到的任何其他变量都在使用该变量时用它们的值递归扩展，而不是声明它时的值

立即设置

VARIABLE := value

通过简单扩展内部值来设置变量 - 其中的值在声明时扩展。

如果不存在则延迟设置

VARIABLE ?= value

仅当变量没有值时才设置变量。value总是在VARIABLE访问时评估。它相当于

ifeq ($(origin VARIABLE), undefined)
  VARIABLE = value
endif

有关详细信息，请参阅文档。

附加

VARIABLE += value

将提供的值附加到现有值（如果变量不存在，则设置为该值）


更多建议

使用 = 会导致变量被赋值。如果变量已经有一个值，它将被替换。这个值在使用的时候会被扩展。例如：

HELLO = world
HELLO_WORLD = $(HELLO) world!

# This echoes "world world!"
echo $(HELLO_WORLD)

HELLO = hello

# This echoes "hello world!"
echo $(HELLO_WORLD)

使用 := 类似于使用=。但是，不是在使用时扩展值，而是在赋值期间扩展它。例如：

HELLO = world
HELLO_WORLD := $(HELLO) world!

# This echoes "world world!"
echo $(HELLO_WORLD)

HELLO = hello

# Still echoes "world world!"
echo $(HELLO_WORLD)

HELLO_WORLD := $(HELLO) world!

# This echoes "hello world!"
echo $(HELLO_WORLD)

如果 以前未分配变量，则使用为变量 ?= 分配一个值。如果变量之前被分配了一个空值（）， 我认为 它仍然被认为是集合。否则，功能完全一样。 VAR= =

using += 与 using 类似=，但不是替换值，而是将值附加到当前值，中间有一个空格。如果变量以前设置为:=， 我认为 它会被扩展。 结果值在我认为 使用时会扩展。例如：

HELLO_WORLD = hello
HELLO_WORLD += world!

# This echoes "hello world!"
echo $(HELLO_WORLD)

如果HELLO_WORLD = $(HELLO_WORLD) world!使用了类似的东西，就会导致递归，这很可能会结束 Makefile 的执行。如果A := $(A) $(B)使用 were，结果将与 using 不完全相同，+=因为Bis expanded with :=while+=不会导致B被扩展。
