如何从 git 存储库中只签出一个文件？



解答

最初，我在 2012 年提到 git archive （参见Jared Forsyth的回答和Robert Knight的回答），自git1.7.9.5（2012 年 3 月）以来，Paul Brannan的回答：

git archive --format=tar --remote=origin HEAD:path/to/directory -- filename | tar -O -xf -

但是：在 2013 年，远程 https://github.com URL不再可能。
请参阅旧页面“我可以存档存储库吗？ ”

当前（2018 年）页面“关于在 GitHub 上存档内容和数据”建议使用第三方服务，例如GHTorrent或GH Archive。

所以你也可以处理本地副本/克隆：

如果您有本答案中提到的裸存储库的本地副本，您也可以执行以下操作，

git --no-pager --git-dir /path/to/bar/repo.git show branch:path/to/file >file

或者你必须首先克隆 repo，这意味着你可以获得完整的历史记录：

  • 在 .git 仓库中

  • 在工作树中。

  • 但是 你可以做一个 稀疏结账 （如果你使用的是 Git1.7+），：

    • 启用稀疏结帐选项 ( git config core.sparsecheckout true)
    • 添加你想在.git/info/sparse-checkout文件中看到的内容
    • 重新阅读工作树以仅显示您需要的内容

重新阅读工作树：

$ git read-tree -m -u HEAD

这样，您最终会得到一个工作树，其中包含您想要的内容（即使它只是一个文件）

稀疏结帐文档指出锥形模式模式集仅限于递归包含目录或匹配目录中所有文件的模式的模式。
在稀疏结帐文件中，前者以以下形式出现：

/A/B/C/

而后者成为一对形式的模式：

/A/B/
!/A/B/*/

或者在匹配顶层文件的特殊情况下：

/*
!/*/

“ add_pattern_to_hashsets()” 函数包含用于在遇到非锥形图案时禁用锥形模式的检查。
但是，当模式列表尝试匹配单个文件或目录时，这些不会捕获，例如以下形式的模式：

/A/B/C

当这种模式位于稀疏结帐文件中并且启用了圆锥模式时，这会导致稀疏结帐表现出意外行为。

具体来说，对于上述模式，非锥形模式下的稀疏结帐将仅包含位于“ /A/B/C”的目录或文件。
但是，启用锥形模式后，稀疏结帐将只显示顶层文件，而不显示位于 ' /A/B/C' 的任何文件。

相关地，在使用“”进行部分克隆时提供相同类型的过滤器时会出现问题--filter=sparse:oid=<oid>。
' upload-pack' 将正确地只包括匹配非锥形模式匹配的对象。
这意味着使用相同的过滤器检查新克隆的 repo，但启用了圆锥模式，由于缺少对象而失败。

要解决这些问题，请添加锥形模式模式检查，断言每个模式都是目录匹配项或模式“ ” /*。
添加一个测试来验证新的模式检查并修改另一个以反映非目录模式被更早捕获。


更多建议

首先使用 -n 选项克隆 repo，它禁止所有文件的默认签出，以及 --depth 1 选项，这意味着它只获取每个文件的最新修订版

git clone -n git://path/to/the_repo.git --depth 1

然后像这样检查你想要的文件：

cd the_repo
git checkout HEAD name_of_file
