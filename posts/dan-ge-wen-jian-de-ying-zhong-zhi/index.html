单个文件的硬重置

Arya

2023-02-17 01:16

如何放弃对单个文件的更改并用新的 HEAD 副本覆盖它？git reset --hard我只想对一个文件做。



解答

要将 Git 索引中的工作副本my-file.txt及其状态重置为 HEAD 的工作副本：

git checkout HEAD -- my-file.txt

--表示 “将此点之后的每个参数都视为文件名” 。此答案中的更多详细信息。感谢VonC指出这一点。


更多建议

自Git 2.23 （2019 年 8 月）起，您可以使用 restore （更多信息）：

git restore pathTo/MyFile

以上将在当前分支上恢复（最后一次提交）MyFile。HEAD

如果你想从其他提交中获取更改，你可以在提交历史上倒退。下面的命令将MyFile在最后一次之前获得两次提交。您现在需要-s( --source) 选项，因为现在您在恢复源时使用master~2而不是master（默认）：

git restore -s master~2 pathTo/MyFile

您也可以从其他分支获取文件！

git restore -s my-feature-branch pathTo/MyFile
