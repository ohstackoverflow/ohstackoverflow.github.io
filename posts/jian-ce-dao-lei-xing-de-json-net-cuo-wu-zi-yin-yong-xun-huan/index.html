检测到类型的 JSON.NET 错误自引用循环

Arya

2023-02-18 01:08

我尝试序列化从 Entity Data Model .edmx 自动生成的 POCO 类，当我使用

JsonConvert.SerializeObject

我收到以下错误：

检测到类型 System.data.entity 的错误自引用循环发生。

我该如何解决这个问题？



解答

最好的解决方案取自Web API 中的 Loop Reference handling（大部分答案是从中复制的）：

修复 1：全局忽略循环引用

（我已经选择/尝试过这个，其他许多人也是如此）

json.net 序列化器有一个选项可以忽略循环引用。将以下代码放入WebApiConfig.cs文件中：

config.Formatters.JsonFormatter.SerializerSettings.ReferenceLoopHandling = Newtonsoft.Json.ReferenceLoopHandling.Ignore;

简单的修复将使序列化程序忽略将导致循环的引用。但是，它有局限性：

  • 数据丢失循环引用信息
  • 该修复仅适用于 JSON.net
  • 如果有很深的引用链，则无法控制引用的级别

如果你想在非 api ASP.NET 项目中使用此修复程序，你可以将上面的行添加到Global.asax.cs，但首先添加：

var config = GlobalConfiguration.Configuration;

如果你想在 .Net Core 项目中使用它，你可以更改Startup.cs为：

var mvc = services.AddMvc(options =>
{
   ...
})
.AddJsonOptions(x => x.SerializerSettings.ReferenceLoopHandling = Newtonsoft.Json.ReferenceLoopHandling.Ignore);

修复 2：全局保留循环引用

第二个修复与第一个类似。只需将代码更改为：

config.Formatters.JsonFormatter.SerializerSettings.ReferenceLoopHandling
     = Newtonsoft.Json.ReferenceLoopHandling.Serialize;

config.Formatters.JsonFormatter.SerializerSettings.PreserveReferencesHandling = Newtonsoft.Json.PreserveReferencesHandling.Objects;

应用此设置后，数据形状将发生变化。

[
   {
      "$id":"1",
      "Category":{
         "$id":"2",
         "Products":[
            {
               "$id":"3",
               "Category":{
                  "$ref":"2"
               },
               "Id":2,
               "Name":"Yogurt"
            },
            {
               "$ref":"1"
            }
         ],
         "Id":1,
         "Name":"Diary"
      },
      "Id":1,
      "Name":"Whole Milk"
   },
   {
      "$ref":"3"
   }
]

$id 和 $ref 保留所有引用并使对象图级别平坦，但客户端代码需要知道形状变化才能使用数据，它也仅适用于 JSON.NET 序列化程序。

修复 3：忽略并保留参考属性

此修复是装饰模型类的属性以控制模型或属性级别的序列化行为。要忽略该属性：

public class Category
{
    public int Id { get; set; }
    public string Name { get; set; }

    [JsonIgnore]
    [IgnoreDataMember]
    public virtual ICollection<Product> Products { get; set; }
}

JsonIgnore 适用于 JSON.NET，IgnoreDataMember 适用于 XmlDCSerializer。保留参考：

// Fix 3
[JsonObject(IsReference = true)]
public class Category
{
    public int Id { get; set; }
    public string Name { get; set; }

    // Fix 3
    //[JsonIgnore]
    //[IgnoreDataMember]
    public virtual ICollection<Product> Products { get; set; }
}

[DataContract(IsReference = true)]
public class Product
{
    [Key]
    public int Id { get; set; }

    [DataMember]
    public string Name { get; set; }

    [DataMember]
    public virtual Category Category { get; set; }
}

JsonObject(IsReference = true)]适用于 JSON.NET 并且[DataContract(IsReference = true)]适用于 XmlDCSerializer。需要注意的是：在类上申请后DataContract，需要添加DataMember到要序列化的属性中。

这些属性可以应用于 json 和 xml 序列化程序，并提供对模型类的更多控制。


更多建议

使用 JsonSerializerSettings

  • ReferenceLoopHandling.Error如果遇到参考循环，（默认）将出错。 这就是你得到异常的原因。
  • ReferenceLoopHandling.Serialize如果对象嵌套但不是无限期嵌套，则很有用。
  • ReferenceLoopHandling.Ignore如果对象是其自身的子对象，则不会序列化该对象。

例子：

JsonConvert.SerializeObject(YourPOCOHere, Formatting.Indented, 
new JsonSerializerSettings 
{ 
        ReferenceLoopHandling = ReferenceLoopHandling.Serialize
});

如果您必须序列化无限嵌套的对象，您可以使用PreserveObjectReferences来避免 StackOverflowException。

例子：

JsonConvert.SerializeObject(YourPOCOHere, Formatting.Indented, 
new JsonSerializerSettings 
{ 
        PreserveReferencesHandling = PreserveReferencesHandling.Objects
});

选择对您正在序列化的对象有意义的内容。
