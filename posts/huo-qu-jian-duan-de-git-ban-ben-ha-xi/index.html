获取简短的 Git 版本哈希

Arya

2023-02-18 03:59

有没有更简洁的方法从 Git 获取的短版本哈希HEAD？

我想看到与从以下位置获得的相同的输出：

 git log -n 1 | head -n 1 | sed -e 's/^commit //' | head -c 8

我最初使用上面的命令生成版本字符串，但这更好：

git describe --tags

它将输出类似0.1.12(tagged commit) 或0.1.11-5-g0c85fbc(five commits after the tag) 的字符串。



解答

尝试这个：

git rev-parse --short HEAD

该命令git rev-parse可以做很多不同的事情，因此您需要非常仔细地阅读文档才能发现这一点。


更多建议

你可以做任何你想要的格式--pretty=format:

git log -1 --pretty=format:%h

%h, 来自,的含义man git log是：

%h
缩写的提交哈希

要查看其他格式选项，请查看man git log并搜索以短语“ Placeholders that expand to information extracted from the commit: ”开头的部分。
