如何在 MySQL 中执行“如果不存在则插入”？

Arya

2023-02-17 06:49

我从谷歌搜索开始，找到了文章 如何在标准 SQL 中编写 INSERT if NOT EXISTS 查询， 其中讨论了互斥表。

我有一张包含约 1400 万条记录的表。如果我想以相同的格式添加更多数据，有没有办法在不使用一对查询的情况下确保我要插入的记录不存在（即，一个要检查的查询和一个要插入的结果集是空的）？

解答

使用INSERT IGNORE INTO table。

还有语法，您可以在 13.2.6.2 INSERT ... ON DUPLICATE KEY UPDATE StatementINSERT … ON DUPLICATE KEY UPDATE中找到解释。 __

根据 Google 的网络缓存从bogdan.org.ua发布：

2007 年 10 月 18 日

首先：从最新的 MySQL 开始，标题中提供的语法是不可能的。但是有几种非常简单的方法可以使用现有功能完成预期的工作。

有 3 种可能的解决方案：使用 INSERT IGNORE、REPLACE 或 INSERT … ON DUPLICATE KEY UPDATE。

假设我们有一张桌子：

CREATE TABLE `transcripts` (
`ensembl_transcript_id` varchar(20) NOT NULL,
`transcript_chrom_start` int(10) unsigned NOT NULL,
`transcript_chrom_end` int(10) unsigned NOT NULL,
PRIMARY KEY (`ensembl_transcript_id`)
) ENGINE=InnoDB DEFAULT CHARSET=latin1;

现在想象一下，我们有一个从 Ensembl 导入转录本元数据的自动管道，并且由于各种原因管道可能在执行的任何步骤中断。因此，我们需要确保两件事：

  1. 管道的重复执行不会破坏我们的 > 数据库

  2. 重复执行不会因为'duplicate > primary key'错误而死亡。

方法一：使用REPLACE

这很简单：

REPLACE INTO `transcripts`
SET `ensembl_transcript_id` = 'ENSORGT00000000001',
`transcript_chrom_start` = 12345,
`transcript_chrom_end` = 12678;

如果记录存在，它将被覆盖；如果它还不存在，它将被创建。然而，使用这种方法对我们的案例来说效率不高：我们不需要覆盖现有记录，跳过它们就可以了。

方法二：使用 INSERT IGNORE 也很简单：

INSERT IGNORE INTO `transcripts`
SET `ensembl_transcript_id` = 'ENSORGT00000000001',
`transcript_chrom_start` = 12345,
`transcript_chrom_end` = 12678;

在这里，如果数据库中已经存在“ensembl_transcript_id”，它将被静默跳过（忽略）。（更准确地说，引用 MySQL 参考手册中的一段话：“如果使用 IGNORE 关键字，则执行 INSERT 语句时发生的错误将被视为警告。例如，没有 IGNORE，重复现有 UNIQUE 索引的行or PRIMARY KEY value in the table causes a duplicate-key error and the statement is aborted.”。）如果记录尚不存在，则会创建它。

第二种方法有几个潜在的弱点，包括在发生任何其他问题时不会中止查询（请参阅手册）。因此，如果之前在没有使用 IGNORE 关键字的情况下进行过测试，则应该使用它。

方法 3：使用 INSERT … ON DUPLICATE KEY UPDATE：

第三种选择是使用INSERT … ON DUPLICATE KEY UPDATE语法，在 UPDATE 部分什么都不做，做一些无意义的（空）操作，比如计算 0+0（Geoffray 建议为 MySQL 优化引擎做 id=id 赋值以忽略此操作）。这种方法的优点是它只忽略重复的键事件，并且仍然会在其他错误时中止。

最后声明：这篇文章的灵感来自 Xaprb。我还建议参考他的另一篇关于编写灵活的 SQL 查询的文章。


更多建议

解决方案：

INSERT INTO `table` (`value1`, `value2`) 
SELECT 'stuff for value1', 'stuff for value2' FROM DUAL 
WHERE NOT EXISTS (SELECT * FROM `table` 
      WHERE `value1`='stuff for value1' AND `value2`='stuff for value2' LIMIT 1)

解释：

最内层查询

SELECT * FROM `table` 
      WHERE `value1`='stuff for value1' AND `value2`='stuff for value2' LIMIT 1

用作WHERE NOT EXISTS-condition 检测是否已经存在包含要插入的数据的行。找到这样的一行后，查询可能会停止，因此LIMIT 1（微优化，可能会被省略）。

中间查询

SELECT 'stuff for value1', 'stuff for value2' FROM DUAL

表示要插入的值。DUAL指的是所有 Oracle 数据库中默认存在的特殊的一行一列表（请参阅https://en.wikipedia.org/wiki/DUAL_table）。在 MySQL-Server 版本 5.7.26 上，我在省略时得到了一个有效查询FROM DUAL，但旧版本（如 5.5.60）似乎需要该FROM信息。WHERE NOT EXISTS如果最内层查询找到匹配数据，则使用中间查询返回一个空结果集。

外部查询

INSERT INTO `table` (`value1`, `value2`)

插入数据（如果中间查询返回任何数据）。
