如何在不使用存储库的情况下将 Docker 映像从一台主机复制到另一台主机

Arya

2023-02-16 22:43

如何在不使用存储库的情况下将 Docker 映像从一台机器传输到另一台机器，无论是私有存储库还是公共存储库？

我在 VirtualBox 中创建自己的映像，完成后我尝试部署到其他机器上以供实际使用。

由于它基于我自己的镜像（如 Red Hat Linux），因此无法从 Dockerfile 重新创建。我的 dockerfile 不容易移植。

我可以使用简单的命令吗？还是另一种解决方案？



解答

您需要将 Docker 映像保存为 tar 文件：

docker save -o <path for generated tar file> <image name>

然后使用常规文件传输工具将您的图像复制到新系统，例如cp,scp或rsync（大文件首选）。之后，您必须将图像加载到 Docker 中：

docker load -i <path to image tar file>

PS：您可能需要sudo所有命令。

编辑：您应该使用 -o 添加文件名（而不仅仅是目录），例如：

docker save -o c:/myfile.tar centos:16


更多建议

通过 SSH 传输 Docker 镜像，动态压缩内容：

docker save <image> | bzip2 | ssh user@host docker load

请注意，它docker load会自动为您解压缩图像。它支持gzip和bzip2。xz

pv将管道放在中间以查看传输情况也是一个好主意：

docker save <image> | bzip2 | pv | ssh user@host docker load

（更多信息pv：主页，手册页）。

来自@Thomas Steinbach 的重要说明：在高带宽上，bzip压缩速度不够快。如果您可以 10 MB/s 或更高的速度上传，gzip/gunzip将比bzip2.

如果您使用的是 3G 并且您的互联网速度很慢， @jgmjgm 建议您可以使用xz：它提供更高的压缩率。
