如何在提交前查看git中的文件差异

Arya

2023-02-18 03:54

这经常发生在我身上：

我在一两天内同时处理几个相关的更改，当需要提交时，我最终忘记了特定文件中的更改。（这只是一个个人 git 仓库，所以我可以接受在一次提交中有多个更新。）

有什么方法可以预览即将签入的本地文件与该文件的最后一次提交之间的更改？

就像是：

git diff --changed /myfile.txt

它会打印出类似的东西：

line 23
  (last commit): var = 2+2
  (current):     var = myfunction() + 2

line 149
  (last commit): return var
  (current):     return var / 7

这样，我可以快速查看自上次签入该文件以来我在该文件中所做的事情。



解答

如果您想查看尚未git add编辑的内容：

git diff myfile.txt

或者如果您想查看已添加的更改

git diff --cached myfile.txt


更多建议

git diff HEAD file

将显示您从上次提交添加到工作树的更改。将显示所有更改（已上演或未上演）。
