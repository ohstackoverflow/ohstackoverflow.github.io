HTML“no-js”类的目的是什么？

Arya

2023-02-18 05:18

我注意到在许多模板引擎中，在HTML5 Boilerplate中，在各种框架中和在普通的 php 站点中，标签no-js上都添加了类<HTML>。

为什么这样做？是否有某种对此类作出反应的默认浏览器行为？为什么总是包括它？如果没有 no-"no-js" 情况并且可以直接处理 html，这不会使类本身过时吗？

以下是 HTML5 样板 index.html 中的示例：

<!--[if lt IE 7 ]> <html lang="en" class="no-js ie6"> <![endif]-->
<!--[if IE 7 ]>    <html lang="en" class="no-js ie7"> <![endif]-->
<!--[if IE 8 ]>    <html lang="en" class="no-js ie8"> <![endif]-->
<!--[if IE 9 ]>    <html lang="en" class="no-js ie9"> <![endif]-->
<!--[if (gt IE 9)|!(IE)]><!--> <html lang="en" class="no-js"> <!--<![endif]-->

如您所见，<html>元素将始终具有此类。有人可以解释为什么经常这样做吗？



解答

当 Modernizr 运行时，它会删除“no-js”类并将其替换为“js”。这是一种根据是否启用 Javascript 支持来应用不同 CSS 规则的方法。

请参阅Modernizer 的源代码。


更多建议

该类由Modernizrno-js特征检测库使用。当 Modernizr 加载时，它会替换为. 如果 JavaScript 被禁用，该类将保留。这使您可以编写轻松针对任一条件的 CSS。no-js``js

来自Modernizrs 的注释来源 （不再维护） ：

从元素中删除“no-js”类，如果它存在： docElement.className=docElement.className.replace(/\bno-js\b/,'') + ' js';

这是 Paul Irish 描述这种方法的博客文章：http://www.paulirish.com/2009/avoiding-the-fouc-v3/

我喜欢做同样的事情，但没有 Modernizr。如果启用了 JavaScript，我将以下内容<script>更改<head>为类。js我更喜欢使用.replace("no- js","js")正则表达式版本，因为它不那么神秘并且适合我的需要。

<script>
    document.documentElement.className = 
       document.documentElement.className.replace("no-js","js");
</script>

在使用这种技术之前，我通常只是直接使用 JavaScript 应用依赖于 js 的样式。例如：

$('#someSelector').hide();
$('.otherStuff').css({'color' : 'blue'});

有了这个no-js技巧，现在可以通过以下方式完成css：

.js #someSelector {display: none;}
.otherStuff { color: blue; }
.no-js .otherStuff { color: green }

这是可取的，因为：

  • 它加载速度更快，没有 FOUC（无样式内容的闪光）
  • 关注点分离等...
