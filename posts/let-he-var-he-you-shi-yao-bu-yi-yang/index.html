"let" 和 "var" 和有什么不一样？

Arya

2023-02-16 18:33

ECMAScript 6 引入了声明let。

我听说它被描述为局部变量，但我仍然不太确定它的行为与 var 关键字有何不同。

有什么区别？什么时候应该let使用而不是var？



解答

范围规则

主要区别在于范围规则。由关键字声明的变量的作用域var为直接函数体（因此为函数作用域），而let变量的作用域为由表示的直接 封闭 块{ }（因此为块作用域）。

function run() {
  var foo = "Foo";
  let bar = "Bar";

  console.log(foo, bar); // Foo Bar

  {
    var moo = "Mooo"
    let baz = "Bazz";
    console.log(moo, baz); // Mooo Bazz
  }

  console.log(moo); // Mooo
  console.log(baz); // ReferenceError
}

run();

let将关键字引入该语言的原因是函数范围令人困惑，并且是 JavaScript 中错误的主要来源之一。

从另一个 Stack Overflow 问题看这个例子：

var funcs = [];
// let's create 3 functions
for (var i = 0; i < 3; i++) {
  // and store them in funcs
  funcs[i] = function() {
    // each should log its value.
    console.log("My value: " + i);
  };
}
for (var j = 0; j < 3; j++) {
  // and now let's run each one to see
  funcs[j]();
}

My value: 3每次调用时都会输出到控制台funcs[j]();，因为匿名函数绑定到同一个变量。

人们必须创建立即调用的函数以从循环中捕获正确的值，但这也很棘手。

吊装

var虽然使用关键字声明的变量被提升（undefined在代码运行之前使用初始化），这意味着它们甚至在声明之前就可以在其封闭范围内访问：

function run() {
  console.log(foo); // undefined
  var foo = "Foo";
  console.log(foo); // Foo
}

run();

let变量在其定义被评估之前不会被初始化。在初始化之前访问它们会导致ReferenceError. 据说从块的开始到处理初始化，该变量都处于“时间死区”。

function checkHoisting() {
  console.log(foo); // ReferenceError
  let foo = "Foo";
  console.log(foo); // Foo
}

checkHoisting();

创建全局对象属性

在顶层，let与 不同var，它不会在全局对象上创建属性：

var foo = "Foo";  // globally scoped
let bar = "Bar"; // not allowed to be globally scoped

console.log(window.foo); // Foo
console.log(window.bar); // undefined

重新声明

在严格模式下，var将允许您在同一范围内重新声明同一变量，同时let引发 SyntaxError。

'use strict';
var foo = "foo1";
var foo = "foo2"; // No problem, 'foo1' is replaced with 'foo2'.

let bar = "bar1"; 
let bar = "bar2"; // SyntaxError: Identifier 'bar' has already been declared


更多建议

let也可用于避免闭包问题。它绑定新值而不是保留旧引用，如下面的示例所示。

for(var i=1; i<6; i++) {
  $("#div" + i).click(function () { console.log(i); });
}

上面的代码演示了一个经典的 JavaScript 闭包问题。对i变量的引用存储在单击处理程序闭包中，而不是 的实际值i。

每个单击处理程序都将引用同一个对象，因为只有一个计数器对象包含 6 个，因此每次单击都会得到 6 个。

一般的解决方法是将其包装在匿名函数中并i作为参数传递。现在也可以通过使用letinstead来避免此类问题var，如下面的代码所示。

（在 Chrome 和 Firefox 50 中测试）

for(let i=1; i<6; i++) {
  $("#div" + i).click(function () { console.log(i); });
}
