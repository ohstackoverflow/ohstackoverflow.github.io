Node / Express：EADDRINUSE，地址已在使用 - 杀死服务器

Arya

2023-02-17 14:41

我有一个使用连接在 node.js 中运行的简单服务器：

var server = require('connect').createServer();
//actions...
server.listen(3000);

在我的代码中，我有实际的处理程序，但这是基本的想法。我不断遇到的问题是

EADDRINUSE, Address already in use

在之前崩溃或错误后再次运行我的应用程序时，我收到此错误。因为我没有打开一个新的终端实例，所以我用ctr + z.

我相当确定我所要做的就是关闭服务器或连接。我试过打电话，server.close()但process.on('exit', ...);没有成功。



解答

process.on('exit', ..)如果进程崩溃或被终止，则不会调用。它仅在事件循环结束时被调用，并且由于server.close() 某种程度上 结束了事件循环（它仍然必须在这里和那里等待当前正在运行的堆栈）将它放在退出事件中是没有意义的......

崩溃时，做process.on('uncaughtException', ..)和杀死时做process.on('SIGTERM', ..)

也就是说，SIGTERM（默认终止信号）让应用程序清理，而 SIGKILL（立即终止）不会让应用程序做任何事情。


更多建议

你也可以走命令行路线：

ps aux | grep node

获取进程ID。

然后：

kill -9 PID

在 kill 上执行 -9 会发送一个 SIGKILL（而不是 SIGTERM）。SIGTERM 有时会被节点忽略。
