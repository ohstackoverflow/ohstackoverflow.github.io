require() 和 library() 有什么区别？

Arya

2023-02-17 20:35

require()和 和有什么不一样library()？



解答

除了已经给出的好建议之外，我还要补充一点：

最好避免使用，require() 除非 您实际上将使用它返回的值，例如在一些错误检查循环中，例如 thierry 给出的。

在大多数其他情况下，最好使用library()，因为如果包不可用，这将在包加载时给出错误消息。require()如果包不存在，只会失败而不会出现错误。这是确定软件包是否需要安装（或者可能因为拼写错误而根本不存在）的最佳时机。尽早并在相关时间获得错误反馈将避免在跟踪后来的代码尝试使用库例程时失败的原因时可能出现的麻烦


更多建议

的另一个好处require()是它默认返回一个逻辑值。TRUE如果包被加载，FALSE如果没有。

> test <- library("abc")
Error in library("abc") : there is no package called 'abc'
> test
Error: object 'test' not found
> test <- require("abc")
Loading required package: abc
Warning message:
In library(package, lib.loc = lib.loc, character.only = TRUE, logical.return = TRUE,  :
  there is no package called 'abc'
> test
[1] FALSE

所以你可以require()在像下面这样的结构中使用。如果你想将你的代码分发到我们的 R 安装包可能没有安装，这主要很方便。

if(require("lme4")){
    print("lme4 is loaded correctly")
} else {
    print("trying to install lme4")
    install.packages("lme4")
    if(require(lme4)){
        print("lme4 installed and loaded")
    } else {
        stop("could not install lme4")
    }
}
