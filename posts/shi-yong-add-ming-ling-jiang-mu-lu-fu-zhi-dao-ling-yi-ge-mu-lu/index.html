使用 ADD 命令将目录复制到另一个目录

Arya

2023-02-18 04:26

我已经阅读了http://docs.docker.com/engine/reference/builder/#add但是我遇到了一个问题。我想将本地目录复制go到docker/user/local/

我试过：

ADD go /usr/local/

和：

ADD /go/ /usr/local/

还：

RUN chmod 0755 /usr/local/go/src/make.bash

但是，我看到以下错误消息：

/usr/local/go/src/make.bash: No such file or directory

但本地go目录确实包含make.bash.



解答

ADD go /usr/local/

会将本地目录的 内容 复制到docker 镜像的目录中。go``/usr/local/

要复制go正在使用的目录本身/usr/local/：

ADD go /usr/local/go

或者

COPY go /usr/local/go


更多建议

确实ADD go /usr/local/会添加文件夹的 内容go而不是文件夹本身，您可以使用 Thomasleveil 解决方案，或者如果由于某种原因它不起作用，您可以更改WORKDIR为/usr/local/然后将目录添加到其中，例如：

WORKDIR /usr/local/
COPY go go/

或者

WORKDIR /usr/local/go
COPY go ./

但是如果你想添加 多个 文件夹，像这样添加它们会很烦人，我从我当前的问题中看到的唯一解决方案是使用并COPY . .排除所有不需要的目录和文件.dockerignore，假设我有文件夹和文件:

- src 
- tmp 
- dist 
- assets 
- go 
- justforfun 
- node_modules 
- scripts 
- .dockerignore 
- Dockerfile 
- headache.lock 
- package.json

我想补充src assets package.json justforfun go一点：

在 Dockerfile 中：

FROM galaxy:latest

WORKDIR /usr/local/
COPY . .

在 .dockerignore 文件中：

node_modules
headache.lock
tmp
dist

这样，你就忽略了，node_modules headache.lock tmp dist所以他们不会被添加！

或者为了更有趣（或者你想迷惑更多的人让他们也受苦 :P）可以是：

*
!src 
!assets 
!go 
!justforfun 
!scripts 
!package.json

通过这种方式，您可以忽略所有内容，但仅从“忽略列表”中排除您想要复制或添加的内容。

这是一个迟到的答案，但增加更多的方法来做同样的事情涵盖了更多的情况。
