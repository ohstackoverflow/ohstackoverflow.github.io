SQL JOIN - WHERE 子句与 ON 子句

Arya

2023-02-17 09:59

阅读后，这不是 Explicit vs Implicit SQL Joins的副本。答案可能相关（甚至相同），但 问题 不同。

有什么区别，每个应该包含什么？

如果我正确地理解了这个理论，查询优化器应该能够互换使用两者。



解答

它们不是同一件事。

考虑这些查询：

SELECT *
FROM Orders
LEFT JOIN OrderLines ON OrderLines.OrderID=Orders.ID
WHERE Orders.ID = 12345

和

SELECT *
FROM Orders
LEFT JOIN OrderLines ON OrderLines.OrderID=Orders.ID 
    AND Orders.ID = 12345

第一个将返回一个订单及其行，如果有的话，订单号为12345。
第二个将返回所有订单，但只有订单12345会有与之关联的任何行。

使用INNER JOIN，子句 实际上 是等价的。然而，仅仅因为它们在功能上相同，因为它们产生相同的结果，并不意味着这两种从句具有相同的语义。


更多建议

  • 对内部联接无关紧要

  • 外部连接的事项

A。WHERE条款：加入 后 。加入后将过滤记录。

b. ON条款 -加入 之前 。记录（来自右表）将在加入之前被过滤。这可能最终在结果中为 null（自 OUTER join 以来）。

示例 ：考虑下表：

  1. 文件：
ID 姓名
1个 文件1
2个 文件2
3个 文件3
4个 文件4
5个 文件5
  1. 下载：
ID 文件编号 用户名
1个 1个 桑迪普
2个 1个 西米
3个 2个 桑迪普
4个 2个 雷亚
5个 3个 西米

a) 内部WHERE条款：

   SELECT documents.name, downloads.id
     FROM documents
     LEFT OUTER JOIN downloads
       ON documents.id = downloads.document_id
     WHERE username = 'sandeep'

对于上述查询，中间连接表将如下所示。

id（来自文档） 姓名 id（来自下载） 文件编号 用户名
1个 文件1 1个 1个 桑迪普
1个 文件1 2个 1个 西米
2个 文件2 3个 2个 桑迪普
2个 文件2 4个 2个 雷亚
3个 文件3 5个 3个 西米
4个 文件4 无效的 无效的 无效的
5个 文件5 无效的 无效的 无效的

应用WHERE子句并选择列出的属性后，结果将是：

姓名 ID
文件1 1个
文件2 3个

b) 内部JOIN条款

   SELECT documents.name, downloads.id
   FROM documents
     LEFT OUTER JOIN downloads
       ON documents.id = downloads.document_id
         AND username = 'sandeep'

对于上述查询，中间连接表将如下所示。

id（来自文档） 姓名 id（来自下载） 文件编号 用户名
1个 文件1 1个 1个 桑迪普
2个 文件2 3个 2个 桑迪普
3个 文件3 无效的 无效的 无效的
4个 文件4 无效的 无效的 无效的
5个 文件5 无效的 无效的 无效的

documents请注意不匹配两个条件的行是如何填充NULL值的。

选择列出的属性后，结果将是：

姓名 ID
文件1 1个
文件2 3个
文件3 无效的
文件4 无效的
文件5 无效的
