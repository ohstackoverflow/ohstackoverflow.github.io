zlib、gzip 和 zip 有什么关系？它们有什么共同点，它们有何不同？

Arya

2023-02-17 05:03

zlib 中使用的压缩算法与 gzip 和 zip 中的压缩算法基本相同。什么是 gzip 和 zip ？它们有什么不同，它们有什么相同之处？



解答

简写：

.zip是一种存档格式，通常使用Deflate 压缩方法。.gzgzip 格式适用于单个文件，也使用 Deflate 压缩方法。通常 gzip 与 tar 结合使用以制作压缩存档格式，.tar.gz. zlib库提供了供 zip、gzip、png（在 deflate 数据上使用zlib 包装器）和许多其他应用程序使用的 Deflate 压缩和解压缩代码。

长表：

ZIP格式由 Phil Katz 开发为具有开放规范的开放格式，他的实现 PKZIP 是共享软件。它是一种存储文件及其目录结构的存档格式，其中每个文件都被单独压缩。文件类型是.zip. 可以选择对文件以及目录结构进行加密。

ZIP 格式支持多种压缩方法：

    0 - The file is stored (no compression)
    1 - The file is Shrunk
    2 - The file is Reduced with compression factor 1
    3 - The file is Reduced with compression factor 2
    4 - The file is Reduced with compression factor 3
    5 - The file is Reduced with compression factor 4
    6 - The file is Imploded
    7 - Reserved for Tokenizing compression algorithm
    8 - The file is Deflated
    9 - Enhanced Deflating using Deflate64(tm)
   10 - PKWARE Data Compression Library Imploding (old IBM TERSE)
   11 - Reserved by PKWARE
   12 - File is compressed using BZIP2 algorithm
   13 - Reserved by PKWARE
   14 - LZMA
   15 - Reserved by PKWARE
   16 - IBM z/OS CMPSC Compression
   17 - Reserved by PKWARE
   18 - File is compressed using IBM TERSE (new)
   19 - IBM LZ77 z Architecture 
   20 - deprecated (use method 93 for zstd)
   93 - Zstandard (zstd) Compression 
   94 - MP3 Compression 
   95 - XZ Compression 
   96 - JPEG variant
   97 - WavPack compressed data
   98 - PPMd version I, Rev 1
   99 - AE-x encryption marker (see APPENDIX E)

方法 1 到 7 是历史的，没有被使用。方法 9 到 98 是相对较新的添加，并且使用量不一。在 ZIP 格式中真正广泛使用的唯一方法是方法 8，Deflate，以及某种程度上完全没有压缩的方法 0。.zip实际上，您在野外遇到的每个文件都将专门使用方法 8 和 0，可能只是方法 8。（方法 8 也有一种有效存储数据的方法，无需压缩和相对较少的扩展，并且方法 0 无法流式传输而方法 8 可以。）

文件容器的ISO /IEC 21320-1:2015 标准是一种受限制的 zip 格式，例如用于 Java 存档文件 (.jar)、Office Open XML 文件（Microsoft Office .docx、.xlsx、.pptx）、Office Document格式文件（.odt、.ods、.odp）和 EPUB 文件 (.epub)。该标准将压缩方法限制为 0 和 8，以及其他限制，例如不加密或签名。

1990 年左右，Info-ZIP 小组编写了可移植的、免费的、开源的实现zip和unzip实用程序，支持使用 Deflate 格式进行压缩，以及对该格式和更早格式的解压缩。这极大地扩展了格式的使用范围.zip。

在 20 世纪 90 年代初期，gzip 格式被开发为Unixcompress实用程序的替代品，它源自 Info-ZIP 实用程序中的 Deflate 代码。Unixcompress旨在压缩单个文件或流，将 a 附加.Z到文件名。compress使用LZW 压缩算法，该算法当时处于专利状态，其免费使用受到专利持有人的争议。虽然 Deflate 的某些特定实现已获得 Phil Katz 的专利，但格式并未获得专利，因此可以编写不侵犯任何专利的 Deflate 实现。在过去的 20 多年里，这种实施并没有受到如此大的挑战。Unixgzip实用程序旨在作为compress，并且实际上能够解压缩compress- 压缩数据（假设您能够解析该句子）。gzip将 a 附加.gz到文件名。gzip使用 Deflate 压缩数据格式，压缩比 Unix 好很多compress，解压速度非常快，并添加了 CRC-32 作为数据的完整性检查。头部格式还允许存储比compress格式允许的更多信息，例如原始文件名和文件修改时间。

虽然compress只压缩单个文件，但通常使用该tar实用程序将文件、它们的属性和目录结构创建到一个.tar文件中，然后将其压缩成compress一个.tar.Z文件。事实上，该tar实用程序曾经并且仍然可以选择同时进行压缩，而不必将 的输出通过管道传输tar到compress. 这一切都继承到了 gzip 格式，并且tar有一个选项可以直接压缩到该.tar.gz格式。该tar.gz格式比方法压缩得更好.zip，因为 a 的压缩.tar可以利用跨文件的冗余，尤其是许多小文件。.tar.gz是 Unix 上最常用的归档格式，因为它具有非常高的可移植性，但也有更有效的压缩方法在使用，所以你会经常看到和.tar.bz2归档.tar.xz。

与 不同.tar，.zip在末尾有一个中央目录，它提供了一个内容列表。这和单独的压缩提供了对.zip文件中各个条目的随机访问。一个.tar文件必须被解压缩并从头到尾扫描才能构建一个目录，这就是文件的.tar列出方式。

在 gzip 引入后不久，大约在 1990 年代中期，同样的专利纠纷对图像格式的免费使用提出了质疑，该图像格式.gif在公告牌和万维网上非常广泛使用（在当时是一个新事物）。因此，一个小组创建了 PNG 无损压缩图像格式，文件类型为.png，以取代.gif. 该格式还使用 Deflate 格式进行压缩，在图像数据的过滤器暴露更多冗余后应用。为了促进 PNG 格式的广泛使用，创建了两个免费代码库。libpng和zlib. libpng 处理了 PNG 格式的所有特性，而 zlib 提供了压缩和解压缩代码供 libpng 以及其他应用程序使用。zlib 改编自gzip代码。

所有提到的专利都已过期。

zlib 库支持 Deflate 压缩和解压缩，以及三种环绕 deflate 流的方法。这些根本没有包装（“原始”放气）、zlib 包装（用于 PNG 格式数据块）和 gzip 包装（为程序员提供 gzip 例程）。zlib 和 gzip 包装之间的主要区别是 zlib 包装更紧凑，6 个字节，而 gzip 至少 18 个字节，并且完整性检查 Adler-32 比 gzip 使用的 CRC-32 运行得更快。原始 deflate 由读取和写入格式的程序使用.zip，这是另一种环绕 deflate 压缩数据的格式。

zlib 现在广泛用于数据传输和存储。例如，服务器和浏览器的大多数 HTTP 事务使用 zlib 压缩和解压缩数据，具体而言，HTTP 标头Content-Encoding: deflate表示包裹在 zlib 数据格式中的 deflate 压缩方法。

deflate 的不同实现可能会导致相同输入数据的不同压缩输出，正如可选择的压缩级别的存在所证明的那样，这些压缩级别允许在压缩效率与 CPU 时间之间进行权衡。zlib 和 PKZIP 并不是 deflate 压缩和解压缩的唯一实现。7-Zip 归档实用程序和Google 的zopfli 库都能够使用比 zlib 更多的 CPU 时间，以便在使用 deflate 格式时尽可能地挤出最后几位，与 zlib 的最高压缩大小相比减少了几个百分点压缩级别。pigz 实用程序，gzip 的并行实现，包括使用 zlib（压缩级别 1-9）或 zopfli（压缩级别 11）的选项，并且通过在多个处理器和内核上拆分大文件的压缩来在一定程度上减轻使用 zopfli 的时间影响。


更多建议

ZIP是一种文件格式，用于存储任意数量的文件和文件夹以及无损压缩。它对所使用的压缩方法没有做出严格的假设，但最常与DEFLATE一起使用。

Gzip既是一种基于 DEFLATE 但较少受到潜在专利等阻碍的压缩算法，也是一种用于存储单个压缩文件的文件格式。与tar结合使用时，它支持压缩任意数量的文件和文件夹。生成的文件的扩展名为.tgzor .tar.gz，通常称为 tarball 。

zlib是一个函数库，将 DEFLATE 封装在其最常见的LZ77化身中。
