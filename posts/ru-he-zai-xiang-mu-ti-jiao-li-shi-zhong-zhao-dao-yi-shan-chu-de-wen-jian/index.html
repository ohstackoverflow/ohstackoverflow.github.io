如何在项目提交历史中找到已删除的文件？

Arya

2023-02-17 00:23

曾几何时，我的项目中有一个文件，我现在希望能够获取该文件。

问题是：我不知道我什么时候删除了它以及它在哪条路径上。

当文件存在时，如何找到它的提交？



解答

如果您不知道您可以使用的确切路径

git log --all --full-history -- "**/thefile.*"

如果你知道文件所在的路径，你可以这样做：

git log --all --full-history -- <path-to-file>

这应该显示触及该文件的所有分支中的提交列表。然后，你可以找到你想要的文件的版本，并显示它......

git show <SHA> -- <path-to-file>

或者将其恢复到您的工作副本中：

git checkout <SHA>^ -- <path-to-file>

请注意插入符号 ( )，它会 先于^已识别的文件进行检出，因为在提交时文件已被删除，我们需要查看上一次提交以获取已删除文件的内容 __<SHA>


更多建议

获取已删除文件的列表并复制已删除文件的完整路径

git log --diff-filter=D --summary | grep delete

执行下一个命令以查找该提交的提交 ID 并复制提交 ID

git log --all -- FILEPATH

显示已删除文件的差异

git show COMMIT_ID -- FILE_PATH

>请记住，您可以使用like将输出写入文件

git show COMMIT_ID -- FILE_PATH > deleted.diff
