如何在 Express 中获取完整的 URL？

Arya

2023-02-17 20:08

假设我的示例 URL 是

http://example.com/one/two

我说我有以下路线

app.get('/one/two', function (req, res) {
    var url = req.url;
}

的价值url将是/one/two。

我如何在 Express 中获取 完整的 URL ？例如，在上面的例子中，我想收到http://example.com/one/two.



解答

  1. 该协议可作为req.protocol. 文档在这里

    1. 在 express 3.0 之前，您可以假设的协议是，http除非您看到它req.get('X-Forwarded-Protocol')已设置并具有值https，在这种情况下您知道那是您的协议
    2. 主机来自req.get('host')Gopal 所指示的

  2. 希望您的 URL 中不需要非标准端口，但如果您确实需要知道它，您会在应用程序状态中拥有它，因为它是您在服务器启动时传递给的任何内容app.listen。但是，在非标准端口上进行本地开发的情况下，Chrome 似乎将端口包含在主机标头中，因此req.get('host')会返回localhost:3000，例如。因此，至少对于标准端口上的生产站点并直接浏览到您的 Express 应用程序（没有反向代理）的情况，标头host似乎对 URL 中的端口做了正确的事情。

  3. 路径来自req.originalUrl（感谢@pgrassant）。请注意，这确实包括查询字符串。在 req.url 和 req.originalUrl 上的文档。根据您打算对 URL 执行的操作，originalUrl与 相比，可能是也可能不是正确的值req.url。

将所有这些组合在一起以重建绝对 URL。

  var fullUrl = req.protocol + '://' + req.get('host') + req.originalUrl;


更多建议

您可以使用node.js API 获取 URL并URL.format()从 express 传递信息，而不是自己将它们连接在一起。

例子：

var url = require('url');

function fullUrl(req) {
  return url.format({
    protocol: req.protocol,
    host: req.get('host'),
    pathname: req.originalUrl
  });
}
