如何从多个远程位置拉/推？

Arya

2023-02-17 11:47

简而言之： 有没有办法让 git repo 推送到远程 repos 列表（而不是单个“来源”）并从中拉取？

长： 我经常遇到这样的情况，当我在多台计算机上开发一个应用程序时，具有不同的连接——比如在运输途中使用笔记本电脑，当我在某个位置时使用计算机“A”，以及另一台计算机“B”而在另一个。此外，膝上型电脑可能仅与“A”或“B”连接，有时与两者连接。

我想要的是 git 总是从它当前可以连接到的所有计算机“拉”和“推”到它，因此更容易从一台机器跳到另一台机器并继续无缝工作。



解答

您可以使用以下git remote命令配置多个远程存储库：

git remote add alt alt-machine:/path/to/repo

要从所有已配置的遥控器中获取并更新跟踪分支，但不合并到 中HEAD，请执行以下操作：

git remote update

如果它当前没有连接到其中一个遥控器，它将超时或抛出错误，然后继续下一个。您必须从获取的存储库中手动合并，或者cherry- pick，这取决于您希望如何组织收集更改。

要从 alt 获取 master 分支并将其拉入当前头部，请执行以下操作：

git pull alt master

所以实际上git pull几乎是简写git pull origin HEAD（实际上它在配置文件中查找以确定这一点，但你明白了）。

要推送更新，您必须手动对每个 repo 执行此操作。
我认为，推送是在设计时考虑到中央存储库工作流程的。
