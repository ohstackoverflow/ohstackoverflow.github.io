如何将一些文件从一个 git repo 移动到另一个（不是克隆），保留历史

Arya

2023-02-17 22:26

我们的 Git 存储库最初是作为单个怪物 SVN 存储库的一部分，其中每个项目都有自己的树，如下所示：

project1/branches
        /tags
        /trunk
project2/branches
        /tags
        /trunk

显然，将文件从一个文件移动到另一个文件非常容易svn mv。但是在 Git 中，每个项目都在自己的存储库中，今天我被要求将一个子目录project2从project1. 我做了这样的事情：

$ git clone project2 
$ cd project2
$ git filter-branch --subdirectory-filter deeply/buried/java/source/directory/A -- --all
$ git remote rm origin  # so I don't accidentally overwrite the repo ;-)
$ mkdir -p deeply/buried/different/java/source/directory/B
$ for f in *.java; do 
>  git mv $f deeply/buried/different/java/source/directory/B
>  done
$ git commit -m "moved files to new subdirectory"
$ cd ..
$
$ git clone project1
$ cd project1
$ git remote add p2 ../project2
$ git fetch p2
$ git branch p2 remotes/p2/master
$ git merge p2 # --allow-unrelated-histories for git 2.9+
$ git remote rm p2
$ git push

但这似乎很复杂。一般来说，有没有更好的方法来做这种事情？还是我采用了正确的方法？

请注意，这涉及将历史合并到现有存储库中，而不是简单地从另一个存储库的一部分创建一个新的独立存储库（如前面的问题）。



解答

是的，点击--subdirectory-filteroffilter-branch是关键。您使用它的事实基本上证明没有更简单的方法 - 您别无选择，只能重写历史记录，因为您只想得到文件的一个（重命名的）子集，并且根据定义，这会更改哈希值。由于没有标准命令（例如pull）重写历史记录，因此您无法使用它们来完成此操作。

当然，您可以改进细节——您的一些克隆和分支并不是绝对必要的——但总体方法是好的！很遗憾它很复杂，但是当然，git 的目的并不是要让重写历史变得容易。


更多建议

在尝试了将文件或文件夹从一个 Git 存储库移动到另一个 Git 存储库的各种方法后，下面概述了唯一似乎可靠工作的方法。

它涉及克隆要从中移动文件或文件夹的存储库，将该文件或文件夹移动到根目录，重写 Git 历史记录，克隆目标存储库并将具有历史记录的文件或文件夹直接拉入此目标存储库。

第一阶段

  1. 按照以下步骤制作存储库 A 的副本，对您不应推送的此副本进行重大更改！

    git clone --branch <branch> --origin origin --progress \

    -v # eg. git clone --branch master --origin origin --progress \

    -v https://username@giturl/scm/projects/myprojects.git

    (assuming myprojects is the repository you want to copy from)

  2. 进入它

    cd <git repository A directory>

    eg. cd /c/Working/GIT/myprojects

  3. 删除原始存储库的链接以避免意外进行任何远程更改（例如通过推送）

    git remote rm origin

  4. 浏览你的历史和文件，删除不在目录 1 中的任何东西。结果是目录 1 的内容被喷到存储库 A 的底部。

    git filter-branch --subdirectory-filter <directory> -- --all

    eg. git filter-branch --subdirectory-filter subfolder1/subfolder2/FOLDER_TO_KEEP -- --all

  5. 仅适用于单个文件移动：遍历剩下的内容并删除所需文件以外的所有内容。（您可能需要删除不需要的同名文件并提交。）

    git filter-branch -f --index-filter \

    'git ls-files -s | grep $'\t'FILE_TO_KEEP$ | GIT_INDEX_FILE=$GIT_INDEX_FILE.new \ git update-index --index-info && \ mv $GIT_INDEX_FILE.new $GIT_INDEX_FILE || echo "Nothing to do"' --prune-empty -- --all

    eg. FILE_TO_KEEP = pom.xml to keep only the pom.xml file from FOLDER_TO_KEEP

第二阶段

  1. 清理步骤

    git reset --hard

  2. 清理步骤

    git gc --aggressive

  3. 清理步骤

    git prune

您可能希望将这些文件导入目录而非根目录中的存储库 B：

  1. 制作那个目录

    mkdir <base directory>             eg. mkdir FOLDER_TO_KEEP

  2. 将文件移动到该目录

    git mv * <base directory>          eg. git mv * FOLDER_TO_KEEP

  3. 将文件添加到该目录

    git add .

  4. 提交您的更改，我们已准备好将这些文件合并到新的存储库中

    git commit

第三阶段

  1. 如果您还没有存储库 B，请复制一份
    git clone <git repository B url>

    eg. git clone https://username@giturl/scm/projects/FOLDER_TO_KEEP.git

（假设 FOLDER_TO_KEEP 是您要复制到的新存储库的名称）

  1. 进入它

    cd <git repository B directory>

    eg. cd /c/Working/GIT/FOLDER_TO_KEEP

  2. 创建到存储库 A 的远程连接作为存储库 B 中的分支

    git remote add repo-A-branch <git repository A directory>

    (repo-A-branch can be anything - it's just an arbitrary name)

    eg. git remote add repo-A-branch /c/Working/GIT/myprojects

  3. 从此分支（仅包含您要移动的目录）拉入存储库 B。

    git pull repo-A-branch master --allow-unrelated-histories

pull 复制文件和历史记录。注意：您可以使用合并而不是拉取，但拉取效果更好。

  1. 最后，您可能想通过删除与存储库 A 的远程连接来清理一下

    git remote rm repo-A-branch

  2. 按下，一切就绪。

    git push
