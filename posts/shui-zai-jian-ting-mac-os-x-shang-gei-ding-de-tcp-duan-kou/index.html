谁在监听 Mac OS X 上给定的 TCP 端口？

Arya

2023-02-16 23:29

在 Linux 上，我可以使用netstat -pntl | grep $PORTorfuser -n tcp $PORT找出哪个进程 (PID) 正在侦听指定的 TCP 端口。我如何在 Mac OS X 上获得相同的信息？



解答

在 macOSBig Sur及更高版本上，使用此命令：

sudo lsof -i -P | grep LISTEN | grep :$PORT

或者只看 IPv4：

sudo lsof -nP -i4TCP:$PORT | grep LISTEN

在旧版本上，使用以下形式之一：

sudo lsof -nP -iTCP:$PORT | grep LISTEN
sudo lsof -nP -i:$PORT | grep LISTEN

替换$PORT为端口号或以逗号分隔的端口号列表。

如果您需要有关 #1024 以下端口的信息，请在前面添加sudo（后跟一个空格）。

该-n标志用于显示 IP 地址而不是主机名。这使得命令执行得更快，因为获取主机名的 DNS 查找可能很慢（对于许多主机来说需要几秒或一分钟）。

该-P标志用于显示原始端口号而不是解析名称，如http，ftp或更深奥的服务名称，如dpserve, socalia。

有关更多选项，请参阅评论。

为了完整起见，因为经常一起使用：

要杀死 PID：

sudo kill -9 <PID>
# kill -9 60401


更多建议

直到 macOS 13 Ventura ，每个版本的 macOS 都支持这个：

sudo lsof -iTCP -sTCP:LISTEN -n -P

就个人而言，我最终在我的中得到了这个简单的功能~/.bash_profile：

listening() {
    if [ $# -eq 0 ]; then
        sudo lsof -iTCP -sTCP:LISTEN -n -P
    elif [ $# -eq 1 ]; then
        sudo lsof -iTCP -sTCP:LISTEN -n -P | grep -i --color $1
    else
        echo "Usage: listening [pattern]"
    fi
}

然后listening命令会为您提供监听某个端口的进程列表，并listening smth根据某种模式对其进行 greps。

有了这个，就很容易询问特定的过程，例如listening dropbox，或端口，例如listening 22。

lsof命令有一些专门的选项用于询问端口、协议、进程等。但我个人发现上面的功能更方便，因为我不需要记住所有这些低级选项。lsof是非常强大的工具，但不幸的是使用起来不是很舒服。
