如何将现有的非空目录转换为Git工作目录并将文件推送到远程仓库

Arya

2023-02-17 13:56

  1. 我有一个非空目录（例如 /etc/something），其中包含无法重命名、移动或删除的文件。

  2. 我想将这个目录检查到 git 中。

  3. 我希望能够使用“git push”或类似的东西将这个存储库的状态推送到远程存储库（在另一台机器上）。

使用 Subversion 很简单（目前我们使用 Subversion 来做）：

svn mkdir <url> -m <msg>
cd <localdir>
svn co <url> .
svn add <files etc>
svn commit -m <msg>

什么是 git 等价物？

我可以“git clone”到一个空目录并简单地移动 .git 目录并让一切正常吗？



解答

假设你已经设置了一个 git 守护进程<url>和一个空的存储库：

cd <localdir>
git init
git add .
git commit -m 'message'
git remote add origin <url>
git push -u origin main
