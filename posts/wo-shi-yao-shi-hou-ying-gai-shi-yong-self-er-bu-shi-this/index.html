我什么时候应该使用“self”而不是“$this”？

Arya

2023-02-16 22:16

self在 PHP 5 中，using和之间有什么区别$this？

什么时候合适？



解答

简答

用于$this引用当前对象。用于self引用当前类。换句话说，用于$this->member非静态成员，用于self::$member静态成员。

完整答案

以下是正确 使用$this非静态和静态成员变量的示例self：

<?php
class X {
    private $non_static_member = 1;
    private static $static_member = 2;

    function __construct() {
        echo $this->non_static_member . ' '
           . self::$static_member;
    }
}

new X();
?>

以下是非静态和静态成员变量的 错误 使用示例：$this``self

<?php
class X {
    private $non_static_member = 1;
    private static $static_member = 2;

    function __construct() {
        echo self::$non_static_member . ' '
           . $this->static_member;
    }
}

new X();
?>

以下是成员函数的 多态性 示例：$this

<?php
class X {
    function foo() {
        echo 'X::foo()';
    }

    function bar() {
        $this->foo();
    }
}

class Y extends X {
    function foo() {
        echo 'Y::foo()';
    }
}

$x = new Y();
$x->bar();
?>

下面是一个通过使用for 成员函数来 抑制多态行为 的例子self：

<?php
class X {
    function foo() {
        echo 'X::foo()';
    }

    function bar() {
        self::foo();
    }
}

class Y extends X {
    function foo() {
        echo 'Y::foo()';
    }
}

$x = new Y();
$x->bar();
?>

这个想法是$this->foo()调用foo()当前对象的确切类型的成员函数。如果对象属于type X，则它调用X::foo()。如果对象属于type Y，它会调用Y::foo()。但是对于 self::foo()，X::foo()总是被调用。

更多建议

关键字 self 不仅仅 指 “当前类”，至少不是以将您限制为静态成员的方式。在非静态成员的上下文中，还提供了一种绕过当前对象的selfvtable（请参阅 wiki on vtable ）的方法。正如您可以使用parent::methodName()调用父版本的函数一样，您也可以使用self::methodName()调用当前类实现的方法。

class Person {
    private $name;

    public function __construct($name) {
        $this->name = $name;
    }

    public function getName() {
        return $this->name;
    }

    public function getTitle() {
        return $this->getName()." the person";
    }

    public function sayHello() {
        echo "Hello, I'm ".$this->getTitle()."<br/>";
    }

    public function sayGoodbye() {
        echo "Goodbye from ".self::getTitle()."<br/>";
    }
}

class Geek extends Person {
    public function __construct($name) {
        parent::__construct($name);
    }

    public function getTitle() {
        return $this->getName()." the geek";
    }
}

$geekObj = new Geek("Ludwig");
$geekObj->sayHello();
$geekObj->sayGoodbye();

这将输出：

你好，我是 Ludwig the geek
来自 Ludwig the person 再见

sayHello()使用$this指针，因此调用 vtable 来调用Geek::getTitle(). sayGoodbye()uses self::getTitle()，因此 vtable 未被使用，而是Person::getTitle()被调用。在这两种情况下，我们都在处理实例化对象的方法，并且可以访问被$this调用函数中的指针。
