如何将空目录（不包含任何文件）添加到 Git 存储库？



解答

使目录（在存储库中）保持（几乎）为空的另一种方法是在该目录中创建一个.gitignore包含以下四行的文件：

# Ignore everything in this directory
*
# Except this file
!.gitignore

这样您就不必像在 m104 的解决方案中那样正确地获得订单。

这也带来了好处，即当您执行 git status 时，该目录中的文件不会显示为“未跟踪”。

我认为值得注意的是，这个解决方案正是按照问题的要求进行的，但也许并不是许多正在寻找这个问题的人一直在寻找的。此解决方案保证目录保持为空。它说“我真的不想在这里签入文件”。与“我这里没有任何文件要签入，但我需要这里的目录，文件可能会在稍后出现”相反。

你不能。请参阅Git 常见问题解答。

目前 git 索引（暂存区）的设计只允许列出文件，没有人有足够的能力做出允许空目录的更改，已经足够关心这种情况来补救它。

在其中添加文件时，目录会自动添加。也就是说，目录永远不必添加到存储库中，也不会自行跟踪。

您可以说“ git add <dir>”，它会在其中添加文件。

如果你真的需要一个目录存在于结帐中，你应该在其中创建一个文件。.gitignore 可以很好地用于此目的；您可以将其留空，或填写您希望在目录中显示的文件的名称。
