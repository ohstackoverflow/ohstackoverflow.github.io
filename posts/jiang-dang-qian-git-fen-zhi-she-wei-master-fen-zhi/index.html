将当前 Git 分支设为 master 分支

Arya

2023-02-16 23:10

我在 Git 中有一个存储库。我创建了一个分支，然后对 master 和分支做了一些更改。

然后，几十次提交后，我意识到分支的状态比主分支好得多，所以我希望分支“成为”主分支并忽略主分支上的更改。

我无法合并它，因为我不想将更改保留在 master 上。我应该怎么办？

额外 ：在这种情况下，“旧”master 已经被push-ed 到另一个存储库，例如 GitHub。这如何改变事情？



解答

其他两个答案的问题是新主人没有旧主人作为祖先，所以当你推动它时，其他人都会搞砸了。这就是你想要做的：

git checkout better_branch
git merge --strategy=ours master    # keep the content of this branch, but record a merge
git checkout master
git merge better_branch             # fast-forward master up to the merge

如果你想让你的历史更清楚一点，我建议在合并提交消息中添加一些信息，以明确你做了什么。将第二行更改为：

git merge --strategy=ours --no-commit master
git commit          # add information to the template merge message


更多建议

确保所有内容都被推送到您的远程存储库 (GitHub)：

git checkout main

用“better_branch”覆盖“main”：

git reset --hard better_branch

强制推送到您的远程存储库：

git push -f origin main
