HTTP 标头是否区分大小写？

Arya

2023-02-17 09:01

在博客文章中，我使用以下 PHP 来设置响应的内容类型：

header('content-type: application/json; charset=utf-8');

我刚在那篇文章中收到一条评论，说content-type需要大写，Content-type。它是否正确？它似乎适用于所有小写字母，我假设 HTTP 标头不区分大小写。或者它只是因为浏览器很好才起作用？



解答

标头名称不区分大小写。

来自RFC 2616 - “超文本传输​​协议 - HTTP/1.1”，第 4.2 节，“消息头”：

每个标头字段由一个名称后跟一个冒号（“:”）和字段值组成。字段名称区分 大小写 。

更新的 RFC 7230在此部分未列出RFC 2616 的任何更改。


更多建议

根据RFC 2616，HTTP 标头名称不区分大小写：

4.2：

每个标头字段由一个名称后跟一个冒号（“:”）和字段值组成。字段名称不区分大小写。

（字段 值 可能区分大小写，也可能不区分大小写。）

如果您相信主要浏览器会遵守这一点，那么您就万事大吉了。

顺便说一句，与大多数 HTTP 不同，方法（动词） 区分 大小写：

5.1.1 方法

Method 标记指示要对
Request-URI 标识的资源执行的方法。该方法区分大小写。

   Method         = "OPTIONS"                ; Section 9.2
                  | "GET"                    ; Section 9.3
                  | "HEAD"                   ; Section 9.4
                  | "POST"                   ; Section 9.5
                  | "PUT"                    ; Section 9.6
                  | "DELETE"                 ; Section 9.7
                  | "TRACE"                  ; Section 9.8
                  | "CONNECT"                ; Section 9.9
                  | extension-method
   extension-method = token
