如何 grep Git 提交差异或特定单词的内容

Arya

2023-02-17 13:24

在 Git 代码存储库中，我想列出包含特定单词的所有提交。我试过这个

git log -p | grep --context=4 "word"

但它不一定会给我返回文件名（除非它与我搜索的单词相距不到五行。我也试过

git grep "word"

但它只给我当前的文件而不是历史。

如何搜索整个历史记录以便跟踪特定单词的变化？我打算在我的代码库中搜索单词的出现以跟踪更改（在文件历史记录中搜索）。



解答

如果要查找 提交消息 包含给定单词的所有提交，请使用

$ git log --grep=word

如果你想找到所有在文件内容 中添加或删除“word”的提交（更准确地说：“word”出现次数发生变化的地方），即搜索提交 内容 ，使用所谓的'鹤嘴锄搜索

$ git log -Sword

在现代 Git 中还有

$ git log -Gword

查找添加或删除的行与“word”匹配的 差异 （也 提交 contents ）。

有几点需要注意：

  • -G默认情况下接受正则表达式，同时-S接受字符串，但可以使用--pickaxe-regex.
  • -S查找“word”出现次数发生变化的提交，同时-G查找差异中出现“word”的提交。
  • 这意味着-S<regex> --pickaxe-regex和-G<regex>不做完全一样的事情。

该git diff文档对差异有很好的解释：

-S<regex> --pickaxe-regex为了说明和之间的区别-G<regex>，请考虑在同一文件中使用以下差异进行提交：

+    return frotz(nitfol, two->ptr, 1, 0);
...
-    hit = frotz(nitfol, mf2.ptr, 1, 0);

虽然git log -G"frotz\(nitfol"会显示此提交，git log -S"frotz\(nitfol" --pickaxe- regex但不会（因为该字符串的出现次数没有改变）。


更多建议

git log的镐将找到包含更改的提交，包括“word”和git log -Sword
