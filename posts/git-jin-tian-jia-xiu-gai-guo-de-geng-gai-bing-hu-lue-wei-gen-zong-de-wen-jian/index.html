git 仅添加修改过的更改并忽略未跟踪的文件

Arya

2023-02-17 11:30

我运行了“git status”，下面列出了一些被修改的文件/或在“changes not staged for commit”标题下。它还列出了一些我想忽略的未跟踪文件（我在这些目录中有一个“.gitignore”文件）。

我想将修改后的文件暂存，以便我可以提交它们。当我运行“git add .”时，它添加了修改后的文件和我想忽略的文件到暂存区。

如果出现下面的 git 状态，我如何只添加修改过的文件并忽略未跟踪的文件。

另外，我的“.gitignore”文件是否正常工作？

$ git status
# On branch addLocation
# Changes not staged for commit:
#   (use "git add <file>..." to update what will be committed)
#   (use "git checkout -- <file>..." to discard changes in working directory)
#
#       modified:   someProject/path/domain/viewer/LocationDO.java
#       modified:   someProject/path/service/ld/LdService.java
#       modified:   someProject/path/service/ld/LdServiceImpl.java
#       modified:   someProject/path/web/jsf/viewer/LocationFormAction.java
#       modified:   someProject/war/WEB-INF/classes/message/viewer/viewer.properties
#       modified:   someProject/war/page/viewer/searchForm.xhtml
#
# Untracked files:
#   (use "git add <file>..." to include in what will be committed)
#
#       .metadata/
#       someProject/build/
no changes added to commit (use "git add" and/or "git commit -a")



解答

理想情况下，您.gitignore应该防止未跟踪（和忽略）的文件显示在状态中，添加使用git add等。所以我会要求您更正您的.gitignore

您可以这样做git add -u，它会暂存修改和删除的文件。

您也可以git commit -a只提交修改和删除的文件。

请注意，如果你有 2.0 之前版本的 Git 并使用过git add .，那么你需要使用git add -u .（参见““ ”和“ ”的区别git add -A``git add .”）。


更多建议

这对我有用：

#!/bin/bash
git add `git status | grep modified | sed 's/\(.*modified:\s*\)//'`

或者更好：

$ git ls-files --modified | xargs git add
