Emacs 和 Vim 之间的区别

Arya

2023-02-17 13:56

Emacs 和 Vim 之间有哪些实际的、客观的区别？例如，什么可以使用一个而不是另一个来完成（或者用一个比另一个更容易完成）？为了选择一个学习，我应该知道什么？



解答

（以下文字是我的观点，不应被视为事实或侮辱）

使用 Emacs，您应该让它 24/7 全天候打开并在程序中运行，几乎您所做的一切都可以从那里完成。您编写自己的扩展，将其用于记笔记、组织、游戏、编程、shell 访问、文件访问、听音乐、网页浏览。需要数周和数周的时间，你才会对它感到满意，然后你就会一直学到新东西。当您无法访问它并不断更改您的配置时，您会很生气。您将无法轻松使用其他人的 emacs 版本，而且它不会被安装。它使用 Lisp，这很棒。你可以把它变成你想要的任何东西。（任何东西）

对于 Vim，它几乎总是预装的。它很快。您打开一个文件进行快速编辑，然后退出。如果您在别人的机器上，您可以使用基本设置。它不是那么可编辑，但它仍然比大多数文本编辑器好得多。它认识到大部分时间您是在阅读/编辑而不是打字，并使该部分更快。您不会遭受 emacs pinkie 的 困扰。没那么气人。它更容易学习。

即使我每天整天都使用 Emacs（并且喜欢它），除非你打算在你选择的程序上花费大量时间我会选择 vim


更多建议

Vim 不是外壳。而且它不能很好地与子流程沟通。这几乎是设计使然，而在 Emacs 中，这些元素是设计使然的。这意味着有些东西，比如嵌入调试器或解释器（产生一种 IDE），在 Vim 中很难。

此外，Emacs 快捷键主要通过修饰符访问，显然 Vim 界面是著名的模态界面，可以访问大量的直接键进行操作。

Emacs 曾经是这两者中唯一可编程的编辑器，而 Vim 的可编程性有很多奇怪的级别，加上 Python 和 Ruby 绑定（还有更多，我忘记了），Vim 在大多数方面也是可编程的你会关心的。

我使用 Vim，我对它相当满意。
