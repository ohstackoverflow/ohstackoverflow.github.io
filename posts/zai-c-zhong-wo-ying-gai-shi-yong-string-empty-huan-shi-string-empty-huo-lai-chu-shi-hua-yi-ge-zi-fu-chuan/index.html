在 C# 中，我应该使用 string.Empty 还是 String.Empty 或 "" 来初始化一个字符串？

Arya

2023-02-17 13:52

在 C# 中，我想用一个空字符串初始化一个字符串值。

我应该怎么做？什么是正确的方法，为什么？

string willi = string.Empty;

或者

string willi = String.Empty;

或者

string willi = "";

或者是什么？



解答

使用您和您的团队认为最易读的内容。

其他答案建议每次使用时都会创建一个新字符串""。这不是真的——由于字符串驻留，它将在每个程序集或每个 AppDomain 中创建一次（或者可能在整个过程中创建一次——在这方面不确定）。这种差异可以忽略不计——非常 非常 微不足道。

但是，您发现哪个更具可读性是另一回事。这是主观的，因人而异——所以我建议你找出你团队中大多数人喜欢什么，并且所有人都喜欢它以保持一致性。我个人觉得""更容易阅读。

""和很容易被误认为彼此的论点" "并没有真正洗掉我。除非您使用的是比例字体（而且我还没有与 任何 开发人员合作过），否则很容易分辨出差异。


更多建议

从性能和代码生成的角度来看，确实没有区别。在性能测试中，他们在哪一个比另一个更快之间来回切换，而且只有几毫秒。

在查看幕后代码时，您也确实看不出任何区别。唯一的区别在于 IL，它string.Empty使用 opcodeldsfld和""使用 opcode ldstr，但这只是因为它string.Empty是静态的，并且两条指令做同样的事情。如果您查看生成的组件，它完全相同。

C#代码

private void Test1()
{
    string test1 = string.Empty;    
    string test11 = test1;
}

private void Test2()
{
    string test2 = "";    
    string test22 = test2;
}
