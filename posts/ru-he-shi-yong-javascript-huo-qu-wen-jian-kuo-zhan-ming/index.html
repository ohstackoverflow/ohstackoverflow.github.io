解答

较新的编辑：自从最初发布这个问题以来，很多事情都发生了变化——wallacer 的修订答案以及VisioN 的出色分解中有很多非常好的信息

编辑： 仅仅因为这是公认的答案；wallacer 的回答确实好得多：

return filename.split('.').pop();

我的旧答案：

return /[^.]+$/.exec(filename);

应该做的。

编辑： 为了回应 PhiLho 的评论，请使用类似以下内容的内容：

return (/[.]/.exec(filename)) ? /[^.]+$/.exec(filename) : undefined;


更多建议

return filename.split('.').pop();

编辑：

这是我认为更有效的另一种非正则表达式解决方案：

return filename.substring(filename.lastIndexOf('.')+1, filename.length) || filename;

下面的VisioN 答案可以更好地处理一些极端情况，尤其是没有扩展名（.htaccess包括 etc ）的文件。

""它的性能非常好，并且可以通过在点之前没有点或没有字符串时返回而不是完整字符串来以一种可以说是更好的方式处理极端情况。这是一个精心设计的解决方案，尽管难以阅读。把它放在你的助手库中并使用它。

旧编辑：

如果您要遇到没有扩展名的文件或没有扩展名的隐藏文件（请参阅 VisioN 对上面 Tom 的回答的评论），那么更安全的实现将遵循这些原则

var a = filename.split(".");
if( a.length === 1 || ( a[0] === "" && a.length === 2 ) ) {
    return "";
}
return a.pop();    // feel free to tack .toLowerCase() here if you want

如果a.length是一个，它是一个没有扩展名的可见文件，即。 文件

如果a[0] === ""它a.length === 2是一个没有扩展名的隐藏文件，即。 .htaccess

这应该可以解决稍微复杂一些的问题。在性能方面，我认为这个解决方案在大多数浏览器中比正则表达式慢一点。但是，对于大多数常见用途，此代码应该完全可用。
