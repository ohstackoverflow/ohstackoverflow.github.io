Java 中的 HashMap 和 Hashtable 有什么区别？

Arya

2023-02-16 19:06

HashMapJava中的a和a有什么区别Hashtable？

哪个对于非线程应用程序更有效？



解答

HashMapJava 之间和在 Java 中有几个区别Hashtable：

  1. Hashtable是同步的，而HashMap不是。这HashMap对非线程应用程序更好，因为非同步对象通常比同步对象执行得更好。

  2. Hashtable不允许null键或值。HashMap允许一个null键和任意数量的null值。

  3. HashMap 的一个子类是LinkedHashMap，所以如果您想要可预测的迭代顺序（默认情况下是插入顺序），您可以轻松地换出 for HashMapa LinkedHashMap。如果您使用Hashtable.

由于同步对您来说不是问题，因此我建议HashMap. 如果同步成为问题，您还可以查看ConcurrentHashMap.


更多建议

请注意，很多答案都指出 Hashtable 是同步的。 在实践中，这给你带来的好处很少。 访问器/修改器方法上的同步将阻止两个线程并发地从映射中添加或删除，但在现实世界中，您通常需要额外的同步。

一个非常常见的习惯用法是“检查然后放置”——即在 中查找一个条目Map，如果它不存在则添加它。Hashtable无论您使用or ，这都不是原子操作HashMap。

HashMap可以通过以下方式获得等效同步：

Collections.synchronizedMap(myMap);

但要正确实现此逻辑，您需要对表单 进行额外的同步：

synchronized(myMap) {
    if (!myMap.containsKey("tomato"))
        myMap.put("tomato", "red");
}

即使遍历 aHashtable的条目（或HashMap由 获得的Collections.synchronizedMapa ）也不是线程安全的，除非您还防止Map通过额外的同步进行修改。

接口的实现ConcurrentMap（例如ConcurrentHashMap）通过包含 线程安全的“先检查后执行”语义 来解决其中的一些问题，例如：

ConcurrentMap.putIfAbsent(key, value);
