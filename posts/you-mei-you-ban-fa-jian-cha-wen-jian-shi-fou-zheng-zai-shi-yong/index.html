有没有办法检查文件是否正在使用？

Arya

2023-02-17 08:29

我正在用 C# 编写一个需要重复访问 1 个图像文件的程序。大多数时候它都可以工作，但如果我的电脑运行速度很快，它会在文件被保存回文件系统之前尝试访问该文件并抛出错误：

“另一个进程正在使用文件”

我想找到解决这个问题的方法，但我所有的谷歌搜索都只通过使用异常处理产生了创建检查。这违背了我的宗教信仰，所以我想知道是否有人有更好的方法来做到这一点？



解答

更新了此解决方案的注意事项 ：检查FileAccess.ReadWrite只读文件将失败，因此已修改解决方案以检查FileAccess.Read。

原文： 在过去的几年里，我一直在使用这段代码，而且我没有遇到任何问题。

理解您对使用异常的犹豫，但您无法一直避免它们：

protected virtual bool IsFileLocked(FileInfo file)
{
    try
    {
        using(FileStream stream = file.Open(FileMode.Open, FileAccess.Read, FileShare.None))
        {
            stream.Close();
        }
    }
    catch (IOException)
    {
        //the file is unavailable because it is:
        //still being written to
        //or being processed by another thread
        //or does not exist (has already been processed)
        return true;
    }

    //file is not locked
    return false;
}


更多建议

您可能会遇到线程竞争条件，有记录的示例将其用作安全漏洞。如果您检查该文件是否可用，然后尝试使用它，您可能会在此时抛出，恶意用户可能会使用它来强制和利用您的代码。

最好的选择是尝试获取文件句柄的 try catch / finally 。

try
{
   using (Stream stream = new FileStream("MyFilename.txt", FileMode.Open))
   {
        // File/Stream manipulating code here
   }
} catch {
  //check here why it failed and ask user to retry if the file is in use.
}
