没有微秒组件的Python日期时间到字符串

Arya

2023-02-18 05:41

我正在将 UTC 时间字符串添加到目前仅包含阿姆斯特丹 (!) 时间字符串的 Bitbucket API 响应中。为了与其他地方返回的 UTC 时间字符串保持一致，所需的格式是2011-11-03 11:07:04（后跟+00:00，但这并不相关）。

从 具有 微秒组件的实例创建这样的字符串（ 没有 微秒组件）的最佳方法是什么？datetime __

 >>> import datetime
>>> print unicode(datetime.datetime.now())
2011-11-03 11:13:39.278026

我将添加我想到的最佳选项作为可能的答案，但很可能有更优雅的解决方案。

编辑： 我应该提一下，我 实际上 并没有打印当前时间——我曾经datetime.now提供过一个简单的例子。因此，该解决方案不应假定datetime它接收到的任何实例都将包含微秒组件。



解答

如果要以datetime不同于标准格式的特定格式格式化对象，最好明确指定该格式：

>>> import datetime
>>> datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
'2011-11-03 18:21:26'

有关指令的解释，请参阅的文档datetime.strftime()``%。

从 Python 3.6 开始，该isoformat()方法足够灵活，也可以生成这种格式：

datetime.datetime.now().isoformat(sep=" ", timespec="seconds")


更多建议

>>> import datetime
>>> now = datetime.datetime.now()
>>> print unicode(now.replace(microsecond=0))
2011-11-03 11:19:07
