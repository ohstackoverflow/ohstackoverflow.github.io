MySQL 错误 #1071 - 指定的密钥太长；最大密钥长度为 767 字节

Arya

2023-02-17 20:06

当我执行以下命令时：

ALTER TABLE `mytable` ADD UNIQUE (
`column1` ,
`column2`
);

我收到此错误消息：

#1071 - Specified key was too long; max key length is 767 bytes

column1 和 column2 的相关信息：

column1 varchar(20) utf8_general_ci
column2  varchar(500) utf8_general_ci

我认为varchar(20)只需要 21 个字节而varchar(500)只需要 501 个字节。所以总字节数是 522，小于 767。那么为什么我会收到错误消息？

#1071 - Specified key was too long; max key length is 767 bytes



解答

MySQL 5.6 版 （及之前的版本）中的 767 字节是InnoDB 表的规定前缀限制。MyISAM 表的长度为 1,000 字节。 在 MySQL 5.7 版 （及更高版本）中，此限制已增加到3072 字节。

varchar您还必须注意，如果您在编码的大字符或字段上设置索引utf8mb4，则必须将 767 字节（或 3072 字节）的最大索引前缀长度除以 4，得到 191 。这是因为一个字符的最大长度utf8mb4是四个字节。对于一个utf8字符，它将是三个字节，导致最大索引前缀长度为 255 （或减去空终止符，254 个字符）。

您的一个选择是只对您的字段设置下限VARCHAR。

另一种选择（根据对这个问题的回应）是获取列的子集而不是全部，即：

ALTER TABLE `mytable` ADD UNIQUE ( column1(15), column2(200) );

调整您需要获得应用的密钥，但我想知道是否值得审查您关于该实体的数据模型以查看是否有可能进行改进，这将允许您在不达到 MySQL 限制的情况下实施预期的业务规则.


更多建议

当你达到极限时。设定以下。

  • 创新数据库utf8 VARCHAR(255)
  • 创新数据库utf8mb4 VARCHAR(191)
