Bootstrap 中的 col-lg-*、col-md-* 和 col-sm-* 有什么区别？

Arya

2023-02-18 02:06

col-lg-*和Twitter Bootstrapcol-md-*之间有什么区别col-sm-*？



解答

2020 年更新...

引导程序 5

在 Bootstrap 5 (alpha) 中有一个新的-xxl-大小：

col-*- 0 (xs)
col-sm-*- 576px
col-md-*- 768px
col-lg-*- 992px
col-xl-*- 1200px
col-xxl-*- 1400px

Bootstrap 5 网格演示

引导程序 4

在 Bootstrap 4 中有一个新的-xl-大小，请参阅此演示。此外， -xs- 中缀已被删除，因此最小的列只是col-1, col-2..col-12等。

col-*- 0 (xs)
col-sm-*- 576px
col-md-*- 768px
col-lg-*- 992px
col-xl-*- 1200px

Bootstrap 4 网格演示

此外，Bootstrap 4 包括新的自动布局列。这些也有响应断点（col、col-sm、col-md等），但没有定义 % 宽度。因此，自动布局列在行中填充 相等的宽度。

引导程序 3

Bootstrap 3 网格分为 4 层（或“断点”）...

  • 超小（适用于智能手机.col-xs-*）
  • 小号（适用于平板电脑.col-sm-*）
  • 中等（适用于笔记本电脑.col-md-*）
  • 大号（适用于笔记本电脑/台式机.col-lg-*）。

这些网格大小使您能够控制不同宽度的网格行为。不同的层由 CSS媒体查询控制。

所以在 Bootstrap 的 12 列网格中......

col-sm-3 在典型的小 设备宽度（> 768 像素）上是 12 列中的 3 列宽 (25%)

col-md-3 在典型的中等 设备宽度（> 992 像素）上是 12 列中的 3 列宽 (25%)

较小的层（xs或sm）md也定义较大屏幕宽度的尺寸。因此，对于所有层上 相同 大小的列，只需设置最小视口的宽度...

<div class="col-lg-3 col-md-3 col-sm-3">..</div>是相同的，

<div class="col-sm-3">..</div>

隐含了更大的层级 。 因为col-sm-3意味着3 units on sm-and-up，除非被使用不同大小的更大层明确覆盖。

xs（默认）> 被覆盖sm> 被覆盖md> 被覆盖lg

组合 这些类以在 不同的 网格大小上使用更改列宽。这将创建响应式布局。

<div class="col-md-3 col-sm-6">..</div>

sm和网格将在小于 768 像素的屏幕/视口上垂直“堆叠” md。lg这就是xs网格适合的地方。使用col-xs-*类的列不会垂直 堆叠 ，并会在最小的屏幕上继续缩小。

使用此演示调整浏览器大小，您将看到网格缩放效果。

bootstrap文档确实对此进行了解释，但我还是花了一些时间才弄明白。当我以两种方式之一向自己解释时，它更有意义：

如果您认为列是水平开始的，那么您可以选择何时让它们堆叠 。

例如，如果您从列开始：ABC

你决定他们什么时候应该像这样堆叠：

A

乙

C

如果您选择 col-lg，那么当宽度 < 1200px 时，列将堆叠。

如果您选择 col-md，那么当宽度 < 992px 时，列将堆叠。

如果您选择 col-sm，那么当宽度 < 768px 时，列将堆叠。

如果您选择 col-xs，那么列将永远不会堆叠。

另一方面，如果您认为列开始是堆叠的，那么您可以选择它们在什么时候变成水平的 ：

如果您选择 col-sm，那么当宽度 >= 768px 时，列将变成水平的。

如果您选择 col-md，那么当宽度 >= 992px 时，列将变成水平的。

如果您选择 col-lg，那么当宽度 >= 1200px 时，列将变为水平。
