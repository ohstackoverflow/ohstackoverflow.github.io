何时在 C++ 中使用 extern

Arya

2023-02-18 07:05

我正在阅读“用 C++ 思考”，它只是介绍了声明extern。例如：

extern int x;
extern float y;

我想我明白了意思（没有定义的声明），但我想知道它什么时候有用。

有人可以举个例子吗？



解答

当你有全局变量时，这很有用。您在标头中声明全局变量的 存在 ，以便每个包含标头的源文件都知道它，但您只需要在其中一个源文件中“定义”它一次。

澄清一下，usingextern int x;告诉编译器类型为intcalled的对象x存在于 某处 。知道它存在的位置不是编译器的工作，它只需要知道类型和名称以便知道如何使用它。x一旦所有源文件都被编译，链接器将解析对它在其中一个已编译源文件中找到的一个定义的所有引用。为了让它工作，变量的定义x需要有所谓的“外部链接”，这基本上意味着它需要在函数外部（通常称为“文件范围”）声明，并且没有关键字static。

标头：

#ifndef HEADER_H
#define HEADER_H

// any source file that includes this will be able to use "global_x"
extern int global_x;

void print_global_x();

#endif

来源 1：

#include "header.h"

// since global_x still needs to be defined somewhere,
// we define it (for example) in this source file
int global_x;

int main()
{
    //set global_x here:
    global_x = 5;

    print_global_x();
}

来源 2：

#include <iostream>
#include "header.h"

void print_global_x()
{
    //print global_x here:
    std::cout << global_x << std::endl;
}


更多建议

当您在几个模块之间共享一个变量时，它很有用。您在一个模块中定义它，并在其他模块中使用 extern。

例如：

在 file1.cpp 中：

int global_int = 1;

在 file2.cpp 中：

extern int global_int;
//in some function
cout << "global_int = " << global_int;
