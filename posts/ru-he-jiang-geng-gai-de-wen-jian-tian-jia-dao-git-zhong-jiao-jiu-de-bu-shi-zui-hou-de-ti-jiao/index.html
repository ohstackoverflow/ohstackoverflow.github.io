解答

使用git rebase。具体来说：

  1. 用于git stash存储要添加的更改。
  2. 使用git rebase -i HEAD~10（或者无论你想看到多少次提交）。
  3. 通过将行开头的a0865...单词更改为 来标记有问题的提交 () 以进行编辑。不要删除其他行，因为那样会删除提交。[^vimnote]pick``edit
  4. 保存 rebase 文件，git 将返回到 shell 并等待您修复该提交。
  5. 使用弹出存储git stash pop
  6. 添加您的文件git add <file>。
  7. 用 修改提交git commit --amend --no-edit。
  8. 做一个git rebase --continue将根据新提交重写其余提交的操作。
  9. 如果您标记了多个提交进行编辑，请从第 2 步开始重复。

[^vimnote]：如果你正在使用vim，那么你必须按下Insert键进行编辑，然后Esc输入:wq以保存文件，退出编辑器，并应用更改。或者，您可以使用配置一个用户友好的 git 提交编辑器git config --global core.editor "nano"。


更多建议

在不更改旧提交的提交消息的情况下，通过一个小的更改来“修复”旧提交，其中OLDCOMMIT类似于091b73a：

git add <my fixed files>
git commit --fixup=OLDCOMMIT
git rebase --interactive --autosquash OLDCOMMIT^

您还可以git commit --squash=OLDCOMMIT在变基期间用于编辑旧的提交消息。

请参阅git commit和git rebase的文档。与往常一样，在重写 git 历史时，你应该只修复或压缩你尚未发布给其他任何人（包括随机互联网用户和构建服务器）的提交。

详细解释

  • git commit --fixup=OLDCOMMIT复制OLDCOMMIT提交消息并自动添加前缀fixup!，以便在交互式变基期间可以将其放入正确的顺序。（--squash=OLDCOMMIT做同样的事情，但前缀squash!。）
  • git rebase --interactive将调出一个文本编辑器（可以配置）来确认（或编辑） 变基指令序列 。文件中有rebase 指令更改的信息；只需 保存并退出 编辑器 ( :wqinvim ) 即可继续进行变基。
  • --autosquash``--fixup=OLDCOMMIT将自动以正确的顺序放置任何提交。请注意，--autosquash只有在--interactive使用该选项时才有效。
  • in表示它是对之前提交^的OLDCOMMIT^引用OLDCOMMIT。(OLDCOMMIT^是 . 的第一个父母OLDCOMMIT。)

可选自动化

上述步骤适用于验证和/或修改 变基指令序列 ，但也可以通过以下方式跳过/自动化交互式变基文本编辑器：

  • 设置GIT_SEQUENCE_EDITOR为脚本。
  • 创建一个git 别名来自动自动压缩所有排队的修正。
  • 创建一个git 别名来自动修复单个提交。
