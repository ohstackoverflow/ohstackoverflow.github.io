直接铸造与'as'运算符？

Arya

2023-02-17 11:50

考虑以下代码：

void Handler(object o, EventArgs e)
{
   // I swear o is a string
   string s = (string)o; // 1
   //-OR-
   string s = o as string; // 2
   // -OR-
   string s = o.ToString(); // 3
}

三种类型的铸造之间有什么区别（好吧，第三种不是铸造，但你明白了）。应该首选哪一个？



解答

string s = (string)o; // 1

如果不是 则抛出InvalidCastException。否则，分配给，即使是。o``string``o``s``o``null

string s = o as string; // 2

分配null给sif ois not astring或 if ois null。因此，您不能将它与值类型一起使用（null在这种情况下，运算符永远不会返回）。否则，分配o给s.

string s = o.ToString(); // 3

如果是 ，则导致NullReferenceException。将任何返回分配给，无论类型是什么。o``null``o.ToString()``s``o

大多数转换使用 1 - 它简单明了。我倾向于几乎从不使用 2，因为如果某些东西的类型不正确，我通常预计会发生异常。我只看到使用错误代码的设计糟糕的库需要这种返回 null 类型的功能（例如，返回 null = 错误，而不是使用异常）。

3 不是强制转换，只是一个方法调用。当您需要非字符串对象的字符串表示时使用它。


更多建议

  1. string s = (string)o; 当某件事绝对 应该是另一件事时使用。
  2. string s = o as string; 当某事可能是 另一件事时使用。
  3. string s = o.ToString();当您不关心它是什么但只想使用可用的字符串表示形式时使用。
