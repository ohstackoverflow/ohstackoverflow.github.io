将 div 放入锚中是否正确？

Arya

2023-02-18 01:15

我听说将块元素放在行内元素中是一种 HTML 错误：

<a href="http://example.com">
    <div>
        What we have here is a problem. 
        You see, an anchor element is an inline element,
        and the div element is a block level element.
    </div>
</a>

display:block但是，如果您像样式表中那样设置外部锚点的样式怎么办？还是错了？块级和内联元素的 HTML 4.01 规范似乎是这么认为的：

样式表提供了指定任意元素呈现的方法，包括元素是呈现为块还是内联。在某些情况下，例如列表元素的内联样式，这可能是合适的，但一般来说，不鼓励作者以这种方式覆盖 HTML 元素的常规解释。

有没有人对这个问题有任何进一步的提示？



解答

根据您要使用的 HTML 版本：

  • HTML 5 声明该<a>元素“可以环绕整个段落、列表、表格等，甚至整个部分，只要其中没有交互式内容（例如按钮或其他链接）”。

  • HTML 4.01 规定<a>元素只能包含内联元素。A<div>是块元素，因此它可能不会出现在 an 中<a>。

当然，您可以自由地设置内联元素的样式，使其 看起来 像一个块，或者确实设置一个块的样式，使其呈现为内联。术语inline和block在 HTML 中的使用指的是元素与文档语义结构的关系，而 CSS 中的相同术语更多地与元素的视觉样式相关。如果您使内联元素以块状方式显示，那很好。

但是，您应该确保当 CSS 不存在时文档的结构仍然有意义，例如当通过屏幕阅读器等辅助技术访问时 - 或者实际上当被强大的 Googlebot 检查时。


更多建议

不，它不会验证，但是它通常可以在现代浏览器中工作。话虽这么说，在你的锚中使用一个跨度，并display: block在它上面设置，这肯定会在任何地方工作，并且它会验证！
