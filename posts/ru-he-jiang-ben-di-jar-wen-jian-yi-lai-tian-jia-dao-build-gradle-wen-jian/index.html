如何将本地 .jar 文件依赖添加到 build.gradle 文件？

Arya

2023-02-17 14:01

我试图将我的本地 .jar 文件依赖项添加到我的 build.gradle 文件中：

apply plugin: 'java'

sourceSets {
    main {
        java {
            srcDir 'src/model'
        }
    }
}

dependencies {
    runtime files('libs/mnist-tools.jar', 'libs/gson-2.2.4.jar')
    runtime fileTree(dir: 'libs', include: '*.jar')
}

您可以看到我将 .jar 文件添加到此处的 referencedLibraries 文件夹中： https: //github.com/WalnutiQ/wAlnut/tree/version-2.3.1/referencedLibraries

但问题是，当我在命令行上运行命令：gradle build 时，出现以下错误：

error: package com.google.gson does not exist
import com.google.gson.Gson;

这是我的整个回购协议：https://github.com/WalnutiQ/wAlnut/tree/version-2.3.1



解答

如果你真的需要从本地目录中获取那个 .jar，

在您的模块 gradle 旁边添加（不是应用程序 gradle 文件）：

repositories {
   flatDir {
       dirs("libs")
   }
}


dependencies {
   implementation("gson-2.2.4")
}

但是，作为实际 Maven 存储库中的标准 .jar，您为什么不试试这个呢？

repositories {
   mavenCentral()
}
dependencies {
   implementation("com.google.code.gson:gson:2.2.4")
}
