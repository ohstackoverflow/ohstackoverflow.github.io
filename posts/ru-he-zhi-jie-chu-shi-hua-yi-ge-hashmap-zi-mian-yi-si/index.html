如何直接初始化一个HashMap（字面意思）？

Arya

2023-02-17 01:42

有没有办法像这样初始化 Java HashMap？：

Map<String,String> test = 
    new HashMap<String, String>{"test":"test","test":"test"};

正确的语法是什么？我还没有发现任何关于此的信息。这可能吗？我正在寻找最短/最快的方法来将一些“最终/静态”值放在地图中，这些值永远不会改变并且在创建地图时预先知道。



解答

所有版本

如果您碰巧只需要一个条目：有Collections.singletonMap("key", "value").

对于 Java 版本 9 或更高版本：

是的，现在这是可能的。在 Java 9 中添加了几个工厂方法来简化映射的创建：

// this works for up to 10 elements:
Map<String, String> test1 = Map.of(
    "a", "b",
    "c", "d"
);

// this works for any number of elements:
import static java.util.Map.entry;    
Map<String, String> test2 = Map.ofEntries(
    entry("a", "b"),
    entry("c", "d")
);

在上面的示例中，test和test2将是相同的，只是表达 Map 的方式不同。该Map.of方法是为地图中最多十个元素定义的，而该Map.ofEntries方法则没有这样的限制。

请注意，在这种情况下，生成的映射将是不可变映射。如果您希望地图可变，您可以再次复制它，例如使用mutableMap = new HashMap<>(Map.of("a", "b"));. 另请注意，在这种情况下，键和值不能是null.

（另请参阅JEP 269和Javadoc）

对于 Java 版本 8：

不，您必须手动添加所有元素。您可以在匿名子类中使用初始化程序来缩短语法：

Map<String, String> myMap = new HashMap<String, String>() {{
    put("a", "b");
    put("c", "d");
}};

但是，匿名子类在某些情况下可能会引入不需要的行为。这包括例如：

  • 它生成一个额外的类，增加内存消耗、磁盘空间消耗和启动时间
  • 在非静态方法的情况下：它持有对调用创建方法的对象的引用。这意味着当创建的映射对象仍然被引用时，外部类的对象不能被垃圾回收，从而阻塞额外的内存

使用函数进行初始化还可以让您在初始化程序中生成映射，但可以避免讨厌的副作用：

Map<String, String> myMap = createMap();

private static Map<String, String> createMap() {
    Map<String,String> myMap = new HashMap<String,String>();
    myMap.put("a", "b");
    myMap.put("c", "d");
    return myMap;
}


更多建议

这是一种方式。

Map<String, String> h = new HashMap<String, String>() {{
    put("a","b");
}};

但是，您应该小心并确保您理解上面的代码（它创建了一个继承自 HashMap 的新类）。因此，您应该在这里阅读更多内容：http://www.c2.com/cgi/wiki? DoubleBraceInitialization ，或者简单地使用 Guava：

Map<String, Integer> left = ImmutableMap.of("a", 1, "b", 2, "c", 3);

ImmutableMap.of适用于最多 5 个条目。否则，使用构建器：source。
