（以下假设github.com自己 没有 关闭，正如eri0o在评论中指出的那样：www.githubstatus.com确保看到）

如果 GitHub 存储库看到新的提交推送到它，而你在本地工作，我建议使用：

git pull --rebase
git push

完整的语法是：

git pull --rebase origin main
git push origin main

使用 Git 2.6+（2015 年 9 月），完成后（一次）

git config --global pull.rebase true
git config --global rebase.autoStash true

一个简单的git pull就够了。
（注意：使用Git 2.27 Q2 2020，amerge.autostash也可用于您的常规拉取，无需变基）

这样，您将在新更新的（或: ）--rebase之上重播（部分）您的本地提交。origin/main``origin/yourBranch``git pull origin yourBranch

请参阅Git Pocket Book的第 6 章使用 rebase 拉取更完整的示例。

我会推荐一个：

# add and commit first
#
git push -u origin main

# Or git 2.37 Q2 2022+
git config --global push.autoSetupRemote true
git push

这将在您的本地主分支及其上游分支之间建立跟踪关系。
之后，该分支的任何未来推送都可以通过简单的方式完成：

git push

同样，使用Git 2.37+ 及其全局选项push.autoSetupRemote，git push即使对于第一个选项也很简单（即：在本地main分支与其上游分支之间建立跟踪关系origin/main）。

请参阅“为什么我需要显式推送一个新分支？ ”。

由于 OP 已经在以下位置重置并重做其提交origin/main：

git reset --mixed origin/main
git add .
git commit -m "This is a new commit for what I originally planned to be amended"
git push origin main

没有必要pull --rebase。

注意：git reset --mixed origin/main也可以写成git reset origin/main，因为该--mixed选项是使用时的默认选项git reset。
