Moment.js 转换为日期对象

Arya

2023-02-17 10:26

使用 Moment.js 我无法将正确的时刻对象转换为具有时区的日期对象。我无法获得正确的日期。

例子：

var oldDate = new Date(),
    momentObj = moment(oldDate).tz("MST7MDT"),
    newDate = momentObj.toDate();
console.log("start date " + oldDate)
console.log("Format from moment with offset " + momentObj.format())
console.log("Format from moment without offset " + momentObj.utc().format())
console.log("(Date object) Time with offset " + newDate)
console.log("(Date object) Time without offset "+ moment.utc(newDate).toDate())



解答

使用它来将 moment 对象转换为 date 对象：

来自http://momentjs.com/docs/#/displaying/as-javascript-date/

moment().toDate();

产量：

Tue Nov 04 2014 14:04:01 GMT-0600 (CST)


更多建议

只要您使用所需区域的数据初始化了moment-timezone，您的代码就会按预期工作。

您正在正确地将时间转换为时区，这反映在 的第二行输出中momentObj.format()。

切换到 UTC 不仅会降低偏移量，还会变回 UTC 时区。如果您要这样做，则.tz()根本不需要原始电话。你可以这样做moment.utc()。

也许您只是想更改输出格式字符串？如果是这样，只需指定您想要的方法参数format：

momentObj.format("YYYY-MM-DD HH:mm:ss")

关于代码的最后几行 - 当您使用 返回对象时Date，toDate()您将放弃 moment.js 的行为并返回到 JavaScript 的行为。JavaScriptDate对象将始终在其运行的计算机的本地时区打印。moment.js 对此无能为力。

其他一些小事：

  • 虽然 moment constructor 可以 使用Date，但通常最好不要使用。对于“现在”，不要使用moment(new Date()). 相反，只需使用moment(). 两者都可以工作，但这是不必要的冗余。如果您从字符串中解析，请将该字符串直接传递给 moment. 不要试图首先解析它Date。您会发现 moment 的解析器更加可靠。

  • 像这样的时区MST7MDT是出于向后兼容的原因。它们源于 POSIX 风格的时区，只有少数在 TZDB 数据中。除非绝对必要，否则您应该使用诸如America/Denver.
