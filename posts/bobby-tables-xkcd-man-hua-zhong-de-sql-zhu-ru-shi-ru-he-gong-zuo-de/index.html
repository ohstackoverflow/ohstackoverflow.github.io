“Bobby Tables”XKCD 漫画中的 SQL 注入是如何工作的？

Arya

2023-02-17 04:28

只是看着：

（来源： https: //xkcd.com/327/）

这个 SQL 做了什么：

Robert'); DROP TABLE STUDENTS; --

我都知道'并且--正在征求意见，但是这个词是否也DROP得到评论，因为它是同一行的一部分？



解答

它删除了学生表。

学校程序中的原始代码可能看起来像

q = "INSERT INTO Students VALUES ('" + FNMName.Text + "', '" + LName.Text + "')";

这是将文本输入添加到查询中的幼稚方法，而且 非常糟糕 ，正如您将看到的那样。

在名字、中间名文本框 FNMName.Text （即Robert'); DROP TABLE STUDENTS; --）和姓氏文本框 LName.Text （我们称之为Derper）的值与查询的其余部分连接后，结果现在实际上是 两个查询， 由语句终止符（分号）。第二个查询已被 注入 到第一个中。当代码对数据库执行此查询时，它将如下所示

INSERT INTO Students VALUES ('Robert'); DROP TABLE Students; --', 'Derper')

其中，用简单的英语，大致翻译成两个查询：

将名称值为“Robert”的新记录添加到 Students 表

和

删除学生表

第二个查询之后的所有内容都标记为评论：--', 'Derper')

'学生姓名中的 不是注释，它是结束字符串分隔符。由于学生的姓名是一个字符串，因此在句法上需要它来完成假设的查询。注入攻击仅 在它们注入的 SQL 查询导致有效 SQL 时才起作用 。

更多建议

假设名称在变量中使用，$Name. 然后运行此查询：

INSERT INTO Students VALUES ( '$Name' )

该代码错误地将用户提供的任何内容放置为变量。你希望 SQL 是：

插入学生值 (' `Robert Tables`` )

但是聪明的用户可以提供他们想要的任何东西：

插入学生值 (' Robert'); DROP TABLE Students; --')

你得到的是：

INSERT INTO Students VALUES ( 'Robert' );  DROP TABLE STUDENTS; --' )

The --only 评论该行的其余部分。
