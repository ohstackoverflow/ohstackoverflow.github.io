在 SQL Server 中应该选择 MONEY 还是 DECIMAL(x,y) 数据类型？

Arya

2023-02-18 07:01

money我很好奇数据类型和类似的东西之间是否存在真正的区别decimal(19,4)（我相信这是金钱在内部使用的东西）。

我知道这money是特定于 SQL Server 的。我想知道是否有令人信服的理由选择其中一个；大多数 SQL Server 示例（例如 AdventureWorks 数据库）使用moneyand 不decimal用于诸如价格信息之类的事情。

我应该继续使用 money 数据类型，还是改用 decimal 有好处吗？金钱意味着输入的字符更少，但这不是一个正当理由 :)



解答

永远不要用钱。它不精确，纯粹是垃圾；始终使用十进制/数字。

运行这个看看我的意思：

DECLARE
    @mon1 MONEY,
    @mon2 MONEY,
    @mon3 MONEY,
    @mon4 MONEY,
    @num1 DECIMAL(19,4),
    @num2 DECIMAL(19,4),
    @num3 DECIMAL(19,4),
    @num4 DECIMAL(19,4)

    SELECT
    @mon1 = 100, @mon2 = 339, @mon3 = 10000,
    @num1 = 100, @num2 = 339, @num3 = 10000

    SET @mon4 = @mon1/@mon2*@mon3
    SET @num4 = @num1/@num2*@num3

    SELECT @mon4 AS moneyresult,
    @num4 AS numericresult

输出：2949.0000 2949.8525

更多建议

SQLMenace 说钱是不准确的。但是你不能用钱乘/除钱！3 美元乘以 50 美分是多少？150 美分？您将钱乘以/除以标量，标量应该是小数。

DECLARE
@mon1 MONEY,
@mon4 MONEY,
@num1 DECIMAL(19,4),
@num2 DECIMAL(19,4),
@num3 DECIMAL(19,4),
@num4 DECIMAL(19,4)

SELECT
@mon1 = 100,
@num1 = 100, @num2 = 339, @num3 = 10000

SET @mon4 = @mon1/@num2*@num3
SET @num4 = @num1/@num2*@num3

SELECT @mon4 AS moneyresult,
@num4 AS numericresult

结果正确：

moneyresult 数值结果
-------------------------------------------------- ----------
2949.8525 2949.8525

money只要您不需要超过 4 个十进制数字就很好，并且您确保您的标量（不代表金钱）是decimals。
