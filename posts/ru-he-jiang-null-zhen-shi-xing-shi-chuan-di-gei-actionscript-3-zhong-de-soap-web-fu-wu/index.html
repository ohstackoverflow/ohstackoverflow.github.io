如何将“Null”（真实姓氏！）传递给 ActionScript 3 中的 SOAP Web 服务

Arya

2023-02-16 18:54

我们有一个姓 Null 的员工。当姓氏被用作搜索词时（现在这种情况经常发生），我们的员工查找应用程序就会被终止。收到的错误（感谢 Fiddler！）是：

<soapenv:Fault>
   <faultcode>soapenv:Server.userException</faultcode>
   <faultstring>coldfusion.xml.rpc.CFCInvocationException: [coldfusion.runtime.MissingArgumentException : The SEARCHSTRING parameter to the getFacultyNames function is required but was not passed in.]</faultstring>

可爱吧？

参数类型为string.

我在用：

  • WSDL（肥皂）
  • 弹性 3.5
  • 动作脚本 3
  • 冷聚变 8

请注意，当从 ColdFusion 页面将 Web 服务作为对象调用时， 不会 发生该错误。



解答

追踪它

起初我以为这是一个强制错误，它null被强制执行"null"并通过了测试"null" == null。它不是。 我很接近，但非常非常错误。 对于那个很抱歉！

从那以后，我在 wonderfl.net 上做了很多摆弄，并跟踪了mx.rpc.xml.*. 在XMLEncoder（在 3.5 源代码中）的第 1795 行，在 中setValue，所有 XMLEncoding 归结为

currentChild.appendChild(xmlSpecialCharsFilter(Object(value)));

这与以下内容基本相同：

currentChild.appendChild("null");

根据我原来的小提琴，这段代码返回一个空的 XML 元素。但为什么？

原因

根据评论员 Justin Mclean 在错误报告FLEX-33664中的说法，以下是罪魁祸首（请参阅我的小提琴中的最后两个测试来验证这一点）：

var thisIsNotNull:XML = <root>null</root>;
if(thisIsNotNull == null){
    // always branches here, as (thisIsNotNull == null) strangely returns true
    // despite the fact that thisIsNotNull is a valid instance of type XML
}

当currentChild.appendChild传递 string时"null"，它首先将其转换为带有 text 的根 XML 元素null，然后根据 null 文字测试该元素。这是一个弱相等性测试，因此包含 null 的 XML 被强制转换为 null 类型，或者 null 类型被强制转换为包含字符串“null”的根 xml 元素，并且测试通过了它可能会失败的地方。一种解决方法可能是在检查 XML（或其他任何东西）是否为“空值”时始终使用严格的相等性测试。

解决方案

除了在每个该死的 ActionScript 版本中修复此错误之外，我能想到的唯一合理的解决方法是测试字段是否为“null”并将 它们转义为 CDATA 值。

CDATA 值是改变整个文本值的最合适方法，否则会导致编码/解码问题。 例如，十六进制编码适用于单个字符。当您转义元素的整个文本时，首选 CDATA 值。这样做的最大原因是它保持了人类的可读性。


更多建议

在xkcd note上，Bobby Tables 网站有很好的建议，可以避免在各种语言（包括ColdFusion）的 SQL 查询中对用户数据（在本例中为字符串“Null”）的不当解释。

从问题中不清楚这是问题的根源，并且考虑到第一个答案的评论中指出的解决方案（将参数嵌入结构中）似乎可能是其他原因。
