我应该在 MySQL 中使用 datetime 还是 timestamp 数据类型？

Arya

2023-02-16 20:10

您会推荐使用日期时间还是时间戳字段，为什么（使用 MySQL）？

我在服务器端使用 PHP。



解答

MySQL 中的时间戳一般用于跟踪记录的变化，并且经常在每次记录变化时更新。如果要存储特定值，则应使用日期时间字段。

如果您的意思是要在使用 UNIX 时间戳或本机 MySQL 日期时间字段之间做出决定，请使用本机格式DATETIME。您可以通过这种方式在 MySQL 中进行计算，如果您想使用 PHP 对其进行操作，则在查询记录时将("SELECT DATE_ADD(my_datetime, INTERVAL 1 DAY)")值的格式更改为 UNIX 时间戳很简单。("SELECT UNIX_TIMESTAMP(my_datetime)")


更多建议

在 MySQL 5 及以上版本中， TIMESTAMP 值从当前时区转换为 UTC 进行存储，并从 UTC 转换回当前时区进行检索。（这只发生在 TIMESTAMP 数据类型上，而 不会 发生在其他类型上，例如 DATETIME。）

默认情况下，每个连接的当前时区是服务器的时间。可以在每个连接的基础上设置时区，如 MySQL 服务器时区支持 中所述。
