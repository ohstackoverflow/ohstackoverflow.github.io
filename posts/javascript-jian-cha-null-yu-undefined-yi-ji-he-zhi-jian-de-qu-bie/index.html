null... 和之间有什么区别undefined？

它们都是通常用来表示某物不存在的值。undefined是更通用的，用作变量的默认值，直到它们被分配一些其他值，作为调用函数时未提供的函数参数的值，以及当您询问对象时获得的值对于它没有的属性。但它也可以明确地用于所有这些情况。（没有属性的对象与具有 value 的属性之间存在差异；使用参数值undefined调用函数与完全不使用该参数之间存在差异。）undefined

null比 稍微更具体undefined：它是一个空白对象引用。JavaScript 是松散类型的，当然，但并非所有与 JavaScript 交互的东西都是松散类型的。如果像浏览器中的 DOM 这样的 API 需要一个空白的对象引用，我们使用null, 而不是undefined. 类似地，DOM 的getElementById操作返回一个对象引用——一个有效的引用（如果它找到了 DOM 元素），或者null（如果没有）。

有趣的是（或不是），它们是它们自己的类型。也就是说，null是Null类型中的唯一值，undefined也是Undefined类型中的唯一值。

“==”和“===”有什么区别

它们之间的唯一区别是==将进行类型强制以尝试使值匹配，而===不会。所以例如"1" == 1是真的，因为"1"强制到1. 但是"1" === 1是 false ，因为类型不匹配。（"1" !== 1是真的。）的第一步（真正的）===是“操作数的类型是否相同？” 如果答案为“否”，则结果为false. 如果类型相同，它会完全按照原样执行==。

类型强制使用相当复杂的规则并且可以产生令人惊讶的结果（例如，"" == 0为真）。

区别很微妙。

在 JavaScript 中，undefined变量是从未声明过或从未赋值的变量。假设您声明var a;例如，然后a将是undefined，因为它从未被分配任何值。

但是如果你然后分配a = null;那么a现在将是null。在 JavaScript 中null是一个对象（typeof null如果您不相信我，请在 JavaScript 控制台中尝试），这意味着 null 是一个值（实际上 evenundefined是一个值）。

例子：

var a;
typeof a;     # => "undefined"

a = null;
typeof null;  # => "object"

这在函数参数中很有用。您可能想要一个默认值，但认为 null 是可以接受的。在这种情况下你可以这样做：

function doSomething(first, second, optional) {
    if (typeof optional === "undefined") {
        optional = "three";
    }
    // do something
}

如果您省略optional参数doSomething(1, 2) thenoptional 将是"three"字符串，但如果您传递doSomething(1, 2, null)则 optional 将是null。

至于相等==和严格相等===比较器，第一个是弱类型，而严格相等也会检查值的类型。这意味着0 == "0"它将返回 true；while0 === "0"将返回 false，因为数字不是字符串。

您可以使用这些运算符undefined在null. 例如：

null === null            # => true
undefined === undefined  # => true
undefined === null       # => false
undefined == null        # => true

最后一种情况很有趣，因为它允许您检查变量是否为未定义或 null 而不是其他任何东西：

function test(val) {
    return val == null;
}
test(null);       # => true
test(undefined);  # => true
