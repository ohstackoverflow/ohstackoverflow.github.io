如何附加到文件？

Arya

2023-02-16 23:17

如何附加到文件而不是覆盖它？



解答

将模式设置open()为"a"（附加）而不是"w"（写入）：

with open("test.txt", "a") as myfile:
    myfile.write("appended text")

该文档列出了所有可用的模式。


更多建议

您需要通过将“a”或“ab”设置为模式来以追加模式打开文件。参见 open() 。

当您使用“a”模式打开时，写入位置将 始终 位于文件末尾（追加）。您可以使用“a+”打开以允许读取、向后查找和读取（但所有写入仍将位于文件末尾！）。

例子：

>>> with open('test1','wb') as f:
        f.write('test')
>>> with open('test1','ab') as f:
        f.write('koko')
>>> with open('test1','rb') as f:
        f.read()
'testkoko'

注意 ：使用 'a' 与使用 'w' 打开并查找到文件末尾不同 - 考虑如果另一个程序打开文件并在查找和写入之间开始写入可能会发生什么。在某些操作系统上，使用“a”打开文件可以保证所有后续写入都将自动附加到文件末尾（即使文件因其他写入而增长）。

关于“a”模式如何运行的更多细节（ 仅在 Linux 上测试 ）。即使您向后查找，每次写入都会附加到文件末尾：

>>> f = open('test','a+') # Not using 'with' just to simplify the example REPL session
>>> f.write('hi')
>>> f.seek(0)
>>> f.read()
'hi'
>>> f.seek(0)
>>> f.write('bye') # Will still append despite the seek(0)!
>>> f.seek(0)
>>> f.read()
'hibye'

事实上，fopen 联机帮助页指出：

以追加模式（a 作为模式的第一个字符）打开文件会导致对该流的所有后续写入操作发生在文件末尾，就像在调用之前一样：

fseek(stream, 0, SEEK_END);

旧的简化答案（不使用with）：

示例：（ 在真实程序中 用于with关闭文件- 请参阅文档）

>>> open("test","wb").write("test")
>>> open("test","a+b").write("koko")
>>> open("test","rb").read()
'testkoko'
