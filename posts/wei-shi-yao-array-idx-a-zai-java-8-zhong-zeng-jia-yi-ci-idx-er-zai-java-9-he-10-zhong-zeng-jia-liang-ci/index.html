为什么 array[idx++]+="a" 在 Java 8 中增加一次 idx 而在 Java 9 和 10 中增加两次？

Arya

2023-02-17 13:52

为了挑战，一位代码高尔夫球手 编写了以下代码：

import java.util.*;
public class Main {
  public static void main(String[] args) {
    int size = 3;
    String[] array = new String[size];
    Arrays.fill(array, "");
    for (int i = 0; i <= 100;) {
      array[i++ % size] += i + " ";
    }
    for (String element: array) {
      System.out.println(element);
    }
  }
}

使用 Java 10 完全不使用编号。那么这里发生了什么？这是Java 10中的错误吗？

跟进评论：

  • 使用 Java 9 或更高版本编译时会出现此问题（我们在 Java 10 中发现了它）。在 Java 8 上编译此代码，然后在 Java 9 或任何更高版本（包括 Java 11 抢先体验版）中运行，会得到预期的结果。

  • 这种代码是非标准的，但根据规范是有效的。它是由Kevin Cruijssen在一场 高尔夫挑战 赛的讨论 ****中发现的 ，因此遇到了奇怪的用例。

  • Didier L用这个更小、更容易理解的代码简化了这个问题：

  • 该问题似乎仅限于字符串连接和赋值运算符 ( +=) 以及带有副作用的表达式作为左操作数，例如 in array[test()]+="a"、array[ix++]+="a"、test()[index]+="a"或test().field+="a"。要启用字符串连接，至少其中一侧必须具有 type String。尝试在其他类型或构造上重现此失败。



解答

这是从 JDK 9 开始的一个错误javac（它在字符串连接方面做了一些更改，我怀疑这是问题的一部分），正如团队在错误 ID JDK-8204322 下确认的那样javac。如果您查看该行的相应字节码：

array[i++%size] += i + " ";

这是：

  21: aload_2
  22: iload_3
  23: iinc          3, 1
  26: iload_1
  27: irem
  28: aload_2
  29: iload_3
  30: iinc          3, 1
  33: iload_1
  34: irem
  35: aaload
  36: iload_3
  37: invokedynamic #5,  0 // makeConcatWithConstants:(Ljava/lang/String;I)Ljava/lang/String;
  42: aastore

最后一个aaload是阵列的实际负载。然而，部分

  21: aload_2             // load the array reference
  22: iload_3             // load 'i'
  23: iinc          3, 1  // increment 'i' (doesn't affect the loaded value)
  26: iload_1             // load 'size'
  27: irem                // compute the remainder

大致对应于表达式array[i++%size]（减去实际加载和存储），在那里两次。这是不正确的，正如jls-15.26.2中的规范所说：

形式的复合赋值表达式等价E1 op= E2于E1 = (T) ((E1) op (E2))，其中T是 的类型E1， 只是E1仅计算一次。

因此，对于表达式array[i++%size] += i + " ";，该部分array[i++%size]只应计算一次。但它被评估了两次（一次用于加载，一次用于存储）。

所以是的，这是一个错误。

一些更新：

该错误已在 JDK 11 中得到修复，并向后移植到 JDK 10（此处和此处），但未移植到 JDK 9，因为它不再接收公共更新。

Aleksey Shipilev 在JBS 页面上提到（以及此处评论中的@DidierL）：

解决方法：编译-XDstringConcat=inline

这将恢复为用于StringBuilder进行串联，并且没有错误。
