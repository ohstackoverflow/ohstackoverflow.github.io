是否有“git merge -s ours”的“他们的”版本？

Arya

2023-02-17 06:16

使用 将主题分支“B”合并到“A”时git merge，我遇到了一些冲突。我知道所有的冲突都可以使用“B”中的版本来解决。

我知道git merge -s ours。但我想要的是类似的东西git merge -s theirs。

为什么不存在？在与现有命令发生冲突合并后，如何获得相同的结果git？（git checkout来自 B 的每个未合并文件）

只是丢弃分支 A 中的任何内容（合并提交点到树的 B 版本）的“解决方案”不是我想要的。



解答

一个类似的替代方案是--strategy-option（缩写形式-X）选项，它接受theirs. 例如：

git checkout branchA
git merge -X theirs branchB

但是，这更等同于-X oursthan -s ours。关键区别在于-X执行常规递归合并，使用所选方解决任何冲突，而-s ours将合并更改为完全忽略另一方。

-X theirs在某些情况下，使用而不是假设的主要问题-s theirs是删除文件。在这种情况下，只需git rm使用已删除的任何文件的名称运行：

git rm {DELETED-FILE-NAME}

之后，-X theirs可能会按预期工作。

当然，使用git rm命令进行实际删除将从一开始就避免冲突的发生。


更多建议

将 branchB 合并到我们签出的 branchA 的可能且经过测试的解决方案：

# in case branchA is not our current branch
git checkout branchA

# make merge commit but without conflicts!!
# the contents of 'ours' will be discarded later
git merge -s ours branchB

# make temporary branch to merged commit
git branch branchTEMP

# get contents of working tree and index to the one of branchB
git reset --hard branchB

# reset to our merged commit but 
# keep contents of working tree and index
git reset --soft branchTEMP

# change the contents of the merged commit
# with the contents of branchB
git commit --amend

# get rid off our temporary branch
git branch -D branchTEMP

# verify that the merge commit contains only contents of branchB
git diff HEAD branchB

要使其自动化，您可以使用 branchA 和 branchB 作为参数将其包装到脚本中。

该解决方案保留了合并提交的第一个和第二个父级，正如您对git merge -s theirs branchB.
