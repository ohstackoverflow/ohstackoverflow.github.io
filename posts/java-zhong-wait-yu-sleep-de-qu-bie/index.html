Java 中“wait()”与“sleep()”的区别

Arya

2023-02-17 03:15

wait()a和in Threads之间有什么区别sleep()？

我的理解是wait()-ing 线程仍处于运行模式并使用 CPU 周期，但 -ingsleep()不消耗任何 CPU 周期是否正确？

为什么我们 同时 wait()拥有和sleep()？

它们的实现在较低级别上有何不同？



解答

A可以被另一个调用正在等待的监视器的wait线程“唤醒”，而 A不能。此外，（和）必须发生在监视器对象的块中，而不会：notify``sleep``wait``notify``synchronized``sleep

Object mon = ...;
synchronized (mon) {
    mon.wait();
}

此时当前正在执行的线程等待 并释放监视器 。另一个线程可能会做

synchronized (mon) { mon.notify(); }

（在同一个mon对象上）和第一个线程（假设它是唯一一个在监视器上等待的线程）将被唤醒。

notifyAll如果监视器上有多个线程在等待，您也可以调用——这将唤醒 所有线程 。然而，只有一个线程能够获取监视器（记住它wait在一个synchronized块中）并继续运行——其他线程将被阻塞，直到它们获得监视器的锁。

另一点是你调用wait它Object自己（即你在一个对象的监视器上等待）而你sleep调用Thread.

还有一点是您可以从中获得 虚假唤醒wait（即正在等待的线程无缘无故地恢复）。您应该 始终wait在某些情况下旋转，如下所示：

synchronized {
    while (!condition) { mon.wait(); }
}


更多建议

尚未提及的一个关键区别是：

  • sleep()不 释放 它在线程上持有的锁，

    synchronized(LOCK) {
Thread.sleep(1000); // LOCK is held

    }

  • wait() 释放 它持有的对象上的锁。

     synchronized(LOCK) {
 LOCK.wait(); // LOCK is not held

    }
