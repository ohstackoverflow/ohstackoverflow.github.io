删除不再在远程的跟踪分支

Arya

2023-02-16 23:14

有没有一种简单的方法可以删除所有远程等效项不再存在的跟踪分支？

例子：

分支机构（本地和远程）

  • 掌握
  • 起源/主人
  • 起源/错误修复-a
  • 起源/错误修复-b
  • 起源/错误修复-c

在本地，我只有一个 master 分支。现在我需要处理 bug-fix-a ，所以我检查它，处理它，然后将更改推送到远程。接下来我对 bug-fix- b 做同样的事情。

分支机构（本地和远程）

  • 掌握
  • 错误修复-a
  • 错误修复-b
  • 起源/主人
  • 起源/错误修复-a
  • 起源/错误修复-b
  • 起源/错误修复-c

现在我有本地分支 master ， bug-fix-a ， bug-fix-b 。Master 分支维护者会将我的更改合并到 master 中，并删除他已经合并的所有分支。

所以现在的状态是：

分支机构（本地和远程）

  • 掌握
  • 错误修复-a
  • 错误修复-b
  • 起源/主人
  • 起源/错误修复-c

现在我想调用一些命令来删除分支（在本例中为 bug-fix-a ， bug-fix-b ），它们不再出现在远程存储库中。

它会像现有的命令git remote prune origin，但更像git local prune origin。



解答

git remote prune origin修剪跟踪分支不在远程。

git branch --merged列出已合并到当前分支的分支。

xargs git branch -d删除标准输入中列出的分支。

小心删除 列出的分支git branch --merged。该列表可能包括master或您不想删除的其他分支。

为了让自己有机会在删除分支之前编辑列表，您可以在一行中执行以下操作：

git branch --merged >/tmp/merged-branches && \
  vi /tmp/merged-branches && xargs git branch -d </tmp/merged-branches


更多建议

最安全的方法是使用git for-each- ref带有插值变量的“管道”命令%(upstream:track)，这将[gone]在分支不再位于远程时发生：

git fetch -p && for branch in $(git for-each-ref --format '%(refname) %(upstream:track)' refs/heads | awk '$2 == "[gone]" {sub("refs/heads/", "", $1); print $1}'); do git branch -D $branch; done

这种方法比使用“瓷器”命令更安全，因为不存在意外匹配部分提交消息的风险。这是一个使用“porcelain”git 命令的版本：

git fetch -p && for branch in $(git branch -vv | grep ': gone]' | awk '{print $1}'); do git branch -D $branch; done

它的工作方式是在命令之后

git fetch -p

删除远程引用，当你运行

git branch -vv

它将显示“消失”作为远程状态。例如，

$ git branch -vv
  master                 b900de9 [origin/master: behind 4] Fixed bug
  release/v3.8           fdd2f4e [origin/release/v3.8: behind 2] Fixed bug
  release/v3.9           0d680d0 [origin/release/v3.9: behind 2] Updated comments
  bug/1234               57379e4 [origin/bug/1234: gone] Fixed bug

这就是脚本迭代的内容。
