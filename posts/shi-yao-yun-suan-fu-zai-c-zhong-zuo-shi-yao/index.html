什么？？！？？！运算符在 C 中做什么？

Arya

2023-02-16 21:16

我看到一行 C 看起来像这样：

!ErrorHasOccured() ??!??! HandleError();

它编译正确并且似乎运行正常。似乎它正在检查是否发生了错误，如果发生了错误，它就会进行处理。但我不太确定它实际上在做什么或它是如何做的。看起来程序员确实在尝试表达他们对错误的感受。

我以前从未??!??!在任何编程语言中见过它，而且我在任何地方都找不到它的文档。（Google 不会帮助搜索诸如 之类的字词??!??!）。它有什么作用以及代码示例如何工作？



解答

??!是翻译成的三字母|。所以它说：

!ErrorHasOccured() || HandleError();

由于短路，它相当于：

if (ErrorHasOccured())
    HandleError();

Guru of the Week（与 C++ 相关，但与此处相关），我从中挑选了它。

三字母的可能来源或正如@DwB 在评论中指出的那样，它更有可能是由于 EBCDIC 很难（再次）。IBM developerworks 论坛上的讨论似乎支持该理论。

来自 ISO/IEC 9899:1999 §5.2.1.1，脚注 12 (h/t @Random832 )：

三字符序列允许输入未在不变代码集中定义的字符，如 ISO/IEC 646 中所述，它是七位美国 ASCII 代码集的子集。


更多建议

好吧，为什么这通常存在可能与它在您的示例中存在的原因不同。

这一切都始于半个世纪前，将硬拷贝通信终端重新用作计算机用户界面。在最初的 Unix 和 C 时代是 ASR-33 电传打字机。

这个设备很慢（10 cps），嘈杂且丑陋，它的 ASCII 字符集视图以 0x5f 结尾，所以它（仔细看图片）没有任何键：

{ | } ~

三字母被定义为解决特定问题。这个想法是 C 程序可以使用在 ASR-33 和其他缺少高 ASCII 值的环境中找到的 ASCII 子集。

你的例子实际上是两个??!，每个意思|，所以结果是||。

然而，几乎按照定义，编写 C 代码的人拥有现代设备，1所以我的猜测是： 有人炫耀或自娱自乐， 在代码中留下一种复活节彩蛋供您查找。

它确实有效，它导致了一个广受欢迎的 SO 问题。

ASR-33 电传打字机

1. 就此而言，三字母组是由 ANSI 委员会发明的，该委员会在C 取得巨大成功 后首次开会，因此原始 C 代码或编码人员都不会使用它们。
