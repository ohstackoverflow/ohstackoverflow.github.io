什么是词法作用域的简要介绍？



解答

我通过例子来理解它们。:)

首先， 词法范围 （也称为 static 范围 ），类似于 C 的语法：

void fun()
{
    int x = 5;

    void fun2()
    {
        printf("%d", x);
    }
}

每个内层都可以访问其外层。

还有另一种方法，称为Lisp的第一个实现使用的 动态作用域 ，也是一种类似 C 的语法：

void fun()
{
    printf("%d", x);
}

void dummy1()
{
    int x = 5;

    fun();
}

void dummy2()
{
    int x = 10;

    fun();
}

这里fun可以访问xindummy1或dummy2，或任何调用其中声明的x函数。fun``x

dummy1();

将打印 5，

dummy2();

将打印 10。

第一个称为静态的，因为它可以在编译时推导，第二个称为动态的，因为外部作用域是动态的并且取决于函数的链式调用。

我发现静态范围更适合眼睛。大多数语言最终都采用了这种方式，甚至 Lisp（两者都可以，对吧？）。动态作用域类似于将所有变量的引用传递给被调用函数。

作为编译器为何无法推断出函数的外部动态范围的示例，请考虑我们的最后一个示例。如果我们这样写：

if(/* some condition */)
    dummy1();
else
    dummy2();

调用链取决于运行时条件。如果为真，则调用链如下所示：

dummy1 --> fun()

如果条件为假：

dummy2 --> fun()

在这两种情况下， 的外部范围fun都是调用者 加上调用者的调用者等等 。

顺便提一下，C 语言不允许嵌套函数，也不允许动态作用域。

词法作用域 定义了如何在嵌套函数中解析变量名： 内部函数包含父函数的作用域，即使父函数已经返回 。
