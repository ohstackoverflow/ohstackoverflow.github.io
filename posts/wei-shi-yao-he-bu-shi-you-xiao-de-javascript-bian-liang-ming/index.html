为什么 ◎﫫◎ 和 ☺ 不是有效的 JavaScript 变量名？

Arya

2023-02-17 15:47

我注意到在 Internet Explorer 中（但不幸的是，在我测试的其他浏览器中没有），您可以使用一些 Unicode 变量名。这让我很高兴，我非常高兴我可以像这样编写有趣的 Unicode 代码：

var ктоείναι草泥马 = "You dirty horse.",
    happy☺n☺mat☺p☺eia = ":)Yay!",
    ಠ_ಠ = "emoticon";

alert(ктоείναι草泥马 + happy☺n☺mat☺p☺eia + ಠ_ಠ);

但是，出于某种原因，◎ܫ◎和♨_♨不是☺有效的变量名。

为什么 doಠ_ಠ和草泥马work 而◎ܫ◎不是♨_♨or ☺？



解答

ಠ_ಠ和草泥马只包含实际字母表中使用的“字母”；也就是说，ಠ是来自卡纳达语字母表的符号，草泥马由汉字组成。

然而，◎ 和 ☺ 纯粹是符号；它们不与任何字母相关联。

ECMAScript标准第 7.6 章（除 Internet Explorer 之外的所有浏览器都遵循）规定标识符必须以下列之一开头。

  • 一个 Unicode 字母
  • $或者_
  • \后跟一个 unicode 转义序列。

标识符的后续字符必须是以下字符之一。

  • 开头允许的任何字符
  • Unicode 组合标记
  • 一个 Unicode 数字
  • Unicode 连接器标点符号
  • 零宽度非连接器
  • 零宽度连接器

IE 超出了标准并且足够宽容，允许使用一些符号，例如 ☺。

根据 ECMAScript 5.1 和 Unicode 6.1，有一个工具会告诉您输入的任何字符串是否是有效的 JavaScript 变量名称。


更多建议

EcmaScript 262，第 7.6 节规定名称必须以 $、_ 或 Unicode 字母开头，之后可能包含这些字符，或 Unicode 组合标记、Unicode 数字或 Unicode 连接符（以及一些格式控制字符是特定于语言的。）

因此，在您的案例中允许和不允许的标识符之间的区别可能是前导字符是否被视为“字母”。
