Dockerfile 中 RUN 和 CMD 的区别

Arya

2023-02-18 00:47

我对什么时候应该使用CMDvs感到困惑RUN。例如，要执行 bash/shell 命令（即ls -la），我会一直使用CMD，或者在某些情况下我会使用RUN？试图了解关于这两个相似Dockerfile指令的最佳实践。



解答

RUN是一个镜像构建步骤，一个命令后容器的状态RUN将被提交到容器镜像中。一个 Dockerfile 可以有许多RUN步骤，这些步骤相互叠加以构建图像。

CMD是启动构建镜像时容器默认执行的命令。一个 Dockerfile 只会使用最终CMD定义的。CMD使用 启动容器时可以覆盖docker run $image $other_command。

ENTRYPOINTCMD也与容器从镜像启动的方式密切相关并且可以修改。


更多建议

RUN- 命令在我们构建 docker 镜像时触发。

CMD- 命令在我们启动创建的 docker 镜像时触发。
