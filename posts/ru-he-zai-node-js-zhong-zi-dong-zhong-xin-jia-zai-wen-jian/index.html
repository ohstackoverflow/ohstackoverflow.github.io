如何在 Node.js 中自动重新加载文件？

Arya

2023-02-18 04:19

关于如何在 Node.js 中实现文件自动重新加载的任何想法？每次更改文件时我都厌倦了重新启动服务器。显然 Node.js 的require()函数不会重新加载文件，如果它们已经被需要的话，所以我需要做这样的事情：

var sys     = require('sys'), 
    http    = require('http'),
    posix   = require('posix'),
    json    = require('./json');

var script_name = '/some/path/to/app.js';
this.app = require('./app').app;

process.watchFile(script_name, function(curr, prev){
    posix.cat(script_name).addCallback(function(content){
        process.compile( content, script_name );
    });
});

http.createServer(this.app).listen( 8080 );

在 app.js 文件中我有：

var file = require('./file');
this.app = function(req, res) { 
    file.serveFile( req, res, 'file.js');  
}

但这也不起作用——我在process.compile()声明中收到一条错误消息，指出“要求”未定义。process.compile正在评估 app.js ，但对 node.js 全局变量一无所知。



解答

一个好的、最新的替代方法supervisor是 nodemon ：

监控 node.js 应用程序中的任何更改并自动重启服务器 - 非常适合开发

要与没有（v8.1 及以下，不建议）nodemon的 Node 版本一起使用：npx

$ npm install nodemon -g
$ nodemon app.js

nodemon或者与捆绑在 (v8.2+) 中的 Node 版本一起使用npx：

$ npm install nodemon
$ npx nodemon app.js

或者作为 devDependency 在 package.json 中使用 npm 脚本：

"scripts": {
  "start": "nodemon app.js"
},
"devDependencies": {
  "nodemon": "..."
}


更多建议

节点主管很棒

旧节点版本保存时重启的用法（不建议）：

npm install supervisor -g
supervisor app.js

对于随附的 Node 版本，在保存时重新启动的用法npx：

npm install supervisor
npx supervisor app.js

或者直接调用supervisornpm 脚本：

"scripts": {
  "start": "supervisor app.js"
}
