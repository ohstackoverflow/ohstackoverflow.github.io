JavaScript Promises - 拒绝与抛出

Arya

2023-02-18 00:17

我已经阅读了几篇关于这个主题的文章，但我仍然不清楚Promise.reject与抛出错误之间是否有区别。例如，

使用 Promise.reject

return asyncIsPermitted()
    .then(function(result) {
        if (result === true) {
            return true;
        }
        else {
            return Promise.reject(new PermissionDenied());
        }
    });

使用抛出

return asyncIsPermitted()
    .then(function(result) {
        if (result === true) {
            return true;
        }
        else {
            throw new PermissionDenied();
        }
    });

我的偏好是仅仅因为它更短而使用它throw，但我想知道两者之间是否有任何优势。



解答

使用一个与另一个相比没有优势，但是，有一个特定的情况是throw行不通的。但是，这些情况是可以解决的。

任何时候你在 promise 回调中，你都可以使用throw. 但是，如果您处于任何其他异步回调中，则必须使用reject.

例如，这不会触发捕获：

new Promise(function() {
  setTimeout(function() {
    throw 'or nah';
    // return Promise.reject('or nah'); also won't work
  }, 1000);
}).catch(function(e) {
  console.log(e); // doesn't happen
});

相反，您会留下未解决的承诺和未捕获的异常。在这种情况下，您可能希望改用reject. 但是，您可以通过两种方式解决此问题。

  1. 通过在超时内使用原始 Promise 的拒绝功能：

    new Promise(function(resolve, reject) { setTimeout(function() { reject('or nah'); }, 1000); }).catch(function(e) { console.log(e); // works! });

  2. 通过承诺超时：

    function timeout(duration) { // Thanks joews return new Promise(function(resolve) { setTimeout(resolve, duration); }); }

    timeout(1000).then(function() { throw 'worky!'; // return Promise.reject('worky'); also works }).catch(function(e) { console.log(e); // 'worky!' });


更多建议

另一个重要的事实是，它reject() 不会 像return语句那样终止控制流。相反，throw确实终止了控制流。

例子：

new Promise((resolve, reject) => {
  throw "err";
  console.log("NEVER REACHED");
})
.then(() => console.log("RESOLVED"))
.catch(() => console.log("REJECTED"));

对比

new Promise((resolve, reject) => {
  reject(); // resolve() behaves similarly
  console.log("ALWAYS REACHED"); // "REJECTED" will print AFTER this
})
.then(() => console.log("RESOLVED"))
.catch(() => console.log("REJECTED"));
