在 SQL Server 中使用 JOIN 更新表？

Arya

2023-02-17 09:04

我想更新表中的列，以便在其他表上进行连接，例如：

UPDATE table1 a 
INNER JOIN table2 b ON a.commonfield = b.[common field] 
SET a.CalculatedColumn= b.[Calculated Column]
WHERE 
    b.[common field]= a.commonfield
AND a.BatchNO = '110'

但它在抱怨：

消息 170，级别 15，状态 1，第 2 行
第 2 行：“a”附近的语法不正确。

这里有什么问题？



解答

您还没有完全UPDATE FROM理解 SQL Server 的专有语法。也不确定为什么您需要加入CommonField并随后对其进行过滤。尝试这个：

UPDATE t1
  SET t1.CalculatedColumn = t2.[Calculated Column]
  FROM dbo.Table1 AS t1
  INNER JOIN dbo.Table2 AS t2
  ON t1.CommonField = t2.[Common Field]
  WHERE t1.BatchNo = '110';

如果你在做一些愚蠢的事情——比如不断地尝试将一列的值设置为另一列的聚合（这违反了避免存储冗余数据的原则），你可以使用 CTE（公共表表达式）——参见这里和此处了解更多详情：

;WITH t2 AS
(
  SELECT [key], CalculatedColumn = SUM(some_column)
    FROM dbo.table2
    GROUP BY [key]
)
UPDATE t1
  SET t1.CalculatedColumn = t2.CalculatedColumn
  FROM dbo.table1 AS t1
  INNER JOIN t2
  ON t1.[key] = t2.[key];

这很愚蠢的原因是，每次更改任何行时，您都必须重新运行整个更新table2。ASUM是您始终可以在运行时计算的东西，这样做时，永远不必担心结果过时。


更多建议

像这样尝试：

    UPDATE a 
    SET a.CalculatedColumn= b.[Calculated Column]
    FROM table1 a INNER JOIN table2 b ON a.commonfield = b.[common field] 
    WHERE a.BatchNO = '110'
