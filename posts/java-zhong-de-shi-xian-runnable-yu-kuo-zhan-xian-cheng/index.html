Java 中的“实现 Runnable”与“扩展线程”

Arya

2023-02-16 21:44

从我花在线程上的时间开始Java，我发现了这两种编写线程的方法：

带 工具Runnable：

public class MyRunnable implements Runnable {
    public void run() {
        //Code
    }
}
//Started with a "new Thread(new MyRunnable()).start()" call

或者，使用 extendsThread：

public class MyThread extends Thread {
    public MyThread() {
        super("MyThread");
    }
    public void run() {
        //Code
    }
}
//Started with a "new MyThread().start()" call

这两个代码块有什么显着差异吗？

更多建议

tl;dr：实现 Runnable 更好。 但是，警告很重要。

一般来说，我会推荐使用类似Runnablerather than 的东西Thread，因为它允许您将您的工作仅与您选择的并发性松散耦合。例如，如果您使用 aRunnable并稍后决定这实际上不需要它自己的Thread，您可以只调用 threadA.run()。

警告： 在这里，我强烈反对使用原始线程。我更喜欢使用Callables和FutureTasks（来自 javadoc：“A cancellable asynchronous computation”）。现代并发支持的超时、适当取消和线程池的集成对我来说都比一堆原始线程有用得多。

跟进： 有一个FutureTask构造函数允许您使用 Runnables（如果这是您最熟悉的）并且仍然可以从现代并发工具中获益。引用 javadoc：

如果您不需要特定结果，请考虑使用以下形式的结构：

Future<?> f = new FutureTask<Object>(runnable, null)

因此，如果我们将 their 替换runnable为 your threadA，我们将得到以下内容：

new FutureTask<Object>(threadA, null)

另一个让您更接近 Runnables 的选项是ThreadPoolExecutor。您可以使用execute方法传入一个 Runnable 来执行“将来某个时间的给定任务”。

如果您想尝试使用线程池，上面的代码片段将变成如下所示（使用 Executors.newCachedThreadPool ()工厂方法）：

ExecutorService es = Executors.newCachedThreadPool();
es.execute(new ThreadA());
