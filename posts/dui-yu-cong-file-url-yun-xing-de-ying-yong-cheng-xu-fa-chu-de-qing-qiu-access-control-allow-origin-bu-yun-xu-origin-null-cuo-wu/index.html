对于从 file:// URL 运行的应用程序发出的请求，“Access-Control-Allow-Origin 不允许 Origin null”错误

Arya

2023-02-18 01:29

我正在开发一个页面，通过 jQuery 的 AJAX 支持从 Flickr 和 Panoramio 中提取图像。

Flickr 端工作正常，但当我尝试$.get(url, callback)从 Panoramio 访问时，我在 Chrome 的控制台中看到错误：

XMLHttpRequest 无法加载http://www.panoramio.com/wapi/data/get_photos?v=1&key=dummykey&tag=test&offset=0&length=20&callback=processImages&minx=-30&miny=0&maxx=0&maxy=150。Access- Control-Allow-Origin 不允许 Origin null。

如果我直接从浏览器查询该 URL，它工作正常。这是怎么回事，我可以解决这个问题吗？我是否错误地编写了我的查询，或者这是 Panoramio 阻碍我尝试做的事情？

Google 没有在错误消息中找到任何有用的匹配项。

编辑

下面是一些显示问题的示例代码：

$().ready(function () {
  var url = 'http://www.panoramio.com/wapi/data/get_photos?v=1&key=dummykey&tag=test&offset=0&length=20&callback=processImages&minx=-30&miny=0&maxx=0&maxy=150';

  $.get(url, function (jsonp) {
    var processImages = function (data) {
      alert('ok');
    };

    eval(jsonp);
  });
});

您可以联机运行示例。

感谢 Darin 在这方面的帮助。 上面的代码是错误的。 改用这个：

$().ready(function () {

解答

作为记录，据我所知，你有两个问题：

  1. 您没有将“jsonp”类型说明符传递给您的$.get，因此它使用的是普通的 XMLHttpRequest。但是，您的浏览器支持 CORS（跨源资源共享）以允许跨域 XMLHttpRequest（如果服务器同意）。这就是Access-Control-Allow-Origin标题进来的地方。

  2. 我相信您提到过您是从 file:// URL 运行它的。CORS 标头有两种方式表明跨域 XHR 正常。一种是发送Access-Control-Allow-Origin: *（如果您通过 访问 Flickr $.get，他们一定已经这样做了），而另一种是回显Origin标头的内容。但是，file://URL 会产生一个空值Origin，无法通过回显进行授权。

第一个是通过 Darin 的建议以迂回的方式解决的$.getJSON。callback=?如果它看到URL 中的子字符串，它将请求类型从默认的“json”更改为“jsonp”，这有点神奇。

通过不再尝试从 URL 执行 CORS 请求，解决了第二个问题file://。

为了向其他人澄清，以下是简单的故障排除说明：

  1. 如果您尝试使用 JSONP，请确保满足以下条件之一：
    • 您正在使用$.get并设置dataType为jsonp。
    • 您正在使用$.getJSON并包含callback=?在 URL 中。
  2. 如果您尝试通过 CORS 执行跨域 XMLHttpRequest ...
    1. 确保您正在通过http://. 通过运行的脚本file://对 CORS 的支持有限。
    2. 确保浏览器确实支持 CORS。（Opera 和 Internet Explorer 迟到了）


更多建议

您可能需要在调用的脚本中添加一个 HEADER，这是我在 PHP 中必须做的：

header('Access-Control-Allow-Origin: *');
