Grunt、NPM 和 Bower 之间的区别（package.json 与 bower.json）

Arya

2023-02-17 22:42

当我想添加一个包（并将依赖项签入 git）时，它属于哪里 - intopackage.json或 into bower.json？

据我所知，
运行bower install会获取包并将其放入/vendor目录，
运行npm install它会获取包并将其放入/node_modules目录。

This SO answer说 bower 用于前端，npm 用于后端。
Ember-app-kit乍一看似乎坚持这种区别......但是 gruntfile 中用于启用某些功能的说明给出了两个明确的命令，所以我在这里完全困惑。

直觉上我猜

  1. npm install --save-dev package-name 相当于将 package-name 添加到我的 package.json

  2. bower install --save package-name 可能与将包添加到我的 bower.json 并运行 bower install 相同吗？

如果是这样的话，我什么时候应该像那样显式安装包而不将它们添加到管理依赖项的文件中（除了全局安装命令行工具）？

你的问题：

当我想添加一个包（并将依赖项签入 git）时，它属于哪里 - 进入 package.json 或进入 bower.json

  • 现在一切都属于 package.json

  • 构建所需的依赖项在“devDependencies”中，即npm install require-dir --save-dev（--save-dev 通过向 devDependencies 添加条目来更新你的 package.json）

  • 您的应用程序在运行时所需的依赖项位于“依赖项”中，即npm install lodash --save（--save 通过向依赖项添加条目来更新您的 package.json）

如果是这样的话，我什么时候应该像那样显式安装包而不将它们添加到管理依赖项的文件中（除了全局安装命令行工具）？

总是 。只是因为舒服。当您添加标志（--save-dev或--save）时，管理 deps 的文件 (package.json) 会自动更新。不要浪费时间手动编辑其中的依赖项。npm install --save-dev package-name是的快捷方式npm i -D package-name和npm install --save package-name是的快捷方式npm i -S package-name
