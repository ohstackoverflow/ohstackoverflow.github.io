为什么Java的main方法是静态的？

Arya

2023-02-18 06:11

Java 方法的方法签名main是：

public static void main(String[] args) {
    ...
}

这个方法必须是静态的有什么原因吗？



解答

该方法是静态的，否则会产生歧义：应该调用哪个构造函数？特别是如果你的班级是这样的：

public class JavaClass{
  protected JavaClass(int x){}
  public void main(String[] args){
  }
}

JVM 应该调用吗new JavaClass(int)？它应该传递什么x？

如果不是，JVM 是否应该JavaClass在不运行任何构造方法的情况下进行实例化？我认为不应该，因为这会特殊情况下你的整个类 - 有时你有一个尚未初始化的实例，你必须在每个可以调用的方法中检查它。

有太多的边缘情况和歧义，使得 JVM 在调用入口点之前必须实例化一个类是没有意义的。这就是为什么main是静态的。

我不知道为什么main总是被标记public。
