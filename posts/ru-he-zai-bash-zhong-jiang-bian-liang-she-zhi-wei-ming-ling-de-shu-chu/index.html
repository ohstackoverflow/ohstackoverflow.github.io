如何在 Bash 中将变量设置为命令的输出？

Arya

2023-02-16 21:51

我有一个非常简单的脚本，类似于以下内容：

#!/bin/bash
VAR1="$1"
MOREF='sudo run command against $VAR1 | grep name | cut -c7-'
echo $MOREF

当我从命令行运行此脚本并将参数传递给它时，我没有得到任何输出。但是，当我运行变量中包含的命令时$MOREF，我能够获得输出。

如何获取需要在脚本中运行的命令的结果，将其保存到变量中，然后将该变量输出到屏幕上？



解答

除了反引号command，命令替换$(command)还可以用or完成"$(command)"，我觉得这样更容易阅读，并且允许嵌套。

OUTPUT=$(ls -1)
echo "${OUTPUT}"

MULTILINE=$(ls \
   -1)
echo "${MULTILINE}"

引用 ( ) 对保留 多行变量值"很重要；它在赋值的右侧是可选的，因为不执行分词，所以可以正常工作。 ****OUTPUT=$(ls -1)


更多建议

$(sudo run command)

如果您要使用撇号，则需要``, 而不是'`. 这个字符称为“反引号”（或“重音符”）：

#!/bin/bash
VAR1="$1"
VAR2="$2"
MOREF=`sudo run command against "$VAR1" | grep name | cut -c7-`
echo "$MOREF"
