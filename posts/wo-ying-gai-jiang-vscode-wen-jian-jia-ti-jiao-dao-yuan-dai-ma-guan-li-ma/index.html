我应该将 .vscode 文件夹提交到源代码管理吗？

Arya

2023-02-18 02:44

该.vscode文件夹是否要提交给源代码管理？

在一个新项目中，文件夹是空的，除了settings.json文件。什么样的东西会进入这个文件夹？它是否像文件夹一样特定于机器、特定于开发人员.vs，因此不会被提交？还是所有开发人员都应该共享这个文件夹，因此应该提交它？

文件顶部的评论.vscode/settings.json指出：

// Place your settings in this file to overwrite default and user settings.
{
}

这似乎暗示该文件夹应包含特定于项目的设置，因此应包含在源代码中。此外，这篇关于 UserVoice 的帖子似乎暗示会在那里输入一些内容，这也表明它应该被提交。



解答

.vscode如果您想与团队共享设置、任务配置和调试配置，请签入该文件夹。如果您想在团队中强制执行设置，我认为通常与团队共享设置（例如空格与制表符）是有意义的。我们在 VS Code 团队中也共享调试和任务特定设置，因为我们希望我们的团队拥有相同的 VS Code 调试目标和任务目标集。

顺便说一句，您不需要.vscode在项目中有一个文件夹来进行设置。您还可以在用户级别配置设置。


更多建议

提交.vscode文件夹中的一些文件

建议通常排除.vscode文件夹，但保留允许其他开发人员重新创建共享设置的精选 JSON 文件。

要包括的设置示例：

  • 用于运行测试套件的语言特定测试配置 ( settings.json)
  • settings.jsonlinters 和代码格式化工具的扩展设置，以强制执行此 repo ( )中使用的语言规则
  • 运行和调试配置 ( launch.json)
  • 共享任务 - 如果使用 VS Code 管理 ( tasks.json)

请注意，某些设置可以存储在用户设置或工作区文件中，或从.vscode文件夹传输到其中。见下文。

示例.gitignore代码

以下是https://gitignore.io中建议的设置。您可以在那里搜索“VisualStudioCode”以获取最新的推荐.gitignore文件。.gitignore我使用这个网站作为我大部分新回购协议的起点：

# Created by https://www.gitignore.io/api/visualstudiocode
# Edit at https://www.gitignore.io/?templates=visualstudiocode

### VisualStudioCode ###
.vscode/*      # Maybe .vscode/**/* instead - see comments
!.vscode/settings.json
!.vscode/tasks.json
!.vscode/launch.json
!.vscode/extensions.json

### VisualStudioCode Patch ###
# Ignore all local history of files
**/.history

# End of https://www.gitignore.io/api/visualstudiocode

在上面的.gitignore文件中，该.vscode/*行（注意：关于是否*应该包含的一些争论 - 请参阅评论；.vscode/**/*也可能更好地忽略嵌套文件夹）表示排除文件夹中的所有内容.vscode，但随后这些!.vscode/a_specific_file行告诉 git “不”忽略该文件夹中的某些特定文件（settings.json、launch.json等）。最终结果是.vscode除了在其他行之一中特别命名的文件外，文件夹中的所有内容都被排除在外。

其他因素

在您的存储库中包含该.vscode文件夹实际上不会 伤害 使用不同 IDE（或文本/代码编辑器）的任何人。

但是，如果这些文件包含需要特定于您的环境的通用设置（例如 repo 安装的绝对路径），它可能会给使用 VS Code 的其他人带来问题，或者某些设置可能无法正确加载。关键是为避免保存对您的本地环境自定义的设置，只共享那些可以（或者，在这个 repo 的情况下，应该）被每个人使用的设置。

例如，如果 IDE 设置文件有 repo 或任何文件/库等的绝对路径，那就不好了，不要共享。但是，如果所有引用都是相对的，那么它们应该适用于任何使用 repo 的人（尽管要注意 Windows/Unix 之间的路径规范差异......）。

关于用户、工作区和文件夹设置

注意：当您更改设置的文件夹版本时，文件夹中的 设置 文件.vscode通常会更新- 但这似乎取决于各个扩展名的编码方式，因为我遇到过此规则的多个例外情况。 __

  • 如果您更改了 用户 设置，它们通常存储在别处（位置取决于操作系统设置，通常在主目录中）。
  • 如果您对 工作区 设置进行更改，它们通常存储在*.code-workspace您当前使用的文件中。如果您没有工作区（您直接打开了一个文件夹），那么他们可能会转到该.vscode文件夹​​，但是，总的来说，这可能取决于拥有该设置的扩展程序。

因此，您通常应该将个人 PC 的自定义设置放入 用户 设置，将通用设置放入 工作区 或 文件夹 设置。
