如何在迭代时从列表中删除项目？

Arya

2023-02-17 09:15

这个问题的答案是 社区的努力。编辑现有答案以改进这篇文章。它目前不接受新的答案或互动。

我正在 Python 中迭代一个元组列表，如果它们满足特定条件，我将尝试删除它们。

for tup in somelist:
    if determine(tup):
         code_to_remove_tup

我应该用什么代替code_to_remove_tup？我不知道如何以这种方式删除该项目。



解答

您可以使用列表理解来创建一个仅包含您不想删除的元素的新列表：

somelist = [x for x in somelist if not determine(x)]

或者，通过分配给 slice somelist[:]，您可以改变现有列表以仅包含您想要的项目：

somelist[:] = [x for x in somelist if not determine(x)]

如果有其他参考somelist需要反映更改，则此方法可能会有用。

除了理解，您还可以使用itertools. 在 Python 2 中：

from itertools import ifilterfalse
somelist[:] = ifilterfalse(determine, somelist)

或者在 Python 3 中：

from itertools import filterfalse
somelist[:] = filterfalse(determine, somelist)


更多建议

建议列表理解的答案 几乎 是正确的——除了他们构建了一个全新的列表，然后给它与旧列表相同的名称，他们没有 修改 旧列表。这与您通过选择性删除所做的不同，如Lennart 的建议——它更快，但如果您的列表是通过多个引用访问的，那么您只是重新设置其中一个引用而不 更改 列表对象本身的事实可以导致微妙的、灾难性的错误。

幸运的是，获得列表理解的速度和就地更改所需的语义非常容易——只需编写代码：

somelist[:] = [tup for tup in somelist if determine(tup)]

请注意与其他答案的细微差别：这个答案没有 分配 给裸名。它分配给恰好是整个列表的列表切片，从而替换 同一个 Python 列表对象中的列表 内容 ，而不是像其他答案那样只重新设置一个引用（从前一个列表对象到新列表对象）。 ****
