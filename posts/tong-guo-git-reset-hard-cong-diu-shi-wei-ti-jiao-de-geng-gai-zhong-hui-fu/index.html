通过“git reset --hard”从丢失未提交的更改中恢复

Arya

2023-02-17 21:25

有什么方法可以从 a 恢复对工作目录的未提交更改git reset --hard HEAD？



解答

一般来说，您 无法取回未提交的更改。

先前暂存的更改 ( git add) 应该可以从索引对象中恢复，因此如果您这样做了，请使用git fsck --lost- found来定位与其相关的对象。（这会将对象写入目录.git/lost-found/；您可以从那里git show <filename>查看每个文件的内容。）

如果没有，这里的答案是：查看您的备份。 也许您的编辑器/IDE 将临时副本存储 在 /tmp 或 C:\TEMP 以及类似的东西下。[1]

git reset HEAD@{1}

这将恢复到以前的 HEAD

[1] vim eg 可选地存储持久撤销， eclipse IDE 存储 本地历史 ；这些功能可能会节省您的时间**
