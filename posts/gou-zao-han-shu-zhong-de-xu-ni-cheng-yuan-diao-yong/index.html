构造函数中的虚拟成员调用

Arya

2023-02-17 02:18

我从 ReSharper 收到关于从我的对象构造函数调用虚拟成员的警告。

为什么这是不应该做的事情？



解答

当构造一个用 C# 编写的对象时，会发生这样的情况：初始化器按顺序从最派生类到基类运行，然后构造函数按从基类到最派生类的顺序运行（详见 Eric Lippert 的博客至于这是为什么）。

同样在 .NET 中，对象在构造时不会更改类型，而是作为派生程度最高的类型开始，方法表用于派生程度最高的类型。这意味着虚方法调用总是在最派生的类型上运行。

当您将这两个事实结合起来时，您会遇到这样一个问题：如果您在构造函数中进行虚方法调用，并且它不是其继承层次结构中最派生的类型，那么它将在构造函数尚未被调用的类上调用运行，因此可能不处于调用该方法的合适状态。

当然，如果您将类标记为密封的以确保它是继承层次结构中派生程度最高的类型，那么这个问题就会得到缓解——在这种情况下调用虚方法是完全安全的。


更多建议

为了回答您的问题，请考虑以下问题：Child实例化对象时，以下代码将打印出什么？

class Parent
{
    public Parent()
    {
        DoSomething();
    }

    protected virtual void DoSomething() 
    {
    }
}

class Child : Parent
{
    private string foo;

    public Child() 
    { 
        foo = "HELLO"; 
    }

    protected override void DoSomething()
    {
        Console.WriteLine(foo.ToLower()); //NullReferenceException!?!
    }
}

答案是实际上aNullReferenceException会被抛出，因为foo是null。 一个对象的基础构造函数在它自己的构造函数之前被调用 。通过调用virtual对象的构造函数，您引入了继承对象将在完全初始化之前执行代码的可能性。
