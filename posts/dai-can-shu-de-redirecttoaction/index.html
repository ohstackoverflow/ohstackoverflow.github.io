带参数的 RedirectToAction

Arya

2023-02-17 17:36

因此，我有一个从锚点调用的动作，Site/Controller/Action/ID哪里ID是int.

稍后我需要从 Controller 重定向到同一个 Action。

有没有聪明的方法来做到这一点？目前我正在存储ID临时数据，但是当你返回后再次按 f5 刷新页面时，临时数据消失了，页面崩溃了。



解答

您可以将 id 作为 RedirectToAction() 方法的 routeValues 参数的一部分传递。

return RedirectToAction("Action", new { id = 99 });

这将导致重定向到 Site/Controller/Action/99。不需要临时或任何类型的视图数据。


更多建议

根据我的研究， Kurt 的回答应该是正确的，但是当我尝试它时，我必须这样做才能让它真正为我工作：

return RedirectToAction( "Main", new RouteValueDictionary( 
    new { controller = controllerName, action = "Main", Id = Id } ) );

如果我没有指定控制器和其中的操作，RouteValueDictionary它就不起作用。

同样，当这样编码时，第一个参数 (Action) 似乎被忽略了。所以如果只是在Dict中指定controller，期望第一个参数指定Action，也是不行的。
