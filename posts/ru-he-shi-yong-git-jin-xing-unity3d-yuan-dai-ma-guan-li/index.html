如何使用 Git 进行 Unity3D 源代码管理？

Arya

2023-02-18 03:50

使用Git的最佳实践是什么源代码控制的最佳实践是什么，特别是在处理 Unity 3D 项目的二进制性质时？请描述工作流程，.gitignore 中将包含哪些路径，Unity 和/或项目中应设置哪些设置，以及任何其他应注意的特殊事项。

注意：我知道使用 Asset Server 是 Unity 推荐的方式，但出于各种原因我想使用 Git。请不要回答说或争辩我应该只使用资产服务器。资产服务器真的不是我的选择。



解答

以下是我个人博客 的节选。

在 3D 游戏中使用 Git

2015 年 10 月更新： GitHub 此后发布了一个名为Git LFS的 Git 插件，它直接处理以下问题。您现在可以轻松高效地对大型二进制文件进行版本控制！

Git 可以开箱即用地处理 3D 游戏。然而，这里的主要警告是，随着提交历史的膨胀，从长远来看，对大型（> 5 MB）媒体文件进行版本控制可能会成为一个问题。我们已经解决了我们项目中的这个潜在问题，方法是仅在二进制资产被认为是最终版本时对其进行版本控制。我们的 3D 美术师使用Dropbox来处理 WIP 资产，这既是出于上述原因，也是因为它 更快 更简单（没有多少美术师会积极使用 Git！）。

Git 工作流程

您的 Git 工作流程在很大程度上需要您根据自己的团队经验以及合作方式自行决定。然而。我强烈推荐原作者在此处描述的适当命名的 Git Flow 方法。

我不会在这里深入探讨该方法的工作原理，因为作者完美地描述了它，而且用很少的文字也很容易理解。我和我的团队一起使用了一段时间，这是迄今为止我们尝试过的最好的工作流程。

Git GUI 客户端应用程序

这实际上是个人偏好，因为在 Git GUI 或是否使用 GUI 方面有很多选择。但我想推荐免费的SourceTree 应用程序，因为它可以完美地插入 Git Flow 扩展。在此处阅读有关在其应用程序中实施 Git Flow 方法的 SourceTree 教程。

Unity3D 设置

对于 Unity 3D v4.3 及更高版本：

  1. （在 v4.5 及更高版本中跳过此步骤）External在 中启用选项Unity → Preferences → Packages → Repository。
  2. 打开Edit菜单并选择Project Settings → Editor：
    1. 切换Version Control Mode到Visible Meta Files。
    2. 切换Asset Serialization Mode到Force Text。
  3. 从菜单中保存场景和项目File。

查看我的博客文章，了解如何在此处执行此操作的步骤。

附加配置

在 Unity3D 项目中使用 Git 的几个主要烦恼之一是 Git 不关心目录，并且会在从目录中删除文件后愉快地留下空目录。Unity3D 将为这些目录生成 *.meta 文件，当 Git 提交不断添加和删除这些元文件时，可能会导致团队成员之间发生争执。

将此 Git 合并后挂钩添加到/.git/hooks/其中包含 Unity3D 项目的存储库的文件夹中。在任何 Git pull/merge 之后，它会查看哪些文件已被删除，检查它所在的目录是否为空，如果是，则将其删除。


更多建议

在 Unity 4.3 中，您还必须从首选项中启用外部选项，但自从 Unity 4.5 他们放弃了该选项，因此完整的设置过程如下所示：

  1. 切换Visible Meta Files到Editor → Project Settings → Editor → Version Control Mode
  2. 切换Force Text到Editor → Project Settings → Editor → Asset Serialization Mode
  3. File从菜单中保存场景和项目
