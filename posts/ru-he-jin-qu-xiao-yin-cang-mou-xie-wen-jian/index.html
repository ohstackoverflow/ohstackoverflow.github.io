如何仅取消隐藏某些文件？

Arya

2023-02-18 06:23

我藏起了我的变化。现在我只想从存储中取消存储一些文件。我怎样才能做到这一点？



解答

如下所述，并在“我如何从 git 存储中提取单个文件（或对文件的更改）？ ”中进行了详细说明，您可以应用 usegit checkout或git show恢复特定文件。

git checkout stash@{0} -- <filename>

在 Git 2.23+（2019 年 8 月）中，使用git restore, 替换令人困惑的git checkout命令：

git restore --source=stash@{0} -- <filename>

这确实会 覆盖 filename：确保您没有本地修改，或者您可能想要合并隐藏的文件。

（正如Jaime M.评论的那样，对于像 tcsh 这样需要转义特殊字符的某些 shell，语法为：）git checkout 'stash@{0}' -- <filename>

或者以另一个文件名保存它：

git show stash@{0}:<full filename>  >  <newfile>

（请注意，这里<full filename>是文件相对于项目顶级目录的完整路径名（认为：相对于stash@{0}））。

yucer在评论中建议：

如果您想手动选择要从该文件应用哪些更改：

git difftool stash@{0}..HEAD -- <filename>

Vivek在评论中补充道：

看起来“ git checkout stash@{0} -- <filename>”恢复了执行隐藏时文件的版本——它不 应用 （仅）该文件的隐藏更改。
做后者：

git diff stash@{0}^1 stash@{0} -- <filename> | git apply

（正如peterflynn评论的那样，在某些情况下您可能需要从传统差异路径中删除一个 ( ) 前导斜杠）| git apply -p1``p1

正如所评论的那样：“unstash”（git stash pop），然后：

  • 添加你想保留的索引 ( git add)
  • 把剩下的藏起来：git stash --keep-index

最后一点是什么允许您在隐藏其他文件的同时保留一些文件。
它在“如何从已更改的多个文件中仅存储一个文件”中进行了说明。


更多建议

git checkout stash@{N} <File(s)/Folder(s) path>

例如。要仅从上次隐藏的 ./test.c 文件和 ./include 文件夹恢复，

git checkout stash@{0} ./test.c ./include
