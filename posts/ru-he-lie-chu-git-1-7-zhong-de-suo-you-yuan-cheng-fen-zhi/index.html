如何列出 Git 1.7+ 中的所有远程分支？

Arya

2023-02-17 11:15

我已经尝试过git branch -r，但这只列出了我在本地跟踪的远程分支。我如何找到那些我没有的列表？ （命令是列出所有 远程分支还是仅列出未跟踪的分支对我来说并不重要。）



解答

对于这里的绝大多数 [1]访问者，“如何在 Git 1.7+ 中列出所有远程分支？”这个问题的正确和最简单的答案。是：

git branch -r

对于少数[1] git branch -r不起作用。如果git branch -r不起作用，请尝试：

git ls-remote --heads <remote-name>

如果git branch -r不起作用，那么也许就像Cascabel所说的那样“你已经修改了默认的 refspec，所以git fetch不要git remote update获取所有remote的分支”。

[1]截至 2018 年 2 月撰写此脚注时，我查看了评论，发现git branch -r绝大多数人（大约 90% 或140人中的125 人）的作品。

如果git branch -r不起作用，请根据此答案git config --get remote.origin.fetch检查是否包含通配符 ( )*


更多建议

remote show显示远程上的所有分支，包括本地未跟踪的分支，甚至还未获取的分支。

git remote show <remote-name>

它还会尝试显示分支相对于本地存储库的状态：

> git remote show origin
* remote origin
  Fetch URL: C:/git/.\remote_repo.git
  Push  URL: C:/git/.\remote_repo.git
  HEAD branch: master
  Remote branches:
    branch_that_is_not_even_fetched new (next fetch will store in remotes/origin)
    branch_that_is_not_tracked      tracked
    branch_that_is_tracked          tracked
    master                          tracked
  Local branches configured for 'git pull':
    branch_that_is_tracked merges with remote branch_that_is_tracked
    master                 merges with remote master
  Local refs configured for 'git push':
    branch_that_is_tracked pushes to branch_that_is_tracked (fast-forwardable)
    master                 pushes to master                 (up to date)
