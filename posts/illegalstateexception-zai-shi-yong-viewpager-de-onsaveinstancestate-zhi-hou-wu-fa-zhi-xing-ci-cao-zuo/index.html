解答

请在这里检查我的答案。基本上我只需要：

@Override
protected void onSaveInstanceState(Bundle outState) {
    //No call for super(). Bug on API Level > 11.
}

不要调用super()onsaveInstanceState方法。这把事情搞砸了...

这是支持包中的已知错误。

如果您需要保存实例并向您添加一些内容，outState Bundle您可以使用以下命令：

@Override
protected void onSaveInstanceState(Bundle outState) {
    outState.putString("WORKAROUND_FOR_BUG_19917_KEY", "WORKAROUND_FOR_BUG_19917_VALUE");
    super.onSaveInstanceState(outState);
}

最后，正确的解决方案是（如评论中所示）使用：

transaction.commitAllowingStateLoss();

添加或执行FragmentTransaction导致Exception.


更多建议

类似的错误信息有很多相关的问题。检查此特定堆栈跟踪的第二行。此异常与对FragmentManagerImpl.popBackStackImmediate.

如果会话状态已经保存，此方法调用popBackStack将 始终 失败，如 。IllegalStateException检查来源。您无法采取任何措施来阻止抛出此异常。

  • 删除呼叫super.onSaveInstanceState将无济于事。
  • 创建片段commitAllowingStateLoss将无济于事。

这是我观察到的问题：

  • 有一个带有提交按钮的表单。
  • 单击该按钮时，将创建一个对话框并启动一个异步过程。
  • 用户在进程完成之前单击主页键 -onSaveInstanceState被调用。
  • 该过程完成，进行回调并popBackStackImmediate尝试。
  • IllegalStateException被抛出。

这是我为解决它所做的工作：

由于无法避免IllegalStateException回调中的 ，因此捕获并忽略它。

try {
    activity.getSupportFragmentManager().popBackStackImmediate(name);
} catch (IllegalStateException ignored) {
    // There's no way to avoid getting this if saveInstanceState has already been called.
}

这足以阻止应用程序崩溃。但是现在用户将恢复应用程序并看到他们认为他们按下的按钮根本没有被按下（他们认为）。表单片段还在显示！

要解决此问题，请在创建对话框时创建一些状态以指示进程已启动。

progressDialog.show(fragmentManager, TAG);
submitPressed = true;

并将此状态保存在包中。

@Override
public void onSaveInstanceState(Bundle outState) {
    ...
    outState.putBoolean(SUBMIT_PRESSED, submitPressed);
}

不要忘记重新加载它onViewCreated

然后，在恢复时，如果之前尝试提交，则回滚片段。这可以防止用户返回到看似未提交的表单。

@Override
public void onResume() {
    super.onResume();
    if (submitPressed) {
        // no need to try-catch this, because we are not in a callback
        activity.getSupportFragmentManager().popBackStackImmediate(name);
        submitPressed = false;
    }
}
