从 GitHub 下载单个文件

Arya

2023-02-17 10:46

从 GitHub 存储库下载单个文件有哪些技巧？

我不想要显示原始文件的 URL；对于二进制文件，什么都没有。

http://support.github.com/discussions/feature-requests/41-download-single-file

甚至可以将 GitHub 用作“下载服务器”吗？

如果我们决定切换到 Google Code，那里是否提供了上述功能？

或者是否有针对开源项目的免费托管和 VCS？



解答

Git 不支持下载部分存储库。你必须下载所有这些。但是你应该可以用 GitHub 做到这一点。

当您查看文件时，它有一个指向“原始”版本的链接。URL是这样构造的

https://raw.githubusercontent.com/user/repository/branch/filename

通过填写 URL 中的空白，您可以使用Wget或cURL（带有-L选项，见下文）或其他任何工具来下载单个文件。同样，您不会通过这样做获得 Git 使用的任何不错的版本控制功能。

更新：我注意到你提到这不适用于二进制文件。您可能不应该在 Git 存储库中使用二进制文件，但 GitHub 为每个存储库提供了一个下载部分，您可以使用它来上传文件。如果您需要多个二进制文件，您可以使用 .zip 文件。下载上传文件的 URL 是：

https://github.com/downloads/user/repository/filename

请注意，上面给出的 URL，从 上的链接github.com，将重定向到raw.githubusercontent.com。您不应直接使用此 HTTP 302 重定向给出的 URL，因为根据RFC 2616：“由于有时可能会更改重定向，因此客户端应该继续使用 Request-URI 进行未来的请求。”
