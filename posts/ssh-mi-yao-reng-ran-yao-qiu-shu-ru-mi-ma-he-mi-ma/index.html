SSH 密钥 - 仍然要求输入密码和密码

Arya

2023-02-17 14:12

当我克隆存储库时，Github 总是要求我提供用户名和密码，这让我有些“忍受”。我想绕过这一步，因为它是我工作流程中的一个烦恼。

我尝试使用本指南设置 SSH 密钥（我成功地做到了）。https://help.github.com/articles/generating-ssh- keys我成功了。

我的问题是，在克隆存储库（使用 SSH）时，系统仍然要求我提供 github 密码和密码。我的理解是，在设置此 SSH 密钥后，我将不再需要这样做。

我有点不确定要问什么，所以我只陈述我的目标。

我希望能够克隆存储库而不必一直输入我的 Github 信息 。

我的 SSH 密钥缺少什么？如果有人可以提供一些指导或资源，我将不胜感激，因为在 GitHub 中进行 SSH 身份验证时，我总是感到有些迷茫。

据我所知，这是一个测试是否正常工作的命令，这是我的控制台的输出：

~ $ ssh -T git@github.com
Saving password to keychain failed
Enter passphrase for key '/Users/MYNAME/.ssh/id_rsa':
Hi MYNAME! You've successfully authenticated, but GitHub does not provide shell access.

当我输入密码时，应该先失败吗？然后，当我输入我的密码时，它通过了。



解答

如果你使用HTTPsurl，它总是会询问你的用户名/密码。这可以使用@Manavalan Gajapathy 的评论（在此处复制）来解决：

请参阅此github 文档以将远程 URL 从 https 转换为 ssh。要检查远程的 URL 是 ssh 还是 https，请使用git remote -v. 从 https 切换到 ssh：

git remote set-url origin git@github.com:USERNAME/REPOSITORY.git

如果您SSH在克隆/设置遥控器时正确使用：确保您有一个 ssh-agent 来记住您的密码（请参阅@Komu 的回答）。这样，您只需在终端会话中输入一次密码。

如果它仍然很烦人，那么只需设置一个没有密码的 ssh-key。
