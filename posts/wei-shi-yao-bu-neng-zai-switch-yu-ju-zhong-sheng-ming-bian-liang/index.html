为什么不能在 switch 语句中声明变量？

Arya

2023-02-17 06:02

我一直想知道这个——为什么不能在 switch 语句中的 case 标签之后声明变量？在 C++ 中，您几乎可以在任何地方声明变量（并且在接近首次使用时声明它们显然是一件好事）但以下内容仍然不起作用：

switch (val)  
{  
case VAL:  
  // This won't work
  int newVal = 42;  
  break;
case ANOTHER_VAL:  
  ...
  break;
}

以上给了我以下错误（MSC）：

'newVal' 的初始化被 'case' 标签跳过

这似乎也是其他语言的限制。为什么会出现这样的问题？



解答

Case陈述只是 标签 。这意味着编译器会将其解释为直接跳转到标签。在 C++ 中，这里的问题是作用域之一。您的大括号将范围定义为switch语句内的所有内容。这意味着您将留下一个范围，在该范围内将进一步执行跳转到跳过初始化的代码中。

处理这个问题的正确方法是定义一个特定于该case语句的范围并在其中定义您的变量：

switch (val)
{   
case VAL:  
{
  // This will work
  int newVal = 42;  
  break;
}
case ANOTHER_VAL:  
...
break;
}


更多建议

这个问题最初被标记为C和C++同时。原来的代码在C和C++中确实是无效的，但是出于完全不同的不相关的原因。

  • 在 C++ 中，此代码是无效的，因为case ANOTHER_VAL:标签绕过其初始化跳入了变量的范围newVal。绕过自动对象初始化的跳转在 C++ 中是非法的。大多数答案都正确解决了问题的这一方面。

  • 然而，在 C 语言中绕过变量初始化并不是错误。在 C 语言中，在变量的初始化过程中跳转到变量的作用域是合法的。它只是意味着变量未被初始化。由于完全不同的原因，原始代码无法在 C 中编译。原始代码中的标签case VAL:附在变量的声明中newVal。在 C 语言中，声明不是语句。他们不能被贴上标签。当此代码被解释为 C 代码时，这就是导致错误的原因。

      switch (val)

    {
    case VAL: / <- C error is here / int newVal = 42;
    break; case ANOTHER_VAL: / <- C++ error is here / ... break; }

添加一个额外的{}块可以修复 C++ 和 C 的问题，即使这些问题恰好非常不同。在 C++ 方面，它限制了 的范围newVal，确保case ANOTHER_VAL:不再跳入该范围，从而消除了 C++ 问题。在 C 端，extra{}引入了复合语句，从而使case VAL:标签应用于语句，从而消除了 C 问题。

  • 在 C 案例中，无需{}. 只需在标签后添加一个空语句case VAL:，代码就会生效
      switch (val)

    {
    case VAL:; / Now it works in C! / int newVal = 42;
    break; case ANOTHER_VAL:
    ... break; }

请注意，即使从 C 的角度来看它现在是有效的，但从 C++ 的角度来看它仍然无效。

  • 对称地，在 C++ 情况下，无需{}. 只需从变量声明中删除初始值设定项，代码就会生效
      switch (val)

    {
    case VAL: int newVal; newVal = 42;
    break; case ANOTHER_VAL: / Now it works in C++! / ... break; }

请注意，即使从 C++ 的角度来看它现在是有效的，但从 C 的角度来看它仍然无效。

从 C23 开始，C 语言中的所有标签都将被解释为标签隐含空语句 ( N2508 )，即无法将标签放在 C 声明前面的问题将不再存在，并且不再;需要基于上述的修复.
