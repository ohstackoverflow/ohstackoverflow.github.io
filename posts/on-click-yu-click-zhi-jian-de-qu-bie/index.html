.on('click') 与 .click() 之间的区别

Arya

2023-02-18 02:19

以下代码之间有什么区别吗？

$('#whatever').on('click', function() {
     /* your code here */
});

和

$('#whatever').click(function() {
     /* your code here */
});



解答

我认为，区别在于使用模式。

我.on更喜欢，.click因为前者 可以 使用更少的内存并且可以处理动态添加的元素。

考虑以下 html：

<html>
    <button id="add">Add new</button>
    <div id="container">
        <button class="alert">alert!</button>
    </div>
</html>

我们通过添加新按钮的地方

$("button#add").click(function() {
    var html = "<button class='alert'>Alert!</button>";
    $("button.alert:last").parent().append(html);
});

并想要“警报！” 显示警报。我们可以为此使用“点击”或“打开”。

当我们使用click 

$("button.alert").click(function() {
    alert(1);
});

有了上面的内容，就会为每个与选择器匹配的 元素 创建一个 单独的处理程序。 这意味着 __

  1. 许多匹配元素会创建许多相同的处理程序，从而增加内存占用
  2. 动态添加的项目将没有处理程序——即，在上面的 html 中，新添加的“Alert!” 除非您重新绑定处理程序，否则按钮将不起作用。

当我们使用.on 

$("div#container").on('click', 'button.alert', function() {
    alert(1);
});

有了上面的内容， 所有与您的选择器匹配的元素的 单个 处理程序，包括动态创建的元素。 __

...使用的另一个原因.on

正如 Adrien 在下面评论的那样，使用的另一个原因.on是命名空间事件。

如果您添加一个处理程序，.on("click", handler)您通常会删除它，.off("click", handler)这将删除该处理程序。显然，这只有在您引用该函数时才有效，如果没有怎么办？您使用名称空间：

$("#element").on("click.someNamespace", function() { console.log("anonymous!"); });

通过解除绑定

$("#element").off("click.someNamespace");

以下代码之间有什么区别吗？

不，您问题中的两个代码示例之间没有功能差异。.click(fn)是 . 的“快捷方式” .on("click", fn)。从文档中.on()：

某些事件有简写方法，例如.click()可用于附加或触发事件处理程序的方法。有关速记方法的完整列表，请参阅事件类别。

请注意，它.on()的不同之处.click()在于它能够通过传递参数来创建委托事件处理selector程序，而.click()不能。当.on()不带参数调用时selector，它的行为与.click(). 如果您想要事件委托，请使用.on().
