您应该将 .gitignore 提交到 Git 存储库中吗？

Arya

2023-02-17 23:48

.gitignore你认为提交到 Git 仓库是一个好习惯吗？

有些人不喜欢它，但我认为它很好，因为您可以跟踪文件的历史记录。不是吗？



解答

通常是的，.gitignore对每个想要使用存储库的人都有用。有时你会想忽略更多私人的东西（也许你经常创造LOG或其他东西。在那些情况下你可能不想把它强加给任何人。


更多建议

你通常 会 commit .gitignore。事实上，我个人会尽量确保我的索引在我不处理某事时始终保持干净。（git status应该什么都不显示。）

在某些情况下，您想要忽略确实不是项目特定的东西。例如，您的文本编辑器可能会创建自动*~备份文件，或者另一个示例是.DS_Store由 OS X 创建的文件。

我会说，如果其他人抱怨这些规则使您的 . 变得混乱.gitignore，请将它们排除在外，而不是将它们放在全局排除文件中。

默认情况下，此文件位于$XDG_CONFIG_HOME/git/ignore（默认为~/.config/git/ignore），但可以通过设置选项更改此位置core.excludesfile。例如：

git config --global core.excludesfile ~/.gitignore

只需根据您的喜好创建和编辑全局排除文件；它将应用于您在该机器上工作的每个 git 存储库。
