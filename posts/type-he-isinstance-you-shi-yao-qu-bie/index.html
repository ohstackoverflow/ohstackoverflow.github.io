type() 和 isinstance() 有什么区别？

Arya

2023-02-17 01:46

这两个代码片段之间有什么区别？

使用type：

import types

if type(a) is types.DictType:
    do_something()
if type(b) in types.StringTypes:
    do_something_else()

使用isinstance：

if isinstance(a, dict):
    do_something()
if isinstance(b, str) or isinstance(b, unicode):
    do_something_else()



解答

总结其他（已经很好！）答案的内容，isinstance迎合继承（派生类的实例 也是 基类的实例），同时检查是否相等type（它要求类型的标识并拒绝实例子类型，AKA 子类）。

通常，在 Python 中，您当然希望您的代码支持继承（因​​为继承非常方便，所以阻止使用您的代码使用它会很糟糕！），所以比检查 s 的身份要好得多，因为它无缝isinstance支持type遗产。

请注意，这并不是什么isinstance好事 —— 它只是比检查类型的相等性 要好。 正常的 Pythonic 首选解决方案几乎总是“鸭子打字”：尝试使用参数， 就好像 它是某种所需的类型一样，在try/except语句中执行它，捕获如果参数实际上不是该类型可能出现的所有异常类型（或任何其他类型很好地模仿它；-），并在子句中except尝试其他类型（使用参数“好像”它是其他类型）。

basestring 然而，这是 一个非常特殊的情况——一种内置类型，它的存在 只是 为了让你使用isinstance（两者str和unicode子类basestring）。字符串是序列（你可以遍历它们，索引它们，切片它们，......），但你通常希望将它们视为“标量”类型 - 处理各种类型有点不方便（但这是一个相当频繁的用例）字符串（可能还有其他标量类型，即你不能循环的那些）以一种方式，所有容器（列表，集合，字典，......）以另一种方式，加上帮助你做到这一点 - 这个的basestring整体isinstance结构成语是这样的：

if isinstance(x, basestring)
  return treatasscalar(x)
try:
  return treatasiter(iter(x))
except TypeError:
  return treatasscalar(x)

你可以说这basestring是一个 抽象基类 （“ABC”）——它不为子类提供具体的功能，而是作为一个“标记”存在，主要用于与isinstance. 这个概念在 Python 中显然是一个不断发展的概念，因为PEP 3119引入了它的概括，被接受并从 Python 2.6 和 3.0 开始实现。

PEP 清楚地表明，虽然 ABC 通常可以代替鸭子打字，但通常没有太大的压力去做（见这里）。然而，在最近的 Python 版本中实现的 ABC 确实提供了额外的好处：isinstance(and issubclass) 现在不仅仅意味着“派生类的[一个实例]”（特别是，任何类都可以用 ABC“注册”，这样它就可以显示为子类，其实例为 ABC 的实例）；并且 ABC 还可以通过模板方法设计模式应用程序以非常自然的方式为实际子类提供额外的便利（有关 TM DP 的更多信息，请参见此处和此处[[第二部分]]，一般而言，特别是在 Python 中，独立于 ABC） .

有关 Python 2.6 中提供的 ABC 支持的基础机制，请参见此处；他们的 3.1 版本非常相似，请参见此处。在这两个版本中，标准库模块集合（即 3.1 版本——对于非常相似的 2.6 版本，请参见此处）提供了几个有用的 ABC。

出于这个答案的目的，要保留关于 ABC 的关键事情（除了可以说是更自然的 TM DP 功能放置，与混合类的经典 Python 替代方案（例如UserDict.DictMixin相比）是它们使（和isinstance）issubclass更多与过去（在 Python 2.6 和之前的版本中）相比，它们具有吸引力和普遍性（在 Python 2.6 和以后的版本中），因此，相比之下，检查类型相等性在最近的 Python 版本中成为比过去更糟糕的做法。


更多建议

isinstance这是一个实现无法实现的东西的示例type：

class Vehicle:
    pass

class Truck(Vehicle):
    pass

在这种情况下，卡车对象是车辆，但您会得到：

isinstance(Vehicle(), Vehicle)  # returns True
type(Vehicle()) == Vehicle      # returns True
isinstance(Truck(), Vehicle)    # returns True
type(Truck()) == Vehicle        # returns False, and this probably won't be what you want.

换句话说，isinstance子类也是如此。
