将远程重置为某个提交

Arya

2023-02-17 07:42

我想放弃提交后所做的所有更改<commit-hash>。所以我做了：

git reset --hard <commit-hash>

现在我想用我的遥控器做同样的事情。我怎样才能做到这一点？之后我做了一些提交（和推送），<commit-hash>我只想放弃它们。只是在途中出现了 严重 错误，我不想让它变得比现在更糟。;(

我基本上想倒回我origin/master的<commit-hash>



解答

假设您的分支master在此处和远程都被调用，并且您的远程被调用，origin您可以这样做：

 git reset --hard <commit-hash>
 git push -f origin master

但是，如果其他人正在使用您的远程存储库并拉取了您的更改，您应该避免这样做。在这种情况下，最好还原不需要的提交，然后照常推送。

更新： 你已经在下面解释过其他人已经撤回了你推送的更改，所以最好 创建一个新的提交来恢复所有这些更改 。 在Jakub Narębski 的这个回答 中，您可以很好地解释执行此操作的选项。哪一个最方便取决于您要还原多少提交，以及哪种方法对您最有意义。

由于从您的问题中可以清楚地看出您已经使用过git reset --hard重置master分支，因此您可能需要先使用git reset --hard ORIG_HEAD将分支移回原来的位置。（与往常一样git reset --hard，确保它git status是干净的，您在正确的分支上，并且您知道git reflog这是一种恢复明显丢失的提交的工具。）您还应该检查ORIG_HEAD指向正确提交的点，使用git show ORIG_HEAD.

故障排除：

如果您收到类似“ ! [remote rejected] a60f7d85 - > master (pre-receive hook declined) ”的消息

那么您必须允许为特定分支重写分支历史记录。例如在 BitBucket 中它说“不允许重写分支历史”。有一个名为Allow rewriting branch history您必须选中的复选框。


更多建议

使用其他一些答案可能会导致不必要的本地状态丢失。 更改远程并非固有地需要本地更改。如果您选择了错误的提交返回，此方法仍然会破坏您的远程，但即使这样您通常也可以找到正确的提交并重试。

您 必须 在您希望远程匹配的本地存储库中的某处进行所需的提交。

  1. 不要 做 任何重置。
  2. 用于git log查找您希望远程所在的提交。用于git log -p查看更改或git log --graph --all --oneline --decorate查看紧凑的树。
  3. 复制提交的散列、标记或（如果是提示）其分支名称。
  4. 运行如下命令：
    git push --force <remote> <commit-ish>:<the remote branch>

例如

    git push --force origin 606fdfaa33af1844c86f4267a136d4666e576cdc:master

或者

    git push --force staging v2.4.0b2:releases

如果强制推送失败，它可能被遥控器禁用了。receive.denyNonFastForwards这可以通过临时更改和receive.denyDeletes中的一个或两个来解决。如果您的遥控器托管在没有 shell 访问权限的服务上，则它可能具有您可以更改的设置以允许强制推送。

我在步骤 2 中使用方便的别名 ( git go) 来查看历史记录，可以像这样添加：

git config --global alias.go 'log --graph --all --decorate --oneline'
