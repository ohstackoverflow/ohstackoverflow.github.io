我怎样才能在 Git 中只存储分阶段的更改？

Arya

2023-02-17 18:34

有没有办法可以只存储我的分阶段更改？我遇到问题的场景是我在给定时间处理了几个错误，并且有几个未暂存的更改。我希望能够单独暂存这些文件，创建我的 .patch 文件，并将它们隐藏起来，直到代码获得批准。这样，当它被批准时，我可以隐藏我的整个（当前）会话，弹出那个错误并推送代码。

我会以错误的方式解决这个问题吗？我是否误解了 git 如何以其他方式工作来简化我的流程？



解答

这个答案被接受但很旧。 现在你可以git stash -S看到https://stackoverflow.com/a/70231955/683157

原始答案： 是的，可以使用 DOUBLE STASH

  1. 暂存所有需要存储的文件。
  2. 运行git stash --keep-index。 此命令将创建一个包含所有 更改（ 已暂存和未暂存 ）的存储，但会将已暂存的更改保留在您的工作目录中（仍处于已暂存状态）。
  3. 运行git stash push -m "good stash"（添加-u标志以在存储中包含新文件）
  4. 现在你"good stash"只有 暂存文件 。

现在，如果您在存储之前需要未暂存的文件，只需应用第一个存储（ 创建的那个--keep-index），现在您可以删除存储到的文件"good stash"。

更多建议

git stash push -S

自 2.35 以来的旧
git stash push --staged#

仅存储当前暂存的更改。这类似于基本的git commit，除了状态被提交到存储而不是当前分支。

老的

使用最新的 git 你可以使用--patch选项

git stash push --patch   # since 2.14.6

git stash save --patch   # for older git versions

git 会询问您文件中的每个更改是否添加到存储中。
你只要回答y或n

**DOUBLE STASH的 旧 UPD
别名： ****

git config --global alias.stash-staged '!bash -c "git stash --keep-index; git stash push -m "staged" --keep-index; git stash pop stash@{1}"'

现在您可以暂存文件，然后运行git stash-staged​​.
结果， 您暂存的文件将保存到 stash 中 。

如果您不想保留暂存文件并希望将它们移动到存储中。然后你可以添加另一个别名并运行git move-staged：

git config --global alias.move-staged '!bash -c "git stash-staged;git commit -m "temp"; git stash; git reset --hard HEAD^; git stash pop"'
