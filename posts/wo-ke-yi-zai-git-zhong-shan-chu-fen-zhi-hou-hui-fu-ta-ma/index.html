我可以在 Git 中删除分支后恢复它吗？

Arya

2023-02-17 02:14

如果我运行git branch -d XYZ，有没有办法恢复分支？有没有办法像我没有运行删除分支命令一样返回？



解答

是的，您应该能够git reflog --no-abbrev在已删除分支的尖端找到提交的 SHA1，然后就可以了git checkout [sha]。一旦你进行了那个提交，你就可以git checkout -b [branchname]从那里重新创建分支。

感谢@Cascabel 提供此压缩/单行版本，感谢@Snowcrash 了解如何获取 sha。

如果您刚刚删除了分支，您将在终端中看到类似这样的内容Deleted branch <your-branch> (was <sha>)。<sha>然后在这个单行中使用它：

git checkout -b <your-branch> <sha>


更多建议

当你的提交在reflog

大多数情况下，无法访问的提交都在 reflog 中。因此， 首先要尝试的是 使用命令查看 reflog git reflog（显示 的 reflog HEAD）。

git reflog name-of-my- branch如果提交是特定且仍然存在的分支的一部分，那么使用命令可能更容易。它也可以与遥控器一起使用，例如，如果你用力推动（尽管应该使用它git push --force-with-lease来防止错误并且更容易恢复）。

当他们不在reflog

如果您的提交不在您的 reflog 中（可能它们被不写入 reflog 的第 3 方工具删除），您可以先尝试此命令来创建一个包含所有悬空提交的文件

git fsck --full --no-reflogs --unreachable --lost-found | grep commit | cut -d\  -f3 | xargs -n 1 git log -n 1 --pretty=oneline > .git/lost-found.txt

然后读取丢失的提交的 SHA 并将你的分支重置到它。

git rescue频繁的用户可以使用创建别名

git config --global alias.rescue '!git fsck --full --no-reflogs --unreachable --lost-found | grep commit | cut -d\  -f3 | xargs -n 1 git log -n 1 --pretty=oneline > .git/lost-found.txt'

以下是一些示例，展示了如何分析找到的提交

显示提交元数据（作者、创建日期和提交消息）：

git cat-file -p 48540dfa438ad8e442b18e57a5a255c0ecad0560

另见差异：

git log -p 48540dfa438ad8e442b18e57a5a255c0ecad0560

在找到的提交上创建一个分支：

git branch commit_rescued 48540dfa438ad8e442b18e57a5a255c0ecad0560

Windows GUI 可以通过菜单=> =>使用GitExtensions轻松恢复提交（也包括未提交的暂存文件）Repository``Git maintenance``Recover lost objects...
