angular.service 与 angular.factory

Arya

2023-02-17 06:14

我已经看到angular.factory()和angular.service()都用于声明服务；但是，我在官方文档中找不到任何地方。 angular.service

这两种方法有什么区别？
哪个应该用于什么（假设他们做不同的事情）？



解答

  angular.service('myService', myServiceFunction);
  angular.factory('myFactory', myFactoryFunction);

在我这样对自己说之前，我很难理解这个概念：

服务 ：您编写的 函数 将是 新 的：

  myInjectedService  <----  new myServiceFunction()

工厂 ：将 调用您编写的 函数 （构造函数） ： ****

  myInjectedFactory  <---  myFactoryFunction()

你用它做什么取决于你，但有一些有用的模式......

比如写一个 服务 函数暴露一个公共API：

function myServiceFunction() {
  this.awesomeApi = function(optional) {
    // calculate some stuff
    return awesomeListOfValues;
  }
}
---------------------------------------------------------------------------------
// Injected in your controller
$scope.awesome = myInjectedService.awesomeApi();

或者使用 工厂 函数公开公共 API：

function myFactoryFunction() {
  var aPrivateVariable = "yay";

  function hello() {
    return "hello mars " + aPrivateVariable;
  }

  // expose a public API
  return {
    hello: hello
  };
}
---------------------------------------------------------------------------------
// Injected in your controller
$scope.hello = myInjectedFactory.hello();

或者使用 工厂 函数返回构造函数：

function myFactoryFunction() {
    return function() {
        var a = 2;
        this.a2 = function() {
            return a*2;
        };
    };
}
---------------------------------------------------------------------------------
// Injected in your controller
var myShinyNewObject = new myInjectedFactory();
$scope.four = myShinyNewObject.a2();

用哪个好？...

你可以用两者完成同样的事情。然而，在某些情况下， 工厂 为您提供了更多的灵活性，可以使用更简单的语法创建可注入的对象。这是因为虽然 myInjectedService 必须始终是对象，但 myInjectedFactory 可以是对象、函数引用或任何值。例如，如果您编写了一个服务来创建构造函数（如上面最后一个示例所示），则必须像这样对其进行实例化：

var myShinyNewObject = new myInjectedService.myFunction()

这可以说比这更不可取：

var myShinyNewObject = new myInjectedFactory();

（但是首先你应该警惕使用这种类型的模式，因为控制器中的 new -ing 对象会创建难以跟踪的依赖关系，这些依赖关系难以模拟以进行测试。最好让服务管理对象集合你比new()狡猾地使用。）

还有一点，他们都是单身人士......

还要记住，在这两种情况下，角度都在帮助您管理单身人士。无论您在何处或多少次注入您的服务或功能，您都将获得对同一对象或功能的相同引用。（当工厂简单地返回一个值如数字或字符串时除外。在这种情况下，您将始终获得相同的值，但不是引用。）


更多建议

简单的说 ..

const user = {
  firstName: 'john'
};

// Factory
const addLastNameFactory = (user, lastName) => ({
  ...user,
  lastName,
});

console.log(addLastNameFactory(user, 'doe'));

// Service
const addLastNameService = (user, lastName) => {
  user.lastName = lastName; // BAD! Mutation
  return user;
};

console.log(addLastNameService(user, 'doe'));
