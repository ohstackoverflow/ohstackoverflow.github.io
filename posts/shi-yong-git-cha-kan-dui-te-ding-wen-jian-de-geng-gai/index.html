使用 git 查看对特定文件的更改

Arya

2023-02-17 19:33

我知道我可以使用git diff命令来检查更改，但据我所知，它是基于目录的。这意味着它给出了当前目录中所有文件的所有更改。

如何只检查一个特定文件中的更改？ 说，我已经更改了文件file_1.rb，，file_2.rb...，，file_N.rb但我只对文件中的更改感兴趣file_2.rb。那么我如何检查这些更改（在我提交之前）？



解答

使用如下命令：

git diff file_2.rb

请参阅git diff文档以获取有关您可以获得差异的各种事物的完整信息。

通常，它本身会显示整个 存储库git diff中的所有更改（而不仅仅是当前目录）。 __


更多建议

另一种方法（在这个 SO answer中提到）将在终端中保留历史记录，并为您提供文件本身的非常深入的跟踪记录：

git log --follow -p -- file

这将显示文件的整个历史记录（包括重命名之后的历史记录和每次更改的差异）。

换句话说，如果名为 bar 的文件曾经被命名为 foo，那么 git log -p bar（没有 --follow 选项）将只显示文件在重命名之前的历史记录——它不会显示文件被称为 foo 时的历史记录。使用 git log --follow -p bar 将显示文件的整个历史记录，包括文件被称为 foo 时的任何更改。
