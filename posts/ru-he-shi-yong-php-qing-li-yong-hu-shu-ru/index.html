如何使用 PHP 清理用户输入？

Arya

2023-02-17 04:06

是否有一个 catchall 函数可以很好地为 SQL 注入和 XSS 攻击清理用户输入，同时仍然允许某些类型的 HTML 标记？



解答

用户输入可以被过滤是一个常见的误解。PHP 甚至有一个（现已弃用）“功能”，称为magic- quotes，它建立在这个想法之上。这是胡说八道。忘掉过滤（或清洁，或人们所说的任何东西）。

为避免出现问题，您应该做的非常简单：无论何时在外部代码中嵌入一段数据，都必须根据该代码的格式规则处理它。但您必须明白，此类规则可能过于复杂，无法尝试手动遵循它们。例如，在 SQL 中，字符串、数字和标识符的规则都是不同的。为了您的方便，在大多数情况下，有一个专门用于此类嵌入的工具。例如，当您需要在 SQL 查询中使用 PHP 变量时，您必须使用准备好的语句，它将处理所有正确的格式/处理。

另一个例子是 HTML：如果在 HTML 标记中嵌入字符串，则必须使用htmlspecialchars. 这意味着每个echoorprint语句都应该使用htmlspecialchars.

第三个示例可能是 shell 命令：如果您要将字符串（例如参数）嵌入到外部命令中，并使用 调用它们，exec那么您必须使用escapeshellcmdand escapeshellarg。

此外，一个非常引人注目的例子是 JSON。规则如此众多和复杂，以至于您永远无法手动遵循它们。这就是为什么你永远不应该手动创建 JSON 字符串，而应该始终使用专用函数来json_encode()正确格式化每一位数据。

等等等等 ...

您需要主动过滤数据的唯一情况是您接受预先格式化的输入 。 例如，如果您让您的用户发布您计划在网站上显示的 HTML 标记。但是，您应该明智地不惜一切代价避免这种情况，因为无论您如何过滤它，它始终是一个潜在的安全漏洞。


更多建议

不要试图通过清理输入数据来防止 SQL 注入。

相反， 不允许在创建 SQL 代码时使用数据 。使用使用绑定变量的准备语句（即在模板查询中使用参数）。这是唯一可以防止 SQL 注入的方法。
