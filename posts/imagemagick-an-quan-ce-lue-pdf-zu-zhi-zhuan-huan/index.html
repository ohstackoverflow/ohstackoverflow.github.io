ImageMagick 安全策略“PDF”阻止转换

Arya

2023-02-17 18:35

Imagemagick 安全策略似乎不允许我执行从 pdf 到 png 的转换。转换其他扩展名似乎有效，只是不适用于 pdf。自从安装 imagemagick 后，我没有更改任何设置...如果操作系统很重要，我正在使用 Arch Linux。

user@machine $ convert -density 300 -depth 8 -quality 90 input.pdf output.png
convert: attempt to perform an operation not allowed by the security policy `PDF' @ error/constitute.c/IsCoderAuthorized/408.
convert: no images defined `output.png' @ error/convert.c/ConvertImageCommand/3288.



解答

在修复 Ghostscript 后，ImageMagick 的更改得以保留，因为应用程序（尤其是 Web 应用程序）经常将任意用户提供的文件提供给 ImageMagick，并不总是正确地执行格式限制，而且由于 Postscript（PDF 使用）是一个图灵完备的编程在沙箱中运行的语言，沙箱中总是存在另一个漏洞的可能性。

最好保留配置，这样 ImageMagick 拒绝处理需要运行程序的文件，而是在您 有意 允许 Postscript 渲染时直接调用 Ghostscript。

这将通过这样的 Ghostscript 命令来完成：

gs -dSAFER -r600 -sDEVICE=pngalpha -o foo.png myfile.pdf

是的，这是 GhostScript 命令 ImageMagic 调用的变体。（参见 ImageMagick 的delegates.xml.-o是 的简写-dBATCH -dNOPAUSE -sOutputFile=）

重要的是 ImageMagick 保持锁定状态，您不会不必要地调用中间程序，并且您可以更好地控制渲染参数。（例如，-r600渲染的 DPI 是否-sDEVICE=pngalpha允许您直接渲染成您想要的格式）


更多建议

此问题是安全漏洞的解决方法。该漏洞已在 Ghostscript 9.24 中得到解决，因此如果您拥有该版本或更新版本，则不再需要解决方法。

在 Ubuntu 19.04 到 22.04 以及可能带有 ImageMagick 6 的任何更高版本上，以下是通过删除该解决方法来解决问题的方法：

  1. 确保你的 Ghostscript ≥9.24：

    gs --version

  2. 如果是，只需从以下部分中删除整个以下部分/etc/ImageMagick-6/policy.xml：

    <!-- disable ghostscript format types -->

仅删除带有 inside 的行pattern="PDF"就足以重新启用 PDF 转换。不过，我看不出为其他基于 PostScript 的文件类型保留解决方法的充分理由。
