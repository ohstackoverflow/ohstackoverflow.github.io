vim 的“write with sudo”技巧是如何工作的？

Arya

2023-02-17 00:50

你们中的许多人可能已经看到允许您在需要 root 权限的文件上写入的命令，即使您忘记使用 sudo 打开 vim：

:w !sudo tee %

问题是我不明白这里到底发生了什么。

我已经想到了：w是为了这个

                                                        *:w_c* *:write_c*
:[range]w[rite] [++opt] !{cmd}
                        Execute {cmd} with [range] lines as standard input
                        (note the space in front of the '!').  {cmd} is
                        executed like with ":!{cmd}", any '!' is replaced with
                        the previous command |:!|.

所以它将所有行作为标准输入传递。

该!sudo tee部分tee以管理员权限调用。

为了使一切都有意义，%应该输出文件名（作为 的参数tee），但我找不到有关此行为的帮助的参考。

tl; dr 有人可以帮我剖析这个命令吗？



解答

在:w !sudo tee %...

%意思是“当前文件”

正如eugene y 指出的那样，%确实意味着“当前文件名”，传递给它tee以便它知道要覆盖哪个文件。

（在替换命令中，它略有不同；如图:help :%所示，它是equal to 1,$ (the entire file)（感谢@Orafu 指出这不会计算出文件名）。例如，:%s/foo/bar表示“ 在当前文件中 ，将出现的内容替换为foo” bar。如果您突出显示键入之前的一些文本:s，您会看到突出显示的行取代了%您的替换范围。）

:w没有更新你的文件

这个技巧的一个令人困惑的部分是您可能认为:w正在修改您的文件，但事实并非如此。如果打开修改file1.txt后运行:w file2.txt，就是“另存为”；file1.txt不会被修改，但当前缓冲区内容将被发送到file2.txt.

取而代之的是file2.txt，您可以 用 shell 命令代替来接收缓冲区内容 。例如，:w !cat将只显示内容。

如果 Vim 没有以 sudo 访问权限运行，它就:w不能修改受保护的文件，但如果它将缓冲区内容传递给 shell，则 可以 使用 sudo 运行 shell 中的命令 __。 在这种情况下，我们使用tee.

认识tee

至于tee，将命令想象tee成正常 bash 管道情况下的 T 形管道：它将输出定向到指定的文件，并将 其发送到标准输出 ，可以被下一个管道命令捕获。

例如，在 中ps -ax | tee processes.txt | grep 'foo'，进程列表将写入一个文本文件 并 传递给grep.

     +-----------+    tee     +------------+
     |           |  --------  |            |
     | ps -ax    |  --------  | grep 'foo' |
     |           |     ||     |            |
     +-----------+     ||     +------------+
                       ||   
               +---------------+
               |               |
               | processes.txt |
               |               |
               +---------------+

（用Asciiflow创建的图表。）

有关更多信息，请参见tee手册页。

Tee 作为黑客

在您的问题描述的情况下， 使用tee是一种 hack，因为我们忽略了它所做的一半。sudo tee写入我们的文件并将缓冲区内容发送到标准输出，但 我们忽略标准输出 。在这种情况下，我们不需要将任何东西传递给另一个管道命令；我们只是将其用作tee另一种写入文件的方式，以便我们可以使用sudo.

让这个技巧变得简单

您可以将其添加到您的应用程序中.vimrc，使该技巧易于使用：只需键入:w!!.

" Allow saving of files as sudo when I forgot to start vim using sudo.
cmap w!! w !sudo tee > /dev/null %

该> /dev/null部分 明确地 丢弃了标准输出，因为正如我所说，我们不需要将任何东西传递给另一个管道命令。


更多建议

在执行的命令行中，%代表 当前文件名 。这记录在:help cmdline-special：

In Ex commands, at places where a file name can be used, the following
characters have a special meaning.
        %       Is replaced with the current file name.

正如您已经发现的那样，:w !cmd将当前缓冲区的内容通过管道传递给另一个命令。所做tee的是将标准输入复制到一个或多个文件，以及复制到标准输出。因此， 作为 root:w !sudo tee % > /dev/null有效地将当前缓冲区的内容写入当前文件。另一个可用于此的命令是： ****dd

:w !sudo dd of=% > /dev/null

作为快捷方式，您可以将此映射添加到您的.vimrc：

" Force saving files that require root permission 
cnoremap w!! w !sudo tee > /dev/null %

使用上面的内容，您可以键入:w!!<Enter>以将文件另存为根目录。
