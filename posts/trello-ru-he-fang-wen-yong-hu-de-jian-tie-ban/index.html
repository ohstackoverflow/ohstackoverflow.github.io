Trello 如何访问用户的剪贴板？

Arya

2023-02-17 08:36

当您将鼠标悬停在Trello中的卡片上并按Ctrl+时C，该卡片的 URL 将被复制到剪贴板。他们如何做到这一点？

据我所知，没有涉及 Flash 电影。我已经安装了Flashblock，但 Firefox 网络选项卡显示没有加载 Flash 电影。（这是通常的方法，例如 ZeroClipboard。）

他们如何实现这种魔力？

（就在这一刻，我想我顿悟了：你不能选择页面上的文本，所以我假设他们有一个不可见的元素，他们通过 JavaScript 代码创建一个文本选择，并且 + 触发浏览器的默认行为，复制那个不Ctrl可见C的节点的文本值。）



解答

披露： 我编写了 Trello 使用的代码；下面的代码是 Trello 用来完成剪贴板技巧的实际源代码。

Ctrl我们实际上并没有“访问用户的剪贴板”，而是通过在用户按下+时选择一些有用的东西来帮助用户C。

听起来你已经明白了；我们利用了这样一个事实，即当你想点击Ctrl+时C，你必须Ctrl先点击 key。按下该键时，我们会弹出一个文本区域，其中包含我们希望在剪贴板上结束的文本，并选择其中的所有文本，因此在按下Ctrl该键时所有选择都已设置。（然后我们在按键出现C时隐藏文本区域。）Ctrl

具体来说，Trello 是这样做的：

TrelloClipboard = new class
  constructor: ->
    @value = ""

    $(document).keydown (e) =>
      # Only do this if there's something to be put on the clipboard, and it
      # looks like they're starting a copy shortcut
      if !@value || !(e.ctrlKey || e.metaKey)
        return

      if $(e.target).is("input:visible,textarea:visible")
        return

      # Abort if it looks like they've selected some text (maybe they're trying
      # to copy out a bit of the description or something)
      if window.getSelection?()?.toString()
        return

      if document.selection?.createRange().text
        return

      _.defer =>
        $clipboardContainer = $("#clipboard-container")
        $clipboardContainer.empty().show()
        $("<textarea id='clipboard'></textarea>")
        .val(@value)
        .appendTo($clipboardContainer)
        .focus()
        .select()

    $(document).keyup (e) ->
      if $(e.target).is("#clipboard")
        $("#clipboard-container").empty().hide()

  set: (@value) ->

在 DOM 中，我们有：

<div id="clipboard-container"><textarea id="clipboard"></textarea></div>

剪贴板内容的 CSS：

#clipboard-container {
  position: fixed;
  left: 0px;
  top: 0px;
  width: 0px;
  height: 0px;
  z-index: 100;
  display: none;
  opacity: 0;
}
#clipboard {
  width: 1px;
  height: 1px;
  padding: 0px;
}

... 并且 CSS 做到了这一点，因此当它弹出时您实际上看不到文本区域 ... 但它“可见”到足以从中复制。

当您将鼠标悬停在卡片上时，它会调用

TrelloClipboard.set(cardUrl)

...这样剪贴板助手就知道Ctrl按下该键时要选择什么。

我实际上构建了一个 Chrome 扩展程序，它可以做到这一点，并且适用于所有网页。源代码在 GitHub 上。

我发现了 Trello 方法的三个错误，我知道这些错误是因为我自己遇到过它们 :)

该副本在这些情况下不起作用：

  1. 如果您已经Ctrl按下然后将鼠标悬停在链接上并点击C，则复制不起作用。
  2. 如果您的光标位于页面中的某个其他文本字段中，则复制不起作用。
  3. 如果您的光标位于地址栏中，则复制不起作用。

我通过总是有一个隐藏的跨度来解决#1，而不是在用户点击Ctrl/时创建一个Cmd。

我通过暂时清除零长度选择、保存插入符号位置、进行复制并恢复插入符号位置来解决#2。

我还没有找到 #3 的修复方法 :)（有关信息，请查看我的 GitHub 项目中的未解决问题）。
