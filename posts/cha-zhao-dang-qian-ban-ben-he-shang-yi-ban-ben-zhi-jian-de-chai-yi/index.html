查找当前版本和上一版本之间的差异

Arya

2023-02-17 11:37

使用 Git，如何找到当前版本和上一版本之间的差异？

git diff last version:HEAD



解答

我真的不明白“最新版本”的含义。

由于可以使用 HEAD^ 访问先前的提交，我认为您正在寻找类似的东西：

git diff HEAD^ HEAD

这也可以申请 :commithash

git diff $commithash^ $commithash

从 Git 1.8.5 开始，@是 的别名HEAD，因此您可以使用：

git diff @~..@

以下内容也将起作用：

git show

如果你想知道 head 和任何提交之间的差异，你可以使用：

git diff commit_id HEAD

这将启动您的视觉差异工具（如果已配置）：

git difftool HEAD^ HEAD

由于与 HEAD 的比较是默认的，因此您可以省略它（正如Orient所指出的）：

git diff @^
git diff HEAD^
git diff commit_id

警告

  • @ScottF 和@Panzercrisis 在评论中解释说在 Windows 上~必须使用字符而不是^.


更多建议

假设“当前版本”是工作目录（未提交的修改）而“最新版本”是HEAD（当前分支的最后提交的修改），只需执行

git diff HEAD

以下归功于用户Cerran。

如果你-a在提交时总是跳过暂存区，那么你可以简单地使用git diff.

概括

  1. git diff显示未暂存的更改。
  2. git diff --cached显示阶段性变化。
  3. git diff HEAD显示所有更改（已上演和未上演）。
