'$(this)' 和 'this' 有什么区别？

Arya

2023-02-18 03:07

我目前正在学习本教程：jQuery 入门

对于下面的两个例子：

$("#orderedlist").find("li").each(function (i) {
    $(this).append(" BAM! " + i);
});
$("#reset").click(function () {
    $("form").each(function () {
        this.reset();
    });
});

请注意，在第一个示例中，我们使用$(this)在每个元素内附加一些文本li。在第二个例子中我们this在重置表单时直接使用。

$(this)似乎比 . 更经常被使用this。

我的猜测是在第一个示例中，$()将每个li元素转换为一个理解函数的 jQuery 对象append()，而在第二个示例中reset()可以直接在表单上调用。

基本上我们需要$()特殊的 jQuery-only 函数。

它是否正确？



解答

$()是的，只有在使用 jQuery 时才需要。如果你想要 jQuery 的帮助来做 DOM 的事情，请记住这一点。

$(this)[0] === this

基本上每次您返回一组元素时，jQuery 都会将其转换为jQuery 对象。如果您知道只有一个结果，那么它将位于第一个元素中。

$("#myDiv")[0] === document.getElementById("myDiv");

等等...


更多建议

$()是 jQuery 构造函数。

this是对调用的 DOM 元素的引用。

所以基本上，在 中$(this)，您只是将thisin$()作为参数传递，这样您就可以调用 jQuery 方法和函数。
