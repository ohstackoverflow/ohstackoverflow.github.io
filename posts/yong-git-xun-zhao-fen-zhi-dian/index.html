用 Git 寻找分支点？

Arya

2023-02-18 04:00

我有一个包含分支 master 和 A 的存储库，并且两者之间有很多合并活动。当基于 master 创建分支 A 时，如何在我的存储库中找到提交？

我的存储库基本上是这样的：

-- X -- A -- B -- C -- D -- F  (master) 
          \     /   \     /
           \   /     \   /
             G -- H -- I -- J  (branch A)

我正在寻找修订版 A，但找不到git merge-base (--all)。



解答

我一直在寻找同样的东西，我发现了这个问题。谢谢你的提问！

但是，我发现我在这里看到的答案似乎并没有 完全 给出您要求的（或我正在寻找的）答案——它们似乎给出了提交G，而不是A提交。

因此，我创建了以下树（按时间顺序分配的字母），因此我可以进行测试：

A - B - D - F - G   <- "master" branch (at G)
     \   \     /
      C - E --'     <- "topic" branch (still at E)

这看起来与你的有点不同，因为我想确保我得到（指的是这张图，而不是你的）B，但不是 A（而不是 D 或 E）。以下是附加到 SHA 前缀和提交消息的字母（如果有人对此感兴趣，可以从此处克隆我的存储库）：

G: a9546a2 merge from topic back to master
F: e7c863d commit on master after master was merged to topic
E: 648ca35 merging master onto topic
D: 37ad159 post-branch commit on master
C: 132ee2a first commit on topic branch
B: 6aafd7f second commit on master before branching
A: 4112403 initial commit on master

所以， 目标：找到 B 。经过一些修补，这是我发现的三种方法：

1. 在视觉上，使用 gitk：

你应该在视觉上看到这样一棵树（从主人那里看）：

B在这两种情况下，我都选择了图表中的提交。单击它后，其完整的 SHA 将显示在图表正下方的文本输入字段中。

2. 视觉上，但从终端：

git log --graph --oneline --all

或者，在纯文本中：

|\

| |\

在任何一种情况下，我们都将 6aafd7f 提交视为最低公共点，即B在我的图表中或A在您的图表中。

3.用壳魔法：

你没有在你的问题中指定你是否想要像上面这样的东西，或者一个单一的命令只会让你得到一个修订，而不是别的。好吧，这是后者：

diff -u <(git rev-list --first-parent topic) \
             <(git rev-list --first-parent master) | \
     sed -ne 's/^ //p' | head -1
6aafd7ff98017c816033df18395c5c1e7829960d

您也可以将其放入您的 ~/.gitconfig 作为 （注意：尾随破折号很重要；感谢 Brian引起注意）：

[alias]
    oldest-ancestor = !zsh -c 'diff -u <(git rev-list --first-parent "${1:-master}") <(git rev-list --first-parent "${2:-HEAD}") | sed -ne \"s/^ //p\" | head -1' -

这可以通过以下（复杂的引用）命令行来完成：

git config --global alias.oldest-ancestor '!zsh -c '\''diff -u <(git rev-list --first-parent "${1:-master}") <(git rev-list --first-parent "${2:-HEAD}") | sed -ne "s/^ //p" | head -1'\'' -'

注意：zsh本来可以很容易bash，但sh不会 起作用 ——<()语法在 vanilla 中不存在sh。（再次感谢@conny，感谢您在对此页面上另一个答案的评论中让我意识到这一点！）

注意：上面的替代版本：

感谢liori指出上面的内容在比较相同的分支时可能会失败，并提出了一个替代的 diff 形式，它从混合中删除了 sed 形式，并使它“更安全”（即它返回一个结果（即，最近的提交）即使你比较大师与大师）：

作为 .git-config 行：

[alias]
    oldest-ancestor = !zsh -c 'diff --old-line-format='' --new-line-format='' <(git rev-list --first-parent "${1:-master}") <(git rev-list --first-parent "${2:-HEAD}") | head -1' -

从外壳：

git config --global alias.oldest-ancestor '!zsh -c '\''diff --old-line-format='' --new-line-format='' <(git rev-list --first-parent "${1:-master}") <(git rev-list --first-parent "${2:-HEAD}") | head -1'\'' -'

因此，在我的测试树中（有一段时间不可用，对不起；它回来了），现在可以在 master 和 topic 上工作（分别给出提交 G 和 B）。再次感谢 liori，提供替代形式。

所以，这就是我 [和 liori] 想出的办法。它似乎对我有用。它还允许额外的几个别名，这些别名可能会很方便：

git config --global alias.branchdiff '!sh -c "git diff `git oldest-ancestor`.."'
git config --global alias.branchlog '!sh -c "git log `git oldest-ancestor`.."'

快乐 git-ing！


更多建议

您可能正在寻找git merge-base：

git merge-base 找到两个提交之间的最佳共同祖先以用于三向合并。如果后者是前者的祖先，则一个共同祖先 优于另一个共同祖先。 没有任何更好的共同祖先的共同祖先是 最佳共同祖先 ，即 合并基础 。请注意，一对提交可以有多个合并基础。
