如何修复提交到错误的 Git 分支？

Arya

2023-02-17 14:26

我刚刚对错误的分支做出了完美的承诺。我如何撤消我的主分支中的最后一次提交，然后进行相同的更改并将它们放入我的升级分支中？



解答

如果您尚未推送更改，您还可以进行软重置：

git reset --soft HEAD^

这将恢复提交，但将提交的更改放回您的索引中。假设分支相对于彼此而言是最新的，git 将允许您检出另一个分支，然后您可以简单地提交：

git checkout branch
git commit -c ORIG_HEAD

该-c ORIG_HEAD部分对于不再键入提交消息很有用。


更多建议

这个话题晚了 4 年，但这可能对某人有帮助。

如果你忘记在提交之前创建一个新的分支并且全部提交到 master 上，无论你做了多少次提交，下面的方法更容易：

git stash                       # skip if all changes are committed
git branch my_feature
git reset --hard origin/master
git checkout my_feature
git stash pop                   # skip if all changes were committed

现在你的 master 分支等于origin/master并且所有新提交都在my_feature. 请注意，这my_feature是本地分支，而不是远程分支。
