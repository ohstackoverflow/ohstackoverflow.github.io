git switch 和 git checkout <branch> 有什么区别

Arya

2023-02-17 19:59

Git 2.23引入了一个新命令git switch——阅读文档后，它看起来几乎相同，git checkout <branchname>有人可以解释一下区别或用例吗？

引入了两个新命令“git switch”和“git restore”来拆分“检出一个分支以推进其历史”和“检出索引和/或树状结构以推进当前的路径”从单个“git checkout”命令中提取“历史记录”。



解答

该switch命令确实与 执行相同的操作checkout，但仅适用于切换分支的那些用法。特别是，不像checkout它不能恢复工作树文件——而是使用restore与switch.

详细解释

正如您在引用的 2.23.0 发行说明部分中所述，引入了switch和命令以将命令拆分为两个单独的部分：restore``checkout

  • “检查一个分支以推进其历史”
  • “检查索引和/或树状结构的路径以推进当前历史”

换句话说，checkout做两件不同的事情，而这个版本将这些不同的事情分成了自己的重点命令。

的双重用途可以在文档checkout中的摘要描述中看到：

git-checkout - 切换分支或恢复工作树文件

添加命令的提交在switch其提交消息中解释了新命令的基本原理：

“git checkout”做太多事情是许多用户混淆的根源（有时甚至会咬老计时器）。为了解决这个问题，该命令将分为两个新命令：switch 和 restore。古老的“git checkout”命令仍然存在，并且会一直存在，直到所有（或大多数用户）都厌倦了它。

由此可见，引入新命令是为了减少混淆，因为它有两个重点命令，而不是一个多用途命令。

请注意，截至 2021 年 12 月，新命令仍列为实验性命令 ( switch, restore)：

此命令是实验性的。行为可能会改变。

  • 用于在切换前创建新分支的-b/选项重命名为/ 。它们也有长选项变体 ( / )，而以前它们只有单字母选项版本。-B``-c``-C``--create``--force-create
  • --detach(or -d) 现在在切换到分离头时总是需要的，以前它对于提交是可选的，但对于分支是必需的。

更多建议

git checkout有点像瑞士军刀，因为它有几个不相关的用途。

如果您修改了一个文件但没有暂存更改，则将git checkout <filename>撤消修改……这是一种取消对文件更改的快速简便的方法。你留在同一个分支。

git checkout <branchname>（如您所述）切换分支。

两个完全不同的目的，如果文件名和分支名称相似，可能会导致混淆。

将它作为两个命令更清楚。
