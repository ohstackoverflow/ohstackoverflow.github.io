如何在 GitHub 上的 README.md 文件中将图像居中？

Arya

2023-02-18 07:29

我一直在研究 GitHub 中使用的 Markdown 语法，但除了将图像调整为页面宽度外README.md，我不知道如何将图像居中。

这可能吗？如果是这样，我该怎么做？



解答

我一直在查看 GitHub [...] 中使用的 Markdown 语法，我不知道如何将图像居中

长话短说

不，你不能仅仅 依靠 Markdown 语法。Markdown 不关心定位。

注意： 一些 Markdown 处理器支持包含 HTML（正如@waldyr.ar 正确指出的那样），在 GitHub 的情况下，您可以回退到类似<div style="text-align:center"><img src="..." /></div>.

请注意，如果您的存储库在不同的托管环境（ CodePlex、Bitbucket等）中分叉，或者如果文档不是通过浏览器读取（Sublime Text Markdown 预览、MarkdownPad、Visual Studio Web Essentials Markdown 预览，...）。

注 2： 请记住，即使在 GitHub 网站内，Markdown 的呈现方式也不统一。例如，wiki 不允许此类 CSS 位置欺骗。

未删节版

Markdown 语法 不提供控制图像位置的能力。

事实上，如“ 哲学”部分所述，允许这种格式设置将违背 Markdown 哲学。

“一个 Markdown 格式的文档应该可以按原样发布，作为纯文本，而不像是用标签或格式说明标记的。”

Markdown 文件由 github.com 网站通过使用 Ruby Redcarpet 库呈现。

Redcarpet 公开了一些不属于标准 Markdown 语法的 扩展 （例如删除线），并提供了额外的“功能”。但是，没有支持的扩展允许您将图像居中。
