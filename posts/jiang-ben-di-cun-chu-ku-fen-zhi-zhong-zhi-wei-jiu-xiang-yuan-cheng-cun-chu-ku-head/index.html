将本地存储库分支重置为就像远程存储库 HEAD

Arya

2023-02-16 18:42

如何将我的本地分支重置为就像远程存储库上的分支一样？

我试过：

git reset --hard HEAD

但git status声称我修改了文件：

On branch master
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)
      modified:   java/com/mycompany/TestContacts.java
      modified:   java/com/mycompany/TestParser.java



解答

将您的分支设置为与远程分支完全匹配可以分两步完成：

git fetch origin
git reset --hard origin/master

如果您想在执行此操作之前保存当前分支的状态（以防万一），您可以执行以下操作：

git commit -a -m "Saving my work, just in case"
git branch my-saved-work

现在，您的工作已保存在分支“my-saved-work”中，以防您决定将其取回（或想稍后查看或将其与更新的分支进行比较）。

请注意，第一个示例假定远程仓库的名称为“origin”，并且远程仓库中名为“master”的分支与本地仓库中当前签出的分支相匹配。

顺便说一句，您所处的这种情况看起来非常像一个常见的情况，在这种情况下，已经对非裸存储库的当前签出分支进行了推送。你最近有没有进入你的本地回购？如果没有，那么不用担心——一定是其他原因导致这些文件意外地被修改。否则，您应该知道不建议推送到非裸存储库（尤其是不要推送到当前签出的分支）。


更多建议

我需要做的（接受的答案中的解决方案）：

git fetch origin
git reset --hard origin/master

其次是：

git clean -f

删除本地文件

要查看将删除哪些文件（而不实际删除它们）：

git clean -n -f
