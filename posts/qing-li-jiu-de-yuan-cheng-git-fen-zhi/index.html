清理旧的远程 git 分支

Arya

2023-02-17 10:34

我在两台不同的计算机（A 和 B）上工作，并在 dropbox 目录中存储一个通用的 git remote。

假设我有两个分支，master 和 develop。两者都在跟踪他们的远程对应方 origin/master 和 origin/devel。

现在在计算机 A 上，我删除了本地和远程的分支开发。

git push origin :heads/devel
git branch -d devel

在计算机 A 上运行git branch -a，我得到以下分支列表。

  • 掌握
  • 起源/头部
  • 起源/主人

在计算机 B 上运行git fetch，我可以使用 删除本地 devel 分支git branch -d devel，但我无法删除远程 devel 分支。

git push origin :heads/devel返回以下错误消息。

error: unable to push to unqualified destination: heads/proxy3d
目标 refspec 既不匹配远程上的现有 ref 也不以 refs/ 开头，我们无法根据源 ref 猜测前缀。
fatal: 远端意外挂断

git branch -a仍然在远程分支中列出 origin/devel。

如何清理计算机 B 的远程分支？



解答

git branch -a首先，在机器 B 上的结果是什么？

第二，你已经删除heads/devel了origin，所以你不能从机器B上删除它。

尝试

git branch -r -d origin/devel

或者

git remote prune origin

或者

git fetch origin --prune

并随意添加--dry-run到语句的末尾git以查看运行结果而不实际运行它。

git remote prune和的文档git branch。


更多建议

考虑运行：

git fetch --prune

定期在每个 repo 中删除本地分支，这些分支一直在跟踪已删除的远程分支（不再存在于远程 GIT repo 中）。

这可以进一步简化为

git config remote.origin.prune true

这是一个per-repo将使任何未来git fetch or git pull自动 修剪的 设置。

要为您的用户设置它，您还可以编辑全局 .gitconfig 并添加

[fetch]
    prune = true

但是，建议使用以下命令完成此操作：

git config --global fetch.prune true

或在系统范围内应用它（不仅仅是为用户）

git config --system fetch.prune true
