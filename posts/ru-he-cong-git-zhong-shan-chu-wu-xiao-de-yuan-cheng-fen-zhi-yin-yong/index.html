如何从 Git 中删除无效的远程分支引用？

Arya

2023-02-17 12:12

在我当前的回购中，我有以下输出：

$ git branch -a
* master
  remotes/origin/master
  remotes/public/master

remotes/public/master我想从分支列表中删除：

$ git branch -d remotes/public/master
error: branch 'remotes/public/master' not found.

此外，输出git remote很奇怪，因为它没有列出public：

$ git remote show 
origin

如何从分支列表中删除“remotes/public/master”？

更新，尝试了git push命令：

$ git push public :master
fatal: 'public' does not appear to be a git repository
fatal: The remote end hung up unexpectedly



解答

您可能需要清理：

git gc --prune=now

或者你可能需要修剪：

git remote prune public

修剪

删除 下的所有陈旧跟踪分支。这些陈旧的分支已经从 引用的远程存储库中删除，但仍然在“remotes/”中本地可用。

使用 --dry-run 选项，报告将修剪哪些分支，但实际上不修剪它们。

但是，看来这些应该早点用

git remote rm public

R M

删除名为 的远程。删除远程的所有远程跟踪分支和配置设置。

所以可能是您手动编辑了您的配置文件，但没有发生，或者您有权限问题。

也许再次运行它，看看会发生什么。

建议语境

如果您查看修订日志，您会注意到我建议了更多“正确”的技术，这些技术出于某种原因不想在他们的存储库上工作。

我怀疑 OP 做了一些让他们的树处于不一致状态的事情，导致它的行为有点奇怪，并且git gc需要修复遗留的 cruft。

通常 git branch -rd origin/badbranch足以对本地跟踪分支进行核对，或git push origin :badbranch对远程分支进行核对， 通常 你 永远 不需要调用git gc


更多建议

您需要做的就是

git fetch -p

它将删除所有远程删除的本地分支。

如果你使用的是 git 1.8.5+，你可以自动设置

git config fetch.prune true

或者

git config --global fetch.prune true
