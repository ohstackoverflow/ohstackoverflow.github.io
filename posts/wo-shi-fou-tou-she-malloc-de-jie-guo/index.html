我是否投射 malloc 的结果？

Arya

2023-02-16 20:42

在这个问题中，有人在评论中建议我 _ 不_ 应该将malloc. 即，我应该这样做：

int *sieve = malloc(sizeof(*sieve) * length);

而不是：

int *sieve = (int *) malloc(sizeof(*sieve) * length);

为什么会这样？



解答

长话短说

int *sieve = (int *) malloc(sizeof(int) * length);

有两个问题。演员和你使用类型而不是变量作为 sizeof 的参数。相反，这样做：

int *sieve = malloc(sizeof *sieve * length);

长版

没有 ；你 不 投结果，因为：

  • 这是不必要的，因为void *在这种情况下会自动且安全地提升为任何其他指针类型。
  • 它增加了代码的混乱，转换不是很容易阅读（特别是如果指针类型很长）。
  • 它让你重复自己，这通常是不好的。
  • 如果您忘记包含，它可以隐藏错误<stdlib.h>。这可能会导致崩溃（或者，更糟的是，直到稍后在代码的某些完全不同的部分 才会导致崩溃）。 考虑一下如果指针和整数的大小不同会发生什么；那么您将通过转换隐藏警告，并且可能会丢失您返回的地址的一部分。注意：从 C99 开始，隐式函数从 C 中消失了，这一点不再相关，因为没有自动假设未声明的函数返回int。

作为澄清，请注意我说的是“你不施法”，而不是“你不需要 施法 ”。在我看来，即使你做对了，也没有包括演员表。这样做根本没有好处，但是有一堆潜在的风险，包括演员表表明您不知道这些风险。

另请注意，正如评论员指出的那样，上面讨论的是直接 C，而不是 C++。我非常坚信 C 和 C++ 是独立的语言。

int要进一步补充，您的代码不必要地重复可能导致错误的类型信息 ( )。最好取消引用用于存储返回值的指针，以将两者“锁定”在一起：

int *sieve = malloc(length * sizeof *sieve);

这也将 the 移到length前面以增加可见性，并用 ; 删除多余的括号sizeof。只有当参数是类型名称时才 需要 它们。许多人似乎不知道（或忽略）这一点，这使得他们的代码更加冗长。记住：sizeof不是函数！:)

虽然在极少数情况下移动length到前面 可能会 增加可见性，但还应注意，在一般情况下，最好将表达式写为：

int *sieve = malloc(sizeof *sieve * length);

由于保留sizeof第一个，在这种情况下，确保乘法至少用数学完成size_t。

Compare: malloc(sizeof *sieve * length * width)vs. malloc(length * width * sizeof *sieve)second 可能会溢出length * widthwhenwidth并且length是比 更小的类型size_t。


更多建议

在 C 中，您不需要强制转换 的返回值malloc。返回的指向 void 的指针malloc会自动转换为正确的类型。但是，如果您希望您的代码使用 C++ 编译器进行编译，则需要强制转换。社区中的首选替代方案是使用以下内容：

int *sieve = malloc(sizeof *sieve * length);

如果您更改sieve.
