获取远程分支到本地 GIT

Arya

2023-02-16 20:16

远程存储库包含各种分支，例如origin/daves_branch：

$ git branch -r
origin/HEAD -> origin/master
origin/daves_branch
origin/master

我如何获取daves_branch到本地存储库以便它跟踪origin/daves_branch？

我试过：

$ git fetch origin discover
$ git checkout discover



解答

更新：使用 Git 开关

下面写的所有信息都是准确的，但是git switch添加了一个新命令来简化工作。

如果daves_branch存在于远程存储库中，但不存在于您的本地分支中，您只需键入：

git switch daves_branch

由于您在本地没有分支，这将自动查看switch远程仓库。然后它还会自动设置远程分支跟踪。

请注意，如果daves_branch本地不存在，您需要git fetch先在本地使用switch.

原帖

您需要创建一个跟踪远程分支的本地分支。以下命令将创建一个名为 daves_branch 的本地分支，跟踪远程分支 origin/daves_branch 。当您推送更改时，远程分支将被更新。

对于最新版本的 Git：

git checkout --track origin/daves_branch

--track是git checkout -b [branch] [remotename]/[branch][remotename] 在这种情况下的 起源 和 [branch] 两次相同的简写，在这种情况下是 daves_branch 。

对于 Git 1.5.6.5 你需要这个：

git checkout --track -b daves_branch origin/daves_branch

对于 Git 1.7.2.3 及更高版本，这就足够了（它可能更早开始，但这是我能很快找到的最早确认）：

git checkout daves_branch

请注意，在最近的 Git 版本中，此命令不会创建本地分支，并且会将您置于“分离的 HEAD”状态。如果您想要本地分支机构，请使用该--track选项。

完整的细节在这里： 3.5 Git 分支 - 远程分支，跟踪分支
