“静态只读”与“常量”

Arya

2023-02-17 01:33

我已经阅读了关于const和static readonly字段的信息。我们有一些类只包含常量值。它们用于我们系统中的各种事物。所以我想知道我的观察是否正确：

这些恒定值是否应该始终适用static readonly于所有公开的东西？并且只const用于internal//protected值private？

你有什么建议吗？我什至不应该使用static readonly字段，而是使用属性吗？



解答

public static readonly字段有点不寻常；public static属性（只有一个get）会更常见（可能由一个private static readonly字段支持）。

const值直接写入调用站点；这是双刃剑：

  • 如果值是在运行时获取的，可能是从配置中获取的，那么它是无用的
  • 如果更改 const 的值，则需要重建所有客户端
  • 但它可以更快，因为它避免了方法调用......
  • ...有时可能已经被 JIT 内联

如果该值 永远 不会改变，那么 const 就可以了 -Zero等等 合理的 consts ;p 除此之外，static属性更常见。


更多建议

static readonly如果 消费者 在不同的程序集中，我会使用。让 theconst和 Consumer在两个不同的程序集中是搬起 石头砸自己脚的好方法。
