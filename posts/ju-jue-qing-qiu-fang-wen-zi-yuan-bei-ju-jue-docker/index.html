我正在通过此链接创建我的第一个 docker Image，它成功运行，现在我正尝试从此链接将此 Image 推送到我的 docker 存储库中。但是每当我试图将这个图像推送到存储库时，我都会遇到这种类型的错误。

denied: requested access to the resource is denied

注意：我已经成功登录到docker



解答

您可能需要在 docker push 之前将 docker repo 切换为私有。

感谢吴院长的回答和ses的这个评论，推送之前记得先 退出 ，然后从命令行 登录你的docker hub账号 * *

# you may need log out first `docker logout` ref. https://stackoverflow.com/a/53835882/248616
docker login

根据文档：

You need to include the namespace for Docker Hub to associate it with your account.
The namespace is the same as your Docker Hub account name.
You need to rename the image to YOUR_DOCKERHUB_NAME/docker-whale.

所以，这意味着你必须在推送之前 标记 你的图像：

docker tag firstimage YOUR_DOCKERHUB_NAME/firstimage

然后你应该能够推动它。

docker push YOUR_DOCKERHUB_NAME/firstimage


更多建议

我有同样的问题，但这里给出的接受答案对我不起作用。我尝试了几步，终于能够绕过它来推动它。

以下是对我有用的步骤：

  1. 登录到码头工人。

码头工人登录-u sirimalla

  1. 标记您的图像构建

我这里的图像名称是： mylocalimage ，默认情况下它有标签： latest
，我的用户名是： sirimalla ，已在 docker cloud 注册，我创建了一个名为： dockerhub的公共存储库

所以我的个人存储库现在变成了： sirimalla/dockerhub ，我想用标签推送我的图像：myfirstimagepush

我标记如下：

docker tag mylocalimage:latest sirimalla/dockerhub:myfirstimagepush
  1. 将图像推送到我的个人 docker 存储库，如下所示

码头推 sirimalla/dockerhub:myfirstimagepush

它成功地推送到我的个人 docker 仓库。
