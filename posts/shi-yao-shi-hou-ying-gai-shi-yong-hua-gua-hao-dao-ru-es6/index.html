什么时候应该使用花括号导入 ES6？

Arya

2023-02-17 05:59

这似乎很明显，但我发现自己对何时使用大括号在 ES6 中导入单个模块感到有点困惑。例如，在我正在处理的 React-Native 项目中，我有以下文件及其内容：

文件 initialState.js 

var initialState = {
    todo: {
        todos: [
            {id: 1, task: 'Finish Coding', completed: false},
            {id: 2, task: 'Do Laundry', completed: false},
            {id: 2, task: 'Shopping Groceries', completed: false},
        ]
    }
};

export default initialState;

在 TodoReducer.js 中，我必须在不使用大括号的情况下导入它：

import initialState from './todoInitialState';

如果我将initialState大括号括起来，我会收到以下代码行的以下错误：

无法读取未定义的属性 todo

文件 TodoReducer.js ：

export default function todos(state = initialState.todo, action) {
    // ...
}

类似的错误也发生在我的带有花括号的组件上。我想知道什么时候我应该对单个导入使用花括号，因为很明显，当导入多个组件/模块时，你必须将它们括在花括号中，我知道这一点。

这里的 Stack Overflow 帖子没有回答我的问题，而是问我 什么时候 应该或不应该使用花括号导入 单个 模块，或者我永远不应该使用花括号导入 ES6 中的单个模块（这显然不是这种情况，正如我所看到的需要花括号的单一导入）。



解答

这是 默认导入 ：

// B.js
import A from './A'

A它仅在具有 默认导出 时才有效：

// A.js
export default 42

在这种情况下，导入时分配给它的名称无关紧要：

// B.js
import A from './A'
import MyA from './A'
import Something from './A'

因为它总是会 解析 为A.

这是一个 名为的命名导入A：

import { A } from './A'

它仅在A包含 名为的命名导出A时才有效：

export const A = 42

在这种情况下，名称很重要，因为您要 通过其导出名称导入特定的东西 ：

// B.js
import { A } from './A'
import { myA } from './A' // Doesn't work!
import { Something } from './A' // Doesn't work!

要使这些工作正常，您需要将 相应的命名导出 添加到A：

// A.js
export const A = 42
export const myA = 43
export const Something = 44

一个模块只能有 一个默认导出 ，但 可以有任意多个命名导出 （零个、一个、两个或多个）。您可以将它们全部导入：

// B.js
import A, { myA, Something } from './A'

在这里，我们将默认导出导入为A，并分别将导出命名为myA和Something。

// A.js
export default 42
export const myA = 43
export const Something = 44

我们还可以在导入时为它们分配不同的名称：

// B.js
import X, { myA as myX, Something as XSomething } from './A'

默认导出往往用于您通常希望从模块中获得的任何内容。命名导出往往用于可能方便但并非总是必需的实用程序。然而，您可以选择如何导出内容：例如，一个模块可能根本没有默认导出。

这是一个很好的 ES 模块指南，解释了默认导出和命名导出之间的区别。
