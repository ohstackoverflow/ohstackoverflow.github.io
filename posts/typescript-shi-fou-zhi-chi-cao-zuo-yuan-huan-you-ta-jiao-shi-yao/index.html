Typescript 是否支持 ?. 操作员？（还有，它叫什么？）

Arya

2023-02-18 02:23

Typescript 目前（或是否有计划）支持的安全导航运算符?.

IE：

var thing = foo?.bar
// same as:
var thing = (foo) ? foo.bar : null;

此外，此运算符是否有更常见的名称（很难用谷歌搜索）。



解答

是的。 从 TypeScript 3.7（2019 年 11 月 5 日发布）开始，支持此功能，称为 Optional Chaining ：

可选链的核心是让我们编写代码，如果遇到 or ，TypeScript 可以立即停止运行某些null表达式undefined。可选链中的明星是 可选属性访问?.的新运算符。 __

有关详细信息，请参阅TypeScript 3.7 发行说明。

在 3.7 版之前，TypeScript 不支持此功能，尽管早在 TypeScript 存储库的 Issue #16（可追溯到 2014 年）就提出了要求。

至于如何 称呼这个 operator ，似乎没有达成共识。除了“可选链”（这也是它在 JavaScript和Swift中的称呼）之外，还有几个其他示例：

  • CoffeeScript 将其称为 存在运算符 （具体来说，是存在运算符的“访问器变体”）：

存在运算符的访问器变体?.可用于吸收属性链中的空引用。.在基值可能为 null 或 undefined 的 情况下，使用它代替点访问器。

  • C# 将此称为 空条件运算符 。

空条件运算符仅在该操作数的计算结果为非空时才对其操作数应用成员访问?.或元素访问操作；?[]否则，它返回null。

  • Kotlin 将其称为 安全调用运算符 。

可能还有很多其他例子。


更多建议

现在可以了，请参阅用户“Donut”的回答。

旧答案：关于布尔运算符的标准 JavaScript 行为可能有所帮助。布尔方法在比较对象时不返回 true 或 false，但在 OR 的情况下返回等于 true 的第一个值。

不如单个 ?，但它有效：

var thing = foo && foo.bar || null;

您可以根据需要使用任意数量的 && ：

var thing = foo && foo.bar && foo.bar.check && foo.bar.check.x || null;

默认值也是可能的：

var name = person && person.name || "Unknown user";
