`self` 参数的目的是什么？为什么需要它？

Arya

2023-02-17 03:30

考虑这个例子：

class MyClass:
    def func(self, name):
        self.name = name

我知道那是指self的具体实例MyClass。但是为什么必须func显式包含self为参数呢？为什么我们需要self在方法的代码中使用？其他一些语言将此隐式化，或使用特殊语法代替。

解答

您需要使用的原因self.是因为 Python 不使用特殊语法来引用实例属性。Python 决定以一种使方法所属的实例自动 传递 但不自动 接收 的方式来处理方法：方法的第一个参数是调用该方法的实例。这使得方法与函数完全相同，并让您使用实际名称（尽管这self是惯例，当您使用其他名称时人们通常会对您皱眉。）self代码并不特殊，它只是另一个对象.

Python 可以做一些其他的事情来区分普通名称和属性——像 Ruby 那样的特殊语法，或者像 C++ 和 Java 那样需要声明，或者可能更不同的东西——但它没有。Python 就是为了让事情变得明确，让什么是什么变得显而易见，虽然它并不是在所有地方都这样做，但它确实为实例属性做到了。这就是为什么分配给一个实例属性需要知道要分配给什么实例，这就是为什么它需要self..


更多建议

假设您有一个类ClassA，其中包含定义为的方法methodA：

def methodA(self, arg1, arg2):
    # do something

并且objectA是这个类的一个实例。

现在 whenobjectA.methodA(arg1, arg2)被调用时，python 在内部为您将其转换为：

ClassA.methodA(objectA, arg1, arg2)

变量self指的是对象本身。
