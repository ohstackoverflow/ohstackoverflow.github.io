接口和抽象类有什么区别？

Arya

2023-02-16 23:19

接口和抽象类之间到底有什么区别？



解答

接口

接口是一种 契约 ：编写接口的人说，“ 嘿，我接受这样的东西 ”，而使用接口的人说“ 好的，我写的类看起来那样 ”。

接口是一个空壳 。只有方法的签名，这意味着方法没有主体。接口什么也做不了。这只是一个模式。

例如（伪代码）：

// I say all motor vehicles should look like this:
interface MotorVehicle
{
    void run();

    int getFuel();
}

// My team mate complies and writes vehicle looking that way
class Car implements MotorVehicle
{

    int fuel;

    void run()
    {
        print("Wrroooooooom");
    }


    int getFuel()
    {
        return this.fuel;
    }
}

实现一个接口消耗的 CPU 很少，因为它不是一个类，只是一堆名称，因此不需要进行任何昂贵的查找。它在重要的时候很棒，例如在嵌入式设备中。

抽象类

与接口不同，抽象类是类。它们的使用成本更高，因为当您从它们继承时需要进行查找。

抽象类看起来很像接口，但它们还有更多东西：您可以为它们定义行为。它更多的是关于一个人说，“ 这些类应该看起来像那样，并且它们有共同点，所以请填空！ ”。

例如：

// I say all motor vehicles should look like this:
abstract class MotorVehicle
{

    int fuel;

    // They ALL have fuel, so lets implement this for everybody.
    int getFuel()
    {
         return this.fuel;
    }

    // That can be very different, force them to provide their
    // own implementation.
    abstract void run();
}

// My teammate complies and writes vehicle looking that way
class Car extends MotorVehicle
{
    void run()
    {
        print("Wrroooooooom");
    }
}

执行

虽然抽象类和接口应该是不同的概念，但实现有时会使该声明不正确。有时，它们甚至不是您认为的那样。

在 Java 中，这条规则被严格执行，而在 PHP 中，接口是没有声明方法的抽象类。

在 Python 中，抽象类更像是一种编程技巧，您可以从 ABC 模块中获得它，并且实际上使用的是元类，因此也就是类。接口与这种语言中的鸭子类型更相关，它是约定和调用描述符的特殊方法（ method 方法）之间的混合体。

与编程一样，有理论、实践和另一种语言的实践:-)


更多建议

抽象类和接口之间的主要技术差异是：

  • 抽象类可以有 常量、成员、方法存根（没有主体的方法）和定义的方法 ，而接口只能有 常量 和 方法存根 。

  • 抽象类的方法和成员可以定义为 任何可见性 ，而接口的所有方法都必须定义为public（默认情况下它们被定义为公共的）。

  • 继承抽象类时， 具体的 子类 必须定义抽象方法 ，而抽象类可以扩展另一个抽象类，而不必定义父类的抽象方法。

  • 同样，扩展另一个接口的接口不 负责实现 父接口的方法。这是因为接口不能定义任何实现。

  • 一个子类只能 扩展一个类 （抽象的或具体的），而一个接口可以扩展或一个类可以 实现多个其他接口 。

  • 子类可以定义具有 相同或更少限制可见性的抽象 方法，而实现接口的类必须定义具有完全相同可见性（公共）的方法。
