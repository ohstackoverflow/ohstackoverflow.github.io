什么是 Visual Studio Code 中的“工作区”？

Arya

2023-02-17 11:31

例如，Visual Studio Code 讨论了在用户级别与工作空间级别应用设置。

一方面，

  • 它可以引用您打开的项目目录；或者
  • 它可以指您在特定窗口中打开的所有内容。

上面引用的页面说

“工作区：这些设置存储在您工作区内的 .vscode 文件夹中，仅在工作区打开时应用。”



解答

什么是工作空间？

一个项目，包含一个或多个根文件夹，以及属于该项目的所有 Visual Studio Code 配置。这些配置包括：

  • 打开该项目时应应用的设置
  • 项目的推荐扩展（与同事共享配置文件时很有用）
  • 项目特定的调试配置

为什么工作空间如此混乱？

Visual Studio Code 并没有在整个 UI 中一致地使用该术语（我已经打开了一个GitHub 问题来解决这个问题）。有时它指的是如上所述的工作区，而其他时候它指的是工作区作为专门与文件关联的项目.code-workspace。

一个很好的例子是最近的文件小部件。请注意，在链接的屏幕截图中，所有项目都分组在相同的“工作区”标题下，这表明那里的一切都有一个工作区。但是随后带有.code- workspace文件的项目被赋予“工作空间”后缀，与标题相矛盾并表明只有那些文件才是真正的工作空间。

什么是.code-workspace文件？

它是一个带有注释的 JSON 文件，除了存储属于工作区的所有根文件夹的位置之外，还存储上述所有配置数据。

我需要一个.code-workspace文件吗？

仅当您正在创建多根工作区.code- workspace时，在这种情况下，除了要在资源管理器中显示的所有根文件夹之外，您将拥有一个自动恢复所有工作区设置的文件。

单文件夹项目呢？

一切都是自动化的。

当您在 Visual Studio Code 中打开一个文件夹并开始对与您当前正在处理的项目特别相关的编辑器进行修改时，Visual Studio Code 会自动创建一个文件夹并将其存储在您创建的项目文件夹的根目录中.vscode。正在努力。此.vscode文件夹包含存储您所做更改的文件。

例如，如果您更改了只想应用于当前项目的Visual Studio Code 设置settings.json，Visual Studio Code 会创建一个包含这些更新的文件，并将该文件存储在该.vscode文件夹中。

.code- workspace如果确实需要，您可以创建一个只包含一个根文件夹的文件。然后您可以直接打开项目文件夹，或打开工作区文件。但我想不出任何这会有益的理由。

如何创建.code-workspace文件？

转到菜单 文件 → 将工作区另存为...

如何将根文件夹添加到工作区？

转到菜单 文件 → 将文件夹添加到工作区...。

如何打开由.code-workspace文件定义的工作空间？

转到菜单 文件 → 打开工作区...。

或者，双击该.code-workspace文件。Visual Studio Code 不会打开实际文件。相反，它将读取该文件并打开属于该工作区的文件夹。

如何查看实际.code-workspace文件？

转到菜单 文件 → 打开... ，然后选择目标.code-workspace文件。

或者，打开与该文件关联的工作区。然后打开命令面板，搜索并选择 工作区：打开工作区配置文件 命令。


更多建议

您可以在工作区级别保存设置，并且可以在一个工作区中打开多个文件夹。如果您想执行其中任何一项操作，请使用工作区，否则，只需打开一个文件夹即可。

Visual Studio Code 工作区是项目文件夹和文件的列表。一个工作区可以包含多个文件夹。您可以自定义工作区的设置和首选项。
