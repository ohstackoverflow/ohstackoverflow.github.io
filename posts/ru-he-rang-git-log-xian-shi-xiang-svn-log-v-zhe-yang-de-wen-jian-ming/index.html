如何让'git log'显示像'svn log -v'这样的文件名

Arya

2023-02-17 04:39

SVN 的日志有一个“-v”模式，输出每次提交中更改的文件的文件名，如下所示：

jes5199$ svn 日志-v
---------------------------------------------- ----------------------
r1 | jes5199 | 2007-01-03 14:39:41 -0800（2007 年 1 月 3 日，星期三）| 1 行
更改路径：
   A /作者
   A /正在复制
   A /更改日志
   A /进化
   /安装
   /MacOSX

有没有一种快速的方法来获取 Git 中每次提交中更改文件的列表？



解答

对于更改文件的完整路径名：

git log --name-only

对于已更改文件的完整路径名和状态：

git log --name-status

对于缩写的路径名和更改文件的 diffstat：

git log --stat

还有很多选择。查看文档。


更多建议

注意： 已弃用，请 改用 git whatchanged``git log

鼓励新用户改用git-log[1]。该命令与git-log[1]whatchanged基本相同，但默认显示原始格式差异输出并跳过合并。

该命令主要出于历史原因而保留；许多在很久以前git log就通过阅读 Linux 内核邮件列表学习 Git 的人的手指被训练来打字。

您可以使用该命令git whatchanged --stat获取每次提交中更改的文件列表（以及提交消息）。
