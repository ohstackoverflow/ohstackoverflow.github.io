如何在 package.json 中指定所需的 Node.js 版本？

Arya

2023-02-17 23:04

我有一个需要 Node 版本 12 或更高版本的 Node.js 项目。有没有办法在packages.json文件中指定这一点，以便安装程序自动检查并通知用户是否需要升级？



解答

engines您可以在您的字段中设置字段，并为其中一个版本或两个版本package.json设置要求：node``npm

  "engines" : { 
    "npm" : ">=8.0.0 <9.0.0",
    "node" : ">=16.0.0 <17.0.0"
  }

要通过 npm 执行此操作，您需要创建一个.npmrc文件（并将其提交到存储库）并将选项设置engines- strict为，如果所需的引擎版本不匹配，true这将导致 npm 命令失败：npm install

# .npmrc
engine-strict=true

如果没有该文件，每个开发人员都需要npm config set engine-strict true在他们的本地工作区中运行以打开此选项。

原始答案

正如您所说的，您的代码肯定不适用于任何较低版本，您可能也需要“engineStrict”标志：

{ "engineStrict" : true }

package.json 文件的文档可以在 npmjs 网站上找到

更新

engineStrict现已弃用，因此只会发出警告。现在取决于用户npm config set engine-strict true是否需要运行。

更新 2

正如 ben 在下面指出的那样，如果 Node 版本不兼容，.npmrc使用文本在项目的根目录（与 package.json 文件相同的级别）创建一个文件将在安装过程中强制出错。engine-strict=true


更多建议

将以下内容添加到package.json：

  "engines": {
    "npm": ">=8.0.0 <9.0.0",
    "node": ">=16.0.0 <17.0.0"
  },

将以下内容添加到.npmrc（与 相同的目录package.json）：

engine-strict=true
