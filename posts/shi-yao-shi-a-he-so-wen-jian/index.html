什么是 .a 和 .so 文件？

Arya

2023-02-18 06:33

我目前正在尝试将 C 应用程序移植到AIX，但感到很困惑。什么是 .a 和 .so 文件以及在构建/运行应用程序时如何使用它们？



解答

存档库 (.a) 是静态链接的，即当您在 gcc 中使用 -c 选项编译程序时。因此，如果库中有任何更改，您需要重新编译和构建您的代码。

.so（共享对象）相对于 .a 库的优势在于它们在运行时即在 gcc 中创建 .o 文件后链接 -o 选项。所以，如果 .so 文件有任何变化，你不需要重新编译你的主程序。但是请确保您的主程序已使用 ln 命令链接到新的 .so 文件。

这将帮助您构建 .so 文件。http://www.yolinux.com/TUTORIALS/LibraryArchives- StaticAndDynamic.html


更多建议

.a 是静态库。如果您使用存储在它们内部的代码，它会从它们中取出并嵌入到您自己的二进制文件中。在 Visual Studio 中，这些将是 .lib 文件。

.so 是动态库。如果您使用存储在其中的代码，它不会被获取并嵌入到您自己的二进制文件中。相反，它只是被引用，因此二进制文件将依赖于它们，并且在运行时添加/加载 so 文件中的代码。在 Visual Studio/Windows 中，这些将是 .dll 文件（带有包含链接信息的小 .lib 文件）。
