如何检查 Bash 脚本中的程序是否存在？

Arya

2023-02-16 20:20

我将如何以返回错误并退出或继续执行脚本的方式验证程序是否存在？

看起来应该很容易，但它一直难倒我。



解答

回答

POSIX 兼容：

command -v <the_command>

使用示例：

if ! command -v <the_command> &> /dev/null
then
    echo "<the_command> could not be found"
    exit
fi

对于 Bash 特定环境：

hash <the_command> # For regular commands. Or...
type <the_command> # To check built-ins and keywords

解释

避免which。它不仅是一个你启动的外部进程，因为它做的很少（意味着内置函数hash，type或者command更便宜），你还可以依靠内置函数来实际做你想做的事情，而外部命令的效果很容易从系统到系统。

为什么要关心？

  • 许多操作系统都有一个which甚至 没有设置退出状态的 a ，这意味着它if which foo甚至不会在那里工作并且 总是 报告foo存在，即使它不存在（请注意一些 POSIX shell 似乎也这样做hash）。
  • 许多操作系统会做一些which自定义的和邪恶的事情，比如改变输出，甚至挂钩到包管理器中。

所以，不要使用which. 而是使用其中之一：

command -v foo >/dev/null 2>&1 || { echo >&2 "I require foo but it's not installed.  Aborting."; exit 1; }



type foo >/dev/null 2>&1 || { echo >&2 "I require foo but it's not installed.  Aborting."; exit 1; }



hash foo 2>/dev/null || { echo >&2 "I require foo but it's not installed.  Aborting."; exit 1; }

（次要注意事项：有些人会建议2>&-是相同的2>/dev/null但更短 - 这是不正确的 。2>&-关闭 FD 2 会在程序尝试写入 stderr 时导致程序 出错 ，这与成功写入它并丢弃输出有很大不同（而且很危险！））

如果你的 hash bang 是/bin/sh那么你应该关心 POSIX 说的是什么。typeandhash的退出代码没有被 POSIX 很好地定义，并且hash当命令不存在时被认为成功退出（还没有看到这个type）。command的退出状态由 POSIX 明确定义，因此使用起来可能是最安全的。

但是，如果您的脚本使用了POSIX 规则bash，那么 POSIX 规则就不再重要了，而且type使用hash起来非常安全。type现在只有 a-P可以搜索PATH并且hash有副作用，命令的位置将被散列（以便下次使用它时更快地查找），这通常是一件好事，因为您可能检查它是否存在以便实际使用它.

作为一个简单的例子，这里有一个函数，gdate如果存在则运行，否则date：

gnudate() {
    if hash gdate 2>/dev/null; then
        gdate "$@"
    else
        date "$@"
    fi
}

具有完整功能集的替代方案

您可以使用scripts-common来满足您的需要。

要检查是否安装了某些东西，您可以执行以下操作：

checkBin <the_command> || errorMessage "This tool requires <the_command>. Install it please, and then run this tool again."


更多建议

以下是检查命令是否存在$PATH 以及 是否可执行的可移植方法：

[ -x "$(command -v foo)" ]

例子：

if ! [ -x "$(command -v git)" ]; then
  echo 'Error: git is not installed.' >&2
  exit 1
fi

需要进行可执行检查，因为如果在$PATH.

另请注意，如果与可执行文件同名的不可执行文件较早存在于 中$PATH，破折号将返回前者，即使后者将被执行。这是一个错误，违反了 POSIX 标准。[错误报告] [标准]
编辑 ：这似乎从 dash 0.5.11 (Debian 11) 开始修复。

此外，如果您正在寻找的命令已被定义为别名，这将失败。
