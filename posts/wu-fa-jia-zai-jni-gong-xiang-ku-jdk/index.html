无法加载 JNI 共享库 (JDK)

Arya

2023-02-17 08:30

当我尝试打开Eclipse时，弹出对话框显示：

无法加载 JNI 共享库“C:/JDK/bin/client/jvm.dll”`。

在此之后，Eclipse 强制关闭。

以下是我想说明的几点：

  • 我检查了那条路径是否存在任何东西。它确实存在。
  • 我的 Eclipse 和Java SE 开发工具包都是 64 位的。我检查了我的系统，它可以处理 64 位。
  • 我在 Google 和 Stack Overflow 上搜索过这个问题，我找到的唯一答案是下载 32 位版本的 JDK 和 Eclipse。

下载 32 位版本是我唯一想做的最后一招。
有什么建议可以解决这个问题？



解答

你需要64-bit三重奏：

  • 64-bit 操作系统
  • 64-bit 爪哇
  • 64-bit 蚀


更多建议

OS、JDK 和 Eclipse 的工作配对：

  • 32-bit 操作系统 | 32-bit JDK | 32-bit Eclipse （ 仅限 32 位 ）
  • 64-bit 操作系统 | 32-bit JDK | 32-bit 蚀
  • 64-bit 操作系统 | 64-bit JDK| 64bit Eclipse （ 仅限 64 位 ）

我安装了几个 JDK 和 JRE。

他们每个人在 PATH 变量中都有自己的条目，或多或少都在工作。

从 PATH 变量来看，一些安装完全没有用，因为它们从未被使用过。当然，如果需要，可以从 Eclipse 中手动引用“非活动”Java，但我从来没有这样做过，所以我真的不需要它们。（至少我当时是这么认为的……）

我清理了混乱，卸载了所有当前的 Java，只安装了 JDK + JRE 1.7 64 位。

Eclipse“安装”之一随后失败，并给出Failed to Load the JNI shared Library了相对于新安装的 JDK 的给定路径，它认为是jvm.dll。

失败的 Eclipse 是我所有 IDE 中唯一一个在我的全 64 位设置上仍然是 32 位版本的。

添加 VM 参数，就像经常提到的那样，eclipse.ini在我的案例中没有用（因为我只有错误的 JDK/JRE 与之相关。）

我也无法找到如何检查这个 Eclipse 是 32 位还是 64 位版本（我无法在任务管理器中查找它，因为这个Eclipse “安装”不会启动。而且因为它有自从我设置它以来已经有一段时间了，我也记不起它的版本了。）

如果您使用较新的 JDK 和较旧的JRE，您也可能会遇到麻烦，但它更有可能java.lang.UnsupportedClassVersionError出现IIRC。
