如何取消设置 JavaScript 变量？

Arya

2023-02-17 19:20

我在 JavaScript 中有一个全局变量（实际上是一个window属性，但我认为这不重要），它已经被以前的脚本填充，但我不希望稍后运行的另一个脚本看到它的值或者它是甚至定义。

我已经把some_var = undefined它用于测试目的typeof some_var == "undefined"，但我真的不认为这是正确的方法。

你怎么认为？



解答

运算delete符从对象中删除属性。它不能删除变量。所以问题的答案取决于全局变量或属性是如何定义的。

(1) 如果是用 创建的var，则不能删除。

例如：

var g_a = 1; //create with var, g_a is a variable
delete g_a; //return false
console.log(g_a); //g_a is still 1

(2) 如果没有创建var，可以删除。

g_b = 1; //create without var, g_b is a property
delete g_b; //return true
console.log(g_b); //error, g_b is not defined

技术说明

1.使用var

在这种情况下，引用g_a是在 ECMAScript 规范称为附加到当前范围的“ VariableEnvironment ”中创建的 - 在var函数内部使用的情况下，这可能是函数执行上下文（尽管它可能会变得有点复杂当您考虑let) 或在“全局”代码的情况下， VariableEnvironment 附加到全局对象（通常window）。

VariableEnvironment 中的引用通常是不可删除的 - ECMAScript 10.5中详述的过程对此进行了详细解释，但足以说明除非您的代码在上下文中执行eval（大多数基于浏览器的开发控制台使用），否则声明的变量var不能被删除。

2. 不使用var

当尝试在不使用关键字的情况下为名称分配值时，JavaScript 会尝试在 ECMAScript 规范称为“ LexicalEnvironmentvar ”的地方找到命名引用，主要区别在于 LexicalEnvironment 是嵌套的 - 即 LexicalEnvironment 有一个父（ ECMAScript 规范称之为“外部环境引用”），当 JavaScript 无法在 LexicalEnvironment 中找到引用时，它会在父 LexicalEnvironment 中查找（详见10.3.1和10.2.2.1）。顶级 LexicalEnvironment 是“ 全局环境 __"，并且它绑定到全局对象，因为它的引用是全局对象的属性。因此，如果您尝试访问未在var当前作用域或任何外部作用域中使用关键字声明的名称，JavaScript 最终将获取一个属性用作该引用的对象window。正如我们之前所了解的，可以删除对象的属性。

笔记

  1. 重要的是要记住var声明是“提升的” - 即它们总是被认为发生在它们所在范围的开头 - 尽管不是可能在语句中完成的值初始化 - 它留在原var处. 所以在下面的代码中，是来自 VariableEnvironmenta的引用而不是属性，它的值将在代码的末尾： __window``10

    function test() { a = 5; var a = 10; }

  2. 上面的讨论是在没有启用“严格模式”的情况下。使用“严格模式”时查找规则有点不同，在没有“严格模式”的情况下解析到窗口属性的词法引用将在“严格模式”下引发“未声明的变量”错误。我真的不明白这是在哪里指定的，但它是浏览器的行为方式。


更多建议

scunliffe 的答案会奏效，但从技术上讲应该是

delete window.some_var;

当目标不是对象属性时， delete 应该是空操作。例如，

(function() {
   var foo = 123;
   delete foo; // wont do anything, foo is still 123
   var bar = { foo: 123 };
   delete bar.foo; // foo is gone
}());

但是由于全局变量实际上是 window 对象的成员，所以它起作用了。

当涉及原型链时，使用 delete 变得更加复杂，因为它只从目标对象中删除属性，而不是原型。例如，

function Foo() {}
Foo.prototype = { bar: 123 };
var foo = new Foo();
// foo.bar is 123
foo.bar = 456;
// foo.bar is now 456
delete foo.bar;
// foo.bar is 123 again.

所以要小心。

注意：我的回答有些不准确（见最后的“误解”）。该链接解释了所有血淋淋的细节，但总结是浏览器之间可能存在很大差异，具体取决于您要删除的对象。delete object.someProp一般只要是应该是安全的object !== window。var尽管在适当的情况下可以，但我仍然不会使用它来删除声明的变量。
