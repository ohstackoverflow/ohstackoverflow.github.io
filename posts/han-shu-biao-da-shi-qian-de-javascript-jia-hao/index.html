函数表达式前的 JavaScript 加号

Arya

2023-02-17 09:54

我一直在寻找有关立即调用的函数的信息，并且在某个地方偶然发现了这种表示法：

+function(){console.log("Something.")}()

有人可以向我解释+函数前面的符号是什么意思吗？



解答

它强制解析器将后面的部分视为+表达式。这通常用于立即调用的函数，例如：

+function() { console.log("Foo!"); }();

如果没有+there，如果解析器处于等待语句（可以是表达式或多个非表达式语句）的状态，则该词看起来像是函数 声明function的开头而不是函数 表达式 ，因此紧随其后（上一行末尾的那些）将是语法错误（在该示例中，名称的缺失也是如此）。 使用 ，它使它成为一个函数表达式，这意味着名称是可选的，并且会导致对可以调用的函数的引用，因此括号是有效的。 () __+

+只是选项之一。它也可以是-, !, ~, 或任何其他一元运算符。或者，您可以使用括号（这更常见，但在语法上不正确）：

(function() { console.log("Foo!"); })();
// or
(function() { console.log("Foo!"); }());


更多建议

@TJCrowder 的回答的附属项，+通常用于强制对值进行数值转换，如此 SO 回答所解释的那样。在这种情况下，它被称为“一元加运算符”（为了便于谷歌搜索）。

var num = +variant;

所以在函数前面，它可以是一种强制将函数的结果解释为数字的方法。我怀疑它是否会发生，但理论上 JIT 可以使用它来将函数编译为纯数值函数等。但是，为了防止一元加号在更大的表达式中使用时成为串联，您需要括号：

blah + (+(function(){ var scope; return "4"; })());
