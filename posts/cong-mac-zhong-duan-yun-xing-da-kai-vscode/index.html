从 Mac 终端运行/打开 VSCode

Arya

2023-02-17 17:21

我想通过运行此命令从 Mac OSX 终端运行/打开 Visual Studio Code code .。我在这里找到了说明：

https://code.visualstudio.com/Docs/setup

显然我需要将其包含在我的.bashrc文件中，所以我这样做了，但无济于事。

code () {
    if [[ $# = 0 ]]
    then
        open -a "Visual Studio Code"
    else
        [[ $1 = /* ]] && F="$1" || F="$PWD/${1#./}"
        open -a "Visual Studio Code" --args "$F"
    fi
}

我.bashrc在这里编辑了文件：

~/.bashrc指向/Users/username/.bashrc

.bashrc我应该编辑哪个？



解答

根据有关从命令行启动的文档：

  1. 打开 Visual Studio 代码
  2. Command使用+ Shift+ P（或F1）打开命令面板
  3. Shell在命令面板中输入
  4. Shell Command: Install code in PATH从建议列表中选择

就是这样。

现在打开你的终端类型。

$ code .

在 MacOS 上重新启动后使此更改持续存在

许多 Mac 用户发现这已被遗忘，需要在任何重启后重新应用。如果 MacOS 已将隔离属性应用于 VS 代码，则可能会发生这种情况，操作系统将其用于“你确定吗？” 通知适用于首次使用从互联网下载的应用程序。

要检查是否应用了此属性，请在此命令返回的列表中查找com.apple.quarantine（如果不是您安装的路径，请更改路径）：

xattr "/Applications/Visual Studio Code.app"

如果确实返回com.apple.quarantine，您可以使用带有标志的相同命令删除该属性-d（以及-r递归地从所有包含的文件中删除它并sudo允许更改）：

sudo xattr -r -d com.apple.quarantine "/Applications/Visual Studio Code.app"

...然后Shell Command : Install code in PATH在删除该属性后执行上述操作，并且它应该在重新启动后仍然存在。

更多建议

我只想从他的评论中提取 Benjamin Pasero 的答案，因为这似乎是最好的解决方案。这是Setting up Visual Studio Code 页面上给出的提示...

如果您想从终端运行 VS Code，请将以下内容附加到您的 ~/.bash_profile 文件（如果您使用 zsh，则为 ~/.zshrc）。

code () { VSCODE_CWD="$PWD" open -n -b "com.microsoft.VSCode" --args $* ;}

现在，您只需输入code .任何文件夹即可开始编辑该文件夹中的文件。[或者code test.txt去处理文件test.txt]
