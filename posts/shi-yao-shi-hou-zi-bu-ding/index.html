什么是猴子补丁？

Arya

2023-02-17 13:05

我想了解，什么是猴子补丁或猴子补丁？

这是方法/运算符重载或委托之类的东西吗？

它与这些东西有什么共同之处吗？



解答

不，它不像那些东西。它只是在运行时动态替换属性。

例如，考虑一个具有方法的类get_data。此方法执行外部查找（例如，在数据库或 Web API 上），类中的各种其他方法调用它。但是，在单元测试中，您不想依赖外部数据源 - 因此您动态地将方法替换为get_data返回一些固定数据的存根。

因为 Python 类是可变的，而方法只是类的属性，所以您可以随心所欲地执行此操作 - 事实上，您甚至可以用完全相同的方式替换模块中的类和函数。

但是，正如一位评论者指出的那样，在进行 monkeypatching 时要小心：

  1. 如果除了你的测试逻辑之外还有其他任何东西调用get_data，它也会调用你的猴子补丁替换而不是原来的——这可能是好也可能是坏。小心点。

  2. get_data如果某些变量或属性在您替换它时也指向该函数，则此别名不会改变其含义并将继续指向原始get_data. （为什么？Python 只是将类中的名称重新绑定get_data到其他一些函数对象；其他名称绑定根本不受影响。）


更多建议

MonkeyPatch 是一段 Python 代码，它在运行时（通常是在启动时）扩展或修改其他代码。

一个简单的示例如下所示：

from SomeOtherProduct.SomeModule import SomeClass

def speak(self):
    return "ook ook eee eee eee!"

SomeClass.speak = speak
