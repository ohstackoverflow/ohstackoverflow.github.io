LINQ聚合算法详解

Arya

2023-02-17 12:16

这听起来可能很蹩脚，但我一直没能找到一个很好的解释Aggregate。

好的意味着简短、描述性、全面以及一个小而清晰的例子。



解答

最容易理解的定义Aggregate是它对列表的每个元素执行一个操作，同时考虑到之前进行的操作。也就是说，它对第一个和第二个元素执行操作并将结果向前推进。然后它对之前的结果和第三个元素进行操作并继续前进。ETC。

示例 1. 对数字求和

var nums = new[]{1,2,3,4};
var sum = nums.Aggregate( (a,b) => a + b);
Console.WriteLine(sum); // output: 10 (1+2+3+4)

这将添加1并2制作3. 然后将3（前一个的结果）和3（顺序的下一个元素）添加到 make 6。然后添加6和4制作10。

示例 2. 从字符串数组创建一个 csv

var chars = new []{"a","b","c","d"};
var csv = chars.Aggregate( (a,b) => a + ',' + b);
Console.WriteLine(csv); // Output a,b,c,d

这以几乎相同的方式工作。连接a一个逗号b并使a,b. 然后a,b用逗号连接c并使a,b,c. 等等。

示例 3. 使用种子将数字相乘

为了完整起见，有一个重载需要Aggregate一个种子值。

var multipliers = new []{10,20,30,40};
var multiplied = multipliers.Aggregate(5, (a,b) => a * b);
Console.WriteLine(multiplied); //Output 1200000 ((((5*10)*20)*30)*40)

与上面的示例非常相似，这从值开始5，然后将其乘以序列的第一个元素，10得到结果50。该结果被结转并乘以序列中的下一个数字，20得到 的结果1000。这将继续通过序列的其余 2 个元素。

附录

上面的示例 2 使用字符串连接创建以逗号分隔的值列表。这是一种简单的方式来解释使用哪个Aggregate是这个答案的目的。但是，如果使用此技术实际创建大量以逗号分隔的数据，则使用 更合适，StringBuilder这与Aggregate使用种子重载启动StringBuilder.

var chars = new []{"a","b","c", "d"};
var csv = chars.Aggregate(new StringBuilder(), (a,b) => {
    if(a.Length>0)
        a.Append(",");
    a.Append(b);
    return a;
});
Console.WriteLine(csv);

更多建议

这在一定程度上取决于您所谈论的是哪种重载，但基本思想是：

  • 从种子作为“当前值”开始
  • 遍历序列。对于序列中的每个值：
    • 应用用户指定的函数转换(currentValue, sequenceValue)为(nextValue)
    • 放currentValue = nextValue
  • 返回决赛currentValue

您可能会发现Aggregate我的 Edulinq 系列中的帖子很有用 - 它包括更详细的描述（包括各种重载）和实现。

一个简单的例子是使用Aggregate作为替代Count：

// 0 is the seed, and for each item, we effectively increment the current value.
// In this case we can ignore "item" itself.
int count = sequence.Aggregate(0, (current, item) => current + 1);

或者可能将字符串序列中的所有字符串长度相加：

int total = sequence.Aggregate(0, (current, item) => current + item.Length);

就我个人而言，我 很少 发现Aggregate有用 - “量身定制”的聚合方法通常对我来说已经足够好了。
