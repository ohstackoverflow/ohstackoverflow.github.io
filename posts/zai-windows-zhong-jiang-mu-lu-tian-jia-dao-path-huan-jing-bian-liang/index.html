在 Windows 中将目录添加到 PATH 环境变量

Arya

2023-02-17 18:17

我想在 Windows 中添加C:\xampp\php到我的系统环境变量。PATH

我已经使用 “环境变量” 对话框添加了它。

但是当我输入我的控制台时：

C:\>path

它不显示新C:\xampp\php目录：

PATH=D:\Program Files\Autodesk\Maya2008\bin;C:\Ruby192\bin;C:\WINDOWS\system32;C:\WINDOWS;

我有两个问题：

  1. 为什么会这样？我做错了什么吗？
  2. 另外，如何PATH使用控制台（以及以编程方式使用批处理文件）将目录添加到我的变量中？



解答

这只会修改注册表。现有流程不会使用这些值。如果新进程在此更改 后启动 并且 不从其父级继承旧环境，则它会这样做。

您没有指定如何启动控制台会话。确保这一点的最佳方法是退出命令 shell 并再次运行它。然后它应该继承更新的 PATH 环境变量。


更多建议

警告： 此解决方案 可能会破坏 您的 PATH 和系统的稳定性。作为副作用，它将合并您的用户和系统 PATH，并将 PATH 截断为 1024 个字符。此命令的效果是不可逆的。首先备份 PATH。请参阅评论以获取更多信息。

不要盲目复制粘贴。谨慎使用。

您可以使用以下命令 永久 添加路径：PATH``setx

setx /M path "%path%;C:\your\path\here\"

/M如果要设置用户PATH而不是系统，请删除该标志PATH。

笔记：

  • 该setx命令仅在 Windows 7 及更高版本中可用。

  • 您应该从提升的命令提示符运行此命令。

  • 如果您只想为当前会话更改它，请使用set。
