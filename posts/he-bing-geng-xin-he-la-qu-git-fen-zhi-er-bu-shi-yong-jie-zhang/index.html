合并、更新和拉取 Git 分支而不使用结帐

Arya

2023-02-17 12:38

我在一个有 2 个分支 A 和 B 的项目上工作。我通常在分支 A 上工作，然后合并分支 B 中的内容。对于合并，我通常会这样做：

git merge origin/branchB

但是，我还想保留分支 B 的本地副本，因为我可能偶尔会在不先与我的分支 A 合并的情况下检查分支。为此，我会这样做：

git checkout branchB
git pull
git checkout branchA

有没有一种方法可以在一个命令中执行上述操作，而不必来回切换分支？我应该git update-ref为此使用吗？如何？



解答

简短的回答

只要你正在做一个 快进 合并，那么你可以简单地使用

git fetch <remote> <sourceBranch>:<destinationBranch>

例子：

# Merge local branch foo into local branch master,
# without having to checkout master first.
# Here `.` means to use the local repository as the "remote":
git fetch . foo:master

# Merge remote branch origin/foo into local branch foo,
# without having to checkout foo first:
git fetch origin foo:foo

虽然Amber 的答案也适用于快进情况，但git fetch以这种方式使用比仅强制移动分支引用更安全一些，因为只要git fetch您不在+参考规范。

长答案

如果不先签出 A，则不能将分支 B 合并到分支 A，否则会导致非快进合并。这是因为需要一个工作副本来解决任何潜在的冲突。

然而， 在快进合并的情况下，这是可能的 ，因为根据定义，这样的合并永远不会导致冲突。要在不首先签出分支的情况下执行此操作，您可以使用git fetchrefspec。

下面是一个更新示例master（不允许非快进更改），如果您feature检出了另一个分支：

git fetch upstream master:master

这个用例很常见，你可能想在你的 git 配置文件中为它创建一个别名，就像这样：

[alias]
    sync = !sh -c 'git checkout --quiet HEAD; git fetch upstream master:master; git checkout --quiet -'

这个别名的作用如下：

  1. git checkout HEAD: 这会将您的工作副本置于分离头状态。master如果您想在碰巧签出它时进行更新，这将很有用。我认为这是必要的，因为否则分支引用将master不会移动，但我不记得那是否真的是我的头脑。

  2. git fetch upstream master:master: 这会将您的本地快进到master与upstream/master.

  3. git checkout -检查您之前检查过的分支（在这种情况下就是这样-做的）。

git fetchfor（非）快进合并的语法

如果您希望fetch命令在更新为非快进时失败，那么您只需使用以下形式的 refspec

git fetch <remote> <remoteBranch>:<localBranch>

如果你想允许非快进更新，那么你+在 refspec 的前面添加一个：

git fetch <remote> +<remoteBranch>:<localBranch>

请注意，您可以使用以下方法将本地存储库作为“远程”参数传递.：

git fetch . <sourceBranch>:<destinationBranch>

文档

从git fetch解释此语法的文档中（强调我的）：

<refspec>

参数的格式<refspec>是一个可选的加号+，后跟源 ref <src>，然后是冒号:，然后是目标 ref <dst>。

获取匹配的远程引用<src>，如果<dst>不是空字符串，则匹配它的本地引用使用<src>. 如果使用可选的 plus+，即使不会导致快进更新，也会更新本地 ref。

更多建议

不，那里没有。必须检出目标分支才能解决冲突等问题（如果 Git 无法自动合并它们）。

但是，如果合并是快进的，则不需要签出目标分支，因为您实际上不需要合并任何东西——您所要做的就是更新分支以指向新头参考。你可以这样做git branch -f：

git branch -f branch-b branch-a

将更新branch-b指向 的头部branch-a。

选项-f代表--force，表示branch-b将被覆盖。

警告 ：使用git fetch一个更安全的选项，它只允许 快进 。

这个方法可以像这样使用：

git branch -f branch-b branch-b@{Upstream}

或更短

git branch -f branch-b branch-b@{U}

强制更新一个分支，而不检查它（比如它们在 rebase 后是否有分歧）
