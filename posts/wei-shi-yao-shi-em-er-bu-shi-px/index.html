为什么是 em 而不是 px？

Arya

2023-02-17 11:45

em我听说您应该在样式表中定义大小和距离而不是像素。那么问题来了，在CSS中定义样式的时候为什么要用eminstead而不是呢？px有一个很好的例子可以说明这一点吗？



解答

我问这个问题的原因是我忘记了如何使用 em，因为那段时间我正在愉快地使用 CSS 进行黑客攻击。人们没有注意到我将问题笼统地保留了下来，因为我并不是在谈论调整字体本身的大小。我更感兴趣的是如何在页面上的 任何给定块元素上定义样式。

正如Henrik Paul和其他人指出的那样，em 与元素中使用的字体大小成正比。以 px 为单位定义块元素的大小是一种常见的做法，但是，在浏览器中调整字体大小通常会破坏这种设计。调整字体大小通常使用快捷键Ctrl++或Ctrl+来完成-。因此，一个好的做法是改用 em。

使用 px 定义宽度

这是一个说明性的例子。假设我们有一个 div 标签，我们想把它变成一个时尚的日期框，我们可能有如下所示的 HTML 代码：

<div class="date-box">
    <p class="month">July</p>
    <p class="day">4</p>
</div>

一个简单的实现将以 px 为单位定义类的宽度date-box：

* { margin: 0; padding: 0; }

p.month { font-size: 10pt; }

p.day { font-size: 24pt; font-weight: bold; }

div.date-box {
    background-color: #DD2222;
    font-family: Arial, sans-serif;
    color: white;
    width: 50px;
}

问题

然而，如果我们想在我们的浏览器中放大文本的大小，这个设计就会被破坏。文本也会溢出框外，这几乎与Flodin指出的 SO 设计所发生的情况相同。这是因为框在锁定到 时将保持相同的宽度大小50px。

改用 em

更聪明的方法是以 em 为单位定义宽度：

div.date-box {
    background-color: #DD2222;
    font-family: Arial, sans-serif;
    color: white;
    width: 2.5em;
}

* { margin: 0; padding: 0; font-size: 10pt; }

// Initial width of date-box = 10 pt x 2.5 em = 25 pt
// Will also work if you used px instead of pt

这样您就可以在日期框上进行流畅的设计，即框的大小将与文本一起按为日期框定义的字体大小的比例放大。在此示例中，字体大小定义为*10pt，并将放大到该字体大小的 2.5 倍。因此，当您在浏览器中调整字体大小时，该框的大小将是该字体大小的 2.5 倍。


更多建议

我有一台高分辨率的小型笔记本电脑，必须以 120% 的文本缩放比例运行 Firefox，才能在不眯着眼睛的情况下阅读。

许多网站都存在这方面的问题。布局变得乱码，按钮中的文本被切成两半或完全消失。甚至 stackoverflow.com 也深受其害：

请注意顶部按钮和页面选项卡是如何重叠的。如果他们使用 em 单位而不是 px，就不会有问题。
