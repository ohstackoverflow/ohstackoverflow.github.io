全局 git 配置数据存储在哪里？

Arya

2023-02-17 04:09

使用git config --global设置时，它将写入哪个文件？

例子：

git config --global core.editor "blah"

我在这些地方找不到它：

C:\Program Files\Git\etc\gitconfig

C:\myapp\.git\config

我没有设置ENV？

我的 Git 版本：1.6.5.1.1367.gcd48 – 在 Windows 7 上



解答

2016 年更新：使用 git 2.8（2016 年 3 月），您可以简单地使用：

git config --list --show-origin

使用 Git 2.26（2020 年第一季度），您可以添加一个--show-scope选项

git config --list --show-origin --show-scope

您将看到在何处设置了哪个配置。
请参阅“我的 Git 配置中的设置来自哪里？ ”

正如Stevoisiak在评论中指出的那样，

它适用于非标准安装位置。（即Git 便携版）

（比如最新的PortableGit-2.14.2-64-bit.7z.exe，可以解压到任何地方）

原始答案 (2010)

从文档：

- 全球的

对于写入选项：写入全局~/.gitconfig文件而不是存储库.git/config。

由于您使用的是Git for Windows，因此可能不清楚这对应于哪个位置。但是如果你查看 etc/profile (in C:\Program Files\Git)，你会看到：

HOME="$HOMEDRIVE$HOMEPATH"

意义：

C:\Users\MyLogin

（在 Windows 7 上）

这意味着该文件适用于 C:\Users\MyLogin\.gitconfig Windows 7 中的 Git。


更多建议

.gitconfig我也在我的 Windows 机器上寻找全局并使用 git 找到了这个巧妙的技巧。

执行 a:git config --global -e然后，如果幸运的话，您将得到一个加载了全局.gitconfig文件的文本编辑器。只需从那里查找文件夹（或尝试另存为...），等等！:-)
