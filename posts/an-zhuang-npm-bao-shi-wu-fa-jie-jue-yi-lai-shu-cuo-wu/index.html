安装 npm 包时无法解决依赖树错误

Arya

2023-02-17 21:53

尝试使用命令安装 npm 包时npm i，出现以下异常：

我尝试重新安装 Node.js 包并使用以下方法将代理设置为关闭：

set HTTP_PROXY=
set HTTPS_PROXY=

问题仍然存在。我做错了什么？

更新：

当我运行以下命令时：

npm install --legacy-peer-deps

显示以下错误：



解答

这与 HTTP 代理无关。

正如它所说，您有 依赖冲突 （ 不正确 且可能 破坏的--force依赖），因此请尝试使用, 或运行命令--legacy-peer- deps。如果不生效， 临时解决办法 是 使用之前版本的Node.js （降级Node.js版本），有时会出现这种错误。

基于OP更新的更新 ：

如您所见，它会引发以下错误：

找不到与@angular/http@^9.1.4 匹配的版本。

看看angular/http页面。 请注意 ，该已弃用软件包的最新版本是在 您请求更高版本（例如， ）7.2.16时！因此，请尝试检查项目依赖项并跟踪引发的错误以解决问题。 ^9.1.4


更多建议

试试这个命令-

npm install --save --legacy-peer-deps
