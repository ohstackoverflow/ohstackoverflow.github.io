错误：'node-sass' 版本 5.0.0 与 ^4.0.0 不兼容

Arya

2023-02-17 20:50

我使用以下命令创建了一个空白的 React 项目：npx create-react-appon npm v7.0.7 and Node.js v15.0.1

安装：

  • 反应 v17.0.1，
  • 节点 sass v5.0.0，

然后我尝试将一个空白的 .scss 文件导入 App 组件：

function App() {

解答

长话短说

  1. npm uninstall node-sass
  2. npm install sass

或者，如果使用纱线

  1. yarn remove node-sass
  2. yarn add sass

Edit3 ：是的，另一个编辑。迁移到 sass (dart-sass) 是最好的解决方案。上一个包括将 node-sass 锁定到版本 4.xx，该版本已有 2 年历史并且缺少更新的 SCSS 功能。

Edit2 ：sass-loader v10.0.5修复了它。问题是您可能没有将它用作项目依赖项，而是更多地用作依赖项的依赖项。CRA 使用固定版本，angular-cli 锁定到 node-sass v4，等等。

目前的建议是：如果您只安装 node-sass，请检查以下解决方法（和注释）。如果你正在处理一个空白项目并且你可以管理你的Webpack配置（不使用 CRA 或 CLI 来构建你的项目），请安装最新的 sass-loader。

编辑 ：此错误来自sass-loader。由于 node-sass @latest 是 v5.0.0 而 sass-loader 期望 ^4.0.0，因此存在语义版本控制不匹配。

他们的存储库中有一个未解决的问题以及需要审查的相关修复。在此之前，请参阅下面的解决方案。

解决方法 ：暂时不要安装 node-sass 5.0.0（主要版本刚刚升级）。

卸载节点sass

npm uninstall node-sass

然后安装最新版本（5.0之前）

npm install node-sass@4.14.1

注意：LibSass（因此也包括 node-sass）已被弃用，dart-sass 是推荐的实现。您可以改用sass，它是编译为纯 JavaScript 的 dart-sass 的 Node.js 发行版。


更多建议

出现此类错误的唯一原因是您的 Node.js 版本与您的 node-sass 版本不兼容。

因此，请务必查看node-sass 上的文档。

或者下图将帮助您决定哪个 Node.js 版本可以使用 node-sass 版本。

例如， 如果您在 Windows 系统上使用 Node.js 版本 12 （“可能”），那么您应该安装 node-sass 版本 4.12 。

npm install node-sass@4.12

是的，就像那样。所以现在你只需要安装node-sass团队推荐的node-sass版本，你的电脑上安装的是Node.js版本。
