检测设备是否为 iOS

Arya

2023-02-18 05:35

我想知道是否有可能检测浏览器是否在 iOS 上运行，类似于使用 Modernizr 进行功能检测的方式（尽管这显然是设备检测而不是功能检测）。

通常我更喜欢功能检测，但我需要找出设备是否是 iOS，因为他们按照这个问题处理视频的方式YouTube API not working with iPad / iPhone / non-Flash device



解答

检测 iOS

在iOS 13 iPad 中，用户代理和平台字符串都发生了变化，iPad 和 MacOS 之间的区别似乎是可能的，因此下面的所有答案现在都需要考虑到这一点。

这可能是最短的替代方案，也涵盖了 iOS 13：

function iOS() {
  return [
    'iPad Simulator',
    'iPhone Simulator',
    'iPod Simulator',
    'iPad',
    'iPhone',
    'iPod'
  ].includes(navigator.platform)
  // iPad on iOS 13 detection
  || (navigator.userAgent.includes("Mac") && "ontouchend" in document)
}

iOS将是true或false

更糟糕的选择：用户代理嗅探

User Agent 嗅探比较危险，经常会出现问题。

在 iPad iOS 13 上，用户代理与 MacOS 13 计算机的用户代理相同 ，但如果您忽略 iPad，这可能会持续一段时间：

var iOS = !window.MSStream && /iPad|iPhone|iPod/.test(navigator.userAgent); // fails on iPad iOS 13

这!window.MSStream是为了不错误地检测 IE11，请参见此处和此处。

注意： 和navigator.userAgent都navigator.platform可以由用户或浏览器扩展伪造。

存在更改 userAgent 或平台的浏览器扩展，因为网站使用过于严厉的检测并且经常禁用某些功能，即使用户的浏览器可以使用该功能也是如此。

为了缓和这种与用户的冲突，建议针对每种情况专门检测您的网站所需的确切功能。然后，当用户获得具有所需功能的浏览器时，它无需额外更改代码即可正常工作。

检测 iOS 版本

检测 iOS 版本的最常见方法是从 User Agent string 中解析它。但也有 特征 检测 推理*；

我们知道在 iOS4 - iOS5 - iOS6 - iOS7 等中history API引入的事实 。 matchMedia API webAudio API WebSpeech API

注意： 以下代码不可靠，如果这些 HTML5 功能中的任何一个在较新的 iOS 版本中被弃用，将会中断。你被警告了！

function iOSversion() {

  if (iOS) { // <-- Use the one here above
    if (window.indexedDB) { return 'iOS 8 and up'; }
    if (window.SpeechSynthesisUtterance) { return 'iOS 7'; }
    if (window.webkitAudioContext) { return 'iOS 6'; }
    if (window.matchMedia) { return 'iOS 5'; }
    if (window.history && 'pushState' in window.history) { return 'iOS 4'; }
    return 'iOS 3 or earlier';
  }

  return 'Not an iOS device';
}


更多建议

在 iOS 13 之后，你应该像这样检测 iOS 设备，因为 iPad 不会被旧方法检测为 iOS 设备（由于新的“桌面”选项，默认启用）：

let isIOS = /iPad|iPhone|iPod/.test(navigator.platform)
|| (navigator.platform === 'MacIntel' && navigator.maxTouchPoints > 1)

第一个条件是 iOS < 13 或禁用桌面模式的 iPhone 或 iPad，第二个条件是默认配置的 iPadOS 13，因为它定位自己像 Macintosh Intel，但实际上是唯一具有多点触控功能的 Macintosh。

与其说是真正的解决方案，不如说是黑客攻击，但对我来说工作可靠

PS 如前所述，您可能应该添加 IE 检查

let isIOS = (/iPad|iPhone|iPod/.test(navigator.platform) ||
(navigator.platform === 'MacIntel' && navigator.maxTouchPoints > 1)) &&
!window.MSStream
