“git commit”和“git push”有什么区别？

Arya

2023-02-17 07:57

在我正在阅读的 Git 教程中，git commit用于存储您所做的更改。

git push那有什么用呢？



解答

基本上，git commit“ 记录对存储库的更改 ”，同时git push“ 更新远程引用以及关联的对象 ”。所以第一个用于与本地存储库相关联，而后一个用于与远程存储库交互。

这是一张来自Oliver Steele的漂亮图片，它解释了 Git 模型和命令：

阅读有关git push推 和拉git pull（我首先提到的文章）的更多信息。 __


更多建议

commit ：将更改添加到本地存储库

push ：将最后一次提交传输到远程服务器
