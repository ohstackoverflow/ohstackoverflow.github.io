自动接受所有 SDK 许可

Arya

2023-02-18 00:52

从 gradle android 插件2.2-alpha4开始：

Gradle 将尝试下载项目所依赖的缺失的 SDK 包

这是非常酷的，并且被认为是JakeWharton 的项目。

但是，要下载 SDK 库，您需要：接受许可协议或 gradle 告诉您：

您尚未接受以下 SDK 组件的许可协议：[Android SDK Build-Tools 24, Android SDK Platform 24]。在构建项目之前，您需要接受许可协议并使用 Android Studio SDK 管理器完成缺少组件的安装。或者，要了解如何将许可协议从一个工作站转移到另一个工作站，请访问http://d.android.com/r/studio- ui/export-licenses.html

这是一个问题，因为我很乐意在执行以下操作时安装所有 sdk 依赖项gradle build.

我正在寻找一种自动接受所有许可证的解决方案。也许是 gradle 脚本？你有什么想法 ？



解答

更新 2021 这应该是公认的答案，因为它简单且最新

AndroidSDK 终于可以接受许可了。

去Android\sdk\tools\bin

yes | sdkmanager --licenses

编辑：

正如@MoOx 在评论中指出的那样，在 macOS 上，你可以这样做

yes | sudo ~/Library/Android/sdk/tools/bin/sdkmanager --licenses

正如@pho、@mikebridge 和@Noitidart 在 Windows 上的评论中指出的那样，您可以

cmd.exe /C"%ANDROID_HOME%\tools\bin\sdkmanager.bat --licenses"

一定要先安装java


更多建议

我收到此错误：

License for package Android SDK Build-Tools 30.0.2 not accepted.

所以，我去了工具 -> SDK 管理器 -> SDK 工具。然后我安装了 Google Play 许可库。它解决了这个问题。
