抓取request.GET中的URL参数

Arya

2023-02-18 00:46

我目前正在定义正则表达式以捕获 URL 中的参数，如教程中所述。如何从 URL 访问参数作为HttpRequest对象的一部分？

我HttpRequest.GET目前返回一个空QueryDict对象。

我想学习如何在没有图书馆的情况下做到这一点，这样我就可以更好地了解 Django。



解答

当 URL 类似于 时domain/search/?q=haha，您将使用request.GET.get('q', '').

q是您想要的参数，如果找不到则为''默认值。q

但是，如果您只是配置您的URLconf**，那么您从中捕获的内容regex将作为参数（或命名参数）传递给函数。

例如：

(r'^user/(?P<username>\w{0,50})/$', views.profile_page,),

那么在你的views.py你会有

def profile_page(request, username):
    # Rest of the method


更多建议

为了澄清camflan的解释，我们假设您有

  • 规则url(regex=r'^user/(?P<username>\w{1,50})/$', view='views.profile_page')
  • 传入请求http://domain/user/thaiyoshi/?message=Hi

URL 调度程序规则将捕获部分 URL 路径 （此处"user/thaiyoshi/"）并将它们与请求对象一起传递给视图函数。

解析查询字符串（此处message=Hi）并将参数存储为QueryDictin request.GET。不会对 HTTP GET 参数进行进一步的匹配或处理。

这个视图函数将使用从 URL 路径中提取的两个部分和一个查询参数：

def profile_page(request, username=None):
    user = User.objects.get(username=username)
    message = request.GET.get('message')

作为旁注，您将在 中找到请求方法（在本例中"GET"，通常对于提交的表单"POST"）request.method。在某些情况下，检查它是否符合您的预期很有用。

更新： 在决定是使用 URL 路径还是查询参数来传递信息时，以下内容可能会有所帮助：

  • 使用 URL 路径来唯一标识资源，例如/blog/post/15/（不是/blog/posts/?id=15）
  • 使用查询参数来更改资源的显示方式，例如/blog/post/15/?show_comments=1或/blog/posts/2008/?sort_by=date&direction=desc
  • 要制作人性化的 URL，请避免使用 ID 号并使用例如日期、类别和/或 slug：/blog/post/2008/09/30/django-urls/
