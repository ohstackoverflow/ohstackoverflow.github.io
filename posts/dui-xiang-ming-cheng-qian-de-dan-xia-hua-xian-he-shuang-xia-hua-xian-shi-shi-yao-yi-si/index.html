对象名称前的单下划线和双下划线是什么意思？

Arya

2023-02-17 00:29

对象名称前的单下划线和双前导下划线在 Python 中代表什么？



解答

单下划线

在一个类中，带有前导下划线的名称向其他程序员表明该属性或方法旨在在该类内部使用。但是，不会以任何方式 强制执行隐私。 在模块中对函数使用前导下划线表示不应从其他地方导入它。

来自PEP-8风格指南：

_single_leading_underscore：弱的“内部使用”指标。例如，from M import *不导入名称以下划线开头的对象。

双下划线（名称修改）

来自Python 文档：

形式的任何标识符__spam（至少两个前导下划线，最多一个尾随下划线）在文本上替换为_classname__spam，其中classname是当前类名，前导下划线被去除。这种重组是在不考虑标识符的句法位置的情况下完成的，因此它可用于定义类私有实例和类变量、方法、存储在全局变量中的变量，甚至存储在实例中的变量。在其他类的实例上对此类私有。

以及来自同一页面的警告：

名称重整旨在为类提供一种简单的方法来定义“私有”实例变量和方法，而不必担心派生类定义的实例变量，或类外代码对实例变量的处理。请注意，改组规则主要是为了避免事故而设计的； 一个坚定的人仍然有可能访问或修改一个被认为是私有的变量。

例子

>>> class MyClass():
...     def __init__(self):
...             self.__superprivate = "Hello"
...             self._semiprivate = ", world!"
...
>>> mc = MyClass()
>>> print mc.__superprivate
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: myClass instance has no attribute '__superprivate'
>>> print mc._semiprivate
, world!
>>> print mc.__dict__
{'_MyClass__superprivate': 'Hello', '_semiprivate': ', world!'}


更多建议

  • _foo: 只是约定俗成。程序员指示变量是私有的（无论这在 Python 中意味着什么）的一种方式。

  • __foo: 这有实际意义。解释器将此名称替换为_classname__foo以确保该名称不会与另一个类中的相似名称重叠。

  • __foo__: 只是约定俗成。Python 系统使用不会与用户名冲突的名称的一种方式。

在 Python 世界中，没有其他形式的下划线有意义。此外，在这些约定中，类、变量、全局等之间没有区别。
