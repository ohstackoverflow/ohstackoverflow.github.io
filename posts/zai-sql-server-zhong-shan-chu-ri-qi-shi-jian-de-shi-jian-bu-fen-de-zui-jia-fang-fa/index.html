在 SQL Server 中删除日期时间的时间部分的最佳方法

Arya

2023-02-18 01:41

从 SQL Server 中的日期时间字段中删除时间部分时，哪种方法提供最佳性能？

a) select DATEADD(dd, DATEDIFF(dd, 0, getdate()), 0)

或者

b) select cast(convert(char(11), getdate(), 113) as datetime)

第二种方法确实会以任何一种方式发送更多字节，但这可能不如转换速度重要。

两者看起来也都很快，但是在处理数十万行或更多行时，速度可能会有所不同吗？

此外，是否有可能有更好的方法来摆脱 SQL 中日期时间的时间部分？



解答

严格来说，方法a是资源占用最少的：

a) select DATEADD(dd, DATEDIFF(dd, 0, getdate()), 0)

事实证明，在相同总持续时间的情况下，CPU 密集度较低，一百万行，手上有太多时间的人：SQL Server 中从日期+时间获取日期的最有效方法？

我在其他地方也看到了类似的测试，结果也类似。

我更喜欢 DATEADD/DATEDIFF 因为：

  • varchar 受语言/日期格式问题的影响
    示例：为什么我的 CASE 表达式是不确定的？

  • float 依赖内部存储

  • 它通过更改“0”基数扩展到计算出月份的第一天、明天等

编辑，2011 年 10 月

对于 SQL Server 2008+，你可以 CAST 到dateie CAST(getdate() AS date)。或者只使用date数据类型，所以不time删除。

编辑，2012 年 1 月

这是多么灵活的一个工作示例：需要在 sql server 中按四舍五入的时间或日期数字计算

编辑，2012 年 5 月

不要在没有考虑的情况下在 WHERE 子句等中使用它：向列添加函数或 CAST 会使索引使用无效。请参阅此处的第 2 个常见 SQL 编程错误

现在，这确实有一个更新的 SQL Server 优化器版本的示例，该版本正确地管理 CAST，但 通常 这不是一个好主意......

编辑，2018 年 9 月，对于 datetime2

DECLARE @datetime2value datetime2 = '02180912 11:45' --this is deliberately within datetime2, year 0218
DECLARE @datetime2epoch datetime2 = '19000101'

select DATEADD(dd, DATEDIFF(dd, @datetime2epoch, @datetime2value), @datetime2epoch)


更多建议

在 SQL Server 2008 中，您可以使用：

CONVERT(DATE, getdate(), 101)
