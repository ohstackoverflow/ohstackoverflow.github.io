计算方法的执行时间

Arya

2023-02-17 18:50

可能重复：
如何测量功能运行的时间？

我有一个 I/O 计时方法，可以将数据从一个位置复制到另一个位置。计算执行时间的最佳和最真实的方法是什么？Thread？Timer？Stopwatch？还有其他解决方案吗？我想要最准确的，并且尽可能简短。



解答

Stopwatch专为此目的而设计，是衡量 .NET 中执行时间的最佳方法之一。

var watch = System.Diagnostics.Stopwatch.StartNew();
// the code that you want to measure comes here
watch.Stop();
var elapsedMs = watch.ElapsedMilliseconds;

不要 使用 DateTime来测量 .NET 中的执行时间。

更多建议

根据个人经验，System.Diagnostics.Stopwatch可以使用类来衡量方法的执行时间，但是， 请注意 ：它并不完全准确！

考虑以下示例：

Stopwatch sw;

for(int index = 0; index < 10; index++)
{
    sw = Stopwatch.StartNew();
    DoSomething();
    Console.WriteLine(sw.ElapsedMilliseconds);
}

sw.Stop();

现在你想知道； “嗯，为什么第一次用了 132 毫秒，而其他时间明显更少？”

答案是它Stopwatch不会补偿 .NET 中的“背景噪音”活动，例如 JITing。因此，当您第一次运行您的方法时，.NET JIT 首先运行。执行此操作所需的时间将添加到执行时间中。同样，其他因素也会导致执行时间发生变化。

您真正应该寻找的绝对准确性是 性能分析 ！
