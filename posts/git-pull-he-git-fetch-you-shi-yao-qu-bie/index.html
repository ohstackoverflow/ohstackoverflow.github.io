git pull和之间有什么区别git fetch？



解答

用最简单的术语来说，git pull执行 agit fetch后跟 a git merge。

git fetch在 下更新您的远程跟踪分支refs/remotes/<remote>/。此操作随时可以安全运行，因为它永远不会更改您在refs/heads.

git pull使本地分支与其远程版本保持最新，同时还更新您的其他远程跟踪分支。

来自 Git 文档git pull：

git pull使用给定的参数运行git fetch，然后根据配置选项或命令行标志，将调用 orgit rebase来git merge协调不同的分支。


更多建议

  • git pull尝试在获取提交后自动合并。 它是上下文相关的 ，因此所有拉取的提交都将合并到您当前活动的分支中。git pull自动合并提交 而不让您先检查它们 。如果您不小心管理您的分支机构，您可能会经常遇到冲突。

  • git fetch从目标分支收集当前分支中不存在的任何提交，并将 它们存储在本地存储库中 。但是， 它不会将它们与您当前的分支合并 。如果您需要使您的存储库保持最新，但正在处理一些如果您更新文件可能会破坏的内容，这将特别有用。要将提交集成到您当前的分支中，您必须git merge在之后使用。
