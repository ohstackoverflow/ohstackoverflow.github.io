什么是 C 中的“静态”函数？

Arya

2023-02-17 21:07

问题是关于普通的C功能，不C++ static方法，如评论中所述。

我明白什么static是变量，但什么是static函数？

为什么如果我声明一个函数，让我们说void print_matrix，让我们说a.c(WITHOUT a.h) 并包含"a.c"- 我得到"print_matrix@@....) already defined in a.obj"，但是如果我声明它那么static void print_matrix它编译？

更新 只是为了澄清问题——我知道包括在内.c是不好的，正如你们中的许多人所指出的那样。我这样做只是为了暂时清理空间，main.c直到我更好地了解如何将所有这些功能分组到适当的.h和.c文件中。只是一个临时的、快速的解决方案。



解答

static函数是仅对同一文件中的其他函数（更准确地说 是同一 翻译单元）可见的函数。

编辑 ：对于那些认为问题的作者意味着“类方法”的人：由于标记了问题，C他意味着一个普通的旧 C 函数。对于 (C++/Java/...) 类方法，static意味着可以在类本身上调用此方法，不需要该类的实例。


更多建议

C中的静态函数和C++中的静态成员函数有很大区别。在 C 语言中，静态函数在其翻译单元之外是不可见的，翻译单元是它被编译成的目标文件。换句话说，使函数静态限制了它的范围。您可以认为静态函数对其 *.c 文件是“私有的”（尽管严格来说这并不正确）。

在C++中，“静态”也可以适用于类的成员函数和数据成员。静态数据成员也称为“类变量”，而非静态数据成员是“实例变量”。这是 Smalltalk 术语。这意味着一个类的所有对象只共享一个静态数据成员的副本，而每个对象都有自己的非静态数据成员副本。所以静态数据成员本质上是一个全局变量，也就是一个类的成员。

非静态成员函数可以访问类的所有数据成员：静态和非静态。静态成员函数只能对静态数据成员进行操作。

一种思考方式是，在 C++ 中，静态数据成员和静态成员函数不属于任何对象，而是属于整个类。
