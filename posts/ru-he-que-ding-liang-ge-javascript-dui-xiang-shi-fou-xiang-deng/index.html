如何确定两个 JavaScript 对象是否相等？

Arya

2023-02-17 09:14

严格相等运算符会告诉您两个对象 类型 是否相等。但是，有没有办法判断两个对象是否相等， 很像 Java 中的哈希码值？

Stack Overflow 问题 JavaScript 中是否有某种 hashCode 函数？ 类似于这个问题，但需要更学术的答案。上面的场景演示了为什么有必要有一个，我想知道是否有任何 等效的解决方案 。



解答

简短的回答

简单的答案是：不，没有通用的方法可以确定一个对象在您的意思上与另一个对象相等。例外情况是您严格认为对象是无类型的。

长答案

这个概念是 Equals 方法的概念，它比较对象的两个不同实例以指示它们在值级别上是否相等。但是，由特定类型来定义方法的Equals实现方式。具有原始值的属性的迭代比较可能是不够的：一个对象可能包含与相等性无关的属性。例如，

 function MyClass(a, b)
 {
     var c;
     this.getCLazy = function() {
         if (c === undefined) c = a * b // imagine * is really expensive
         return c;
     }
  }

在上述情况下，c确定 MyClass 的任何两个实例是否相等并不重要，重要的a只是。b在某些情况下，c实例之间可能会有所不同，但在比较过程中并不重要。

请注意，当成员本身也可能是某个类型的实例并且这些成员都需要具有确定相等性的方法时，此问题适用。

更复杂的是，在 JavaScript 中，数据和方法之间的区别是模糊的。

一个对象可以引用一个被调用为事件处理程序的方法，这可能不会被视为其“值状态”的一部分。而另一个对象很可能被分配了一个执行重要计算的函数，从而使这个实例与其他实例不同，仅仅是因为它引用了一个不同的函数。

如果一个对象的现有原型方法之一被另一个函数覆盖了怎么办？它仍然可以被认为等于另一个在其他方面相同的实例吗？只能在每种类型的每个特定情况下回答该问题。

如前所述，异常将是一个严格无类型的对象。在这种情况下，唯一明智的选择是对每个成员进行迭代和递归比较。即使那样，人们也不得不问一个函数的“价值”是什么？
