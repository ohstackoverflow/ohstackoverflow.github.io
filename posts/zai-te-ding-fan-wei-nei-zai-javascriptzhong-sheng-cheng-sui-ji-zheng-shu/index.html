在特定范围内在JavaScript中生成随机整数

Arya

2023-02-16 21:27

如何在 JavaScript 中的两个指定变量之间生成随机整数，例如x = 4并y = 8输出任何一个4, 5, 6, 7, 8？



解答

Mozilla 开发者网络页面上有一些示例：

/**
 * Returns a random number between min (inclusive) and max (exclusive)
 */
function getRandomArbitrary(min, max) {
    return Math.random() * (max - min) + min;
}

/**
 * Returns a random integer between min (inclusive) and max (inclusive).
 * The value is no lower than min (or the next integer greater than min
 * if min isn't an integer) and no greater than max (or the next integer
 * lower than max if max isn't an integer).
 * Using Math.round() will give you a non-uniform distribution!
 */
function getRandomInt(min, max) {
    min = Math.ceil(min);
    max = Math.floor(max);
    return Math.floor(Math.random() * (max - min + 1)) + min;
}

这是它背后的逻辑。这是三个简单的规则：

Math.random()返回Number0（含）和 1（不含）之间的 a。所以我们有这样的间隔：

[0 .................................... 1)

现在，我们想要一个介于min（含）和max（不含）之间的数字：

[0 .................................... 1)
[min .................................. max)

我们可以使用 来Math.random获取 [min, max) 区间内的对应者。min但是，首先我们应该通过从第二个间隔中减去一点来分解问题：

[0 .................................... 1)
[min - min ............................ max - min)

这给出：

[0 .................................... 1)
[0 .................................... max - min)

我们现在可以申请Math.random，然后计算相应的。让我们选择一个随机数：

                Math.random()
                    |
[0 .................................... 1)
[0 .................................... max - min)
                    |
                    x (what we need)

所以，为了找到x，我们会这样做：

x = Math.random() * (max - min);

别忘了加min回去，这样我们就得到了[min, max)区间内的一个数：

x = Math.random() * (max - min) + min;

这是 MDN 的第一个功能。第二个，返回一个介于min和 之间的整数max，包括两者。

现在要获取整数，您可以使用round,ceil或floor。

您可以使用Math.round(Math.random() * (max - min)) + min，但是这给出了非均匀分布。两者都有，min并且max只有大约一半的机会滚动：

min...min+0.5...min+1...min+1.5   ...    max-0.5....max
└───┬───┘└────────┬───────┘└───── ... ─────┘└───┬──┘   ← Math.round()
   min          min+1                          max

排除在间隔之外max，它滚动的机会甚至比min。

你有Math.floor(Math.random() * (max - min +1)) + min一个完全均匀的分布。

 min...  min+1...    ...      max-1... max....   (max+1 is excluded from interval)
└───┬───┘└───┬───┘└─── ... ┘└───┬───┘└───┬───┘   ← Math.floor()
   min     min+1               max-1    max

你不能在那个等式中使用ceil()and-1因为max现在滚动的机会稍微少一些，但你min-1也可以滚动（不需要的）结果。


更多建议

var randomnumber = Math.floor(Math.random() * (maximum - minimum + 1)) + minimum;
