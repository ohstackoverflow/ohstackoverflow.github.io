解答

我刚遇到同样的问题，只是弄清楚是什么原因。

Github 似乎只支持 ssh 方式来读写 repo，尽管 https 方式也显示为“Read&Write”。

因此，您需要将 PC 上的 repo 配置更改为 ssh 方式：

  1. 编辑.git/config您的 repo 目录下的文件。
  2. 在部分下找到url=条目[remote "origin"]。

  3. Change it from:
    url=https://MichaelDrogalis@github.com/derekerdmann/lunch_call.git
    to: 即把symbol
    url=ssh://git@github.com/derekerdmann/lunch_call.git
    之前的所有文字都改成@``ssh://git

  4. 保存config文件并退出。现在您可以使用它git push origin master在 GitHub 上同步您的存储库。


更多建议

为了绝对能够使用https协议登录，您应该首先将您的 身份验证凭据 设置为 git 远程 URI ：

git remote set-url origin https://yourusername@github.com/user/repo.git

然后你会在尝试时被要求输入密码git push。

其实这是在http认证格式上。您也可以设置密码：

https://youruser:password@github.com/user/repo.git

你应该知道，如果你这样做，你的 github 密码将以明文形式存储在你的 .git 目录中，这显然是不可取的。
