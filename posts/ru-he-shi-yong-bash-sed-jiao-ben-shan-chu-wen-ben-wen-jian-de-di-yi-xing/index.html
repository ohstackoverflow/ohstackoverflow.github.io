如何使用 bash/sed 脚本删除文本文件的第一行？

Arya

2023-02-17 16:01

我需要使用 bash 脚本从一个巨大的文本文件中重复删除第一行。

现在我正在使用sed -i -e "1d" $FILE- 但删除大约需要一分钟。

有没有更有效的方法来完成这个？



解答

试试尾巴：

tail -n +2 "$FILE"

-n x: 只打印最后x一行。tail -n 5会给你输入的最后 5 行。符号+有点颠倒论点并打印tail除第一x-1行以外的任何内容。tail -n +1将打印整个文件，tail -n +2除第一行以外的所有内容，等等。

GNUtail比sed. tail在 BSD 上也可用，并且-n +2标志在两个工具中是一致的。查看FreeBSD或OS X手册页了解更多信息。

不过，BSD 版本可能比 慢得多sed。我想知道他们是如何做到的；tail应该只是逐行读取文件，同时sed进行相当复杂的操作，包括解释脚本、应用正则表达式等。

注意：您可能想使用

# THIS WILL GIVE YOU AN EMPTY FILE!
tail -n +2 "$FILE" > "$FILE"

但这会给你一个 空文件 。原因是重定向 ( >) 发生在tailshell 调用之前：

  1. Shell 截断文件$FILE
  2. Shell 创建一个新进程tail
  3. Shell 将tail进程的标准输出重定向到$FILE
  4. tail从现在空的读取$FILE

如果你想删除文件中的第一行，你应该使用：

tail -n +2 "$FILE" > "$FILE.tmp" && mv "$FILE.tmp" "$FILE"

这&&将确保文件在出现问题时不会被覆盖。


更多建议

您可以使用 -i 来更新文件，而无需使用“>”运算符。以下命令将从文件中删除第一行并将其保存到文件中（在幕后使用临时文件）。

sed -i '1d' filename
