IntelliJ：永远不要使用通配符导入

Arya

2023-02-17 14:18

有没有办法告诉 IntelliJ 永远不要使用通配符导入？在“设置 > 代码样式 > 导入”下，我可以看到您可以在使用通配符导入的 IntelliJ 之前指定“类计数”。但是，如果我不想使用通配符导入，是否可以关闭此功能？

我试过输入 -1 或将该字段留空，但这只是告诉 IntelliJ 始终使用通配符导入。显然，一个不太好的解决方案是输入一个高得离谱的数字，这样你就永远不会遇到通配符导入，但我希望有一个更好的方法来关闭它。



解答

很明显为什么要禁用它：强制 IntelliJ 单独包含每个导入。它使人们更容易弄清楚您正在使用的类的确切来源。

点击工具栏上的Settings“扳手”图标，打开“Code Style”下的“Imports”，勾选“Use single class import”选项。您还可以完全删除“Packages to use import with *”下的条目，或者指定一个阈值，仅*当包中的各个类超过该阈值时才使用“”。

更新： 在 IDEA 13 中“使用单类导入”不会阻止通配符导入。解决方案是转到Preferences（在 macOS 上为⌘+ /在 Windows 和 Linux 上为+ + ）设置并设置为更高的值。任何超过 99 的值似乎都可以正常工作。,``Ctrl``Alt``S``> Editor > Code Style > Java > Imports tab``Class count to use import with '*'``Names count to use static import with '*'


更多建议

  1. 文件\设置... ( Ctrl+ Alt+ S)
  2. 项目设置 > 编辑器 > 代码风格 > Java > 导入选项卡
  3. 将类数 设置为使用带有“*”的导入到 999
  4. 将名称计数 设置为使用带“*”的静态导入到 999
