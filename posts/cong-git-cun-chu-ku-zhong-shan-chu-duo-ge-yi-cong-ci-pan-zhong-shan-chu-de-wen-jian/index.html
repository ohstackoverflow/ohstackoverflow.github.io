从 Git 存储库中删除多个已从磁盘中删除的文件

Arya

2023-02-17 02:50

我有一个 Git 存储库，我已经从使用中删除了四个文件rm（ 不是 git rm），我的 Git 状态如下所示：

#    deleted:    file1.txt
#    deleted:    file2.txt
#    deleted:    file3.txt
#    deleted:    file4.txt

如何从 Git 中删除这些文件，而不必像这样手动检查和添加每个文件：

git rm file1 file2 file3 file4

git add .理想情况下，如果可能的话，我正在寻找以相同方式工作的东西。



解答

对于 Git 1.x

$ git add -u

这告诉 git 自动暂存跟踪文件——包括删除以前跟踪的文件。

对于 Git 2.0

暂存整个工作树：

$ git add -u :/

仅暂存当前路径：

$ git add -u .


更多建议

git ls-files --deleted -z | xargs -0 git rm

可能是你要找的..它对我有用..
