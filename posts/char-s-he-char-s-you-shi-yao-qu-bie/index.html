char s[] 和 char *s 有什么区别？

Arya

2023-02-18 03:11

在 C 中，可以在这样的声明中使用字符串文字：

char s[] = "hello";

或者像这样：

char *s = "hello";

那么区别是什么呢？我想知道在编译和运行时的存储持续时间方面实际发生了什么。



解答

这里的区别在于

char *s = "Hello world";

将放置"Hello world"在 内存的只读部分 ，并指向s该内存的指针使得对该内存的任何写操作都是非法的。

在做的时候：

char s[] = "Hello world";

将文字字符串放入只读内存并将字符串复制到堆栈上新分配的内存中。从而使

s[0] = 'J';

合法的。


更多建议

首先，在函数参数中，它们是完全等价的：

void foo(char *x);
void foo(char x[]); // exactly the same in all respects

在其他情况下，char *分配一个指针，同时char []分配一个数组。你问，在前一种情况下，字符串放在哪里？编译器秘密分配一个静态匿名数组来保存字符串文字。所以：

char *x = "Foo";
// is approximately equivalent to:
static const char __secret_anonymous_array[] = "Foo";
char *x = (char *) __secret_anonymous_array;

请注意，您绝不能尝试通过此指针修改此匿名数组的内容；效果未定义（通常意味着崩溃）：

x[1] = 'O'; // BAD. DON'T DO THIS.

使用数组语法直接将其分配到新内存中。因此修改是安全的：

char x[] = "Foo";
x[1] = 'O'; // No problem.

但是，该数组仅在其包含范围内存在，因此如果您在函数中执行此操作，请不要返回或泄漏指向该数组的指针 - 而是使用或类似的方式进行复制strdup()。如果数组是在全局范围内分配的，当然没问题。
