什么是“关闭”？

Arya

2023-02-18 07:28

我问了一个关于 Currying 的问题，并提到了闭包。什么是闭包？它与柯里化有什么关系？



解答

可变范围

当您声明一个局部变量时，该变量就有一个作用域。通常，局部变量只存在于声明它们的块或函数中。

function() {
  var a = 1;
  console.log(a); // works
}    
console.log(a); // fails

如果我尝试访问局部变量，大多数语言会在当前作用域中查找它，然后通过父作用域向上查找，直到到达根作用域。

var a = 1;
function() {
  console.log(a); // works
}    
console.log(a); // works

当一个块或函数用完后，它的局部变量就不再需要了，而且通常会被清除出内存。

这就是我们通常期望事情运作的方式。

闭包是持久的局部变量作用域

闭包是一个持久作用域，即使在代码执行移出该块之后，它也会保留局部变量。支持闭包的语言（例如 JavaScript、Swift 和 Ruby）将允许您保留对作用域（包括其父作用域）的引用，即使在声明这些变量的块已完成执行后，只要您保留引用到那个块或某个地方的功能。

范围对象及其所有局部变量都与函数相关联，并且只要该函数存在，就会存在。

这为我们提供了功能可移植性。我们可以期望在函数首次定义时在范围内的任何变量在我们稍后调用函数时仍然在范围内，即使我们在完全不同的上下文中调用函数也是如此。

例如

这是一个非常简单的 JavaScript 示例，它说明了这一点：

outer = function() {
  var a = 1;
  var inner = function() {
    console.log(a);
  }
  return inner; // this returns a function
}

var fnc = outer(); // execute outer to get inner 
fnc();

这里我在一个函数中定义了一个函数。内部函数可以访问所有外部函数的局部变量，包括a. 该变量a在内部函数的范围内。

通常当一个函数退出时，它的所有局部变量都会被吹走。但是，如果我们返回内部函数并将其分配给一个变量，fnc以便它在outer退出后仍然存在， 则定义 时范围内的所有变量inner也会持续存在。该变量a已被关闭——它在一个闭包内。

请注意，该变量a对fnc. 这是一种在函数式编程语言（如 JavaScript）中创建私有变量的方法。

正如您可能猜到的那样，当我调用fnc()它时，它会打印 的值a，即“1”。

在没有闭包的语言中，变量a会在函数outer退出时被垃圾收集并丢弃。调用 fnc 会引发错误，因为它a不再存在。

在 JavaScript 中，变量a持久存在是因为变量范围是在函数首次声明时创建的，并且只要函数继续存在就会一直存在。

a属于 的范畴outer。的范围inner有一个指向范围的父指针outer。fnc是一个指向 的变量inner。a只要坚持就fnc坚持。a在闭包内。

更多建议

我将举一个例子（在 JavaScript 中）：

function makeCounter () {
  var count = 0;
  return function () {
    count += 1;
    return count;
  }
}

var x = makeCounter();
x(); returns 1
x(); returns 2
...etc...

这个函数makeCounter所做的是它返回一个我们称为的函数x，每次调用它都会加一。由于我们没有向 提供任何参数x，它必须以某种方式记住计数。它根据所谓的词法作用域知道在哪里找到它——它必须查看定义它的位置以找到值。这个“隐藏”值就是所谓的闭包。

这是我的柯里化示例：

function add (a) {
  return function (b) {
    return a + b;
  }
}

var add3 = add(3);

add3(4); returns 7

您可以看到，当您使用add参数a（3）调用时，该值包含在我们定义为 的返回函数的闭包中add3。这样，当我们调用时add3，它知道在哪里可以找到a执行加法的值。
