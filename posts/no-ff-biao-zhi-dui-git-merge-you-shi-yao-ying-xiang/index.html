`--no-ff` 标志对 `git merge` 有什么影响？

Arya

2023-02-17 03:45

使用，我看不出和gitk log的效果有什么区别。我如何观察差异（使用 git 命令或某些工具）？git merge``git merge --no- ff



解答

如果该标志检测到您的当前是您尝试合并的提交的祖先，则该--no-ff标志会阻止git merge执行“快进” 。HEAD快进是，git 不构建合并提交，而是将分支指针移动到指向传入的提交。这通常发生在git pull没有任何本地更改的情况下。

然而，有时您希望防止这种行为发生，通常是因为您希望维护特定的分支拓扑（例如，您正在合并主题分支并且希望确保它在读取历史记录时看起来是那样的）。为了做到这一点，你可以传递--no- ff标志，并且git merge总是 构造 一个合并而不是快进。

类似地，如果你想执行 agit pull或 usegit merge以明确快进，并且你想在不能快进时退出，那么你可以使用标志--ff- only。这样你就可以经常git pull --ff-only不假思索地做一些事情，然后如果它出错了你可以回去决定是否要合并或变基。

示例工作流

我在我的网站上更新了一个包，不得不返回我的笔记以查看我的工作流程；我认为向这个答案添加一个例子很有用。

我的 git 命令工作流程：

git checkout -b contact-form
(do your work on "contact-form")
git status
git commit -am  "updated form in contact module"
git checkout master
git merge --no-ff contact-form
git branch -d contact-form
git push origin master
