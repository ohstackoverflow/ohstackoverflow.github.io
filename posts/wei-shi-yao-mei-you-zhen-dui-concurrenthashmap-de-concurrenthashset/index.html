为什么没有针对 ConcurrentHashMap 的 ConcurrentHashSet

Arya

2023-02-17 19:27

HashSet 基于 HashMap。

如果我们看一下HashSet<E>实施，一切都在HashMap<E,Object>.

<E>用作 的键HashMap。

我们知道这HashMap不是线程安全的。这就是我们ConcurrentHashMap在 Java 中的原因。

基于此，我很困惑 为什么我们没有应该基于的 ConcurrentHashSetConcurrentHashMap？

还有什么我想念的吗？我需要Set在多线程环境中使用。

另外，如果我想创建自己的，ConcurrentHashSet我可以通过替换HashMaptoConcurrentHashMap并保持其余不变来实现它吗？



解答

没有内置类型，ConcurrentHashSet因为您始终可以从地图中 派生出一个集合。 由于有多种类型的地图，您可以使用一种方法从给定的地图（或地图类）生成一组地图。

在 Java 8 之前，您可以使用并发哈希映射生成并发哈希集Collections.newSetFromMap(map)

在 Java 8 中（@Matt 指出），您可以 通过ConcurrentHashMap.newKeySet(). newSetFromMap这比要求您传入一个空地图对象的旧版本要简单一些。但它是特定于ConcurrentHashMap.

无论如何，Java 设计者可以在每次创建新的地图接口时创建一个新的集合接口，但是当第三方创建他们自己的地图时，这种模式将无法实施。最好有派生新集合的静态方法；该方法始终有效，即使您创建自己的地图实现也是如此。


更多建议

Set<String> mySet = Collections.newSetFromMap(new ConcurrentHashMap<String, Boolean>());
