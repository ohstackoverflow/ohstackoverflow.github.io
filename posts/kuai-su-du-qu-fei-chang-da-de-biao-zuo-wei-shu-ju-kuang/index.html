快速读取非常大的表作为数据框

Arya

2023-02-18 04:18

我有非常大的表（3000 万行），我想将其作为数据帧加载到 R 中。它read.table()有很多方便的功能，但似乎在实现中有很多逻辑会减慢速度。在我的例子中，我假设我提前知道列的类型，该表不包含任何列标题或行名，并且没有我必须担心的任何病态字符。

我知道在表中读取列表使用scan()可以非常快，例如：

datalist <- scan('myfile',sep='\t',list(url='',popularity=0,mintime=0,maxtime=0)))

但是我将其转换为数据框的一些尝试似乎将上述性能降低了 6 倍：

df <- as.data.frame(scan('myfile',sep='\t',list(url='',popularity=0,mintime=0,maxtime=0))))

有更好的方法吗？或者很可能完全不同的问题解决方法？



解答

几年后的更新

这个答案是旧的，R 已经移动了。调整read.table运行速度快一点几乎没有什么好处。您的选择是：

  1. 使用vroomtidyverse 包vroom将数据从 csv/tab 分隔文件直接导入 R tibble。请参阅赫克托的回答。

  2. 使用freadindata.table将 csv/tab 分隔文件中的数据直接导入 R。请参阅mnel 的回答。

  3. read_table在（2015 年 4 月起在 CRAN 上）使用readr。这很像fread上面的工作。 链接中的自述文件 解释了这两个函数之间的区别（readr目前声称比“慢 1.5-2 倍” data.table::fread）。

  4. read.csv.rawfromiotools为快速读取 CSV 文件提供了第三种选择。

  5. 尝试在数据库而不是平面文件中存储尽可能多的数据。（作为更好的永久存储介质，数据以二进制格式传入和传出 R，速度更快。）在包read.csv.sql中sqldf，如JD Long 的回答中所述，将数据导入临时 SQLite 数据库，然后读取它进入 R。另请参阅：包，以及包RODBC页面的反向依赖部分。为您提供一种数据类型，它伪装成数据框，但实际上是底层的 MonetDB，从而提高了性能。使用其功能导入数据。允许您直接使用存储在多种类型数据库中的数据。DBI``MonetDB.R``monetdb.read.csv``dplyr

  6. 以二进制格式存储数据也有助于提高性能。使用saveRDS/ readRDS（见下文）、HDF5 格式的h5或包，或包中的/ 。rhdf5``write_fst``read_fst``fst

原来的答案

无论您使用 read.table 还是 scan，都可以尝试一些简单的操作。

  1. Set nrows= 数据中的记录数 ( nmaxin scan)。

  2. 确保comment.char=""关闭评论的解释。

  3. colClasses使用in显式定义每列的类read.table。

  4. 设置multi.line=FALSE还可以提高扫描性能。

如果这些都不起作用，则使用其中一个分析包来确定哪些线路正在减慢速度。read.table也许你可以根据结果写一个简化版本。

另一种方法是在将数据读入 R 之前对其进行过滤。

或者，如果问题是您必须定期读取它，则使用这些方法一次读取数据，然后将数据帧保存为二进制 blob save saveRDS，然后下次您可以更快地检索它 load readRDS.


更多建议

这是一个使用1.8.7fread的示例data.table

这些示例来自帮助页面fread，以及我的 Windows XP Core 2 duo E8400 上的计时。

library(data.table)
# Demo speedup
n=1e6
DT = data.table( a=sample(1:1000,n,replace=TRUE),
                 b=sample(1:1000,n,replace=TRUE),
                 c=rnorm(n),
                 d=sample(c("foo","bar","baz","qux","quux"),n,replace=TRUE),
                 e=rnorm(n),
                 f=sample(1:1000,n,replace=TRUE) )
DT[2,b:=NA_integer_]
DT[4,c:=NA_real_]
DT[3,d:=NA_character_]
DT[5,d:=""]
DT[2,e:=+Inf]
DT[3,e:=-Inf]

标准读表

write.table(DT,"test.csv",sep=",",row.names=FALSE,quote=FALSE)
cat("File size (MB):",round(file.info("test.csv")$size/1024^2),"\n")    
## File size (MB): 51

system.time(DF1 <- read.csv("test.csv",stringsAsFactors=FALSE))

##   24.71    0.15   25.42
# second run will be faster
system.time(DF1 <- read.csv("test.csv",stringsAsFactors=FALSE))

优化的读表

system.time(DF2 <- read.table("test.csv",header=TRUE,sep=",",quote="",  
                          stringsAsFactors=FALSE,comment.char="",nrows=n,                   
                          colClasses=c("integer","integer","numeric",                        
                                       "character","numeric","integer")))


##    user  system elapsed 
##   10.20    0.03   10.32
