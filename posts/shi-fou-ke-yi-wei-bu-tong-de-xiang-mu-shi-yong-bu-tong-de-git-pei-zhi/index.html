是否可以为不同的项目使用不同的 Git 配置？

Arya

2023-02-18 01:58

.gitconfig通常存放在user.home目录中。

我使用不同的身份来处理公司 A 的项目和公司 B 的其他内容（主要是姓名/电子邮件）。我怎样才能有两个不同的 Git 配置，以便我的签到不与姓名/电子邮件一起使用？



解答

.git/config存储库的特定克隆中的文件是该克隆的本地文件。放置在那里的任何设置只会影响该特定项目的操作。

（默认情况下，git config修改.git/config，而不是~/.gitconfig- 只有 with--global修改后者。）


更多建议

从 git 版本 2.13 开始，git 支持条件配置包括. 在本例中，我们将公司 A 的存储库克隆到~/company_a目录中，将公司 B 的存储库克隆到~/company_b.

在你的文件的末尾.gitconfig，你可以放这样的东西：

[includeIf "gitdir:~/company_a/"]
  path = .gitconfig-company_a
[includeIf "gitdir:~/company_b/"]
  path = .gitconfig-company_b

示例内容.gitconfig-company_a（[core]如果可以使用全局ssh密钥，则该部分可以省略）：

[user]
name = John Smith
email = john.smith@companya.net

[core]
sshCommand = ssh -i ~/.ssh/id_rsa_companya

示例内容.gitconfig-company_b：

[user]
name = John Smith
email = js@companyb.com

[core]
sshCommand = ssh -i ~/.ssh/id_rsa_companyb
