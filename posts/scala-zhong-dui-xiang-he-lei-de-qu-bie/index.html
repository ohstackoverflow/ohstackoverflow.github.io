Scala 中对象和类的区别

Arya

2023-02-17 16:03

我刚刚浏览了 Internet 上的一些 Scala 教程，并注意到在某些示例中，在示例的开头声明了一个对象。

classScala和Scala之间有什么区别object？



解答

tl；博士

  • class C定义一个类，就像在 Java 或 C++ 中一样。
  • object O创建一个单例对象O作为某个匿名类的实例；它可用于保存与某个类的实例无关的静态成员。
  • object O extends T使对象成为;O的实例 trait T然后你可以通过O任何地方，这T是意料之中的。
  • 如果有class C，则为object C类的伴生对象C；请注意，伴生对象 不会 自动成为C.

另请参阅object和class的 Scala 文档。

object作为静态成员的主机

大多数情况下，您需要一个object方法和值/变量来保存这些方法和值/变量，而无需首先实例化某个类的实例。这种用法与 Java 中的成员密切相关static。

object A {
  def twice(i: Int): Int = 2*i
}

然后，您可以使用调用上述方法A.twice(2)。

如果twice是某个类的成员A，那么您需要先创建一个实例：

class A() {
  def twice(i: Int): Int = 2 * i
}

val a = new A()
a.twice(2)

您可以看到这是多么多余，因为twice不需要任何特定于实例的数据。

object作为一个特殊的命名实例

您还可以将object自身用作类或特征的一些特殊实例。当您这样做时，您的对象需要扩展一些trait以便成为它的子类的实例。

考虑以下代码：

object A extends B with C {
  ...
}

B此声明首先声明一个扩展了和的匿名（不可访问）类C，并实例化此类名为 的单个实例A。

这意味着A可以传递给期望类型为Bor C， or的对象的函数B with C。

的附加功能object

Scala 中还存在一些对象的特殊特性。我建议阅读官方文档。

  • def apply(...)启用通常的方法无名称语法A(...)
  • def unapply(...)允许创建自定义模式匹配提取器
  • 如果伴随同名类，则该对象在解析隐式参数时扮演特殊角色


更多建议

Aclass是一个定义，一个描述。它根据其他类型的方法和组合来定义一个类型。

Anobject是单例——保证唯一的类的实例。对于object代码中的每一个，都会创建一个匿名类，它继承自您声明object要实现的任何类。从 Scala 源代码中看不到这个类——尽管您可以通过反射获得它。

object和之间存在关系class。如果一个对象共享相同的名称，则称该对象是类的伴生对象。当这种情况发生时，每个人都可以访问private对方的可见性方法。但是，这些方法不会自动导入。您要么必须显式导入它们，要么在它们前面加上类/对象名称。

例如：

class X {
  // class X can see private members of object X
  // Prefix to call
  def m(x: Int) = X.f(x)

  // Import and use
  import X._
  def n(x: Int) = f(x)

  private def o = 2
}

object X {
  private def f(x: Int) = x * x

  // object X can see private members of class X
  def g(x: X) = {
    import x._
    x.o * o // fully specified and imported
   }
}
