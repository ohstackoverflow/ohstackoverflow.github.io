JavaScript 中是否有“空合并”运算符？

Arya

2023-02-17 01:40

Javascript 中是否有空合并运算符？

例如，在 C# 中，我可以这样做：

String someString = null;
var whatIWant = someString ?? "Cookies!";

我能为 Javascript 找到的最佳近似是使用条件运算符：

var someString = null;
var whatIWant = someString ? someString : 'Cookies!';

恕我直言，这有点恶心。我可以做得更好吗？



解答

更新

JavaScript 现在支持nullish 合并运算符 (??)。当其左侧操作数为nullor时，它返回其右侧undefined操作数，否则返回其左侧操作数。

旧答案

使用前请检查兼容性。

C# 空合并运算符 ( ??) 的 JavaScript 等价物使用逻辑 OR ( ||)：

var whatIWant = someString || "Cookies!";

在某些情况下（下文会阐明）行为与 C# 的行为不匹配，但这是在 JavaScript 中分配默认值/替代值的一般、简洁的方式。

澄清

无论第一个操作数的类型如何，如果将其转换为布尔值导致false，赋值将使用第二个操作数。当心以下所有情况：

alert(Boolean(null)); // false
alert(Boolean(undefined)); // false
alert(Boolean(0)); // false
alert(Boolean("")); // false
alert(Boolean("false")); // true -- gotcha! :)

这意味着：

var whatIWant = null || new ShinyObject(); // is a new shiny object
var whatIWant = undefined || "well defined"; // is "well defined"
var whatIWant = 0 || 42; // is 42
var whatIWant = "" || "a million bucks"; // is "a million bucks"
var whatIWant = "false" || "no way"; // is "false"


更多建议

function coalesce() {
    var len = arguments.length;
    for (var i=0; i<len; i++) {
        if (arguments[i] !== null && arguments[i] !== undefined) {
            return arguments[i];
        }
    }
    return null;
}

var xyz = {};
xyz.val = coalesce(null, undefined, xyz.val, 5);

// xyz.val now contains 5

此解决方案的工作方式类似于 SQL 合并函数，它接受任意数量的参数，如果没有参数则返回 null。它的行为类似于 C# ?? “”、false 和 0 被视为 NOT NULL 并因此算作实际值的意义上的运算符。如果你来自.net 背景，这将是最自然的感觉解决方案。
