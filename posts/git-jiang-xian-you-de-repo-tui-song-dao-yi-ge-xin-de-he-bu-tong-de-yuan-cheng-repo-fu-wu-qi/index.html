Git 将现有的 repo 推送到一个新的和不同的远程 repo 服务器？

Arya

2023-02-17 19:07

假设我在git.fedorahosted.org上有一个存储库，我想将其克隆到我在 github 的帐户中，以便在 fedorahosted 上拥有更“官方”的存储库之外拥有自己的游乐场。最初复制它的步骤是什么？在 github 中有一个很好的“fork”按钮，但由于显而易见的原因我不能使用它。

我将如何跟踪 fedorahosted 存储库中的更改到 github 存储库中？



解答

  1. 在 github 上创建一个新的 repo。
  2. 从 fedorahosted 克隆 repo 到你的本地机器。
  3. git remote rename origin upstream
  4. git remote add origin URL_TO_GITHUB_REPO
  5. git push origin master

现在您可以像使用任何其他 github 存储库一样使用它。要从上游拉入补丁，只需运行git pull upstream master && git push origin master.


更多建议

这个问题有一个已删除的答案，它有一个有用的链接：https ://help.github.com/articles/duplicating-a- repository

要点是

0. create the new empty repository (say, on github)
1. make a bare clone of the repository in some temporary location
2. change to the temporary location
3. perform a mirror-push to the new repository
4. change to another location and delete the temporary location

OP的例子：

在你的本地机器上

$ cd $HOME
$ git clone --bare https://git.fedorahosted.org/the/path/to/my_repo.git
$ cd my_repo.git
$ git push --mirror https://github.com/my_username/my_repo.git
$ cd ..
$ rm -rf my_repo.git
