如何在 Git 存储库中查找和恢复已删除的文件？

Arya

2023-02-16 20:06

假设我在 Git 存储库中。我删除一个文件并提交该更改。我继续工作并做出更多承诺。然后，我发现我需要在删除该文件后恢复该文件。

我知道我可以使用 签出文件git checkout <commit> -- filename.txt，但我不知道该文件何时被删除。

  1. 如何找到删除给定文件名的提交？
  2. 我如何将该文件恢复到我的工作副本中？



解答

查找影响给定路径的最后一次提交。由于该文件不在 HEAD 提交中，因此之前的提交必须已将其删除。

git rev-list -n 1 HEAD -- <file_path>

然后使用脱字符号 ( ) 签出之前提交时的版本^：

git checkout <deleting_commit>^ -- <file_path>

或者在一个命令中，if$file是有问题的文件。

git checkout $(git rev-list -n 1 HEAD -- "$file")^ -- "$file"

如果您使用的是 zsh 并启用了 EXTENDED_GLOB 选项，插入符号将不起作用。您可以改用~1。

git checkout $(git rev-list -n 1 HEAD -- "$file")~1 -- "$file"


更多建议

  1. 获取所有已删除文件的提交，以及已删除的文件：
    git log --diff-filter=D --summary

记下所需的提交哈希，例如e4e6d4d5e5c59c69f3bd7be2.

  1. 将删除的文件从之前的一个提交 ( ~1) 恢复到上面确定的提交 ( e4e6d4d5e5c59c69f3bd7be2)：
    git checkout e4e6d4d5e5c59c69f3bd7be2~1 path/to/file.ext

注意~1.
