将 Flask 开发服务器配置为在网络上可见

Arya

2023-02-17 20:01

我不确定这是否是特定于 Flask 的，但是当我以开发模式 ( http://localhost:5000) 运行应用程序时，我无法从网络上的其他机器访问它 ( 使用http://[dev-host-ip]:5000)。例如，在开发模式下使用 Rails，它工作正常。我找不到任何关于 Flask 开发服务器配置的文档。知道应该配置什么来启用它吗？



解答

虽然这是可能的，但您不应在生产环境中使用 Flask 开发服务器。Flask 开发服务器并不是特别安全、稳定或高效的。请参阅有关部署的文档以获取正确的解决方案。

--host的选项或flask run参数host控制app.run()开发服务器监听的地址。默认情况下它运行于localhost，将其更改为flask run --host=0.0.0.0（或app.run(host="0.0.0.0")）以在您计算机的所有 IP 地址上运行。

0.0.0.0是一个你不能直接在浏览器中使用的特殊值，你需要导航到网络上机器的实际 IP 地址。您可能还需要调整防火墙以允许外部访问该端口。

Flask快速入门文档在“外部可见服务器”部分对此进行了解释：

如果您运行该服务器，您会注意到该服务器只能从您自己的计算机访问，而不能从网络中的任何其他计算机访问。这是默认设置，因为在调试模式下，应用程序的用户可以在您的计算机上执行任意 Python 代码。

如果您禁用了调试器或信任您网络上的用户，则只需在--host=0.0.0.0命令行中添加以下命令即可使服务器公开可用：

$ flask run --host=0.0.0.0

这告诉您的操作系统监听所有公共 IP。


更多建议

如果您使用flask可执行文件来启动您的服务器，请使用flask run --host=0.0.0.0更改默认值 127.0.0.1 并将其打开到非本地连接。
