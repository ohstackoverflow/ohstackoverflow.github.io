如何在“纯”Swift 中创建弱协议引用（没有@objc）

Arya

2023-02-18 01:07

weak引用似乎在 Swift 中不起作用，除非 aprotocol被声明为@objc，我不希望在纯 Swift 应用程序中这样做。

此代码给出编译错误（weak不能应用于非类类型MyClassDelegate）：

class MyClass {
  weak var delegate: MyClassDelegate?
}

protocol MyClassDelegate {
}

我需要在协议前加上前缀@objc，然后它才能工作。

问题：完成 a 的“纯”Swift 方法是什么weak delegate？



解答

您需要将协议的类型声明为AnyObject.

protocol ProtocolNameDelegate: AnyObject {
    // Protocol stuff goes here
}

class SomeClass {
    weak var delegate: ProtocolNameDelegate?
}

用AnyObject你说只有类可以符合这个协议，而结构或枚举不能。

补充回答

我一直对代表是否应该是弱者感到困惑。最近我了解了更多有关委托以及何时使用弱引用的知识，所以让我在这里添加一些补充点以供将来的观众使用。

  • 使用关键字的目的weak是为了避免强引用循环（retain cycles）。当两个类实例彼此具有强引用时，就会发生强引用循环。它们的引用计数永远不会归零，因此它们永远不会被释放。

  • weak只有当委托是一个类时才需要使用。Swift 结构和枚举是值类型（它们的值在创建新实例时被复制），而不是引用类型，因此它们不会产生强 引用 循环。

  • weak引用始终是可选的（否则您会使用unowned）并且始终使用var（不是let）以便可以nil在释放时将可选设置为。

  • 父类自然应该对其子类有强引用，因此不使用关键字weak。但是，当子项想要引用其父项时，应该使用关键字将其设为弱引用weak。

  • weak当您想要引用不属于您的类时应该使用，而不仅仅是引用其父类的子类。当两个非层次类需要相互引用时，选择一个弱的。您选择哪个取决于情况。有关更多信息，请参阅此问题的答案。

  • 作为一般规则，委托应标记为，weak因为大多数委托都在引用它们不拥有的类。当孩子使用委托与父母通信时，这绝对是正确的。文档建议对委托使用弱引用。（但也请看这个。）

  • 协议可用于引用类型（类）和值类型（结构、枚举）。因此，在可能需要使委托变弱的情况下，您必须使其成为仅对象协议。这样做的方法是添加AnyObject到协议的继承列表中。（过去您使用class关键字来执行此操作，但AnyObject现在首选.）

    protocol MyClassDelegate: AnyObject {
// ...

    }

    class SomeClass { weak var delegate: MyClassDelegate? }
