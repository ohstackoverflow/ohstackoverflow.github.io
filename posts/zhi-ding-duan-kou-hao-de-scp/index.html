指定端口号的scp

Arya

2023-02-17 06:23

我正在尝试将文件从远程服务器 scp 到我的本地机器。只能访问端口 80。

我试过：

scp -p 80 username@www.myserver.com:/root/file.txt .

但出现此错误：cp: 80: No such file or directory

如何在 scp 命令中指定端口号？



解答

与 ssh 不同，scp 使用大写的 P 开关来设置端口，而不是小写的 p：

scp -P 80 ... # Use port 80 to bypass the firewall, instead of the scp default

小写的 p 开关与 scp 一起使用，用于保存时间和模式。

以下是 scp 手册页的摘录，其中包含有关这两个开关的所有详细信息，以及为什么为 scp 选择大写 P 的解释：

-P port 指定要连接到远程主机上的端口。请注意，此选项以大写字母“P”书写，因为 -p 已保留用于保留 rcp(1) 中文件的时间和模式。

-p 保留原始文件的修改时间、访问时间和模式。

额外提示 ：如何确定 SSH 守护程序正在使用的端口来接受 SSH 连接？

这个问题可以使用实用程序来回答netstat，如下所示：

sudo netstat -tnlp | grep sshd

或者，使用更易读的基于单词的 netstat 选项名称：

sudo netstat --tcp --numeric-ports --listening --program | grep sshd

假设您的 ssh 守护程序配置了默认值，您将看到的输出如下所示（在列之间略微修剪了空白，以便无需滚动即可显示整个表）：

Active Internet connections (only servers)
Proto Recv-Q Send-Q Local Address  Foreign Address  State       ID/Program name
tcp      0      0   0.0.0.0:22     0.0.0.0:*        LISTEN      888/sshd: /usr/sbin 
tcp6     0      0   :::22          :::*             LISTEN      888/sshd: /usr/sbin

对于上面的示例，sudo用于以管理员权限运行 netstat，以便能够看到 所有 的 Program Names 。如果您以普通用户身份运行 netstat（即，没有 sudo 并假设您没有通过其他方法授予您管理员权限），您将只会看到程序名称显示给拥有您的 UID 作为所有者的套接字。 属于其他用户的套接字的程序名称 将不会显示（即，将被隐藏，而是显示占位符连字符）：

更新并放在一边以解决其中一条（强烈赞成的）评论 ：

关于阿卜杜勒关于scp期权订单的评论，他的建议是：

scp -r some_directory -P 80 ...

...，散布选项和参数，因为-rswitch 不接受额外的参数并且some_directory被视为命令的第一个参数，使得-P和所有后续命令行参数看起来像命令的附加参数（即，连字符前缀的参数不再是视为开关）。

getopt(1)明确定义参数必须在选项（即开关） 之后 并且不能与它们穿插，willy-nilly：

调用 getopt 的参数可以分为两部分：修改 getopt 进行解析的方式的选项（SYNOPSIS 中的选项和 optstring），以及要解析的参数（SYNOPSIS 中的参数）。第二部分将从第一个不是选项参数的非选项参数开始，或者在第一次出现“--”之后开始。如果在第一部分中找不到“-o”或“--options”选项，则第二部分的第一个参数将用作短选项字符串。

由于-r命令行选项没有进一步的参数，some_directory因此是“第一个不是选项参数的非选项参数”。因此，正如手册页中明确指出的那样getopt(1)，它后面的所有后续命令行参数（即-P 80 ...）都被假定为非选项（和非选项参数）。

因此，实际上，这就是getopt(1)示例的显示方式，其中选项的末尾和参数的开头由灰色文本划定：

scp -rsome_directory -P 80 ...

这与行为无关scp，而与 POSIX 标准应用程序如何使用 C 函数集解析命令行选项有关getopt(3)。

有关命令行排序和处理的更多详细信息，请getopt(1)使用以下命令阅读联机帮助页：

man 1 getopt


更多建议

一个额外的提示。在 scp 命令之后放置“-P”选项，无论您要通过 ssh 进入的机器是否是第二台（也称为目标）。例子：

scp -P 2222 /absolute_path/source-folder/some-file user@example.com:/absolute_path/destination-folder
