为什么我需要一直执行 `--set-upstream`？

Arya

2023-02-16 23:51

我在 Git 中创建了一个新分支：

git branch my_branch

推它：

git push origin my_branch

现在说有人在服务器上做了一些更改，我想从origin/my_branch. 我愿意：

git pull

但我得到：

You asked me to pull without telling me which branch you
want to merge with, and 'branch.my_branch.merge' in
your configuration file does not tell me, either. Please
specify which branch you want to use on the command line and
try again (e.g. 'git pull <repository> <refspec>').
See git-pull(1) for details.

If you often merge with the same branch, you may want to
use something like the following in your configuration file:

    [branch "my_branch"]
    remote = <nickname>
    merge = <remote-ref>

    [remote "<nickname>"]
    url = <url>
    fetch = <refspec>

See git-config(1) for details.

我了解到我可以使用它：

git branch --set-upstream my_branch origin/my_branch

但是为什么我需要为我创建的每个分支都这样做呢？my_branch如果我推入origin/my_branch，那么我想拉origin/my_branch入，这不是很明显吗my_branch？我怎样才能使它成为默认行为？



解答

Git v2.37.1 及以上版本

如果您使用的是上述版本或更高版本，您可以使用这个新的配置条目来自动设置远程跟踪：

git config --global push.autoSetupRemote true

之后，当您进行git push跟踪时，会自动设置。不需要git push -u origin my_branch

一个不依赖于记住git branch --set-upstream 1语法的快捷方式是：

git push -u origin my_branch

... 第一次推送该分支。或者，从同名分支推送到当前分支（方便使用别名）：

git push -u origin HEAD

您只需要使用一次，就可以像以前一样-u在您的分支机构和 at 分支机构之间建立关联。origin``git branch --set- upstream

就个人而言，我认为必须明确地在您的分支机构和远程分支机构之间建立关联是一件好事。遗憾的是和的规则不同git push``git pull。

2012-10-11 更新 ：显然我不是唯一发现容易出错的人！感谢VonC指出 git 1.8.0 引入了更明显的git branch --set- upstream-to，如果你在分支上，可以按如下方式使用my_branch：

git branch --set-upstream-to origin/my_branch

...或使用简短选项：

git branch -u origin/my_branch

此更改及其原因在git 1.8.0 的发行说明中进行了描述，发布候选版本 1：

很想说git branch --set-upstream origin/master，但这告诉 Git 安排本地分支origin/master与当前签出的分支集成，这不太可能是用户的意思。该选项已弃用；请改用新的--set-upstream- to（带有简短的-u）选项。


更多建议

您可以通过更少的输入来实现这一点。首先，改变推送的方式：

git config --global push.default current

这将推断出该origin my_branch部分，因此您可以这样做：

git push -u

这将创建具有相同名称的远程分支并跟踪它。
